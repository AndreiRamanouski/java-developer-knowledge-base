## Overview

OAuth2 is an authorization framework that enables third-party applications to obtain limited access to user accounts. JWT (JSON Web Tokens) are commonly used as the token format in OAuth2 implementations. Understanding both deeply is crucial for building secure, modern APIs.

**Key Concept**: OAuth2 is about **authorization** (what you can access), not authentication (who you are). JWT is a token format that can carry both authentication and authorization information.

---

## 1. OAuth2 Fundamentals

### OAuth2 Roles

```
┌─────────────────────────────────────────────────────────────┐
│                      OAuth2 Roles                            │
└─────────────────────────────────────────────────────────────┘

1. Resource Owner (User)
   - Owns the protected resources
   - Grants access to their resources
   Example: User who wants to let mobile app access their Google Drive

2. Client (Application)
   - Application requesting access to resources
   - Must be registered with Authorization Server
   Example: Mobile app, Web app, Third-party service

3. Authorization Server
   - Issues access tokens after authenticating user
   - Validates client credentials
   Example: Google OAuth, Auth0, Keycloak, Spring Authorization Server

4. Resource Server (API)
   - Hosts protected resources
   - Validates access tokens
   - Serves resources if token is valid
   Example: Google Drive API, Your REST API
```

### OAuth2 Token Types

```java
/**
 * OAuth2 Token Types
 */

// 1. Access Token - Short-lived token for API access
// Format: JWT or opaque string
// Lifetime: 15 minutes - 1 hour (typical)
// Example:
String accessToken = "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...";

// 2. Refresh Token - Long-lived token to get new access tokens
// Format: Usually opaque string
// Lifetime: Days, weeks, or months
// Example:
String refreshToken = "tGzv3JOkF0XG5Qx2TlKWIA";

// 3. ID Token (OpenID Connect) - Contains user identity information
// Format: Always JWT
// Example:
String idToken = "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...";
```

### Authorization Code Flow

**Most secure flow, recommended for web and mobile apps**

```
┌─────────────────────────────────────────────────────────────┐
│           Authorization Code Flow (Step-by-Step)             │
└─────────────────────────────────────────────────────────────┘

1. User clicks "Login with Google" in Client App
         ↓
2. Client redirects to Authorization Server:
   GET /authorize?
     response_type=code
     &client_id=CLIENT_ID
     &redirect_uri=https://client.app/callback
     &scope=read:profile
     &state=random_string
         ↓
3. Authorization Server shows login page
         ↓
4. User enters credentials and consents
         ↓
5. Authorization Server redirects to Client with code:
   GET https://client.app/callback?
     code=AUTH_CODE
     &state=random_string
         ↓
6. Client exchanges code for tokens:
   POST /token
     grant_type=authorization_code
     &code=AUTH_CODE
     &redirect_uri=https://client.app/callback
     &client_id=CLIENT_ID
     &client_secret=CLIENT_SECRET
         ↓
7. Authorization Server returns tokens:
   {
     "access_token": "eyJhbGci...",
     "token_type": "Bearer",
     "expires_in": 3600,
     "refresh_token": "tGzv3JOk...",
     "scope": "read:profile"
   }
         ↓
8. Client uses access token to call Resource Server:
   GET /api/profile
   Authorization: Bearer eyJhbGci...
         ↓
9. Resource Server validates token and returns data
```

```java
/**
 * Authorization Code Flow Implementation
 */

// Step 1: Redirect to authorization endpoint
@GetMapping("/oauth2/authorization/google")
public void redirectToAuthServer(HttpServletResponse response) throws IOException {
    String authUrl = "https://accounts.google.com/o/oauth2/v2/auth" +
        "?response_type=code" +
        "&client_id=" + clientId +
        "&redirect_uri=" + URLEncoder.encode("https://myapp.com/callback", "UTF-8") +
        "&scope=openid%20profile%20email" +
        "&state=" + generateRandomState();
    
    response.sendRedirect(authUrl);
}

// Step 2: Handle callback with authorization code
@GetMapping("/callback")
public ResponseEntity<?> handleCallback(
        @RequestParam("code") String code,
        @RequestParam("state") String state) {
    
    // Validate state parameter (CSRF protection)
    if (!validateState(state)) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body("Invalid state");
    }
    
    // Exchange code for tokens
    TokenResponse tokens = exchangeCodeForTokens(code);
    
    // Store tokens securely
    storeTokens(tokens);
    
    return ResponseEntity.ok(tokens);
}

// Step 3: Exchange authorization code for tokens
private TokenResponse exchangeCodeForTokens(String code) {
    RestTemplate restTemplate = new RestTemplate();
    
    MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
    params.add("grant_type", "authorization_code");
    params.add("code", code);
    params.add("redirect_uri", "https://myapp.com/callback");
    params.add("client_id", clientId);
    params.add("client_secret", clientSecret);
    
    HttpHeaders headers = new HttpHeaders();
    headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
    
    HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(params, headers);
    
    ResponseEntity<TokenResponse> response = restTemplate.postForEntity(
        "https://oauth2.googleapis.com/token",
        request,
        TokenResponse.class
    );
    
    return response.getBody();
}

@Data
public class TokenResponse {
    private String access_token;
    private String token_type;
    private Integer expires_in;
    private String refresh_token;
    private String scope;
    private String id_token;  // OpenID Connect
}
```

**Why Authorization Code Flow?**

- ✅ Access token never exposed to browser
- ✅ Client secret protected on backend
- ✅ Most secure for server-side apps
- ✅ Supports refresh tokens

### Client Credentials Flow

**For machine-to-machine (M2M) communication, no user involved**

```
┌─────────────────────────────────────────────────────────────┐
│              Client Credentials Flow                         │
└─────────────────────────────────────────────────────────────┘

1. Client sends credentials to Authorization Server:
   POST /token
     grant_type=client_credentials
     &client_id=CLIENT_ID
     &client_secret=CLIENT_SECRET
     &scope=read:data
         ↓
2. Authorization Server validates credentials and returns token:
   {
     "access_token": "eyJhbGci...",
     "token_type": "Bearer",
     "expires_in": 3600,
     "scope": "read:data"
   }
         ↓
3. Client uses access token to call Resource Server:
   GET /api/data
   Authorization: Bearer eyJhbGci...
```

```java
/**
 * Client Credentials Flow Implementation
 */
@Service
public class ClientCredentialsService {
    
    @Value("${oauth2.client.id}")
    private String clientId;
    
    @Value("${oauth2.client.secret}")
    private String clientSecret;
    
    @Value("${oauth2.token.url}")
    private String tokenUrl;
    
    private final RestTemplate restTemplate = new RestTemplate();
    
    // ✅ Get access token using client credentials
    public String getAccessToken() {
        MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
        params.add("grant_type", "client_credentials");
        params.add("client_id", clientId);
        params.add("client_secret", clientSecret);
        params.add("scope", "read:data write:data");
        
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
        
        HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(params, headers);
        
        ResponseEntity<TokenResponse> response = restTemplate.postForEntity(
            tokenUrl,
            request,
            TokenResponse.class
        );
        
        return response.getBody().getAccess_token();
    }
    
    // ✅ Call API with access token
    public <T> T callApi(String url, Class<T> responseType) {
        String accessToken = getAccessToken();
        
        HttpHeaders headers = new HttpHeaders();
        headers.setBearerAuth(accessToken);
        
        HttpEntity<Void> request = new HttpEntity<>(headers);
        
        ResponseEntity<T> response = restTemplate.exchange(
            url,
            HttpMethod.GET,
            request,
            responseType
        );
        
        return response.getBody();
    }
}

/**
 * Example: Microservice A calling Microservice B
 */
@Service
public class OrderService {
    
    @Autowired
    private ClientCredentialsService oauth2Client;
    
    public Inventory checkInventory(Long productId) {
        // Get token and call inventory service
        return oauth2Client.callApi(
            "https://inventory-service/api/products/" + productId,
            Inventory.class
        );
    }
}
```

**Use Cases**:

- Microservice-to-microservice communication
- Backend services accessing APIs
- Scheduled jobs calling APIs
- CLI tools

**Characteristics**:

- ❌ No user context (no refresh token)
- ✅ Simple and fast
- ✅ Client authenticates itself
- ⚠️ Client secret must be protected

### PKCE (Proof Key for Code Exchange)

**Extension to Authorization Code Flow for public clients (mobile apps, SPAs)**

```
┌─────────────────────────────────────────────────────────────┐
│     Authorization Code Flow with PKCE (Recommended)         │
└─────────────────────────────────────────────────────────────┘

1. Client generates random code_verifier:
   code_verifier = "dBjftJeZ4CVP-mB92K27uhbUJU1p1r_wW1gFWFOEjXk"
         ↓
2. Client creates code_challenge from code_verifier:
   code_challenge = BASE64URL(SHA256(code_verifier))
                  = "E9Melhoa2OwvFrEMTJguCHaoeK1t8URWbuGJSstw-cM"
         ↓
3. Client requests authorization with code_challenge:
   GET /authorize?
     response_type=code
     &client_id=CLIENT_ID
     &redirect_uri=https://app.com/callback
     &scope=read
     &code_challenge=E9Melhoa2OwvFrEMTJguCHaoeK1t8URWbuGJSstw-cM
     &code_challenge_method=S256
         ↓
4. Authorization Server returns authorization code
         ↓
5. Client exchanges code with code_verifier:
   POST /token
     grant_type=authorization_code
     &code=AUTH_CODE
     &redirect_uri=https://app.com/callback
     &client_id=CLIENT_ID
     &code_verifier=dBjftJeZ4CVP-mB92K27uhbUJU1p1r_wW1gFWFOEjXk
         ↓
6. Authorization Server validates code_verifier matches code_challenge
         ↓
7. If valid, returns tokens
```

```java
/**
 * PKCE Implementation
 */
@Service
public class PKCEService {
    
    private static final SecureRandom secureRandom = new SecureRandom();
    private static final Base64.Encoder base64Encoder = Base64.getUrlEncoder().withoutPadding();
    
    // ✅ Step 1: Generate code_verifier
    public String generateCodeVerifier() {
        byte[] codeVerifier = new byte[32];
        secureRandom.nextBytes(codeVerifier);
        return base64Encoder.encodeToString(codeVerifier);
    }
    
    // ✅ Step 2: Generate code_challenge from code_verifier
    public String generateCodeChallenge(String codeVerifier) throws NoSuchAlgorithmException {
        byte[] bytes = codeVerifier.getBytes(StandardCharsets.US_ASCII);
        MessageDigest messageDigest = MessageDigest.getInstance("SHA-256");
        byte[] digest = messageDigest.digest(bytes);
        return base64Encoder.encodeToString(digest);
    }
    
    // ✅ Complete PKCE flow
    public PKCEParameters generatePKCEParameters() throws NoSuchAlgorithmException {
        String codeVerifier = generateCodeVerifier();
        String codeChallenge = generateCodeChallenge(codeVerifier);
        
        return new PKCEParameters(codeVerifier, codeChallenge);
    }
}

@Data
@AllArgsConstructor
public class PKCEParameters {
    private String codeVerifier;
    private String codeChallenge;
}

/**
 * Authorization Request with PKCE
 */
@RestController
public class OAuth2Controller {
    
    @Autowired
    private PKCEService pkceService;
    
    // Step 1: Initiate OAuth2 flow with PKCE
    @GetMapping("/oauth2/authorize")
    public void authorize(HttpServletResponse response, HttpSession session) 
            throws NoSuchAlgorithmException, IOException {
        
        // Generate PKCE parameters
        PKCEParameters pkce = pkceService.generatePKCEParameters();
        
        // Store code_verifier in session for later use
        session.setAttribute("code_verifier", pkce.getCodeVerifier());
        
        // Build authorization URL
        String authUrl = "https://authorization-server.com/authorize" +
            "?response_type=code" +
            "&client_id=" + clientId +
            "&redirect_uri=" + URLEncoder.encode(redirectUri, "UTF-8") +
            "&scope=openid%20profile%20email" +
            "&state=" + generateRandomState() +
            "&code_challenge=" + pkce.getCodeChallenge() +
            "&code_challenge_method=S256";
        
        response.sendRedirect(authUrl);
    }
    
    // Step 2: Handle callback and exchange code
    @GetMapping("/oauth2/callback")
    public ResponseEntity<?> callback(
            @RequestParam("code") String code,
            @RequestParam("state") String state,
            HttpSession session) {
        
        // Retrieve code_verifier from session
        String codeVerifier = (String) session.getAttribute("code_verifier");
        
        // Exchange code for tokens with code_verifier
        TokenResponse tokens = exchangeCodeWithPKCE(code, codeVerifier);
        
        return ResponseEntity.ok(tokens);
    }
    
    private TokenResponse exchangeCodeWithPKCE(String code, String codeVerifier) {
        RestTemplate restTemplate = new RestTemplate();
        
        MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
        params.add("grant_type", "authorization_code");
        params.add("code", code);
        params.add("redirect_uri", redirectUri);
        params.add("client_id", clientId);
        params.add("code_verifier", codeVerifier);  // PKCE parameter
        
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
        
        HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(params, headers);
        
        ResponseEntity<TokenResponse> response = restTemplate.postForEntity(
            "https://authorization-server.com/token",
            request,
            TokenResponse.class
        );
        
        return response.getBody();
    }
}
```

**PKCE Benefits**:

- ✅ Protects against authorization code interception
- ✅ No client secret needed (safe for public clients)
- ✅ Recommended for mobile apps and SPAs
- ✅ Backwards compatible (works with clients that don't use PKCE)

**PKCE Methods**:

- **S256** (SHA-256): Recommended, more secure
- **plain**: Legacy, less secure (code_challenge = code_verifier)

```java
@Configuration
public class AuthorizationServerConfig {

    @Bean
    public RegisteredClientRepository registeredClientRepository() {
        RegisteredClient webClient = RegisteredClient.withId(UUID.randomUUID().toString())
            .clientId("web-client")
            .clientSecret("{bcrypt}" + passwordEncoder().encode("secret"))
            .clientAuthenticationMethod(ClientAuthenticationMethod.CLIENT_SECRET_BASIC)
            .clientAuthenticationMethod(ClientAuthenticationMethod.CLIENT_SECRET_POST)
            
            // Authorization Code Flow
            .authorizationGrantType(AuthorizationGrantType.AUTHORIZATION_CODE)
            .authorizationGrantType(AuthorizationGrantType.REFRESH_TOKEN)
            
            // Client Credentials Flow
            .authorizationGrantType(AuthorizationGrantType.CLIENT_CREDENTIALS)
            
            .redirectUri("http://localhost:8080/login/oauth2/code/web-client")
            .redirectUri("http://localhost:8080/authorized")
            
            .scope(OidcScopes.OPENID)
            .scope(OidcScopes.PROFILE)
            .scope(OidcScopes.EMAIL)
            .scope("read")
            .scope("write")
            
            .clientSettings(ClientSettings.builder()
                .requireAuthorizationConsent(true)  // Show consent screen
                .requireProofKey(true)              // Require PKCE
                .build())
            
            .tokenSettings(TokenSettings.builder()
                .accessTokenTimeToLive(Duration.ofMinutes(30))
                .refreshTokenTimeToLive(Duration.ofDays(30))
                .reuseRefreshTokens(false)  // Rotate refresh tokens
                .build())
            
            .build();
        
        RegisteredClient mobileClient = RegisteredClient.withId(UUID.randomUUID().toString())
            .clientId("mobile-client")
            .clientAuthenticationMethod(ClientAuthenticationMethod.NONE)  // Public client
            .authorizationGrantType(AuthorizationGrantType.AUTHORIZATION_CODE)
            .authorizationGrantType(AuthorizationGrantType.REFRESH_TOKEN)
            .redirectUri("myapp://authorized")
            .scope(OidcScopes.OPENID)
            .scope("read")
            .clientSettings(ClientSettings.builder()
                .requireAuthorizationConsent(false)
                .requireProofKey(true)  // PKCE required for public clients
                .build())
            .build();
        
        return new InMemoryRegisteredClientRepository(webClient, mobileClient);
    }
    
    // ✅ 2. JWT configuration
    @Bean
    public JWKSource<SecurityContext> jwkSource() throws Exception {
        KeyPairGenerator keyPairGenerator = KeyPairGenerator.getInstance("RSA");
        keyPairGenerator.initialize(2048);
        KeyPair keyPair = keyPairGenerator.generateKeyPair();
        
        RSAPublicKey publicKey = (RSAPublicKey) keyPair.getPublic();
        RSAPrivateKey privateKey = (RSAPrivateKey) keyPair.getPrivate();
        
        RSAKey rsaKey = new RSAKey.Builder(publicKey)
            .privateKey(privateKey)
            .keyID(UUID.randomUUID().toString())
            .build();
        
        JWKSet jwkSet = new JWKSet(rsaKey);
        return (jwkSelector, securityContext) -> jwkSelector.select(jwkSet);
    }
    
    // ✅ 3. JWT decoder for token introspection
    @Bean
    public JwtDecoder jwtDecoder(JWKSource<SecurityContext> jwkSource) {
        return OAuth2AuthorizationServerConfiguration.jwtDecoder(jwkSource);
    }
    
    // ✅ 4. Authorization server settings
    @Bean
    public AuthorizationServerSettings authorizationServerSettings() {
        return AuthorizationServerSettings.builder()
            .issuer("http://localhost:9000")  // Issuer URL
            .authorizationEndpoint("/oauth2/authorize")
            .tokenEndpoint("/oauth2/token")
            .jwkSetEndpoint("/oauth2/jwks")
            .tokenRevocationEndpoint("/oauth2/revoke")
            .tokenIntrospectionEndpoint("/oauth2/introspect")
            .oidcClientRegistrationEndpoint("/connect/register")
            .oidcUserInfoEndpoint("/userinfo")
            .build();
    }
    
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

### Resource Server Configuration

```java
/**
 * Resource Server Configuration (Validates Access Tokens)
 */
@Configuration
@EnableWebSecurity
public class ResourceServerConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/api/public/**").permitAll()
                .requestMatchers("/api/admin/**").hasAuthority("SCOPE_admin")
                .requestMatchers("/api/**").hasAuthority("SCOPE_read")
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt
                    .decoder(jwtDecoder())
                    .jwtAuthenticationConverter(jwtAuthenticationConverter())
                )
            );
        
        return http.build();
    }
    
    // ✅ JWT decoder with issuer validation
    @Bean
    public JwtDecoder jwtDecoder() {
        NimbusJwtDecoder jwtDecoder = NimbusJwtDecoder
            .withJwkSetUri("http://localhost:9000/oauth2/jwks")
            .build();
        
        // Validate issuer
        jwtDecoder.setJwtValidator(JwtValidators.createDefaultWithIssuer("http://localhost:9000"));
        
        return jwtDecoder;
    }
    
    // ✅ Convert JWT to Authentication with authorities
    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtGrantedAuthoritiesConverter grantedAuthoritiesConverter = new JwtGrantedAuthoritiesConverter();
        grantedAuthoritiesConverter.setAuthoritiesClaimName("scope");
        grantedAuthoritiesConverter.setAuthorityPrefix("SCOPE_");
        
        JwtAuthenticationConverter jwtAuthenticationConverter = new JwtAuthenticationConverter();
        jwtAuthenticationConverter.setJwtGrantedAuthoritiesConverter(grantedAuthoritiesConverter);
        
        return jwtAuthenticationConverter;
    }
}

 * Resource Server Endpoints

@RestController
@RequestMapping("/api")
public class ResourceController {

    @GetMapping("/public/info")

    @GetMapping("/users")
    public List<User> getUsers(@AuthenticationPrincipal Jwt jwt) {
        String username = jwt.getSubject();
        List<String> scopes = jwt.getClaimAsStringList("scope");
        
        log.info("User {} with scopes {} accessing /api/users", username, scopes);
        return userService.findAll();
    }
    
    // Admin endpoint (requires 'admin' scope)
    @PreAuthorize("hasAuthority('SCOPE_admin')")
    @DeleteMapping("/users/{id}")
    public void deleteUser(@PathVariable Long id) {
        userService.delete(id);
    }
    
    // Extract claims from JWT
    @GetMapping("/me")
    public Map<String, Object> getCurrentUser(@AuthenticationPrincipal Jwt jwt) {
        return Map.of(
            "sub", jwt.getSubject(),
            "email", jwt.getClaimAsString("email"),
            "scopes", jwt.getClaimAsStringList("scope"),
            "exp", jwt.getExpiresAt(),
            "iat", jwt.getIssuedAt()
        );
    }
}
```

### JWT Structure

```
┌─────────────────────────────────────────────────────────────┐
│                    JWT Structure                             │
└─────────────────────────────────────────────────────────────┘

JWT = Header.Payload.Signature

Example JWT:
eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiYWRtaW4iOnRydWUsImlhdCI6MTUxNjIzOTAyMn0.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

Part 1: Header (Base64URL encoded)
{
  "alg": "RS256",        // Algorithm: RSA with SHA-256
  "typ": "JWT",          // Type: JSON Web Token
  "kid": "key-id-123"    // Key ID (identifies signing key)
}

Part 2: Payload (Base64URL encoded)
{
  // Standard claims (registered)
  "iss": "http://localhost:9000",           // Issuer
  "sub": "1234567890",                      // Subject (user ID)
  "aud": ["api.example.com"],               // Audience
  "exp": 1735689600,                        // Expiration time (Unix timestamp)
  "nbf": 1735686000,                        // Not before
  "iat": 1735686000,                        // Issued at
  "jti": "unique-token-id",                 // JWT ID
  
  // Public claims (custom)
  "name": "John Doe",
  "email": "john@example.com",
  "roles": ["ROLE_USER", "ROLE_ADMIN"],
  "scope": ["read", "write"]
}

Part 3: Signature
RSASHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  privateKey
)
```

```java
/**
 * JWT Structure Breakdown
 */
@Service
public class JwtService {
    
    // ✅ Parse JWT and extract parts
    public void parseJWT(String token) {
        String[] parts = token.split("\\.");
        
        if (parts.length != 3) {
            throw new IllegalArgumentException("Invalid JWT format");
        }
        
        // Decode header
        String headerJson = new String(Base64.getUrlDecoder().decode(parts[0]));
        System.out.println("Header: " + headerJson);
        
        // Decode payload
        String payloadJson = new String(Base64.getUrlDecoder().decode(parts[1]));
        System.out.println("Payload: " + payloadJson);
        
        // Signature (binary, cannot decode to string)
        byte[] signature = Base64.getUrlDecoder().decode(parts[2]);
        System.out.println("Signature length: " + signature.length + " bytes");
    }
    
    // ✅ Create JWT manually
    public String createJWT(PrivateKey privateKey) throws Exception {
        // 1. Create header
        String header = Base64.getUrlEncoder().withoutPadding()
            .encodeToString("{\"alg\":\"RS256\",\"typ\":\"JWT\"}".getBytes());
        
        // 2. Create payload
        String payload = Base64.getUrlEncoder().withoutPadding()
            .encodeToString(createPayload().getBytes());
        
        // 3. Create signature
        String signatureInput = header + "." + payload;
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initSign(privateKey);
        signature.update(signatureInput.getBytes());
        byte[] signatureBytes = signature.sign();
        
        String signatureEncoded = Base64.getUrlEncoder().withoutPadding()
            .encodeToString(signatureBytes);
        
        // 4. Combine parts
        return header + "." + payload + "." + signatureEncoded;
    }
    
    private String createPayload() {
        return """
            {
              "sub": "user123",
              "name": "John Doe",
              "iat": %d,
              "exp": %d
            }
            """.formatted(
                Instant.now().getEpochSecond(),
                Instant.now().plusSeconds(3600).getEpochSecond()
            );
    }
}
```

### JWS vs JWE

```java
/**
 * JWS (JSON Web Signature) vs JWE (JSON Web Encryption)
 */

// JWS: Signed token (integrity, not confidentiality)
// Format: header.payload.signature
// Anyone can READ the payload (Base64 decoded)
// Signature ensures token was not tampered with

String jws = "eyJhbGci..."; // Standard JWT
// Payload visible: {"sub": "user123", "role": "admin"} ← Anyone can see this!

// JWE: Encrypted token (confidentiality AND integrity)
// Format: header.encrypted_key.iv.ciphertext.tag
// Only holder of decryption key can READ the payload

String jwe = "eyJhbGci..."; // Encrypted JWT
// Payload encrypted: Cannot see contents without decryption key ✅

/**
 * When to use JWS vs JWE
 */

// ✅ Use JWS (standard JWT) when:
// - Token contains non-sensitive data (user ID, roles, permissions)
// - Token transmitted over HTTPS (encrypted in transit)
// - Performance is important (no encryption overhead)
// - Most common use case

// ✅ Use JWE when:
// - Token contains sensitive data (SSN, credit card, medical info)
// - Token might be stored insecurely (localStorage, logs)
// - Extra security layer needed beyond HTTPS
// - Compliance requirements (HIPAA, PCI-DSS)
```

```java
@Configuration
public class JWEConfig {
    
    @Bean
    public JwtEncoder jweEncoder() throws Exception {
        // Generate RSA key pair
        KeyPairGenerator keyPairGenerator = KeyPairGenerator.getInstance("RSA");
        keyPairGenerator.initialize(2048);
        KeyPair keyPair = keyPairGenerator.generateKeyPair();
        
        RSAPublicKey publicKey = (RSAPublicKey) keyPair.getPublic();
        RSAPrivateKey privateKey = (RSAPrivateKey) keyPair.getPrivate();
        
        // Create JWK with encryption
        RSAKey rsaKey = new RSAKey.Builder(publicKey)
            .privateKey(privateKey)
            .keyUse(KeyUse.ENCRYPTION)
            .algorithm(JWEAlgorithm.RSA_OAEP_256)
            .build();
        
        JWKSource<SecurityContext> jwkSource = (jwkSelector, context) -> 
            jwkSelector.select(new JWKSet(rsaKey));
        
        return new NimbusJwtEncoder(jwkSource);
    }
    
    @Bean
    public JwtDecoder jweDecoder() throws Exception {
        // Configure decoder to handle JWE
        return NimbusJwtDecoder.withPublicKey((RSAPublicKey) publicKey)
            .build();
    }
```

```java
/**
 * Comprehensive Token Validation

@Service
public class TokenValidationService {
    
    @Autowired
    private JwtDecoder jwtDecoder;
    
    @Autowired
    private TokenBlacklistService blacklistService;
    
    // ✅ Complete token validation
    public boolean validateToken(String token) {
        try {
            // 1. Decode and verify signature
            Jwt jwt = jwtDecoder.decode(token);
            
            // 2. Check expiration
            if (!validateExpiration(jwt)) {
                log.warn("Token expired");
                return false;
            }
            
            // 3. Check not-before
            if (!validateNotBefore(jwt)) {
                log.warn("Token not yet valid");
                return false;
            }
            
            // 4. Validate issuer
            if (!validateIssuer(jwt)) {
                log.warn("Invalid issuer");
                return false;
            }
            
            // 5. Validate audience
            if (!validateAudience(jwt)) {
                log.warn("Invalid audience");
                return false;
            }
            
            // 6. Check if token is blacklisted
            if (blacklistService.isBlacklisted(jwt.getId())) {
                log.warn("Token is blacklisted");
                return false;
            }
            
            // 7. Validate custom claims
            if (!validateCustomClaims(jwt)) {
                log.warn("Invalid custom claims");
                return false;
            }
            
            return true;
            
        } catch (JwtException e) {
            log.error("Token validation failed", e);
            return false;
        }
    }
    
    private boolean validateExpiration(Jwt jwt) {
        Instant exp = jwt.getExpiresAt();
        return exp != null && exp.isAfter(Instant.now());
    }
    
    private boolean validateNotBefore(Jwt jwt) {
        Instant nbf = jwt.getNotBefore();
        return nbf == null || nbf.isBefore(Instant.now());
    }
    
    private boolean validateIssuer(Jwt jwt) {
        String expectedIssuer = "http://localhost:9000";
        return expectedIssuer.equals(jwt.getIssuer().toString());
    }
    
    private boolean validateAudience(Jwt jwt) {
        List<String> expectedAudiences = List.of("api.example.com", "mobile-app");
        List<String> tokenAudiences = jwt.getAudience();
        
        return tokenAudiences.stream()
            .anyMatch(expectedAudiences::contains);
    }
    
    private boolean validateCustomClaims(Jwt jwt) {
        // Validate email claim
        String email = jwt.getClaimAsString("email");
        if (email == null || !email.matches("^[A-Za-z0-9+_.-]+@(.+)$")) {
            return false;
        }
        
        // Validate tenant
        String tenantId = jwt.getClaimAsString("tenant_id");
        if (!tenantService.exists(tenantId)) {
            return false;
        }
        
        return true;
    }
}

/**
```
```java
 * Refresh Token Implementation
 */
@Service
public class RefreshTokenService {
    
    @Autowired
    private RefreshTokenRepository refreshTokenRepository;
    
    @Autowired
    private JwtService jwtService;
    
    // ✅ Generate refresh token
    public RefreshToken createRefreshToken(String username) {
        RefreshToken refreshToken = new RefreshToken();
        refreshToken.setToken(UUID.randomUUID().toString());
        refreshToken.setUsername(username);
        refreshToken.setExpiryDate(Instant.now().plusSeconds(86400 * 30)); // 30 days
        refreshToken.setCreatedAt(Instant.now());
        
        return refreshTokenRepository.save(refreshToken);
    }
    
    // ✅ Validate refresh token
    public RefreshToken validateRefreshToken(String token) {
        RefreshToken refreshToken = refreshTokenRepository.findByToken(token)
            .orElseThrow(() -> new TokenException("Refresh token not found"));
        
        if (refreshToken.getExpiryDate().isBefore(Instant.now())) {
            refreshTokenRepository.delete(refreshToken);
            throw new TokenException("Refresh token expired");
        }
        
        return refreshToken;
    }
    
    // ✅ Refresh access token
    public TokenResponse refreshAccessToken(String refreshTokenValue) {
        // Validate refresh token
        RefreshToken refreshToken = validateRefreshToken(refreshTokenValue);
        
        // Generate new access token
        String accessToken = jwtService.generateToken(refreshToken.getUsername());
        
        // Optionally rotate refresh token
        RefreshToken newRefreshToken = rotateRefreshToken(refreshToken);
        
        return new TokenResponse(
            accessToken,
            "Bearer",
            3600,
            newRefreshToken.getToken()
        );
    }
    
    // ✅ Rotate refresh token (recommended for security)
    private RefreshToken rotateRefreshToken(RefreshToken oldToken) {
        // Delete old token
        refreshTokenRepository.delete(oldToken);
        
        // Create new token
        return createRefreshToken(oldToken.getUsername());
    }
    
    // ✅ Revoke refresh token
    public void revokeRefreshToken(String token) {
        refreshTokenRepository.findByToken(token)
            .ifPresent(refreshTokenRepository::delete);
    }
    
    // ✅ Revoke all refresh tokens for user
    public void revokeAllUserTokens(String username) {
        refreshTokenRepository.deleteAllByUsername(username);
    }
}

    @PostMapping("/refresh")
    public ResponseEntity<TokenResponse> refresh(@RequestBody RefreshRequest request) {
        try {
            TokenResponse response = refreshTokenService.refreshAccessToken(
                request.getRefreshToken()
            );
            
            return ResponseEntity.ok(response);
            
        } catch (TokenException e) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED)
                .body(null);
        }
    }
    
    @PostMapping("/revoke")
    public ResponseEntity<Void> revoke(@RequestBody RevokeRequest request) {
        refreshTokenService.revokeRefreshToken(request.getRefreshToken());
        return ResponseEntity.ok().build();
    }
}

@Data
public class RefreshRequest {
    private String refreshToken;
}

@Data
public class RevokeRequest {
    private String refreshToken;
}
```

```java
// Strategy 1: Token Blacklist (JTI-based)
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

@Service
public class TokenBlacklistService {
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    private static final String BLACKLIST_PREFIX = "blacklist:";
    
    // ✅ Revoke token by adding to blacklist
    public void revokeToken(Jwt jwt) {
        String jti = jwt.getId();  // JWT ID
        Instant exp = jwt.getExpiresAt();
        
        // Store in Redis with TTL = token expiration
        Duration ttl = Duration.between(Instant.now(), exp);
        redisTemplate.opsForValue().set(
            BLACKLIST_PREFIX + jti, 
            "revoked", 
            ttl
        );
    }
    
    // ✅ Check if token is blacklisted
    public boolean isBlacklisted(String jti) {
        return Boolean.TRUE.equals(
            redisTemplate.hasKey(BLACKLIST_PREFIX + jti)
        );
    }
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// Strategy 2: Short-lived Access Tokens + Refresh Tokens
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

@Configuration
public class TokenLifetimeConfig {
    
    @Bean
    public TokenSettings tokenSettings() {
        return TokenSettings.builder()
            .accessTokenTimeToLive(Duration.ofMinutes(15))    // Short-lived
            .refreshTokenTimeToLive(Duration.ofDays(30))      // Long-lived
            .reuseRefreshTokens(false)                        // Rotate on use
            .build();
    }
}

// Benefits:
// - Access token compromised → Limited damage (expires in 15 min)
// - Revoke refresh token → User must re-authenticate
// - No need to check blacklist for every request

// Strategy 3: Token Versioning
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

@Service
public class TokenVersioningService {
    
    @Autowired
    private UserRepository userRepository;
    
    // ✅ Generate token with version
    public String generateToken(User user) {
        return Jwts.builder()
            .setSubject(user.getUsername())
            .claim("token_version", user.getTokenVersion())  // Add version
            .setExpiration(new Date(System.currentTimeMillis() + 3600000))
            .signWith(SignatureAlgorithm.HS512, secret)
            .compact();
    }
    
    // ✅ Validate token version
    public boolean validateTokenVersion(Jwt jwt) {
        String username = jwt.getSubject();
        Integer tokenVersion = jwt.getClaim("token_version");
        
        User user = userRepository.findByUsername(username).orElse(null);
        
        return user != null && tokenVersion.equals(user.getTokenVersion());
    }
    
    // ✅ Revoke all tokens for user
    public void revokeAllUserTokens(String username) {
        User user = userRepository.findByUsername(username).orElseThrow();
        user.setTokenVersion(user.getTokenVersion() + 1);  // Increment version
        userRepository.save(user);
        
        // All existing tokens now invalid (version mismatch)
    }
}

// Strategy 4: Refresh Token Rotation
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

@Service
public class RefreshTokenRotationService {
    
    // ✅ Refresh token rotation (best practice)
    public TokenResponse refreshToken(String oldRefreshToken) {
        // 1. Validate old refresh token
        RefreshToken refreshToken = validateRefreshToken(oldRefreshToken);
        
        // 2. Generate new access token
        String newAccessToken = generateAccessToken(refreshToken.getUsername());
        
        // 3. Generate new refresh token (rotate)
        RefreshToken newRefreshToken = createRefreshToken(refreshToken.getUsername());
        
        // 4. Delete old refresh token
        refreshTokenRepository.delete(refreshToken);
        
        // 5. Return new tokens
        return new TokenResponse(newAccessToken, newRefreshToken.getToken());
    }
    
    // ✅ Detect refresh token reuse (security)
    public TokenResponse refreshWithReuseDetection(String refreshToken) {
        Optional<RefreshToken> token = refreshTokenRepository.findByToken(refreshToken);
        
        if (token.isEmpty()) {
            // Token doesn't exist → might be reused!
            // Revoke all tokens for this user family
            revokeTokenFamily(refreshToken);
            throw new SecurityException("Refresh token reuse detected");
        }
        
        return refreshToken(refreshToken);
    }
    
    private void revokeTokenFamily(String token) {
        // Find all tokens in same family (issued from same original token)
        // and revoke them all
        String tokenFamily = extractTokenFamily(token);
        refreshTokenRepository.deleteAllByTokenFamily(tokenFamily);
    }
}
```

|Strategy|Pros|Cons|Use Case|
|---|---|---|---|
|**Blacklist (JTI)**|Immediate revocation|Requires storage & lookup|Emergency revocation|
|**Short-lived tokens**|No revocation needed|Frequent token refresh|High-security apps|
|**Token versioning**|Simple, efficient|Revokes ALL user tokens|Account compromise|
|**Refresh rotation**|Detects token theft|Complex implementation|Production systems|

**Recommended Approach**:

- Short-lived access tokens (15 min)
- Refresh token rotation
- Token versioning for emergency revocation
- Blacklist as last resort

## Summary & Best Practices

### OAuth2 Flows

|Flow|Use Case|Client Type|Security|Recommended|
|---|---|---|---|---|
|**Authorization Code + PKCE**|Web/Mobile apps|Public/Confidential|⭐⭐⭐⭐⭐|✅ Yes|
|**Client Credentials**|M2M, microservices|Confidential|⭐⭐⭐⭐|✅ Yes|
|**Resource Owner Password**|Legacy migration|Trusted first-party|⭐⭐|⚠️ Avoid|
|**Implicit**|SPAs (old)|Public|⭐|❌ Deprecated|

### JWT Best Practices

1. **Keep tokens small** (< 1-2 KB)
2. **Use short expiration** (15-60 minutes)
3. **Always validate** signature, expiration, issuer, audience
4. **Use RS256** for production (not HS256)
5. **Rotate refresh tokens** on use
6. **Never store sensitive data** in JWT (it's Base64, not encrypted)
7. **Use HTTPS** always (tokens in transit)
8. **Implement token versioning** for emergency revocation
9. **Monitor token usage** (detect anomalies)
10. **Use JWE** for truly sensitive data

### Security Checklist

✅ PKCE for public clients  
✅ Client secret protection for confidential clients  
✅ Short-lived access tokens  
✅ Refresh token rotation  
✅ Token blacklist for revocation  
✅ HTTPS everywhere  
✅ Validate all claims  
✅ Rate limit token endpoints  
✅ Monitor for token theft  
✅ Regular key rotation
