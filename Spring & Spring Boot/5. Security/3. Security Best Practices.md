## Overview

Security is not optional—it's fundamental. Understanding common vulnerabilities and protection mechanisms is crucial for building secure applications. Spring Security provides many protections by default, but knowing how they work helps you configure them correctly.

**Key Concept**: Defense in depth—use multiple layers of security so that if one fails, others still protect your application.

---

## 1. CSRF Protection

### How CSRF Works

```
┌─────────────────────────────────────────────────────────────┐
│            CSRF Attack (Cross-Site Request Forgery)          │
└─────────────────────────────────────────────────────────────┘

SCENARIO: User is logged into bank.com

1. User logs into bank.com
   → Session cookie set: JSESSIONID=ABC123
   
2. User visits malicious site evil.com (in different tab)

3. evil.com contains hidden form:
   <form action="https://bank.com/transfer" method="POST">
     <input name="to" value="attacker-account" />
     <input name="amount" value="10000" />
   </form>
   <script>document.forms[0].submit();</script>
   
4. Browser automatically includes bank.com cookies with request!
   POST https://bank.com/transfer
   Cookie: JSESSIONID=ABC123  ← User's valid session
   
5. bank.com processes request → Money transferred! ❌

WHY IT WORKS:
- Browser sends cookies automatically
- bank.com sees valid session cookie
- bank.com cannot distinguish between legitimate request and CSRF attack
```

```java
/**
 * CSRF Attack Example
 */

// Victim's legitimate request (works fine)
// POST /transfer
// Cookie: JSESSIONID=abc123
// CSRF-Token: xyz789
// Body: to=12345&amount=100

// Attacker's CSRF request (blocked by CSRF protection)
// POST /transfer
// Cookie: JSESSIONID=abc123  ← Browser sends this automatically
// NO CSRF-Token!              ← Attacker doesn't know this
// Body: to=attacker&amount=10000
```

### CSRF Tokens

```java
/**
 * CSRF Protection Configuration
 */
@Configuration
@EnableWebSecurity
public class CsrfConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf
                // ✅ Default: CSRF protection enabled for all POST, PUT, DELETE, PATCH
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
                
                // Ignore CSRF for specific endpoints (e.g., webhooks)
                .ignoringRequestMatchers("/api/webhooks/**")
            )
            .authorizeHttpRequests(authorize -> authorize
                .anyRequest().authenticated()
            );
        
        return http.build();
    }
}

/**
 * CSRF Token Flow
 */
@RestController
public class CsrfController {
    
    // ✅ Step 1: Client gets CSRF token
    @GetMapping("/api/csrf-token")
    public CsrfToken getCsrfToken(CsrfToken csrfToken) {
        // Spring automatically generates token and adds to response
        return csrfToken;
    }
    
    // Response:
    // {
    //   "token": "4c9f5e88-1234-5678-9abc-def012345678",
    //   "headerName": "X-CSRF-TOKEN",
    //   "parameterName": "_csrf"
    // }
    
    // ✅ Step 2: Client includes token in subsequent requests
    @PostMapping("/api/data")
    public String saveData(@RequestBody DataRequest data) {
        // Request must include:
        // Header: X-CSRF-TOKEN: 4c9f5e88-1234-5678-9abc-def012345678
        // OR
        // Body parameter: _csrf=4c9f5e88-1234-5678-9abc-def012345678
        
        return "Data saved";
    }
}

/**
 * CSRF Token in HTML Forms

// Thymeleaf automatically includes CSRF token

    <button type="submit">Transfer</button>

// Generated HTML:

    <input type="text" name="to" />

/**
 * CSRF Token in JavaScript (AJAX)
 */
// Get token from meta tag (added by Spring Security)
<meta name="_csrf" content="${_csrf.token}"/>
<meta name="_csrf_header" content="${_csrf.headerName}"/>

// Include in AJAX request
const token = document.querySelector('meta[name="_csrf"]').content;
const header = document.querySelector('meta[name="_csrf_header"]').content;

fetch('/api/data', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        [header]: token  // X-CSRF-TOKEN: 4c9f5e88-...
    },
    body: JSON.stringify({ data: 'value' })
});

/**
 * Cookie-based CSRF Token (for SPAs)
 */
@Configuration
public class CsrfCookieConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
                // Token stored in cookie: XSRF-TOKEN
                // JavaScript can read it (not HttpOnly)
                // Include in request header: X-XSRF-TOKEN
            );
        
        return http.build();
    }
}

// 1. Read token from XSRF-TOKEN cookie
// 2. Send in X-XSRF-TOKEN header

// Angular does this automatically!
```

### When to Disable CSRF

```java
/**
 * Disabling CSRF (Stateless REST APIs)
 */
@Configuration
@EnableWebSecurity
public class StatelessApiConfig {
    
    @Bean
    public SecurityFilterChain apiSecurityFilterChain(HttpSecurity http) throws Exception {
        http
            .securityMatcher("/api/**")
            .csrf(csrf -> csrf.disable())  // ✅ Disable for stateless API
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))  // No sessions
            .authorizeHttpRequests(authorize -> authorize
                .anyRequest().authenticated())
            .oauth2ResourceServer(oauth2 -> oauth2.jwt());  // JWT-based auth
        
        return http.build();
    }
}

/**
 * When to DISABLE CSRF:
 */
// ✅ Stateless REST API with JWT/Bearer tokens
//    - No cookies, no sessions
//    - Authentication via Authorization header
//    - Example: Mobile app backend, microservices

// ✅ Public read-only API
//    - No state-changing operations
//    - All endpoints are GET requests

// ✅ Webhooks from third-party services
//    - Cannot provide CSRF tokens
//    - Authenticated via signature/secret

/**
 * When to KEEP CSRF enabled:
 */
// ✅ Session-based authentication (cookies)
//    - Traditional web applications
//    - Form-based login

// ✅ Any application using cookies for authentication
//    - Even if it's a REST API
//    - Cookies sent automatically by browser → CSRF risk

/**
 * Hybrid Configuration: CSRF for web, disabled for API
 */
@Configuration
@EnableWebSecurity
public class HybridSecurityConfig {
    
    @Bean
    @Order(1)
    public SecurityFilterChain apiSecurityFilterChain(HttpSecurity http) throws Exception {
        http
            .securityMatcher("/api/**")
            .csrf(csrf -> csrf.disable())  // No CSRF for API
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .oauth2ResourceServer(oauth2 -> oauth2.jwt());
        
        return http.build();
    }
    
    @Bean
    @Order(2)
    public SecurityFilterChain webSecurityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(Customizer.withDefaults())  // CSRF enabled for web
            .authorizeHttpRequests(authorize -> authorize
                .anyRequest().authenticated())
            .formLogin(Customizer.withDefaults());
        
        return http.build();
    }
}
```

## 2. CORS (Cross-Origin Resource Sharing)

### Same-Origin Policy

```
┌─────────────────────────────────────────────────────────────┐
│                   Same-Origin Policy                         │
└─────────────────────────────────────────────────────────────┘

ORIGIN = Protocol + Domain + Port

Examples:
https://example.com:443
│      │          │        │
└─ Protocol       │        └─ Port
                  └─ Domain

SAME ORIGIN:
https://example.com/page1 → https://example.com/page2  ✅
http://example.com:80      → http://example.com:80     ✅

DIFFERENT ORIGIN (CORS required):
https://example.com  → http://example.com    ❌ (different protocol)
https://example.com  → https://api.example.com ❌ (different subdomain)
https://example.com  → https://example.com:8080 ❌ (different port)
```

```java
/**
 * Same-Origin Policy Example
 */

// Scenario: Frontend at https://myapp.com
//           API at https://api.myapp.com

// Without CORS configuration:
fetch('https://api.myapp.com/data')
  .then(response => response.json())
  .catch(error => {
    // ❌ Error: CORS policy: No 'Access-Control-Allow-Origin' header
  });

// Browser blocks request because origins are different!
```

### CORS Configuration

```java
/**
 * Global CORS Configuration
 */
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            // ✅ Allow specific origin
            .allowedOrigins("https://myapp.com")
            
            // ✅ Allow multiple origins
            .allowedOrigins("https://myapp.com", "https://admin.myapp.com")
            
            // ⚠️ Allow all origins (not recommended for production)
            // .allowedOrigins("*")
            
            // ✅ Allow specific HTTP methods
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            
            // ✅ Allow all methods
            // .allowedMethods("*")
            
            // ✅ Allow specific headers
            .allowedHeaders("Authorization", "Content-Type", "X-Requested-With")
            
            // ✅ Allow all headers
            // .allowedHeaders("*")
            
            // ✅ Expose specific headers to JavaScript
            .exposedHeaders("X-Total-Count", "X-Page-Number")
            
            // ✅ Allow credentials (cookies, authorization headers)
            .allowCredentials(true)
            
            // ✅ Pre-flight request cache duration (seconds)
            .maxAge(3600);  // 1 hour
    }
}

/**
 * Security-based CORS Configuration
 */
@Configuration
@EnableWebSecurity
public class SecurityCorsConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(authorize -> authorize
                .anyRequest().authenticated());
        
        return http.build();
    }
    
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        
        // ✅ Production: Specific origins only
        configuration.setAllowedOrigins(Arrays.asList(
            "https://myapp.com",
            "https://admin.myapp.com"
        ));
        
        // ✅ Development: Pattern-based origins
        // configuration.setAllowedOriginPatterns(Arrays.asList("http://localhost:[*]"));
        
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setExposedHeaders(Arrays.asList("Authorization", "X-Total-Count"));
        configuration.setAllowCredentials(true);
        configuration.setMaxAge(3600L);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", configuration);
        
        return source;
    }
}

/**
 * Controller-level CORS
 */
@RestController
@RequestMapping("/api")
@CrossOrigin(
    origins = "https://myapp.com",
    methods = {RequestMethod.GET, RequestMethod.POST},
    allowedHeaders = "*",
    exposedHeaders = "X-Total-Count",
    allowCredentials = "true",
    maxAge = 3600
)
public class ApiController {
    
    @GetMapping("/data")
    public List<Data> getData() {
        return dataService.findAll();
    }
    
    // ✅ Method-level CORS (overrides class-level)
    @PostMapping("/data")
    @CrossOrigin(origins = "https://admin.myapp.com")
    public Data createData(@RequestBody Data data) {
        return dataService.save(data);
    }
}
```

### Pre-flight Requests

```
┌─────────────────────────────────────────────────────────────┐
│              CORS Pre-flight Request                         │
└─────────────────────────────────────────────────────────────┘

When browser sends pre-flight OPTIONS request:
1. Non-simple requests (PUT, DELETE, PATCH)
2. Custom headers (Authorization, X-Custom-Header)
3. Content-Type other than application/x-www-form-urlencoded,
   multipart/form-data, or text/plain

Example Flow:

1. JavaScript makes request:
   fetch('https://api.example.com/data', {
     method: 'DELETE',
     headers: {
       'Authorization': 'Bearer token123',
       'Content-Type': 'application/json'
     }
   });

2. Browser sends pre-flight OPTIONS request:
   OPTIONS /data HTTP/1.1
   Host: api.example.com
   Origin: https://myapp.com
   Access-Control-Request-Method: DELETE
   Access-Control-Request-Headers: Authorization, Content-Type

3. Server responds to OPTIONS:
   HTTP/1.1 200 OK
   Access-Control-Allow-Origin: https://myapp.com
   Access-Control-Allow-Methods: GET, POST, PUT, DELETE
   Access-Control-Allow-Headers: Authorization, Content-Type
   Access-Control-Allow-Credentials: true
   Access-Control-Max-Age: 3600

4. Browser caches pre-flight response for 3600 seconds

5. Browser sends actual DELETE request:
   DELETE /data HTTP/1.1
   Host: api.example.com
   Origin: https://myapp.com
   Authorization: Bearer token123
   Content-Type: application/json

6. Server responds with data:
   HTTP/1.1 200 OK
   Access-Control-Allow-Origin: https://myapp.com
   Access-Control-Allow-Credentials: true
```

```java
/**
 * CORS Pre-flight Handling
 */
@Configuration
public class CorsPreflightConfig {
    
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowedOrigins(Arrays.asList("https://myapp.com"));
        config.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        config.setAllowedHeaders(Arrays.asList("*"));
        config.setAllowCredentials(true);
        
        // ✅ Cache pre-flight response for 1 hour
        config.setMaxAge(3600L);  // Reduces pre-flight requests
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        
        return source;
    }
}

/**
 * Common CORS Errors and Solutions
 */

// ❌ Error: "Access-Control-Allow-Origin header is missing"
// Solution: Add .allowedOrigins("https://myapp.com")

// ❌ Error: "Access-Control-Allow-Credentials: true, but Access-Control-Allow-Origin is '*'"
// Solution: Cannot use allowCredentials(true) with allowedOrigins("*")
//          Must specify exact origins

// ❌ Error: "Method DELETE is not allowed by Access-Control-Allow-Methods"
// Solution: Add "DELETE" to .allowedMethods()

// ❌ Error: "Authorization header is not allowed by Access-Control-Allow-Headers"
// Solution: Add "Authorization" to .allowedHeaders() or use .allowedHeaders("*")
```

## 3. Common Vulnerabilities

### SQL Injection

```java
/**
 * SQL Injection Prevention
 */

// ❌ VULNERABLE: String concatenation
@Repository
public class VulnerableUserRepository {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    // ❌ DO NOT DO THIS
    public User findByUsername(String username) {
        String sql = "SELECT * FROM users WHERE username = '" + username + "'";
        return jdbcTemplate.queryForObject(sql, userRowMapper);
    }
    
    // Attack: username = "admin' OR '1'='1"
    // Result SQL: SELECT * FROM users WHERE username = 'admin' OR '1'='1'
    // Returns ALL users! ❌
}

// ✅ SAFE: JPA with parameter binding
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    
    // ✅ Safe: Spring Data JPA uses prepared statements
    Optional<User> findByUsername(String username);
    
    // ✅ Safe: JPQL with named parameters
    @Query("SELECT u FROM User u WHERE u.username = :username")
    Optional<User> findByUsernameJPQL(@Param("username") String username);
    
    // ✅ Safe: Native query with parameters
    @Query(value = "SELECT * FROM users WHERE username = ?1", nativeQuery = true)
    Optional<User> findByUsernameNative(String username);
}

// ✅ SAFE: JdbcTemplate with prepared statements
@Repository
public class SafeUserRepository {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    // ✅ Safe: Uses PreparedStatement with placeholders
    public User findByUsername(String username) {
        String sql = "SELECT * FROM users WHERE username = ?";
        return jdbcTemplate.queryForObject(sql, userRowMapper, username);
    }
    
    // ✅ Safe: Named parameters
    public User findByUsernameNamed(String username) {
        String sql = "SELECT * FROM users WHERE username = :username";
        
        MapSqlParameterSource params = new MapSqlParameterSource();
        params.addValue("username", username);
        
        return namedParameterJdbcTemplate.queryForObject(sql, params, userRowMapper);
    }
}

// ✅ SAFE: EntityManager with parameters
@Repository
public class SafeEntityManagerRepository {
    
    @PersistenceContext
    private EntityManager entityManager;
    
    public User findByUsername(String username) {
        TypedQuery<User> query = entityManager.createQuery(
            "SELECT u FROM User u WHERE u.username = :username", 
            User.class
        );
        query.setParameter("username", username);
        return query.getSingleResult();
    }
}

/**
 * SQL Injection in Dynamic Queries
 */
@Repository
public class DynamicQueryRepository {
    
    @PersistenceContext
    private EntityManager entityManager;
    
    // ❌ VULNERABLE: Dynamic ORDER BY
    public List<User> findAllSorted(String sortField) {
        // Attack: sortField = "username; DROP TABLE users--"
        String jpql = "SELECT u FROM User u ORDER BY u." + sortField;  // ❌
        return entityManager.createQuery(jpql, User.class).getResultList();
    }
    
    // ✅ SAFE: Whitelist approach
    public List<User> findAllSortedSafe(String sortField) {
        // Validate against whitelist
        Set<String> allowedFields = Set.of("username", "email", "createdAt");
        if (!allowedFields.contains(sortField)) {
            throw new IllegalArgumentException("Invalid sort field");
        }
        
        String jpql = "SELECT u FROM User u ORDER BY u." + sortField;  // ✅ Safe
        return entityManager.createQuery(jpql, User.class).getResultList();
    }
    
    // ✅ SAFE: Criteria API
    public List<User> findAllSortedCriteria(String sortField) {
        CriteriaBuilder cb = entityManager.getCriteriaBuilder();
        CriteriaQuery<User> query = cb.createQuery(User.class);
        Root<User> root = query.from(User.class);
        
        // Safe: No string concatenation
        query.select(root).orderBy(cb.asc(root.get(sortField)));
        
        return entityManager.createQuery(query).getResultList();
    }
}
```

```java
/**
 * XSS Prevention
 */

// ❌ VULNERABLE: Unescaped user input
@Controller
public class VulnerableController {
    
    @GetMapping("/welcome")
    public String welcome(@RequestParam String name, Model model) {

// welcome.html (Thymeleaf)
// ❌ VULNERABLE: Unescaped
<div th:utext="${name}"></div>  
// Output: <div><script>alert('XSS')</script></div>  ❌

// ✅ SAFE: Escaped by default
<div th:text="${name}"></div>
// Output: <div>&lt;script&gt;alert('XSS')&lt;/script&gt;</div>  ✅

/**
 * XSS Types and Prevention
 */

// Type 1: Stored XSS (most dangerous)
@RestController
public class CommentController {
    
    @PostMapping("/comments")
    public Comment createComment(@RequestBody CommentRequest request) {
        Comment comment = new Comment();
        
        // ✅ Sanitize input before storing
        comment.setText(sanitize(request.getText()));
        
        return commentRepository.save(comment);
    }
    
    private String sanitize(String input) {
        // Use OWASP Java HTML Sanitizer
        PolicyFactory policy = new HtmlPolicyBuilder()
            .allowElements("b", "i", "u", "p", "br")
            .allowAttributes("href").onElements("a")
            .requireRelNofollowOnLinks()
            .toFactory();
        
        return policy.sanitize(input);
    }
}

// Type 2: Reflected XSS
@GetMapping("/search")
public String search(@RequestParam String query, Model model) {
    // ❌ VULNERABLE: query = "<script>...</script>"
    model.addAttribute("query", query);
    
    // Search results...
    return "search-results";
}

// search-results.html
// ✅ SAFE: Thymeleaf escapes by default
<p>You searched for: <span th:text="${query}">query</span></p>

// Type 3: DOM-based XSS (client-side)
// JavaScript vulnerability - sanitize on backend

/**
 * Content Security Policy (CSP) - Additional XSS protection
 */
@Configuration
public class SecurityHeadersConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            .contentSecurityPolicy(csp -> csp
                .policyDirectives("default-src 'self'; " +
                    "script-src 'self' 'nonce-{nonce}'; " +  // Only scripts with nonce
                    "style-src 'self' 'unsafe-inline'; " +
                    "img-src 'self' data: https:; " +
                    "font-src 'self'; " +
                    "connect-src 'self'; " +
                    "frame-ancestors 'none'")
            )
        );
        
        return http.build();
    }
}

/**
 * OWASP Java HTML Sanitizer
 */
// pom.xml
<dependency>
    <groupId>com.googlecode.owasp-java-html-sanitizer</groupId>
    <artifactId>owasp-java-html-sanitizer</artifactId>
    <version>20220608.1</version>
</dependency>

@Service
public class HtmlSanitizer {
    
    private final PolicyFactory policy;
    
    public HtmlSanitizer() {
        this.policy = new HtmlPolicyBuilder()
            // Allow basic formatting
            .allowElements("b", "i", "u", "strong", "em", "p", "br", "ul", "ol", "li")
            
            // Allow links with restrictions
            .allowElements("a")
            .allowAttributes("href").onElements("a")
            .requireRelNofollowOnLinks()
            .allowUrlProtocols("http", "https")
            
            // Allow images with restrictions
            .allowElements("img")
            .allowAttributes("src", "alt").onElements("img")
            .allowUrlProtocols("http", "https")
            
            .toFactory();
    }
    
    public String sanitize(String untrustedHtml) {
        if (untrustedHtml == null) {
            return null;
        }
        return policy.sanitize(untrustedHtml);
    }
}
```

### Clickjacking

```java
/**
 * Clickjacking Prevention
 */

// Attack: Attacker embeds your site in invisible iframe
// <iframe src="https://bank.com/transfer" style="opacity:0"></iframe>
// <button>Win iPhone!</button>  ← User clicks this, actually clicks hidden iframe

@Configuration
@EnableWebSecurity
public class ClickjackingConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            // ✅ DENY: Cannot be embedded in iframe at all (most secure)
            .frameOptions(frame -> frame.deny())
            
            // ✅ SAMEORIGIN: Can be embedded only by same origin
            // .frameOptions(frame -> frame.sameOrigin())
            
            // ⚠️ ALLOW-FROM: Specific origin (deprecated, use CSP instead)
            // .frameOptions(frame -> frame.disable())
        );
        
        return http.build();
    }
}

/**
 * X-Frame-Options Header
 */
// Response headers:
// X-Frame-Options: DENY              ← Cannot be framed
// X-Frame-Options: SAMEORIGIN        ← Can be framed by same origin
// X-Frame-Options: ALLOW-FROM https://trusted.com  ← Deprecated

/**
 * Content-Security-Policy (Modern approach)
 */
@Configuration
public class CspClickjackingConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            .contentSecurityPolicy(csp -> csp
                .policyDirectives("frame-ancestors 'none'")  // Same as X-Frame-Options: DENY
                // .policyDirectives("frame-ancestors 'self'")  // Same as SAMEORIGIN
                // .policyDirectives("frame-ancestors https://trusted.com")  // Whitelist
            )
        );
        
        return http.build();
    }
}
```

### Session Fixation

```java
/**
 * Session Fixation Prevention
 */

// Attack:
// 1. Attacker creates session: JSESSIONID=attacker-session
// 2. Attacker sends link to victim: http://bank.com/?JSESSIONID=attacker-session
// 3. Victim logs in using attacker's session ID
// 4. Attacker uses same session ID → logged in as victim! ❌

@Configuration
@EnableWebSecurity
public class SessionFixationConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .sessionManagement(session -> session
                // ✅ RECOMMENDED: Create new session, migrate attributes
                .sessionFixation().migrateSession()  // Default
                
                // Alternative: Create new session, don't migrate attributes
                // .sessionFixation().newSession()
                
                // Alternative: Change session ID only (Servlet 3.1+)
                // .sessionFixation().changeSessionId()
                
                // ❌ DANGEROUS: Don't change session (not recommended)
                // .sessionFixation().none()
            );
        
        return http.build();
    }
```

```java
/**
 * Content-Security-Policy Configuration
 */
@Configuration
@EnableWebSecurity
public class CspConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            .contentSecurityPolicy(csp -> csp
                .policyDirectives(buildCspPolicy())
            )
        );
        
        return http.build();
    }
    
    private String buildCspPolicy() {
        return """
            default-src 'self';
            script-src 'self' 'nonce-{nonce}' https://trusted-cdn.com;
            style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
            img-src 'self' data: https:;
            font-src 'self' https://fonts.gstatic.com;
            connect-src 'self' https://api.example.com;
            frame-src 'none';
            object-src 'none';
            base-uri 'self';
            form-action 'self';
            frame-ancestors 'none';
            upgrade-insecure-requests;
            block-all-mixed-content;
            """.replaceAll("\n", " ").trim();
    }
}

/**
 * CSP Directives Explained
 */

// default-src: Fallback for other directives
// default-src 'self'  → Only load from same origin

// script-src: JavaScript sources
// script-src 'self' 'nonce-ABC123'  → Self + scripts with nonce
// script-src 'self' 'unsafe-inline'  → Self + inline scripts (not recommended)
// script-src 'self' https://cdn.example.com  → Self + specific CDN

// style-src: CSS sources
// style-src 'self' 'unsafe-inline'  → Self + inline styles

// img-src: Image sources
// img-src 'self' data: https:  → Self + data URLs + HTTPS images

// connect-src: AJAX, WebSocket, EventSource
// connect-src 'self' https://api.example.com

// font-src: Font sources
// font-src 'self' https://fonts.gstatic.com

// frame-src: iframe sources
// frame-src 'none'  → No iframes allowed

// object-src: <object>, <embed>, <applet>
// object-src 'none'  → No plugins allowed

// base-uri: <base> element
// base-uri 'self'  → Restrict base URL

// form-action: Form submission targets
// form-action 'self'  → Forms can only submit to same origin

// frame-ancestors: Who can embed this page in iframe
// frame-ancestors 'none'  → Cannot be embedded (replaces X-Frame-Options)

// upgrade-insecure-requests: Upgrade HTTP to HTTPS
// block-all-mixed-content: Block HTTP content on HTTPS page

/**
 * CSP with Nonce
 */
@Controller
public class CspNonceController {
    
    @GetMapping("/page")
    public String page(HttpServletRequest request, Model model) {
        // Generate nonce
        String nonce = UUID.randomUUID().toString();
        
        // Add to model for template
        model.addAttribute("cspNonce", nonce);
        
        // Add to CSP header
        String csp = "default-src 'self'; script-src 'self' 'nonce-" + nonce + "'";
        ((HttpServletResponse) request).setHeader("Content-Security-Policy", csp);
        
        return "page";
    }
}
```

```java
            .frameOptions(frame -> frame

/**
 * X-Frame-Options Values
 */
// X-Frame-Options: DENY
// → Page cannot be displayed in iframe, frame, embed, or object

// X-Frame-Options: SAMEORIGIN
// → Page can be displayed in iframe only if origin matches

// X-Frame-Options: ALLOW-FROM https://example.com  ← DEPRECATED
// → Not supported by modern browsers, use CSP frame-ancestors instead

/**
 * When to use SAMEORIGIN vs DENY
 */

// Use DENY when:
// - Banking, financial applications
// - Admin panels
// - Authentication pages
// - Any page that shouldn't be embedded

// Use SAMEORIGIN when:
// - Your app uses iframes internally
// - Dashboard with embedded reports
// - Multi-window applications
```

### Strict-Transport-Security (HSTS)

```java
/**
 * HSTS Configuration
 */
@Configuration
@EnableWebSecurity
public class HstsConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            .httpStrictTransportSecurity(hsts -> hsts
                .maxAgeInSeconds(31536000)        // 1 year
                .includeSubDomains(true)          // Include all subdomains
                .preload(true)                    // Eligible for browser preload list
            )
        );
        
        return http.build();
    }
}

/**
 * HSTS Header Explained
 */
// Response header:
// Strict-Transport-Security: max-age=31536000; includeSubDomains; preload

// max-age=31536000
// → Browser remembers for 1 year (31536000 seconds)
// → All HTTP requests automatically upgraded to HTTPS

// includeSubDomains
// → Applies to all subdomains (api.example.com, www.example.com, etc.)

// preload
// → Site eligible for HSTS preload list
// → Browsers have built-in list of HTTPS-only sites
// → Submit at: https://hstspreload.org/

/**
 * HSTS Flow
 */
// First visit:
// 1. User types: http://example.com
// 2. Browser makes HTTP request
// 3. Server redirects to HTTPS with HSTS header
// 4. Browser remembers HSTS policy

// Subsequent visits (within max-age):
// 1. User types: http://example.com
// 2. Browser automatically converts to https://example.com
// 3. No HTTP request sent ✅
// 4. Protects against SSL stripping attacks

/**
 * HSTS Considerations
 */
// ✅ Enable HSTS in production with valid SSL certificate
// ⚠️ Start with short max-age (e.g., 300 seconds) for testing
// ⚠️ Increase max-age gradually (1 day → 1 week → 1 month → 1 year)
// ⚠️ includeSubDomains requires ALL subdomains to support HTTPS
// ⚠️ preload is permanent (very difficult to remove)

@Configuration
public class HstsProductionConfig {
    
    @Bean
    @Profile("production")
    public SecurityFilterChain productionFilterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            .httpStrictTransportSecurity(hsts -> hsts
                .maxAgeInSeconds(31536000)  // 1 year in production
                .includeSubDomains(true)
                .preload(true)
            )
        );
        
        return http.build();
    }
    
    @Bean
    @Profile("dev")
    public SecurityFilterChain devFilterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            .httpStrictTransportSecurity(hsts -> hsts
                .disable()  // Disable HSTS in development
            )
        );
        
        return http.build();
    }
}
```

```java
/**
 * Complete Security Headers Configuration

@Configuration
@EnableWebSecurity
public class ComprehensiveSecurityHeadersConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.headers(headers -> headers
            // ✅ Content Security Policy
            .contentSecurityPolicy(csp -> csp
                .policyDirectives("""
                    default-src 'self';
                    script-src 'self' 'nonce-{nonce}';

                    img-src 'self' data: https:;
                    font-src 'self';
                    connect-src 'self';
                    frame-src 'none';
                    object-src 'none';
                    base-uri 'self';
                    form-action 'self';
                    frame-ancestors 'none';
                    upgrade-insecure-requests;
                    """)
            )
            
            // ✅ X-Frame-Options (Clickjacking protection)
            .frameOptions(frame -> frame.deny())
            
            // ✅ X-Content-Type-Options (MIME sniffing protection)
            .contentTypeOptions(Customizer.withDefaults())
            
            // ✅ Strict-Transport-Security (Force HTTPS)
            .httpStrictTransportSecurity(hsts -> hsts
                .maxAgeInSeconds(31536000)
                .includeSubDomains(true)
                .preload(true)
            )
            
            // ✅ X-XSS-Protection (Legacy, but good to have)
            .xssProtection(xss -> xss
                .headerValue(XXssProtectionHeaderWriter.HeaderValue.ENABLED_MODE_BLOCK)
            )
            
            // ✅ Referrer-Policy
            .referrerPolicy(referrer -> referrer
                .policy(ReferrerPolicyHeaderWriter.ReferrerPolicy.STRICT_ORIGIN_WHEN_CROSS_ORIGIN)
            )
            
            // ✅ Permissions-Policy (formerly Feature-Policy)
            .permissionsPolicy(permissions -> permissions
                .policy("geolocation=(), microphone=(), camera=()")
            )
        );
        
        return http.build();
    }
}
```

## Summary & Best Practices

### Security Checklist

✅ **CSRF Protection**

- Enabled for session-based authentication
- Disabled for stateless APIs (JWT)
- Use CookieCsrfTokenRepository for SPAs

✅ **CORS Configuration**

- Specific origins (not "*" in production)
- Appropriate methods and headers
- allowCredentials carefully

✅ **Input Validation**

- Use JPA/prepared statements (prevent SQL injection)
- Sanitize HTML (prevent XSS)
- Validate all user input

✅ **Security Headers**

- Content-Security-Policy (XSS protection)
- X-Frame-Options (Clickjacking protection)
- X-Content-Type-Options (MIME sniffing protection)
- Strict-Transport-Security (Force HTTPS)

✅ **Session Management**

- Session fixation protection enabled
- Appropriate session timeout
- Secure session cookies

### Common Mistakes

❌ Disabling CSRF without understanding implications  
❌ Using CORS "*" in production  
❌ String concatenation in SQL queries  
❌ Rendering unescaped user input  
❌ Missing security headers  
❌ Not using HTTPS in production  
❌ Storing sensitive data in JWT  
❌ Not validating all user input

### Production Deployment Checklist

1. **Enable HTTPS** (mandatory)
2. **Configure HSTS** with appropriate max-age
3. **Enable all security headers** (CSP, X-Frame-Options, etc.)
4. **CSRF protection** for session-based auth
5. **CORS** with specific origins
6. **Input validation** on all endpoints
7. **SQL injection prevention** via JPA/prepared statements
8. **XSS prevention** via output encoding
9. **Rate limiting** on sensitive endpoints
10. **Security auditing** and logging
