# Spring Cloud - Microservices Patterns

## Overview

Spring Cloud provides tools for building and managing microservices architectures. It offers solutions for common distributed system patterns like service discovery, configuration management, circuit breakers, and distributed tracing. Spring Cloud builds on Spring Boot to make microservices development easier.

**Key Concept**: Microservices introduce complexity (service discovery, fault tolerance, distributed configuration). Spring Cloud provides battle-tested patterns to solve these challenges.

## Microservices Architecture

```
┌─────────────────────────────────────────────────────────────┐
│              Microservices Architecture                      │
└─────────────────────────────────────────────────────────────┘

Monolithic Architecture:
┌──────────────────────────────────────┐
│         Single Application           │
│  ┌────────┐ ┌────────┐ ┌────────┐  │
│  │  User  │ │ Order  │ │Product │  │

│         Single Database              │
└──────────────────────────────────────┘
✅ Simple deployment
❌ Hard to scale
❌ All-or-nothing deploy

Microservices Architecture:
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│User Service │  │Order Service│  │Product Svc  │
│   + DB      │  │   + DB      │  │   + DB      │
└─────────────┘  └─────────────┘  └─────────────┘
       │                 │                 │
       └─────────────────┴─────────────────┘
                         │
              ┌──────────┴──────────┐
              │    API Gateway      │
              └─────────────────────┘
✅ Independent scaling
✅ Independent deployment
✅ Technology diversity
❌ Complexity
❌ Distributed system challenges
```

### Microservices Challenges

```
┌─────────────────────────────────────────────────────────────┐
│           Microservices Challenges & Solutions               │
└─────────────────────────────────────────────────────────────┘

Challenge 1: Service Discovery
Problem: How do services find each other?
Solution: Eureka, Consul (service registry)

Challenge 2: Load Balancing
Problem: How to distribute requests across instances?
Solution: Spring Cloud LoadBalancer, Ribbon

Challenge 3: Fault Tolerance
Problem: What if a service is down?
Solution: Resilience4j (circuit breakers, retry, bulkhead)

Challenge 4: API Gateway
Problem: Single entry point, routing, authentication?
Solution: Spring Cloud Gateway

Challenge 5: Configuration Management
Problem: Managing config across many services?
Solution: Spring Cloud Config

Challenge 6: Distributed Tracing
Problem: How to trace requests across services?
Solution: Spring Cloud Sleuth + Zipkin

Challenge 7: Security
Problem: Authentication and authorization across services?
Solution: OAuth2, JWT, Spring Security
```

### Service Discovery in Action

```java
/**
 * Order Service calling User Service via Discovery
 */
@SpringBootApplication
@EnableDiscoveryClient
public class OrderServiceApplication {
    
    public static void main(String[] args) {
        SpringApplication.run(OrderServiceApplication.class, args);
    }
    
    // ✅ RestTemplate with load balancing
    @Bean
    @LoadBalanced  // Enable client-side load balancing
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

/**
 * Order Service calling User Service
 */
@Service
public class OrderService {
    
    @Autowired
    @LoadBalanced
    private RestTemplate restTemplate;
    
    @Autowired
    private DiscoveryClient discoveryClient;
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Method 1: Using @LoadBalanced RestTemplate (Recommended)
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Order createOrder(Long userId, Long productId) {
        // Use service name instead of hostname:port
        String userServiceUrl = "http://user-service/api/users/" + userId;
        
        // RestTemplate automatically:
        // 1. Resolves "user-service" to actual instances via Eureka
        // 2. Load balances across available instances
        // 3. Handles failover
        User user = restTemplate.getForObject(userServiceUrl, User.class);
        
        String productServiceUrl = "http://product-service/api/products/" + productId;
        Product product = restTemplate.getForObject(productServiceUrl, Product.class);
        
        return new Order(user, product);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Method 2: Using DiscoveryClient directly (Manual)
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public List<String> getUserServiceInstances() {
        List<ServiceInstance> instances = discoveryClient.getInstances("user-service");
        
        return instances.stream()
            .map(instance -> instance.getHost() + ":" + instance.getPort())
            .collect(Collectors.toList());
        
        // Output: ["localhost:8081", "localhost:8082"]
    }
    
    public User getUserManually(Long userId) {
        List<ServiceInstance> instances = discoveryClient.getInstances("user-service");
        
        if (instances.isEmpty()) {
            throw new ServiceUnavailableException("User service not available");
        }
        
        // Pick first instance (no load balancing)
        ServiceInstance instance = instances.get(0);
        String url = "http://" + instance.getHost() + ":" + instance.getPort() 
                   + "/api/users/" + userId;
        
        RestTemplate plainRestTemplate = new RestTemplate();
        return plainRestTemplate.getForObject(url, User.class);
    }
}
```

```java
/**
 * Load Balancer Configuration
 */
@Configuration
public class LoadBalancerConfig {
    
    // ✅ Default: Round Robin
    @Bean
    public ReactorLoadBalancer<ServiceInstance> roundRobinLoadBalancer(
            Environment environment,
            LoadBalancerClientFactory loadBalancerClientFactory) {
        
        String name = environment.getProperty(LoadBalancerClientFactory.PROPERTY_NAME);
        return new RoundRobinLoadBalancer(
            loadBalancerClientFactory.getLazyProvider(name, ServiceInstanceListSupplier.class),
            name
        );
    }
    
    // ✅ Custom: Random
    @Bean
    public ReactorLoadBalancer<ServiceInstance> randomLoadBalancer(
            Environment environment,
            LoadBalancerClientFactory loadBalancerClientFactory) {
        
        String name = environment.getProperty(LoadBalancerClientFactory.PROPERTY_NAME);
        return new RandomLoadBalancer(
            loadBalancerClientFactory.getLazyProvider(name, ServiceInstanceListSupplier.class),
            name
        );
    }
}

            .get()
            .uri("http://user-service/api/users/{id}", id)  // Service name
            .retrieve()

        // WebClient automatically:
        // 1. Resolves service name
        // 2. Load balances
        // 3. Handles failover
    }
}

/**
 * Custom Load Balancer Algorithm
 */
public class CustomLoadBalancer implements ReactorServiceInstanceLoadBalancer {
    
    private final ObjectProvider<ServiceInstanceListSupplier> serviceInstanceListSupplierProvider;
    private final String serviceId;
    
    public CustomLoadBalancer(
            ObjectProvider<ServiceInstanceListSupplier> serviceInstanceListSupplierProvider,
            String serviceId) {
        this.serviceInstanceListSupplierProvider = serviceInstanceListSupplierProvider;
        this.serviceId = serviceId;
    }
    
    @Override
    public Mono<Response<ServiceInstance>> choose(Request request) {
        ServiceInstanceListSupplier supplier = serviceInstanceListSupplierProvider
            .getIfAvailable(NoopServiceInstanceListSupplier::new);
        
        return supplier.get(request)
            .next()
            .map(instances -> {
                if (instances.isEmpty()) {
                    return new EmptyResponse();
                }
                
                // Custom algorithm: Select instance with lowest load
                ServiceInstance selected = selectInstanceWithLowestLoad(instances);
                return new DefaultResponse(selected);
            });
    }
    
    private ServiceInstance selectInstanceWithLowestLoad(List<ServiceInstance> instances) {
        // Custom logic: check metrics, current load, etc.
        return instances.get(0);  // Simplified
    }
}
```

```java
/**
 * Circuit Breaker States
 */
/*
┌─────────────────────────────────────────────────────────────┐
│              Circuit Breaker States                          │
└─────────────────────────────────────────────────────────────┘

CLOSED (Normal):
- Requests go through
- Failures counted
- If failure rate > threshold → OPEN

OPEN (Circuit broken):
- All requests immediately fail
- No calls to downstream service
- After timeout → HALF_OPEN

HALF_OPEN (Testing):
- Limited requests allowed
- If successful → CLOSED
- If failed → OPEN
*/

/**
 * Using Circuit Breaker
 */
@Service
public class UserService {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @Autowired
    private CircuitBreakerFactory circuitBreakerFactory;
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Method 1: Using @CircuitBreaker annotation
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @CircuitBreaker(name = "userService", fallbackMethod = "getUserFallback")
    @Retry(name = "userService")
    @RateLimiter(name = "userService")
    public User getUser(Long id) {
        String url = "http://user-service/api/users/" + id;
        return restTemplate.getForObject(url, User.class);
    }
    
    // Fallback method (same signature + Throwable)
    public User getUserFallback(Long id, Throwable throwable) {
        log.error("Circuit breaker fallback for user {}", id, throwable);
        return new User(id, "Fallback User", "fallback@example.com");
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Method 2: Programmatic Circuit Breaker
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public User getUserProgrammatic(Long id) {
        CircuitBreaker circuitBreaker = circuitBreakerFactory.create("userService");
        
        return circuitBreaker.run(
            () -> {
                // Primary call
                String url = "http://user-service/api/users/" + id;
                return restTemplate.getForObject(url, User.class);
            },
            throwable -> {
                // Fallback
                log.error("Circuit breaker fallback", throwable);
                return new User(id, "Fallback User", "fallback@example.com");
            }
        );
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Multiple Fallback Methods (different exceptions)
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @CircuitBreaker(name = "userService", fallbackMethod = "handleException")
    public User getUserWithMultipleFallbacks(Long id) {
        String url = "http://user-service/api/users/" + id;
        return restTemplate.getForObject(url, User.class);
    }
    
    // Specific exception fallback
    public User handleException(Long id, HttpServerErrorException ex) {
        log.error("Server error for user {}", id, ex);
        return new User(id, "Server Error", "error@example.com");
    }
    
    // Generic exception fallback
    public User handleException(Long id, Exception ex) {
        log.error("Generic error for user {}", id, ex);
        return new User(id, "Generic Error", "error@example.com");
    }
}

/**
 * Bulkhead Pattern (Limit Concurrent Calls)
 */
@Service
public class OrderService {
    
    @Bulkhead(name = "userService", fallbackMethod = "bulkheadFallback", type = Bulkhead.Type.SEMAPHORE)
    public Order createOrder(Long userId, Long productId) {
        // Only 10 concurrent calls allowed (from config)
        // Additional calls wait or fail
        return processOrder(userId, productId);
    }
    
    public Order bulkheadFallback(Long userId, Long productId, BulkheadFullException ex) {
        log.error("Bulkhead full, rejecting request", ex);
        throw new ServiceUnavailableException("Service is busy, try again later");
    }
}

/**
 * Reactive Circuit Breaker
 */
@Service
public class ReactiveUserService {
    
    @Autowired
    private WebClient.Builder webClientBuilder;
    
    @CircuitBreaker(name = "userService", fallbackMethod = "getUserFallback")
    public Mono<User> getUser(Long id) {
        return webClientBuilder.build()
            .get()
            .uri("http://user-service/api/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class);
    }
    
    public Mono<User> getUserFallback(Long id, Throwable throwable) {
        log.error("Circuit breaker fallback", throwable);
        return Mono.just(new User(id, "Fallback User", "fallback@example.com"));
    }
}

/**
 * Circuit Breaker Events (Monitoring)
 */
@Component
public class CircuitBreakerEventListener {
    
    @Autowired
    private CircuitBreakerRegistry circuitBreakerRegistry;
    
    @PostConstruct
    public void registerEventListener() {
        CircuitBreaker circuitBreaker = circuitBreakerRegistry.circuitBreaker("userService");
        
        circuitBreaker.getEventPublisher()
            .onStateTransition(event -> {
                log.info("Circuit breaker state changed: {} -> {}", 
                    event.getStateTransition().getFromState(),
                    event.getStateTransition().getToState());
            })
            .onError(event -> {
                log.error("Circuit breaker error: {}", event.getThrowable().getMessage());
            })
            .onSuccess(event -> {
                log.info("Circuit breaker success");
            });
    }
}
```

```java
        return builder.routes()
            // ✅ Basic routing
            .route("user-service", r -> r
                .path("/api/users/**")
                .uri("lb://user-service"))
            
            // ✅ With filters
            .route("product-service", r -> r
                .path("/api/products/**")
                .filters(f -> f
                    .addRequestHeader("X-Gateway", "true")
                    .addResponseHeader("X-Response-Time", "123ms")
                    .stripPrefix(1))  // Remove /api from path
                .uri("lb://product-service"))
            
            // ✅ With predicates
            .route("order-service", r -> r
                .path("/api/orders/**")
                .and()
                .method(HttpMethod.POST, HttpMethod.PUT)
                .filters(f -> f
                    .circuitBreaker(config -> config
                        .setName("orderCircuitBreaker")
                        .setFallbackUri("forward:/fallback/orders")))
                .uri("lb://order-service"))
            
            // ✅ Query parameter routing
            .route("search-service", r -> r
                .path("/api/search")
                .and()
                .query("version", "v2")
                .uri("lb://search-service-v2"))
            
            // ✅ Header-based routing
            .route("mobile-api", r -> r
                .path("/api/**")
                .and()
                .header("X-Client-Type", "mobile")
                .filters(f -> f.rewritePath("/api/(?<segment>.*)", "/mobile/${segment}"))
                .uri("lb://mobile-api-service"))
            
            // ✅ Weighted routing (A/B testing)
            .route("user-service-v1", r -> r
                .path("/api/users/**")
                .and()
                .weight("user-service-group", 80)  // 80% traffic
                .uri("lb://user-service-v1"))
            
            .route("user-service-v2", r -> r
                .path("/api/users/**")
                .and()
                .weight("user-service-group", 20)  // 20% traffic
                .uri("lb://user-service-v2"))
            
            .build();
    }

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();

                duration);
        }));
    }

    public int getOrder() {

    }

/**
 * Authentication Filter
 */
@Component
public class AuthenticationFilter implements GlobalFilter, Ordered {
    
    @Autowired
    private JwtUtil jwtUtil;
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        
        // Skip authentication for public endpoints
        if (isPublicEndpoint(request.getPath().toString())) {
            return chain.filter(exchange);
        }
        
        // Extract token
        String token = extractToken(request);
        
        if (token == null || !jwtUtil.validateToken(token)) {
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
        }
        
        // Add user info to headers
        String userId = jwtUtil.getUserIdFromToken(token);
        ServerHttpRequest modifiedRequest = exchange.getRequest()
            .mutate()
            .header("X-User-Id", userId)
            .build();
        
        return chain.filter(exchange.mutate().request(modifiedRequest).build());
    }
    
    @Override
    public int getOrder() {
        return 0;
    }
    
    private boolean isPublicEndpoint(String path) {
        return path.startsWith("/api/public") || path.startsWith("/api/auth");
    }
    
    private String extractToken(ServerHttpRequest request) {
        String authHeader = request.getHeaders().getFirst(HttpHeaders.AUTHORIZATION);
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            return authHeader.substring(7);
        }
        return null;
    }
}

/**
```
```java
 * Distributed Tracing in Action
 */

// Request Flow:
// API Gateway → User Service → Order Service → Payment Service

// Each service automatically:
// 1. Receives trace ID and span ID from headers
// 2. Creates new span for its operation
// 3. Passes trace ID to next service
// 4. Sends span data to Zipkin

/**
 * User Service
 */
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @Autowired
    private Tracer tracer;  // Sleuth tracer
    
    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        // Automatically traced
        log.info("Getting user {}", id);  // Log includes traceId and spanId
        
        User user = userRepository.findById(id);
        
        // Call order service (trace ID propagated automatically)
        String orderUrl = "http://order-service/api/orders/user/" + id;
        List<Order> orders = restTemplate.getForObject(orderUrl, List.class);
        
        user.setOrders(orders);
        return user;
    }
    
    // ✅ Custom spans for detailed tracing
    @GetMapping("/{id}/detailed")
    public User getUserDetailed(@PathVariable Long id) {
        Span span = tracer.nextSpan().name("getUserDetailed").start();
        
        try (Tracer.SpanInScope ws = tracer.withSpan(span)) {
            // Database query
            Span dbSpan = tracer.nextSpan().name("db-query").start();
            try {
                User user = userRepository.findById(id);
                dbSpan.tag("user.id", id.toString());
                return user;
            } finally {
                dbSpan.end();
            }
        } finally {
            span.end();
        }
    }
}

/**
 * Order Service
 */
@RestController
@RequestMapping("/api/orders")
public class OrderController {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @GetMapping("/user/{userId}")
    public List<Order> getOrdersByUser(@PathVariable Long userId) {
        // Automatically traced with same trace ID from User Service
        log.info("Getting orders for user {}", userId);
        
        List<Order> orders = orderRepository.findByUserId(userId);
        
        // Call payment service
        for (Order order : orders) {
            String paymentUrl = "http://payment-service/api/payments/order/" + order.getId();
            Payment payment = restTemplate.getForObject(paymentUrl, Payment.class);
            order.setPayment(payment);
        }
        
        return orders;
    }
}
```

## Summary & Best Practices

### Microservices Patterns Quick Reference

|Pattern|Solution|Use Case|
|---|---|---|
|**Service Discovery**|Eureka, Consul|Dynamic service locations|
|**Load Balancing**|Spring Cloud LoadBalancer|Distribute requests|
|**Circuit Breaker**|Resilience4j|Fault tolerance|
|**API Gateway**|Spring Cloud Gateway|Single entry point|
|**Config Management**|Spring Cloud Config|Centralized configuration|
|**Distributed Tracing**|Sleuth + Zipkin|Request tracking|

### Best Practices

1. **Service Discovery**: Always use service names, not hardcoded URLs
2. **Circuit Breakers**: Implement fallbacks for all external calls
3. **API Gateway**: Single entry point for authentication, routing, rate limiting
4. **Configuration**: Externalize config, use profiles for environments
5. **Tracing**: Enable distributed tracing from day one
6. **Monitoring**: Use Actuator endpoints, integrate with monitoring tools
7. **Security**: Implement authentication at gateway, authorization in services
8. **Resilience**: Use retry, timeout, bulkhead patterns
9. **Testing**: Test failure scenarios (circuit open, service down)
10. **Documentation**: Document service contracts (OpenAPI/Swagger)

### Common Mistakes

❌ Hardcoding service URLs  
❌ No circuit breakers (cascading failures)  
❌ No fallbacks  
❌ Configuration in code  
❌ No distributed tracing  
❌ No health checks  
❌ No retry logic  
❌ Tight coupling between services  
❌ No API versioning  
❌ Ignoring network latency

### Microservices Checklist

✅ Service discovery configured  
✅ Circuit breakers implemented  
✅ Fallbacks defined  
✅ API gateway deployed  
✅ Distributed tracing enabled  
✅ Configuration externalized  
✅ Health checks implemented  
✅ Monitoring and alerts set up  
✅ Security implemented (OAuth2/JWT)  
✅ Load balancing configured
