## Overview

Spring WebFlux is Spring's reactive web framework, built on Project Reactor. It provides non-blocking, asynchronous request handling that can scale to handle massive concurrency with a small number of threads. WebFlux is an alternative to Spring MVC, not a replacement.

**Key Concept**: Traditional Spring MVC uses one thread per request (blocking). WebFlux uses event loop with few threads handling many requests (non-blocking). This makes WebFlux ideal for I/O-bound applications with high concurrency.

## 1. Reactive Web

### Traditional MVC vs WebFlux

```
┌─────────────────────────────────────────────────────────────┐
│           Spring MVC vs Spring WebFlux                       │
└─────────────────────────────────────────────────────────────┘

Spring MVC (Blocking):
Thread Pool (200 threads)
┌─────┐ ┌─────┐ ┌─────┐
│ T1  │ │ T2  │ │ T3  │ ... (200 threads)
└─────┘ └─────┘ └─────┘
   │       │       │
Request → Thread blocks waiting for DB/HTTP
   │       │       │
Max 200 concurrent requests

Spring WebFlux (Non-blocking):
Event Loop (4-8 threads)
┌─────┐ ┌─────┐
│ T1  │ │ T2  │ (4-8 threads)
└─────┘ └─────┘
   │       │
Request → Thread handles, delegates I/O, continues
   │       │
Handles thousands of concurrent requests
```

```xml
<!-- pom.xml -->
<dependencies>
    <!-- WebFlux (replaces spring-boot-starter-web) -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-webflux</artifactId>

    <!-- Reactive data access -->
    <dependency>
```

### Annotated Controllers

```java
/**
 * Reactive REST Controller (Annotated Style)
 */
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // GET - Single Item (Mono)
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @GetMapping("/{id}")
    public Mono<User> getUser(@PathVariable Long id) {
        return userService.findById(id);
        // Returns Mono<User> - framework subscribes automatically
    }
    
    // ✅ With ResponseEntity
    @GetMapping("/{id}/details")
    public Mono<ResponseEntity<User>> getUserWithStatus(@PathVariable Long id) {
        return userService.findById(id)
            .map(user -> ResponseEntity.ok(user))
            .defaultIfEmpty(ResponseEntity.notFound().build());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // GET - Multiple Items (Flux)
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @GetMapping
    public Flux<User> getAllUsers() {
        return userService.findAll();
        // Returns Flux<User> - streams items to client
    }
    
    // ✅ With pagination
    @GetMapping("/paginated")
    public Flux<User> getUsersPaginated(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size) {
        return userService.findAll()
            .skip(page * size)
            .take(size);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // POST - Create
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public Mono<User> createUser(@RequestBody @Valid UserRequest request) {
        return userService.createUser(request);
    }
    
    // ✅ With location header
    @PostMapping("/with-location")
    public Mono<ResponseEntity<User>> createUserWithLocation(
            @RequestBody @Valid UserRequest request) {
        return userService.createUser(request)
            .map(user -> ResponseEntity
                .created(URI.create("/api/users/" + user.getId()))
                .body(user));
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // PUT - Update
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @PutMapping("/{id}")
    public Mono<User> updateUser(
            @PathVariable Long id,
            @RequestBody @Valid UserRequest request) {
        return userService.updateUser(id, request);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // DELETE - Delete
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @DeleteMapping("/{id}")
    @ResponseStatus(HttpStatus.NO_CONTENT)
    public Mono<Void> deleteUser(@PathVariable Long id) {
        return userService.deleteUser(id);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Server-Sent Events (SSE) - Streaming
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<User> streamUsers() {
        return userService.streamUsers();
        // Continuously streams users to client
    }
    
    // ✅ SSE with interval
    @GetMapping(value = "/updates", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<User>> streamUserUpdates() {
        return userService.findAll()
            .repeat()  // Repeat query
            .delayElements(Duration.ofSeconds(5))  // Every 5 seconds
            .map(user -> ServerSentEvent.<User>builder()
                .id(String.valueOf(user.getId()))
                .event("user-update")
                .data(user)
                .build());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Error Handling
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFound(UserNotFoundException ex) {
        return ResponseEntity.status(HttpStatus.NOT_FOUND)
            .body(new ErrorResponse(ex.getMessage()));
    }
    
    @ExceptionHandler(ValidationException.class)
    public ResponseEntity<ErrorResponse> handleValidation(ValidationException ex) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST)
            .body(new ErrorResponse(ex.getMessage()));
    }
}

/**
 * Service Layer
 */
@Service
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    @Autowired
    private EmailService emailService;
    
    public Mono<User> findById(Long id) {
        return userRepository.findById(id)
            .switchIfEmpty(Mono.error(new UserNotFoundException("User not found: " + id)));
    }
    
    public Flux<User> findAll() {
        return userRepository.findAll();
    }
    
    public Mono<User> createUser(UserRequest request) {
        User user = new User();
        user.setEmail(request.getEmail());
        user.setName(request.getName());
        
        return userRepository.save(user)
            .flatMap(savedUser -> 
                emailService.sendWelcomeEmail(savedUser)
                    .thenReturn(savedUser)  // Return user after email sent
            );
    }
    
    public Mono<User> updateUser(Long id, UserRequest request) {
        return userRepository.findById(id)
            .switchIfEmpty(Mono.error(new UserNotFoundException("User not found")))
            .flatMap(user -> {
                user.setEmail(request.getEmail());
                user.setName(request.getName());
                return userRepository.save(user);
            });
    }
    
    public Mono<Void> deleteUser(Long id) {
        return userRepository.deleteById(id);
    }
    
    public Flux<User> streamUsers() {
        return userRepository.findAll()
            .delayElements(Duration.ofMillis(100));  // Throttle stream
    }
}
```

```java
/**
 * Functional Endpoints (Router Functions)

@Configuration
public class UserRouter {
    
    @Bean
    public RouterFunction<ServerResponse> userRoutes(UserHandler handler) {
        return RouterFunctions
            // ✅ GET /api/users
            .route(GET("/api/users"), handler::getAllUsers)
            
            // ✅ GET /api/users/{id}
            .andRoute(GET("/api/users/{id}"), handler::getUser)
            
            // ✅ POST /api/users
            .andRoute(POST("/api/users"), handler::createUser)
            
            // ✅ PUT /api/users/{id}
            .andRoute(PUT("/api/users/{id}"), handler::updateUser)
            
            // ✅ DELETE /api/users/{id}
            .andRoute(DELETE("/api/users/{id}"), handler::deleteUser)
            
            // ✅ GET /api/users/search?email={email}
            .andRoute(GET("/api/users/search"), handler::searchUsers);
    }
    
    // ✅ Advanced routing with predicates
    @Bean
    public RouterFunction<ServerResponse> advancedRoutes(UserHandler handler) {
        return RouterFunctions.nest(
            RequestPredicates.path("/api/v2/users"),
            RouterFunctions
                // GET with Accept header
                .route(GET("").and(accept(MediaType.APPLICATION_JSON)), 
                    handler::getAllUsers)
                
                // POST with Content-Type
                .andRoute(POST("").and(contentType(MediaType.APPLICATION_JSON)), 
                    handler::createUser)
                
                // Custom predicate
                .andRoute(RequestPredicates.method(HttpMethod.GET)
                    .and(path("/{id}"))
                    .and(accept(MediaType.APPLICATION_JSON)), 
                    handler::getUser)
        );
    }
}

/**
 * Handler Functions
 */
@Component
public class UserHandler {
    
    @Autowired
    private UserService userService;
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // GET All
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> getAllUsers(ServerRequest request) {
        Flux<User> users = userService.findAll();
        
        return ServerResponse.ok()
            .contentType(MediaType.APPLICATION_JSON)
            .body(users, User.class);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // GET Single
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> getUser(ServerRequest request) {
        Long id = Long.valueOf(request.pathVariable("id"));
        
        return userService.findById(id)
            .flatMap(user -> ServerResponse.ok()
                .contentType(MediaType.APPLICATION_JSON)
                .bodyValue(user))
            .switchIfEmpty(ServerResponse.notFound().build());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // POST Create
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> createUser(ServerRequest request) {
        Mono<UserRequest> userRequestMono = request.bodyToMono(UserRequest.class);
        
        return userRequestMono
            .flatMap(userService::createUser)
            .flatMap(user -> ServerResponse
                .created(URI.create("/api/users/" + user.getId()))
                .contentType(MediaType.APPLICATION_JSON)
                .bodyValue(user))
            .onErrorResume(ValidationException.class, error ->
                ServerResponse.badRequest()
                    .bodyValue(new ErrorResponse(error.getMessage()))
            );
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // PUT Update
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> updateUser(ServerRequest request) {
        Long id = Long.valueOf(request.pathVariable("id"));
        Mono<UserRequest> userRequestMono = request.bodyToMono(UserRequest.class);
        
        return userRequestMono
            .flatMap(userRequest -> userService.updateUser(id, userRequest))
            .flatMap(user -> ServerResponse.ok()
                .contentType(MediaType.APPLICATION_JSON)
                .bodyValue(user))
            .switchIfEmpty(ServerResponse.notFound().build());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // DELETE
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> deleteUser(ServerRequest request) {
        Long id = Long.valueOf(request.pathVariable("id"));
        
        return userService.deleteUser(id)
            .then(ServerResponse.noContent().build());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Query Parameters
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> searchUsers(ServerRequest request) {
        String email = request.queryParam("email").orElse("");
        
        Flux<User> users = userService.searchByEmail(email);
        
        return ServerResponse.ok()
            .contentType(MediaType.APPLICATION_JSON)
            .body(users, User.class);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // SSE Streaming
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ServerResponse> streamUsers(ServerRequest request) {
        Flux<User> users = userService.streamUsers();
        
        return ServerResponse.ok()
            .contentType(MediaType.TEXT_EVENT_STREAM)
            .body(users, User.class);
    }
}

/**
 * Annotated vs Functional Comparison
 */

// Annotated (Controller):
// ✅ Familiar Spring MVC style
// ✅ Easy to read and understand
// ✅ Built-in validation support
// ✅ Exception handling with @ExceptionHandler
// ❌ Less flexible
// ❌ More magic (annotations)

// Functional (Router):
// ✅ More explicit and functional
// ✅ More flexible routing
// ✅ Easier to test (pure functions)
// ✅ Better for complex routing logic
// ❌ More verbose
// ❌ Steeper learning curve

// You can use BOTH in the same application!
```

### WebClient - Reactive HTTP Client

```java
/**
 * WebClient Configuration
 */
@Configuration
public class WebClientConfig {
    
    @Bean
    public WebClient webClient(WebClient.Builder builder) {
        return builder
            .baseUrl("https://api.example.com")
            .defaultHeader(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
            .defaultHeader(HttpHeaders.USER_AGENT, "MyApp/1.0")
            
            // Connection timeout
            .clientConnector(new ReactorClientHttpConnector(
                HttpClient.create()
                    .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 5000)
                    .responseTimeout(Duration.ofSeconds(5))
            ))
            
            // Filters
            .filter(ExchangeFilterFunction.ofRequestProcessor(request -> {
                log.info("Request: {} {}", request.method(), request.url());
                return Mono.just(request);
            }))
            
            .filter(ExchangeFilterFunction.ofResponseProcessor(response -> {
                log.info("Response: {}", response.statusCode());
                return Mono.just(response);
            }))
            
            .build();
    }
}

@Service
public class ExternalApiService {
    
    @Autowired
    private WebClient webClient;
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // GET Request
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<User> getUser(Long id) {
        return webClient.get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class);
    }
    
    // ✅ GET with query parameters
    public Flux<User> searchUsers(String name, int page, int size) {
        return webClient.get()
            .uri(uriBuilder -> uriBuilder
                .path("/users/search")
                .queryParam("name", name)
                .queryParam("page", page)
                .queryParam("size", size)
                .build())
            .retrieve()
            .bodyToFlux(User.class);
    }
    
    // ✅ GET with headers
    public Mono<User> getUserWithAuth(Long id, String token) {
        return webClient.get()
            .uri("/users/{id}", id)
            .header(HttpHeaders.AUTHORIZATION, "Bearer " + token)
            .header("X-Custom-Header", "value")
            .retrieve()
            .bodyToMono(User.class);
    }

        return webClient.post()
            .uri("/users")
            .contentType(MediaType.APPLICATION_JSON)
            .bodyValue(request)  // Or .body(Mono.just(request), UserRequest.class)

            .bodyToMono(User.class);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    public Mono<Void> deleteUser(Long id) {
        return webClient.delete()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(Void.class);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Error Handling
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<User> getUserWithErrorHandling(Long id) {
        return webClient.get()
            .uri("/users/{id}", id)
            .retrieve()
            
            // Handle 4xx errors
            .onStatus(HttpStatusCode::is4xxClientError, response -> {
                if (response.statusCode() == HttpStatus.NOT_FOUND) {
                    return Mono.error(new UserNotFoundException("User not found"));
                }
                return Mono.error(new ClientException("Client error: " + response.statusCode()));
            })
            
            // Handle 5xx errors
            .onStatus(HttpStatusCode::is5xxServerError, response ->
                Mono.error(new ServerException("Server error: " + response.statusCode()))
            )
            
            .bodyToMono(User.class)
            
            // Fallback
            .onErrorResume(WebClientException.class, error -> {
                log.error("Network error", error);
                return Mono.empty();
            });
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Response with Headers and Status
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<ResponseEntity<User>> getUserWithResponse(Long id) {
        return webClient.get()
            .uri("/users/{id}", id)
            .retrieve()
            .toEntity(User.class);  // Returns ResponseEntity with headers and status
    }
    
    // ✅ Using exchangeToMono for full control
    public Mono<User> getUserWithExchange(Long id) {
        return webClient.get()
            .uri("/users/{id}", id)
            .exchangeToMono(response -> {
                if (response.statusCode().equals(HttpStatus.OK)) {
                    return response.bodyToMono(User.class);
                } else if (response.statusCode().equals(HttpStatus.NOT_FOUND)) {
                    return Mono.empty();
                } else {
                    return response.createException()
                        .flatMap(Mono::error);
                }
            });
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Streaming Response
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Flux<User> streamUsers() {
        return webClient.get()
            .uri("/users/stream")
            .accept(MediaType.TEXT_EVENT_STREAM)
            .retrieve()
            .bodyToFlux(User.class);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Retry and Timeout
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<User> getUserWithRetry(Long id) {
        return webClient.get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class)
            
            // Timeout
            .timeout(Duration.ofSeconds(5))
            
            // Retry
            .retryWhen(Retry.backoff(3, Duration.ofSeconds(1))
                .filter(throwable -> throwable instanceof WebClientRequestException)
            )
            
            // Log errors
            .doOnError(error -> log.error("Failed to get user", error));
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // Parallel Requests
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    public Mono<UserDetails> getUserDetails(Long userId) {
        Mono<User> userMono = getUser(userId);
        Mono<Profile> profileMono = getProfile(userId);
        Mono<List<Order>> ordersMono = getOrders(userId).collectList();
        
        // Execute in parallel, combine results
        return Mono.zip(userMono, profileMono, ordersMono)
            .map(tuple -> new UserDetails(
                tuple.getT1(),  // User
                tuple.getT2(),  // Profile
                tuple.getT3()   // Orders
            ));
    }
```

```java
/**
 * Use Case: API Gateway handling many concurrent requests
 */
@RestController
@RequestMapping("/api/gateway")
public class GatewayController {
    
    @Autowired
    private WebClient webClient;
    
    // ✅ GOOD FOR REACTIVE: Many parallel external calls
    @GetMapping("/user-dashboard/{userId}")
    public Mono<UserDashboard> getUserDashboard(@PathVariable Long userId) {
        // Execute 4 API calls in parallel
        Mono<User> userMono = webClient.get()
            .uri("/users/{id}", userId)
            .retrieve()
            .bodyToMono(User.class);
        
        Mono<Profile> profileMono = webClient.get()
            .uri("/profiles/{id}", userId)
            .retrieve()
            .bodyToMono(Profile.class);
        
        Mono<List<Order>> ordersMono = webClient.get()
            .uri("/orders?userId={id}", userId)
            .retrieve()
            .bodyToFlux(Order.class)
            .collectList();
        
        Mono<Statistics> statsMono = webClient.get()
            .uri("/statistics/{id}", userId)
            .retrieve()
            .bodyToMono(Statistics.class);
        
        // Combine all results
        return Mono.zip(userMono, profileMono, ordersMono, statsMono)
            .map(tuple -> new UserDashboard(
                tuple.getT1(),  // User
                tuple.getT2(),  // Profile
                tuple.getT3(),  // Orders
                tuple.getT4()   // Statistics
            ));
        
        // With Spring MVC (blocking):
        // - Would need 4 threads
        // - Each thread blocks waiting for response
        // - Can handle ~200 concurrent requests (with 200 thread pool)
        
        // With WebFlux (non-blocking):
        // - Uses 4-8 threads (event loop)
        // - Threads don't block
        // - Can handle thousands of concurrent requests
    }
}
```

```java
@Service

    private UserRepository userRepository;

    private ProductRepository productRepository;

    private PaymentService paymentService;

            userRepository.findById(request.getUserId()),      // I/O 1: Database
            productRepository.findById(request.getProductId()), // I/O 2: Database
            paymentService.processPayment(request.getPayment()) // I/O 3: External API
        )
        .flatMap(tuple -> {
            User user = tuple.getT1();
            Product product = tuple.getT2();
            Payment payment = tuple.getT3();
            
            Order order = new Order(user, product, payment);
            return orderRepository.save(order);  // I/O 4: Database
        })
        .flatMap(order -> 
            emailService.sendConfirmation(order)  // I/O 5: Email service
                .thenReturn(order)
        );
        
        // All I/O operations are non-blocking
        // Thread is free to handle other requests while waiting
    }
```

### When NOT to Use Reactive

```java
/**
 * ❌ DON'T USE REACTIVE: CPU-bound operations
 */
@Service
public class ImageProcessingService {
    
    // ❌ BAD: CPU-intensive operation
    public Mono<BufferedImage> processImage(BufferedImage image) {
        return Mono.fromCallable(() -> {
            // Heavy CPU work: resize, filters, transformations
            return heavyImageProcessing(image);  // Blocks thread!
        });
        
        // Problem: CPU work blocks the event loop thread
        // Event loop threads should NEVER be blocked
        // Solution: Use blocking thread pool or traditional Spring MVC
    }
    
    // ✅ GOOD: Offload to blocking thread pool
    public Mono<BufferedImage> processImageCorrectly(BufferedImage image) {
        return Mono.fromCallable(() -> heavyImageProcessing(image))
            .subscribeOn(Schedulers.boundedElastic());  // Use blocking thread pool
    }
}

/**
 * ❌ DON'T USE REACTIVE: Blocking dependencies
 */
@Service
public class LegacyService {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;  // ❌ Blocking JDBC
    
    // ❌ BAD: Blocking operation in reactive pipeline
    public Mono<User> getUser(Long id) {
        return Mono.fromCallable(() -> {
            // Blocking JDBC call
            return jdbcTemplate.queryForObject(
                "SELECT * FROM users WHERE id = ?",
                userRowMapper,
                id
            );
        });
        // Blocks event loop thread → defeats purpose of reactive
        
        // Solution: Use R2DBC instead of JDBC
        // Or: Offload to blocking thread pool
    }
}

/**
 * ❌ DON'T USE REACTIVE: Simple CRUD application
 */
// If your application is just:
// - Simple CRUD operations
// - Low concurrency (< 100 concurrent users)
// - No streaming requirements
// - Team not familiar with reactive

// Use traditional Spring MVC:
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);  // Blocking, simple
    }
    
    // Spring MVC is perfectly fine for most applications!
    // Don't use reactive just because it's "cool"
}

/**
 * ❌ DON'T USE REACTIVE: Debugging and monitoring are priorities
 */
// Reactive stack traces are harder to read
// Example reactive stack trace:
//   at reactor.core.publisher.Mono.lambda$onErrorMap$29
//   at reactor.core.publisher.FluxOnErrorResume$ResumeSubscriber
//   at reactor.core.publisher.Operators$MultiSubscriptionSubscriber
//   ... (many reactor internals)

// Blocking stack trace:
//   at com.example.UserService.getUser(UserService.java:42)
//   at com.example.UserController.getUser(UserController.java:25)
//   ... (clear application code)

// Use reactive only if:
// - Team is comfortable debugging reactive code
// - Have proper monitoring (Micrometer, reactive spans)
```

### Decision Matrix

```
┌─────────────────────────────────────────────────────────────┐
│          When to Use Spring MVC vs Spring WebFlux            │
└─────────────────────────────────────────────────────────────┘

Use Spring MVC when:
✅ Simple CRUD operations
✅ Low to medium concurrency (< 1000 concurrent requests)
✅ CPU-bound operations
✅ Team familiar with blocking paradigm
✅ Using blocking dependencies (JDBC, blocking APIs)
✅ Quick debugging needed
✅ Short time to market

Use Spring WebFlux when:
✅ High concurrency (> 1000 concurrent requests)
✅ I/O-bound operations (many external calls)
✅ Streaming data requirements (SSE, WebSocket)
✅ Microservices with many inter-service calls
✅ Need backpressure handling
✅ Team experienced with reactive programming
✅ Can use reactive drivers (R2DBC, reactive MongoDB)

Hybrid Approach (Both!):
✅ Most endpoints: Spring MVC (CRUD)
✅ High-concurrency endpoints: WebFlux (streaming, gateway)
✅ Gradual migration path
```

## Summary & Best Practices

### WebFlux Quick Reference

|Feature|Spring MVC|Spring WebFlux|
|---|---|---|
|**Programming Model**|Imperative|Reactive|
|**Threading**|Thread per request|Event loop (few threads)|
|**Concurrency**|Medium (100-1000)|Very High (10,000+)|
|**Latency**|Higher (thread context switching)|Lower (non-blocking)|
|**Debugging**|Easy|Harder|
|**Learning Curve**|Low|High|
|**Data Access**|JDBC (blocking)|R2DBC, Reactive MongoDB|

### Annotated vs Functional

|Aspect|Annotated (@RestController)|Functional (RouterFunction)|
|---|---|---|
|**Style**|Spring MVC-like|Functional programming|
|**Familiarity**|✅ Easy|⚠️ Learning curve|
|**Flexibility**|⚠️ Limited|✅ Very flexible|
|**Testing**|Good|✅ Excellent (pure functions)|
|**Routing**|Annotations|Code|

### Best Practices

1. **Never block** in reactive code (no `.block()`, no blocking I/O)
2. **Use R2DBC** instead of JDBC for databases
3. **WebClient** instead of RestTemplate for HTTP
4. **Offload CPU work** to `Schedulers.boundedElastic()`
5. **Handle errors** with `onErrorResume`, `retry`
6. **Use backpressure** operators (`onBackpressureBuffer`, `onBackpressureDrop`)
7. **Test reactively** with `StepVerifier`
8. **Monitor** with Micrometer and reactive spans
9. **Start simple** - don't use reactive everywhere
10. **Educate team** before adopting reactive

### Common Mistakes

❌ Blocking in reactive pipeline  
❌ Not handling backpressure  
❌ Using JDBC instead of R2DBC  
❌ Calling `.block()` in controllers  
❌ No error handling  
❌ Mixing blocking and non-blocking code  
❌ Using reactive when not needed  
❌ Not understanding reactive operators
