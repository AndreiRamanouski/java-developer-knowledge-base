# Spring Boot - Deep Interview Questions & Answers

## Overview

This guide contains in-depth answers to Spring Boot interview questions, focusing on auto-configuration, application setup, debugging, and custom starter creation.

**Interview Tip**: Spring Boot questions often test understanding of "convention over configuration" and how the framework makes intelligent decisions based on classpath and properties.

---

## Spring Boot Questions

### Q1: How does auto-configuration work?

**Short Answer**: Auto-configuration uses @Conditional annotations to automatically configure beans based on classpath presence, existing beans, and properties. It's activated by @EnableAutoConfiguration (part of @SpringBootApplication).

**Core Mechanism**:

```
1. @SpringBootApplication includes @EnableAutoConfiguration
2. @EnableAutoConfiguration imports AutoConfigurationImportSelector
3. Spring reads META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
   (or META-INF/spring.factories in Spring Boot 2.x)
4. Loads all auto-configuration classes listed
5. Each auto-configuration class uses @Conditional to decide if it should apply
6. Beans are created only if conditions are met
```

**Example: DataSource Auto-Configuration**:

```java
@Configuration
@ConditionalOnClass({ DataSource.class, EmbeddedDatabaseType.class })
@ConditionalOnMissingBean(DataSource.class)
@EnableConfigurationProperties(DataSourceProperties.class)
public class DataSourceAutoConfiguration {
    
    @Bean
    @ConditionalOnProperty(
        name = "spring.datasource.type", 
        havingValue = "com.zaxxer.hikari.HikariDataSource",
        matchIfMissing = true
    )
    public DataSource hikariDataSource(DataSourceProperties properties) {
        return properties.initializeDataSourceBuilder()
                        .type(HikariDataSource.class)
                        .build();
    }
}
```

**How It Works Step-by-Step**:

```java
// Step 1: Spring Boot starts
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

// Step 2: @SpringBootApplication is expanded
@Configuration
@EnableAutoConfiguration  // <-- This triggers auto-configuration
@ComponentScan
public @interface SpringBootApplication { }

// Step 3: AutoConfigurationImportSelector processes
public class AutoConfigurationImportSelector {
    
    public String[] selectImports(AnnotationMetadata metadata) {
        // Read from META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
        List<String> configurations = SpringFactoriesLoader.loadFactoryNames(
            EnableAutoConfiguration.class,
            classLoader
        );
        
        // Returns list like:
        // org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
        // org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration
        // org.springframework.boot.autoconfigure.web.servlet.WebMvcAutoConfiguration
        // ... hundreds more
        
        return configurations.toArray(new String[0]);
    }
}

// Step 4: Each auto-configuration is evaluated
@Configuration
@ConditionalOnClass(DataSource.class) // Is DataSource on classpath?
@ConditionalOnMissingBean(DataSource.class) // No DataSource bean exists?
public class DataSourceAutoConfiguration {
    
    @Bean
    public DataSource dataSource() {
        // Only created if conditions pass
        return new HikariDataSource();
    }
}
```

**Conditional Annotations**:

```java
// 1. Class must be on classpath
@ConditionalOnClass(RedisOperations.class)
// Checks: Is spring-data-redis dependency present?

// 2. Bean must NOT exist
@ConditionalOnMissingBean(DataSource.class)
// Checks: Has user defined custom DataSource?

// 3. Property must exist and match
@ConditionalOnProperty(
    name = "spring.redis.enabled", 
    havingValue = "true",
    matchIfMissing = true  // Default if property not set
)

// 4. Resource must exist
@ConditionalOnResource(resources = "classpath:schema.sql")
// Checks: Is schema.sql file present?

// 5. Web application type
@ConditionalOnWebApplication(type = WebApplicationType.SERVLET)
// Checks: Is this a servlet-based web app?

// 6. Expression must evaluate to true
@ConditionalOnExpression("${my.feature.enabled:true} and ${my.feature.version} >= 2")
// Checks: Complex SpEL expressions

// 7. Bean must exist
@ConditionalOnBean(DataSource.class)
// Checks: DataSource bean exists?

// 8. Single candidate
@ConditionalOnSingleCandidate(DataSource.class)
// Checks: Only one DataSource bean or one marked @Primary?

// 9. Java version
@ConditionalOnJava(JavaVersion.SEVENTEEN)
// Checks: Running on Java 17+?

// 10. Cloud platform
@ConditionalOnCloudPlatform(CloudPlatform.KUBERNETES)
// Checks: Running on Kubernetes?
```

**Complete Auto-Configuration Example**:

```java
// Redis Auto-Configuration (simplified from Spring Boot source)
@Configuration
@ConditionalOnClass(RedisOperations.class)
@EnableConfigurationProperties(RedisProperties.class)
public class RedisAutoConfiguration {
    
    @Configuration
    @ConditionalOnClass(Jedis.class)
    @ConditionalOnMissingBean(RedisConnectionFactory.class)
    protected static class JedisConnectionConfiguration {
        
        @Bean
        public JedisConnectionFactory redisConnectionFactory(
                RedisProperties properties) {
            
            RedisStandaloneConfiguration config = 
                new RedisStandaloneConfiguration(
                    properties.getHost(), 
                    properties.getPort()
                );
            
            if (properties.getPassword() != null) {
                config.setPassword(properties.getPassword());
            }
            
            return new JedisConnectionFactory(config);
        }
    }
    
    @Bean
    @ConditionalOnMissingBean(name = "redisTemplate")
    public RedisTemplate<Object, Object> redisTemplate(
            RedisConnectionFactory connectionFactory) {
        
        RedisTemplate<Object, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(connectionFactory);
        return template;
    }
    
    @Bean
    @ConditionalOnMissingBean
    public StringRedisTemplate stringRedisTemplate(
            RedisConnectionFactory connectionFactory) {
        
        return new StringRedisTemplate(connectionFactory);
    }
}

// Properties binding
@ConfigurationProperties(prefix = "spring.redis")
public class RedisProperties {
    private String host = "localhost";
    private int port = 6379;
    private String password;
    private int database = 0;
    
    // Getters and setters
}
```

**Auto-Configuration Order**:

```java
// Control when auto-configurations run relative to each other

@Configuration
@AutoConfigureAfter(DataSourceAutoConfiguration.class)
public class MyAutoConfiguration {
    // Runs AFTER DataSourceAutoConfiguration
    // Useful when you need DataSource to exist first
}

@Configuration
@AutoConfigureBefore(WebMvcAutoConfiguration.class)
public class SecurityAutoConfiguration {
    // Runs BEFORE WebMvcAutoConfiguration
    // Security filters need to be set up before MVC
}

@Configuration
@AutoConfigureOrder(Ordered.HIGHEST_PRECEDENCE)
public class PriorityAutoConfiguration {
    // Explicit ordering using Spring's Ordered interface
}
```

**Real-World Example - JPA Auto-Configuration**:

```java
@Configuration
@ConditionalOnClass({ LocalContainerEntityManagerFactoryBean.class, EntityManager.class })
@ConditionalOnBean(DataSource.class)
@EnableConfigurationProperties(JpaProperties.class)
@AutoConfigureAfter({ DataSourceAutoConfiguration.class })
public class HibernateJpaAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean
    public LocalContainerEntityManagerFactoryBean entityManagerFactory(
            DataSource dataSource,
            JpaProperties properties) {
        
        LocalContainerEntityManagerFactoryBean factory = 
            new LocalContainerEntityManagerFactoryBean();
        
        factory.setDataSource(dataSource);
        factory.setPackagesToScan(properties.getPackagesToScan());
        factory.setJpaVendorAdapter(new HibernateJpaVendorAdapter());
        
        Map<String, Object> jpaProperties = new HashMap<>();
        jpaProperties.put("hibernate.hbm2ddl.auto", properties.getHibernate().getDdlAuto());
        jpaProperties.put("hibernate.dialect", properties.getHibernate().getDialect());
        jpaProperties.put("hibernate.show_sql", properties.isShowSql());
        
        factory.setJpaPropertyMap(jpaProperties);
        
        return factory;
    }
    
    @Bean
    @ConditionalOnMissingBean
    public JpaTransactionManager transactionManager(EntityManagerFactory emf) {
        return new JpaTransactionManager(emf);
    }
}
```

**Understanding Auto-Configuration Flow**:

```java
// What happens when you add a dependency:

// 1. Add dependency to pom.xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>

// 2. Dependency brings these classes to classpath:
//    - RedisOperations.class
//    - RedisTemplate.class
//    - JedisConnectionFactory.class

// 3. Spring Boot finds RedisAutoConfiguration via spring.factories

// 4. RedisAutoConfiguration checks conditions:
@ConditionalOnClass(RedisOperations.class) // ✅ Present on classpath
@ConditionalOnMissingBean(RedisConnectionFactory.class) // ✅ User hasn't defined one

// 5. Beans are created automatically:
//    - RedisConnectionFactory
//    - RedisTemplate
//    - StringRedisTemplate

// 6. You can now inject and use:
@Service
public class CacheService {
    @Autowired
    private RedisTemplate<String, Object> redisTemplate; // Auto-configured!
    
    public void cache(String key, Object value) {
        redisTemplate.opsForValue().set(key, value);
    }
}
```

**Custom Condition Example**:

```java
// Create custom condition
public class OnDevelopmentEnvironmentCondition implements Condition {
    
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        Environment env = context.getEnvironment();
        String[] profiles = env.getActiveProfiles();
        
        return Arrays.asList(profiles).contains("dev") 
            || Arrays.asList(profiles).contains("development");
    }
}

// Use custom condition
@Configuration
@Conditional(OnDevelopmentEnvironmentCondition.class)
public class DevelopmentToolsAutoConfiguration {
    
    @Bean
    public DevToolsService devToolsService() {
        return new DevToolsService();
    }
}
```

### Q2: Explain @SpringBootApplication

**Short Answer**: @SpringBootApplication is a convenience annotation combining three annotations: @Configuration, @EnableAutoConfiguration, and @ComponentScan.

**Complete Breakdown**:

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration  // = @Configuration
@EnableAutoConfiguration
@ComponentScan(excludeFilters = {
    @Filter(type = FilterType.CUSTOM, classes = TypeExcludeFilter.class),
    @Filter(type = FilterType.CUSTOM, classes = AutoConfigurationExcludeFilter.class)
})
public @interface SpringBootApplication {
    
    // Exclude auto-configurations
    Class<?>[] exclude() default {};
    
    // Exclude auto-configurations by name
    String[] excludeName() default {};
    
    // Component scan base packages
    String[] scanBasePackages() default {};
    
    // Component scan base package classes
    Class<?>[] scanBasePackageClasses() default {};
    
    // Proxy bean methods
    boolean proxyBeanMethods() default true;
}
```

**What Each Part Does**:

**1. @Configuration (via @SpringBootConfiguration)**:

```java
// Makes this class a source of bean definitions
@SpringBootApplication
public class MyApplication {
    
    // You can define beans directly
    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
    
    @Bean
    public ObjectMapper objectMapper() {
        ObjectMapper mapper = new ObjectMapper();
        mapper.configure(SerializationFeature.INDENT_OUTPUT, true);
        return mapper;
    }
}

// Equivalent to XML configuration:
// <bean id="restTemplate" class="org.springframework.web.client.RestTemplate"/>
```

**2. @EnableAutoConfiguration**:

```java
// Enables Spring Boot's auto-configuration mechanism
// Automatically configures beans based on:
// - Classpath contents (what JARs are present)
// - Property values (application.properties/yml)
// - Existing beans (what you've already defined)

// Examples of what gets auto-configured:

// If spring-boot-starter-web is present:
// ✅ Embedded Tomcat server
// ✅ DispatcherServlet
// ✅ Spring MVC configuration
// ✅ Jackson JSON converter
// ✅ Error page handling

// If spring-boot-starter-data-jpa is present:
// ✅ DataSource
// ✅ EntityManagerFactory
// ✅ TransactionManager
// ✅ JPA repositories

// If spring-boot-starter-security is present:
// ✅ Security filter chain
// ✅ Default authentication
// ✅ CSRF protection
// ✅ Security headers
```

**3. @ComponentScan**:

```java
// Scans for Spring components in:
// - Same package as @SpringBootApplication class
// - All sub-packages

package com.example.myapp; // Base package

@SpringBootApplication
public class MyApplication { 
    // Scans com.example.myapp.**
}

// ✅ These will be found:
package com.example.myapp.controller;
@RestController
public class UserController { } // Found

package com.example.myapp.service;
@Service
public class UserService { } // Found

package com.example.myapp.repository;
@Repository
public class UserRepository { } // Found

package com.example.myapp.config;
@Configuration
public class AppConfig { } // Found

package com.example.myapp.component;
@Component
public class MyComponent { } // Found

// ❌ This will NOT be found:
package com.another.package;
@Service
public class ExternalService { } // NOT found (outside scan path)
```

**Customization Options**:

**Option 1: Exclude Auto-Configurations**:

```java
@SpringBootApplication(exclude = {
    DataSourceAutoConfiguration.class,
    SecurityAutoConfiguration.class,
    RedisAutoConfiguration.class
})
public class MyApplication {
    // These auto-configurations won't run
}
```

**Option 2: Exclude by Name**:

```java
@SpringBootApplication(excludeName = {
    "org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration",
    "org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration"
})
public class MyApplication {
    // Useful when you don't have the class on compile classpath
}
```

**Option 3: Custom Component Scan Packages**:

```java
@SpringBootApplication(scanBasePackages = {
    "com.example.myapp",      // Main application
    "com.example.common",     // Shared library
    "com.example.integration" // Third-party integration
})
public class MyApplication {
    // Scans multiple package trees
}
```

**Option 4: Scan by Marker Classes**:

```java
@SpringBootApplication(scanBasePackageClasses = {
    MyApplication.class,      // Scans com.example.myapp.**
    CommonConfig.class,       // Scans com.example.common.**
    IntegrationConfig.class   // Scans com.example.integration.**
})
public class MyApplication {
    // Type-safe way to specify scan packages
}
```

**Option 5: Disable Proxy Bean Methods**:

```java
@SpringBootApplication(proxyBeanMethods = false)
public class MyApplication {
    // Faster startup, less memory
    // But @Bean methods won't be proxied
    // Multiple calls = multiple instances
    
    @Bean
    public MyService myService() {
        return new MyService(myRepository()); // New instance each call
    }
    
    @Bean
    public MyRepository myRepository() {
        return new MyRepository();
    }
}

// vs default (proxyBeanMethods = true):
@SpringBootApplication
public class MyApplication {
    
    @Bean
    public MyService myService() {
        return new MyService(myRepository()); // Returns singleton
    }
    
    @Bean
    public MyRepository myRepository() {
        return new MyRepository();
    }
}
```

**Equivalent Long Form**:

```java
// Instead of @SpringBootApplication, you could write:

@Configuration
@EnableAutoConfiguration
@ComponentScan(basePackages = "com.example.myapp")
public class MyApplication {
    
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

// Or even more verbose:

@SpringBootConfiguration
@EnableAutoConfiguration(exclude = {DataSourceAutoConfiguration.class})
@ComponentScan(
    basePackages = {"com.example.myapp"},
    excludeFilters = {
        @ComponentScan.Filter(
            type = FilterType.CUSTOM, 
            classes = TypeExcludeFilter.class
        )
    }
)
public class MyApplication {
    
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

**Common Patterns**:

**Pattern 1: Standard Application Structure**:

```java
package com.example.myapp;

@SpringBootApplication
public class MyApplication {
    
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

// Directory structure:
// com.example.myapp
// ├── MyApplication.java          (main class)
// ├── controller/
// │   ├── UserController.java
// │   └── ProductController.java
// ├── service/
// │   ├── UserService.java
// │   └── ProductService.java
// ├── repository/
// │   ├── UserRepository.java
// │   └── ProductRepository.java
// ├── model/
// │   ├── User.java
// │   └── Product.java
// └── config/
//     ├── SecurityConfig.java
//     └── DatabaseConfig.java
```

**Pattern 2: Multi-Module Application**:

```java
// Main application module
package com.example.myapp;

@SpringBootApplication(scanBasePackages = {
    "com.example.myapp",        // Main application
    "com.example.common",       // Common library
    "com.example.notifications", // Notification module
    "com.example.payments"      // Payment module
})
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

**Pattern 3: Profile-Specific Configuration**:

```java
@SpringBootApplication
public class MyApplication {
    
    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(MyApplication.class);
        
        // Set active profiles programmatically
        app.setAdditionalProfiles("dev", "debug");
        
        app.run(args);
    }
    
    @Bean
    @Profile("dev")
    public DataSource devDataSource() {
        // H2 in-memory database for development
        return new EmbeddedDatabaseBuilder()
            .setType(EmbeddedDatabaseType.H2)
            .build();
    }
    
    @Bean
    @Profile("prod")
    public DataSource prodDataSource() {
        // PostgreSQL for production
        HikariDataSource dataSource = new HikariDataSource();
        dataSource.setJdbcUrl("jdbc:postgresql://prod-db:5432/myapp");
        return dataSource;
    }
}
```

**Pattern 4: Custom Banner and Startup Logic**:

```java
@SpringBootApplication
public class MyApplication {
    
    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(MyApplication.class);
        
        // Disable banner
        app.setBannerMode(Banner.Mode.OFF);
        
        // Custom banner
        app.setBanner((environment, sourceClass, out) -> {
            out.println("=================================");
            out.println("   My Awesome Application v1.0   ");
            out.println("=================================");
        });
        
        // Add listeners
        app.addListeners(new ApplicationStartupListener());
        
        // Add initializers
        app.addInitializers(ctx -> {
            System.out.println("Custom initialization logic");
        });
        
        app.run(args);
    }
}
```

**Pattern 5: Conditional Component Scanning**:

```java
@SpringBootApplication
public class MyApplication {
    
    // Scan additional packages only in dev mode
    @Configuration
    @Profile("dev")
    @ComponentScan("com.example.myapp.dev")
    static class DevConfiguration { }
    
    // Scan additional packages only in prod mode
    @Configuration
    @Profile("prod")
    @ComponentScan("com.example.myapp.prod")
    static class ProdConfiguration { }
}
```

**What @SpringBootApplication Does Behind the Scenes**:

```java
// When you annotate with @SpringBootApplication
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

// Spring Boot does this:

// 1. Registers configuration class
context.register(MyApplication.class);

// 2. Scans for components starting from MyApplication's package
context.scan("com.example.myapp");

// 3. Processes auto-configuration
//    - Reads META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
//    - Loads ~200+ auto-configuration classes
//    - Evaluates @Conditional annotations for each
//    - Creates beans for configurations that match

// 4. Refreshes context
context.refresh();

// 5. Publishes ApplicationReadyEvent

// 6. Application is ready to serve requests
```

**Common Mistakes**:

**Mistake 1: Wrong Package Structure**:

```java
// ❌ Wrong - Main class too deep
package com.example.myapp.config;

@SpringBootApplication
public class MyApplication { }
// Only scans com.example.myapp.config and below
// Misses com.example.myapp.controller, service, etc.

// ✅ Correct - Main class at root
package com.example.myapp;

@SpringBootApplication
public class MyApplication { }
// Scans all of com.example.myapp.**
```

**Mistake 2: Conflicting Annotations**:

```java
// ❌ Wrong - Redundant and potentially conflicting
@SpringBootApplication
@Configuration  // Already included in @SpringBootApplication
@EnableAutoConfiguration  // Already included
@ComponentScan  // Already included
public class MyApplication { }

// ✅ Correct
@SpringBootApplication
public class MyApplication { }
```

### Q3: How to debug which auto-configurations are applied?

**Short Answer**: Use the `--debug` flag, Actuator endpoints, or application properties to see detailed condition evaluation reports showing which auto-configurations were applied and why others were skipped.

**Method 1: Debug Flag (Easiest)**:

```bash
# Command line
java -jar myapp.jar --debug

# Maven
mvn spring-boot:run -Dspring-boot.run.arguments=--debug

# Gradle
./gradlew bootRun --args='--debug'

# IDE (IntelliJ/Eclipse)
# Add to VM options: -Ddebug=true
# Or program arguments: --debug
```

**Or via application.properties**:

```properties
# application.properties
debug=true

# Or more specific
logging.level.org.springframework.boot.autoconfigure=DEBUG
```

@RestController
public class DebugController {
    
    @Autowired
    private ApplicationContext context;
    
    @GetMapping("/debug/conditions")
    public Map<String, Object> getConditionsReport() {
        ConditionEvaluationReport report = ConditionEvaluationReport
            .get(context.getEnvironment());
        
        Map<String, Object> result = new HashMap<>();
        
        // Positive matches (auto-configs that applied)
        Map<String, List<String>> positiveMatches = new HashMap<>();
        report.getConditionAndOutcomesBySource().forEach((source, conditions) -> {
            if (conditions.isFullMatch()) {
                List<String> messages = conditions.getConditionOutcomes()
                    .stream()
                    .map(ConditionOutcome::getMessage)
                    .collect(Collectors.toList());
                positiveMatches.put(source, messages);
            }
        });
        result.put("positiveMatches", positiveMatches);
        
        // Negative matches (auto-configs that were skipped)
        Map<String, List<String>> negativeMatches = new HashMap<>();
        report.getConditionAndOutcomesBySource().forEach((source, conditions) -> {
            if (!conditions.isFullMatch()) {
                List<String> messages = conditions.getConditionOutcomes()
                    .stream()
                    .filter(outcome -> !outcome.isMatch())
                    .map(ConditionOutcome::getMessage)
                    .collect(Collectors.toList());
                negativeMatches.put(source, messages);
            }
        });
        result.put("negativeMatches", negativeMatches);
        
        // Exclusions
        result.put("exclusions", report.getExclusions());
        
        // Unconditional classes (always applied)
        result.put("unconditionalClasses", report.getUnconditionalClasses());
        
        return result;
    }
    
    @GetMapping("/debug/beans")
    public String[] getAllBeans() {
        return context.getBeanDefinitionNames();
    }
    
    @GetMapping("/debug/beans/{name}")
    public Map<String, Object> getBeanInfo(@PathVariable String name) {
        Map<String, Object> info = new HashMap<>();
        
        Object bean = context.getBean(name);
        info.put("class", bean.getClass().getName());
        info.put("singleton", context.isSingleton(name));
        info.put("prototype", context.isPrototype(name));
        
        // Check if it's a proxy
        info.put("isAopProxy", AopUtils.isAopProxy(bean));
        info.put("isJdkDynamicProxy", AopUtils.isJdkDynamicProxy(bean));
        info.put("isCglibProxy", AopUtils.isCglibProxy(bean));
        
        if (AopUtils.isAopProxy(bean)) {
            info.put("targetClass", AopUtils.getTargetClass(bean).getName());
        }
        
        return info;
    }

@Component
public class AutoConfigurationAnalyzer implements ApplicationListener<ApplicationReadyEvent> {

        log.info("=".repeat(80));

        log.info("\n✅ APPLIED AUTO-CONFIGURATIONS:");
        long appliedCount = report.getConditionAndOutcomesBySource()
              .entrySet()
              .stream()
              .filter(e -> e.getValue().isFullMatch())
              .peek(e -> {
                  log.info("  📦 {}", getSimpleName(e.getKey()));
                  e.getValue().getConditionOutcomes().forEach(outcome ->
                      log.info("    ✓ {}", outcome.getMessage())
                  );
              })
              .count();
        log.info("Total applied: {}", appliedCount);
        
        // Analyze negative matches
        log.info("\n❌ SKIPPED AUTO-CONFIGURATIONS:");
        long skippedCount = report.getConditionAndOutcomesBySource()
              .entrySet()
              .stream()
              .filter(e -> !e.getValue().isFullMatch())
              .peek(e -> {
                  log.info("  📦 {}", getSimpleName(e.getKey()));
                  e.getValue().getConditionOutcomes()
                      .stream()
                      .filter(outcome -> !outcome.isMatch())
                      .forEach(outcome ->
                          log.info("    ✗ {}", outcome.getMessage())
                      );
              })
              .count();
        log.info("Total skipped: {}", skippedCount);
        
        // Exclusions
        if (!report.getExclusions().isEmpty()) {
            log.info("\n🚫 EXCLUDED:");
            report.getExclusions().forEach(exclusion ->
                log.info("  - {}", getSimpleName(exclusion))
            );
        }
        
        // Unconditional classes
        log.info("\n⚡ UNCONDITIONAL (always applied):");
        report.getUnconditionalClasses().forEach(unconditional ->
            log.info("  - {}", getSimpleName(unconditional))
        );
        
        log.info("\n" + "=".repeat(80));
    }
    
    private String getSimpleName(String fullClassName) {
        return fullClassName.substring(fullClassName.lastIndexOf('.') + 1);

**Common Debugging Scenarios**:

**Scenario 1: DataSource Not Auto-Configured**:

```
Problem: Expected DataSource bean but it wasn't created

Debug output:
  DataSourceAutoConfiguration:
    Did not match:
      - @ConditionalOnClass did not find required class 
        'javax.sql.DataSource' (OnClassCondition)

Analysis: Missing JDBC driver dependency

Solution:
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-jdbc</artifactId>
</dependency>

Or add specific driver:
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
</dependency>
```

**Scenario 2: Wrong Database Auto-Configured**:

```
Problem: H2 database auto-configured but you want PostgreSQL

Debug output:
  DataSourceAutoConfiguration matched
  H2ConsoleAutoConfiguration matched

Analysis: H2 is on classpath (often from test scope)

Solution:
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>test</scope> <!-- Only for tests -->
</dependency>

Or exclude H2 auto-configuration:
@SpringBootApplication(exclude = H2ConsoleAutoConfiguration.class)
```

**Scenario 3: Custom Bean Prevents Auto-Configuration**:

```
Problem: Redis auto-configuration not working

Debug output:
  RedisAutoConfiguration:
    Did not match:
      - @ConditionalOnMissingBean (types: RedisConnectionFactory) 
        found beans: customRedisConnectionFactory (OnBeanCondition)

Analysis: You defined custom RedisConnectionFactory, so auto-config backs off

Solution: Either remove custom bean or configure it properly
```

**Scenario 4: Property Disables Auto-Configuration**:

```
Problem: JPA repositories not auto-configured

Debug output:
  JpaRepositoriesAutoConfiguration:
    Did not match:
      - @ConditionalOnProperty (spring.data.jpa.repositories.enabled=false)
        (OnPropertyCondition)

Analysis: Property explicitly disabled repositories

Solution: Remove or change property:
# application.properties
spring.data.jpa.repositories.enabled=true
```

**Scenario 5: Missing Configuration Properties**:

```
Problem: MongoDB not connecting

Debug output:
  MongoAutoConfiguration matched but connection fails

Analysis: Auto-config applied but missing connection properties

Solution: Add properties:
# application.yml
spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/mydb
      # Or individual properties
      host: localhost
      port: 27017
      database: mydb
```

**Debug Checklist**:

```
When auto-configuration doesn't work, check:

1. ✅ Is the required dependency on the classpath?
   - Check pom.xml or build.gradle
   - Run: mvn dependency:tree

2. ✅ Are there conflicting dependencies?
   - Multiple versions of same library
   - Transitive dependency issues

3. ✅ Did you define a custom bean that conflicts?
   - Check for @Bean methods
   - Check @ConditionalOnMissingBean conditions

4. ✅ Are properties configured correctly?
   - Check application.properties/yml
   - Check environment variables
   - Use actuator /env endpoint

5. ✅ Is auto-configuration explicitly excluded?
   - Check @SpringBootApplication(exclude = ...)
   - Check spring.autoconfigure.exclude property

6. ✅ Are profiles active?
   - Some auto-configs are profile-specific
   - Check spring.profiles.active

7. ✅ Is the auto-configuration class registered?
   - Check META-INF/spring.factories
   - Check META-INF/spring/...AutoConfiguration.imports
```

### Q4: Custom starter creation

**Short Answer**: A custom starter is a module that bundles auto-configuration and dependencies for a specific feature. It follows naming convention `xxx-spring-boot-starter` and includes auto-configuration classes registered in `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`.

**Starter Structure**:

```
myservice-spring-boot-starter/
├── pom.xml
└── src/main/
    ├── java/com/example/myservice/
    │   ├── MyService.java
    │   ├── MyServiceProperties.java
    │   ├── MyServiceAutoConfiguration.java
    │   └── MyServiceHealthIndicator.java
    └── resources/
        └── META-INF/
            └── spring/
                └── org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

// MyService.java - The actual service
public class MyService {
    private final String apiKey;
    private final String endpoint;
    private final RestTemplate restTemplate;
    private final int timeout;
    
    public MyService(String apiKey, String endpoint, int timeout) {
        this.apiKey = apiKey;
        this.endpoint = endpoint;
        this.timeout = timeout;
        
        // Configure RestTemplate
        this.restTemplate = new RestTemplate();
        HttpComponentsClientHttpRequestFactory factory = 
            new HttpComponentsClientHttpRequestFactory();
        factory.setConnectTimeout(timeout);
        factory.setReadTimeout(timeout);
        restTemplate.setRequestFactory(factory);
    }
    
    public String getData(String id) {
        String url = endpoint + "/data/" + id + "?key=" + apiKey;
        return restTemplate.getForObject(url, String.class);
    }
    
    public void sendData(String id, Object data) {
        String url = endpoint + "/data/" + id + "?key=" + apiKey;
        restTemplate.postForLocation(url, data);
    }
    
    public boolean isHealthy() {
        try {
            String url = endpoint + "/health?key=" + apiKey;
            String response = restTemplate.getForObject(url, String.class);
            return "OK".equals(response);
        } catch (Exception e) {
            return false;
        }
    }
}

**Starter Best Practices**:

```
1. ✅ Naming Convention
   - xxx-spring-boot-starter (third-party)
   - spring-boot-starter-xxx (official Spring)

2. ✅ Use @ConditionalOnMissingBean
   - Allow users to override with custom beans

3. ✅ Provide sensible defaults
   - Users shouldn't HAVE to configure anything

4. ✅ Use @ConfigurationProperties
   - Type-safe configuration
   - IDE autocomplete support

5. ✅ Include metadata
   - spring-boot-configuration-processor
   - Generates metadata for IDE support

6. ✅ Make dependencies optional
   - Use <optional>true</optional>
   - Conditional auto-configuration

7. ✅ Document properties
   - JavaDoc on properties classes
   - README with examples

8. ✅ Provide health indicators
   - Integrate with Actuator

9. ✅ Support metrics
   - Integrate with Micrometer

10. ✅ Version compatibility
    - Test with multiple Spring Boot versions
```
