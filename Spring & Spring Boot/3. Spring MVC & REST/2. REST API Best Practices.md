## Overview

REST (Representational State Transfer) is an architectural style for designing networked applications. This guide covers best practices for building production-ready RESTful APIs with Spring Boot.

**Key Principle**: Use standard HTTP methods, status codes, and conventions to create intuitive, predictable APIs.

---

## 1. RESTful Design

### Resource Modeling

Design your API around resources (nouns), not actions (verbs).

**Good resource design:**

```
GET    /api/users              # Get all users
GET    /api/users/{id}         # Get specific user
POST   /api/users              # Create user
PUT    /api/users/{id}         # Update user (full)
PATCH  /api/users/{id}         # Update user (partial)
DELETE /api/users/{id}         # Delete user

GET    /api/users/{id}/orders  # Get user's orders
POST   /api/users/{id}/orders  # Create order for user
```

**Bad resource design (RPC-style):**

```
❌ POST /api/createUser
❌ GET  /api/getUserById?id=123
❌ POST /api/updateUserEmail
❌ POST /api/deleteUser
```

#### Resource Naming Conventions

|Convention|Example|Note|
|---|---|---|
|Use nouns|`/users`, `/orders`|Not verbs like `/getUsers`|
|Use plurals|`/users` not `/user`|Consistent collection names|
|Lowercase|`/users` not `/Users`|Case-sensitive URLs|
|Hyphens for multi-word|`/shipping-addresses`|Not underscores or camelCase|
|Hierarchical structure|`/users/{id}/orders`|Parent-child relationships|
|No trailing slash|`/users` not `/users/`|Consistent endpoints|

#### Resource Hierarchy Example

```java
@RestController
@RequestMapping("/api")
public class ResourceHierarchyExample {
    
    // Top-level resources
    @GetMapping("/users")
    public List<User> getUsers() { }
    
    @GetMapping("/users/{userId}")
    public User getUser(@PathVariable Long userId) { }
    
    // Nested resources (user's orders)
    @GetMapping("/users/{userId}/orders")
    public List<Order> getUserOrders(@PathVariable Long userId) { }
    
    @GetMapping("/users/{userId}/orders/{orderId}")
    public Order getUserOrder(
        @PathVariable Long userId,
        @PathVariable Long orderId
    ) { }
    
    // Alternative: Direct access if order ID is globally unique
    @GetMapping("/orders/{orderId}")
    public Order getOrder(@PathVariable Long orderId) { }
}
```

**When to use nested vs flat resources:**

```java
// Nested - when resource is dependent on parent
GET /users/{userId}/addresses        # User's addresses only
POST /users/{userId}/addresses       # Create address for this user

// Flat - when resource is independent
GET /orders/{orderId}                # Order exists independently
GET /products/{productId}            # Product exists independently

// Both - for flexibility
GET /addresses/{addressId}           # Direct access by ID
GET /users/{userId}/addresses        # Filter by user
```

### HTTP Methods (Proper Usage)

Use HTTP methods according to their semantics.

#### Method Characteristics

|Method|Safe|Idempotent|Request Body|Response Body|
|---|---|---|---|---|
|GET|✅ Yes|✅ Yes|❌ No|✅ Yes|
|POST|❌ No|❌ No|✅ Yes|✅ Yes|
|PUT|❌ No|✅ Yes|✅ Yes|✅ Yes|
|PATCH|❌ No|❌ No|✅ Yes|✅ Yes|
|DELETE|❌ No|✅ Yes|❌ No|✅ Optional|
|HEAD|✅ Yes|✅ Yes|❌ No|❌ No|
|OPTIONS|✅ Yes|✅ Yes|❌ No|✅ Yes|

**Definitions:**

- **Safe**: No side effects (read-only)
- **Idempotent**: Same result regardless of how many times executed

#### GET - Retrieve Resources

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    // Get all users (with pagination)
    @GetMapping
    public ResponseEntity<Page<User>> getUsers(
        @RequestParam(defaultValue = "0") int page,
        @RequestParam(defaultValue = "10") int size
    ) {
        Page<User> users = userService.findAll(PageRequest.of(page, size));
        return ResponseEntity.ok(users);
    }
    
    // Get specific user
    @GetMapping("/{id}")
    public ResponseEntity<User> getUser(@PathVariable Long id) {
        return userService.findById(id)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }
    
    // Search/filter
    @GetMapping("/search")
    public ResponseEntity<List<User>> searchUsers(
        @RequestParam(required = false) String name,
        @RequestParam(required = false) String email
    ) {
        List<User> users = userService.search(name, email);
        return ResponseEntity.ok(users);
    }
}
```

**Best practices:**

- Use query parameters for filtering, sorting, pagination
- Return 200 OK with data or 404 Not Found
- GET must be safe and idempotent (no side effects!)
- Cache-friendly (use ETag, Last-Modified headers)

#### POST - Create Resources

```java
@PostMapping
public ResponseEntity<User> createUser(@Valid @RequestBody User user) {
    User created = userService.create(user);
    
    // Return 201 Created with Location header
    URI location = ServletUriComponentsBuilder
        .fromCurrentRequest()
        .path("/{id}")
        .buildAndExpand(created.getId())
        .toUri();
    
    return ResponseEntity.created(location).body(created);
}

// Non-idempotent operations
@PostMapping("/{id}/reset-password")
public ResponseEntity<Void> resetPassword(@PathVariable Long id) {
    userService.resetPassword(id);
    return ResponseEntity.ok().build();
}
```

**Best practices:**

- Return 201 Created with Location header pointing to new resource
- Include created resource in response body
- Return 400 Bad Request for validation errors
- POST is NOT idempotent (multiple calls create multiple resources)

#### PUT - Full Update (Replace)

```java
@PutMapping("/{id}")
public ResponseEntity<User> updateUser(
    @PathVariable Long id,
    @Valid @RequestBody User user
) {
    if (!userService.exists(id)) {
        return ResponseEntity.notFound().build();
    }
    
    // PUT replaces entire resource
    user.setId(id);
    User updated = userService.update(user);
    
    return ResponseEntity.ok(updated);
}
```

**Best practices:**

- Require complete resource representation
- Replace entire resource
- Return 200 OK with updated resource
- Return 404 if resource doesn't exist
- PUT is idempotent (multiple identical calls = same result)

#### PATCH - Partial Update

```java
@PatchMapping("/{id}")
public ResponseEntity<User> partialUpdateUser(
    @PathVariable Long id,
    @RequestBody Map<String, Object> updates
) {
    if (!userService.exists(id)) {
        return ResponseEntity.notFound().build();
    }
    
    // PATCH updates only specified fields
    User updated = userService.partialUpdate(id, updates);
    
    return ResponseEntity.ok(updated);
}

// Alternative with DTO
public class UserPatchRequest {
    @Email
    private String email;
    private String phone;
    // Only fields that can be updated
}

@PatchMapping("/{id}")
public ResponseEntity<User> patchUser(
    @PathVariable Long id,
    @Valid @RequestBody UserPatchRequest patch
) {
    User updated = userService.patch(id, patch);
    return ResponseEntity.ok(updated);
}
```

**Best practices:**

- Accept partial resource representation
- Update only provided fields
- Return 200 OK with updated resource
- Use for incremental updates
- PATCH may or may not be idempotent (depends on implementation)

#### DELETE - Remove Resources

```java
@DeleteMapping("/{id}")
public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
    if (!userService.exists(id)) {
        return ResponseEntity.notFound().build();
    }
    
    userService.delete(id);
    
    // Return 204 No Content (no response body)
    return ResponseEntity.noContent().build();
}

// Soft delete (return deleted resource)
@DeleteMapping("/{id}/soft")
public ResponseEntity<User> softDeleteUser(@PathVariable Long id) {
    User deleted = userService.softDelete(id);
    return ResponseEntity.ok(deleted);
}
```

**Best practices:**

- Return 204 No Content on success (no body)
- Or return 200 OK with deleted resource
- Return 404 if resource doesn't exist
- DELETE is idempotent (deleting same resource multiple times = same result)

### Status Codes (When to Use Which)

#### Success Codes (2xx)

|Code|Name|When to Use|Example|
|---|---|---|---|
|200|OK|Successful GET, PUT, PATCH, DELETE with body|`GET /users/123`|
|201|Created|Successful POST (resource created)|`POST /users`|
|202|Accepted|Request accepted, processing async|`POST /users/123/process`|
|204|No Content|Successful DELETE, PUT, PATCH (no body)|`DELETE /users/123`|

```java
@RestController
@RequestMapping("/api/users")
public class StatusCodeExamples {
    
    // 200 OK
    @GetMapping("/{id}")
    public ResponseEntity<User> getUser(@PathVariable Long id) {
        User user = userService.findById(id);
        return ResponseEntity.ok(user);  // 200 OK
    }
    
    // 201 Created
    @PostMapping
    public ResponseEntity<User> createUser(@RequestBody User user) {
        User created = userService.create(user);
        URI location = /* ... */;
        return ResponseEntity.created(location).body(created);  // 201 Created
    }
    
    // 202 Accepted (async processing)
    @PostMapping("/{id}/export")
    public ResponseEntity<Void> exportData(@PathVariable Long id) {
        exportService.scheduleExport(id);
        return ResponseEntity.accepted().build();  // 202 Accepted
    }
    
    // 204 No Content
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        userService.delete(id);
        return ResponseEntity.noContent().build();  // 204 No Content
    }
}
```

#### Client Error Codes (4xx)

|Code|Name|When to Use|Example|
|---|---|---|---|
|400|Bad Request|Invalid request (validation error)|Invalid JSON, missing required fields|
|401|Unauthorized|Missing/invalid authentication|No Authorization header|
|403|Forbidden|Authenticated but not authorized|User not admin|
|404|Not Found|Resource doesn't exist|`GET /users/99999`|
|405|Method Not Allowed|HTTP method not supported|`DELETE /users` not allowed|
|409|Conflict|Resource conflict|Email already exists|
|422|Unprocessable Entity|Validation error (semantic)|Invalid business rule|
|429|Too Many Requests|Rate limit exceeded|Too many API calls|

```java
// 400 Bad Request
@PostMapping
public ResponseEntity<?> createUser(@Valid @RequestBody User user, 
                                    BindingResult result) {
    if (result.hasErrors()) {
        return ResponseEntity.badRequest().body(result.getAllErrors());
    }
    // ...
}

// 401 Unauthorized (Spring Security)
@GetMapping("/protected")
public ResponseEntity<String> protectedEndpoint() {
    // If no valid token → Spring Security returns 401
    return ResponseEntity.ok("Protected data");
}

// 403 Forbidden
@PreAuthorize("hasRole('ADMIN')")
@DeleteMapping("/all")
public ResponseEntity<Void> deleteAllUsers() {
    // If user not admin → 403 Forbidden
    userService.deleteAll();
    return ResponseEntity.noContent().build();
}

// 404 Not Found
@GetMapping("/{id}")
public ResponseEntity<User> getUser(@PathVariable Long id) {
    return userService.findById(id)
        .map(ResponseEntity::ok)
        .orElse(ResponseEntity.notFound().build());  // 404
}

// 409 Conflict
@PostMapping
public ResponseEntity<?> createUser(@RequestBody User user) {
    if (userService.emailExists(user.getEmail())) {
        return ResponseEntity.status(HttpStatus.CONFLICT)
            .body("Email already exists");  // 409 Conflict
    }
    // ...
}

// 422 Unprocessable Entity
@PostMapping
public ResponseEntity<?> createOrder(@RequestBody Order order) {
    if (order.getTotal().compareTo(BigDecimal.ZERO) <= 0) {
        return ResponseEntity.unprocessableEntity()
            .body("Order total must be positive");  // 422
    }
    // ...
}
```

#### Server Error Codes (5xx)

|Code|Name|When to Use|
|---|---|---|
|500|Internal Server Error|Unexpected server error|
|502|Bad Gateway|Invalid response from upstream|
|503|Service Unavailable|Service temporarily down|
|504|Gateway Timeout|Upstream timeout|

```java
// 500 Internal Server Error (caught by exception handler)
@GetMapping("/{id}")
public User getUser(@PathVariable Long id) {
    // If unexpected exception → 500
    return userService.findById(id);
}

// 503 Service Unavailable
@GetMapping("/health")
public ResponseEntity<Void> health() {
    if (!databaseService.isAvailable()) {
        return ResponseEntity.status(HttpStatus.SERVICE_UNAVAILABLE).build();
    }
    return ResponseEntity.ok().build();
}
```

### HATEOAS (Hypermedia as the Engine of Application State)

HATEOAS makes APIs self-documenting by including links to related resources.

**Add dependency:**

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-hateoas</artifactId>
</dependency>
```

**Basic HATEOAS example:**

```java
@RestController
@RequestMapping("/api/users")
public class UserHateoasController {
    
    @GetMapping("/{id}")
    public EntityModel<User> getUser(@PathVariable Long id) {
        User user = userService.findById(id);
        
        // Add links
        EntityModel<User> resource = EntityModel.of(user);
        
        // Self link
        resource.add(linkTo(methodOn(UserHateoasController.class)
            .getUser(id)).withSelfRel());
        
        // Related links
        resource.add(linkTo(methodOn(UserHateoasController.class)
            .getUserOrders(id)).withRel("orders"));
        
        resource.add(linkTo(methodOn(UserHateoasController.class)
            .getUserAddress(id)).withRel("address"));
        
        return resource;
    }
    
    @GetMapping("/{id}/orders")
    public CollectionModel<Order> getUserOrders(@PathVariable Long id) {
        // Implementation...
        return null;
    }
    
    @GetMapping("/{id}/address")
    public Address getUserAddress(@PathVariable Long id) {
        // Implementation...
        return null;
    }
}
```

## 2. Request/Response Handling

### HttpMessageConverter (Jackson, Gson, XML)

`HttpMessageConverter` converts HTTP request/response to/from Java objects.

#### How It Works

```
Request Flow:
1. Client sends JSON → DispatcherServlet
2. DispatcherServlet finds MappingJackson2HttpMessageConverter
3. Converter reads JSON → Java object
4. Object passed to @RequestBody parameter

Response Flow:
1. Controller returns Java object
2. DispatcherServlet finds MappingJackson2HttpMessageConverter
3. Converter writes object → JSON
4. JSON sent to client
```

#### Built-in Converters

|Converter|Handles|Media Type|
|---|---|---|
|`MappingJackson2HttpMessageConverter`|JSON (Jackson)|`application/json`|
|`GsonHttpMessageConverter`|JSON (Gson)|`application/json`|
|`Jaxb2RootElementHttpMessageConverter`|XML (JAXB)|`application/xml`|
|`StringHttpMessageConverter`|Plain text|`text/plain`|
|`ByteArrayHttpMessageConverter`|Binary data|`application/octet-stream`|
|`ResourceHttpMessageConverter`|Files|Various|

#### Jackson Configuration

**application.yml:**

```yaml
spring:
  jackson:
    serialization:
      indent-output: true              # Pretty print
      write-dates-as-timestamps: false # ISO-8601 dates
      fail-on-empty-beans: false       # Allow empty objects
    deserialization:
      fail-on-unknown-properties: false # Ignore unknown fields
      accept-single-value-as-array: true
    default-property-inclusion: non_null  # Omit null fields
    time-zone: UTC
    date-format: yyyy-MM-dd'T'HH:mm:ss.SSSZ
```

**Custom ObjectMapper:**

```java
@Configuration
public class JacksonConfig {
    
    @Bean
    public ObjectMapper objectMapper() {
        ObjectMapper mapper = new ObjectMapper();
        
        // Serialization features
        mapper.enable(SerializationFeature.INDENT_OUTPUT);
        mapper.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);
        
        // Deserialization features
        mapper.disable(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES);
        
        // Include only non-null fields
        mapper.setSerializationInclusion(JsonInclude.Include.NON_NULL);
        
        // Custom date format
        mapper.setDateFormat(new SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ss"));
        
        // Register custom modules
        mapper.registerModule(new JavaTimeModule());
        
        return mapper;
    }
}
```

**Jackson annotations:**

```java
public class User {

}
```

### Content Negotiation

Client specifies desired format via `Accept` header.

**Request:**

```http
GET /api/users/123
Accept: application/json
```

**Or:**

```http
GET /api/users/123
Accept: application/xml
```

**Configuration:**

```yaml
spring:
  mvc:
    contentnegotiation:
      favor-parameter: true  # Enable ?format=json
      parameter-name: format
      media-types:
        json: application/json
        xml: application/xml
```

**Usage:**

```
GET /api/users/123?format=json  # Returns JSON
GET /api/users/123?format=xml   # Returns XML
```

**Controller:**

```java
@GetMapping(produces = {"application/json", "application/xml"})
public User getUser(@PathVariable Long id) {
    // Spring automatically converts based on Accept header
    return userService.findById(id);
}
```

### Custom HttpMessageConverter

Create custom converter for specific format.

**Example: CSV converter**

```java
public class CsvHttpMessageConverter extends AbstractHttpMessageConverter<List<User>> {
    
    public CsvHttpMessageConverter() {
        super(new MediaType("text", "csv"));
    }
    
    @Override
    protected boolean supports(Class<?> clazz) {
        return List.class.isAssignableFrom(clazz);
    }
    
    @Override
    protected List<User> readInternal(
            Class<? extends List<User>> clazz,
            HttpInputMessage inputMessage
    ) throws IOException {
        // Read CSV from request
        BufferedReader reader = new BufferedReader(
            new InputStreamReader(inputMessage.getBody())
        );
        
        List<User> users = new ArrayList<>();
        String line;
        while ((line = reader.readLine()) != null) {
            String[] parts = line.split(",");
            User user = new User();
            user.setName(parts[0]);
            user.setEmail(parts[1]);
            users.add(user);
        }
        
        return users;
    }
    
    @Override
    protected void writeInternal(
            List<User> users,
            HttpOutputMessage outputMessage
    ) throws IOException {
        // Write CSV to response
        OutputStream out = outputMessage.getBody();
        PrintWriter writer = new PrintWriter(out);
        
        // Header
        writer.println("id,name,email");
        
        // Data
        for (User user : users) {
            writer.println(String.format("%d,%s,%s",
                user.getId(), user.getName(), user.getEmail()));
        }
        
        writer.flush();
    }
}
```

**Register converter:**

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void configureMessageConverters(
            List<HttpMessageConverter<?>> converters) {
        converters.add(new CsvHttpMessageConverter());
    }
}
```

**Usage:**

```java
@GetMapping(produces = "text/csv")
public List<User> getUsersCsv() {
    return userService.findAll();
}
```

### @ResponseBody and @RequestBody Deep Dive

#### @ResponseBody

Converts return value to HTTP response body.

```java
// Without @ResponseBody (traditional MVC)
@Controller
public class TraditionalController {
    
    @GetMapping("/home")
    public String home(Model model) {
        model.addAttribute("message", "Hello");
        return "home";  // Returns view name, not data
    }
}

// With @ResponseBody (REST API)
@Controller
public class RestController {
    
    @GetMapping("/users/{id}")
    @ResponseBody  // Converts User object to JSON
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);
    }
}

// @RestController = @Controller + @ResponseBody on all methods
@RestController
public class UserRestController {
    
    @GetMapping("/users/{id}")
    public User getUser(@PathVariable Long id) {
        // Automatically converted to JSON
        return userService.findById(id);
    }
}
```

**How @ResponseBody works:**

```java
1. Controller returns User object
2. Spring checks for @ResponseBody
3. Finds appropriate HttpMessageConverter (Jackson for JSON)
4. Converter serializes User → JSON
5. Sets Content-Type: application/json
6. Writes JSON to response body
```

#### @RequestBody

Converts HTTP request body to Java object.

```java
@PostMapping("/users")
public User createUser(@RequestBody User user) {
    // Spring automatically:
    // 1. Reads request body
    // 2. Finds HttpMessageConverter for Content-Type
    // 3. Deserializes JSON → User object
    // 4. Injects User into parameter
    
    return userService.save(user);
}
```

**Request:**

```http
POST /api/users
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john@example.com"
}
```

**With validation:**

```java
@PostMapping("/users")
public ResponseEntity<?> createUser(
        @Valid @RequestBody User user,
        BindingResult result) {
    
    if (result.hasErrors()) {
        return ResponseEntity.badRequest().body(result.getAllErrors());
    }
    
    User created = userService.save(user);
    return ResponseEntity.ok(created);
}
```

## 3. Exception Handling

### @ExceptionHandler (Controller-Specific)

Handle exceptions within a specific controller.

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return userService.findById(id)
            .orElseThrow(() -> new UserNotFoundException(id));
    }
    
    // Handle UserNotFoundException in this controller only
    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFound(
            UserNotFoundException ex) {
        
        ErrorResponse error = new ErrorResponse(
            HttpStatus.NOT_FOUND.value(),
            ex.getMessage(),
            Instant.now()
        );
        
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }
    
    // Handle validation errors
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map<String, String>> handleValidationErrors(
            MethodArgumentNotValidException ex) {
        
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors().forEach(error ->
            errors.put(error.getField(), error.getDefaultMessage())
        );
        
        return ResponseEntity.badRequest().body(errors);
    }
}
```

### @ControllerAdvice (Global Exception Handling)

Handle exceptions across all controllers.

```java
@ControllerAdvice
public class GlobalExceptionHandler {
    
    // Resource not found
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleResourceNotFound(
            ResourceNotFoundException ex,
            WebRequest request) {
        
        ErrorResponse error = ErrorResponse.builder()
            .timestamp(Instant.now())
            .status(HttpStatus.NOT_FOUND.value())
            .error("Not Found")
            .message(ex.getMessage())
            .path(request.getDescription(false))
            .build();
        
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }
    
    // Validation errors
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ValidationErrorResponse> handleValidationErrors(
            MethodArgumentNotValidException ex) {
        
        List<FieldError> fieldErrors = ex.getBindingResult()
            .getFieldErrors()
            .stream()
            .map(error -> new FieldError(
                error.getField(),
                error.getDefaultMessage(),
                error.getRejectedValue()
            ))
            .collect(Collectors.toList());
        
        ValidationErrorResponse response = new ValidationErrorResponse(
            HttpStatus.BAD_REQUEST.value(),
            "Validation failed",
            fieldErrors
        );
        
        return ResponseEntity.badRequest().body(response);
    }
    
    // Constraint violation (path variables, request params)
    @ExceptionHandler(ConstraintViolationException.class)
    public ResponseEntity<ErrorResponse> handleConstraintViolation(
            ConstraintViolationException ex) {
        
        String message = ex.getConstraintViolations()
            .stream()
            .map(ConstraintViolation::getMessage)
            .collect(Collectors.joining(", "));
        
        ErrorResponse error = new ErrorResponse(
            HttpStatus.BAD_REQUEST.value(),
            message,
            Instant.now()
        );
        
        return ResponseEntity.badRequest().body(error);
    }
    
    // Duplicate resource
    @ExceptionHandler(DuplicateResourceException.class)
    public ResponseEntity<ErrorResponse> handleDuplicateResource(
            DuplicateResourceException ex) {
        
        ErrorResponse error = new ErrorResponse(
            HttpStatus.CONFLICT.value(),
            ex.getMessage(),
            Instant.now()
        );
        
        return ResponseEntity.status(HttpStatus.CONFLICT).body(error);
    }
    
    // Unauthorized access
    @ExceptionHandler(AccessDeniedException.class)
    public ResponseEntity<ErrorResponse> handleAccessDenied(
            AccessDeniedException ex) {
        
        ErrorResponse error = new ErrorResponse(
            HttpStatus.FORBIDDEN.value(),
            "Access denied: " + ex.getMessage(),
            Instant.now()
        );
        
        return ResponseEntity.status(HttpStatus.FORBIDDEN).body(error);
    }
    
    // Generic exception (catch-all)
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleGenericException(
            Exception ex,
            WebRequest request) {
        
        log.error("Unexpected error", ex);
        
        ErrorResponse error = new ErrorResponse(
            HttpStatus.INTERNAL_SERVER_ERROR.value(),
            "An unexpected error occurred",
            Instant.now()
        );
        
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(error);
    }
}
```

### ResponseEntityExceptionHandler

```java
@ControllerAdvice
public class CustomExceptionHandler extends ResponseEntityExceptionHandler {
    
    // Override built-in exception handling
    @Override
    protected ResponseEntity<Object> handleMethodArgumentNotValid(
            MethodArgumentNotValidException ex,
            HttpHeaders headers,
            HttpStatusCode status,
            WebRequest request) {
        
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors().forEach(error ->
            errors.put(error.getField(), error.getDefaultMessage())
        );
        
        ErrorResponse errorResponse = ErrorResponse.builder()
            .timestamp(Instant.now())
            .status(status.value())
            .error("Validation Failed")
            .message("Input validation error")
            .errors(errors)
            .build();
        
        return new ResponseEntity<>(errorResponse, headers, status);
    }
    
    @Override
    protected ResponseEntity<Object> handleHttpMessageNotReadable(
            HttpMessageNotReadableException ex,
            HttpHeaders headers,
            HttpStatusCode status,
            WebRequest request) {
        
        ErrorResponse error = ErrorResponse.builder()
            .timestamp(Instant.now())
            .status(HttpStatus.BAD_REQUEST.value())
            .error("Malformed JSON")
            .message("Request body is not valid JSON")
            .build();
        
        return new ResponseEntity<>(error, HttpStatus.BAD_REQUEST);
    }
    
    @Override
    protected ResponseEntity<Object> handleHttpRequestMethodNotSupported(
            HttpRequestMethodNotSupportedException ex,
            HttpHeaders headers,
            HttpStatusCode status,

        ErrorResponse error = ErrorResponse.builder()
            .timestamp(Instant.now())

            .error("Method Not Allowed")
            .message(ex.getMessage())
            .build();
        
        return new ResponseEntity<>(error, HttpStatus.METHOD_NOT_ALLOWED);
    }
    
    // Add custom exception handlers
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleResourceNotFound(
            ResourceNotFoundException ex) {
        
        ErrorResponse error = ErrorResponse.builder()
            .timestamp(Instant.now())
            .status(HttpStatus.NOT_FOUND.value())
            .error("Not Found")
            .message(ex.getMessage())
            .build();

    }
```

### Custom Validators

**Step 1: Create annotation**

```java
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneNumberValidator.class)
public @interface ValidPhoneNumber {
    String message() default "Invalid phone number format";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}
```

**Step 2: Create validator**

```java
public class PhoneNumberValidator 
        implements ConstraintValidator<ValidPhoneNumber, String> {
    
    private static final Pattern PHONE_PATTERN = 
        Pattern.compile("^\\+?[1-9]\\d{1,14}$");
    
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null || value.isEmpty()) {
            return true;  // Use @NotBlank for null check
        }
        
        return PHONE_PATTERN.matcher(value).matches();
    }
}
```

**Step 3: Use it**

```java
public class User {
    
    @ValidPhoneNumber
    private String phone;
}
```

**Example: Password strength validator**

```java
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PasswordStrengthValidator.class)
public @interface StrongPassword {
    String message() default "Password must be at least 8 characters with uppercase, lowercase, and number";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

public class PasswordStrengthValidator 
        implements ConstraintValidator<StrongPassword, String> {
    
    @Override
    public boolean isValid(String password, ConstraintValidatorContext context) {
        if (password == null) {
            return false;
        }
        
        // At least 8 characters
        if (password.length() < 8) {
            return false;
        }
        
        // Contains uppercase
        if (!password.matches(".*[A-Z].*")) {
            return false;
        }
        
        // Contains lowercase
        if (!password.matches(".*[a-z].*")) {
            return false;
        }
        
        // Contains number
        if (!password.matches(".*\\d.*")) {
            return false;
        }
        
        return true;
    }
}
```

### Validation Groups

Use groups to apply different validations for different scenarios.

**Define groups:**

```java
public interface CreateValidation {}
public interface UpdateValidation {}
```

**Apply to fields:**

```java
public class User {
    
    @Null(groups = CreateValidation.class)
    @NotNull(groups = UpdateValidation.class)
    private Long id;
    
    @NotBlank(groups = {CreateValidation.class, UpdateValidation.class})
    @Size(min = 2, max = 100, groups = {CreateValidation.class, UpdateValidation.class})
    private String name;
    
    @NotBlank(groups = CreateValidation.class)
    @StrongPassword(groups = CreateValidation.class)
    private String password;
    
    // Password not required for updates
}
```

**Use in controller:**

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    // Validate with CreateValidation group
    @PostMapping
    public User createUser(
            @Validated(CreateValidation.class) @RequestBody User user) {
        // ID must be null
        // Password required and must be strong
        return userService.save(user);
    }
    
    // Validate with UpdateValidation group
    @PutMapping("/{id}")
    public User updateUser(
            @PathVariable Long id,
            @Validated(UpdateValidation.class) @RequestBody User user) {
        // ID must not be null
        // Password not required
        user.setId(id);
        return userService.update(user);
    }
}
```

**Ordered groups:**

```java
@GroupSequence({BasicValidation.class, AdvancedValidation.class})
public interface OrderedValidation {}

public class Product {
    
    @NotBlank(groups = BasicValidation.class)
    private String name;
    
    @UniqueProductName(groups = AdvancedValidation.class)
    // Only validated if BasicValidation passes
    private String name;
}
```

## Best Practices Summary

### RESTful Design ✅

1. Use nouns for resources, not verbs
2. Use HTTP methods correctly (GET, POST, PUT, PATCH, DELETE)
3. Return appropriate status codes
4. Use hierarchical URLs for relationships
5. Implement HATEOAS for discoverability

### Request/Response ✅

1. Use consistent JSON naming (camelCase or snake_case)
2. Configure Jackson properly
3. Support content negotiation
4. Version your API (in URL or header)
5. Use pagination for collections

### Exception Handling ✅

1. Use @ControllerAdvice for global handling
2. Return consistent error format
3. Include meaningful error messages
4. Log server errors (5xx)
5. Use RFC 7807 Problem Details

### Validation ✅

1. Validate at controller layer (@Valid)
2. Validate at service layer (method validation)
3. Use custom validators for complex rules
4. Use validation groups when needed
5. Return field-specific error messages
