## Overview

Aspect-Oriented Programming (AOP) is a powerful paradigm that enables you to modularize cross-cutting concerns that span multiple classes and methods. Spring AOP provides a proxy-based implementation that integrates seamlessly with the Spring container.

```
┌─────────────────────────────────────────────────────────────────┐
│                    SPRING AOP ARCHITECTURE                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Cross-Cutting Concerns                                         │
│  ├── Logging                                                    │
│  ├── Security                                                   │
│  ├── Transactions                                               │
│  ├── Caching                                                    │
│  └── Performance Monitoring                                     │
│       ↓                                                         │
│  AOP Proxy (Wraps Target Object)                               │
│  ┌─────────────────────────────────────┐                       │
│  │ Before Advice                        │                       │
│  │   ↓                                  │                       │
│  │ Target Method Execution              │                       │
│  │   ↓                                  │                       │
│  │ AfterReturning / AfterThrowing       │                       │
│  │   ↓                                  │                       │
│  │ After (Finally) Advice               │                       │
│  └─────────────────────────────────────┘                       │
│                                                                 │
│  Proxy Types:                                                   │
│  1. JDK Dynamic Proxy (Interface-based)                        │
│  2. CGLIB Proxy (Class-based)                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 1. AOP Fundamentals

### 1.1 Cross-Cutting Concerns

**Problems AOP Solves:**

```java
/**
 * WITHOUT AOP - Code Duplication and Tangled Concerns
 */
@Service
public class UserService {
    
    private static final Logger logger = LoggerFactory.getLogger(UserService.class);
    
    public User createUser(User user) {
        // Logging concern
        logger.info("Creating user: {}", user.getUsername());
        long startTime = System.currentTimeMillis();
        
        try {
            // Security concern
            if (!SecurityContextHolder.getContext().getAuthentication().isAuthenticated()) {
                throw new SecurityException("Not authenticated");
            }
            
            // Actual business logic (buried in cross-cutting concerns)
            User savedUser = userRepository.save(user);
            
            // Caching concern
            cache.put("user:" + savedUser.getId(), savedUser);
            
            // Logging concern
            long duration = System.currentTimeMillis() - startTime;
            logger.info("User created in {}ms", duration);
            
            return savedUser;
            
        } catch (Exception e) {
            // Error handling concern
            logger.error("Error creating user", e);
            throw e;
        }
    }
    
    // Same cross-cutting concerns repeated in EVERY method!
    public User updateUser(User user) {
        logger.info("Updating user: {}", user.getUsername());
        // ... repeat all the same concerns
    }
}

/**
 * WITH AOP - Clean Separation of Concerns
 */
@Service
public class UserService {
    
    // Only business logic - clean and focused!
    public User createUser(User user) {
        return userRepository.save(user);
    }
    
    public User updateUser(User user) {
        return userRepository.save(user);
    }
}

@Aspect
@Component
public class LoggingAspect {
    // All logging in one place
}

@Aspect
@Component
public class SecurityAspect {
    // All security in one place
}

@Aspect
@Component
public class PerformanceAspect {
    // All performance monitoring in one place
}
```

**Cross-Cutting Concerns:**

```
Logging:
  ✓ Method entry/exit logging
  ✓ Parameter logging
  ✓ Exception logging

Security:
  ✓ Authorization checks
  ✓ Authentication validation
  ✓ Role-based access control

Transaction Management:
  ✓ Begin/commit/rollback transactions
  ✓ Isolation levels
  ✓ Propagation behavior

Performance Monitoring:
  ✓ Execution time tracking
  ✓ Method call counting
  ✓ Resource usage monitoring

Caching:
  ✓ Cache lookups
  ✓ Cache updates
  ✓ Cache invalidation

Error Handling:
  ✓ Exception translation
  ✓ Retry logic
  ✓ Circuit breaking
```

### 1.2 AOP Terminology

**Visual Representation:**

```
┌─────────────────────────────────────────────────────────────┐
│                  AOP TERMINOLOGY                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Target Object:     UserService (the actual bean)          │
│  Proxy:             UserService$$EnhancerBySpringCGLIB     │
│                                                             │
│  Aspect:            @Aspect class containing advice         │
│  Join Point:        createUser() execution                 │
│  Pointcut:          "execution(* com.example..*Service.*(..))" │
│  Advice:            @Before, @After, @Around methods        │
│  Weaving:           Creating proxy at runtime              │
│                                                             │
│  ┌──────────────────────────────────────────┐              │
│  │ Client calls: userService.createUser()   │              │
│  │           ↓                               │              │
│  │ Proxy intercepts call                    │              │
│  │           ↓                               │              │
│  │ Execute @Before advice                   │              │
│  │           ↓                               │              │
│  │ Call actual method on target object      │              │
│  │           ↓                               │              │
│  │ Execute @After advice                    │              │
│  │           ↓                               │              │
│  │ Return result to client                  │              │
│  └──────────────────────────────────────────┘              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**Definitions:**

```java
/**
 * 1. ASPECT
 * A modularization of a cross-cutting concern
 */
@Aspect
@Component
public class LoggingAspect {
    // This entire class is an ASPECT
}

/**
 * 2. JOIN POINT
 * A point in the execution of the program (e.g., method execution)
 */
// Join points in Spring AOP:
// - Method execution (most common)
// - Constructor execution (limited support)
// NOT supported: field access, method call (only execution)

/**
 * 3. POINTCUT
 * A predicate that matches join points
 */
@Pointcut("execution(* com.example.service.*.*(..))")
public void serviceMethods() {}
// This pointcut matches all methods in service package

/**
 * 4. ADVICE
 * Action taken at a join point
 */
@Before("serviceMethods()")  // This method is ADVICE
public void logBefore(JoinPoint joinPoint) {
    System.out.println("Before: " + joinPoint.getSignature());
}

/**
 * 5. TARGET OBJECT
 * The actual object being advised (proxied)
 */
@Service
public class UserService {  // This is the TARGET OBJECT
    public void createUser() { }
}

/**
 * 6. AOP PROXY
 * The object created by AOP framework to implement aspect contracts
 */
// Spring creates: UserService$$EnhancerBySpringCGLIB
// This proxy wraps the target object

/**
 * 7. WEAVING
 * Linking aspects with other application types to create advised objects
 * In Spring AOP: Runtime weaving via proxies
 */
```

### 1.3 Proxy-Based AOP

**JDK Dynamic Proxy vs CGLIB:**

```java
/**
 * JDK Dynamic Proxy (Interface-based)
 */
public interface UserService {
    void createUser(User user);
}

@Service
public class UserServiceImpl implements UserService {
    @Override
    public void createUser(User user) {
        // Implementation
    }
}

/**
 * Spring creates JDK dynamic proxy:
 * 1. UserService interface exists
 * 2. Proxy implements same interface
 * 3. Delegates to actual implementation
 * 
 * Proxy structure:
 * class $Proxy0 implements UserService {
 *     private UserServiceImpl target;
 *     
 *     public void createUser(User user) {
 *         // Execute before advice
 *         target.createUser(user);
 *         // Execute after advice
 *     }
 * }
 */

/**
 * CGLIB Proxy (Class-based)
 */
@Service
public class UserService {  // No interface
    public void createUser(User user) {
        // Implementation
    }
}

/**
 * Spring creates CGLIB proxy:
 * 1. No interface available
 * 2. Creates subclass of UserService
 * 3. Overrides methods to add advice
 * 
 * Proxy structure:
 * class UserService$$EnhancerBySpringCGLIB extends UserService {
 *     
 *     @Override
 *     public void createUser(User user) {
 *         // Execute before advice
 *         super.createUser(user);
 *         // Execute after advice
 *     }
 * }
 */
```

**Comparison:**

```
┌──────────────────────────────────────────────────────────────┐
│         JDK DYNAMIC PROXY vs CGLIB                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Aspect          JDK Proxy         CGLIB Proxy              │
│  ────────────────────────────────────────────────────────   │
│  Requires        Interface         No requirement           │
│  Mechanism       Implements        Extends class            │
│  Performance     Slightly faster   Slightly slower          │
│  Final classes   Works              Cannot proxy            │
│  Final methods   Works              Cannot proxy            │
│  Private methods Cannot proxy      Cannot proxy            │
│  Constructor     Called once       Called twice            │
│  Dependencies    JDK only          CGLIB library            │
│  Spring default  If interface      If no interface         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**Force CGLIB Proxy:**

```java
// Option 1: @EnableAspectJAutoProxy
@Configuration
@EnableAspectJAutoProxy(proxyTargetClass = true)  // Force CGLIB
public class AppConfig {
}

// Option 2: Application properties
// application.properties
spring.aop.proxy-target-class=true

// Option 3: @Scope with CGLIB
@Service
@Scope(proxyMode = ScopedProxyMode.TARGET_CLASS)  // CGLIB proxy
public class UserService {
}
```

**Limitations of Both:**

```java
/**
 * Limitations of Proxy-Based AOP
 */

// 1. Cannot intercept private methods
@Service
public class UserService {
    
    public void publicMethod() {
        privateMethod();  // AOP won't intercept this call
    }
    
    private void privateMethod() {
        // Advice won't apply
    }
}

// 2. Cannot intercept final methods (CGLIB only)
@Service
public class UserService {
    
    public final void finalMethod() {
        // CGLIB cannot override final methods
        // Advice won't apply
    }
}

// 3. Cannot intercept final classes (CGLIB only)
@Service
public final class UserService {  // CGLIB cannot extend final class
    public void method() {
        // If no interface, AOP won't work
    }
}

// 4. Cannot intercept static methods
@Service
public class UserService {
    
    public static void staticMethod() {
        // Proxy cannot intercept static methods
        // Advice won't apply
    }
}
```

### 1.4 Self-Invocation Problem

**The Problem:**

```java
@Service
public class UserService {
    
    @Transactional
    public void publicMethod() {
        // Transaction starts here
        
        this.privateMethod();  // ⚠️ SELF-INVOCATION - bypasses proxy!
        
        // Transaction ends here
    }
    
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    private void privateMethod() {
        // This transaction annotation is IGNORED!
        // Called directly, not through proxy
    }
}

/**
 * Why it happens:
 * 
 * Client → Proxy → publicMethod()
 *                      ↓
 *                  this.privateMethod()  ← Direct call, bypasses proxy!
 * 
 * The 'this' refers to the actual object, not the proxy
 * So advice doesn't execute for privateMethod()
 */
```

**Demonstration:**

```java
@Aspect
@Component
public class LoggingAspect {
    
    @Before("execution(* com.example.UserService.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        System.out.println("Before: " + joinPoint.getSignature().getName());
    }
}

@Service
public class UserService {
    
    public void method1() {
        System.out.println("Method1 executing");
        this.method2();  // Self-invocation
    }
    
    public void method2() {
        System.out.println("Method2 executing");
    }
}

// Calling method1():
// Before: method1          ← Advice executes
// Method1 executing
// Method2 executing        ← Advice DOES NOT execute for method2!

// Calling method2() directly:
// Before: method2          ← Advice executes
// Method2 executing
```

**Solutions:**

**Solution 1: Self-Injection (Spring 4.3+)**

```java
@Service
public class UserService {
    
    @Autowired
    private UserService self;  // Inject the proxy
    
    @Transactional
    public void publicMethod() {
        self.privateMethod();  // ✓ Goes through proxy
    }
    
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void privateMethod() {
        // Transaction annotation now works!
    }
}
```

**Solution 2: ApplicationContext Lookup**

```java
@Service
public class UserService implements ApplicationContextAware {
    
    private ApplicationContext applicationContext;
    
    @Override
    public void setApplicationContext(ApplicationContext applicationContext) {
        this.applicationContext = applicationContext;
    }
    
    @Transactional
    public void publicMethod() {
        UserService proxy = applicationContext.getBean(UserService.class);
        proxy.privateMethod();  // ✓ Goes through proxy
    }
    
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void privateMethod() {
        // Transaction annotation now works!
    }
}
```

**Solution 3: Refactor into Separate Class**

```java
@Service
public class UserService {
    
    @Autowired
    private UserHelper userHelper;
    
    @Transactional
    public void publicMethod() {
        userHelper.helperMethod();  // ✓ Different bean, goes through proxy
    }
}

@Service
public class UserHelper {
    
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void helperMethod() {
        // Transaction annotation works!
    }
}
```

**Solution 4: AspectJ Weaving (Load-Time or Compile-Time)**

```java
// Use AspectJ instead of Spring AOP
// Weaves aspects directly into bytecode
// No proxy, no self-invocation problem
// But requires AspectJ configuration
```

## 2. Pointcut Expressions

### 2.1 execution() - Most Common

**Basic Syntax:**

```
execution(modifiers? return-type declaring-type? method-name(parameters) throws?)

Components:
  modifiers:       public, protected (optional)
  return-type:     void, String, * (wildcard) (required)
  declaring-type:  com.example.UserService (optional)
  method-name:     save, find*, * (wildcard) (required)
  parameters:      (), (..), (*), (String, *) (required)
  throws:          throws Exception (optional)
```

**Examples:**

```java
@Aspect
@Component
public class PointcutExamples {
    
    // 1. Any public method
    @Pointcut("execution(public * *(..))")
    public void anyPublicMethod() {}
    
    // 2. Any method in UserService
    @Pointcut("execution(* com.example.UserService.*(..))")
    public void anyUserServiceMethod() {}
    
    // 3. Methods starting with "get"
    @Pointcut("execution(* get*(..))")
    public void anyGetter() {}
    
    // 4. Methods in service package
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void anyServiceMethod() {}
    
    // 5. Methods in service package and subpackages
    @Pointcut("execution(* com.example.service..*.*(..))")
    public void anyServiceMethodIncludingSubpackages() {}
    
    // 6. Methods returning String
    @Pointcut("execution(String *(..))")
    public void methodsReturningString() {}
    
    // 7. Methods with specific parameters
    @Pointcut("execution(* *(String, int))")
    public void methodsWithStringAndInt() {}
    
    // 8. Methods with any number of parameters
    @Pointcut("execution(* *(..))")
    public void methodsWithAnyParameters() {}
    
    // 9. Methods with at least one parameter
    @Pointcut("execution(* *(*))")  // One parameter
    public void methodsWithOneParameter() {}
    
    @Pointcut("execution(* *(*,*))")  // Two parameters
    public void methodsWithTwoParameters() {}
    
    @Pointcut("execution(* *(*,..))")  // One or more parameters
    public void methodsWithAtLeastOneParameter() {}
    
    // 10. Methods throwing specific exception
    @Pointcut("execution(* *(..) throws java.io.IOException)")
    public void methodsThrowingIOException() {}
    
    // 11. Complete example
    @Pointcut("execution(public User com.example.service.UserService.findById(Long))")
    public void findUserById() {}
    
    // 12. Multiple return types
    @Pointcut("execution(User+ *(..))")  // User or subtype
    public void methodsReturningUserOrSubtype() {}
}
```

### 2.3 Annotation-Based Pointcuts

```java
/**
 * Custom annotations for pointcuts
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Loggable {
    String value() default "";
}

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Auditable {
}

@Aspect
@Component
public class AnnotationPointcuts {
    
    // 1. Methods annotated with @Loggable
    @Pointcut("@annotation(com.example.Loggable)")
    public void loggableMethods() {}
    
    @Before("loggableMethods()")
    public void logMethod(JoinPoint joinPoint) {
        System.out.println("Logging: " + joinPoint.getSignature());
    }
    
    // 2. Methods in classes annotated with @Auditable
    @Pointcut("@within(com.example.Auditable)")
    public void auditableClasses() {}
    
    // 3. Methods with @Transactional
    @Pointcut("@annotation(org.springframework.transaction.annotation.Transactional)")
    public void transactionalMethods() {}
    
    // 4. Accessing annotation values
    @Before("@annotation(loggable)")
    public void logWithAnnotationValue(JoinPoint joinPoint, Loggable loggable) {
        System.out.println("Log level: " + loggable.value());
    }
}

// Usage
@Service
@Auditable
public class UserService {
    
    @Loggable("INFO")
    public void createUser(User user) {
        // Method will be logged
    }
}
```

### 2.5 Combining Pointcuts

```java
@Aspect
@Component
public class CombinedPointcuts {
    
    // Define reusable pointcuts
    @Pointcut("execution(* com.example.service..*.*(..))")
    public void serviceLayer() {}
    
    @Pointcut("execution(* com.example.repository..*.*(..))")
    public void repositoryLayer() {}
    
    @Pointcut("execution(public * *(..))")
    public void publicMethod() {}
    
    @Pointcut("@annotation(org.springframework.transaction.annotation.Transactional)")
    public void transactionalMethod() {}
    
    // 1. AND operator (&&)
    @Before("serviceLayer() && publicMethod()")
    public void publicServiceMethods(JoinPoint joinPoint) {
        // Applies to public methods in service layer only
    }
    
    // 2. OR operator (||)
    @Before("serviceLayer() || repositoryLayer()")
    public void dataAccessLayers(JoinPoint joinPoint) {
        // Applies to service OR repository layer
    }
    
    // 3. NOT operator (!)
    @Before("serviceLayer() && !transactionalMethod()")
    public void nonTransactionalServiceMethods(JoinPoint joinPoint) {
        // Service methods that are NOT transactional
    }
    
    // 4. Complex combination
    @Around("(serviceLayer() || repositoryLayer()) && publicMethod() && !transactionalMethod()")
    public Object monitorNonTransactionalDataAccess(ProceedingJoinPoint pjp) throws Throwable {
        // Public, non-transactional methods in service or repository layer
        long start = System.currentTimeMillis();
        try {
            return pjp.proceed();
        } finally {
            long duration = System.currentTimeMillis() - start;
            System.out.println(pjp.getSignature() + " took " + duration + "ms");
        }
    }
    
    // 5. Combining with parameters
    @Before("serviceLayer() && args(user, ..)")
    public void serviceMethodsWithUserParameter(User user) {
        System.out.println("Service method called with user: " + user.getUsername());
    }
}
```

## 3. Advice Types

### 3.1 @Before Advice

```java
@Aspect
@Component
public class BeforeAdviceExamples {
    
    /**
     * @Before - Executes before the method
     * Cannot prevent method execution (unless throws exception)
     */
    
    // Basic @Before
    @Before("execution(* com.example.service..*.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        System.out.println("Before: " + joinPoint.getSignature());
    }
    
    // Access method parameters
    @Before("execution(* saveUser(..)) && args(user)")
    public void validateUser(User user) {
        if (user.getUsername() == null) {
            throw new IllegalArgumentException("Username cannot be null");
        }
    }
    
    // Access target object
    @Before("execution(* com.example..*.*(..))")
    public void beforeMethod(JoinPoint joinPoint) {
        Object target = joinPoint.getTarget();
        Object proxy = joinPoint.getThis();
        
        System.out.println("Target class: " + target.getClass().getName());
        System.out.println("Proxy class: " + proxy.getClass().getName());
        
        Object[] args = joinPoint.getArgs();
        System.out.println("Arguments: " + Arrays.toString(args));
    }
    
    // Pre-condition checking
    @Before("@annotation(requiresAuth)")
    public void checkAuthentication(RequiresAuth requiresAuth) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth == null || !auth.isAuthenticated()) {
            throw new SecurityException("Not authenticated");
        }
        
        String requiredRole = requiresAuth.role();
        if (!auth.getAuthorities().contains(new SimpleGrantedAuthority(requiredRole))) {
            throw new SecurityException("Insufficient permissions");
        }
    }
}
```

### 3.3 @AfterReturning Advice

```java
@Aspect
@Component
public class AfterReturningAdviceExamples {
    
    /**
     * @AfterReturning - Executes after successful method return
     * Can access return value
     * Does NOT execute if method throws exception
     */

    @AfterReturning("execution(* find*(..))")

    @AfterReturning(
        pointcut = "execution(* findUser(..))",
        returning = "user"

            cache.put("user:" + user.getId(), user);
            System.out.println("Cached user: " + user.getUsername());
        }
    }
    
    // Modify return value (doesn't work - return value already determined)
    @AfterReturning(
        pointcut = "execution(* getPrice(..))",
        returning = "price"
    )
    public void applyDiscount(BigDecimal price) {
        // ⚠️ This DOES NOT modify the return value
        // price = price.multiply(new BigDecimal("0.9"));
        // Use @Around if you need to modify return value
    }
    
    // Logging successful operations
    @AfterReturning(
        pointcut = "execution(* save*(..)) && args(entity)",
        returning = "saved"
    )
    public void logSuccessfulSave(Object entity, Object saved) {
        log.info("Successfully saved {} with id {}", 
            entity.getClass().getSimpleName(),
            extractId(saved));
    }
}
```

### 3.4 @AfterThrowing Advice

```java
@Aspect
```

### 3.5 @Around Advice - Most Powerful

```java
@Aspect
@Component
public class AroundAdviceExamples {
    
    /**
     * @Around - Most powerful advice type
     * Can:
     * - Execute code before and after
     * - Modify parameters
     * - Modify return value
     * - Suppress exceptions
     * - Prevent method execution
     * 
     * Must call pjp.proceed() to execute target method
     */
    
    // Basic @Around
    @Around("execution(* com.example.service..*.*(..))")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        System.out.println("Before method");
        
        try {
            Object result = pjp.proceed();  // Execute target method
            System.out.println("After successful execution");
            return result;
        } catch (Exception e) {
            System.out.println("After exception");
            throw e;
        } finally {
            System.out.println("After (finally)");
        }
    }
    
    // Performance monitoring
    @Around("execution(* com.example.service..*.*(..))")
    public Object measureExecutionTime(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        String methodName = pjp.getSignature().toShortString();
        
        try {
            Object result = pjp.proceed();
            long duration = System.currentTimeMillis() - start;
            
            if (duration > 1000) {
                log.warn("{} took {}ms", methodName, duration);
            } else {
                log.debug("{} took {}ms", methodName, duration);
            }
            
            return result;
        } catch (Exception e) {
            long duration = System.currentTimeMillis() - start;
            log.error("{} failed after {}ms", methodName, duration, e);
            throw e;
        }
    }
    
    // Modify parameters
    @Around("execution(* saveUser(..)) && args(user)")
    public Object sanitizeUser(ProceedingJoinPoint pjp, User user) throws Throwable {
        // Modify parameter before proceeding
        user.setUsername(user.getUsername().trim().toLowerCase());
        user.setEmail(user.getEmail().trim().toLowerCase());
        
        // Proceed with modified parameter
        return pjp.proceed(new Object[]{user});
    }
    
    // Modify return value
    @Around("execution(* getPrice(..))")
    public Object applyDiscount(ProceedingJoinPoint pjp) throws Throwable {
        BigDecimal price = (BigDecimal) pjp.proceed();
        
        // Apply 10% discount
        BigDecimal discounted = price.multiply(new BigDecimal("0.9"));
        
        return discounted;
    }
    
    // Caching
    @Around("execution(* findById(..)) && args(id)")
    public Object cache(ProceedingJoinPoint pjp, Long id) throws Throwable {
        String cacheKey = pjp.getSignature().toShortString() + ":" + id;
        
        // Check cache
        Object cached = cache.get(cacheKey);
        if (cached != null) {
            log.debug("Cache hit for {}", cacheKey);
            return cached;
        }
        
        // Cache miss - proceed with method
        log.debug("Cache miss for {}", cacheKey);
        Object result = pjp.proceed();
        
        // Store in cache
        if (result != null) {
            cache.put(cacheKey, result);
        }
        
        return result;
    }
    
    // Retry logic
    @Around("@annotation(retryable)")
    public Object retry(ProceedingJoinPoint pjp, Retryable retryable) throws Throwable {
        int maxAttempts = retryable.maxAttempts();
        long delay = retryable.delay();
        
        Throwable lastException = null;
        
        for (int attempt = 1; attempt <= maxAttempts; attempt++) {
            try {
                return pjp.proceed();
            } catch (Exception e) {
                lastException = e;
                log.warn("Attempt {}/{} failed: {}", attempt, maxAttempts, e.getMessage());
                
                if (attempt < maxAttempts) {
                    Thread.sleep(delay);
                }
            }
        }
        
        throw lastException;
    }
    
    // Prevent execution conditionally
    @Around("execution(* expensiveOperation(..))")
    public Object skipIfDisabled(ProceedingJoinPoint pjp) throws Throwable {
        if (!featureToggle.isEnabled("expensive-operation")) {
            log.info("Skipping expensive operation - feature disabled");
            return null;  // Don't proceed
        }
        
        return pjp.proceed();
    }
    
    // Exception handling and translation
    @Around("execution(* externalServiceCall(..))")
    public Object handleExternalServiceErrors(ProceedingJoinPoint pjp) throws Throwable {
        try {
            return pjp.proceed();
        } catch (ExternalServiceException e) {
            // Translate to application exception
            throw new ServiceUnavailableException("External service failed", e);
        } catch (TimeoutException e) {
            // Return fallback value
            log.warn("External service timeout, using fallback");
            return getFallbackValue();
        }
    }
}
```
