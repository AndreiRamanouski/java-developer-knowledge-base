## Overview

Spring Boot Actuator provides production-ready features for monitoring and managing applications. It exposes operational information about the running application through HTTP endpoints and JMX.

**Key Concept**: Actuator gives you insight into your application's health, metrics, environment, and more without writing custom monitoring code.

---

## 1. Production-Ready Features

### Getting Started

**Add dependency:**

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

**Default endpoints exposed:**

```
http://localhost:8080/actuator
```

Returns a list of available endpoints:

```json
{
  "_links": {
    "self": { "href": "http://localhost:8080/actuator" },
    "health": { "href": "http://localhost:8080/actuator/health" },
    "info": { "href": "http://localhost:8080/actuator/info" }
  }
}
```

By default, only `/health` and `/info` are exposed over HTTP for security reasons.

### Enabling All Endpoints

**application.yml:**

```yaml
management:
  endpoints:
    web:
      exposure:
        include: "*"  # Expose all endpoints (use cautiously in production!)
        # Or selectively:
        # include: health,info,metrics,env,beans,mappings
      base-path: /actuator  # Default base path
  endpoint:
    health:
      show-details: always  # Show detailed health info
```

**Security warning**: Always secure actuator endpoints in production!

#### Custom Health Indicators

Create custom health checks for external dependencies.

**Example 1: Simple custom health check**

```java
@Component
public class CustomHealthIndicator implements HealthIndicator {
    
    @Override
    public Health health() {
        // Perform health check logic
        boolean isHealthy = checkExternalService();
        
        if (isHealthy) {
            return Health.up()
                .withDetail("service", "external-api")
                .withDetail("status", "reachable")
                .build();
        } else {
            return Health.down()
                .withDetail("service", "external-api")
                .withDetail("error", "Connection timeout")
                .build();
        }
    }
    
    private boolean checkExternalService() {
        try {
            // Check external service
            RestTemplate restTemplate = new RestTemplate();
            ResponseEntity<String> response = restTemplate.getForEntity(
                "https://api.external-service.com/ping", 
                String.class
            );
            return response.getStatusCode().is2xxSuccessful();
        } catch (Exception e) {
            return false;
        }
    }
}
```

**Example 2: Advanced custom health check with metrics**

```java
@Component
public class PaymentGatewayHealthIndicator implements HealthIndicator {
    
    private final PaymentGatewayClient paymentClient;
    private final MeterRegistry meterRegistry;
    
    public PaymentGatewayHealthIndicator(
            PaymentGatewayClient paymentClient,
            MeterRegistry meterRegistry) {
        this.paymentClient = paymentClient;
        this.meterRegistry = meterRegistry;
    }
    
    @Override
    public Health health() {
        try {
            long startTime = System.currentTimeMillis();
            
            // Ping payment gateway
            boolean isAvailable = paymentClient.ping();
            
            long responseTime = System.currentTimeMillis() - startTime;
            
            // Record metrics
            meterRegistry.timer("payment.gateway.health.check")
                .record(responseTime, TimeUnit.MILLISECONDS);
            
            if (isAvailable && responseTime < 5000) {
                return Health.up()
                    .withDetail("gateway", "payment-provider")
                    .withDetail("responseTime", responseTime + "ms")
                    .withDetail("lastCheck", Instant.now())
                    .build();
            } else if (responseTime >= 5000) {
                return Health.down()
                    .withDetail("gateway", "payment-provider")
                    .withDetail("error", "Response time exceeds threshold")
                    .withDetail("responseTime", responseTime + "ms")
                    .build();
            } else {
                return Health.down()
                    .withDetail("gateway", "payment-provider")
                    .withDetail("error", "Gateway unreachable")
                    .build();
            }
        } catch (Exception e) {
            return Health.down()
                .withDetail("gateway", "payment-provider")
                .withDetail("error", e.getMessage())
                .withException(e)
                .build();
        }
    }
}
```

**Example 3: Reactive health indicator (WebFlux)**

```java
@Component
public class ReactiveApiHealthIndicator implements ReactiveHealthIndicator {
    
    private final WebClient webClient;
    
    public ReactiveApiHealthIndicator(WebClient.Builder webClientBuilder) {
        this.webClient = webClientBuilder
            .baseUrl("https://api.external.com")
            .build();
    }
    
    @Override
    public Mono<Health> health() {
        return webClient.get()
            .uri("/health")
            .retrieve()
            .toBodilessEntity()
            .map(response -> Health.up()
                .withDetail("api", "external-service")
                .withDetail("status", response.getStatusCode())
                .build())
            .onErrorResume(ex -> Mono.just(
                Health.down()
                    .withDetail("api", "external-service")
                    .withDetail("error", ex.getMessage())
                    .build()
            ))
            .timeout(Duration.ofSeconds(3))
            .onErrorResume(TimeoutException.class, ex -> Mono.just(
                Health.down()
                    .withDetail("api", "external-service")
                    .withDetail("error", "Timeout after 3 seconds")
                    .build()
            ));
    }
}
```

### Auditing

Track "who did what when" in your application.

**Enable auditing:**

```java
@Configuration
@EnableJpaAuditing
public class AuditConfig {
    
    @Bean
    public AuditorAware<String> auditorProvider() {
        return () -> {
            // Get current user from SecurityContext
            Authentication auth = SecurityContextHolder.getContext().getAuthentication();
            if (auth != null && auth.isAuthenticated()) {
                return Optional.of(auth.getName());
            }
            return Optional.of("system");
        };
    }
}
```

```java
    @Column(nullable = false, updatable = false)
    private String createdBy;
    
    @LastModifiedBy
    private String lastModifiedBy;
    
    // Getters and setters...
}
```

```java
@Component
public class RedisHttpTraceRepository implements HttpTraceRepository {
    
    private final RedisTemplate<String, HttpTrace> redisTemplate;
    private static final String TRACE_KEY = "http:traces";
    private static final int MAX_TRACES = 1000;
    
    public RedisHttpTraceRepository(RedisTemplate<String, HttpTrace> redisTemplate) {
        this.redisTemplate = redisTemplate;
    }
    
    @Override
    public List<HttpTrace> findAll() {
        return redisTemplate.opsForList().range(TRACE_KEY, 0, -1);
    }
    
    @Override
    public void add(HttpTrace trace) {
        redisTemplate.opsForList().leftPush(TRACE_KEY, trace);
        redisTemplate.opsForList().trim(TRACE_KEY, 0, MAX_TRACES - 1);
    }
}
```

### Basic Custom Endpoint

```java
@Endpoint(id = "features")
@Component
public class FeaturesEndpoint {
    
    private final Map<String, Boolean> features = new ConcurrentHashMap<>();
    
    public FeaturesEndpoint() {
        features.put("analytics", true);
        features.put("notifications", false);
        features.put("darkMode", true);
    }
    
    @ReadOperation
    public Map<String, Boolean> features() {
        return features;
    }
    
    @ReadOperation
    public Boolean feature(@Selector String name) {
        return features.get(name);
    }
    
    @WriteOperation
    public void configureFeature(@Selector String name, boolean enabled) {
        features.put(name, enabled);
    }
    
    @DeleteOperation
    public void deleteFeature(@Selector String name) {
        features.remove(name);
    }
}
```

### Advanced Custom Endpoint

```java
@Endpoint(id = "cache-management")
@Component
public class CacheManagementEndpoint {
    
    private final CacheManager cacheManager;
    private final MeterRegistry meterRegistry;
    
    public CacheManagementEndpoint(
            CacheManager cacheManager,
            MeterRegistry meterRegistry) {
        this.cacheManager = cacheManager;
        this.meterRegistry = meterRegistry;
    }
    
    @ReadOperation
    public Map<String, CacheStatistics> caches() {
        Map<String, CacheStatistics> stats = new HashMap<>();
        
        cacheManager.getCacheNames().forEach(cacheName -> {
            Cache cache = cacheManager.getCache(cacheName);
            if (cache != null) {
                stats.put(cacheName, getCacheStats(cacheName));
            }
        });
        
        return stats;
    }
    
    @ReadOperation
    public CacheStatistics cache(@Selector String name) {
        return getCacheStats(name);
    }
    
    @WriteOperation
    public void evictCache(@Selector String name) {
        Cache cache = cacheManager.getCache(name);
        if (cache != null) {
            cache.clear();
            meterRegistry.counter("cache.evictions", "cache", name).increment();
        }
    }
    
    @WriteOperation
    public void evictAll() {
        cacheManager.getCacheNames().forEach(cacheName -> {
            Cache cache = cacheManager.getCache(cacheName);
            if (cache != null) {
                cache.clear();
            }
        });
    }
    
    private CacheStatistics getCacheStats(String cacheName) {
        // Get statistics from cache
        // This is implementation-specific (Caffeine, EhCache, etc.)
        return new CacheStatistics(
            cacheName,
            0L,  // hits
            0L,  // misses
            0L,  // size
            0.0  // hit rate
        );
    }
    
    public static class CacheStatistics {
        private final String name;
        private final long hits;
        private final long misses;
        private final long size;
        private final double hitRate;
        
        public CacheStatistics(String name, long hits, long misses, long size, double hitRate) {
            this.name = name;
            this.hits = hits;
            this.misses = misses;
            this.size = size;
            this.hitRate = hitRate;
        }
        
        // Getters...
        public String getName() { return name; }
        public long getHits() { return hits; }
        public long getMisses() { return misses; }
        public long getSize() { return size; }
        public double getHitRate() { return hitRate; }
    }
}
```

### Security for Actuator Endpoints

Secure actuator endpoints in production.

**Add Spring Security:**

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

**Security configuration:**

```java
@Configuration
@EnableWebSecurity
public class ActuatorSecurityConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                // Public endpoints
                .requestMatchers("/actuator/health").permitAll()
                .requestMatchers("/actuator/info").permitAll()
                
                // Require authentication for other actuator endpoints
                .requestMatchers("/actuator/**").hasRole("ADMIN")
                
                // Application endpoints
                .requestMatchers("/api/**").authenticated()
                .anyRequest().permitAll()
            )
            .httpBasic(Customizer.withDefaults())
            .csrf(csrf -> csrf
                .ignoringRequestMatchers("/actuator/**")  // Disable CSRF for actuator
            );
        
        return http.build();
    }
    
    @Bean
    public UserDetailsService userDetailsService() {
        UserDetails admin = User.builder()
            .username("admin")
            .password("{noop}admin123")  // Use proper password encoder in production!
            .roles("ADMIN")
            .build();
        
        return new InMemoryUserDetailsManager(admin);
    }
}
```

**Different security for different endpoints:**

```java
@Configuration
public class ActuatorSecurityConfig {
    
    @Bean
    public SecurityFilterChain actuatorSecurity(HttpSecurity http) throws Exception {
        http
            .securityMatcher(EndpointRequest.toAnyEndpoint())
            .authorizeHttpRequests(auth -> auth
                // Health and info - public
                .requestMatchers(EndpointRequest.to("health", "info")).permitAll()
                
                // Metrics - requires METRICS role
                .requestMatchers(EndpointRequest.to("metrics", "prometheus")).hasRole("METRICS")
                
                // Dangerous operations - requires ADMIN
                .requestMatchers(EndpointRequest.to("shutdown", "threaddump", "heapdump"))
                    .hasRole("ADMIN")
                
                // Everything else - authenticated
                .anyRequest().authenticated()
            )
            .httpBasic(Customizer.withDefaults());
        
        return http.build();
    }
}
```

**Network-based security (restrict to localhost/internal network):**

```yaml
management:
  server:
    port: 9090  # Different port for actuator
    address: 127.0.0.1  # Only localhost can access
```

Or use Kubernetes network policies, AWS security groups, etc.

#### Timer - Track Duration

```java
@Service
public class PaymentService {
    
    private final Timer paymentProcessingTimer;
    
    public PaymentService(MeterRegistry meterRegistry) {
        this.paymentProcessingTimer = Timer.builder("payment.processing")
            .description("Payment processing time")
            .tag("gateway", "stripe")
            .publishPercentiles(0.5, 0.95, 0.99)  // Percentiles
            .publishPercentileHistogram()
            .register(meterRegistry);
    }
    
    public PaymentResult processPayment(Payment payment) {
        return paymentProcessingTimer.record(() -> {
            // Process payment
            return performPayment(payment);
        });
    }
    
    // Or manual timing
    public PaymentResult processPaymentManual(Payment payment) {
        Timer.Sample sample = Timer.start();
        try {
            PaymentResult result = performPayment(payment);
            sample.stop(paymentProcessingTimer);
            return result;
        } catch (Exception e) {
            sample.stop(Timer.builder("payment.processing")
                .tag("gateway", "stripe")
                .tag("status", "error")
                .register(meterRegistry));
```

### Dimensional Metrics (Tags)

Add dimensions to metrics for better filtering and aggregation.

```java
@Service
public class ApiMetricsService {
    
    private final MeterRegistry meterRegistry;
    
    public ApiMetricsService(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }
    
    public void recordApiCall(String endpoint, String method, int statusCode, long duration) {
        // Record with multiple tags
        Timer.builder("api.requests")
            .tag("endpoint", endpoint)
            .tag("method", method)
            .tag("status", String.valueOf(statusCode))
            .tag("status_class", statusCode / 100 + "xx")
            .tag("region", "us-east-1")
            .description("API request duration")
            .register(meterRegistry)
            .record(duration, TimeUnit.MILLISECONDS);
    }
}
```

**Query metrics by tags:**

```
GET /actuator/metrics/api.requests?tag=endpoint:/api/users&tag=method:GET&tag=status:200
```

**Tag naming best practices:**

- Use lowercase with dots: `http.method`, `db.operation`
- Keep cardinality low (avoid user IDs, timestamps)
- Use meaningful names
- Be consistent across services

### Complete Metrics Example

```java
@Service
public class ComprehensiveMetricsService {
    
    private final MeterRegistry meterRegistry;
    private final Counter requestCounter;
    private final Timer requestTimer;
    private final Gauge activeConnectionsGauge;
    
    public ComprehensiveMetricsService(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        
        // Counter
        this.requestCounter = Counter.builder("api.requests.total")
            .description("Total API requests")
            .tag("service", "user-service")
            .register(meterRegistry);
        
        // Timer with percentiles
        this.requestTimer = Timer.builder("api.request.duration")
            .description("API request duration")
            .tag("service", "user-service")
            .publishPercentiles(0.5, 0.95, 0.99)
            .publishPercentileHistogram()
            .serviceLevelObjectives(
                Duration.ofMillis(100),
                Duration.ofMillis(500),
                Duration.ofMillis(1000)
            )
            .minimumExpectedValue(Duration.ofMillis(1))
            .maximumExpectedValue(Duration.ofSeconds(10))
            .register(meterRegistry);
        
        // Gauge for active connections
        AtomicInteger activeConnections = new AtomicInteger(0);
        this.activeConnectionsGauge = Gauge.builder("api.connections.active", activeConnections, AtomicInteger::get)
            .description("Active connections")
            .tag("service", "user-service")
            .register(meterRegistry);
    }
    
    public void processRequest(String endpoint, Runnable requestHandler) {
        // Increment counter
        requestCounter.increment();
        
        // Record timing
        requestTimer.record(() -> {
            // Add tags dynamically
            Tags tags = Tags.of("endpoint", endpoint);
            
            try {
                requestHandler.run();
                
                // Success metric
                meterRegistry.counter("api.requests.success", tags).increment();
            } catch (Exception e) {
                // Error metric
                meterRegistry.counter("api.requests.error", 
                    tags.and("error", e.getClass().getSimpleName())
                ).increment();
                
                throw e;
            }
        });
    }
    
    // Custom metrics for business logic
    public void recordUserSignup(String plan) {
        meterRegistry.counter("users.signups", "plan", plan).increment();
    }
    
    public void recordPayment(BigDecimal amount, String currency) {
        meterRegistry.summary("payments.amount", "currency", currency)
            .record(amount.doubleValue());
    }
    
    public void recordCacheHit(String cacheName) {
        meterRegistry.counter("cache.requests", 
            "cache", cacheName, 
            "result", "hit"
        ).increment();
    }
}
```

## Best Practices

### DO ✅

1. **Expose health and info publicly** - Safe for monitoring
2. **Secure sensitive endpoints** - Require authentication
3. **Use different port for actuator** - Separate management traffic
4. **Create custom health indicators** - Monitor critical dependencies
5. **Use meaningful metric names** - Follow naming conventions
6. **Add tags to metrics** - Enable filtering and aggregation
7. **Set up alerts** - Based on health checks and metrics
8. **Monitor JVM metrics** - Memory, GC, threads
9. **Use percentiles** - For latency metrics (p50, p95, p99)
10. **Export to monitoring system** - Prometheus, DataDog, etc.

### DON'T ❌

1. **Don't expose all endpoints publicly** - Security risk
2. **Don't use InMemoryHttpTraceRepository in production** - Limited capacity
3. **Don't create high-cardinality tags** - User IDs, timestamps
4. **Don't ignore health check failures** - They indicate real problems
5. **Don't skip security** - Always protect actuator endpoints
6. **Don't over-instrument** - Too many metrics → noise
7. **Don't forget SLOs** - Set service level objectives for timers
8. **Don't ignore metric exporters** - Use appropriate backend
