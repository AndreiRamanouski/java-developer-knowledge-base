## Overview

Unit testing is the foundation of software quality. Testing Spring components and mastering mocking frameworks like Mockito are essential skills for Java developers. This guide covers everything from basic unit tests to advanced mocking patterns.

**Key Concept**: Unit tests should be fast, isolated, and deterministic. Test one unit of work at a time, mock external dependencies, and ensure tests can run independently in any order.

```java
    @TestConfiguration
    static class TestConfig {
        
        @Bean
        @Primary  // Override production bean

        @Bean
        public TestDataBuilder testDataBuilder() {
            return new TestDataBuilder();
        }
    }
```

### @MockBean and @SpyBean

```java
/**
 * @MockBean - Replace Spring Bean with Mock
 */
@SpringBootTest
class UserServiceTest {
    
    @Autowired
    private UserService userService;  // Real service
    
    @MockBean
    private UserRepository userRepository;  // Mocked repository
    
    @MockBean
    private EmailService emailService;  // Mocked email service
    
    @Test
    void testCreateUser() {
        // Given: Mock behavior
        User user = new User("john@example.com");
        when(userRepository.save(any(User.class))).thenReturn(user);
        
        // When: Call service
        User created = userService.createUser(user);
        
        // Then: Verify
        assertNotNull(created);
        verify(userRepository).save(any(User.class));
        verify(emailService).sendWelcomeEmail(user);
    }
}

/**
 * @SpyBean - Partial Mock (Real Object with Some Methods Mocked)
 */
@SpringBootTest
class OrderServiceTest {
    
    @Autowired
    private OrderService orderService;
    
    @SpyBean
    private PriceCalculator priceCalculator;  // Real bean, but can mock specific methods
    
    @Test
    void testCreateOrderWithCustomPrice() {
        // Real method calls work normally
        // But we can override specific method
        doReturn(new BigDecimal("99.99"))
            .when(priceCalculator).calculateTax(any());
        
        Order order = orderService.createOrder(new OrderRequest());
        
        // calculateTax() returns mocked value
        // Other PriceCalculator methods work normally
        assertEquals(new BigDecimal("99.99"), order.getTax());
    }
}

/**
 * @MockBean vs @SpyBean
 */

// @MockBean:
// - Replaces bean with Mockito mock
// - All methods return default values (null, 0, false)
// - Must stub every method you call
// - Use when: External dependencies (repositories, APIs)

// @SpyBean:
// - Wraps real bean with Mockito spy
// - Real methods execute by default
// - Can stub specific methods
// - Use when: Partial mocking needed

/**
 * Multiple Mocks of Same Type
 */
@SpringBootTest
class PaymentServiceTest {
    
    @Autowired
    private PaymentService paymentService;
    
    @MockBean
    @Qualifier("stripeGateway")
    private PaymentGateway stripeGateway;
    
    @MockBean
    @Qualifier("paypalGateway")
    private PaymentGateway paypalGateway;
    
    @Test
    void testPaymentWithStripe() {
        when(stripeGateway.processPayment(any())).thenReturn(new PaymentResult(true));
        
        paymentService.processPayment(PaymentMethod.STRIPE, new BigDecimal("100"));
        
        verify(stripeGateway).processPayment(any());
        verify(paypalGateway, never()).processPayment(any());
    }
}

/**
 * Field vs Method Injection with @MockBean
 */
@SpringBootTest
class UserServiceTest {
    
    // ✅ Field injection (most common)
    @MockBean
    private UserRepository userRepository;
    
    @Autowired
    private UserService userService;
    
    // ✅ Constructor injection (preferred for production, works in tests)
    private final EmailService emailService;
    
    @Autowired
    UserServiceTest(@MockBean EmailService emailService) {
        this.emailService = emailService;
    }
}

/**
 * @MockBean Lifecycle
 */
// 1. Spring context starts
// 2. @MockBean creates Mockito mock
// 3. Mock replaces real bean in context
// 4. All @Autowired injections get the mock
// 5. After each test: Mockito resets the mock
// 6. Context is cached for subsequent tests (unless @DirtiesContext)
```

```java
    @Mock
```

### when(), thenReturn(), thenThrow()

```java
/**
 * Stubbing Method Behavior
 */
@ExtendWith(MockitoExtension.class)
class StubbingTest {
    
    @Mock
    private UserRepository userRepository;
    
    @Test
    void testThenReturn() {
        User user = new User("john");
        
        // ✅ Basic stubbing
        when(userRepository.findById(1L))
            .thenReturn(Optional.of(user));
        
        // Multiple calls return same value
        assertEquals(user, userRepository.findById(1L).get());
        assertEquals(user, userRepository.findById(1L).get());
    }
    
    @Test
    void testMultipleThenReturn() {
        User user1 = new User("john");
        User user2 = new User("jane");
        
        // ✅ Different return values for consecutive calls
        when(userRepository.findById(1L))
            .thenReturn(Optional.of(user1))
            .thenReturn(Optional.of(user2))
            .thenReturn(Optional.empty());
        
        assertEquals(user1, userRepository.findById(1L).get());  // First call
        assertEquals(user2, userRepository.findById(1L).get());  // Second call
        assertFalse(userRepository.findById(1L).isPresent());    // Third call
        assertFalse(userRepository.findById(1L).isPresent());    // Fourth+ calls
    }
    
    @Test
    void testThenThrow() {
        // ✅ Throw exception
        when(userRepository.findById(1L))
            .thenThrow(new RuntimeException("Database error"));
        
        assertThrows(RuntimeException.class, () -> {
            userRepository.findById(1L);
        });
    }
    
    @Test
    void testThenThrowMultipleExceptions() {
        // ✅ Different exceptions
        when(userRepository.findById(1L))
            .thenThrow(new RuntimeException("Error 1"))
            .thenThrow(new RuntimeException("Error 2"));
        
        assertThrows(RuntimeException.class, () -> userRepository.findById(1L));
        assertThrows(RuntimeException.class, () -> userRepository.findById(1L));
    }
    
    @Test
    void testThenAnswer() {
        // ✅ Custom logic with Answer
        when(userRepository.findById(anyLong()))
            .thenAnswer(invocation -> {
                Long id = invocation.getArgument(0);
                return Optional.of(new User("user-" + id));
            });
        
        assertEquals("user-1", userRepository.findById(1L).get().getName());
        assertEquals("user-2", userRepository.findById(2L).get().getName());
    }
    
    @Test
    void testThenCallRealMethod() {
        // ✅ Call real method (requires spy or @SpyBean)
        UserRepository spy = spy(new UserRepositoryImpl());
        
        when(spy.findById(1L)).thenCallRealMethod();
        
        // Real implementation called
    }
}

/**
 * Chaining Stubbing
 */
@Test
void testChainingStubbing() {
    when(userRepository.findById(1L))
        .thenReturn(Optional.of(new User("john")))   // First call
        .thenReturn(Optional.of(new User("jane")))   // Second call
        .thenThrow(new RuntimeException("Error"));   // Third call
}

/**
 * Stubbing with Conditions
 */
@Test
void testConditionalStubbing() {
    // Return different values based on argument
    when(userRepository.findById(1L))
        .thenReturn(Optional.of(new User("john")));
    
    when(userRepository.findById(2L))
        .thenReturn(Optional.of(new User("jane")));
    
    when(userRepository.findById(3L))
        .thenReturn(Optional.empty());
}

/**
 * doReturn() vs when().thenReturn()
 */
@Test
void doReturnVsWhenThenReturn() {
    UserRepository spy = spy(new UserRepositoryImpl());
    
    // ❌ when().thenReturn() calls real method first
    when(spy.findById(1L)).thenReturn(Optional.of(new User("john")));
    // Real method executed, then stubbed
    
    // ✅ doReturn() doesn't call real method
    doReturn(Optional.of(new User("john")))
        .when(spy).findById(1L);
    // Real method NOT executed, just stubbed
}

/**
 * Stubbing Void Methods
 */
@Test
void testVoidMethodStubbing() {
    EmailService emailService = mock(EmailService.class);
    
    // ✅ Void method does nothing by default (safe)
    emailService.sendEmail("test@example.com");  // Does nothing
    
    // ✅ Make void method throw exception
    doThrow(new RuntimeException("Email failed"))
        .when(emailService).sendEmail(anyString());
    
    assertThrows(RuntimeException.class, () -> {
        emailService.sendEmail("test@example.com");
    });
    
    // ✅ Void method with custom behavior
    doAnswer(invocation -> {
        String email = invocation.getArgument(0);
        System.out.println("Sending email to: " + email);
        return null;  // void
    }).when(emailService).sendEmail(anyString());
}
```

### Argument Matchers

```java
/**
 * Argument Matchers
 */
@ExtendWith(MockitoExtension.class)
class ArgumentMatchersTest {
    
    @Mock
    private UserRepository userRepository;
    
    @Mock
    private EmailService emailService;
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // ANY MATCHERS
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testAnyMatchers() {
        // ✅ any() - Matches any object
        when(userRepository.save(any(User.class)))
            .thenReturn(new User("john"));
        
        // ✅ anyLong(), anyInt(), anyString(), etc.
        when(userRepository.findById(anyLong()))
            .thenReturn(Optional.of(new User("john")));
        
        // ✅ anyList(), anySet(), anyMap(), anyCollection()
        when(userRepository.saveAll(anyList()))
            .thenReturn(Collections.emptyList());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // EQ MATCHER
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testEqMatcher() {
        // ✅ eq() - Matches exact value
        when(userRepository.findById(eq(1L)))
            .thenReturn(Optional.of(new User("john")));
        
        // ⚠️ When mixing matchers and literals, ALL must be matchers
        // ❌ Wrong:
        // when(emailService.sendEmail("john@example.com", any()))
        
        // ✅ Correct:
        when(emailService.sendEmail(eq("john@example.com"), any()))
            .thenReturn(true);
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // CUSTOM MATCHERS
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testArgThat() {
        // ✅ argThat() - Custom condition
        when(userRepository.save(argThat(user -> 
            user.getEmail().endsWith("@example.com")
        ))).thenReturn(new User("john"));
        
        // Matches: email ends with @example.com
        User user1 = new User("john@example.com");
        assertNotNull(userRepository.save(user1));
        
        // Doesn't match
        User user2 = new User("john@other.com");
        assertNull(userRepository.save(user2));  // Returns default null
    }
    
    @Test
    void testArgThatAdvanced() {
        // ✅ Complex conditions
        when(userRepository.save(argThat(user ->
            user.getEmail() != null &&
            user.getEmail().contains("@") &&
            user.getAge() >= 18
        ))).thenReturn(new User("valid"));
        
        User validUser = new User("john@example.com", 25);
        assertNotNull(userRepository.save(validUser));
        
        User invalidUser = new User("john@example.com", 15);
        assertNull(userRepository.save(invalidUser));
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // NULL MATCHERS
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testNullMatchers() {
        // ✅ isNull(), isNotNull()
        when(userRepository.save(isNull()))
            .thenThrow(new IllegalArgumentException("User cannot be null"));
        
        when(userRepository.save(isNotNull()))
            .thenReturn(new User("john"));
        
        assertThrows(IllegalArgumentException.class, () -> {
            userRepository.save(null);
        });
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // STRING MATCHERS
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testStringMatchers() {
        // ✅ contains()
        when(userRepository.findByEmail(contains("@example.com")))
            .thenReturn(Optional.of(new User("john")));
        
        // ✅ startsWith(), endsWith()
        when(userRepository.findByEmail(startsWith("admin")))
            .thenReturn(Optional.of(new User("admin")));
        
        // ✅ matches() - Regex
        when(userRepository.findByEmail(matches(".*@example\\.com")))
            .thenReturn(Optional.of(new User("john")));
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // COLLECTION MATCHERS
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testCollectionMatchers() {
        // ✅ anyList(), anySet(), anyMap()
        when(userRepository.saveAll(anyList()))
            .thenReturn(Collections.emptyList());
        
        // ✅ argThat with collections
```

```java
    ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
    emailService.sendEmail("test@example.com");
    verify(emailService).sendEmail(captor.capture());
    assertEquals("test@example.com", captor.getValue());
    
    // Pattern 4: Custom side effects
    doAnswer(invocation -> {
        String email = invocation.getArgument(0);
        // Custom logic (logging, tracking, etc.)
        return null;
    }).when(emailService).sendEmail(anyString());
}
```

### Spies vs Mocks

```java
/**
 * Spies vs Mocks
 */
@ExtendWith(MockitoExtension.class)
class SpiesVsMocksTest {
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // MOCK: All methods return defaults (null, 0, false)
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testMockBehavior() {
        // ✅ Mock: All methods return defaults
        List<String> mockList = mock(ArrayList.class);
        
        mockList.add("element");  // Does nothing (void)
        int size = mockList.size();  // Returns 0 (default)
        String element = mockList.get(0);  // Returns null (default)
        
        assertEquals(0, size);
        assertNull(element);
        
        // Must stub to get real behavior
        when(mockList.size()).thenReturn(1);
        assertEquals(1, mockList.size());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // SPY: Real methods execute unless stubbed
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testSpyBehavior() {
        // ✅ Spy: Real methods execute
        List<String> spyList = spy(new ArrayList<>());
        
        spyList.add("element");  // Real method executes
        int size = spyList.size();  // Real method executes
        String element = spyList.get(0);  // Real method executes
        
        assertEquals(1, size);
        assertEquals("element", element);
        
        // Can still stub specific methods
        when(spyList.size()).thenReturn(100);
        assertEquals(100, spyList.size());  // Stubbed
        assertEquals("element", spyList.get(0));  // Still real
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // SPY WITH @Spy ANNOTATION
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Spy
    private List<String> spyList = new ArrayList<>();
    
    @Test
    void testSpyAnnotation() {
        spyList.add("element");
        assertEquals(1, spyList.size());
        
        // Stub specific method
        when(spyList.size()).thenReturn(100);
        assertEquals(100, spyList.size());
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // WHEN vs DO_RETURN WITH SPIES
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testSpyWhenVsDoReturn() {
        List<String> spy = spy(new ArrayList<>());
        
        // ❌ when() calls real method THEN stubs
        when(spy.get(0)).thenReturn("stubbed");  // Throws IndexOutOfBoundsException!
        
        // ✅ doReturn() doesn't call real method
        doReturn("stubbed").when(spy).get(0);  // Safe, just stubs
        assertEquals("stubbed", spy.get(0));
    }
    
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // REAL-WORLD SPY USAGE
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    @Test
    void testSpyRealWorldExample() {
        // ✅ Spy on real service
        UserService spy = spy(new UserService(new UserRepositoryImpl()));
        
        // Most methods work normally
        // But stub expensive operation
        doReturn(true).when(spy).validateWithExternalApi(anyString());
        
        // Real service logic executes
        // But external API call is stubbed
        User user = spy.createUser(new User("john@example.com"));
        assertNotNull(user);
    }
}

/**
 * Mock vs Spy Decision Guide
 */

// USE MOCK when:
// ✅ Testing external dependencies (repositories, APIs)
// ✅ You want complete control over behavior
// ✅ Don't care about real implementation
// ✅ Most common case

// USE SPY when:
// ✅ Testing partial functionality
// ✅ Want real behavior except for specific methods
// ✅ Testing abstract classes or inheritance
// ✅ Legacy code refactoring

// AVOID SPY when:
// ❌ Can use simple mock instead
// ❌ Testing becomes confusing (real vs stubbed)
// ❌ Class has complex initialization

/**
 * Comparison Table
 */
@Test
void mockVsSpyComparison() {
    // Mock
    List<String> mock = mock(ArrayList.class);
    mock.add("element");  // Does nothing
    assertEquals(0, mock.size());  // Returns 0 (default)
    
    // Spy
    List<String> spy = spy(new ArrayList<>());
    spy.add("element");  // Real add() executes
    assertEquals(1, spy.size());  // Real size() executes
    
    // Both can be stubbed
    when(mock.size()).thenReturn(100);
    when(spy.size()).thenReturn(100);
    
    assertEquals(100, mock.size());
    assertEquals(100, spy.size());
}
```

## Summary & Best Practices

### Testing Quick Reference

|Annotation|Purpose|Use Case|
|---|---|---|
|`@ExtendWith(SpringExtension.class)`|Enable Spring|Spring component testing|
|`@SpringBootTest`|Full application context|Integration-like tests|
|`@ContextConfiguration`|Load specific config|Custom test configuration|
|`@TestConfiguration`|Test-specific beans|Override production beans|
|`@MockBean`|Replace Spring bean with mock|External dependencies|
|`@SpyBean`|Partial mock of Spring bean|Partial mocking needed|
|`@Mock`|Create Mockito mock|Pure unit tests|
|`@Spy`|Create Mockito spy|Partial mocking|
|`@InjectMocks`|Inject mocks into test subject|Constructor/field injection|
|`@Captor`|Capture arguments|Verify argument values|

### Mockito Best Practices

1. **Use @ExtendWith(MockitoExtension.class)** for automatic mock initialization
2. **Prefer @Mock over mock()** for cleaner test setup
3. **Use @InjectMocks** for automatic dependency injection
4. **Prefer specific matchers** over any() when possible
5. **Use argument captors** for complex argument verification
6. **Verify important interactions**, not implementation details
7. **Use doReturn() instead of when()** with spies
8. **Keep tests focused** - one behavior per test
9. **Name tests clearly** - testMethodName_Scenario_ExpectedResult
10. **Reset mocks** between tests (MockitoExtension does this)

### Common Mistakes

❌ Mixing matchers and literals without eq()  
❌ Over-verifying (checking every method call)  
❌ Not verifying important interactions  
❌ Using when() instead of doReturn() with spies  
❌ Complex test setup (too many mocks)  
❌ Testing implementation details  
❌ Not using argument captors when needed  
❌ Forgetting to verify void methods

### Test Structure (AAA Pattern)

```java
@Test
void testMethodName() {
    // Arrange (Given)
    User user = new User("john@example.com");
    when(userRepository.save(any())).thenReturn(user);
    
    // Act (When)
    User created = userService.createUser(user);
    
    // Assert (Then)
    assertNotNull(created);
    verify(userRepository).save(any());
    verify(emailService).sendWelcomeEmail(user);
}
```
