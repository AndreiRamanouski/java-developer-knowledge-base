## Overview

Integration testing verifies that multiple components work together correctly. Spring Boot provides powerful testing tools that make integration testing straightforward. Unlike unit tests that mock dependencies, integration tests use real components and configurations.

**Key Concept**: Integration tests are slower but more realistic than unit tests. Use test slices to test specific layers efficiently, and full context tests for end-to-end scenarios.

---

## 1. Spring Boot Test

### @SpringBootTest - Full Application Context

```java
/**
 * @SpringBootTest - Load Full Application Context
 */
@SpringBootTest
class UserServiceIntegrationTest {
    
    @Autowired
    private UserService userService;
    
    @Autowired
    private UserRepository userRepository;
    
    @Test
    void testCreateUser() {
        // Real service with real repository
        User user = new User("john@example.com");
        User created = userService.createUser(user);
        
        assertNotNull(created.getId());
        assertTrue(userRepository.existsById(created.getId()));
    }
}

/**
 * How @SpringBootTest Works
 */
// 1. Searches for @SpringBootApplication class
// 2. Loads full application context
// 3. Auto-configuration enabled
// 4. All beans available for injection
// 5. Can override specific beans with @MockBean

/**
 * @SpringBootTest Web Environment Options
 */

 * When to Use @SpringBootTest

// - Testing multiple components together
// - End-to-end testing
```

### @WebMvcTest - Controller Slice Testing

```java
/**
 * @WebMvcTest - Test MVC Controllers Only
 */
@WebMvcTest(UserController.class)
class UserControllerTest {
    
    @Autowired
    private MockMvc mockMvc;
    
    @MockBean  // Service layer is mocked
    private UserService userService;
    
    @Test
    void testGetUser() throws Exception {
        // Given: Mock service behavior
        User user = new User(1L, "john@example.com");
        when(userService.findById(1L)).thenReturn(user);
        
        // When & Then: Test controller
        mockMvc.perform(get("/api/users/1"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.id").value(1))
            .andExpect(jsonPath("$.email").value("john@example.com"));
    }
    
    @Test
    void testCreateUser() throws Exception {
        User user = new User(1L, "john@example.com");
        when(userService.createUser(any())).thenReturn(user);
        
        mockMvc.perform(post("/api/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("{\"email\":\"john@example.com\"}"))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.id").value(1));
    }
}

/**
 * What @WebMvcTest Loads
 */
// ✅ Loaded:

// - @ControllerAdvice, @RestControllerAdvice
// - @JsonComponent
// - @Converter, @Formatter
// - WebMvcConfigurer
// - HandlerInterceptor
// - MockMvc auto-configured

// ❌ NOT Loaded:
// - @Service, @Repository, @Component
// - Regular @Configuration classes
```

### @DataJpaTest - Repository Slice Testing

```java
/**
 * @DataJpaTest - Test JPA Repositories Only
 */
@DataJpaTest
class UserRepositoryTest {
    
    @Autowired
    private UserRepository userRepository;
    
    @Autowired
    private TestEntityManager entityManager;  // Alternative to repository for setup
    
    @Test
    void testFindByEmail() {
        // Given: Persist test data
        User user = new User("john@example.com");
        entityManager.persist(user);
        entityManager.flush();
        
        // When: Query
        Optional<User> found = userRepository.findByEmail("john@example.com");
        
        // Then: Verify
        assertTrue(found.isPresent());
        assertEquals("john@example.com", found.get().getEmail());
    }
    
    @Test
    void testCustomQuery() {
        // Test custom JPQL query
        User user1 = new User("john@example.com");
        User user2 = new User("jane@example.com");
        entityManager.persist(user1);
        entityManager.persist(user2);
        entityManager.flush();
        
        List<User> users = userRepository.findAllActive();
        
        assertEquals(2, users.size());
    }
    
    @Test
    void testNativeQuery() {
        // Test native SQL query
        User user = new User("john@example.com");
        entityManager.persist(user);
        entityManager.flush();
        
        List<User> users = userRepository.findByEmailNative("john@example.com");
        
        assertEquals(1, users.size());
    }
}

/**
 * What @DataJpaTest Loads
 */
// ✅ Loaded:
// - JPA repositories
// - DataSource configuration
// - JPA entities
// - TestEntityManager
// - In-memory database (H2 by default)
// - Transaction management

// ❌ NOT Loaded:
// - @Service, @Controller, @Component
// - Full auto-configuration
// - Web layer

/**
 * @DataJpaTest with Real Database
 */
@DataJpaTest
@AutoConfigureTestDatabase(replace = AutoConfigureTestDatabase.Replace.NONE)
class UserRepositoryWithRealDbTest {
    
    // Uses actual database configured in application.properties
    // Not in-memory H2
    
    @Autowired
    private UserRepository userRepository;
    
    @Test
    void testWithRealDatabase() {
        User user = new User("john@example.com");
        User saved = userRepository.save(user);
        
        assertNotNull(saved.getId());
    }
}

/**
 * TestEntityManager vs Repository
 */
@DataJpaTest
class TestEntityManagerVsRepositoryTest {
    
    @Autowired
    private UserRepository userRepository;
    
    @Autowired
    private TestEntityManager entityManager;
    
    @Test
    void testWithRepository() {
        // ✅ Use repository (most common)
        User user = new User("john@example.com");
        userRepository.save(user);
        
        Optional<User> found = userRepository.findById(user.getId());
        assertTrue(found.isPresent());
    }
    
    @Test
    void testWithEntityManager() {
        // ✅ Use entity manager for setup/verification
        User user = new User("john@example.com");
        entityManager.persist(user);
        entityManager.flush();  // Force SQL execution
        
        // Clear persistence context
        entityManager.clear();
        
        // Query bypasses first-level cache
        User found = entityManager.find(User.class, user.getId());
        assertNotNull(found);
    }
}
```

### Test Slices vs Full Context

```java
/**
 * Test Slices vs @SpringBootTest Comparison
 */

// Test Slice (@WebMvcTest, @DataJpaTest, etc.)
// ✅ Fast startup (loads only specific layer)
// ✅ Isolated testing (one layer at a time)
// ✅ Focused tests
// ❌ Requires mocking dependencies
// ❌ Doesn't test integration between layers

// Full Context (@SpringBootTest)
// ✅ Tests real integration
// ✅ No mocking needed (can use real beans)
// ✅ Catches configuration issues
// ❌ Slow startup (loads entire application)
// ❌ Tests may be harder to debug

/**
 * Test Strategy: Pyramid Approach
 */
// Many: Unit tests (@ExtendWith(MockitoExtension.class))
//   ↑   Fast, isolated, mock everything
//   |
// Some: Slice tests (@WebMvcTest, @DataJpaTest)
//   ↑   Medium speed, test one layer
//   |
// Few:  Integration tests (@SpringBootTest)
//   ↑   Slow, end-to-end, real components

/**
 * Choosing the Right Test Type
 */

// @ExtendWith(MockitoExtension.class) - Unit Test
@ExtendWith(MockitoExtension.class)
class UserServiceUnitTest {
    @Mock private UserRepository userRepository;
    @InjectMocks private UserService userService;
    
    @Test
    void testBusinessLogic() {
        // Test service logic in isolation
    }
}

// @WebMvcTest - Controller Slice
@WebMvcTest(UserController.class)
class UserControllerSliceTest {
    @Autowired private MockMvc mockMvc;
    @MockBean private UserService userService;
    
    @Test
    void testControllerEndpoint() {
        // Test controller without service/repository
    }
}

// @DataJpaTest - Repository Slice
@DataJpaTest
class UserRepositorySliceTest {
    @Autowired private UserRepository userRepository;
    
    @Test
    void testDatabaseQuery() {
        // Test repository with real database
    }
}

// @SpringBootTest - Integration Test
@SpringBootTest
class UserIntegrationTest {
    @Autowired private UserService userService;
    @Autowired private UserRepository userRepository;
    
    @Test
    void testEndToEnd() {
        // Test full stack (controller → service → repository → database)
    }
}
```

```java
            .andExpect(status().isCreated())
            .andExpect(header().exists("Location"))
```

```java
/**
```

```java
        mockMvc.perform(get("/api/users/1"))
            // Status matchers
            .andExpect(status().isOk())                    // 200
            .andExpect(status().is(200))                   // Exact status code
            // .andExpect(status().isCreated())            // 201
            // .andExpect(status().isNoContent())          // 204
            // .andExpect(status().isBadRequest())         // 400
            // .andExpect(status().isUnauthorized())       // 401
            // .andExpect(status().isForbidden())          // 403
```

## Summary & Best Practices

### Test Type Quick Reference

|Test Type|Annotation|Speed|Scope|Use Case|
|---|---|---|---|---|
|**Unit**|`@ExtendWith(MockitoExtension)`|⚡⚡⚡⚡|Single class|Business logic|
|**Slice**|`@WebMvcTest`, `@DataJpaTest`|⚡⚡⚡|Single layer|Controllers, repositories|
|**Integration**|`@SpringBootTest`|⚡⚡|Full app|End-to-end|

### Testing Strategy

1. **Many unit tests** (fast, isolated)
2. **Some slice tests** (medium speed, one layer)
3. **Few integration tests** (slow, full stack)

### Test Annotations

✅ **@SpringBootTest** - Full application context, integration tests  
✅ **@WebMvcTest** - Controller layer only, fast  
✅ **@DataJpaTest** - Repository layer, embedded database  
✅ **@RestClientTest** - REST client testing  
✅ **@MockBean** - Replace Spring beans with mocks  
✅ **@TestConfiguration** - Test-specific beans  
✅ **@Sql** - Execute SQL scripts  
✅ **@Transactional** - Auto-rollback  
✅ **@ActiveProfiles** - Activate test profiles

### Database Testing

✅ Use **H2** for fast tests  
✅ Use **Testcontainers** for realistic tests  
✅ Use **@Transactional** for automatic cleanup  
✅ Use **@Sql** for complex test data

### Best Practices

1. **Use test slices** when possible (faster than @SpringBootTest)
2. **Isolate tests** - each test should be independent
3. **Use @Transactional** for automatic rollback
4. **Mock external dependencies** with @MockBean
5. **Use Testcontainers** for realistic database tests
6. **Keep integration tests fast** - use embedded databases
7. **Name tests clearly** - describe what's being tested
8. **Use @ActiveProfiles("test")** consistently
9. **Don't test Spring framework** - test your code
10. **Clean up resources** - close connections, files
