## Overview

Migrating from Maven to Gradle can significantly improve build performance and flexibility. While Gradle provides automated migration tools, understanding the differences and following a structured approach ensures a smooth transition.

```
┌─────────────────────────────────────────────────────────────────┐
│              MAVEN TO GRADLE MIGRATION STRATEGY                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Phase 1: Assessment                                            │
│  ├── Analyze current Maven structure                           │
│  ├── Identify custom plugins                                   │
│  ├── Document build requirements                               │
│  └── Plan migration approach                                   │
│                                                                 │
│  Phase 2: Automated Conversion                                  │
│  ├── Run: gradle init                                          │
│  ├── Convert pom.xml → build.gradle                           │
│  ├── Generate Gradle wrapper                                   │
│  └── Initial conversion (80-90% complete)                      │
│                                                                 │
│  Phase 3: Manual Adjustments                                    │
│  ├── Fix dependencies (compile → implementation)               │
│  ├── Convert custom plugins                                    │
│  ├── Migrate build profiles                                    │
│  └── Test thoroughly                                           │
│                                                                 │
│  Phase 4: Optimization                                          │
│  ├── Enable build cache                                        │
│  ├── Configure parallel execution                              │
│  ├── Create convention plugins                                 │
│  └── Optimize performance                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 1. Automatic Migration with `gradle init`

### 1.1 Basic Conversion

```bash
# In directory with pom.xml
gradle init

# Interactive prompts:
# 1. Select type of project to generate: 2: pom.xml (Maven project)
# 2. Select build script DSL: 1: Groovy
# 3. Generate build using new APIs: yes

# Generates:
# ✓ build.gradle (converted from pom.xml)
# ✓ settings.gradle (project structure)
# ✓ gradle/ (wrapper files)
# ✓ gradlew, gradlew.bat (wrapper scripts)
```

**What Gets Converted:**

```
✓ Dependencies
✓ Basic plugins
✓ Repositories
✓ Properties
✓ Multi-module structure
✓ Test configuration

✗ Custom plugins (needs manual conversion)
✗ Complex build logic
✗ Maven profiles (partial conversion)
✗ Custom tasks
```

## 2. Key Differences: Maven vs Gradle

### 2.1 Dependency Configurations

**Maven Scopes vs Gradle Configurations:**

```
┌──────────────────────────────────────────────────────────────┐
│         MAVEN SCOPE → GRADLE CONFIGURATION                   │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  compile          → implementation (preferred)               │
│  compile          → api (for libraries)                      │
│  provided         → compileOnly                              │
│  runtime          → runtimeOnly                              │
│  test             → testImplementation                       │
│  test (runtime)   → testRuntimeOnly                          │
│  system           → (use files() or fileTree())              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**Maven:**

```xml
<dependencies>
    <!-- compile -->
    <dependency>
        <groupId>com.google.guava</groupId>
        <artifactId>guava</artifactId>
        <version>32.1.3-jre</version>
    </dependency>
    
    <!-- provided -->
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <version>1.18.30</version>
        <scope>provided</scope>
    </dependency>
    
    <!-- runtime -->
    <dependency>
        <groupId>org.postgresql</groupId>
        <artifactId>postgresql</artifactId>
        <version>42.6.0</version>
        <scope>runtime</scope>
    </dependency>
    
    <!-- test -->
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter</artifactId>
        <version>5.10.0</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

**Gradle:**

```groovy
dependencies {
    // Maven compile → Gradle implementation
    implementation 'com.google.guava:guava:32.1.3-jre'
    
    // Maven provided → Gradle compileOnly
    compileOnly 'org.projectlombok:lombok:1.18.30'
    annotationProcessor 'org.projectlombok:lombok:1.18.30'
    
    // Maven runtime → Gradle runtimeOnly
    runtimeOnly 'org.postgresql:postgresql:42.6.0'
    
    // Maven test → Gradle testImplementation
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}
```

**Maven:**


**Gradle:**

```groovy
// Method 1: Extra properties
ext {
    javaVersion = JavaVersion.VERSION_17
    springBootVersion = '3.2.0'
}

    implementation "org.springframework.boot:spring-boot-starter-web:${springBootVersion}"

// Method 2: Project properties (gradle.properties)
// gradle.properties

dependencies {
    implementation libs.spring.boot.web
```

### 2.6 Profiles vs Configuration

**Maven Profiles:**

```xml
<profiles>
    <profile>
        <id>dev</id>
        <properties>
            <env>development</env>
        </properties>
        <dependencies>
            <dependency>
                <groupId>com.h2database</groupId>
                <artifactId>h2</artifactId>
            </dependency>
        </dependencies>
    </profile>
    
    <profile>
        <id>prod</id>
        <properties>
            <env>production</env>
        </properties>
        <dependencies>
            <dependency>
                <groupId>org.postgresql</groupId>
                <artifactId>postgresql</artifactId>
            </dependency>
        </dependencies>
    </profile>
</profiles>
```

**Gradle:**

```groovy
// Method 1: Project properties
def env = project.hasProperty('env') ? project.env : 'dev'

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    
    if (env == 'dev') {
        runtimeOnly 'com.h2database:h2'
    } else if (env == 'prod') {
        runtimeOnly 'org.postgresql:postgresql'
    }
}

// Usage: ./gradlew build -Penv=prod

// Method 2: Source sets
sourceSets {
    dev
    prod
}

configurations {
    devImplementation.extendsFrom implementation
    prodImplementation.extendsFrom implementation
}

dependencies {
    devRuntimeOnly 'com.h2database:h2'
    prodRuntimeOnly 'org.postgresql:postgresql'
}
```

## 3. Multi-Module Migration

### 3.1 Maven Multi-Module

**Maven Structure:**

```
parent/
├── pom.xml (parent)
├── module-a/
│   └── pom.xml
├── module-b/
│   └── pom.xml
└── module-c/
    └── pom.xml
```

**Parent pom.xml:**

```xml
<project>
    <groupId>com.example</groupId>
    <artifactId>parent</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>
    
    <modules>
        <module>module-a</module>
        <module>module-b</module>
        <module>module-c</module>
    </modules>
    
    <properties>
        <java.version>17</java.version>
    </properties>
    
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>com.google.guava</groupId>
                <artifactId>guava</artifactId>
                <version>32.1.3-jre</version>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>
```

**Module pom.xml:**

```xml
<project>
    <parent>
        <groupId>com.example</groupId>
        <artifactId>parent</artifactId>
        <version>1.0.0</version>
    </parent>
    
    <artifactId>module-a</artifactId>
    
    <dependencies>
        <dependency>
            <groupId>com.google.guava</groupId>
            <artifactId>guava</artifactId>
            <!-- Version from parent -->
        </dependency>
    </dependencies>
</project>
```

### 3.2 Gradle Multi-Module Equivalent

**Gradle Structure:**

```
root/
├── settings.gradle
├── build.gradle (root)
├── module-a/
│   └── build.gradle
├── module-b/
│   └── build.gradle
└── module-c/
    └── build.gradle
```

**settings.gradle:**

```groovy
rootProject.name = 'parent'

include 'module-a'
include 'module-b'
include 'module-c'
```

**Root build.gradle:**

```groovy
plugins {
    id 'java' apply false
}

allprojects {
    group = 'com.example'
    version = '1.0.0'
    
    repositories {
        mavenCentral()
    }
}

subprojects {
    apply plugin: 'java'
    
    java {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }
    
    dependencies {
        // Common dependencies
        testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
    }
    
    test {
        useJUnitPlatform()
    }
}
```

**Module build.gradle:**

```groovy
dependencies {
    implementation 'com.google.guava:guava:32.1.3-jre'
}
```

## 4. Gradual Migration Strategy

### 4.1 Coexistence Approach

**Phase 1: Preparation**

```
1. Keep Maven build working
2. Add Gradle build alongside
3. Ensure both produce identical artifacts
4. Run both in CI/CD
```

**Directory Structure During Migration:**

```
project/
├── pom.xml                    # Maven (keep working)
├── build.gradle               # Gradle (new)
├── settings.gradle            # Gradle
├── gradlew, gradlew.bat       # Gradle wrapper
├── .mvn/                      # Maven wrapper (keep)
├── gradle/                    # Gradle wrapper
├── module-a/
│   ├── pom.xml               # Maven
│   └── build.gradle          # Gradle
└── module-b/
    ├── pom.xml               # Maven
    └── build.gradle          # Gradle
```

---

### 4.2 Module-by-Module Migration

**Step 1: Migrate Leaf Modules First**

```
Dependency Tree:
  common (no dependencies) → Migrate first
     ↑
  service-a (depends on common) → Migrate second
     ↑
  web (depends on service-a) → Migrate last
```

**Step 2: Convert One Module**

```bash
# In module directory
cd module-a
gradle init

# Test Gradle build
../gradlew build

# Compare outputs
diff target/module-a-1.0.0.jar build/libs/module-a-1.0.0.jar
```

**Step 3: Update Dependent Modules**

```groovy
// Other Gradle modules
dependencies {
    implementation project(':module-a')
}
```

**Step 4: Publish for Maven Modules**

```groovy
// module-a/build.gradle
plugins {
    id 'maven-publish'
}

publishing {
    publications {
        maven(MavenPublication) {
            from components.java
        }
    }
    repositories {
        maven {
            url = "${rootProject.buildDir}/repo"
        }
    }
}

// Other Maven modules can use this
// mvn install:install-file ...
```

---

### 4.3 Parallel Builds in CI/CD

```yaml
# .github/workflows/build.yml
name: Build

on: [push, pull_request]

jobs:
  maven-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-java@v3
        with:
          java-version: '17'
      - name: Maven Build
        run: mvn clean verify
  
  gradle-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-java@v3
        with:
          java-version: '17'
      - name: Gradle Build
        run: ./gradlew build
      
      # Compare artifacts
      - name: Compare Outputs
        run: |
          diff target/*.jar build/libs/*.jar || echo "Artifacts differ!"
```

## 5. Common Migration Challenges

### 5.1 Custom Maven Plugins

**Maven Custom Plugin:**

```xml
<plugin>
    <groupId>com.example</groupId>
    <artifactId>custom-maven-plugin</artifactId>
    <version>1.0.0</version>
    <executions>
        <execution>
            <goals>
                <goal>custom-goal</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

**Gradle Equivalent Options:**

**Option 1: Find Gradle Plugin**

```groovy
// Look for equivalent Gradle plugin
plugins {
    id 'com.example.custom-gradle-plugin' version '1.0.0'
}
```

**Option 2: Create Custom Task**

```groovy
task customGoal {
    doLast {
        // Implement plugin functionality
    }
}
```

**Option 3: Execute Maven Plugin from Gradle**

```groovy
task runMavenPlugin(type: Exec) {
    commandLine 'mvn', 'com.example:custom-maven-plugin:1.0.0:custom-goal'
}
```

---

### 5.2 Assembly/Packaging

**Maven Assembly:**

```xml
<plugin>
    <artifactId>maven-assembly-plugin</artifactId>
    <configuration>
        <descriptors>
            <descriptor>assembly.xml</descriptor>
        </descriptors>
    </configuration>
</plugin>
```

**Gradle Equivalent:**

```groovy
// Option 1: Distribution plugin
plugins {
    id 'distribution'
}

distributions {
    main {
        contents {
            from 'src/dist'
            from jar
            into('lib') {
                from configurations.runtimeClasspath
            }
        }
    }
}

// Option 2: Custom Zip task
task createDistribution(type: Zip) {
    archiveFileName = "${project.name}-${version}.zip"
    destinationDirectory = file("${buildDir}/distributions")
    
    from jar
    from(configurations.runtimeClasspath) {
        into 'lib'
    }
    from('src/dist') {
        into '.'
    }
}
```

---

### 5.3 Resource Filtering

**Maven:**

```xml
<resources>
    <resource>
        <directory>src/main/resources</directory>
        <filtering>true</filtering>
    </resource>
</resources>
```

**Gradle:**

```groovy
processResources {
    filesMatching('**/*.properties') {
        expand(project.properties)
    }
}

// Or for specific properties
processResources {
    inputs.property 'version', version
    
    filesMatching('application.properties') {
        expand(
            'version': version,
            'name': project.name
        )
    }
}
```

## 6. Publishing to Maven Repositories

### 6.1 Maven Publishing Plugin

```groovy
plugins {
    id 'java-library'
    id 'maven-publish'
}

publishing {
    publications {
        maven(MavenPublication) {
            from components.java
            
            groupId = 'com.example'
            artifactId = 'my-library'
            version = '1.0.0'
            
            pom {
                name = 'My Library'
                description = 'A library for doing things'
                url = 'https://github.com/example/my-library'
                
                licenses {
                    license {
                        name = 'The Apache License, Version 2.0'
                        url = 'http://www.apache.org/licenses/LICENSE-2.0.txt'
                    }
                }
                
                developers {
                    developer {
                        id = 'dev'
                        name = 'Developer Name'
                        email = 'dev@example.com'
                    }
                }
                
                scm {
                    connection = 'scm:git:git://github.com/example/my-library.git'
                    developerConnection = 'scm:git:ssh://github.com/example/my-library.git'
                    url = 'https://github.com/example/my-library'
                }
            }
        }
    }
    
    repositories {
        maven {
            name = 'nexus'
            def releasesUrl = 'https://nexus.company.com/repository/maven-releases/'
            def snapshotsUrl = 'https://nexus.company.com/repository/maven-snapshots/'
            url = version.endsWith('SNAPSHOT') ? snapshotsUrl : releasesUrl
            
            credentials {
                username = project.findProperty('nexusUsername') ?: System.getenv('NEXUS_USERNAME')
                password = project.findProperty('nexusPassword') ?: System.getenv('NEXUS_PASSWORD')
            }
        }
    }
}
```

## Summary

### Migration Comparison

```
┌──────────────────────────────────────────────────────────────┐
│               MAVEN vs GRADLE EQUIVALENTS                    │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Maven                    Gradle                             │
│  ─────────────────────────────────────────────────────       │
│  pom.xml                  build.gradle                       │
│  settings.xml             gradle.properties                  │
│  mvn                      ./gradlew                          │
│  <dependencies>           dependencies { }                   │
│  <plugins>                plugins { }                        │
│  <properties>             ext { } or gradle.properties       │
│  <modules>                include in settings.gradle         │
│  <profiles>               Project properties or source sets  │
│  compile scope            implementation                     │
│  provided scope           compileOnly                        │
│  runtime scope            runtimeOnly                        │
│  test scope               testImplementation                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```
