# Dynamic Programming Patterns

## Overview

Dynamic programming solves problems by breaking them into overlapping subproblems, solving each once, and storing the result. The key insight: if a problem has **optimal substructure** (optimal solution built from optimal solutions to subproblems) and **overlapping subproblems** (same subproblems solved repeatedly), DP transforms exponential brute force into polynomial time.

**The mental model:**

```
Brute force:      O(2ⁿ) — recomputes same subproblems exponentially
Memoization:      O(n × states) — caches results, top-down recursion
Tabulation:       O(n × states) — fills table bottom-up, no recursion
Space-optimized:  O(states per layer) — discard rows no longer needed
```

**Two questions to recognize DP:**

```
1. "Count ways to..." / "Find minimum/maximum..." / "Is it possible to..."
2. Can the problem be expressed as: "what choices do I have at each step,
   and what's the optimal/total result of all choices?"
```

## 1. Memoization vs Tabulation

### 1.1 Memoization — top-down

Start with the recursive solution, add a cache. Natural for problems where not all subproblems need to be solved.

```java
// Template: top-down memoization
public class Memoization {

    // Step 1: Define state — what uniquely identifies a subproblem?
    // Step 2: Write recursive relation
    // Step 3: Add HashMap or array cache
    // Step 4: Handle base cases before recursion

    private Map<String, Long> memo = new HashMap<>();

    public long solve(int param1, int param2) {
        String key = param1 + "," + param2;
        if (memo.containsKey(key)) return memo.get(key);

        // Base cases
        if (param1 == 0) return 0;
        if (param2 == 0) return 1;

        // Recurrence
        long result = solve(param1 - 1, param2) + solve(param1, param2 - 1);
        memo.put(key, result);
        return result;
    }
}

// Array memo when state is bounded integers — faster than HashMap
public long solveArray(int n, int m, long[][] memo) {
    if (n == 0 || m == 0) return 1; // base case
    if (memo[n][m] != -1) return memo[n][m];

    memo[n][m] = solveArray(n - 1, m, memo) + solveArray(n, m - 1, memo);
    return memo[n][m];
}
```

### 1.2 Tabulation — bottom-up

Fill a table from base cases upward. Avoids recursion stack overhead — essential for large inputs.

```java
// Template: bottom-up tabulation
public long solveTabulation(int n, int m) {
    long[][] dp = new long[n + 1][m + 1];

    // Step 1: Initialize base cases
    for (int i = 0; i <= n; i++) dp[i][0] = 1;
    for (int j = 0; j <= m; j++) dp[0][j] = 1;

    // Step 2: Fill table in dependency order
    // dp[i][j] depends on dp[i-1][j] and dp[i][j-1]
    // → fill left-to-right, top-to-bottom
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1];

    return dp[n][m];
}
```

### 1.3 Space optimization — rolling array

When `dp[i][j]` only depends on the previous row, discard completed rows:

```java
// 2D → 1D space optimization
public long solveSO(int n, int m) {
    long[] dp = new long[m + 1];
    Arrays.fill(dp, 1); // base: dp[0][j] = 1

    for (int i = 1; i <= n; i++) {
        // dp[j] currently holds dp[i-1][j]
        // after update, dp[j] holds dp[i][j]
        for (int j = 1; j <= m; j++) {
            dp[j] += dp[j - 1]; // dp[i][j] = dp[i-1][j] + dp[i][j-1]
            //           ^ dp[j] = prev row          ^ dp[j-1] = current row, already updated
        }
    }
    return dp[m];
}
```

### 1.4 Memoization vs Tabulation comparison

||Memoization|Tabulation|
|---|---|---|
|Direction|Top-down (recursive)|Bottom-up (iterative)|
|Stack overhead|Yes — can stack overflow|No|
|Subproblems solved|Only needed ones|All subproblems|
|Code clarity|Closer to recursive definition|Requires figuring out fill order|
|Space optimization|Harder|Natural (rolling array)|
|Best for|Trees, graphs, irregular state|Sequences, grids, large inputs|

## 2. Fibonacci Variants

Fibonacci is the entry point — every DP pattern generalises it.

### 2.1 Classic Fibonacci — all four forms

```java
// Brute force: O(2ⁿ) — exponential recomputation
int fibBrute(int n) {
    if (n <= 1) return n;
    return fibBrute(n - 1) + fibBrute(n - 2);
}

// Memoization: O(n) time, O(n) space
int fibMemo(int n, int[] memo) {
    if (n <= 1) return n;
    if (memo[n] != 0) return memo[n];
    return memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
}

// Tabulation: O(n) time, O(n) space
int fibTab(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[1] = 1;
    for (int i = 2; i <= n; i++) dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}

// Space-optimized: O(n) time, O(1) space
int fibOpt(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

### 2.2 Climbing stairs (generalised Fibonacci)

```java
// You can climb 1 or 2 steps — how many ways to reach step n?
// dp[n] = dp[n-1] + dp[n-2]  → exactly Fibonacci with offset
public int climbStairs(int n) {
    if (n <= 2) return n;
    int a = 1, b = 2;
    for (int i = 3; i <= n; i++) { int c = a + b; a = b; b = c; }
    return b;
}

// Generalisation: can climb 1..k steps at a time
// dp[i] = sum of dp[i-1] + dp[i-2] + ... + dp[i-k]
// Use sliding window sum to maintain O(n) not O(n×k)
public int climbStairsK(int n, int k) {
    int[] dp = new int[n + 1];
    dp[0] = 1;
    int windowSum = 1; // sum of dp[max(0,i-k)..i-1]

    for (int i = 1; i <= n; i++) {
        dp[i] = windowSum;
        windowSum += dp[i];
        if (i >= k) windowSum -= dp[i - k]; // remove element falling out of window
    }
    return dp[n];
}
```

### 2.3 House robber

```java
// Adjacent houses can't both be robbed — max sum of non-adjacent elements
// dp[i] = max(dp[i-1], dp[i-2] + nums[i])
//   Don't rob i: dp[i-1]
//   Rob i:       dp[i-2] + nums[i]
public int rob(int[] nums) {
    if (nums.length == 1) return nums[0];
    int prev2 = 0, prev1 = 0;
    for (int num : nums) {
        int curr = Math.max(prev1, prev2 + num);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// House robber II — circular array (first and last are adjacent)
// Split into two problems: rob [0..n-2] OR rob [1..n-1], take max
public int robCircular(int[] nums) {
    if (nums.length == 1) return nums[0];
    return Math.max(
        robRange(nums, 0, nums.length - 2),
        robRange(nums, 1, nums.length - 1)
    );
}

private int robRange(int[] nums, int start, int end) {
    int prev2 = 0, prev1 = 0;
    for (int i = start; i <= end; i++) {
        int curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1; prev1 = curr;
    }
    return prev1;
}
```

### 2.4 Decode ways (Fibonacci variant with constraints)

```java
// "12" can decode as "AB"(1,2) or "L"(12) — count valid decodings
// dp[i] = ways to decode s[0..i-1]
// dp[i] += dp[i-1]  if s[i-1] is valid single digit (1-9)
// dp[i] += dp[i-2]  if s[i-2..i-1] forms valid two-digit (10-26)
public int numDecodings(String s) {
    int n = s.length();
    int[] dp = new int[n + 1];
    dp[0] = 1;                                 // empty string = 1 way
    dp[1] = s.charAt(0) == '0' ? 0 : 1;       // single char

    for (int i = 2; i <= n; i++) {
        int oneDigit = Integer.parseInt(s.substring(i - 1, i));
        int twoDigit = Integer.parseInt(s.substring(i - 2, i));

        if (oneDigit >= 1)                dp[i] += dp[i - 1];
        if (twoDigit >= 10 && twoDigit <= 26) dp[i] += dp[i - 2];
    }
    return dp[n];
}
```

## 3. Knapsack Problem

The knapsack pattern covers a family of problems: given items with weights and values, select items subject to a capacity constraint to maximise value.

### 3.1 0/1 Knapsack — each item used at most once

```java
// dp[i][w] = max value using first i items with capacity w
// Choice: skip item i → dp[i-1][w]
//         take item i → dp[i-1][w - weight[i]] + value[i]  (if weight[i] <= w)
public int knapsack01(int[] weights, int[] values, int capacity) {
    int n = weights.length;
    int[][] dp = new int[n + 1][capacity + 1];

    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= capacity; w++) {
            dp[i][w] = dp[i - 1][w]; // skip item i
            if (weights[i - 1] <= w) {
                dp[i][w] = Math.max(dp[i][w],
                    dp[i - 1][w - weights[i - 1]] + values[i - 1]); // take item i
            }
        }
    }
    return dp[n][capacity];
}

// Space-optimized 0/1 knapsack — 1D dp, iterate capacity BACKWARDS
// Backwards prevents item from being used twice in same pass
public int knapsack01Optimized(int[] weights, int[] values, int capacity) {
    int[] dp = new int[capacity + 1];

    for (int i = 0; i < weights.length; i++) {
        for (int w = capacity; w >= weights[i]; w--) { // ← RIGHT TO LEFT
            dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
```

**Why iterate backwards for 0/1 knapsack:**

```
Forward pass (wrong for 0/1):
  When computing dp[w], dp[w - weight[i]] may already be updated
  in this pass → item i could be counted twice

Backward pass (correct):
  When computing dp[w], dp[w - weight[i]] still holds the value
  from before this item was considered → item i counted at most once
```

### 3.2 Unbounded knapsack — each item usable unlimited times

```java
// Each item can be used any number of times
// dp[w] = max value achievable with capacity exactly w
// Iterate capacity FORWARDS — allows reusing same item
public int unboundedKnapsack(int[] weights, int[] values, int capacity) {
    int[] dp = new int[capacity + 1];

    for (int w = 1; w <= capacity; w++) {
        for (int i = 0; i < weights.length; i++) {
            if (weights[i] <= w) {
                dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
            }
        }
    }
    return dp[capacity];
}

// Coin Change — minimum coins to make amount (unbounded knapsack variant)
// dp[w] = min coins to make amount w
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1); // "infinity"
    dp[0] = 0;

    for (int w = 1; w <= amount; w++) {
        for (int coin : coins) {
            if (coin <= w) {
                dp[w] = Math.min(dp[w], dp[w - coin] + 1);
            }
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}

// Coin change II — count ways to make amount (order doesn't matter)
// Outer loop: coins (ensures no duplicate combinations like [1,2] and [2,1])
// Inner loop: amounts
public int coinChangeWays(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    dp[0] = 1; // one way to make 0: use no coins

    for (int coin : coins) {           // ← iterate coins in outer loop
        for (int w = coin; w <= amount; w++) {
            dp[w] += dp[w - coin];
        }
    }
    return dp[amount];
}
```

### 3.3 Subset sum and partition problems

```java
// Subset Sum — does any subset sum to target?
// dp[w] = true if some subset sums to w
public boolean subsetSum(int[] nums, int target) {
    boolean[] dp = new boolean[target + 1];
    dp[0] = true;

    for (int num : nums) {
        for (int w = target; w >= num; w--) { // backwards for 0/1
            dp[w] = dp[w] || dp[w - num];
        }
    }
    return dp[target];
}

// Partition Equal Subset Sum (LeetCode 416)
// Can we split array into two equal-sum halves?
// Equivalent to: does any subset sum to total/2?
public boolean canPartition(int[] nums) {
    int total = Arrays.stream(nums).sum();
    if (total % 2 != 0) return false;
    return subsetSum(nums, total / 2);
}

// Target sum — assign +/- to each number, count ways to reach target
// Insight: let P = sum of positive numbers, N = sum of negative
//   P + N = total, P - N = target → P = (total + target) / 2
// → count subsets summing to P (coin change II style)
public int findTargetSumWays(int[] nums, int target) {
    int total = Arrays.stream(nums).sum();
    if ((total + target) % 2 != 0 || Math.abs(target) > total) return 0;
    int subsetTarget = (total + target) / 2;

    int[] dp = new int[subsetTarget + 1];
    dp[0] = 1;
    for (int num : nums)
        for (int w = subsetTarget; w >= num; w--)
            dp[w] += dp[w - num];
    return dp[subsetTarget];
}
```

### 3.4 Knapsack pattern summary

```
"Can we achieve exactly X?"          → boolean dp[], subset sum
"How many ways to achieve X?"        → count dp[], coin change II style
"Maximum value within weight W?"     → max dp[], classic knapsack
"Minimum items/cost to achieve X?"   → min dp[], coin change I style

Each item used at most once:          → iterate capacity backwards
Each item used unlimited:            → iterate capacity forwards
```

## 4. Longest Common Subsequence

LCS and its variants form a major DP family: edit distance, longest common substring, shortest common supersequence, and many others share the same core recurrence.

### 4.1 LCS — core recurrence

```java
// LCS(i,j) = length of LCS of s1[0..i-1] and s2[0..j-1]
//
// If s1[i-1] == s2[j-1]: LCS(i,j) = LCS(i-1,j-1) + 1
// Else:                   LCS(i,j) = max(LCS(i-1,j), LCS(i,j-1))

public int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i - 1) == s2.charAt(j - 1)) {
                dp[i][j] = dp[i - 1][j - 1] + 1; // characters match
            } else {
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]); // skip one
            }
        }
    }
    return dp[m][n];
}

// Reconstruct the LCS string
public String lcsString(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (s1.charAt(i - 1) == s2.charAt(j - 1)) dp[i][j] = dp[i-1][j-1] + 1;
            else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);

    // Backtrack from dp[m][n]
    StringBuilder sb = new StringBuilder();
    int i = m, j = n;
    while (i > 0 && j > 0) {
        if (s1.charAt(i - 1) == s2.charAt(j - 1)) {
            sb.append(s1.charAt(i - 1));
            i--; j--;
        } else if (dp[i - 1][j] > dp[i][j - 1]) i--;
        else j--;
    }
    return sb.reverse().toString();
}

// Space-optimized LCS — only need previous row
public int lcsSpaceOpt(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[] prev = new int[n + 1], curr = new int[n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i - 1) == s2.charAt(j - 1)) curr[j] = prev[j - 1] + 1;
            else curr[j] = Math.max(prev[j], curr[j - 1]);
        }
        int[] tmp = prev; prev = curr; curr = tmp; // swap rows
        Arrays.fill(curr, 0);
    }
    return prev[n];
}
```

### 4.2 Edit Distance (Levenshtein)

```java
// Min operations (insert, delete, replace) to transform s1 into s2
// dp[i][j] = edit distance between s1[0..i-1] and s2[0..j-1]
//
// If s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] (free — chars match)
// Else:  dp[i][j] = 1 + min(
//           dp[i-1][j],   // delete from s1
//           dp[i][j-1],   // insert into s1
//           dp[i-1][j-1]  // replace in s1
//        )

public int editDistance(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];

    // Base cases: transform prefix to empty string
    for (int i = 0; i <= m; i++) dp[i][0] = i; // delete all of s1
    for (int j = 0; j <= n; j++) dp[0][j] = j; // insert all of s2

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i - 1) == s2.charAt(j - 1)) {
                dp[i][j] = dp[i - 1][j - 1];
            } else {
                dp[i][j] = 1 + Math.min(dp[i - 1][j - 1],
                               Math.min(dp[i - 1][j], dp[i][j - 1]));
            }
        }
    }
    return dp[m][n];
}
```

### 4.3 Longest Common Substring (contiguous)

```java
// Unlike LCS, substring must be contiguous
// dp[i][j] = length of longest common substring ending at s1[i-1], s2[j-1]
// Only extends on match (no "skip" option)
public int longestCommonSubstring(String s1, String s2) {
    int m = s1.length(), n = s2.length(), maxLen = 0;
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i - 1) == s2.charAt(j - 1)) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
                maxLen = Math.max(maxLen, dp[i][j]);
            }
            // No else — mismatching chars reset to 0 (implicit from new array)
        }
    }
    return maxLen;
}
```

### 4.4 Shortest Common Supersequence

```java
// Shortest string containing both s1 and s2 as subsequences
// Length = m + n - LCS(s1, s2)
public String shortestCommonSupersequence(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (s1.charAt(i-1) == s2.charAt(j-1)) dp[i][j] = dp[i-1][j-1] + 1;
            else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);

    // Reconstruct: include LCS chars once, non-LCS chars from both strings
    StringBuilder sb = new StringBuilder();
    int i = m, j = n;
    while (i > 0 && j > 0) {
        if (s1.charAt(i-1) == s2.charAt(j-1)) {
            sb.append(s1.charAt(i-1)); i--; j--;       // shared char — take once
        } else if (dp[i-1][j] > dp[i][j-1]) {
            sb.append(s1.charAt(i-1)); i--;             // take from s1
        } else {
            sb.append(s2.charAt(j-1)); j--;             // take from s2
        }
    }
    while (i > 0) { sb.append(s1.charAt(i-1)); i--; }  // remaining s1
    while (j > 0) { sb.append(s2.charAt(j-1)); j--; }  // remaining s2
    return sb.reverse().toString();
}
```

### 4.5 Longest Increasing Subsequence

```java
// LIS — O(n²) DP
// dp[i] = length of LIS ending at index i
public int lisDP(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1); // each element is a LIS of length 1

    int maxLen = 1;
    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
        maxLen = Math.max(maxLen, dp[i]);
    }
    return maxLen;
}

// LIS — O(n log n) with binary search (patience sorting)
// tails[i] = smallest tail element of all increasing subsequences of length i+1
public int lisOptimal(int[] nums) {
    List<Integer> tails = new ArrayList<>();

    for (int num : nums) {
        // Binary search for first tail >= num
        int lo = 0, hi = tails.size();
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (tails.get(mid) < num) lo = mid + 1;
            else hi = mid;
        }
        if (lo == tails.size()) tails.add(num);  // extend
        else tails.set(lo, num);                  // replace (maintain smallest tails)
    }
    return tails.size();
}
```

## 5. Matrix Chain Multiplication

Matrix chain multiplication is the canonical **interval DP** problem — the optimal split of a range `[i, j]` is derived from optimal splits of sub-ranges. The same pattern applies to burst balloons, palindrome partitioning, and optimal BST construction.

### 5.1 Matrix chain — O(n³)

```java
// Given matrices A₁, A₂, ..., Aₙ with dimensions p[0]×p[1], p[1]×p[2], ..., p[n-1]×p[n]
// Find the minimum number of scalar multiplications to compute A₁×A₂×...×Aₙ
// Key: matrix multiplication is associative but the cost depends on parenthesization
//
// Cost of Aᵢ (pᵢ₋₁×pᵢ) × Aⱼ (pⱼ₋₁×pⱼ):
//   If Aᵢ..ₖ has dim pᵢ₋₁×pₖ and Aₖ₊₁..ⱼ has dim pₖ×pⱼ:
//   cost = pᵢ₋₁ × pₖ × pⱼ  (scalar multiplications for this split)
//
// dp[i][j] = min cost to multiply matrices i through j (1-indexed)
// dp[i][j] = min over k in [i, j-1] of:
//            dp[i][k] + dp[k+1][j] + p[i-1] × p[k] × p[j]

public int matrixChain(int[] p) {
    int n = p.length - 1; // number of matrices
    int[][] dp = new int[n + 1][n + 1]; // dp[i][j] = min cost for matrices i..j

    // Base case: single matrix, 0 cost
    // dp[i][i] = 0 (already initialised)

    // Fill by increasing chain length
    for (int len = 2; len <= n; len++) {           // chain length
        for (int i = 1; i <= n - len + 1; i++) {  // start index
            int j = i + len - 1;                   // end index
            dp[i][j] = Integer.MAX_VALUE;

            for (int k = i; k < j; k++) {          // split point
                int cost = dp[i][k] + dp[k + 1][j]
                         + p[i - 1] * p[k] * p[j];
                dp[i][j] = Math.min(dp[i][j], cost);
            }
        }
    }
    return dp[1][n];
}
```

**Trace for matrices with dims [10, 30, 5, 60] (3 matrices: 10×30, 30×5, 5×60):**

```
p = [10, 30, 5, 60], n = 3

Base: dp[1][1]=0, dp[2][2]=0, dp[3][3]=0

len=2:
  dp[1][2]: k=1: dp[1][1]+dp[2][2]+p[0]*p[1]*p[2] = 0+0+10*30*5 = 1500
  dp[2][3]: k=2: dp[2][2]+dp[3][3]+p[1]*p[2]*p[3] = 0+0+30*5*60 = 9000

len=3:
  dp[1][3]: 
    k=1: dp[1][1]+dp[2][3]+p[0]*p[1]*p[3] = 0+9000+10*30*60 = 27000
    k=2: dp[1][2]+dp[3][3]+p[0]*p[2]*p[3] = 1500+0+10*5*60  =  4500
  dp[1][3] = min(27000, 4500) = 4500

Optimal: ((A₁×A₂)×A₃) costs 4500 multiplications
```

### 5.2 Interval DP pattern generalisation

```java
// Burst Balloons (LeetCode 312) — interval DP
// dp[i][j] = max coins from bursting all balloons in (i..j) exclusive
// Think LAST balloon to burst in range [i+1..j-1] → gains nums[i]*nums[k]*nums[j]
public int maxCoins(int[] nums) {
    int n = nums.length;
    int[] balls = new int[n + 2];
    balls[0] = balls[n + 1] = 1;
    for (int i = 1; i <= n; i++) balls[i] = nums[i - 1];

    int[][] dp = new int[n + 2][n + 2];

    for (int len = 1; len <= n; len++) {
        for (int left = 1; left <= n - len + 1; left++) {
            int right = left + len - 1;
            for (int k = left; k <= right; k++) { // k = last to burst in [left,right]
                int coins = balls[left - 1] * balls[k] * balls[right + 1]
                          + dp[left][k - 1] + dp[k + 1][right];
                dp[left][right] = Math.max(dp[left][right], coins);
            }
        }
    }
    return dp[1][n];
}

// Palindrome Partitioning II — min cuts to partition string into palindromes
// Interval DP: isPalin[i][j] + DP on cut positions
public int minCut(String s) {
    int n = s.length();
    boolean[][] isPalin = new boolean[n][n];

    // Precompute palindromes — all lengths
    for (int len = 1; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            isPalin[i][j] = s.charAt(i) == s.charAt(j)
                && (len <= 2 || isPalin[i + 1][j - 1]);
        }

    int[] dp = new int[n]; // dp[i] = min cuts for s[0..i]
    for (int i = 0; i < n; i++) {
        if (isPalin[0][i]) { dp[i] = 0; continue; } // whole prefix is palindrome

        dp[i] = i; // worst case: cut before every character
        for (int j = 1; j <= i; j++) {
            if (isPalin[j][i]) {
                dp[i] = Math.min(dp[i], dp[j - 1] + 1);
            }
        }
    }
    return dp[n - 1];
}
```

## 6. Recognizing DP Problems

### 6.1 The four signals

```
Signal 1: Optimization — "minimum", "maximum", "longest", "shortest", "largest"
Signal 2: Counting — "how many ways", "count the number of"
Signal 3: Feasibility — "is it possible", "can we achieve"
Signal 4: The choices compound — each decision affects future options

NOT DP (usually):
  "Find all solutions" → backtracking
  "Find any one solution" → greedy or backtracking
  "Data changes dynamically" → balanced BST or segment tree
```

### 6.2 DP vs Greedy vs Divide-and-Conquer

```
Greedy: make locally optimal choice at each step, never reconsider.
  Works when: greedy choice property holds (local = global optimal)
  Examples: activity selection, Huffman coding, Dijkstra's

Divide & Conquer: split into INDEPENDENT subproblems, combine results.
  Subproblems DON'T overlap — no need to cache
  Examples: MergeSort, QuickSort, binary search

Dynamic Programming: subproblems OVERLAP and have optimal substructure.
  Cache overlapping subproblems — each computed once
  Examples: knapsack, LCS, shortest paths with negative weights

Test: if the same subproblem is solved multiple times in the recursion tree → DP
```

### 6.3 State design — the hardest part

```
Step 1: What changes between calls? → Those are your state variables
Step 2: What are you returning? → The "value" of each state
Step 3: What choices do you make at each state? → Transitions
Step 4: What are the base cases? → When you stop recursing

Common state patterns:
  dp[i]              — up to index i
  dp[i][j]           — between indices i and j (interval DP)
  dp[i][j]           — grid cell (i, j)
  dp[i][capacity]    — up to item i with remaining capacity
  dp[i][last]        — at position i, last choice was "last"
  dp[mask]           — bitmask of chosen items (bitmask DP)
  dp[i][j][k]        — three-dimensional state (3 varying constraints)
```

### 6.4 Common DP patterns taxonomy

```
Linear DP (1D):
  dp[i] depends on dp[i-1], dp[i-2], ...
  → Fibonacci, climbing stairs, house robber, coin change

Knapsack (capacity dimension):
  dp[i][w] — item and capacity
  → 0/1 knapsack, subset sum, partition, target sum

String/Sequence DP (2D):
  dp[i][j] — two pointers into one or two strings
  → LCS, edit distance, LIS, palindrome problems

Interval DP:
  dp[i][j] — range [i..j]
  → matrix chain, burst balloons, palindrome partition, optimal BST

Grid DP:
  dp[r][c] — grid cell
  → unique paths, minimum path sum, dungeon game

Tree DP:
  dp[node][state] — defined recursively on tree structure
  → house robber on tree, diameter, path sum

Bitmask DP:
  dp[mask] — subset of elements chosen
  → TSP, minimum cost to cover all, assignment problems
  → Only feasible for n ≤ 20

State machine DP:
  dp[i][state] — position + discrete state
  → best time to buy/sell stock (held/not-held/cooldown)
  → regex matching (in-wildcard/not)
```

### 6.5 Stock problems — state machine DP

```java
// Best Time to Buy and Sell Stock with Cooldown (LeetCode 309)
// States: HELD (own stock), SOLD (just sold — cooldown), REST (available to buy)
public int maxProfitCooldown(int[] prices) {
    int held = Integer.MIN_VALUE; // max profit while holding stock
    int sold = 0;                 // max profit day after selling (in cooldown)
    int rest = 0;                 // max profit while resting (available to buy)

    for (int price : prices) {
        int prevHeld = held, prevSold = sold, prevRest = rest;

        held = Math.max(prevHeld, prevRest - price);  // keep holding OR buy
        sold = prevHeld + price;                       // sell today
        rest = Math.max(prevRest, prevSold);           // keep resting OR cooldown ends
    }
    return Math.max(sold, rest);
}

// Best Time to Buy and Sell Stock IV (at most k transactions)
// dp[k][0] = max profit, k transactions, not holding
// dp[k][1] = max profit, k transactions, holding
public int maxProfitK(int k, int[] prices) {
    int[][] dp = new int[k + 1][2];
    for (int[] row : dp) row[1] = Integer.MIN_VALUE; // holding = -infinity initially

    for (int price : prices) {
        for (int t = k; t >= 1; t--) {
            dp[t][0] = Math.max(dp[t][0], dp[t][1] + price); // sell
            dp[t][1] = Math.max(dp[t][1], dp[t-1][0] - price); // buy
        }
    }
    return dp[k][0];
}
```

### 6.6 Grid DP

```java
// Unique paths in grid — can only move right or down
// dp[r][c] = paths[r-1][c] + paths[r][c-1]
public int uniquePaths(int m, int n) {
    int[] dp = new int[n];
    Arrays.fill(dp, 1); // first row: all 1s

    for (int r = 1; r < m; r++)
        for (int c = 1; c < n; c++)
            dp[c] += dp[c - 1]; // dp[c] = above + left

    return dp[n - 1];
}

// Minimum path sum — each cell has a cost
public int minPathSum(int[][] grid) {
    int m = grid.length, n = grid[0].length;
    int[] dp = new int[n];
    dp[0] = grid[0][0];

    for (int c = 1; c < n; c++) dp[c] = dp[c - 1] + grid[0][c]; // first row

    for (int r = 1; r < m; r++) {
        dp[0] += grid[r][0]; // first column
        for (int c = 1; c < n; c++)
            dp[c] = Math.min(dp[c], dp[c - 1]) + grid[r][c]; // min(above, left)
    }
    return dp[n - 1];
}

// Dungeon game — find min initial health to reach bottom-right
// Must work BACKWARDS: dp[r][c] = min health needed at cell (r,c)
public int calculateMinimumHP(int[][] dungeon) {
    int m = dungeon.length, n = dungeon[0].length;
    int[][] dp = new int[m + 1][n + 1];
    // Sentinel: beyond last cell, need 1 health
    for (int[] row : dp) Arrays.fill(row, Integer.MAX_VALUE);
    dp[m][n - 1] = dp[m - 1][n] = 1; // adjacent to exit

    for (int r = m - 1; r >= 0; r--) {
        for (int c = n - 1; c >= 0; c--) {
            int need = Math.min(dp[r + 1][c], dp[r][c + 1]) - dungeon[r][c];
            dp[r][c] = Math.max(need, 1); // must have at least 1 health
        }
    }
    return dp[0][0];
}
```

### 6.7 Bitmask DP

```java
// Minimum cost to visit all cities exactly once (TSP-like)
// dp[mask][i] = min cost to have visited cities in 'mask', currently at city i
// O(2ⁿ × n²) — feasible for n ≤ 20
public int minCostVisitAll(int[][] cost, int n) {
    int FULL = (1 << n) - 1;
    int[][] dp = new int[1 << n][n];
    for (int[] row : dp) Arrays.fill(row, Integer.MAX_VALUE / 2);
    dp[1][0] = 0; // start at city 0, only city 0 visited

    for (int mask = 1; mask <= FULL; mask++) {
        for (int u = 0; u < n; u++) {
            if ((mask & (1 << u)) == 0) continue; // u not in current set
            if (dp[mask][u] == Integer.MAX_VALUE / 2) continue;

            for (int v = 0; v < n; v++) {
                if ((mask & (1 << v)) != 0) continue; // v already visited
                int nextMask = mask | (1 << v);
                dp[nextMask][v] = Math.min(dp[nextMask][v], dp[mask][u] + cost[u][v]);
            }
        }
    }

    int result = Integer.MAX_VALUE;
    for (int u = 0; u < n; u++)
        result = Math.min(result, dp[FULL][u]); // end at any city
    return result;
}
```

## 7. Debugging DP Solutions

```
Common mistakes and fixes:

1. Wrong base case
   Symptom: correct for small inputs, wrong for larger
   Fix: trace manually for n=0, n=1, n=2

2. Off-by-one in array dimensions
   Symptom: ArrayIndexOutOfBoundsException or wrong answer at boundary
   Fix: dp[n+1] not dp[n] when indices run 1..n

3. Wrong iteration order (0/1 knapsack vs unbounded)
   Symptom: values counted multiple times
   Fix: backwards for 0/1 (each item once), forwards for unbounded

4. Integer overflow
   Symptom: negative or impossibly large values
   Fix: use long, or cap at Integer.MAX_VALUE/2 before adding

5. Not initializing "infinity" correctly
   Symptom: minimum queries return 0 instead of correct answer
   Fix: Arrays.fill(dp, Integer.MAX_VALUE) or large sentinel

6. Forgetting to update the result variable
   Symptom: only returns dp[n], missing dp[0..n-1] candidates
   Fix: track maxLen / minCost across all states, not just final state

// Debugging trick: print the DP table for small inputs
void printDP(int[][] dp) {
    for (int[] row : dp)
        System.out.println(Arrays.toString(row));
}
```

## 8. Interview Questions

**Q: What is the difference between memoization and tabulation, and when would you prefer each?**

A: Both achieve the same asymptotic complexity by ensuring each subproblem is solved once. Memoization is top-down — it starts with the original problem, recurses into subproblems, and caches results in a HashMap or array, solving only the subproblems that are actually needed. Tabulation is bottom-up — it fills a table from base cases upward, always solving all subproblems in dependency order. Prefer memoization when only a fraction of subproblems are needed (e.g., sparse graph DP, tree DP), when the dependency order is complex to determine, or when recursion maps naturally to the problem definition. Prefer tabulation for large inputs where recursion would overflow the call stack, when space optimization via rolling arrays is needed, and when all subproblems will be needed anyway.

---

**Q: Why does 0/1 knapsack iterate capacity backwards in the space-optimized version?**

A: In the 2D version, `dp[i][w]` uses `dp[i-1][w - weight]` — the previous row. When compressed to 1D, `dp[w]` is reused to represent the current row, so we must ensure that when computing `dp[w]`, the value `dp[w - weight]` still holds the previous row's value (not the current row's updated value). By iterating capacity from high to low, when we compute `dp[w]`, we haven't yet updated `dp[w - weight]` (since `w - weight < w`) — so it still holds the previous item's result. If we iterated forward, `dp[w - weight]` would already be updated with the current item, potentially counting the same item twice — which is the unbounded knapsack behaviour.

---

**Q: What is the key insight of interval DP, and what makes matrix chain multiplication a canonical example?**

A: Interval DP solves problems where the optimal solution for a range `[i, j]` depends on the optimal solutions for all sub-ranges `[i, k]` and `[k+1, j]`. The key insight is that you enumerate the "last operation" or "split point" within the range and combine results from sub-ranges. Matrix chain multiplication is canonical because the choice of parenthesization is equivalent to choosing a split point — at each split, the cost is the product of the outer dimensions plus the recursive costs of each half. The recurrence `dp[i][j] = min over k of dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j]` directly encodes this, and filling by increasing interval length guarantees sub-ranges are solved before the ranges that need them. The same structure appears in burst balloons, palindrome partitioning, and optimal BST.

---

**Q: How do you recognise that a problem requires DP rather than greedy or backtracking?**

A: The three key indicators are overlapping subproblems, optimal substructure, and the inability of a greedy choice to guarantee global optimality. Check for overlapping subproblems by drawing the recursion tree — if the same function call with the same arguments appears multiple times, DP is needed. Check for optimal substructure by asking: "can I build the optimal solution from optimal solutions to smaller versions of the same problem?" If yes, DP or greedy applies. To distinguish DP from greedy, ask: "does making the locally optimal choice now ever prevent a better global solution?" If yes — like in the knapsack problem where taking a high-value-per-weight item can prevent fitting items that combine for greater value — you need DP's full exploration. Backtracking is appropriate when you need all solutions or when the search space is too irregular for DP's structured states.

---

**Q: Explain the LCS recurrence — why does it correctly compute the longest common subsequence?**

A: The recurrence `dp[i][j]` represents the LCS length for `s1[0..i-1]` and `s2[0..j-1]`. When `s1[i-1] == s2[j-1]`, the last characters match — we must include them in the LCS (since including them never makes the answer worse), giving `dp[i-1][j-1] + 1`. When they don't match, we must skip the last character of one string or the other — the best we can do is the maximum of skipping `s1`'s last character (`dp[i-1][j]`) or `s2`'s last character (`dp[i][j-1]`). This correctly solves all subproblems because the LCS of the full strings is built from LCS of its prefixes, and every subproblem `dp[i][j]` is smaller than `dp[m][n]` — ensuring the table can be filled bottom-up with no circular dependencies.

## 9. Key Takeaways

- **Memoization**: top-down, cache-on-demand, natural recursion — use for sparse subproblems and tree DP
- **Tabulation**: bottom-up, fills all states, no stack overhead — use for large inputs and rolling array optimization
- **0/1 Knapsack**: iterate capacity **backwards** (prevent reuse); **Unbounded**: iterate **forwards** (allow reuse)
- **LCS family**: match → diagonal + 1; mismatch → max(up, left) — same template for edit distance (with +1 cost on mismatch)
- **Interval DP**: fill by increasing length; recurrence enumerates the "last operation" split point `k`
- **State design** is the hardest part: identify what changes, what you return, what choices you make
- **DP taxonomy**: linear (Fibonacci), knapsack (capacity), string (LCS), interval (matrix chain), grid, tree, bitmask, state machine (stocks)
- **Bitmask DP** only for n ≤ 20 (2²⁰ ≈ 1M states); **3D DP** rarely beyond n = 500
- **Space optimization**: any DP where `dp[i][j]` only uses `dp[i-1][...]` can be reduced to 1D with a rolling array
