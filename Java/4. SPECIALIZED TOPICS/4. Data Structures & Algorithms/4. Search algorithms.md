# Search Algorithms

## Overview

Search algorithms find a target value within a data structure, or determine whether a path or pattern exists. The right algorithm depends on three axes: **data structure** (sorted array, graph, string), **query type** (exact match, prefix, pattern, shortest path), and **constraints** (preprocessing cost vs query speed, memory).

**The search landscape:**

```
Sorted array         → Binary Search and variants — O(log n)
Rotated sorted array → Modified Binary Search — O(log n)
String pattern       → KMP, Rabin-Karp — O(n + m)
Prefix queries       → Trie — O(L)
Weighted graph       → A* — O(E log V) with good heuristic
Unsorted             → Linear scan — O(n) — no better option
```

## 1. Binary Search Variations

Binary search eliminates half the search space per step by comparing the target against the middle element of a **sorted** array. The template matters — off-by-one errors in boundary conditions are the single most common binary search bug.

### 1.1 The canonical template

```java
// Template: find exact target, return -1 if not found
public int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;

    while (lo <= hi) {               // ← inclusive bounds: [lo, hi]
        int mid = lo + (hi - lo) / 2; // ← avoids (lo+hi) integer overflow

        if      (arr[mid] == target) return mid;
        else if (arr[mid] < target)  lo = mid + 1; // target in right half
        else                         hi = mid - 1; // target in left half
    }
    return -1; // not found
}
```

### 1.2 Find first occurrence (leftmost position)

```java
// Returns the leftmost index where arr[i] == target, or -1
public int findFirst(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1, result = -1;

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] == target) {
            result = mid;      // record candidate
            hi = mid - 1;      // keep searching LEFT for earlier occurrence
        } else if (arr[mid] < target) lo = mid + 1;
        else                          hi = mid - 1;
    }
    return result;
}
```

### 1.3 Find last occurrence (rightmost position)

```java
// Returns the rightmost index where arr[i] == target, or -1
public int findLast(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1, result = -1;

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] == target) {
            result = mid;      // record candidate
            lo = mid + 1;      // keep searching RIGHT for later occurrence
        } else if (arr[mid] < target) lo = mid + 1;
        else                          hi = mid - 1;
    }
    return result;
}

// Count occurrences using first + last
public int countOccurrences(int[] arr, int target) {
    int first = findFirst(arr, target);
    if (first == -1) return 0;
    return findLast(arr, target) - first + 1;
}
```

### 1.4 Lower bound — first index where arr[i] >= target

```java
// Returns smallest i such that arr[i] >= target
// Equivalent to Java's Collections.binarySearch when element exists,
// but returns insertion point when it doesn't
public int lowerBound(int[] arr, int target) {
    int lo = 0, hi = arr.length; // hi = length (not length-1) — allows "past end"

    while (lo < hi) {            // ← strictly less: [lo, hi)
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] < target) lo = mid + 1;
        else                   hi = mid;   // arr[mid] >= target, keep as candidate
    }
    return lo; // lo == hi: insertion point, range [0, arr.length]
}

// Upper bound — first index where arr[i] > target
public int upperBound(int[] arr, int target) {
    int lo = 0, hi = arr.length;

    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] <= target) lo = mid + 1; // include equals in "left" side
        else                    hi = mid;
    }
    return lo;
}

// Range [lowerBound, upperBound) contains all occurrences of target
// Count = upperBound(target) - lowerBound(target)
```

### 1.5 Binary search on answer (parametric search)

The most powerful binary search pattern: **binary search over the answer space** rather than the array index.

```java
// "Find the minimum X such that condition(X) is true"
// Works when condition is monotone: false, false, ..., true, true, ...

// Example: Koko eating bananas (LeetCode 875)
// Given piles of bananas and H hours, find minimum eating speed K
// such that Koko can eat all bananas in H hours
public int minEatingSpeed(int[] piles, int h) {
    int lo = 1, hi = Arrays.stream(piles).max().getAsInt();

    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (canFinish(piles, h, mid)) hi = mid; // mid works, try smaller
        else                          lo = mid + 1;
    }
    return lo;
}

private boolean canFinish(int[] piles, int h, int speed) {
    int hours = 0;
    for (int pile : piles) hours += (pile + speed - 1) / speed; // ceil division
    return hours <= h;
}

// Template for "minimum satisfying value":
// lo = smallest possible answer, hi = largest possible answer
// condition(mid) = true means mid is a valid answer
// → after loop, lo = minimum valid answer
```

```java
// Example: Minimum days to make m bouquets (LeetCode 1482)
// Binary search on the number of days
public int minDays(int[] bloomDay, int m, int k) {
    if ((long) m * k > bloomDay.length) return -1;
    int lo = 1, hi = Arrays.stream(bloomDay).max().getAsInt();

    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (canMakeBouquets(bloomDay, m, k, mid)) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}

private boolean canMakeBouquets(int[] bloomDay, int m, int k, int day) {
    int bouquets = 0, consecutive = 0;
    for (int d : bloomDay) {
        if (d <= day) { if (++consecutive == k) { bouquets++; consecutive = 0; } }
        else consecutive = 0;
    }
    return bouquets >= m;
}
```

### 1.6 Binary search on floating-point

```java
// Find square root to precision ε — no integer boundary issues
public double sqrtBinarySearch(double x, double eps) {
    double lo = 0, hi = Math.max(1, x);

    while (hi - lo > eps) {           // converge until gap smaller than epsilon
        double mid = (lo + hi) / 2;
        if (mid * mid <= x) lo = mid;
        else                hi = mid;
    }
    return (lo + hi) / 2;
}
// ~50 iterations for eps=1e-15 regardless of x — O(log(range/eps))
```

### 1.7 Binary search complexity

|Variant|Time|Notes|
|---|---|---|
|Exact search|O(log n)|Terminates early on hit|
|First/last occurrence|O(log n)|Always scans to boundary|
|Lower/upper bound|O(log n)|Returns insertion point|
|Parametric search|O(log(range) × f(n))|f(n) = cost of condition check|
|Floating-point|O(log(range/ε))|~50 iters for double precision|

## 2. Searching in Rotated Arrays

A sorted array may be **rotated** at some pivot: `[4,5,6,7,0,1,2]` is `[0,1,2,4,5,6,7]` rotated at index 4. Binary search still works in O(log n) — the key insight is that **at least one half of any split is always sorted**.

### 2.1 Search in rotated sorted array (no duplicates)

```java
// LeetCode 33 — guaranteed no duplicates
public int searchRotated(int[] nums, int target) {
    int lo = 0, hi = nums.length - 1;

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return mid;

        // Determine which half is sorted
        if (nums[lo] <= nums[mid]) {
            // LEFT half [lo..mid] is sorted
            if (nums[lo] <= target && target < nums[mid]) {
                hi = mid - 1; // target is in the sorted left half
            } else {
                lo = mid + 1; // target must be in right half
            }
        } else {
            // RIGHT half [mid..hi] is sorted
            if (nums[mid] < target && target <= nums[hi]) {
                lo = mid + 1; // target is in the sorted right half
            } else {
                hi = mid - 1; // target must be in left half
            }
        }
    }
    return -1;
}
```

**Trace on `[4,5,6,7,0,1,2]`, target = 0:**

```
lo=0, hi=6, mid=3, nums[3]=7
nums[lo=4] <= nums[mid=7] → LEFT [4,5,6,7] is sorted
target=0 not in [4,7) → lo = 4

lo=4, hi=6, mid=5, nums[5]=1
nums[lo=0] <= nums[mid=1] → LEFT [0,1] is sorted
target=0 in [0,1) → hi = 4

lo=4, hi=4, mid=4, nums[4]=0 == target → return 4 ✓
```

### 2.2 Search with duplicates

```java
// LeetCode 81 — may have duplicates → worst case O(n)
// Duplicates make it impossible to always determine which half is sorted
public boolean searchRotatedWithDups(int[] nums, int target) {
    int lo = 0, hi = nums.length - 1;

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return true;

        // Ambiguous case: nums[lo] == nums[mid] == nums[hi]
        // Can't determine sorted half → shrink both boundaries
        if (nums[lo] == nums[mid] && nums[mid] == nums[hi]) {
            lo++; hi--;
        } else if (nums[lo] <= nums[mid]) {
            if (nums[lo] <= target && target < nums[mid]) hi = mid - 1;
            else lo = mid + 1;
        } else {
            if (nums[mid] < target && target <= nums[hi]) lo = mid + 1;
            else hi = mid - 1;
        }
    }
    return false;
}
```

### 2.3 Find minimum in rotated sorted array

```java
// Find the pivot (minimum element) — O(log n)
public int findMin(int[] nums) {
    int lo = 0, hi = nums.length - 1;

    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;

        if (nums[mid] > nums[hi]) {
            // Minimum is in the right half (past the rotation point)
            lo = mid + 1;
        } else {
            // Minimum is in left half (or at mid)
            hi = mid;
        }
    }
    return nums[lo]; // lo == hi = index of minimum
}

// Find rotation index (index of minimum)
// Then any element can be found in O(log n):
//   search left sorted half [0..pivot-1] or right [pivot..n-1]
public int findRotationIndex(int[] nums) {
    int lo = 0, hi = nums.length - 1;
    if (nums[lo] <= nums[hi]) return 0; // not rotated

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] > nums[mid + 1]) return mid + 1; // drop point found
        if (nums[mid] < nums[lo])      hi = mid - 1;
        else                           lo = mid + 1;
    }
    return 0;
}
```

## 3. KMP String Matching

The Knuth-Morris-Pratt algorithm finds all occurrences of a **pattern** of length m in a **text** of length n in O(n + m) — versus O(n × m) for naive search. The key is the **failure function** (also called partial match table or LPS — Longest Proper Prefix which is also Suffix) that allows the algorithm to skip redundant comparisons.

### 3.1 The failure function (LPS array)

```java
// LPS[i] = length of longest proper prefix of pattern[0..i]
//           that is also a suffix
// "Proper" means not the string itself

// Example: pattern = "ABABAB"
// LPS[0] = 0 (no proper prefix/suffix for single char)
// LPS[1] = 0 ("AB" — no proper prefix = suffix)
// LPS[2] = 1 ("ABA" — "A" is both prefix and suffix, length 1)
// LPS[3] = 2 ("ABAB" — "AB" is both prefix and suffix, length 2)
// LPS[4] = 3 ("ABABA" — "ABA" matches, length 3)
// LPS[5] = 4 ("ABABAB" — "ABAB" matches, length 4)
// LPS = [0, 0, 1, 2, 3, 4]

public int[] computeLPS(String pattern) {
    int m = pattern.length();
    int[] lps = new int[m];
    lps[0] = 0;

    int len = 0; // length of previous longest prefix suffix
    int i = 1;

    while (i < m) {
        if (pattern.charAt(i) == pattern.charAt(len)) {
            lps[i++] = ++len; // extend current prefix-suffix
        } else {
            if (len != 0) {
                // Fall back: try shorter prefix-suffix
                len = lps[len - 1]; // don't increment i
            } else {
                lps[i++] = 0; // no prefix-suffix possible
            }
        }
    }
    return lps;
}
```

### 3.2 KMP search

```java
public List<Integer> kmpSearch(String text, String pattern) {
    List<Integer> matches = new ArrayList<>();
    int n = text.length(), m = pattern.length();
    if (m == 0) return matches;

    int[] lps = computeLPS(pattern);
    int i = 0; // index into text
    int j = 0; // index into pattern

    while (i < n) {
        if (text.charAt(i) == pattern.charAt(j)) {
            i++; j++;
        }

        if (j == m) {
            // Full match found
            matches.add(i - j); // start index in text
            j = lps[j - 1];     // use LPS to skip — don't restart from 0
        } else if (i < n && text.charAt(i) != pattern.charAt(j)) {
            if (j != 0) {
                j = lps[j - 1]; // skip back using LPS, don't move i
            } else {
                i++;             // no prefix match at all — advance text
            }
        }
    }
    return matches;
}
```

**Trace on text = "ABABDABABAB", pattern = "ABABAB":**

```
LPS for "ABABAB" = [0, 0, 1, 2, 3, 4]

i=0,j=0: A==A → i=1,j=1
i=1,j=1: B==B → i=2,j=2
i=2,j=2: A==A → i=3,j=3
i=3,j=3: B==B → i=4,j=4
i=4,j=4: D≠A → j=lps[3]=2 (don't reset to 0 — "AB" prefix already matched)
i=4,j=2: D≠A → j=lps[1]=0
i=4,j=0: D≠A → i=5,j=0
i=5,j=0: A==A → i=6,j=1
i=6,j=1: B==B → i=7,j=2
i=7,j=2: A==A → i=8,j=3
i=8,j=3: B==B → i=9,j=4
i=9,j=4: A==A → i=10,j=5
i=10,j=5: B==B → i=11,j=6=m → MATCH at index 5
  j = lps[5] = 4 (reuse "ABAB" prefix already matched)
→ matches: [5]
```

### 3.3 Why KMP is O(n + m)

```
The key: i never decreases. Every character in text is examined at most once.
When a mismatch occurs after j matches, we set j = lps[j-1]:
  - We don't move i backwards
  - We reuse already-matched prefix via LPS
  - j can only decrease to 0 in total as many times as it increased

Total increments of i: n
Total increments of j: at most n (since i always advances with j)
Total decrements of j: at most n (bounded by total increments)
→ O(n) for search + O(m) for LPS construction = O(n + m)
```

## 4. Rabin-Karp Algorithm

Rabin-Karp uses **rolling hash** to compare the pattern against substrings in O(1) per slide (after O(m) initial hash). It achieves O(n + m) average-case and is particularly powerful for **multi-pattern search**.

### 4.1 Rolling hash

```java
public class RabinKarp {
    private static final long BASE  = 31;    // base for polynomial hash
    private static final long MOD   = 1_000_000_007L; // large prime

    // Compute initial hash of pattern
    // hash(s) = s[0]*BASE^(m-1) + s[1]*BASE^(m-2) + ... + s[m-1]*BASE^0
    private long hash(String s) {
        long h = 0, power = 1;
        for (int i = s.length() - 1; i >= 0; i--) {
            h = (h + (s.charAt(i) - 'a' + 1) * power) % MOD;
            power = (power * BASE) % MOD;
        }
        return h;
    }

    public List<Integer> search(String text, String pattern) {
        List<Integer> matches = new ArrayList<>();
        int n = text.length(), m = pattern.length();
        if (m > n) return matches;

        long patternHash = hash(pattern);

        // Precompute BASE^(m-1) % MOD for rolling hash removal
        long highPower = 1;
        for (int i = 0; i < m - 1; i++) highPower = (highPower * BASE) % MOD;

        // Initial window hash
        long windowHash = hash(text.substring(0, m));

        for (int i = 0; i <= n - m; i++) {
            if (windowHash == patternHash) {
                // Hash match — verify to handle collisions
                if (text.substring(i, i + m).equals(pattern)) {
                    matches.add(i);
                }
            }

            // Roll: remove leftmost char, add next char
            if (i < n - m) {
                windowHash = (windowHash
                    - (text.charAt(i) - 'a' + 1) * highPower % MOD + MOD) % MOD;
                windowHash = (windowHash * BASE) % MOD;
                windowHash = (windowHash + (text.charAt(i + m) - 'a' + 1)) % MOD;
            }
        }
        return matches;
    }
}
```

**Rolling hash mechanics:**

```
Window "ABC" → "BCD" (remove A, add D):

Old hash = A*BASE² + B*BASE + C
Remove A: hash - A*BASE² = B*BASE + C
Shift:    (B*BASE + C) * BASE = B*BASE² + C*BASE
Add D:    B*BASE² + C*BASE + D = new hash ✓

Time: O(1) per slide instead of O(m) per recompute
```

### 4.3 Multi-pattern search — Rabin-Karp shines

```java
// Search for ANY of k patterns simultaneously — O(n + m_total)
// Using a HashSet of pattern hashes
public List<String> multiPatternSearch(String text, List<String> patterns) {
    // Assume all patterns have the same length m (simplification)
    int m = patterns.get(0).length();
    Set<Long> patternHashes = new HashSet<>();
    Map<Long, String> hashToPattern = new HashMap<>();

    for (String p : patterns) {
        long h = hash(p);
        patternHashes.add(h);
        hashToPattern.put(h, p);
    }

    Set<String> found = new HashSet<>();
    long windowHash = hash(text.substring(0, m));
    long highPower = 1;
    for (int i = 0; i < m - 1; i++) highPower = (highPower * BASE) % MOD;

    for (int i = 0; i <= text.length() - m; i++) {
        if (patternHashes.contains(windowHash)) {
            String candidate = text.substring(i, i + m);
            if (hashToPattern.containsKey(windowHash)
                    && candidate.equals(hashToPattern.get(windowHash))) {
                found.add(candidate);
            }
        }
        if (i < text.length() - m) {
            // Roll the window
            windowHash = ((windowHash
                - (text.charAt(i) - 'a' + 1) * highPower % MOD + MOD) % MOD
                * BASE % MOD
                + (text.charAt(i + m) - 'a' + 1)) % MOD;
        }
    }
    return new ArrayList<>(found);
}
```

### 4.4 KMP vs Rabin-Karp

||KMP|Rabin-Karp|
|---|---|---|
|Preprocessing|O(m) LPS array|O(m) hash computation|
|Search|O(n) — no false positives|O(n) avg, O(nm) worst (collisions)|
|Multiple patterns|Run KMP k times: O(n×k + m×k)|Single pass with hash set: O(n + Σm)|
|DNA/text search|✓ Standard choice|✓ Good with double hashing|
|2D pattern search|Complex|Natural extension (2D rolling hash)|
|Simplicity|Moderate (LPS logic)|Simple rolling hash|

## 5. Trie-Based Search

Covered in depth in [[Tree Structures]] — this section focuses on search-specific operations and real-world applications.

### 5.1 Prefix search and autocomplete

```java
public class AutocompleteSystem {
    private static class TrieNode {
        Map<Character, TrieNode> children = new HashMap<>();
        boolean isEnd;
        int frequency; // how often this word was queried
    }

    private final TrieNode root = new TrieNode();

    public void insert(String word, int freq) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            node.children.putIfAbsent(c, new TrieNode());
            node = node.children.get(c);
        }
        node.isEnd = true;
        node.frequency = freq;
    }

    // Return top-k completions for prefix, ranked by frequency
    public List<String> topKCompletions(String prefix, int k) {
        TrieNode node = root;
        for (char c : prefix.toCharArray()) {
            if (!node.children.containsKey(c)) return Collections.emptyList();
            node = node.children.get(c);
        }

        // Collect all words with this prefix via DFS
        PriorityQueue<String[]> maxHeap = new PriorityQueue<>(
            (a, b) -> Integer.compare(Integer.parseInt(b[1]), Integer.parseInt(a[1]))
        );
        collectWords(node, new StringBuilder(prefix), maxHeap);

        List<String> result = new ArrayList<>();
        for (int i = 0; i < k && !maxHeap.isEmpty(); i++)
            result.add(maxHeap.poll()[0]);
        return result;
    }

    private void collectWords(TrieNode node, StringBuilder current,
                               PriorityQueue<String[]> heap) {
        if (node.isEnd) heap.offer(new String[]{current.toString(),
            String.valueOf(node.frequency)});
        for (Map.Entry<Character, TrieNode> e : node.children.entrySet()) {
            current.append(e.getKey());
            collectWords(e.getValue(), current, heap);
            current.deleteCharAt(current.length() - 1);
        }
    }
}
```

### 5.2 Wildcard and fuzzy search

```java
// Search with '.' wildcard (matches any single character) — LeetCode 211
public boolean searchWithWildcard(TrieNode root, String word) {
    return searchRec(root, word, 0);
}

private boolean searchRec(TrieNode node, String word, int i) {
    if (i == word.length()) return node.isEnd;

    char c = word.charAt(i);
    if (c == '.') {
        // Try all possible characters
        for (TrieNode child : node.children.values())
            if (searchRec(child, word, i + 1)) return true;
        return false;
    } else {
        TrieNode child = node.children.get(c);
        return child != null && searchRec(child, word, i + 1);
    }
}

// Search within edit distance k (fuzzy matching) — dynamic programming on Trie
// At each Trie node, maintain the DP row for edit distance computation
public List<String> fuzzySearch(TrieNode root, String query, int maxDistance) {
    List<String> results = new ArrayList<>();

    // Initial DP row: distance from empty prefix to query prefixes
    int[] initialRow = new int[query.length() + 1];
    for (int i = 0; i <= query.length(); i++) initialRow[i] = i;

    for (Map.Entry<Character, TrieNode> e : root.children.entrySet())
        fuzzyDFS(e.getValue(), e.getKey(), query, initialRow,
            maxDistance, new StringBuilder().append(e.getKey()), results);

    return results;
}

private void fuzzyDFS(TrieNode node, char letter, String query,
                       int[] prevRow, int maxDist,
                       StringBuilder current, List<String> results) {
    int[] currentRow = new int[query.length() + 1];
    currentRow[0] = prevRow[0] + 1;

    for (int i = 1; i <= query.length(); i++) {
        int insertCost  = currentRow[i - 1] + 1;
        int deleteCost  = prevRow[i] + 1;
        int replaceCost = prevRow[i - 1] + (query.charAt(i - 1) == letter ? 0 : 1);
        currentRow[i]   = Math.min(insertCost, Math.min(deleteCost, replaceCost));
    }

    if (currentRow[query.length()] <= maxDist && node.isEnd)
        results.add(current.toString());

    // Prune: if minimum of currentRow > maxDist, no word below can match
    if (Arrays.stream(currentRow).min().getAsInt() <= maxDist)
        for (Map.Entry<Character, TrieNode> e : node.children.entrySet()) {
            current.append(e.getKey());
            fuzzyDFS(e.getValue(), e.getKey(), query, currentRow,
                maxDist, current, results);
            current.deleteCharAt(current.length() - 1);
        }
}
```

### 5.3 XOR trie for maximum XOR

```java
// Trie on binary representation of integers — O(32) per operation
// Maximum XOR of two numbers in an array: pick the number that at each bit
// goes the opposite direction to maximize XOR
public int findMaximumXOR(int[] nums) {
    int[][] trie = new int[32 * nums.length][2]; // [node][bit 0 or 1]
    int nodeCount = 1; // root = 0

    for (int num : nums) {
        int node = 0;
        for (int i = 31; i >= 0; i--) {
            int bit = (num >> i) & 1;
            if (trie[node][bit] == 0) trie[node][bit] = nodeCount++;
            node = trie[node][bit];
        }
    }

    int maxXOR = 0;
    for (int num : nums) {
        int node = 0, currXOR = 0;
        for (int i = 31; i >= 0; i--) {
            int bit = (num >> i) & 1;
            int want = 1 - bit; // want opposite bit to maximize XOR
            if (trie[node][want] != 0) { currXOR |= (1 << i); node = trie[node][want]; }
            else                         node = trie[node][bit];
        }
        maxXOR = Math.max(maxXOR, currXOR);
    }
    return maxXOR;
}
```

## 6. A* Search Algorithm

A* is an **informed** best-first graph search that finds the shortest path from a source to a target using a **heuristic function** `h(n)` estimating the cost to the target. It combines Dijkstra's guarantees with heuristic guidance for dramatically better performance in practice.

### 6.1 Core concept

```
f(n) = g(n) + h(n)
  g(n): actual cost from start to node n (known exactly)
  h(n): estimated cost from n to target (heuristic — must be admissible)
  f(n): estimated total path cost through n

Admissible heuristic: h(n) ≤ true cost to target (never overestimates)
  → A* is OPTIMAL: guaranteed to find shortest path
  → Consistent (monotone): h(n) ≤ cost(n→n') + h(n') for all edges
     → guarantees nodes are settled in optimal order (like Dijkstra)

Examples of admissible heuristics:
  Grid (Manhattan distance):    h = |x₁-x₂| + |y₁-y₂|  (no diagonal moves)
  Grid (Euclidean distance):    h = √((x₁-x₂)² + (y₁-y₂)²)
  Grid (Chebyshev distance):    h = max(|x₁-x₂|, |y₁-y₂|)  (8-directional)
  General graph (no heuristic): h = 0 → A* degenerates to Dijkstra
```

### 6.2 A* implementation for grid pathfinding

```java
public class AStar {

    record Point(int row, int col) {}

    record State(int f, int g, Point point) implements Comparable<State> {
        @Override public int compareTo(State other) {
            return Integer.compare(this.f, other.f); // min-heap by f = g + h
        }
    }

    private static final int[][] DIRS = {{0,1},{0,-1},{1,0},{-1,0}};

    // Returns shortest path length, or -1 if no path exists
    // grid[r][c] = 0 (passable) or 1 (blocked)
    public int astar(int[][] grid, Point start, Point end) {
        int rows = grid.length, cols = grid[0].length;

        int[][] gScore = new int[rows][cols];
        for (int[] row : gScore) Arrays.fill(row, Integer.MAX_VALUE);
        gScore[start.row()][start.col()] = 0;

        PriorityQueue<State> open = new PriorityQueue<>();
        open.offer(new State(heuristic(start, end), 0, start));

        boolean[][] closed = new boolean[rows][cols];

        while (!open.isEmpty()) {
            State curr = open.poll();
            Point p = curr.point();

            if (closed[p.row()][p.col()]) continue; // stale entry
            closed[p.row()][p.col()] = true;

            if (p.equals(end)) return curr.g();

            for (int[] dir : DIRS) {
                int nr = p.row() + dir[0], nc = p.col() + dir[1];
                if (nr < 0 || nr >= rows || nc < 0 || nc >= cols
                        || grid[nr][nc] == 1 || closed[nr][nc]) continue;

                int newG = curr.g() + 1; // uniform edge cost
                Point next = new Point(nr, nc);

                if (newG < gScore[nr][nc]) {
                    gScore[nr][nc] = newG;
                    int h = heuristic(next, end);
                    open.offer(new State(newG + h, newG, next));
                }
            }
        }
        return -1; // no path
    }

    // Manhattan distance — admissible for 4-directional grid
    private int heuristic(Point a, Point b) {
        return Math.abs(a.row() - b.row()) + Math.abs(a.col() - b.col());
    }
}
```

### 6.3 A* with path reconstruction

```java
public List<Point> astarWithPath(int[][] grid, Point start, Point end) {
    int rows = grid.length, cols = grid[0].length;

    int[][] gScore = new int[rows][cols];
    Point[][] prev = new Point[rows][cols]; // parent tracking
    for (int[] row : gScore) Arrays.fill(row, Integer.MAX_VALUE);
    gScore[start.row()][start.col()] = 0;

    PriorityQueue<State> open = new PriorityQueue<>();
    open.offer(new State(heuristic(start, end), 0, start));
    boolean[][] closed = new boolean[rows][cols];

    while (!open.isEmpty()) {
        State curr = open.poll();
        Point p = curr.point();
        if (closed[p.row()][p.col()]) continue;
        closed[p.row()][p.col()] = true;

        if (p.equals(end)) {
            // Reconstruct path
            List<Point> path = new ArrayList<>();
            for (Point node = end; node != null; node = prev[node.row()][node.col()])
                path.add(node);
            Collections.reverse(path);
            return path;
        }

        for (int[] dir : DIRS) {
            int nr = p.row() + dir[0], nc = p.col() + dir[1];
            if (nr < 0 || nr >= rows || nc < 0 || nc >= cols
                    || grid[nr][nc] == 1 || closed[nr][nc]) continue;

            int newG = curr.g() + 1;
            Point next = new Point(nr, nc);
            if (newG < gScore[nr][nc]) {
                gScore[nr][nc] = newG;
                prev[nr][nc] = p;
                open.offer(new State(newG + heuristic(next, end), newG, next));
            }
        }
    }
    return Collections.emptyList();
}
```

### 6.4 Weighted graph A* (general case)

```java
// A* on weighted graph — road network, game map with terrain costs
public int astarWeighted(Map<Integer, List<int[]>> adj,
                          int src, int dst,
                          int[] heuristics) { // precomputed h(v) for each vertex
    int[] gScore = new int[heuristics.length];
    Arrays.fill(gScore, Integer.MAX_VALUE);
    gScore[src] = 0;

    // [f-score, vertex]
    PriorityQueue<int[]> open = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
    open.offer(new int[]{heuristics[src], src});

    boolean[] closed = new boolean[heuristics.length];

    while (!open.isEmpty()) {
        int[] curr = open.poll();
        int u = curr[1];

        if (closed[u]) continue;
        closed[u] = true;

        if (u == dst) return gScore[dst];

        for (int[] edge : adj.getOrDefault(u, Collections.emptyList())) {
            int v = edge[0], weight = edge[1];
            if (closed[v]) continue;

            int newG = gScore[u] + weight;
            if (newG < gScore[v]) {
                gScore[v] = newG;
                open.offer(new int[]{newG + heuristics[v], v});
            }
        }
    }
    return -1;
}
```

### 6.5 Heuristic quality — impact on performance

```
h = 0 (no heuristic):   A* = Dijkstra. Explores in all directions.
h = perfect estimate:    A* explores only the optimal path. Minimum nodes visited.
h > true cost (inadmissible): A* may find sub-optimal path. Not guaranteed optimal.
                              (becomes greedy best-first search)

Practical tradeoff:
  Weighted A*: f(n) = g(n) + w×h(n) where w > 1
    → faster (fewer nodes explored) but w-suboptimal solution
    → common in games where "good enough" path is acceptable

In game AI / navigation:
  h = Manhattan:  fast, admissible for 4-dir grid
  h = Euclidean:  admissible for weighted grids with any direction
  h = Diagonal:   Chebyshev distance for 8-directional movement
```

### 6.6 A* vs Dijkstra vs BFS

||BFS|Dijkstra|A*|
|---|---|---|---|
|Edge weights|Unweighted|Weighted ≥ 0|Weighted ≥ 0|
|Heuristic|None|None|Admissible h(n)|
|Optimal|✓ (unweighted)|✓|✓ (admissible h)|
|Nodes explored|All reachable|All reachable|Guided toward target|
|Time|O(V + E)|O((V+E) log V)|O(E log V) best case|
|Use case|Unweighted shortest path|Weighted, no target|Weighted, specific target|
|Maps/games|✗|✓|✓✓ (much faster)|

## 7. Search in Special Structures

### 7.1 Binary search in 2D matrix

```java
// Matrix where each row and column is sorted (LeetCode 240)
// Staircase search — start at top-right corner
public boolean searchMatrix(int[][] matrix, int target) {
    int row = 0, col = matrix[0].length - 1;

    while (row < matrix.length && col >= 0) {
        if      (matrix[row][col] == target) return true;
        else if (matrix[row][col] > target)  col--; // too big → go left
        else                                 row++; // too small → go down
    }
    return false;
}
// O(m + n) — eliminates one row or column per step

// Matrix where rows are sorted and first element of row i+1 > last of row i
// (LeetCode 74) — treat as 1D sorted array
public boolean searchMatrix2(int[][] matrix, int target) {
    int m = matrix.length, n = matrix[0].length;
    int lo = 0, hi = m * n - 1;

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        int val = matrix[mid / n][mid % n]; // map 1D index to 2D
        if      (val == target) return true;
        else if (val < target)  lo = mid + 1;
        else                    hi = mid - 1;
    }
    return false;
}
// O(log(m × n)) — true binary search
```

### 7.2 Exponential search (unbounded/infinite arrays)

```java
// When array size is unknown or very large — find range then binary search
public int exponentialSearch(int[] arr, int target) {
    if (arr[0] == target) return 0;

    // Double the range until we find a boundary
    int bound = 1;
    while (bound < arr.length && arr[bound] < target) bound *= 2;

    // Binary search in [bound/2, min(bound, n-1)]
    int lo = bound / 2, hi = Math.min(bound, arr.length - 1);
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if      (arr[mid] == target) return mid;
        else if (arr[mid] < target)  lo = mid + 1;
        else                         hi = mid - 1;
    }
    return -1;
}
// O(log i) where i = index of target — optimal for unknown-size arrays
```

## 8. Algorithm Selection Guide

```
Data structure / query type              → Algorithm
──────────────────────────────────────────────────────────────────────
Sorted array, exact match                → Binary search — O(log n)
Sorted array, first/last occurrence      → Binary search (leftmost/rightmost)
Sorted array, range count                → lowerBound + upperBound
Find minimum X satisfying condition      → Binary search on answer
Rotated sorted array                     → Modified binary search — O(log n)
String, single pattern search            → KMP — O(n + m)
String, multiple patterns simultaneously → Rabin-Karp with hash set
String, rolling hash / approximate match → Rabin-Karp
Prefix queries, autocomplete             → Trie — O(L)
Fuzzy match (edit distance)              → Trie + DP row
Bit manipulation / XOR problems          → XOR Trie
Weighted graph, source → all             → Dijkstra
Weighted graph, source → specific target → A* with heuristic
Unweighted graph, shortest path          → BFS
Sorted 2D matrix (sorted rows+cols)      → Staircase search — O(m+n)
Sorted 2D matrix (sorted globally)       → Binary search on flat index — O(log mn)
```

## 9. Interview Questions

**Q: What is the difference between lower bound and upper bound in binary search, and when do you need them?**

A: Lower bound returns the first index `i` where `arr[i] >= target`, and upper bound returns the first index `i` where `arr[i] > target`. The range `[lowerBound, upperBound)` contains all occurrences of target, so `upperBound - lowerBound` gives the count. You need these when the array has duplicates and you need more than just "does it exist?" — for example: counting occurrences, finding the insertion position that preserves sorted order, or implementing `floor` and `ceil` functions. The critical implementation detail is that `hi` starts at `arr.length` (not `arr.length - 1`) to allow the result to point one past the end when all elements are smaller than target.

---

**Q: Why does the naive approach fail for rotated array search, and how does the modified binary search fix it?**

A: Naive binary search relies on the assumption that if `arr[mid] < target`, then the target must be in the right half — which fails in a rotated array because the right half might wrap around the rotation point and be out of order relative to `arr[mid]`. The fix uses the observation that when you split a rotated array at `mid`, **at least one of the two halves is always sorted**. By checking whether `arr[lo] <= arr[mid]` (left half sorted) or `arr[mid] <= arr[hi]` (right half sorted), you can determine which half has guaranteed sorted order. Then you check whether the target falls within that sorted half's bounds — if yes, search there; if no, search the other half. This gives O(log n) by eliminating half the array each step.

---

**Q: Explain the KMP failure function — what does it represent and why does it enable O(n) search?**

A: The LPS (Longest Proper Prefix which is also Suffix) array at index `i` stores the length of the longest proper prefix of `pattern[0..i]` that is also a suffix. For pattern "ABABAB", `lps[3] = 2` means "ABAB" has "AB" as both a prefix and suffix. When a mismatch occurs at position `j` in the pattern after matching `j` characters, instead of restarting the pattern comparison from 0, KMP jumps to `j = lps[j-1]` — meaning we reuse the fact that the last `lps[j-1]` characters of what we already matched form a valid prefix of the pattern. This ensures the text pointer `i` never moves backward. Since `i` is monotonically increasing and `j` can only decrease a total number of times bounded by its total increases, the entire algorithm runs in O(n) text traversal plus O(m) LPS construction.

---

__Q: What makes A_ optimal, and what happens if the heuristic overestimates?_*

A: A* is optimal when the heuristic is **admissible** — it never overestimates the true cost to the target. Admissibility guarantees that when A* settles a node (marks it closed), it has found the optimal path to that node. The proof mirrors Dijkstra's: if a node `n` is settled with cost `g(n)`, any alternative path must have `f = g' + h ≥ g' + 0`, and since `h` never overestimates, if a better path existed, its `f`-value would be smaller and it would have been settled first. When the heuristic overestimates (inadmissible), A* may incorrectly settle a node via a suboptimal path because `f` looks small due to the inflated `h`. The algorithm degenerates toward greedy best-first search — faster but no longer guaranteed optimal. Weighted A* (`f = g + w×h`, `w > 1`) deliberately uses an inadmissible heuristic for a `w`-suboptimal but faster solution, useful in games where good-enough paths are acceptable.

---

**Q: When would you choose Rabin-Karp over KMP?**

A: Choose Rabin-Karp when searching for **multiple patterns simultaneously** — running KMP separately for k patterns costs O(n×k + Σm), while Rabin-Karp with a hash set of pattern hashes scans the text once in O(n + Σm). Rabin-Karp is also simpler to implement and extend to 2D pattern matching (using a 2D rolling hash). Choose KMP for single-pattern search when you need guaranteed O(n) with no collision risk — Rabin-Karp's worst case is O(nm) if many hash collisions occur, though this is extremely unlikely with a good hash and large prime. In practice, KMP is preferred for critical single-pattern tasks (DNA search, log parsing), while Rabin-Karp shines for plagiarism detection, duplicate code detection, and other multi-pattern scenarios.

## 10. Key Takeaways

- **Binary search template**: use `lo + (hi - lo) / 2` to avoid overflow; `lo <= hi` for exact search, `lo < hi` for lower/upper bound; `hi = arr.length` (not `-1`) for bounds
- **Parametric binary search**: binary search over the answer space when the condition is monotone — one of the most powerful interview patterns
- **Rotated array**: at every `mid`, one half is always sorted — identify which, then check if target falls in it
- **KMP**: O(m) LPS preprocessing enables O(n) search by never retreating `i` — LPS encodes "how far to fall back on mismatch"
- **Rabin-Karp**: rolling hash gives O(1) per slide; use double hashing to eliminate false positives; single pass for multiple patterns
- **Trie search**: O(L) per query regardless of dataset size; supports prefix, wildcard, fuzzy (edit distance), and XOR queries
- **A***: `f = g + h`, admissible `h` guarantees optimality; degrades to Dijkstra when `h=0`, to greedy best-first when `h` dominates; choose heuristic by movement type (Manhattan, Euclidean, Chebyshev)
- **Staircase search**: O(m+n) for sorted-rows-and-columns matrix; O(log mn) for globally-sorted matrix via flat-index binary search
