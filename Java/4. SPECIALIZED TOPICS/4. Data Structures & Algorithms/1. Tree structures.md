# Tree Structures

## Overview

Trees are hierarchical data structures where nodes are connected by edges, with no cycles. They are the backbone of databases (B-trees), compilers (AST), file systems, routing tables, and autocomplete (tries). Understanding when to use each tree variant — and how to implement the core operations — is essential for both system design and coding interviews.

**Core vocabulary:**

```
Root:     The top node (no parent)
Leaf:     A node with no children
Height:   Longest path from root to a leaf
Depth:    Distance from root to a node
Degree:   Number of children a node has
Balanced: Height is O(log n) — guarantees efficient operations
```

---

## 1. Binary Trees, BST, AVL, Red-Black

### 1.1 Binary Tree — the base structure

A binary tree allows each node to have **at most two children** (left and right). It imposes no ordering constraint.

```java
public class TreeNode<T> {
    T val;
    TreeNode<T> left;
    TreeNode<T> right;

    public TreeNode(T val) {
        this.val = val;
    }
}
```

**Types of binary trees:**

```
Full Binary Tree:     Every node has 0 or 2 children
Complete Binary Tree: All levels filled except possibly the last,
                      filled left to right (heap property)
Perfect Binary Tree:  All internal nodes have 2 children,
                      all leaves at same depth → n = 2^h - 1
Degenerate Tree:      Every node has 1 child → effectively a linked list
                      Worst case for BST (sorted insertion)
```

---

### 1.2 Binary Search Tree (BST)

A BST adds an **ordering invariant**: for every node, all values in its left subtree are less, and all values in its right subtree are greater.

```
       8
      / \
     3   10
    / \    \
   1   6    14
      / \   /
     4   7 13
```

```java
public class BST {
    private TreeNode<Integer> root;

    // Insert — O(h) where h is height. O(log n) balanced, O(n) degenerate
    public void insert(int val) {
        root = insertRec(root, val);
    }

    private TreeNode<Integer> insertRec(TreeNode<Integer> node, int val) {
        if (node == null) return new TreeNode<>(val);

        if (val < node.val)      node.left = insertRec(node.left, val);
        else if (val > node.val) node.right = insertRec(node.right, val);
        // equal: ignore duplicate (or handle per requirements)

        return node;
    }

    // Search — O(h)
    public boolean contains(int val) {
        return containsRec(root, val);
    }

    private boolean containsRec(TreeNode<Integer> node, int val) {
        if (node == null) return false;
        if (val == node.val) return true;
        return val < node.val
            ? containsRec(node.left, val)
            : containsRec(node.right, val);
    }

    // Delete — O(h) — the tricky one
    public void delete(int val) {
        root = deleteRec(root, val);
    }

    private TreeNode<Integer> deleteRec(TreeNode<Integer> node, int val) {
        if (node == null) return null;

        if (val < node.val) {
            node.left = deleteRec(node.left, val);
        } else if (val > node.val) {
            node.right = deleteRec(node.right, val);
        } else {
            // Found the node to delete
            if (node.left == null) return node.right;  // case 1: no left child
            if (node.right == null) return node.left;  // case 2: no right child

            // Case 3: two children
            // Replace with in-order successor (smallest in right subtree)
            int successor = findMin(node.right);
            node.val = successor;
            node.right = deleteRec(node.right, successor); // delete successor
        }
        return node;
    }

    private int findMin(TreeNode<Integer> node) {
        while (node.left != null) node = node.left;
        return node.val;
    }

    // In-order traversal → sorted output
    public List<Integer> inorder() {
        List<Integer> result = new ArrayList<>();
        inorderRec(root, result);
        return result;
    }

    private void inorderRec(TreeNode<Integer> node, List<Integer> result) {
        if (node == null) return;
        inorderRec(node.left, result);
        result.add(node.val);
        inorderRec(node.right, result);
    }
}
```

**BST Complexity:**

|Operation|Average|Worst (degenerate)|
|---|---|---|
|Search|O(log n)|O(n)|
|Insert|O(log n)|O(n)|
|Delete|O(log n)|O(n)|
|Min/Max|O(log n)|O(n)|

**The degenerate case** (inserting sorted data: 1, 2, 3, 4...) produces a right-leaning chain. This is why self-balancing trees (AVL, Red-Black) exist.

### 1.3 AVL Tree

An AVL tree is a self-balancing BST where for **every node**, the heights of the left and right subtrees differ by **at most 1** (balance factor ∈ {-1, 0, 1}). When a rotation violates this, a rotation restores balance.

```
Balance Factor = height(left) - height(right)
  BF = 1:  left-heavy (acceptable)
  BF = 0:  perfectly balanced
  BF = -1: right-heavy (acceptable)
  BF > 1 or BF < -1: → rotate
```

**The four rotation cases:**

```
Case 1: Left-Left (BF > 1, left child BF >= 0) → Right Rotate
    z          y
   / \        / \
  y   T4 →  x   z
 / \        /\ /\
x   T3     T1 T2 T3 T4

Case 2: Right-Right (BF < -1, right child BF <= 0) → Left Rotate
  z              y
   \            / \
    y    →    z   x
   / \       /\   \
  T2  x    T1 T2  T3
       \
       T3

Case 3: Left-Right → Left rotate child, then Right rotate
Case 4: Right-Left → Right rotate child, then Left rotate
```

```java
public class AVLTree {
    private AVLNode root;

    private static class AVLNode {
        int val, height;
        AVLNode left, right;
        AVLNode(int val) { this.val = val; this.height = 1; }
    }

    private int height(AVLNode n) { return n == null ? 0 : n.height; }

    private int balanceFactor(AVLNode n) {
        return n == null ? 0 : height(n.left) - height(n.right);
    }

    private void updateHeight(AVLNode n) {
        n.height = 1 + Math.max(height(n.left), height(n.right));
    }

    // Right rotation — fixes Left-Left imbalance
    private AVLNode rotateRight(AVLNode z) {
        AVLNode y = z.left;
        AVLNode T3 = y.right;

        y.right = z;
        z.left = T3;

        updateHeight(z); // z is now lower — update first
        updateHeight(y);
        return y;        // y is new root
    }

    // Left rotation — fixes Right-Right imbalance
    private AVLNode rotateLeft(AVLNode z) {
        AVLNode y = z.right;
        AVLNode T2 = y.left;

        y.left = z;
        z.right = T2;

        updateHeight(z);
        updateHeight(y);
        return y;
    }

    private AVLNode balance(AVLNode n) {
        updateHeight(n);
        int bf = balanceFactor(n);

        // Left-Left
        if (bf > 1 && balanceFactor(n.left) >= 0)
            return rotateRight(n);

        // Left-Right
        if (bf > 1 && balanceFactor(n.left) < 0) {
            n.left = rotateLeft(n.left);
            return rotateRight(n);
        }

        // Right-Right
        if (bf < -1 && balanceFactor(n.right) <= 0)
            return rotateLeft(n);

        // Right-Left
        if (bf < -1 && balanceFactor(n.right) > 0) {
            n.right = rotateRight(n.right);
            return rotateLeft(n);
        }

        return n; // already balanced
    }

    public void insert(int val) { root = insertRec(root, val); }

    private AVLNode insertRec(AVLNode n, int val) {
        if (n == null) return new AVLNode(val);

        if (val < n.val)      n.left = insertRec(n.left, val);
        else if (val > n.val) n.right = insertRec(n.right, val);
        else return n; // duplicate

        return balance(n); // re-balance on the way back up
    }
}
```

**AVL guarantees**: height ≤ 1.44 log₂(n+2) — all operations strictly O(log n).

### 1.4 Red-Black Tree

A Red-Black tree is a self-balancing BST where each node is colored **red or black**, and five invariants maintain a "loosely balanced" tree (height ≤ 2 log₂(n+1)).

**The five invariants:**

```
1. Every node is RED or BLACK
2. Root is BLACK
3. Every NULL leaf is BLACK
4. RED node's children must both be BLACK
   (no two consecutive red nodes on any path)
5. Every path from a node to its NULL descendants
   contains the same number of BLACK nodes
   (same "black height" for all paths)
```

```java
// Red-Black Tree — high-level structure
// (Full implementation is lengthy — focusing on the key concepts and cases)

public class RedBlackTree {
    private static final boolean RED = true;
    private static final boolean BLACK = false;

    private RBNode root;
    private final RBNode NIL; // sentinel null node

    public RedBlackTree() {
        NIL = new RBNode(0);
        NIL.color = BLACK;
        root = NIL;
    }

    private static class RBNode {
        int val;
        boolean color;
        RBNode left, right, parent;
        RBNode(int val) { this.val = val; this.color = RED; }
    }

    // Left rotation (same as AVL, but maintains parent pointers)
    private void rotateLeft(RBNode x) {
        RBNode y = x.right;
        x.right = y.left;
        if (y.left != NIL) y.left.parent = x;

        y.parent = x.parent;
        if (x.parent == null)      root = y;
        else if (x == x.parent.left) x.parent.left = y;
        else                       x.parent.right = y;

        y.left = x;
        x.parent = y;
    }

    public void insert(int val) {
        RBNode node = new RBNode(val);
        node.left = node.right = NIL;

        // Standard BST insert
        RBNode parent = null;
        RBNode current = root;
        while (current != NIL) {
            parent = current;
            current = val < current.val ? current.left : current.right;
        }
        node.parent = parent;
        if (parent == null)          root = node;
        else if (val < parent.val)   parent.left = node;
        else                         parent.right = node;

        node.color = RED; // new nodes are always red
        fixInsert(node);  // then fix violations
    }

    // Fix Red-Black violations after insert
    // Key insight: only invariant 4 can be violated (red parent + red child)
    private void fixInsert(RBNode z) {
        while (z.parent != null && z.parent.color == RED) {
            if (z.parent == z.parent.parent.left) {
                RBNode uncle = z.parent.parent.right;

                if (uncle.color == RED) {
                    // Case 1: Uncle is RED → recolor, move violation up
                    z.parent.color = BLACK;
                    uncle.color = BLACK;
                    z.parent.parent.color = RED;
                    z = z.parent.parent;

                } else {
                    if (z == z.parent.right) {
                        // Case 2: Uncle BLACK, z is right child → rotate left
                        z = z.parent;
                        rotateLeft(z);
                    }
                    // Case 3: Uncle BLACK, z is left child → recolor + rotate right
                    z.parent.color = BLACK;
                    z.parent.parent.color = RED;
                    rotateRight(z.parent.parent);
                }
            } else {
                // Mirror cases (parent is right child)
                // ... symmetric to above
            }
        }
        root.color = BLACK; // invariant 2: root always black
    }
}
```

**AVL vs Red-Black Tree:**

||AVL|Red-Black|
|---|---|---|
|Balance|Strict (BF ≤ 1)|Loose (height ≤ 2 log n)|
|Lookup speed|Faster (stricter balance)|Slightly slower|
|Insert/Delete|More rotations|Fewer rotations (1-2 max)|
|Use case|Read-heavy|Write-heavy|
|Java stdlib|—|`TreeMap`, `TreeSet`|
|Linux kernel|—|Scheduler, virtual memory|

**Java's `TreeMap` and `TreeSet` use Red-Black Trees** — O(log n) for all operations.

```java
// Java TreeMap internals — Red-Black Tree in practice
TreeMap<Integer, String> map = new TreeMap<>();
map.put(5, "five");   // O(log n) insert + rebalance
map.put(3, "three");
map.get(3);           // O(log n) lookup
map.firstKey();       // O(log n) min
map.lastKey();        // O(log n) max
map.headMap(5);       // all keys < 5 — range query
map.subMap(3, 7);     // keys in [3, 7)
```

## 2. B-Trees and B+ Trees

B-trees and B+ trees are **multi-way search trees** designed for disk storage. Where binary trees minimize comparisons, B-trees minimize disk I/O by storing many keys per node (matching disk page size).

### 2.1 B-Tree

In a B-tree of order `m`:

- Every node has **at most m children**
- Non-root internal nodes have **at least ⌈m/2⌉ children**
- Root has at least 2 children (if not a leaf)
- All leaves at the **same depth**
- A node with `k` children holds **k-1 keys**
- Data records can exist at **any node** (internal or leaf)

```
B-Tree of order 5 (max 4 keys, 5 children per node):

        [10 | 20 | 30 | 40]
       /    |    |    |    \
 [1-9] [11-19] [21-29] [31-39] [41-99]
```

```java
// B-Tree node structure (conceptual — real implementations are complex)
public class BTreeNode {
    int[] keys;         // sorted keys
    Object[] values;    // data at each key (B-Tree stores data at all nodes)
    BTreeNode[] children;
    int keyCount;
    boolean isLeaf;

    public BTreeNode(int order) {
        keys = new int[order - 1];
        values = new Object[order - 1];
        children = new BTreeNode[order];
        keyCount = 0;
        isLeaf = true;
    }
}

// Search in B-Tree — O(log_m(n) × m) = O(log n)
// At each node: binary search among keys, then follow correct child pointer
public Object search(BTreeNode node, int key) {
    int i = 0;
    // Find first key >= target
    while (i < node.keyCount && key > node.keys[i]) i++;

    if (i < node.keyCount && key == node.keys[i]) {
        return node.values[i]; // found — B-Tree can have data at internal nodes
    }
    if (node.isLeaf) return null; // not found

    return search(node.children[i], key); // recurse into correct child
}
```

**Split on insert — the key operation:**

```
When a node is full (m-1 keys), split it:
1. Promote middle key up to parent
2. Left child gets keys to the left of middle
3. Right child gets keys to the right of middle

Before (full, order 4):
  Parent: [20 | 40]
  Child:  [10 | 25 | 30 | 35] ← full, inserting 28

After split (promote 30):
  Parent: [20 | 30 | 40]
  Left:   [10 | 25]
  Right:  [35]
  (then insert 28 into left)
```

### 2.2 B+ Tree

The B+ tree adds one constraint: **all data lives in the leaf nodes**. Internal nodes hold only keys for routing. Leaves are linked together in a doubly linked list for efficient range queries.

```
B+ Tree of order 4:

Internal (routing only):     [20 | 40]
                            /    |    \
Leaves (data):       [10,15] → [20,25,30] → [40,45,50]
                     (← linked list of leaves →)
```

```java
// B+ Tree leaf node structure
public class BPlusLeaf {
    int[] keys;
    Object[] values;  // actual data — only in leaves
    BPlusLeaf next;   // linked list pointer to next leaf
    int size;
}

// B+ Tree internal node — routing only
public class BPlusInternal {
    int[] keys;           // routing keys (copies of leaf keys)
    BPlusNode[] children; // pointers to children
    int size;
}

// Range query — B+ Tree's killer feature
public List<Object> rangeQuery(BPlusTree tree, int low, int high) {
    List<Object> result = new ArrayList<>();

    // Find leaf containing 'low'
    BPlusLeaf leaf = tree.findLeaf(low);

    // Walk the linked list until key > high
    while (leaf != null) {
        for (int i = 0; i < leaf.size; i++) {
            if (leaf.keys[i] > high) return result;
            if (leaf.keys[i] >= low) result.add(leaf.values[i]);
        }
        leaf = leaf.next; // O(1) — linked list traversal
    }
    return result;
}
```

### 2.3 B-Tree vs B+ Tree

||B-Tree|B+ Tree|
|---|---|---|
|Data location|Any node|Leaves only|
|Internal nodes|Keys + data|Keys only (smaller → more keys fit)|
|Leaf linking|No|Yes (doubly linked list)|
|Range queries|Inefficient (may revisit)|O(k) scan of linked leaves|
|Point queries|Slightly faster (can stop at internal node)|Always reach leaf|
|Height|Slightly taller|Slightly shorter (denser internal nodes)|
|Use in databases|Rarely (older systems)|Universal (MySQL InnoDB, PostgreSQL)|

**Why databases use B+ Trees:**

```
Disk page = 16KB (MySQL default)
Integer key = 8 bytes, pointer = 6 bytes

B+ Tree internal node can hold:
  16KB / (8 + 6) bytes = ~1170 keys per node

For 1 million rows:
  Level 1 (root): 1 node
  Level 2:       1170 nodes
  Level 3:       1,170² = 1,368,900 nodes > 1,000,000

→ All 1 million records reachable in 3 disk reads
```

## 3. Trie for String Search

A Trie (prefix tree) stores strings by sharing common prefixes. Each path from root to a marked node represents a word. Optimized for prefix matching, autocomplete, and spell-check.

```
Words: ["can", "cat", "cats", "car", "card"]

        root
         |
         c
         |
         a
        /|\
       n  t  r
       *  |  |
          s  d
          *  *
       (* = end of word marker)
```

```java
public class Trie {
    private static final int ALPHABET = 26;

    private static class TrieNode {
        TrieNode[] children = new TrieNode[ALPHABET];
        boolean isEndOfWord;
        int wordsWithPrefix; // optional: count of words below this node
    }

    private final TrieNode root = new TrieNode();

    // Insert — O(L) where L = word length
    public void insert(String word) {
        TrieNode current = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (current.children[idx] == null) {
                current.children[idx] = new TrieNode();
            }
            current = current.children[idx];
            current.wordsWithPrefix++;
        }
        current.isEndOfWord = true;
    }

    // Search — O(L): exact match
    public boolean search(String word) {
        TrieNode node = findNode(word);
        return node != null && node.isEndOfWord;
    }

    // StartsWith — O(L): prefix check
    public boolean startsWith(String prefix) {
        return findNode(prefix) != null;
    }

    private TrieNode findNode(String s) {
        TrieNode current = root;
        for (char c : s.toCharArray()) {
            int idx = c - 'a';
            if (current.children[idx] == null) return null;
            current = current.children[idx];
        }
        return current;
    }

    // Autocomplete — return all words with given prefix
    public List<String> autocomplete(String prefix) {
        List<String> results = new ArrayList<>();
        TrieNode node = findNode(prefix);
        if (node == null) return results;

        dfsCollect(node, new StringBuilder(prefix), results);
        return results;
    }

    private void dfsCollect(TrieNode node, StringBuilder current, List<String> results) {
        if (node.isEndOfWord) results.add(current.toString());

        for (int i = 0; i < ALPHABET; i++) {
            if (node.children[i] != null) {
                current.append((char) ('a' + i));
                dfsCollect(node.children[i], current, results);
                current.deleteCharAt(current.length() - 1); // backtrack
            }
        }
    }

    // Delete — O(L): mark as not end of word, prune if no other words use node
    public void delete(String word) {
        deleteRec(root, word, 0);
    }

    private boolean deleteRec(TrieNode node, String word, int depth) {
        if (node == null) return false;

        if (depth == word.length()) {
            if (!node.isEndOfWord) return false; // word not in trie
            node.isEndOfWord = false;
            // Return true if this node has no children → safe to delete
            return allChildrenNull(node);
        }

        int idx = word.charAt(depth) - 'a';
        if (deleteRec(node.children[idx], word, depth + 1)) {
            node.children[idx] = null; // prune the child
            // Delete this node if it's not a word endpoint and has no children
            return !node.isEndOfWord && allChildrenNull(node);
        }
        return false;
    }

    private boolean allChildrenNull(TrieNode node) {
        for (TrieNode child : node.children)
            if (child != null) return false;
        return true;
    }
}
```

**Trie complexity:**

|Operation|Time|Space|
|---|---|---|
|Insert|O(L)|O(L × ALPHABET)|
|Search|O(L)|—|
|Prefix check|O(L)|—|
|Delete|O(L)|—|
|Space (total)|—|O(N × L × ALPHABET)|

**Compressed Trie (Patricia Trie / Radix Tree):** collapses single-child chains into one edge labelled with a substring. Used in routing tables and Linux kernel's VFS. Java's `java.util.regex` internals use a similar structure.

## 4. Segment Tree

A segment tree answers **range queries** (sum, min, max) over an array in O(log n), with O(log n) point updates. It's a complete binary tree where each node stores the aggregate of its range.

```
Array: [1, 3, 5, 7, 9, 11]
Range:  0  1  2  3   4   5

Segment Tree (sum):
                [36: 0-5]
               /           \
        [9: 0-2]          [27: 3-5]
        /       \          /       \
  [4: 0-1]  [5: 2-2] [16: 3-4] [11: 5-5]
   /    \
[1:0] [3:1]      [7:3] [9:4]
```

```java
public class SegmentTree {
    private final int[] tree;
    private final int n;

    // Build — O(n)
    public SegmentTree(int[] arr) {
        n = arr.length;
        tree = new int[4 * n]; // safe upper bound for tree size
        build(arr, 0, 0, n - 1);
    }

    private void build(int[] arr, int node, int start, int end) {
        if (start == end) {
            tree[node] = arr[start]; // leaf
        } else {
            int mid = (start + end) / 2;
            build(arr, 2 * node + 1, start, mid);       // left child
            build(arr, 2 * node + 2, mid + 1, end);     // right child
            tree[node] = tree[2 * node + 1] + tree[2 * node + 2]; // merge
        }
    }

    // Range sum query — O(log n)
    public int query(int l, int r) {
        return queryRec(0, 0, n - 1, l, r);
    }

    private int queryRec(int node, int start, int end, int l, int r) {
        if (r < start || end < l) return 0;  // completely outside range
        if (l <= start && end <= r) return tree[node]; // completely inside

        int mid = (start + end) / 2;
        int leftSum  = queryRec(2 * node + 1, start, mid, l, r);
        int rightSum = queryRec(2 * node + 2, mid + 1, end, l, r);
        return leftSum + rightSum;
    }

    // Point update — O(log n)
    public void update(int idx, int newVal) {
        updateRec(0, 0, n - 1, idx, newVal);
    }

    private void updateRec(int node, int start, int end, int idx, int newVal) {
        if (start == end) {
            tree[node] = newVal; // leaf — update directly
        } else {
            int mid = (start + end) / 2;
            if (idx <= mid) updateRec(2 * node + 1, start, mid, idx, newVal);
            else            updateRec(2 * node + 2, mid + 1, end, idx, newVal);
            tree[node] = tree[2 * node + 1] + tree[2 * node + 2]; // recalculate
        }
    }
}
```

**Range Min/Max variant** (change only the merge operation):

```java
// For range minimum:
tree[node] = Math.min(tree[2*node+1], tree[2*node+2]); // merge
// Query base case (outside range): return Integer.MAX_VALUE

// For range maximum:
tree[node] = Math.max(tree[2*node+1], tree[2*node+2]);
// Query base case (outside range): return Integer.MIN_VALUE
```

**Lazy Propagation** — range updates in O(log n):

```java
// Without lazy propagation, range update (add 5 to elements 2-7)
// requires O(n) individual point updates.
// Lazy propagation: mark update on a node, defer to children until needed.

private int[] lazy; // lazy[i] = pending addition for subtree i

private void rangeUpdate(int node, int start, int end, int l, int r, int delta) {
    if (lazy[node] != 0) applyLazy(node, start, end); // push down pending

    if (r < start || end < l) return; // out of range

    if (l <= start && end <= r) {
        // Completely within range — apply and mark lazy
        tree[node] += (end - start + 1) * delta;
        if (start != end) { // not a leaf
            lazy[2 * node + 1] += delta;
            lazy[2 * node + 2] += delta;
        }
        return;
    }

    int mid = (start + end) / 2;
    rangeUpdate(2 * node + 1, start, mid, l, r, delta);
    rangeUpdate(2 * node + 2, mid + 1, end, l, r, delta);
    tree[node] = tree[2 * node + 1] + tree[2 * node + 2];
}

private void applyLazy(int node, int start, int end) {
    tree[node] += lazy[node] * (end - start + 1);
    if (start != end) {
        lazy[2 * node + 1] += lazy[node];
        lazy[2 * node + 2] += lazy[node];
    }
    lazy[node] = 0;
}
```

**Segment tree complexity:**

|Operation|Time|Space|
|---|---|---|
|Build|O(n)|O(n)|
|Range query|O(log n)|—|
|Point update|O(log n)|—|
|Range update (lazy)|O(log n)|O(n) extra|

**Use cases**: range sum/min/max, frequency queries in competitive programming, interval scheduling, and databases' query optimizers for histogram statistics.

## 5. Tree Traversals (DFS, BFS)

### 5.1 Depth-First Search (DFS) — three orders

```
Tree:     1
         / \
        2   3
       / \
      4   5

In-order   (L → Root → R): 4 2 5 1 3  → BST gives sorted output
Pre-order  (Root → L → R): 1 2 4 5 3  → serialize/copy tree
Post-order (L → R → Root): 4 5 2 3 1  → delete tree, evaluate expression
```

```java
// Recursive — elegantly simple
void inorder(TreeNode node) {
    if (node == null) return;
    inorder(node.left);           // L
    System.out.print(node.val);   // Root
    inorder(node.right);          // R
}

void preorder(TreeNode node) {
    if (node == null) return;
    System.out.print(node.val);   // Root
    preorder(node.left);          // L
    preorder(node.right);         // R
}

void postorder(TreeNode node) {
    if (node == null) return;
    postorder(node.left);         // L
    postorder(node.right);        // R
    System.out.print(node.val);   // Root
}

// Iterative in-order — avoids stack overflow for deep trees (interview favourite)
List<Integer> inorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode current = root;

    while (current != null || !stack.isEmpty()) {
        // Go as far left as possible
        while (current != null) {
            stack.push(current);
            current = current.left;
        }
        // Process node
        current = stack.pop();
        result.add(current.val);
        // Move to right subtree
        current = current.right;
    }
    return result;
}

// Iterative pre-order — uses stack explicitly
List<Integer> preorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);

    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.add(node.val);             // process
        if (node.right != null) stack.push(node.right); // right first (LIFO)
        if (node.left != null)  stack.push(node.left);
    }
    return result;
}

// Iterative post-order — two-stack trick
List<Integer> postorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);

    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.addFirst(node.val);        // add to FRONT (reverse pre-order)
        if (node.left != null)  stack.push(node.left);
        if (node.right != null) stack.push(node.right);
    }
    return result; // already in post-order due to addFirst
}
```

### 5.2 Breadth-First Search (BFS) — level-order

```
Tree:     1           → Level 0: [1]
         / \          → Level 1: [2, 3]
        2   3         → Level 2: [4, 5, 6]
       / \   \
      4   5   6
```

```java
// BFS / Level-order traversal — uses a queue
List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size(); // snapshot of current level's size
        List<Integer> level = new ArrayList<>();

        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null)  queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}

// Zigzag level order — alternate left-to-right and right-to-left
List<List<Integer>> zigzagLevelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    boolean leftToRight = true;

    while (!queue.isEmpty()) {
        int size = queue.size();
        LinkedList<Integer> level = new LinkedList<>();

        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            if (leftToRight) level.addLast(node.val);
            else             level.addFirst(node.val); // reverse direction
            if (node.left != null)  queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
        leftToRight = !leftToRight;
    }
    return result;
}
```

### 5.3 Morris Traversal — O(1) space in-order

Uses temporary links in the tree to traverse without a stack or recursion:

```java
// Morris In-order — O(n) time, O(1) space
List<Integer> morrisInorder(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    TreeNode current = root;

    while (current != null) {
        if (current.left == null) {
            result.add(current.val); // process
            current = current.right;
        } else {
            // Find in-order predecessor (rightmost node of left subtree)
            TreeNode predecessor = current.left;
            while (predecessor.right != null && predecessor.right != current) {
                predecessor = predecessor.right;
            }

            if (predecessor.right == null) {
                // Create temporary link: predecessor → current
                predecessor.right = current;
                current = current.left;
            } else {
                // Link already exists → we've processed left subtree
                predecessor.right = null; // remove temporary link
                result.add(current.val);  // process
                current = current.right;
            }
        }
    }
    return result;
}
```

### 5.4 DFS vs BFS — when to use each

||DFS|BFS|
|---|---|---|
|Space|O(h) — stack depth|O(w) — max width|
|Finds shortest path|No|Yes (unweighted)|
|Level-by-level processing|No|Yes|
|Tree height/depth|Yes|No|
|Path existence|Yes|Yes|
|Deep narrow trees|Better|Worse|
|Wide shallow trees|Worse|Better|

## 6. LCA — Lowest Common Ancestor

The LCA of two nodes `p` and `q` is the deepest node that is an ancestor of both. It's a building block for many tree problems (distance between nodes, range queries on trees).

### 6.1 Naive recursive approach — O(n) per query

```java
// Works for any binary tree (not just BST)
TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null) return null;
    if (root == p || root == q) return root; // found one of the targets

    TreeNode left  = lca(root.left,  p, q);
    TreeNode right = lca(root.right, p, q);

    if (left != null && right != null) return root; // p and q in different subtrees
    return left != null ? left : right; // both in same subtree
}
```

```
Example: LCA(4, 5) in tree below
         1
        / \
       2   3
      / \
     4   5

lca(1, 4, 5):
  lca(2, 4, 5):
    lca(4, 4, 5): returns 4 (found p)
    lca(5, 4, 5): returns 5 (found q)
    left=4, right=5, both non-null → return 2
  lca(3, 4, 5): returns null
  left=2, right=null → return 2

LCA(4, 5) = 2  ✓
```

### 6.2 LCA for BST — O(h)

Use the BST ordering property to navigate directly:

```java
// For BST: if both p and q are less than root → LCA is in left subtree
// If both greater → right subtree. Otherwise root is LCA.
TreeNode lcaBST(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null) return null;

    if (p.val < root.val && q.val < root.val)
        return lcaBST(root.left, p, q);   // both in left

    if (p.val > root.val && q.val > root.val)
        return lcaBST(root.right, p, q);  // both in right

    return root; // split point — root is LCA
}

// Iterative BST version — O(h) time, O(1) space
TreeNode lcaBSTIterative(TreeNode root, TreeNode p, TreeNode q) {
    while (root != null) {
        if (p.val < root.val && q.val < root.val) root = root.left;
        else if (p.val > root.val && q.val > root.val) root = root.right;
        else return root;
    }
    return null;
}
```

### 6.3 Binary Lifting — O(n log n) preprocessing, O(log n) per query

For many LCA queries on the same tree, precompute ancestor tables:

```java
public class LCABinaryLifting {
    private final int[][] ancestor; // ancestor[v][k] = 2^k-th ancestor of v
    private final int[] depth;
    private final int LOG = 20;     // supports trees up to 2^20 ≈ 1M nodes

    public LCABinaryLifting(TreeNode root, int n) {
        ancestor = new int[n + 1][LOG];
        depth = new int[n + 1];

        // Initialize with -1 (no ancestor)
        for (int[] row : ancestor) Arrays.fill(row, -1);

        // BFS/DFS to fill depth and direct parent (ancestor[v][0])
        dfs(root, -1, 0);

        // Fill ancestor table: ancestor[v][k] = ancestor[ancestor[v][k-1]][k-1]
        for (int k = 1; k < LOG; k++) {
            for (int v = 0; v < n; v++) {
                if (ancestor[v][k - 1] != -1) {
                    ancestor[v][k] = ancestor[ancestor[v][k - 1]][k - 1];
                }
            }
        }
    }

    private void dfs(TreeNode node, int parent, int d) {
        if (node == null) return;
        depth[node.val] = d;
        ancestor[node.val][0] = parent; // direct parent
        dfs(node.left, node.val, d + 1);
        dfs(node.right, node.val, d + 1);
    }

    // Lift node up by k levels
    private int kthAncestor(int v, int k) {
        for (int i = 0; i < LOG; i++) {
            if (((k >> i) & 1) == 1) { // if bit i is set in k
                v = ancestor[v][i];
                if (v == -1) return -1;
            }
        }
        return v;
    }

    // LCA query — O(log n)
    public int lca(int u, int v) {
        // Bring u and v to the same depth
        if (depth[u] < depth[v]) { int tmp = u; u = v; v = tmp; }
        u = kthAncestor(u, depth[u] - depth[v]); // lift deeper node

        if (u == v) return u; // one is ancestor of the other

        // Binary lift both until they diverge just below LCA
        for (int k = LOG - 1; k >= 0; k--) {
            if (ancestor[u][k] != ancestor[v][k]) {
                u = ancestor[u][k];
                v = ancestor[v][k];
            }
        }
        return ancestor[u][0]; // direct parent is LCA
    }

    // Distance between two nodes = depth[u] + depth[v] - 2 * depth[lca(u,v)]
    public int distance(int u, int v) {
        return depth[u] + depth[v] - 2 * depth[lca(u, v)];
    }
}
```

### 6.4 Euler Tour + Range Minimum Query — O(n) preprocessing, O(1) per query

The fastest LCA approach for offline scenarios:

```
Euler tour: record node visits including re-visits during DFS
Tree:     1
         / \
        2   3
       / \
      4   5

Euler tour: [1, 2, 4, 2, 5, 2, 1, 3, 1]
Depths:     [0, 1, 2, 1, 2, 1, 0, 1, 0]

LCA(4, 5) = min depth node between first occurrences of 4 and 5
  First occ of 4 = index 2
  First occ of 5 = index 4
  Euler[2..4] = [4, 2, 5], depths = [2, 1, 2]
  Min depth is at index 3 → node 2 = LCA(4, 5) ✓

Build a sparse table on depths array → O(1) RMQ → O(1) LCA
```

|Approach|Preprocessing|Per Query|Space|
|---|---|---|---|
|Naive recursion|—|O(n)|O(h)|
|BST property|—|O(h)|O(1)|
|Binary Lifting|O(n log n)|O(log n)|O(n log n)|
|Euler Tour + RMQ|O(n)|O(1)|O(n)|

## 7. Classic Tree Problems

```java
// Height of a binary tree — O(n)
int height(TreeNode root) {
    if (root == null) return 0;
    return 1 + Math.max(height(root.left), height(root.right));
}

// Check if balanced — O(n)
int checkBalanced(TreeNode node) {
    if (node == null) return 0;
    int left  = checkBalanced(node.left);
    int right = checkBalanced(node.right);
    if (left == -1 || right == -1) return -1;           // already unbalanced
    if (Math.abs(left - right) > 1) return -1;          // this node unbalanced
    return 1 + Math.max(left, right);
}
boolean isBalanced(TreeNode root) { return checkBalanced(root) != -1; }

// Diameter (longest path between any two nodes) — O(n)
int maxDiameter = 0;
int diameter(TreeNode node) {
    if (node == null) return 0;
    int left  = diameter(node.left);
    int right = diameter(node.right);
    maxDiameter = Math.max(maxDiameter, left + right); // path through this node
    return 1 + Math.max(left, right);
}

// Serialize / Deserialize — O(n) using pre-order
String serialize(TreeNode root) {
    if (root == null) return "null,";
    return root.val + "," + serialize(root.left) + serialize(root.right);
}

TreeNode deserialize(String data) {
    Queue<String> queue = new LinkedList<>(Arrays.asList(data.split(",")));
    return deserializeHelper(queue);
}
TreeNode deserializeHelper(Queue<String> q) {
    String val = q.poll();
    if ("null".equals(val)) return null;
    TreeNode node = new TreeNode(Integer.parseInt(val));
    node.left  = deserializeHelper(q);
    node.right = deserializeHelper(q);
    return node;
}
```

## 8. Interview Questions

**Q: What is the difference between a B-Tree and a B+ Tree, and why do databases prefer B+ Trees?**

A: In a B-Tree, data records can be stored at any node — internal or leaf. In a B+ Tree, all data records live exclusively in leaf nodes; internal nodes contain only routing keys. This distinction has two major consequences. First, B+ Tree internal nodes can hold more keys because they don't carry data — this reduces tree height for large datasets. Second, B+ Tree leaves are connected in a doubly linked list, enabling range queries by scanning the leaf level without backtracking through the tree. A query like `WHERE price BETWEEN 10 AND 100` traverses the tree once to find the first matching leaf, then scans the linked list — O(k) for k results. Databases like MySQL InnoDB and PostgreSQL use B+ Trees because sequential range scans are extremely common in SQL workloads.

---

**Q: Why are AVL trees preferred for read-heavy workloads and Red-Black Trees for write-heavy workloads?**

A: AVL trees maintain a stricter balance invariant (height difference ≤ 1 at every node), which results in a shorter tree and faster lookups — fewer nodes traversed per search. However, this strictness means more rotations are required after each insert or delete to restore the invariant. Red-Black trees use a looser invariant (no two consecutive red nodes; equal black height), allowing the tree to be up to twice as tall as an AVL tree in the worst case, but requiring at most 2 rotations per insert and 3 per delete. This makes writes cheaper. Java's `TreeMap` and `TreeSet` use Red-Black Trees because Java collections are general-purpose and tend toward balanced read/write workloads, while the Linux kernel's scheduler and memory manager use Red-Black Trees for their predictable worst-case write performance.

---

**Q: When would you use a Trie over a HashMap for string storage?**

A: A HashMap provides O(L) average-case lookup (L = string length) but cannot answer prefix queries — it can only tell you if an exact string exists. A Trie supports prefix queries, autocomplete, and "does any word start with X?" — all in O(L) time. Tries also share storage for common prefixes, which can be more space-efficient when the dataset has many strings with common prefixes (e.g., a dictionary of English words all starting with the same root). Choose a Trie when you need prefix search, autocomplete, spell checking, or IP routing (CIDR prefix matching). Choose a HashMap when you need only exact-match lookups and the set of strings has few shared prefixes — it has lower constant overhead.

---

**Q: Walk through how a Segment Tree answers a range query and why it's O(log n).**

A: A Segment Tree is a complete binary tree where each node stores the aggregate (sum, min, or max) of a contiguous array range. The root covers the full array; each node's children cover the left and right halves. During a range query `[l, r]`, the algorithm visits each node and makes one of three decisions: if the node's range is completely outside `[l, r]`, return the identity value (0 for sum) immediately; if completely inside, return the precomputed aggregate without recursing; if partially overlapping, recurse into both children. The O(log n) bound follows because at each level of the tree, at most 4 nodes are "partially overlapping" — all others are either completely inside or completely outside and handled in O(1). With O(log n) levels and at most 4 active nodes per level, the total work is O(log n).

---

**Q: What is the LCA problem and why is Binary Lifting efficient for multiple queries?**

A: The Lowest Common Ancestor of two nodes p and q is the deepest node that is an ancestor of both. The naive approach traverses from the root each time — O(n) per query. Binary Lifting preprocesses the tree by storing for each node its 1st, 2nd, 4th, 8th... (2^k-th) ancestor in a table — O(n log n) preprocessing. For a query, it first brings the deeper node up to the same depth as the shallower one using binary representation of the depth difference, then both nodes ascend in powers of two until they converge just below the LCA. This is O(log n) per query. Binary Lifting is the practical choice when you have many LCA queries on the same tree — the preprocessing cost is amortized. For a single query, the naive O(n) DFS is simpler; for millions of queries, Binary Lifting or Euler Tour + sparse table RMQ are appropriate.

## 9. Key Takeaways

- **BST** gives O(log n) operations when balanced but degrades to O(n) on sorted insertion — self-balancing trees fix this
- **AVL** is strictly balanced (faster reads, more rotations on write); **Red-Black** is loosely balanced (fewer rotations — Java's `TreeMap`/`TreeSet`)
- **B+ Trees** dominate databases: internal nodes hold only keys (more keys per page), leaves linked in a list (O(k) range scans)
- **Trie** excels at prefix operations (autocomplete, spell check) — O(L) per operation regardless of dataset size
- **Segment Tree** answers range queries in O(log n) with O(log n) updates; lazy propagation extends this to range updates
- **In-order DFS on BST = sorted output**; pre-order = serialization; post-order = deletion/evaluation
- **LCA naive** = O(n); **BST LCA** = O(h); **Binary Lifting** = O(log n) per query after O(n log n) preprocessing
- Always consider tree height: balanced = O(log n), degenerate = O(n) — height determines all operation costs
