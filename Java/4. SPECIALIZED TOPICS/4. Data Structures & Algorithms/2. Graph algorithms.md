# Graph Algorithms

## Overview

A graph is a collection of **vertices** (nodes) connected by **edges**. Graphs model virtually everything: road networks (Dijkstra's), dependency resolution (topological sort), social networks (BFS/DFS), circuit boards (MST), and financial systems (cycle detection). Mastering graphs is indispensable for both system design and coding interviews.

**Core vocabulary:**

```
Directed (digraph): edges have direction A → B
Undirected:         edges are bidirectional A — B
Weighted:           edges carry a cost/distance
Unweighted:         all edges treated as equal
Connected:          every vertex reachable from any other (undirected)
Strongly Connected: every vertex reachable from every other (directed)
DAG:                Directed Acyclic Graph — no directed cycles
Sparse:             E ≈ V (few edges)     → prefer adjacency list
Dense:              E ≈ V² (many edges)   → adjacency matrix acceptable
```

## 1. Graph Representations

### 1.1 Adjacency List

Stores a list of neighbours for each vertex. The standard choice for most algorithms.

```java
// Unweighted undirected graph
public class Graph {
    private final int V;
    private final List<List<Integer>> adj;

    public Graph(int V) {
        this.V = V;
        adj = new ArrayList<>();
        for (int i = 0; i < V; i++) adj.add(new ArrayList<>());
    }

    public void addEdge(int u, int v) {
        adj.get(u).add(v);
        adj.get(v).add(u); // omit for directed graph
    }

    public List<Integer> neighbours(int v) { return adj.get(v); }
}

// Weighted directed graph
public class WeightedGraph {
    record Edge(int to, int weight) {}

    private final int V;
    private final List<List<Edge>> adj;

    public WeightedGraph(int V) {
        this.V = V;
        adj = new ArrayList<>();
        for (int i = 0; i < V; i++) adj.add(new ArrayList<>());
    }

    public void addEdge(int from, int to, int weight) {
        adj.get(from).add(new Edge(to, weight));
        // adj.get(to).add(new Edge(from, weight)); // for undirected
    }

    public List<Edge> neighbours(int v) { return adj.get(v); }
}

// Graph with String labels — common in interview problems
public class LabelledGraph {
    private final Map<String, List<String>> adj = new HashMap<>();

    public void addEdge(String u, String v) {
        adj.computeIfAbsent(u, k -> new ArrayList<>()).add(v);
        adj.computeIfAbsent(v, k -> new ArrayList<>()).add(u);
    }

    public List<String> neighbours(String v) {
        return adj.getOrDefault(v, Collections.emptyList());
    }
}
```

### 1.2 Adjacency Matrix

A 2D boolean (or weight) array where `matrix[u][v] = true` means an edge exists from u to v.

```java
public class GraphMatrix {
    private final int V;
    private final int[][] matrix; // int for weighted, boolean for unweighted

    public GraphMatrix(int V) {
        this.V = V;
        matrix = new int[V][V]; // 0 = no edge
    }

    public void addEdge(int u, int v, int weight) {
        matrix[u][v] = weight;
        matrix[v][u] = weight; // omit for directed
    }

    public boolean hasEdge(int u, int v) { return matrix[u][v] != 0; }
    public int weight(int u, int v)      { return matrix[u][v]; }

    // Iterate all neighbours — O(V) even if sparse
    public List<Integer> neighbours(int u) {
        List<Integer> result = new ArrayList<>();
        for (int v = 0; v < V; v++)
            if (matrix[u][v] != 0) result.add(v);
        return result;
    }
}
```

### 1.3 Edge List

Simply a list of all edges — used by algorithms that iterate over edges (Kruskal's MST, Bellman-Ford).

```java
record Edge(int from, int to, int weight) {}

List<Edge> edges = new ArrayList<>();
edges.add(new Edge(0, 1, 4));
edges.add(new Edge(0, 2, 1));
edges.add(new Edge(2, 1, 2));
```

### 1.4 Representation Comparison

||Adjacency List|Adjacency Matrix|Edge List|
|---|---|---|---|
|Space|O(V + E)|O(V²)|O(E)|
|Add edge|O(1)|O(1)|O(1)|
|Check edge (u,v)|O(degree)|O(1)|O(E)|
|Iterate neighbours|O(degree)|O(V)|O(E)|
|Best for|Sparse graphs, most algorithms|Dense graphs, O(1) edge lookup|Edge-centric algorithms (Kruskal, Bellman-Ford)|

## 2. BFS and DFS on Graphs

### 2.1 Breadth-First Search

BFS explores level by level using a **queue**. Finds the shortest path (fewest edges) in unweighted graphs.

```java
// BFS — shortest path in unweighted graph
public int[] bfs(List<List<Integer>> adj, int src, int V) {
    int[] dist = new int[V];
    Arrays.fill(dist, -1);          // -1 = unvisited
    dist[src] = 0;

    Queue<Integer> queue = new LinkedList<>();
    queue.offer(src);

    while (!queue.isEmpty()) {
        int u = queue.poll();
        for (int v : adj.get(u)) {
            if (dist[v] == -1) {    // unvisited
                dist[v] = dist[u] + 1;
                queue.offer(v);
            }
        }
    }
    return dist; // dist[v] = shortest path length from src to v, -1 if unreachable
}

// BFS with path reconstruction
public List<Integer> shortestPath(List<List<Integer>> adj, int src, int dst, int V) {
    int[] prev = new int[V];
    Arrays.fill(prev, -1);
    boolean[] visited = new boolean[V];
    visited[src] = true;

    Queue<Integer> queue = new LinkedList<>();
    queue.offer(src);

    while (!queue.isEmpty()) {
        int u = queue.poll();
        if (u == dst) break;

        for (int v : adj.get(u)) {
            if (!visited[v]) {
                visited[v] = true;
                prev[v] = u;
                queue.offer(v);
            }
        }
    }

    // Reconstruct path from dst back to src
    if (!visited[dst]) return Collections.emptyList(); // no path

    List<Integer> path = new ArrayList<>();
    for (int v = dst; v != -1; v = prev[v]) path.add(v);
    Collections.reverse(path);
    return path;
}

// Multi-source BFS — start from multiple sources simultaneously
// Example: "0" cells are land, "1" cells are water — find max distance
// to nearest land for each water cell
public int[][] multiSourceBFS(int[][] grid, int rows, int cols) {
    int[][] dist = new int[rows][cols];
    Queue<int[]> queue = new LinkedList<>();

    for (int[] row : dist) Arrays.fill(row, Integer.MAX_VALUE);

    // Enqueue all sources at distance 0
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            if (grid[r][c] == 0) { dist[r][c] = 0; queue.offer(new int[]{r, c}); }

    int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
    while (!queue.isEmpty()) {
        int[] cell = queue.poll();
        for (int[] d : dirs) {
            int nr = cell[0] + d[0], nc = cell[1] + d[1];
            if (nr >= 0 && nr < rows && nc >= 0 && nc < cols
                    && dist[nr][nc] == Integer.MAX_VALUE) {
                dist[nr][nc] = dist[cell[0]][cell[1]] + 1;
                queue.offer(new int[]{nr, nc});
            }
        }
    }
    return dist;
}
```

### 2.2 Depth-First Search

DFS explores as deep as possible along each branch using a **stack** (or recursion).

```java
// Iterative DFS
public void dfs(List<List<Integer>> adj, int src, int V) {
    boolean[] visited = new boolean[V];
    Deque<Integer> stack = new ArrayDeque<>();
    stack.push(src);

    while (!stack.isEmpty()) {
        int u = stack.pop();
        if (visited[u]) continue;
        visited[u] = true;
        System.out.print(u + " ");

        for (int v : adj.get(u))
            if (!visited[v]) stack.push(v);
    }
}

// Recursive DFS — cleaner for most problems
public void dfsRec(List<List<Integer>> adj, int u, boolean[] visited) {
    visited[u] = true;
    System.out.print(u + " ");

    for (int v : adj.get(u))
        if (!visited[v])
            dfsRec(adj, v, visited);
}

// Count connected components
public int countComponents(int V, List<List<Integer>> adj) {
    boolean[] visited = new boolean[V];
    int components = 0;

    for (int v = 0; v < V; v++) {
        if (!visited[v]) {
            dfsRec(adj, v, visited); // DFS marks entire component
            components++;
        }
    }
    return components;
}

// Find all paths from src to dst — DFS with backtracking
public List<List<Integer>> allPaths(List<List<Integer>> adj, int src, int dst) {
    List<List<Integer>> result = new ArrayList<>();
    List<Integer> path = new ArrayList<>();
    path.add(src);
    dfsAllPaths(adj, src, dst, path, result);
    return result;
}

private void dfsAllPaths(List<List<Integer>> adj, int u, int dst,
                          List<Integer> path, List<List<Integer>> result) {
    if (u == dst) { result.add(new ArrayList<>(path)); return; }

    for (int v : adj.get(u)) {
        if (!path.contains(v)) { // avoid cycles
            path.add(v);
            dfsAllPaths(adj, v, dst, path, result);
            path.remove(path.size() - 1); // backtrack
        }
    }
}
```

### 2.3 BFS vs DFS — when to use each

||BFS|DFS|
|---|---|---|
|Data structure|Queue (FIFO)|Stack / Recursion|
|Shortest path|✓ (unweighted)|✗|
|Cycle detection|✓|✓|
|Connected components|✓|✓|
|Topological sort|✓ (Kahn's)|✓ (DFS-based)|
|Memory (wide graphs)|High (stores level)|Low|
|Memory (deep graphs)|Low|High (stack depth)|
|Completeness|Yes|Yes (with visited set)|

## 3. Dijkstra's Shortest Path

Dijkstra's finds the shortest path from a source to all vertices in a **weighted graph with non-negative weights**. It greedily selects the unvisited vertex with the minimum known distance.

```
Graph:
  0 --4-- 1
  |       |
  1       2
  |       |
  2 --5-- 3

From source 0:
  dist = [0, ∞, ∞, ∞]
  Process 0 (dist=0): update 1→4, 2→1
  dist = [0, 4, 1, ∞]
  Process 2 (dist=1): update 3→6
  dist = [0, 4, 1, 6]
  Process 1 (dist=4): update 3→min(6, 4+2)=6
  dist = [0, 4, 1, 6]
  Process 3 (dist=6): no updates
  Final: [0, 4, 1, 6]
```

```java
public class Dijkstra {

    record Edge(int to, int weight) {}

    // O((V + E) log V) with priority queue
    public int[] dijkstra(List<List<Edge>> adj, int src, int V) {
        int[] dist = new int[V];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // Min-heap: [distance, vertex]
        PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        pq.offer(new int[]{0, src});

        while (!pq.isEmpty()) {
            int[] curr = pq.poll();
            int d = curr[0], u = curr[1];

            // Stale entry — already processed with a shorter path
            if (d > dist[u]) continue;

            for (Edge edge : adj.get(u)) {
                int newDist = dist[u] + edge.weight();
                if (newDist < dist[edge.to()]) {
                    dist[edge.to()] = newDist;
                    pq.offer(new int[]{newDist, edge.to()});
                }
            }
        }
        return dist;
    }

    // With path reconstruction
    public int[] dijkstraWithPath(List<List<Edge>> adj, int src, int V, int[] prev) {
        int[] dist = new int[V];
        Arrays.fill(dist, Integer.MAX_VALUE);
        Arrays.fill(prev, -1);
        dist[src] = 0;

        PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        pq.offer(new int[]{0, src});

        while (!pq.isEmpty()) {
            int[] curr = pq.poll();
            int d = curr[0], u = curr[1];
            if (d > dist[u]) continue;

            for (Edge edge : adj.get(u)) {
                int newDist = dist[u] + edge.weight();
                if (newDist < dist[edge.to()]) {
                    dist[edge.to()] = newDist;
                    prev[edge.to()] = u;          // record predecessor
                    pq.offer(new int[]{newDist, edge.to()});
                }
            }
        }
        return dist;
    }

    public List<Integer> reconstructPath(int[] prev, int src, int dst) {
        List<Integer> path = new ArrayList<>();
        for (int v = dst; v != -1; v = prev[v]) path.add(v);
        if (path.get(path.size() - 1) != src) return Collections.emptyList();
        Collections.reverse(path);
        return path;
    }
}
```

**Dijkstra complexity with different data structures:**

|Priority Queue|Time|Use When|
|---|---|---|
|Binary heap (PQ)|O((V + E) log V)|Standard — most problems|
|Fibonacci heap|O(E + V log V)|Dense graphs, theoretical|
|Simple array|O(V²)|Dense graphs with V small|

**Dijkstra's limitation**: fails with **negative weights** — a negative edge could make a previously-settled path shorter, violating the greedy assumption. Use Bellman-Ford for negative weights.

## 4. Bellman-Ford Algorithm

Bellman-Ford finds shortest paths from a source vertex even with **negative weight edges**. It also detects **negative weight cycles** (cycles where the total weight is negative — making some shortest paths undefined as −∞).

**Core idea**: relax ALL edges V-1 times. After V-1 iterations, all shortest paths (which have at most V-1 edges) are finalized. A V-th relaxation that still improves a distance indicates a negative cycle.

```java
public class BellmanFord {

    record Edge(int from, int to, int weight) {}

    // O(V × E) time, O(V) space
    public int[] bellmanFord(List<Edge> edges, int src, int V) {
        int[] dist = new int[V];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // Relax all edges V-1 times
        for (int i = 0; i < V - 1; i++) {
            boolean updated = false;
            for (Edge e : edges) {
                if (dist[e.from()] != Integer.MAX_VALUE
                        && dist[e.from()] + e.weight() < dist[e.to()]) {
                    dist[e.to()] = dist[e.from()] + e.weight();
                    updated = true;
                }
            }
            if (!updated) break; // early exit — already optimal
        }

        // V-th pass: if any distance still improves → negative cycle
        for (Edge e : edges) {
            if (dist[e.from()] != Integer.MAX_VALUE
                    && dist[e.from()] + e.weight() < dist[e.to()]) {
                throw new IllegalStateException("Graph contains a negative weight cycle");
            }
        }

        return dist;
    }

    // Find all vertices reachable through a negative cycle (dist = -∞)
    public boolean[] negativeCycleReachable(List<Edge> edges, int src, int V) {
        int[] dist = new int[V];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        for (int i = 0; i < V - 1; i++)
            for (Edge e : edges)
                if (dist[e.from()] != Integer.MAX_VALUE
                        && dist[e.from()] + e.weight() < dist[e.to()])
                    dist[e.to()] = dist[e.from()] + e.weight();

        // Mark vertices that can be relaxed in V-th pass
        boolean[] inNegCycle = new boolean[V];
        for (Edge e : edges)
            if (dist[e.from()] != Integer.MAX_VALUE
                    && dist[e.from()] + e.weight() < dist[e.to()])
                inNegCycle[e.to()] = true;

        // Propagate: any vertex reachable from a negative-cycle vertex is also -∞
        Queue<Integer> queue = new LinkedList<>();
        for (int v = 0; v < V; v++) if (inNegCycle[v]) queue.offer(v);
        while (!queue.isEmpty()) {
            int u = queue.poll();
            for (Edge e : edges)
                if (e.from() == u && !inNegCycle[e.to()]) {
                    inNegCycle[e.to()] = true;
                    queue.offer(e.to());
                }
        }
        return inNegCycle;
    }
}
```

**Step-by-step trace:**

```
Graph: 0 →(4)→ 1 →(-3)→ 2 →(2)→ 3
               0 →(2)→ 2

Initial: dist = [0, ∞, ∞, ∞]

Iteration 1 (relax all edges):
  Edge 0→1 (w=4): dist[1] = 0+4 = 4
  Edge 0→2 (w=2): dist[2] = 0+2 = 2
  Edge 1→2 (w=-3): dist[2] = min(2, 4-3) = 1
  Edge 2→3 (w=2): dist[3] = 1+2 = 3
  dist = [0, 4, 1, 3]

Iteration 2: no improvements → done
Final: [0, 4, 1, 3]
```

**Dijkstra vs Bellman-Ford:**

||Dijkstra|Bellman-Ford|
|---|---|---|
|Negative weights|✗|✓|
|Negative cycles|Cannot detect|Detects|
|Time complexity|O((V+E) log V)|O(V × E)|
|Space|O(V)|O(V)|
|Single source|✓|✓|
|All pairs|Run V times|SPFA or Floyd-Warshall|

## 5. Topological Sort

Topological sort orders vertices of a **DAG** so that for every directed edge `u → v`, vertex `u` comes before `v`. Used for dependency resolution, build systems, course scheduling.

**Key property**: only possible on DAGs. If a cycle exists, no valid ordering exists.

### 5.1 Kahn's Algorithm (BFS-based)

```java
// Kahn's Algorithm — O(V + E)
// Uses in-degree counting: start with nodes that have no dependencies
public List<Integer> topologicalSortKahn(List<List<Integer>> adj, int V) {
    int[] inDegree = new int[V];
    for (int u = 0; u < V; u++)
        for (int v : adj.get(u))
            inDegree[v]++;

    // Start with all nodes that have no incoming edges
    Queue<Integer> queue = new LinkedList<>();
    for (int v = 0; v < V; v++)
        if (inDegree[v] == 0) queue.offer(v);

    List<Integer> order = new ArrayList<>();

    while (!queue.isEmpty()) {
        int u = queue.poll();
        order.add(u);

        for (int v : adj.get(u)) {
            inDegree[v]--;
            if (inDegree[v] == 0) queue.offer(v); // all dependencies resolved
        }
    }

    // If order doesn't contain all vertices → cycle exists
    if (order.size() != V)
        throw new IllegalStateException("Graph has a cycle — topological sort impossible");

    return order;
}
```

### 5.2 DFS-based Topological Sort

```java
// DFS post-order: push to stack when ALL descendants are processed
// Reverse of stack = topological order
public List<Integer> topologicalSortDFS(List<List<Integer>> adj, int V) {
    boolean[] visited = new boolean[V];
    Deque<Integer> stack = new ArrayDeque<>();

    for (int v = 0; v < V; v++)
        if (!visited[v])
            dfsTopoSort(adj, v, visited, stack);

    List<Integer> order = new ArrayList<>();
    while (!stack.isEmpty()) order.add(stack.pop());
    return order;
}

private void dfsTopoSort(List<List<Integer>> adj, int u,
                          boolean[] visited, Deque<Integer> stack) {
    visited[u] = true;
    for (int v : adj.get(u))
        if (!visited[v])
            dfsTopoSort(adj, v, visited, stack);
    stack.push(u); // push AFTER all descendants processed
}
```

### 5.3 Practical example — course scheduling

```java
// LeetCode 207: Course Schedule — can all courses be taken?
// Equivalent to: does the prerequisite graph contain a cycle?
public boolean canFinish(int numCourses, int[][] prerequisites) {
    List<List<Integer>> adj = new ArrayList<>();
    for (int i = 0; i < numCourses; i++) adj.add(new ArrayList<>());

    int[] inDegree = new int[numCourses];
    for (int[] pre : prerequisites) {
        adj.get(pre[1]).add(pre[0]); // pre[1] → pre[0]
        inDegree[pre[0]]++;
    }

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < numCourses; i++)
        if (inDegree[i] == 0) queue.offer(i);

    int processed = 0;
    while (!queue.isEmpty()) {
        int course = queue.poll();
        processed++;
        for (int next : adj.get(course)) {
            if (--inDegree[next] == 0) queue.offer(next);
        }
    }
    return processed == numCourses; // true = no cycle = all courses takeable
}
```

**Trace on DAG:** `5→0, 5→2, 4→0, 4→1, 2→3, 3→1`

```
In-degrees: [2, 2, 1, 1, 0, 0]
Queue: [4, 5]

Process 4: remove edges 4→0, 4→1. In-degrees: [1, 1, 1, 1, -, 0]
Process 5: remove edges 5→0, 5→2. In-degrees: [0, 1, 0, 1, -, -]
Queue: [0, 2]

Process 0: no outgoing edges.
Process 2: remove 2→3. In-degrees: [-, 1, -, 0, -, -]
Queue: [3]

Process 3: remove 3→1. In-degrees: [-, 0, -, -, -, -]
Queue: [1]

Process 1: done.
Order: [4, 5, 0, 2, 3, 1]
```

## 6. Cycle Detection

### 6.1 Undirected graph — DFS with parent tracking

```java
// A cycle exists in undirected graph if we visit an already-visited
// vertex that is NOT the immediate parent
public boolean hasCycleUndirected(List<List<Integer>> adj, int V) {
    boolean[] visited = new boolean[V];
    for (int v = 0; v < V; v++)
        if (!visited[v] && dfsUndirected(adj, v, -1, visited))
            return true;
    return false;
}

private boolean dfsUndirected(List<List<Integer>> adj, int u,
                               int parent, boolean[] visited) {
    visited[u] = true;
    for (int v : adj.get(u)) {
        if (!visited[v]) {
            if (dfsUndirected(adj, v, u, visited)) return true;
        } else if (v != parent) {
            return true; // back edge to non-parent visited node → cycle
        }
    }
    return false;
}
```

### 6.2 Directed graph — DFS with recursion stack (three-color)

For directed graphs, tracking just visited nodes is not enough — revisiting a node via a different path doesn't indicate a cycle. We need to know if we revisit a node **on the current DFS path**.

```java
// Three states: WHITE (0) = unvisited, GRAY (1) = in current path, BLACK (2) = done
public boolean hasCycleDirected(List<List<Integer>> adj, int V) {
    int[] color = new int[V]; // 0=white, 1=gray, 2=black

    for (int v = 0; v < V; v++)
        if (color[v] == 0 && dfsDirected(adj, v, color))
            return true;
    return false;
}

private boolean dfsDirected(List<List<Integer>> adj, int u, int[] color) {
    color[u] = 1; // mark GRAY — currently on DFS path

    for (int v : adj.get(u)) {
        if (color[v] == 1) return true;  // back edge to GRAY node → cycle
        if (color[v] == 0 && dfsDirected(adj, v, color)) return true;
    }

    color[u] = 2; // mark BLACK — fully processed
    return false;
}

// Equivalent with boolean arrays (cleaner)
public boolean hasCycleDirectedV2(List<List<Integer>> adj, int V) {
    boolean[] visited = new boolean[V];
    boolean[] onPath  = new boolean[V]; // in current recursion stack

    for (int v = 0; v < V; v++)
        if (!visited[v] && dfsOnPath(adj, v, visited, onPath))
            return true;
    return false;
}

private boolean dfsOnPath(List<List<Integer>> adj, int u,
                           boolean[] visited, boolean[] onPath) {
    visited[u] = onPath[u] = true;

    for (int v : adj.get(u)) {
        if (onPath[v]) return true;                         // back edge
        if (!visited[v] && dfsOnPath(adj, v, visited, onPath)) return true;
    }

    onPath[u] = false; // remove from current path on backtrack
    return false;
}
```

### 6.3 Cycle detection with BFS (Kahn's in-degree method)

```java
// If topological sort does not include all vertices → cycle exists
// Already shown in section 5.1 — Kahn's algorithm
public boolean hasCycleBFS(List<List<Integer>> adj, int V) {
    int[] inDegree = new int[V];
    for (int u = 0; u < V; u++)
        for (int v : adj.get(u)) inDegree[v]++;

    Queue<Integer> queue = new LinkedList<>();
    for (int v = 0; v < V; v++) if (inDegree[v] == 0) queue.offer(v);

    int processed = 0;
    while (!queue.isEmpty()) {
        int u = queue.poll(); processed++;
        for (int v : adj.get(u)) if (--inDegree[v] == 0) queue.offer(v);
    }
    return processed != V; // if not all processed → cycle
}
```

### 6.4 Floyd's Cycle Detection (Linked List / fast-slow pointers)

For linked lists or functional sequences (not adjacency-list graphs):

```java
// Floyd's Tortoise and Hare — O(n) time, O(1) space
public boolean hasCycleLinkedList(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true; // cycle detected
    }
    return false;
}

// Find cycle entry point
public ListNode detectCycleStart(ListNode head) {
    ListNode slow = head, fast = head;

    // Phase 1: detect meeting point
    while (fast != null && fast.next != null) {
        slow = slow.next; fast = fast.next.next;
        if (slow == fast) break;
    }
    if (fast == null || fast.next == null) return null; // no cycle

    // Phase 2: move one pointer to head, advance both at same speed
    // They meet at the cycle entry point
    slow = head;
    while (slow != fast) { slow = slow.next; fast = fast.next; }
    return slow; // cycle start
}
```

## 7. Minimum Spanning Tree (Kruskal's, Prim's)

A **Minimum Spanning Tree** (MST) of a connected, undirected, weighted graph is a subset of edges that connects all vertices with the **minimum total weight** and **no cycles** (tree = V-1 edges).

**MST properties:**

```
- Exactly V-1 edges (for V vertices)
- Acyclic (by definition of tree)
- Connects all vertices (spanning)
- Minimum total weight among all spanning trees
- MST may not be unique (equal weights)
- Cut property: the minimum-weight edge crossing any cut is in some MST
```

### 7.1 Kruskal's Algorithm — edge-centric

Sort all edges by weight. Add edge if it doesn't form a cycle (use **Union-Find** to check efficiently).

```java
public class Kruskal {

    // Union-Find (Disjoint Set Union) — essential for Kruskal's
    static class UnionFind {
        int[] parent, rank;

        UnionFind(int n) {
            parent = new int[n];
            rank = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i; // each node is its own root
        }

        int find(int x) {
            if (parent[x] != x)
                parent[x] = find(parent[x]); // path compression
            return parent[x];
        }

        boolean union(int x, int y) {
            int rx = find(x), ry = find(y);
            if (rx == ry) return false; // same component → adding edge creates cycle

            // Union by rank — attach smaller tree under larger
            if (rank[rx] < rank[ry])      parent[rx] = ry;
            else if (rank[rx] > rank[ry]) parent[ry] = rx;
            else { parent[ry] = rx; rank[rx]++; }

            return true; // successfully merged
        }
    }

    record Edge(int from, int to, int weight) {}

    // O(E log E) due to sorting
    public List<Edge> kruskal(int V, List<Edge> edges) {
        // Sort edges by weight ascending
        edges.sort(Comparator.comparingInt(Edge::weight));

        UnionFind uf = new UnionFind(V);
        List<Edge> mst = new ArrayList<>();

        for (Edge e : edges) {
            if (uf.union(e.from(), e.to())) { // only add if no cycle
                mst.add(e);
                if (mst.size() == V - 1) break; // MST complete
            }
        }

        if (mst.size() < V - 1)
            throw new IllegalStateException("Graph is not connected — no MST exists");

        return mst;
    }

    public int mstWeight(int V, List<Edge> edges) {
        return kruskal(V, edges).stream().mapToInt(Edge::weight).sum();
    }
}
```

**Kruskal's trace:**

```
Graph edges (sorted): (0-3,5), (0-1,10), (1-3,15), (0-2,20), (2-3,30)

Add (0-3, w=5): UF: {0,3}, {1}, {2}. MST: [(0,3,5)]
Add (0-1, w=10): UF: {0,1,3}, {2}. MST: [(0,3,5),(0,1,10)]
Add (1-3, w=15): 1 and 3 already connected → skip (cycle)
Add (0-2, w=20): UF: {0,1,2,3}. MST: [(0,3,5),(0,1,10),(0,2,20)]
V-1 = 3 edges → done. Total weight = 35
```

### 7.2 Prim's Algorithm — vertex-centric

Grow the MST from a start vertex by always adding the **cheapest edge that connects a tree vertex to a non-tree vertex**.

```java
public class Prim {

    record Edge(int to, int weight) {}

    // O((V + E) log V) with priority queue
    public int[] prim(List<List<Edge>> adj, int src, int V) {
        int[] minCost = new int[V];  // minimum edge weight to include vertex
        int[] parent  = new int[V];  // MST parent
        boolean[] inMST = new boolean[V];

        Arrays.fill(minCost, Integer.MAX_VALUE);
        Arrays.fill(parent, -1);
        minCost[src] = 0;

        // Min-heap: [cost, vertex]
        PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        pq.offer(new int[]{0, src});

        int totalWeight = 0;

        while (!pq.isEmpty()) {
            int[] curr = pq.poll();
            int cost = curr[0], u = curr[1];

            if (inMST[u]) continue; // already in MST
            inMST[u] = true;
            totalWeight += cost;

            for (Edge e : adj.get(u)) {
                if (!inMST[e.to()] && e.weight() < minCost[e.to()]) {
                    minCost[e.to()] = e.weight();
                    parent[e.to()] = u;
                    pq.offer(new int[]{e.weight(), e.to()});
                }
            }
        }

        System.out.println("MST total weight: " + totalWeight);
        return parent; // parent[v] = u means edge u-v is in MST
    }
}
```

**Prim's trace (starting from vertex 0):**

```
Graph: 0-1(2), 0-3(6), 1-2(3), 1-3(8), 1-4(5), 2-4(7), 3-4(9)

inMST = {}, minCost = [0,∞,∞,∞,∞]
PQ: [(0,0)]

Process 0 (cost=0): update 1→2, 3→6
PQ: [(2,1),(6,3)], minCost=[0,2,∞,6,∞]

Process 1 (cost=2): update 2→3, 4→5 (3<8, 5<∞, 6>2 skip)
PQ: [(3,2),(5,4),(6,3)], minCost=[0,2,3,6,5]

Process 2 (cost=3): update 4→min(5,7)=5 → no change
Process 4 (cost=5): update 3→min(6,9)=6 → no change
Process 3 (cost=6): all neighbours already in MST

MST edges: 0-1(2), 1-2(3), 1-4(5), 0-3(6). Total = 16
```

### 7.3 Kruskal's vs Prim's

||Kruskal's|Prim's|
|---|---|---|
|Approach|Edge-centric (sort then add)|Vertex-centric (grow from seed)|
|Data structure|Union-Find + sorted edges|Priority Queue|
|Time complexity|O(E log E)|O((V + E) log V)|
|Best for|Sparse graphs (E ≈ V)|Dense graphs (E ≈ V²)|
|Implementation|Simpler|Slightly more complex|
|Disconnected graphs|Produces forest|Produces only one component's MST|

## 8. Floyd-Warshall — All-Pairs Shortest Path

When you need shortest paths between **all pairs** of vertices:

```java
// O(V³) time, O(V²) space — works with negative weights (no negative cycles)
public int[][] floydWarshall(int[][] weight, int V) {
    int[][] dist = new int[V][V];

    // Initialize: dist[i][j] = direct edge weight, or ∞ if no edge
    for (int i = 0; i < V; i++) {
        for (int j = 0; j < V; j++) {
            dist[i][j] = (i == j) ? 0 : weight[i][j];
        }
    }

    // For each intermediate vertex k:
    // Can we improve i→j by going i→k→j?
    for (int k = 0; k < V; k++)
        for (int i = 0; i < V; i++)
            for (int j = 0; j < V; j++)
                if (dist[i][k] != Integer.MAX_VALUE
                        && dist[k][j] != Integer.MAX_VALUE)
                    dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);

    // Check for negative cycles: if dist[i][i] < 0, vertex i is in a negative cycle
    for (int i = 0; i < V; i++)
        if (dist[i][i] < 0)
            throw new IllegalStateException("Negative weight cycle detected");

    return dist;
}
```

## 9. Algorithm Selection Guide

```
What is the problem?

Find shortest path, unweighted graph:
    → BFS from source — O(V + E)

Find shortest path, weighted, non-negative weights:
    → Dijkstra's — O((V + E) log V)

Find shortest path, negative weights, detect negative cycles:
    → Bellman-Ford — O(V × E)

All-pairs shortest paths:
    → Floyd-Warshall — O(V³) or run Dijkstra V times — O(V(V+E) log V)

Order tasks respecting dependencies (DAG):
    → Topological Sort (Kahn's or DFS-based) — O(V + E)

Does a cycle exist?
    → Undirected: DFS with parent tracking
    → Directed: DFS with recursion stack (gray/white/black)
    → Kahn's: if topo sort is incomplete

Connect all vertices with minimum total edge weight:
    → Kruskal's (sparse) or Prim's (dense) — O(E log E)

Find connected components:
    → DFS/BFS over all unvisited vertices — O(V + E)

Is a path reachable between u and v?
    → BFS/DFS — O(V + E)
```

## 10. Interview Questions

**Q: Why does Dijkstra's algorithm fail with negative weight edges?**

A: Dijkstra greedily "settles" the vertex with the minimum current distance, assuming it will never be improved. With non-negative weights this holds: once a vertex is settled, no future relaxation through an unvisited vertex can produce a smaller distance (all future paths are at least as long). With a negative edge, a later-discovered path could go through that negative edge and arrive at a settled vertex with a shorter total distance, invalidating the greedy assumption. For example, settling vertex A at distance 5, then discovering a path of 10 + (−8) = 2 through vertex B — Dijkstra misses this since A was already settled. Bellman-Ford handles this by relaxing all edges V-1 times without any greedy settlement.

---

**Q: What is the difference between cycle detection in directed vs undirected graphs?**

A: In an undirected graph, a back edge to any visited node other than the immediate parent indicates a cycle — that's sufficient. In a directed graph, revisiting a node alone doesn't indicate a cycle: you might reach the same node via two entirely separate paths (a diamond shape). The condition is specifically a back edge to a node currently **on the active DFS stack** (marked GRAY). If we reach a node that's fully processed (BLACK), it just means we've found a multi-path convergence, not a cycle. The three-color DFS (white/gray/black) or the `onPath[]` boolean array correctly distinguishes these cases.

---

**Q: When would you use Kruskal's over Prim's and vice versa?**

A: Kruskal's processes edges globally — it sorts all E edges and uses Union-Find to avoid cycles. Its time complexity is O(E log E), making it efficient for sparse graphs where E is close to V. Prim's grows the MST vertex-by-vertex from a seed, using a priority queue to always pick the cheapest next edge. Its complexity is O((V + E) log V), which for dense graphs (E ≈ V²) gives O(V² log V) — better than Kruskal's O(V² log V²). In practice, Kruskal's is simpler to implement and preferred for sparse graphs; Prim's is preferred when the graph is dense or when the graph is given as an adjacency matrix rather than an edge list.

---

**Q: What does the V-th relaxation pass in Bellman-Ford tell you?**

A: After V-1 iterations of relaxing all edges, all shortest paths have been found — because any shortest path in a graph with V vertices can have at most V-1 edges (otherwise it would visit a vertex twice, forming a cycle). If performing a V-th relaxation pass still reduces any distance, it means there is a path that could be made shorter by adding another edge — only possible if there's a cycle with negative total weight. So the V-th pass is specifically a negative cycle detector: any vertex whose distance can still be reduced is either in a negative cycle or reachable from one.

---

**Q: Why does topological sort only work on DAGs, and how does Kahn's algorithm detect cycles?**

A: Topological ordering requires placing every vertex before all vertices it points to. In a cycle (A → B → C → A), A must come before B, B before C, and C before A — a contradiction that makes ordering impossible. Kahn's algorithm naturally detects this: it only processes vertices with in-degree 0 (no remaining dependencies). Vertices in a cycle always have at least one incoming edge from within the cycle, so their in-degree never reaches 0 — they are never added to the queue. At the end, if the number of processed vertices is less than V, the unprocessed vertices form cycles.

---

**Q: Explain the Union-Find data structure and why Kruskal's uses it.**

A: Union-Find (Disjoint Set Union) maintains a collection of disjoint sets and supports two operations: `find(x)` — which root/representative does x belong to — and `union(x, y)` — merge x's set with y's set. Kruskal's needs to answer one question for each candidate edge (u, v): are u and v already in the same connected component? If yes, adding the edge creates a cycle (skip); if no, add it and merge the components. Union-Find with path compression and union by rank answers both operations in amortized O(α(n)) time — nearly O(1) in practice. Without Union-Find, Kruskal's would need an O(V) DFS/BFS cycle check per edge, making the total O(E × V) instead of O(E log E).

## 11. Key Takeaways

- **Adjacency list** is the default for sparse graphs (O(V + E) space); use **adjacency matrix** only when O(1) edge lookup is critical for dense graphs
- **BFS** finds shortest paths in unweighted graphs; **DFS** is simpler for connectivity, cycle detection, and topological sort
- **Dijkstra** is the go-to for weighted shortest paths with non-negative edges; **Bellman-Ford** handles negative edges and detects negative cycles at cost of O(V × E)
- **Topological sort** requires a DAG — Kahn's naturally detects cycles via in-degree; DFS approach uses post-order stack reversal
- **Cycle detection**: undirected uses parent-tracking DFS; directed uses recursion-stack (gray) DFS or Kahn's incomplete topo sort
- **Kruskal's** (sort edges + Union-Find) for sparse MST; **Prim's** (priority queue + vertex expansion) for dense MST
- **Union-Find** with path compression + union by rank is the key primitive for Kruskal's — nearly O(1) per operation
- **Floyd-Warshall** for all-pairs shortest paths in O(V³) — negative weights allowed, negative cycles detectable via diagonal
