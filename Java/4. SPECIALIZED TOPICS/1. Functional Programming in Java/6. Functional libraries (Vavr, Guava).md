# Functional Libraries (Vavr, Guava)

## Overview

Functional libraries extend Java with powerful abstractions from functional programming—Vavr provides persistent collections, pattern matching, tuples, and functional error handling; Guava offers immutable collections, functional utilities, and caching. This guide covers six critical aspects: Vavr Collections (List, Set, Map are immutable and persistent, structural sharing for efficiency, richer API than Java collections, fluent transformations with map/filter/flatMap), Pattern Matching in Vavr (structural pattern matching with Match API, type-safe case expressions, decompose complex structures, eliminate if-else chains and instanceof checks), Function Composition (compose functions with andThen/compose, lift functions to work with Optional, memoize for caching, partial application utilities), Tuple Types (Tuple2 through Tuple8 for multiple return values, typed product types without creating classes, pattern matching on tuples, useful for intermediate results), Try and Either (Try wraps operations that throw, Either for typed errors, Success/Failure containers compose with map/flatMap, eliminates try-catch blocks), and Lazy Values (deferred computation with caching, computed once on first access, thread-safe, useful for expensive initialization). The key insight: these libraries bring functional programming ergonomics to Java—instead of fighting the language, you get clean functional patterns that compose naturally. Common mistakes: mixing Vavr and Java collections carelessly (they don't interoperate directly, need conversion), overusing pattern matching where simple polymorphism would be clearer, creating too many tuple types instead of proper domain objects (Tuple3<String, Integer, Boolean> is less clear than User), importing wrong Either (Vavr's vs others), not understanding persistent collections (they're immutable but efficient through structural sharing).


**The Philosophy:**

```
"Java lacks many functional programming constructs.
 Libraries fill the gap."

Standard Java gives you:
  - Optional (nullable handling)
  - Stream (lazy collections)
  - CompletableFuture (async)
  
But lacks:
  - Persistent collections (immutable + efficient)
  - Pattern matching (structural decomposition)
  - Tuples (multiple returns without classes)
  - Try/Either (functional exception handling)
  - Function composition utilities
  - Lazy values with memoization

Vavr fills these gaps.
Guava complements with utilities.

The result: Functional programming that feels natural.
```

import io.vavr.collection.List;
import io.vavr.collection.Map;
import io.vavr.collection.Set;

/**
 * Vavr collections are:
 *   - Immutable (cannot be modified)
 *   - Persistent (share structure between versions)
 *   - Efficient (O(log n) operations, not O(n) copies)
 *   - Rich API (map, filter, flatMap, etc.)
 */

public class VavrCollectionsIntro {
    
    // ✅ List: Immutable linked list
    public void listExample() {
        List<Integer> list1 = List.of(1, 2, 3);
        
        // Operations return NEW list, original unchanged
        List<Integer> list2 = list1.append(4);
        // list1 is still [1, 2, 3]
        // list2 is [1, 2, 3, 4]
        // Structural sharing: they share the [1, 2, 3] part
        
        List<Integer> list3 = list1.prepend(0);
        // [0, 1, 2, 3]
        
        List<Integer> doubled = list1.map(n -> n * 2);
        // [2, 4, 6]
        
        List<Integer> evens = list1.filter(n -> n % 2 == 0);
        // [2]
    }
    
    // ✅ Set: Immutable hash set
    public void setExample() {
        Set<String> set1 = Set.of("A", "B", "C");
        
        Set<String> set2 = set1.add("D");
        // set1 unchanged: ["A", "B", "C"]
        // set2: ["A", "B", "C", "D"]
        
        Set<String> set3 = set1.remove("B");
        // ["A", "C"]
        
        boolean contains = set1.contains("B");  // true
    }
    
    // ✅ Map: Immutable hash map
    public void mapExample() {
        Map<String, Integer> map1 = Map.of(
            "Alice", 25,
            "Bob", 30
        );
        
        Map<String, Integer> map2 = map1.put("Carol", 28);
        // map1 unchanged
        // map2 has all three entries
        
        Map<String, Integer> map3 = map1.remove("Bob");
        // Only has Alice
        
        Option<Integer> age = map1.get("Alice");
        // Returns Option (Vavr's Optional equivalent)
    }
}

## 2. Pattern Matching in Vavr

### Structural Decomposition

```java
// ========================================
// PATTERN MATCHING: Match API
// ========================================

import static io.vavr.API.*;
import static io.vavr.Predicates.*;

public class PatternMatchingBasics {
    
    // ✅ Basic pattern matching
    public String describeNumber(int n) {
        return Match(n).of(
            Case($(0), "zero"),
            Case($(1), "one"),
            Case($(2), "two"),
            Case($(), "other")  // Default case
        );
    }
    
    // ✅ Predicate matching
    public String classifyNumber(int n) {
        return Match(n).of(
            Case($(isIn(-1, 0, 1)), "small"),
            Case($(isIn(2, 3, 4)), "medium"),
            Case($(i -> i > 10), "large"),
            Case($(), "other")
        );
    }
    
    // ✅ Type matching
    public String describeObject(Object obj) {
        return Match(obj).of(
            Case($(instanceOf(String.class)), s -> "String: " + s),
            Case($(instanceOf(Integer.class)), i -> "Integer: " + i),
            Case($(instanceOf(List.class)), l -> "List of size: " + ((List<?>) l).size()),
            Case($(), o -> "Unknown: " + o.getClass().getSimpleName())
        );
    }
}
```

public class AdvancedPatternMatching {
    
    // ✅ Match on Try
    public String processResult(Try<Integer> result) {
        return Match(result).of(
            Case($Success($()), value -> "Success: " + value),
            Case($Failure($()), error -> "Error: " + error.getMessage())
        );
    }
    
    // ✅ Match on Either
    public String processEither(Either<String, User> result) {
        return Match(result).of(
            Case($Right($()), user -> "User: " + user.getName()),
            Case($Left($()), error -> "Error: " + error)
        );
    }
    
    // ✅ Match on Option
    public String processOption(Option<String> opt) {
        return Match(opt).of(
            Case($Some($()), value -> "Found: " + value),
            Case($None(), "Not found")
        );
    }
    
    // ✅ Nested pattern matching
    public String processNestedOption(Option<Option<String>> nested) {
        return Match(nested).of(
            Case($Some($Some($())), value -> "Found: " + value),
            Case($Some($None()), "Found empty"),
            Case($None(), "Not found")
        );
    }
}

public class HttpResponseHandler {
    
    sealed interface HttpResponse {
        record Success(int statusCode, String body) implements HttpResponse {}
        record ClientError(int statusCode, String message) implements HttpResponse {}
        record ServerError(int statusCode, String message) implements HttpResponse {}
        record NetworkError(String message) implements HttpResponse {}
    }
    
    // ✅ Pattern matching on sealed interface
    public String handleResponse(HttpResponse response) {
        return Match(response).of(
            Case($(instanceOf(HttpResponse.Success.class)), 
                r -> "Success: " + r.body()),
            Case($(instanceOf(HttpResponse.ClientError.class)), 
                r -> "Client error " + r.statusCode() + ": " + r.message()),
            Case($(instanceOf(HttpResponse.ServerError.class)), 
                r -> "Server error " + r.statusCode() + ": " + r.message()),
            Case($(instanceOf(HttpResponse.NetworkError.class)), 
                r -> "Network error: " + r.message())
        );
    }
    
    // ✅ Pattern matching with predicates
    public int getRetryDelay(HttpResponse response) {
        return Match(response).of(
            Case($(instanceOf(HttpResponse.ServerError.class)), r -> 
                r.statusCode() == 503 ? 60 : 10),  // 503: retry after 60s
            Case($(instanceOf(HttpResponse.NetworkError.class)), 5),
            Case($(), 0)  // Don't retry for other errors
        );
    }
}

## 3. Function Composition

### Composing Functions with Vavr

```java
// ========================================
// FUNCTION COMPOSITION: andThen and compose
// ========================================

import io.vavr.Function1;
import io.vavr.Function2;

public class VavrFunctionComposition {
    
    // ✅ Function1: Single-argument function
    public void functionComposition() {
        Function1<String, String> trim = String::trim;
        Function1<String, String> lowercase = String::toLowerCase;
        Function1<String, Integer> length = String::length;
        
        // Compose: trim → lowercase → length
        Function1<String, Integer> pipeline = trim
            .andThen(lowercase)
            .andThen(length);
        
        int result = pipeline.apply("  HELLO  ");  // 5
    }
    
    // ✅ Function2: Two-argument function
    public void biFunctionExample() {
        Function2<Integer, Integer, Integer> add = (a, b) -> a + b;
        
        // Partial application: fix first argument
        Function1<Integer, Integer> add5 = add.apply(5);
        
        int result = add5.apply(10);  // 15
    }
    
    // ✅ Currying: Transform to curried form
    public void curryingExample() {
        Function2<Integer, Integer, Integer> add = (a, b) -> a + b;
        
        Function1<Integer, Function1<Integer, Integer>> curried = add.curried();
        
        int result = curried.apply(5).apply(10);  // 15
    }
}
```

import io.vavr.Tuple;

/**
 * Vavr provides Tuple1 through Tuple8

public class TupleExamples {
    
    // ✅ Tuple2: Two values
    public Tuple2<String, Integer> getNameAndAge(User user) {
        return Tuple.of(user.getName(), user.getAge());
    }
    
    public void useTuple2() {
        Tuple2<String, Integer> result = getNameAndAge(user);
        
        String name = result._1;   // Access first element
        Integer age = result._2;   // Access second element
        
        // Or use method syntax
        String name2 = result._1();
        Integer age2 = result._2();
    }
    
    // ✅ Tuple3: Three values
    public Tuple3<String, String, Integer> parseUserData(String data) {
        String[] parts = data.split(",");
        return Tuple.of(parts[0], parts[1], Integer.parseInt(parts[2]));
    }
    
    // ✅ map: Transform tuple elements
    public void transformTuple() {
        Tuple2<String, Integer> tuple = Tuple.of("Alice", 25);
        
        // Map first element
        Tuple2<String, Integer> mapped1 = tuple.map1(String::toUpperCase);
        // ("ALICE", 25)
        
        // Map second element
        Tuple2<String, Integer> mapped2 = tuple.map2(age -> age + 1);
        // ("Alice", 26)
        
        // Map both
        Tuple2<Integer, String> mapped3 = tuple.map(
            String::length,           // Transform _1
            age -> age + " years"     // Transform _2
        );
        // (5, "25 years")
    }
}

public class TupleAntiPatterns {

    public Tuple3<String, String, Integer> getUserData(String userId) {
        // What are these values? Hard to tell!
        return Tuple.of("Alice", "alice@example.com", 25);

     *   ✓ Returning 2-3 values temporarily

### Functional Error Handling

import io.vavr.control.Try;

public class VavrTryExamples {
    
    // ✅ Try: Wrap operations that throw
    public Try<User> findUserSafe(String userId) {
        return Try.of(() -> userRepository.findById(userId));
        // Success(user) or Failure(exception)
    }
    
    // ✅ Chaining Try operations
    public Try<String> getUserEmail(String userId) {
        return Try.of(() -> userRepository.findById(userId))

        return Try.of(() -> userRepository.findById(userId))
            .recover(UserNotFoundException.class, User.anonymous())
            .get();
    }
    
    // ✅ orElse: Alternative Try on failure
    public Try<User> getUserWithAlternative(String userId) {
        return Try.of(() -> primaryDatabase.findById(userId))
            .orElse(Try.of(() -> fallbackDatabase.findById(userId)));

import io.vavr.control.Either;

public class VavrEitherExamples {
    
    // ✅ Either: Left = Error, Right = Success
    public Either<String, User> createUser(UserRequest request) {
        if (request.getName() == null) {
            return Either.left("Name is required");
        }
        if (request.getAge() < 18) {
            return Either.left("Must be 18 or older");
        }
        
        User user = new User(request.getName(), request.getAge());
        return Either.right(user);
    }
    
    // ✅ Chaining Either operations
    public Either<String, String> processUser(UserRequest request) {
        return createUser(request)                    // Either<String, User>
            .map(User::getName)                       // Either<String, String>
            .map(String::toUpperCase);                // Either<String, String>
    }
    
    // ✅ flatMap: Chain operations that return Either
    public Either<String, Order> createOrder(String userId) {
        return findUser(userId)                       // Either<String, User>
            .flatMap(user -> validateUser(user))      // Either<String, User>
            .flatMap(user -> createOrderForUser(user)); // Either<String, Order>
    }
    
    private Either<String, User> findUser(String userId) {
        return Either.right(new User("Alice", 25));
    }
    
    private Either<String, User> validateUser(User user) {
        return user.isActive() 
            ? Either.right(user) 
            : Either.left("User is inactive");
    }
    
    private Either<String, Order> createOrderForUser(User user) {
        return Either.right(new Order("order_123"));
    }
}

    private final LoadingCache<String, User> userCache = CacheBuilder.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(Duration.ofMinutes(10))
        .build(new CacheLoader<String, User>() {
            @Override
            public User load(String userId) throws Exception {
                System.out.println("Loading user: " + userId);
                return userRepository.findById(userId);
            }
        });
    
    public User getUser(String userId) {
        try {
            return userCache.get(userId);
            // First call: loads from repository
            // Subsequent calls: returns cached value
            // After 10 minutes: reloads
        } catch (ExecutionException e) {
            throw new RuntimeException(e);
        }
    }
    
    // ✅ Cache with statistics
    private final LoadingCache<String, String> dataCache = CacheBuilder.newBuilder()
        .maximumSize(100)
        .recordStats()  // Enable statistics
        .build(new CacheLoader<String, String>() {
            @Override
            public String load(String key) {
                return fetchData(key);
            }
        });
    
    public void printCacheStats() {
        System.out.println(dataCache.stats());
        // Hit rate, miss rate, eviction count, etc.
    }
    
    private String fetchData(String key) {
        return "data";
    }
}
