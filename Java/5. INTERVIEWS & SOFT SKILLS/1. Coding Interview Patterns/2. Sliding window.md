# Sliding Window Technique

## Overview

The sliding window technique maintains a contiguous subarray or substring "window" over a data structure, expanding and contracting it to find an optimal subset satisfying a constraint. It converts O(n²) or O(n³) brute-force solutions — checking every possible subarray — into O(n) by reusing computation from the previous window instead of restarting from scratch.

**The core insight:**

```
Brute force subarray search:
  for every start i:         O(n)
    for every end j ≥ i:     O(n)
      compute property       O(n) or O(1)
  Total: O(n²) or O(n³)

Sliding window:
  right pointer expands window — adds one element
  left pointer shrinks window  — removes one element
  Each element enters and exits the window exactly once → O(n) total

This works when:
  The problem asks for a subarray/substring satisfying a constraint
  The property being tracked can be updated incrementally (add/remove one element)
  The window validity has monotonic structure:
    larger window: more likely to satisfy (find minimum window → shrink when valid)
    larger window: less likely to satisfy (find maximum window → shrink when invalid)
```

**Fixed vs variable window:**

```
Fixed window (size = k):
  Window size never changes — slide by dropping leftmost, adding rightmost
  right − left + 1 = k always
  Use: "find best subarray of exactly size k"

Variable window (size changes):
  Expand right to satisfy constraint; shrink left to optimise
  OR: shrink left to restore constraint; expand right to grow
  Use: "find smallest window satisfying X" or "find largest window satisfying X"
```

## 1. Fixed Window

### 1.1 Maximum sum subarray of size k

```java
// Find the maximum sum of any contiguous subarray of exactly size k.
// Time: O(n)  Space: O(1)
//
// Brute force: every starting position × k elements = O(nk)
// Sliding window: maintain running sum, slide by one element per step = O(n)
public int maxSumSubarray(int[] nums, int k) {
    if (nums.length < k) throw new IllegalArgumentException("Array shorter than k");

    // Build first window
    int windowSum = 0;
    for (int i = 0; i < k; i++) windowSum += nums[i];

    int maxSum = windowSum;

    // Slide: add rightmost element, drop leftmost element
    for (int right = k; right < nums.length; right++) {
        windowSum += nums[right];          // expand right edge
        windowSum -= nums[right - k];      // shrink left edge (element leaving window)
        maxSum = Math.max(maxSum, windowSum);
    }
    return maxSum;
}
// [2, 1, 5, 1, 3, 2], k=3:
//   First window [2,1,5]: sum=8
//   Slide: +1-2=7  [1,5,1]
//   Slide: +3-1=9  [5,1,3]  ← max
//   Slide: +2-5=6  [1,3,2]
//   Return 9 ✓
```

### 1.3 Maximum of all subarrays of size k — LeetCode 239

```java
// Each window's maximum via monotonic deque — O(n) time, O(k) space.
// Deque stores INDICES in decreasing order of their values.
// Front of deque = index of current window maximum.
public int[] maxSlidingWindow(int[] nums, int k) {
    int[] result = new int[nums.length - k + 1];
    Deque<Integer> deque = new ArrayDeque<>(); // stores indices

    for (int right = 0; right < nums.length; right++) {
        // Remove indices outside current window from front
        while (!deque.isEmpty() && deque.peekFirst() < right - k + 1)
            deque.pollFirst();

        // Maintain decreasing order: remove all smaller elements from back
        // (they can never be the maximum while nums[right] is in the window)
        while (!deque.isEmpty() && nums[deque.peekLast()] < nums[right])
            deque.pollLast();

        deque.offerLast(right);

        // Window is full — record maximum (front of deque)
        if (right >= k - 1)
            result[right - k + 1] = nums[deque.peekFirst()];
    }
    return result;
}
// [1,3,-1,-3,5,3,6,7], k=3:
//   right=0: deque=[0(1)]
//   right=1: 3>1 → pop 0; deque=[1(3)]
//   right=2: -1<3; deque=[1(3),2(-1)]; window full → result[0]=nums[1]=3
//   right=3: -3<-1; deque=[1(3),2(-1),3(-3)]; result[1]=nums[1]=3
//   right=4: 5>all → pop all; deque=[4(5)]; result[2]=5
//   ...result=[3,3,5,5,6,7] ✓
```

### 1.4 Find all anagrams in a string — LeetCode 438

```java
// Fixed window of size p.length(). Window is valid iff char frequencies match.
// Time: O(n)  Space: O(1) — alphabet size 26
public List<Integer> findAnagrams(String s, String p) {
    List<Integer> result = new ArrayList<>();
    if (s.length() < p.length()) return result;

    int[] pCount = new int[26];
    int[] wCount = new int[26];

    for (char c : p.toCharArray()) pCount[c - 'a']++;
    int k = p.length();

    // Build first window
    for (int i = 0; i < k; i++) wCount[s.charAt(i) - 'a']++;

    if (Arrays.equals(pCount, wCount)) result.add(0);

    for (int right = k; right < s.length(); right++) {
        // Slide: add right character, remove leftmost character
        wCount[s.charAt(right) - 'a']++;
        wCount[s.charAt(right - k) - 'a']--;

        if (Arrays.equals(pCount, wCount))
            result.add(right - k + 1);
    }
    return result;
}

// Optimised: track "matches" count — avoid O(26) comparison each step
public List<Integer> findAnagramsOpt(String s, String p) {
    List<Integer> result = new ArrayList<>();
    int[] count = new int[26];  // pCount - wCount; 0 means matched
    for (char c : p.toCharArray()) count[c - 'a']++;

    int required = 0;
    for (int c : count) if (c != 0) required++; // number of distinct chars to match

    int matched = 0;
    int k = p.length();

    for (int right = 0; right < s.length(); right++) {
        // Add right character
        int rc = s.charAt(right) - 'a';
        count[rc]--;
        if (count[rc] == 0) matched++;   // this char now fully matched

        // Remove left character (when window exceeds size k)
        if (right >= k) {
            int lc = s.charAt(right - k) - 'a';
            if (count[lc] == 0) matched--; // removing a matched char
            count[lc]++;
        }

        if (matched == required) result.add(right - k + 1);
    }
    return result;
}
```

## 2. Variable Window — Expand to Satisfy, Shrink to Optimise

### 2.1 Longest substring without repeating characters — LeetCode 3

```java
// Expand right always. When duplicate found, shrink left past previous occurrence.
// Time: O(n)  Space: O(1)
public int lengthOfLongestSubstring(String s) {
    int[] lastSeen = new int[128]; // char → most recent index (-1 = unseen)
    Arrays.fill(lastSeen, -1);

    int left = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        int c = s.charAt(right);

        if (lastSeen[c] >= left) {
            // Duplicate inside window → jump left past previous occurrence
            left = lastSeen[c] + 1;
        }
        lastSeen[c] = right;
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
// "abcabcbb":
//   right=0(a): lastSeen=-1; left=0; len=1
//   right=1(b): lastSeen=-1; left=0; len=2
//   right=2(c): lastSeen=-1; left=0; len=3
//   right=3(a): lastSeen[a]=0 ≥ left=0 → left=1; len=3
//   right=4(b): lastSeen[b]=1 ≥ left=1 → left=2; len=3
//   right=5(c): lastSeen[c]=2 ≥ left=2 → left=3; len=3
//   right=6(b): lastSeen[b]=4 ≥ left=3 → left=5; len=2
//   right=7(b): lastSeen[b]=6 ≥ left=5 → left=7; len=1
//   Return 3 ✓
```

### 2.2 Minimum window substring — LeetCode 76

```java
// Find smallest window in s containing all characters of t.
// Expand right until all chars covered; shrink left to minimise; repeat.
// Time: O(n + m)  Space: O(m) where m = |t|
public String minWindow(String s, String t) {
    if (s.isEmpty() || t.isEmpty()) return "";

    int[] need = new int[128];    // how many of each char still needed
    for (char c : t.toCharArray()) need[c]++;

    int required = 0;
    for (int n : need) if (n > 0) required++; // distinct chars to satisfy

    int left = 0, matched = 0;
    int minLen = Integer.MAX_VALUE, minStart = 0;

    for (int right = 0; right < s.length(); right++) {
        // Expand: bring right character into window
        char rc = s.charAt(right);
        need[rc]--;
        if (need[rc] == 0) matched++; // just satisfied this character's requirement

        // Shrink: while all requirements met, try to minimise window
        while (matched == required) {
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                minStart = left;
            }
            char lc = s.charAt(left);
            need[lc]++;
            if (need[lc] > 0) matched--; // removing a char that was exactly satisfied
            left++;
        }
    }
    return minLen == Integer.MAX_VALUE ? "" : s.substring(minStart, minStart + minLen);
}
// s="ADOBECODEBANC", t="ABC":
//   Expand right until A,B,C all covered → window "ADOBEC" (left=0,right=5)
//   Shrink: remove A → "DOBEC" but A missing → stop shrinking
//   Continue expanding right → "DOBECODEBA" has A,B,C
//   Shrink: remove D→ "OBECODEBA"; remove O→ "BECODEBA"... until "BANC" ✓
```

### 2.3 Longest substring with at most k distinct characters — LeetCode 340

```java
// Expand right always. When distinct count exceeds k, shrink left.
// Time: O(n)  Space: O(k)
public int lengthOfLongestSubstringKDistinct(String s, int k) {
    Map<Character, Integer> freq = new HashMap<>(); // char → count in window
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        char rc = s.charAt(right);
        freq.merge(rc, 1, Integer::sum); // add to window

        // Too many distinct chars — shrink until ≤ k
        while (freq.size() > k) {
            char lc = s.charAt(left);
            freq.merge(lc, -1, Integer::sum);
            if (freq.get(lc) == 0) freq.remove(lc); // fully removed from window
            left++;
        }
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
// "eceba", k=2:
//   right=0(e): {e:1}, len=1
//   right=1(c): {e:1,c:1}, len=2
//   right=2(e): {e:2,c:1}, len=3
//   right=3(b): {e:2,c:1,b:1} size=3>2 → shrink:
//     left=0(e): {e:1,c:1,b:1} → size=3>2 → shrink:
//     left=1(c): {e:1,b:1} → size=2=k → stop; len=max(3,2)=3
//   right=4(a): {e:1,b:1,a:1} size=3>2 → shrink:
//     left=2(e): {b:1,a:1} → size=2 → stop; len=max(3,2)=3
//   Return 3 ✓
```

### 2.4 Longest repeating character replacement — LeetCode 424

```java
// Replace at most k characters to make longest substring of one repeated char.
// Key insight: window is valid if (windowSize - maxFreq) ≤ k
//   windowSize - maxFreq = minimum replacements needed
// Time: O(n)  Space: O(1)
public int characterReplacement(String s, int k) {
    int[] freq = new int[26];
    int left = 0, maxFreq = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        freq[s.charAt(right) - 'A']++;
        // Track the maximum frequency of any character seen so far in window
        // (No need to recompute on shrink — maxFreq only grows or stays)
        maxFreq = Math.max(maxFreq, freq[s.charAt(right) - 'A']);

        // Window invalid: too many replacements needed → shrink by 1
        if ((right - left + 1) - maxFreq > k) {
            freq[s.charAt(left) - 'A']--;
            left++;
        }
        // Window always grows by 1 (or stays same) → right - left + 1 is maxLen
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
// "AABABBA", k=1:
//   right=0(A): freq[A]=1, maxFreq=1, size=1, 1-1=0≤1 → len=1
//   right=1(A): freq[A]=2, maxFreq=2, size=2, 2-2=0≤1 → len=2
//   right=2(B): freq[B]=1, maxFreq=2, size=3, 3-2=1≤1 → len=3
//   right=3(A): freq[A]=3, maxFreq=3, size=4, 4-3=1≤1 → len=4
//   right=4(B): freq[B]=2, maxFreq=3, size=5, 5-3=2>1 → shrink: left=1, freq[A]=2
//   right=5(B): freq[B]=3, maxFreq=3, size=5, 5-3=2>1 → shrink: left=2, freq[A]=1
//   right=6(A): freq[A]=2, maxFreq=3, size=5, 5-3=2>1 → shrink: left=3, freq[B]=2
//             : size=4, 4-3=1≤1 → len=max(4,4)=4
//   Return 4 ("AABA" with 1 replacement → "AAAA") ✓
```

## 3. Variable Window — Shrink to Satisfy Constraint

### 3.1 Minimum size subarray sum — LeetCode 209

```java
// Shortest contiguous subarray with sum ≥ target.
// Expand right always; shrink left while sum ≥ target (record minimum, then shrink).
// Time: O(n)  Space: O(1)
public int minSubArrayLen(int target, int[] nums) {
    int left = 0, sum = 0, minLen = Integer.MAX_VALUE;

    for (int right = 0; right < nums.length; right++) {
        sum += nums[right]; // expand

        while (sum >= target) {
            minLen = Math.min(minLen, right - left + 1);
            sum -= nums[left++]; // shrink: record BEFORE shrinking
        }
    }
    return minLen == Integer.MAX_VALUE ? 0 : minLen;
}
// [2,3,1,2,4,3], target=7:
//   right=0: sum=2 < 7
//   right=1: sum=5 < 7
//   right=2: sum=6 < 7
//   right=3: sum=8 ≥ 7 → len=4; shrink: sum=6 < 7 → stop
//   right=4: sum=10 ≥ 7 → len=min(4,3)=3; shrink: sum=7 ≥ 7 → len=min(3,2)=2; shrink: sum=4 < 7
//   right=5: sum=7 ≥ 7 → len=min(2,2)=2; shrink: sum=3 < 7
//   Return 2 ✓
```

### 3.2 Number of subarrays with product less than k — LeetCode 713

```java
// Count subarrays where product of all elements < k.
// Key insight: for each right, count valid subarrays ending at right.
// When window is valid (product < k), subarrays ending at right:
//   [right], [left..right], [left+1..right], ... = (right - left + 1) subarrays
// Time: O(n)  Space: O(1)
public int numSubarrayProductLessThanK(int[] nums, int k) {
    if (k <= 1) return 0;
    int left = 0, product = 1, count = 0;

    for (int right = 0; right < nums.length; right++) {
        product *= nums[right];

        while (product >= k) {
            product /= nums[left++]; // shrink until valid
        }
        // All subarrays ending at right with start in [left..right] are valid
        count += right - left + 1;
    }
    return count;
}
// [10,5,2,6], k=100:
//   right=0: product=10; count += 0-0+1 = 1  (subarrays: [10])
//   right=1: product=50; count += 1-0+1 = 2  (subarrays: [5],[10,5])
//   right=2: product=100 ≥ 100 → shrink: product=10, left=1
//            count += 2-1+1 = 2              (subarrays: [2],[5,2])
//   right=3: product=60; count += 3-1+1 = 3  (subarrays: [6],[2,6],[5,2,6])
//   Total = 1+2+2+3 = 8 ✓
```

    int left = 0, zeros = 0, maxLen = 0;

    for (int right = 0; right < nums.length; right++) {
        if (nums[right] == 0) zeros++;

        while (zeros > 1) {
            if (nums[left] == 0) zeros--;
            left++;
        }

    }

## 5. Optimisation Techniques

### 5.1 The "never shrink" trick — monotonically growing window

```java
// For maximum-length problems, the window never needs to SHRINK smaller than maxLen.
// Once we've found a window of size maxLen, we only care about finding size maxLen+1.
// Instead of shrinking left until valid, just slide the window forward by 1.
// This keeps time O(n) but avoids the inner while loop.

// characterReplacement already uses this: notice we use if() not while() to shrink
// The window size only grows — once maxLen is found, window only shifts right

// General template for "find maximum window length":
public int maxWindowOptimised(int[] nums, int k) {
    // ... window state ...
    int left = 0;

    for (int right = 0; right < nums.length; right++) {
        // expand: add nums[right] to window state

        if (/* window invalid */) {
            // shrink by exactly 1 (not while loop)
            // remove nums[left] from window state
            left++;
        }
        // right - left + 1 is the current (non-decreasing) window size
    }
    return nums.length - left; // final window size = right - left + 1
}
// Why correct: we never need a window smaller than the best found so far.
// Invalid windows slide forward without recording, valid windows record via size.
```

### 5.2 Prefix sum + HashMap for subarray sum problems

```java
// For "number of subarrays with sum = k" — sliding window cannot handle negative numbers.
// Use prefix sum: prefixSum[j] - prefixSum[i] = sum(i+1..j)
// Count of subarrays ending at j with sum k = count of indices i where prefixSum[i] = prefixSum[j] - k
// Time: O(n)  Space: O(n)

// Number of subarrays with sum = k — LeetCode 560
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1); // empty prefix: sum 0 seen once
    int prefixSum = 0, count = 0;

    for (int num : nums) {
        prefixSum += num;
        // How many previous prefixes had sum = prefixSum - k?
        count += prefixCount.getOrDefault(prefixSum - k, 0);
        prefixCount.merge(prefixSum, 1, Integer::sum);
    }
    return count;
}
// [1,1,1], k=2:
//   prefixCount={0:1}
//   num=1: prefix=1; count += prefixCount.get(1-2=-1)=0; {0:1,1:1}
//   num=1: prefix=2; count += prefixCount.get(2-2=0)=1→count=1; {0:1,1:1,2:1}
//   num=1: prefix=3; count += prefixCount.get(3-2=1)=1→count=2; {0:1,1:1,2:1,3:1}
//   Return 2 ✓ (subarrays [1,1] at indices 0-1 and 1-2)

// Longest subarray with sum = k (when negatives present):
public int maxSubarrayLenSumK(int[] nums, int k) {
    Map<Integer, Integer> firstSeen = new HashMap<>(); // prefix sum → first index
    firstSeen.put(0, -1);
    int prefixSum = 0, maxLen = 0;

    for (int i = 0; i < nums.length; i++) {
        prefixSum += nums[i];
        int target = prefixSum - k;
        if (firstSeen.containsKey(target)) {
            maxLen = Math.max(maxLen, i - firstSeen.get(target));
        }
        // Only store FIRST occurrence (we want maximum length → earliest start)
        firstSeen.putIfAbsent(prefixSum, i);
    }
    return maxLen;
}
```

### 5.3 Monotonic deque for range queries within window

```java
// When window needs minimum/maximum efficiently: use monotonic deque.
// Deque head = current window min or max (for monotone increasing or decreasing deque)
// Each element added/removed exactly once → O(n) total

// Sliding window minimum (monotone increasing deque — front = minimum)
public int[] slidingWindowMin(int[] nums, int k) {
    int[] result = new int[nums.length - k + 1];
    Deque<Integer> deque = new ArrayDeque<>(); // stores indices, increasing by value

    for (int right = 0; right < nums.length; right++) {
        // Remove elements outside window
        while (!deque.isEmpty() && deque.peekFirst() < right - k + 1)
            deque.pollFirst();

        // Remove elements larger than current (can never be minimum while nums[right] in window)
        while (!deque.isEmpty() && nums[deque.peekLast()] > nums[right])
            deque.pollLast();

        deque.offerLast(right);

        if (right >= k - 1)
            result[right - k + 1] = nums[deque.peekFirst()];
    }
    return result;
}

// Shortest subarray with sum ≥ k (k can be negative) — LeetCode 862
// Requires prefix sum + monotonic deque (not plain sliding window — negatives!)
public int shortestSubarray(int[] nums, int k) {
    int n = nums.length;
    long[] prefix = new long[n + 1];
    for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] + nums[i];

    int minLen = Integer.MAX_VALUE;
    Deque<Integer> deque = new ArrayDeque<>(); // monotone increasing prefix sums

    for (int right = 0; right <= n; right++) {
        // Shrink from front: prefix[right] - prefix[front] ≥ k → record and remove
        while (!deque.isEmpty() && prefix[right] - prefix[deque.peekFirst()] >= k) {
            minLen = Math.min(minLen, right - deque.pollFirst());
        }
        // Maintain increasing order: remove back elements ≥ current prefix
        while (!deque.isEmpty() && prefix[deque.peekLast()] >= prefix[right])
            deque.pollLast();

        deque.offerLast(right);
    }
    return minLen == Integer.MAX_VALUE ? -1 : minLen;
}
```

### 5.4 When sliding window fails — and what to use instead

```java
/*
Sliding window requires:
  (a) Non-negative numbers for sum problems (negative numbers break monotonicity)
  (b) Contiguous subarray (not subsequence)
  (c) A single constraint that can be maintained incrementally

When it fails:
  Negative numbers in sum problems → prefix sum + HashMap (subarraySum)
  Non-contiguous (subsequence) problems → DP or two pointers
  Multiple constraints simultaneously → harder; may need nested approach
  "Exactly k" constraint → use "at most k" - "at most k-1" trick (see below)
*/

// "Exactly k" constraint trick: count(exactly k) = count(at most k) - count(at most k-1)
// Because sliding window handles "at most" naturally but not "exactly"

// Number of subarrays with exactly k distinct integers — LeetCode 992
public int subarraysWithKDistinct(int[] nums, int k) {
    return atMostK(nums, k) - atMostK(nums, k - 1);
}

private int atMostK(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    int left = 0, count = 0;

    for (int right = 0; right < nums.length; right++) {
        freq.merge(nums[right], 1, Integer::sum);

        while (freq.size() > k) {
            freq.merge(nums[left], -1, Integer::sum);
            if (freq.get(nums[left]) == 0) freq.remove(nums[left]);
            left++;
        }
        count += right - left + 1; // all subarrays ending at right with ≤ k distinct
    }
    return count;
}
// For each right, (right - left + 1) counts subarrays [left..right], [left+1..right], ..., [right..right]
// Subarrays with exactly k = (subarrays with ≤ k) − (subarrays with ≤ k−1)
```

## 8. Interview Questions

**Q: How do you decide whether to use a fixed or variable sliding window?**

A: Fixed window is the choice when the problem specifies a concrete window size — "find the maximum sum subarray of size k" or "find all anagrams of a pattern." The window never changes size; it slides by adding the new rightmost element and removing the oldest leftmost element. Variable window applies when the problem asks to find the longest or shortest subarray satisfying some condition, because the optimal answer has an unknown size. For maximum-length problems, expand the right pointer on every step and shrink the left pointer when the window violates the constraint — this keeps the window as large as possible. For minimum-length problems, expand right until the constraint is satisfied, then shrink left as much as possible while it remains satisfied, recording the minimum each time before shrinking further.

---

**Q: Why can't you use a sliding window for "number of subarrays with sum = k" on an array with negative numbers?**

A: The sliding window relies on monotonicity — when you add an element to the window, the sum predictably increases (for positive numbers), and when you remove from the left, it decreases. This lets you make a definitive decision: if the sum is too large, shrink; if too small, expand. With negative numbers, adding an element can decrease the sum and removing one can increase it, so there's no clear direction to move either pointer. The standard solution is prefix sums with a HashMap: the sum of subarray `[i+1..j]` equals `prefixSum[j] - prefixSum[i]`. For every position `j`, count how many previous prefix sums equal `prefixSum[j] - k` — those are the valid subarrays ending at `j`. This is O(n) time and O(n) space, handling both positive and negative numbers correctly.

---

**Q: Explain the "never shrink" optimisation in the character replacement problem.**

A: In problems asking for the maximum window length, the answer can only improve when we find a window strictly longer than the current best. This means we never need to contract the window to smaller than `maxLen` — if the window becomes invalid, instead of shrinking left until it's valid again (which might reduce it below `maxLen`), we just slide it forward by one: increment both left and right. The window size either stays the same (on invalid step) or grows (on valid step) — it never shrinks. This turns the inner `while` loop into an `if` statement, making the logic simpler, but more importantly it means the window monotonically grows, so the final window size directly gives the answer without needing to track `maxLen` separately. The tradeoff: this only works for maximum-length problems, not minimum-length ones where we need to aggressively shrink the window.

---

**Q: How does the "at most k minus at most k-1" trick work for exact-count problems?**

A: Sliding windows handle "at most k" naturally — maintain a window with at most k distinct values (or k zeros, etc.) by shrinking when the count exceeds k. But "exactly k" is harder because adding one element might jump from below k to above k, and there's no obvious shrink condition. The trick exploits a counting identity: the number of subarrays with exactly k distinct integers equals the number with at most k distinct minus the number with at most k-1 distinct. The proof: any subarray counted by "at most k" either has exactly k distinct or fewer than k distinct; subtracting "at most k-1" removes all subarrays with fewer than k, leaving exactly those with k. Both "at most" counts are solved independently with standard sliding windows, and the difference gives the exact count. The same identity applies to other constraints: exactly k zeros, exactly k replacements, etc.

---

**Q: When should you use a monotonic deque inside a sliding window?**

A: Use a monotonic deque when the problem needs the maximum or minimum of the current window efficiently — not just the sum or count, but the extremum. A frequency map or running sum can be updated in O(1) when the window slides, but naively finding the new maximum after removing the old maximum requires scanning the entire window, giving O(nk) overall. The monotonic deque maintains elements in sorted order (decreasing for maximum, increasing for minimum) by removing from the back any element that becomes dominated when a new element is added. When the window's left boundary advances past the front element, it's removed from the front. Both operations are amortised O(1) — each element is added and removed exactly once — giving O(n) total. The deque stores indices (not values) so boundary checks are straightforward. This pattern also extends beyond fixed windows: the monotonic deque on prefix sums solves shortest subarray with sum ≥ k even with negative numbers.

## 9. Key Takeaways

- **Fixed window**: size is given; slide by `windowSum += nums[right] - nums[right - k]`; use for anagrams, averages, fixed-k max/min
- **Variable window, maximise length**: expand right always; shrink with `while` when window becomes invalid; answer is `right - left + 1`
- **Variable window, minimise length**: expand right until valid; shrink with `while` while still valid; record before each shrink
- **Never-shrink optimisation**: for maximum-length problems, replace inner `while` with `if` — window only grows or slides, never contracts; final window size is the answer
- **Negative numbers break sum-based sliding windows** — use prefix sum + HashMap instead; `subarraySum` and `maxSubarrayLen` with sum = k both use this
- **Exactly k = at most k − at most k-1**: sliding window handles "at most" naturally; "exactly" follows from subtraction
- **Monotonic deque for window min/max**: O(n) overall; decreasing deque for maximum, increasing for minimum; store indices for boundary checking
- **Count formula for subarray counting**: when window is valid at `right`, it contributes `right - left + 1` subarrays (all starting points from `left` to `right`)
- **Recognition signals**: "contiguous subarray/substring", "at most k", "longest/shortest satisfying X", "k distinct", "no repeating" → all point to sliding window
