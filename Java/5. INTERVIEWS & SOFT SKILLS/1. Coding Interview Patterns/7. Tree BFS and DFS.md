# Tree BFS and DFS

## Overview

Tree traversal problems split cleanly into two families. BFS (Breadth-First Search) processes nodes level by level using a queue — natural for problems involving levels, distances, or layer-by-layer comparisons. DFS (Depth-First Search) processes nodes along a path using recursion or an explicit stack — natural for path problems, subtree comparisons, and the three classic orderings (inorder, preorder, postorder).

**Choosing between BFS and DFS:**

```
Use BFS when:                           Use DFS when:
  "level order"                           "path from root to leaf"
  "minimum depth" / "closest node"        "maximum depth" / "all paths"
  "right side view" / "left side view"    "validate BST"
  "average at each level"                 "lowest common ancestor"
  "connect next pointers"                 "subtree sum / count"
  Layer-by-layer processing               Inorder (sorted BST traversal)
  Comparing nodes across same level       Structural subtree problems

BFS uses O(w) space where w = max width (can be O(n) at bottom level of complete tree)
DFS uses O(h) space where h = height (O(log n) balanced, O(n) worst case / skewed)
For very wide trees: DFS uses less space
For very deep/skewed trees: BFS uses less space
```

## 1. Level Order Traversal (BFS) — LeetCode 102

### 1.1 Core BFS template

```java
// Return nodes grouped by level: [[3],[9,20],[15,7]]
// Time: O(n)  Space: O(w) — w = max width of tree
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size(); // snapshot: all nodes at current level
        List<Integer> level = new ArrayList<>();

        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);

            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
```

**Trace on:**

```
      3
     / \
    9  20
       / \
      15   7

queue=[3], levelSize=1
  poll 3 → level=[3], enqueue 9,20
  result=[[3]]

queue=[9,20], levelSize=2
  poll 9 → level=[9], no children
  poll 20 → level=[9,20], enqueue 15,7
  result=[[3],[9,20]]

queue=[15,7], levelSize=2
  poll 15 → level=[15], no children
  poll 7  → level=[15,7], no children
  result=[[3],[9,20],[15,7]] ✓
```

**The `levelSize` snapshot is the key insight:**

```
Without levelSize: queue grows as we enqueue children.
  We'd have no way to know which nodes belong to the current level vs the next.

With levelSize = queue.size() BEFORE the inner loop:
  We process exactly the nodes that were in the queue at the START of this level.
  New children go to the back of the queue, safely outside the levelSize boundary.
  This cleanly separates level k from level k+1.
```

```java
// [[15,7],[9,20],[3]] — leaves to root
public List<List<Integer>> levelOrderBottom(TreeNode root) {
    List<List<Integer>> result = new LinkedList<>(); // LinkedList for O(1) addFirst
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();

    }
    return result;
}
```

### 1.3 Average of levels — LeetCode 637

```java
public List<Double> averageOfLevels(TreeNode root) {
    List<Double> averages = new ArrayList<>();
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size();
        double levelSum = 0;
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            levelSum += node.val;
            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        averages.add(levelSum / levelSize);
    }
    return averages;
}
```

### 1.4 Minimum depth — LeetCode 111

```java
// BFS naturally finds the minimum depth: first leaf encountered is the shallowest.
// DFS would need to traverse the whole tree — BFS stops at first leaf.
public int minDepth(TreeNode root) {
    if (root == null) return 0;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    int depth = 0;

    while (!queue.isEmpty()) {
        depth++;
        int levelSize = queue.size();
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            if (node.left == null && node.right == null) return depth; // first leaf
            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
    }
    return depth;
}
// BFS advantage: can return early at first leaf.
// DFS would explore entire tree even if answer is at depth 2.
```

## 2. Zigzag Level Order Traversal — LeetCode 103

```java
// Even levels: left-to-right. Odd levels: right-to-left.
// [[3],[20,9],[15,7]]
// Time: O(n)  Space: O(w)
public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    boolean leftToRight = true;

    while (!queue.isEmpty()) {
        int levelSize = queue.size();
        LinkedList<Integer> level = new LinkedList<>(); // LinkedList supports addFirst

        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();

            if (leftToRight) level.addLast(node.val);
            else             level.addFirst(node.val); // reverse direction

            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }

        result.add(level);
        leftToRight = !leftToRight; // flip direction each level
    }
    return result;
}
```

**Trace on the same tree:**

```
      3
     / \
    9  20
       / \
      15   7

Level 0 (leftToRight=true):  poll 3  → addLast(3)    → [3]
Level 1 (leftToRight=false): poll 9  → addFirst(9)   → [9]
                              poll 20 → addFirst(20)  → [20,9]
Level 2 (leftToRight=true):  poll 15 → addLast(15)   → [15]
                              poll 7  → addLast(7)    → [15,7]

Result: [[3],[20,9],[15,7]] ✓
```

**Alternative: build level normally, reverse odd levels after:**

```java
// Slightly less elegant but easier to reason about
for (int i = 0; i < levelSize; i++) {
    TreeNode node = queue.poll();
    level.add(node.val);
    // ... enqueue children
}
if (!leftToRight) Collections.reverse(level);
result.add(level);
leftToRight = !leftToRight;
// Costs O(w) extra per reversed level — usually acceptable
```

## 3. Right Side View — LeetCode 199

```java
// Return the value visible from the right side at each level (rightmost node per level).
// Time: O(n)  Space: O(w)
public List<Integer> rightSideView(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size();
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            if (i == levelSize - 1) result.add(node.val); // last node in level
            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
    }
    return result;
}

// DFS alternative: right child first, record first-seen node at each depth
public List<Integer> rightSideViewDFS(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    dfsRight(root, 0, result);
    return result;
}

private void dfsRight(TreeNode node, int depth, List<Integer> result) {
    if (node == null) return;
    if (depth == result.size()) result.add(node.val); // first node at this depth
    dfsRight(node.right, depth + 1, result); // visit right FIRST
    dfsRight(node.left,  depth + 1, result);
}
```

## 4. Connect Next Right Pointers — LeetCode 116 / 117

```java
// Connect each node's "next" pointer to its right neighbour at the same level.
// Perfect binary tree (116): elegant O(1) space solution using already-connected level
// General tree (117): BFS approach works for both
// Time: O(n)  Space: O(1) for perfect tree

// LeetCode 116 — Perfect binary tree, O(1) space
public Node connect116(Node root) {
    if (root == null) return null;
    Node leftmost = root; // start of current level

    while (leftmost.left != null) { // while not at leaf level
        Node curr = leftmost;
        while (curr != null) {
            curr.left.next = curr.right;                             // connect children of same parent
            if (curr.next != null) curr.right.next = curr.next.left; // connect across parents
            curr = curr.next;                                         // move right using already-set next
        }
        leftmost = leftmost.left; // drop to next level
    }
    return root;
}

// LeetCode 117 — General tree (not necessarily perfect)
public Node connect117(Node root) {
    if (root == null) return null;
    Queue<Node> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size();
        for (int i = 0; i < levelSize; i++) {
            Node node = queue.poll();
            // Connect to next node in same level (not last node in level)
            if (i < levelSize - 1) node.next = queue.peek();
            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
    }
    return root;
}
```

## 5. DFS Traversals — Inorder, Preorder, Postorder

### 5.1 The three orderings

```
      1
     / \
    2   3
   / \
  4   5

Inorder   (Left → Root → Right): 4, 2, 5, 1, 3  ← sorted order for BST
Preorder  (Root → Left → Right): 1, 2, 4, 5, 3  ← serialise / clone tree
Postorder (Left → Right → Root): 4, 5, 2, 3, 1  ← delete tree / compute subtree values

Memory aid:
  pre-order:  root comes PRE (before) children
  in-order:   root comes IN between children
  post-order: root comes POST (after) children
```

### 5.2 Recursive implementations

```java
// Inorder — LeetCode 94
public List<Integer> inorderTraversal(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    inorder(root, result);
    return result;
}
private void inorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    inorder(node.left, result);   // Left
    result.add(node.val);          // Root
    inorder(node.right, result);  // Right
}

// Preorder — LeetCode 144
private void preorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    result.add(node.val);           // Root
    preorder(node.left, result);   // Left
    preorder(node.right, result);  // Right
}

// Postorder — LeetCode 145
private void postorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    postorder(node.left, result);  // Left
    postorder(node.right, result); // Right
    result.add(node.val);          // Root
}
```

### 5.3 Iterative inorder — the interview favourite

```java
// Iterative inorder using explicit stack.
// Pattern: go left as deep as possible, process node, then go right.
// Time: O(n)  Space: O(h)
public List<Integer> inorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode curr = root;

    while (curr != null || !stack.isEmpty()) {
        // Go left as deep as possible
        while (curr != null) {
            stack.push(curr);
            curr = curr.left;
        }
        // Process leftmost unprocessed node
        curr = stack.pop();
        result.add(curr.val);
        // Move to right subtree
        curr = curr.right;
    }
    return result;
}
// Tree above: stack=[1,2,4] → pop 4 (no right) → stack=[1,2] → pop 2 → curr=5
// stack=[1,5] → pop 5 (no right) → stack=[1] → pop 1 → curr=3 → stack=[3] → pop 3
// Result: 4,2,5,1,3 ✓
```

### 5.4 Iterative preorder

```java
// Iterative preorder: process root, push right then left (so left processed first).
public List<Integer> preorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);

    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.add(node.val);              // process root
        if (node.right != null) stack.push(node.right); // push right FIRST
        if (node.left  != null) stack.push(node.left);  // push left second (processed first)
    }
    return result;
}
// Push right before left: stack is LIFO, so left gets popped/processed before right
```

### 5.5 Iterative postorder

```java
// Postorder = reverse of (Root → Right → Left) preorder
// Compute modified preorder, then reverse result.
public List<Integer> postorderIterative(TreeNode root) {
    LinkedList<Integer> result = new LinkedList<>(); // addFirst for O(1) prepend
    if (root == null) return result;

    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);

    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.addFirst(node.val);         // prepend (reversal happens here)
        if (node.left  != null) stack.push(node.left);
        if (node.right != null) stack.push(node.right);
    }
    return result;
}
// Why this works:
//   Normal preorder:           Root, Left, Right
//   Push left before right:    Root, Right, Left  (modified preorder)
//   Prepend (addFirst) = reverse: Left, Right, Root = postorder ✓
```

### 5.6 Morris Traversal — O(1) space inorder

```java
// Morris traversal: no stack, no recursion — O(1) space.
// Temporarily threads right pointers from inorder predecessor back to its successor.
// Time: O(n)  Space: O(1)
public List<Integer> morrisInorder(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    TreeNode curr = root;

    while (curr != null) {
        if (curr.left == null) {
            // No left subtree → process curr, move right
            result.add(curr.val);
            curr = curr.right;
        } else {
            // Find inorder predecessor (rightmost node of left subtree)
            TreeNode pred = curr.left;
            while (pred.right != null && pred.right != curr) {
                pred = pred.right;
            }

            if (pred.right == null) {
                // First visit: create thread from predecessor back to curr
                pred.right = curr;
                curr = curr.left;  // descend into left subtree
            } else {
                // Second visit: thread already exists → we've finished left subtree
                pred.right = null; // remove thread (restore tree)
                result.add(curr.val);
                curr = curr.right;
            }
        }
    }
    return result;
}
```

## 6. Path Problems

### 6.1 Maximum path sum — LeetCode 124

```java
// Path can start and end at any node. Find maximum sum.
// Key: at each node, compute max gain going DOWN (either left or right, not both).
//      Check if path THROUGH this node (left + root + right) beats global max.
// Time: O(n)  Space: O(h)
public int maxPathSum(TreeNode root) {
    int[] maxSum = {Integer.MIN_VALUE}; // array to allow mutation in lambda/inner method
    maxGain(root, maxSum);
    return maxSum[0];
}

private int maxGain(TreeNode node, int[] maxSum) {
    if (node == null) return 0;

    // Max gain from left/right subtrees (ignore negative contributions)
    int leftGain  = Math.max(maxGain(node.left,  maxSum), 0);
    int rightGain = Math.max(maxGain(node.right, maxSum), 0);

    // Path through this node: left → node → right
    int pathThroughNode = node.val + leftGain + rightGain;
    maxSum[0] = Math.max(maxSum[0], pathThroughNode);

    // Return max gain going DOWN from this node (can only go one direction)
    return node.val + Math.max(leftGain, rightGain);
}
// Tree: [-10, 9, 20, null, null, 15, 7]
//        -10
//        / \
//       9  20
//         /  \
//        15   7
//
// maxGain(15): left=0, right=0, through=15, return 15
// maxGain(7):  left=0, right=0, through=7,  return 7
// maxGain(20): left=15, right=7, through=42, maxSum=42, return 20+15=35
// maxGain(9):  left=0, right=0, through=9,  return 9
// maxGain(-10): left=9, right=35, through=-10+9+35=34, return -10+35=25
// maxSum = 42 ✓
```

### 6.2 Path sum — LeetCode 112

```java
// Does any root-to-leaf path sum to targetSum?
public boolean hasPathSum(TreeNode root, int targetSum) {
    if (root == null) return false;

    // Leaf node: does remaining sum equal this node's value?
    if (root.left == null && root.right == null) return root.val == targetSum;

    int remaining = targetSum - root.val;
    return hasPathSum(root.left, remaining) || hasPathSum(root.right, remaining);
}
```

### 6.3 Path sum II — find all paths — LeetCode 113

```java
// Find all root-to-leaf paths that sum to targetSum.
// Time: O(n²) — copying path list at each leaf  Space: O(n·h)
public List<List<Integer>> pathSum(TreeNode root, int targetSum) {
    List<List<Integer>> result = new ArrayList<>();
    dfs(root, targetSum, new ArrayList<>(), result);
    return result;
}

private void dfs(TreeNode node, int remaining,
                  List<Integer> path, List<List<Integer>> result) {
    if (node == null) return;

    path.add(node.val);
    remaining -= node.val;

    if (node.left == null && node.right == null && remaining == 0) {
        result.add(new ArrayList<>(path)); // found valid path — copy it
    }

    dfs(node.left,  remaining, path, result);
    dfs(node.right, remaining, path, result);

    path.remove(path.size() - 1); // backtrack: remove current node before returning
}
```

### 6.4 Path sum III — any path, any start/end — LeetCode 437

```java
// Count paths (not necessarily root-to-leaf) with sum = targetSum.
// Paths must go downward (parent → child direction only).
//
// Prefix sum + HashMap: for each node, count how many ancestors have
// prefixSum such that prefixSum[ancestor] = currentPrefixSum - targetSum
// Time: O(n)  Space: O(n)
public int pathSumIII(TreeNode root, int targetSum) {
    Map<Long, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0L, 1); // empty path prefix
    return dfsPrefix(root, 0L, targetSum, prefixCount);
}

private int dfsPrefix(TreeNode node, long currSum, int target,
                       Map<Long, Integer> prefixCount) {
    if (node == null) return 0;

    currSum += node.val;
    // Paths ending at this node with sum = target
    int count = prefixCount.getOrDefault(currSum - target, 0);

    // Record current prefix sum
    prefixCount.merge(currSum, 1, Integer::sum);

    count += dfsPrefix(node.left,  currSum, target, prefixCount);
    count += dfsPrefix(node.right, currSum, target, prefixCount);

    // Backtrack: remove current prefix from map (leaving subtree)
    prefixCount.merge(currSum, -1, Integer::sum);

    return count;
}
// Why backtrack: the prefix map must only contain prefix sums on the
// path from root to current node. When we return to the parent and
// explore the other subtree, this node's contribution must be removed.
```

### 6.5 Binary tree maximum depth — LeetCode 104

```java
// Recursive: max depth = 1 + max(leftDepth, rightDepth)
public int maxDepth(TreeNode root) {
    if (root == null) return 0;
    return 1 + Math.max(maxDepth(root.left), maxDepth(root.right));
}

// Iterative BFS: depth = number of levels
public int maxDepthBFS(TreeNode root) {
    if (root == null) return 0;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    int depth = 0;

    while (!queue.isEmpty()) {
        depth++;
        int size = queue.size();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
    }
    return depth;
}
```

### 6.6 Diameter of binary tree — LeetCode 543

```java
// Diameter = longest path between any two nodes.
// At each node: diameter through it = leftHeight + rightHeight.
// Track global max; return height (not diameter) for parent's computation.
public int diameterOfBinaryTree(TreeNode root) {
    int[] maxDiam = {0};
    height(root, maxDiam);
    return maxDiam[0];
}

private int height(TreeNode node, int[] maxDiam) {
    if (node == null) return 0;
    int left  = height(node.left,  maxDiam);
    int right = height(node.right, maxDiam);
    maxDiam[0] = Math.max(maxDiam[0], left + right); // path through this node
    return 1 + Math.max(left, right);                // height for parent
}
// Same pattern as maxPathSum: track global via array, return local value for parent.
```

## 7. BST-Specific DFS Problems

### 7.1 Validate BST — LeetCode 98

```java
// Validate using min/max bounds passed down the tree.
// Every node must satisfy: min < node.val < max
// Time: O(n)  Space: O(h)
public boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean validate(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false;
    return validate(node.left,  min, node.val) &&   // left must be < node.val
           validate(node.right, node.val, max);     // right must be > node.val
}
// Why Long bounds: int value could equal Integer.MIN_VALUE or MAX_VALUE

// Alternative: inorder traversal → must be strictly increasing
public boolean isValidBSTInorder(TreeNode root) {
    long prev = Long.MIN_VALUE;
    // (use iterative inorder to avoid O(n) space from recursion)
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode curr = root;
    while (curr != null || !stack.isEmpty()) {
        while (curr != null) { stack.push(curr); curr = curr.left; }
        curr = stack.pop();
        if (curr.val <= prev) return false; // not strictly increasing
        prev = curr.val;
        curr = curr.right;
    }
    return true;
}
```

### 7.2 Lowest Common Ancestor — LeetCode 236

```java
// LCA of two nodes p and q in a binary tree (not necessarily BST).
// If current node is p or q: it's the LCA (the other must be in its subtree).
// If left returns non-null AND right returns non-null: current node is LCA.
// Time: O(n)  Space: O(h)
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;

    TreeNode left  = lowestCommonAncestor(root.left,  p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);

    if (left != null && right != null) return root; // p and q split across subtrees
    return left != null ? left : right;             // both in same subtree
}

// BST version (LeetCode 235): use BST property to avoid full scan
public TreeNode lcaBST(TreeNode root, TreeNode p, TreeNode q) {
    while (root != null) {
        if (p.val < root.val && q.val < root.val)      root = root.left;
        else if (p.val > root.val && q.val > root.val) root = root.right;
        else return root; // p and q split: root is LCA
    }
    return null;
}
```

### 7.3 Kth Smallest in BST — LeetCode 230

```java
// Inorder traversal of BST produces sorted order.
// The k-th element in inorder sequence is the k-th smallest.
// Time: O(h + k)  Space: O(h)
public int kthSmallest(TreeNode root, int k) {
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode curr = root;

    while (curr != null || !stack.isEmpty()) {
        while (curr != null) { stack.push(curr); curr = curr.left; }
        curr = stack.pop();
        if (--k == 0) return curr.val; // found k-th smallest
        curr = curr.right;
    }
    return -1;
}
```

## 8. Symmetric Tree and Structural Problems

### 8.1 Symmetric tree — LeetCode 101

```java
// Is the tree a mirror of itself?
// Time: O(n)  Space: O(h)
public boolean isSymmetric(TreeNode root) {
    return isMirror(root, root);
}

private boolean isMirror(TreeNode left, TreeNode right) {
    if (left == null && right == null) return true;
    if (left == null || right == null) return false;
    return left.val == right.val
        && isMirror(left.left,  right.right) // outer pair
        && isMirror(left.right, right.left); // inner pair
}

// Iterative BFS version: check pairs level by level
public boolean isSymmetricBFS(TreeNode root) {
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    queue.offer(root);

    while (!queue.isEmpty()) {
        TreeNode t1 = queue.poll();
        TreeNode t2 = queue.poll();
        if (t1 == null && t2 == null) continue;
        if (t1 == null || t2 == null) return false;
        if (t1.val != t2.val)         return false;
        // Enqueue in mirror order
        queue.offer(t1.left);  queue.offer(t2.right);
        queue.offer(t1.right); queue.offer(t2.left);
    }
    return true;
}
```

```java
// Flatten to right-linked list in preorder (in-place).
// Postorder traversal: process right, then left, then connect to prev.
// Time: O(n)  Space: O(h)
public void flatten(TreeNode root) {
    TreeNode[] prev = {null}; // previously processed node (rightmost so far)
    flattenHelper(root, prev);
}

private void flattenHelper(TreeNode node, TreeNode[] prev) {
    if (node == null) return;
    flattenHelper(node.right, prev); // process right subtree first
    flattenHelper(node.left,  prev);
    node.right = prev[0]; // connect current node to previously processed
    node.left  = null;
    prev[0] = node;
}
// Reverse preorder: right, left, root — equivalent to reverse of root,left,right
```

## 11. Interview Questions

**Q: When does BFS use less space than DFS on a binary tree, and when does DFS win?**

A: BFS uses O(w) space where w is the maximum width of the tree. DFS uses O(h) space where h is the height. For a complete or nearly complete binary tree with n nodes, the last level can hold up to n/2 nodes, making BFS O(n) space while DFS is O(log n) — DFS wins dramatically. For a skewed tree (a linked-list-like tree), height equals n so DFS is O(n) while BFS only ever holds one or two nodes at a time, making BFS O(1) effectively. The crossover point depends on the tree's shape. For balanced trees, DFS uses far less space. For problems that naturally require level-by-level output (level order, zigzag, right side view), BFS is the more natural fit regardless of space — but for pure tree exploration where level grouping isn't needed, DFS is typically preferred for balanced trees.

---

**Q: Why is the iterative inorder traversal pattern structured as an outer `while` with a nested `while` for left descent?**

A: The inner `while (curr != null)` descends left as deep as possible, pushing every node onto the stack. This simulates the recursive call stack: in recursive inorder, `inorder(node.left)` recurses until it hits null, implicitly stacking all the nodes it passes through. When we can't go further left, we process the top of the stack (the leftmost unprocessed node) and then move to its right child. The outer `while (curr != null || !stack.isEmpty())` terminates when both conditions are exhausted: `curr` is null (nowhere to descend) and the stack is empty (nothing left to backtrack to). The two-condition outer loop is necessary because after processing a node and moving to its right child, `curr` may be null (no right child) but the stack still has ancestors — stopping on `curr == null` alone would terminate too early.

---

**Q: Explain the `maxGain` pattern used in Maximum Path Sum and Diameter. Why do they both use a global variable?**

A: Both problems have the same structural challenge: the answer (maximum path sum, or diameter) can span across a node — using both its left and right subtrees — but when computing the value to return to a parent, a node can only contribute one direction (a path can't branch). The `maxGain` function serves dual purposes: it returns the maximum linear extension downward from the current node (for the parent's computation), and it updates a global maximum with the best path that passes through the current node (which may use both subtrees). The global variable — typically an `int[]` array of size 1 to allow mutation from inside a method — captures the answer at any node during the traversal. If we tried to return the answer directly, we'd lose the downward-extension value the parent needs. The pattern decouples "what I report upward" (one-direction gain) from "what I record globally" (two-direction path through me). This same pattern applies to any problem where the local return value to the parent differs from the globally interesting quantity.

---

**Q: How does prefix sum + HashMap work for Path Sum III (paths don't need to start at root)?**

A: The insight is that any downward path from node A to node B can be expressed as the difference between two prefix sums: `sum(root→B) - sum(root→A's parent)`. We maintain a running prefix sum from the root to the current node, and a HashMap counting how many times each prefix sum has been seen along the current root-to-node path. For each node, we ask: "how many ancestors have prefix sum equal to `currentSum - targetSum`?" — those ancestors define valid path starting points. The count from the HashMap is added to the result. We then recurse into children with the updated prefix sum, and critically, backtrack by decrementing the current prefix sum's count when we leave a node. The backtracking is essential: the HashMap must only contain prefix sums on the current root-to-node path. Without it, prefix sums from one branch would incorrectly appear as valid ancestors when processing another branch of the tree.

---

**Q: What is the difference between the LCA algorithm for a general binary tree versus a BST?**

A: For a general binary tree, we use a postorder DFS that searches both subtrees and combines their results. If both `left` and `right` recursive calls return non-null, it means p and q are split across the two subtrees, making the current node their LCA. If only one side returns non-null, both p and q are in that subtree and the non-null result propagates up. The algorithm processes the entire tree in O(n) time. For a BST, we exploit the ordering property: if both p and q are smaller than the current node, the LCA must be in the left subtree; if both are larger, it's in the right subtree; otherwise (one on each side, or one equals the current node), the current node is the LCA. This allows iterative traversal — just follow the correct child each step — achieving O(h) time without visiting every node. The BST version is O(log n) for a balanced BST, far better than the O(n) general case.

## 12. Key Takeaways

- **BFS = queue + levelSize snapshot**: `int levelSize = queue.size()` before the inner loop is the single line that separates level k from level k+1
- **Zigzag**: use `LinkedList` with `addFirst`/`addLast` — O(1) per node vs O(w) for full reversal
- **Iterative inorder**: `while (curr != null || !stack.isEmpty())` — two-condition outer loop; inner `while` descends left; then pop, process, move right
- **Iterative postorder**: compute modified preorder (root, right, left) using `addFirst` to prepend — reversal happens automatically
- **Path problems return dual values**: local return to parent (one-direction extension) vs global update (two-direction path through node) — use `int[]` array for mutable global state
- **Path Sum III**: prefix sum + HashMap with mandatory backtracking — map tracks only current root-to-node path; decrement on exit
- **BST inorder = sorted order**: kth smallest, validate BST (check strictly increasing), and range queries all exploit this
- **Validate BST with bounds**: pass `(min, max)` constraints down — simpler and handles duplicate-value edge cases better than inorder prev-comparison
- **LCA general tree**: both subtrees non-null → split → current node is LCA; one non-null → both in same subtree → propagate up
- **BFS vs DFS space**: complete tree → DFS uses O(log n) vs BFS O(n); skewed tree → opposite; choose BFS when level grouping is needed, DFS for path/subtree problems
