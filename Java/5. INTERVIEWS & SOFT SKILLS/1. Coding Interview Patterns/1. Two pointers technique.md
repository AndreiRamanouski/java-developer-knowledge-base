# Two Pointers Technique

## Overview

The two pointers technique uses two index variables that move through a data structure — typically an array or string — to reduce an O(n²) brute-force search to O(n). Instead of checking every pair with nested loops, the two pointers converge on the answer by moving intelligently based on the current values they point to.

**The core insight:**

```
Brute force: for every i, check every j → O(n²)
Two pointers: i and j move toward each other, skipping impossible pairs → O(n)

This works when:
  1. The array is sorted (or can be sorted), giving directional information
  2. The problem involves finding a pair or partition satisfying a condition
  3. Moving a pointer in one direction monotonically changes the relevant value
```

**Four canonical configurations:**

```
1. Opposite ends (converging):    left→  ←right    (Two Sum, Container Water)
2. Same direction (fast/slow):    slow→  fast→      (Floyd cycle, remove duplicates)
3. Sliding window:                left→  right→     (subarray problems)
4. Split array (partition):       ←left  right→     (Dutch National Flag, quicksort)
```

## 1. When to Use Two Pointers

```
Recognition signals in the problem statement:

"sorted array"              → likely converging two pointers
"find pair with sum = k"    → converging two pointers on sorted array
"remove duplicates in-place"→ fast/slow same-direction pointers
"palindrome"                → converging from both ends
"subarray/substring with X" → sliding window (variant of two pointers)
"partition around pivot"    → Dutch National Flag / quicksort partition
"linked list cycle"         → Floyd's slow/fast pointers
"merge two sorted arrays"   → two pointers one per array

NOT suited for two pointers:
  Unsorted array where no directional insight exists
  Problems requiring tracking more than one pair simultaneously
  Graph / tree problems (use BFS/DFS instead)
  When elements can be reused in complex ways (use DP instead)
```

## 2. Palindrome Check

### 2.1 Basic palindrome — converging pointers

```java
// Is string a palindrome? Compare characters from outside in.
// Time: O(n)  Space: O(1)
public boolean isPalindrome(String s) {
    int left = 0, right = s.length() - 1;

    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) return false;
        left++;
        right--;
    }
    return true;
}
// "racecar": l=r,a=a,c=c → left>=right → true
// "hello":   h≠o → false immediately
```

### 2.2 Valid palindrome II — LeetCode 680

```java
// Can you make s a palindrome by deleting AT MOST one character?
// Time: O(n)  Space: O(1)
public boolean validPalindromeII(String s) {
    int left = 0, right = s.length() - 1;

    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) {
            // Mismatch: try skipping left OR skipping right
            return isPalindromeRange(s, left + 1, right)
                || isPalindromeRange(s, left, right - 1);
        }
        left++; right--;
    }
    return true;
}

private boolean isPalindromeRange(String s, int left, int right) {
    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) return false;
        left++; right--;
    }
    return true;
}
// "abca": a=a, b≠c → try "bc" (false) OR "bc" wait → try skip left: "bca"[1..3]="ca"? no
//   skip right: isPalindromeRange("abca",1,2) = "bc"→ false
//   skip left:  isPalindromeRange("abca",2,3) = "ca"→ false... wait
// "abca": l=0,r=3: 'a'='a' → l=1,r=2: 'b'≠'c'
//   skip b: isPalindromeRange(s,2,2) = single char → true ✓
```

### 2.3 Palindrome with non-alphanumeric characters — LeetCode 125

```java
// "A man, a plan, a canal: Panama" → true (ignore spaces/punctuation, case-insensitive)
public boolean isPalindromeFiltered(String s) {
    int left = 0, right = s.length() - 1;

    while (left < right) {
        // Skip non-alphanumeric characters
        while (left < right && !Character.isLetterOrDigit(s.charAt(left)))  left++;
        while (left < right && !Character.isLetterOrDigit(s.charAt(right))) right--;

        if (Character.toLowerCase(s.charAt(left))
                != Character.toLowerCase(s.charAt(right))) return false;
        left++; right--;
    }
    return true;
}
```

### 2.4 Longest palindromic substring — expand from center

```java
// Expand around each center: O(n²) time, O(1) space
// (Manacher's is O(n) — see String Algorithms guide)
public String longestPalindrome(String s) {
    int start = 0, maxLen = 1;

    for (int i = 0; i < s.length(); i++) {
        // Odd length: center at i
        int len1 = expand(s, i, i);
        // Even length: center between i and i+1
        int len2 = expand(s, i, i + 1);
        int len  = Math.max(len1, len2);

        if (len > maxLen) {
            maxLen = len;
            start = i - (len - 1) / 2;
        }
    }
    return s.substring(start, start + maxLen);
}

private int expand(String s, int left, int right) {
    while (left >= 0 && right < s.length()
           && s.charAt(left) == s.charAt(right)) {
        left--; right++;
    }
    return right - left - 1;  // length when loop exits
}
```

## 3. Two Sum on Sorted Array

### 3.1 Two Sum II — LeetCode 167

```java
// Array is SORTED. Find indices of two numbers that sum to target.
// Time: O(n)  Space: O(1)
//
// Key insight: sorted array gives directional information:
//   sum too small → move left pointer right (increase sum)
//   sum too large → move right pointer left (decrease sum)
public int[] twoSumSorted(int[] numbers, int target) {
    int left = 0, right = numbers.length - 1;

    while (left < right) {
        int sum = numbers[left] + numbers[right];
        if      (sum == target) return new int[]{left + 1, right + 1}; // 1-indexed
        else if (sum < target)  left++;   // need larger sum → advance left
        else                    right--;  // need smaller sum → retreat right
    }
    return new int[]{-1, -1}; // no solution (problem guarantees one exists)
}
// [2,7,11,15], target=9:
//   l=0(2), r=3(15): sum=17 > 9 → r--
//   l=0(2), r=2(11): sum=13 > 9 → r--
//   l=0(2), r=1(7):  sum=9 == 9 → return [1,2] ✓
```

### 3.2 Two Sum on unsorted array — HashMap O(n)

```java
// Array is UNSORTED. Cannot use two pointers directly.
// HashMap: for each element, check if complement exists
// Time: O(n)  Space: O(n)
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>(); // value → index

    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (seen.containsKey(complement)) {
            return new int[]{seen.get(complement), i};
        }
        seen.put(nums[i], i);
    }
    return new int[]{-1, -1};
}
```

### 3.3 3Sum — LeetCode 15

```java
// Find all unique triplets [a, b, c] where a + b + c = 0.
// Sort first, then for each element fix it and use two pointers for the pair.
// Time: O(n²)  Space: O(1) ignoring output
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 2; i++) {
        // Skip duplicates for the first element
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        // Early termination: smallest triplet sum > 0 → no solution possible
        if (nums[i] > 0) break;

        int left = i + 1, right = nums.length - 1;
        int target = -nums[i]; // looking for pair that sums to -nums[i]

        while (left < right) {
            int sum = nums[left] + nums[right];
            if (sum == target) {
                result.add(List.of(nums[i], nums[left], nums[right]));
                // Skip duplicates for second and third elements
                while (left < right && nums[left]  == nums[left + 1])  left++;
                while (left < right && nums[right] == nums[right - 1]) right--;
                left++; right--;
            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }
    }
    return result;
}
// [-4,-1,-1,0,1,2]: fix i=0(-4): target=4, l=1,r=5: -1+2=1<4→l++
//   l=2(-1),r=5(2): 1<4→l++; l=3(0),r=5(2): 2<4→l++; l=4(1),r=5(2): 3<4→l++ done
//   fix i=1(-1): target=1, l=2(-1),r=5(2): -1+2=1=1 → [-1,-1,2] ✓
//   skip dup: nums[2]==nums[3]? -1==0? no. l=3(0),r=4(1): 0+1=1 → [-1,0,1] ✓
```

### 3.4 4Sum — LeetCode 18

```java
// Same idea extended: fix two elements, use two pointers for remaining pair.
// Time: O(n³)  Space: O(1)
public List<List<Integer>> fourSum(int[] nums, int target) {
    Arrays.sort(nums);
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 3; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        for (int j = i + 1; j < nums.length - 2; j++) {
            if (j > i + 1 && nums[j] == nums[j - 1]) continue;

            int left = j + 1, right = nums.length - 1;
            long remain = (long) target - nums[i] - nums[j]; // long: avoid overflow

            while (left < right) {
                int sum = nums[left] + nums[right];
                if (sum == remain) {
                    result.add(List.of(nums[i], nums[j], nums[left], nums[right]));
                    while (left < right && nums[left]  == nums[left + 1])  left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    left++; right--;
                } else if (sum < remain) left++;
                else                     right--;
            }
        }
    }
    return result;
}
```

## 4. Container With Most Water

### 4.1 Problem — LeetCode 11

```java
// n vertical lines at positions 0..n-1 with heights height[i].
// Find two lines that together with x-axis form a container holding most water.
// Water volume = min(height[left], height[right]) × (right - left)
// Time: O(n)  Space: O(1)
//
// Key insight: start with widest container (left=0, right=n-1).
//   To possibly increase volume, we MUST move the SHORTER line inward.
//   Moving the TALLER line inward can only decrease or maintain width
//   while the height is still limited by the shorter line → never helps.
public int maxArea(int[] height) {
    int left = 0, right = height.length - 1;
    int maxWater = 0;

    while (left < right) {
        int water = Math.min(height[left], height[right]) * (right - left);
        maxWater = Math.max(maxWater, water);

        // Move the shorter line inward — only way to potentially find more water
        if (height[left] < height[right]) left++;
        else                              right--;
    }
    return maxWater;
}
// [1,8,6,2,5,4,8,3,7]:
//   l=0(1),r=8(7): water=min(1,7)*8=8  → move left (shorter)
//   l=1(8),r=8(7): water=min(8,7)*7=49 → move right (shorter, tie: either works)
//   l=1(8),r=7(3): water=min(8,3)*6=18 → move right
//   l=1(8),r=6(8): water=min(8,8)*5=40 → move either
//   ...
//   maxWater = 49 ✓
```

### 4.2 Why always move the shorter line — proof

```
Suppose at state (left, right):
  height[left] = 3, height[right] = 7
  current water = 3 × (right - left)

Option A: move left inward to left' = left + 1
  new_water = min(height[left'], 7) × (right - left - 1)
  Could be larger if height[left'] >> 3

Option B: move right inward to right' = right - 1
  new_water = min(3, height[right']) × (right - left - 1)
  = at most 3 × (right - left - 1) < 3 × (right - left) = current water

Moving the taller side (B) can NEVER increase water because:
  - Width decreases by 1
  - Height is still capped at the shorter side (3)
  - Therefore: 3 × (width - 1) < 3 × width always

Conclusion: moving the shorter side is the only way to potentially improve.
The greedy strategy (always move shorter) is provably optimal.
```

### 4.3 Trapping rain water — LeetCode 42

```java
// Different problem: water TRAPPED between bars, not in a two-bar container.
// Water at position i = min(max_left[i], max_right[i]) - height[i]
//
// Two pointers approach (O(n) time, O(1) space):
// Maintain running max from left and from right.
// Process the side with smaller max (its trapped water is determined).
public int trap(int[] height) {
    int left = 0, right = height.length - 1;
    int leftMax = 0, rightMax = 0;
    int water = 0;

    while (left < right) {
        if (height[left] < height[right]) {
            if (height[left] >= leftMax) leftMax = height[left];
            else water += leftMax - height[left]; // trapped water at left position
            left++;
        } else {
            if (height[right] >= rightMax) rightMax = height[right];
            else water += rightMax - height[right]; // trapped water at right position
            right--;
        }
    }
    return water;
}
// [0,1,0,2,1,0,1,3,2,1,2,1]:
//   Process left side when height[left] < height[right]:
//   l=0(0): leftMax=0, height=0≥0 → leftMax=0
//   l=1(1): height=1≥leftMax=0 → leftMax=1
//   l=2(0): height=0 < leftMax=1 → water += 1-0 = 1
//   ... total water = 6 ✓
```

## 5. Fast and Slow Pointers

### 5.1 Linked list cycle detection — Floyd's algorithm

```java
// Detect cycle in linked list. No extra space.
// Slow moves 1 step; fast moves 2 steps.
// If cycle exists: fast laps slow → they meet.
// If no cycle: fast reaches null.
// Time: O(n)  Space: O(1)
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true; // meeting point inside cycle
    }
    return false;
}

// Find ENTRY POINT of cycle:
// After meeting, reset one pointer to head; advance both one step at a time.
// They meet again exactly at the cycle entrance. (mathematical proof via distances)
public ListNode detectCycleEntry(ListNode head) {
    ListNode slow = head, fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            // Cycle detected — find entry point
            ListNode entry = head;
            while (entry != slow) {
                entry = entry.next;
                slow  = slow.next;
            }
            return entry;
        }
    }
    return null; // no cycle
}
```

### 5.2 Find middle of linked list

```java
// Slow pointer reaches middle when fast pointer reaches end.
// Time: O(n)  Space: O(1)
public ListNode findMiddle(ListNode head) {
    ListNode slow = head, fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow; // for even length: returns second of two middle nodes
}
// 1→2→3→4→5: slow=3 (middle) ✓
// 1→2→3→4:   slow=3 (second middle for even length) ✓
```

### 5.3 Happy number — slow/fast on sequence

```java
// A happy number eventually reaches 1 when repeatedly replacing n
// with the sum of squares of its digits. Unhappy numbers loop.
// Use slow/fast to detect cycle in the sequence.
public boolean isHappy(int n) {
    int slow = n;
    int fast = sumOfSquares(n);

    while (fast != 1 && slow != fast) {
        slow = sumOfSquares(slow);
        fast = sumOfSquares(sumOfSquares(fast));
    }
    return fast == 1;
}

private int sumOfSquares(int n) {
    int sum = 0;
    while (n > 0) {
        int d = n % 10;
        sum += d * d;
        n /= 10;
    }
    return sum;
}
```

## 6. Remove Duplicates and Partition

### 6.1 Remove duplicates from sorted array — LeetCode 26

```java
// In-place removal. Slow pointer marks next write position.
// Fast pointer scans ahead for new (non-duplicate) values.
// Time: O(n)  Space: O(1)
public int removeDuplicates(int[] nums) {
    if (nums.length == 0) return 0;
    int slow = 0; // index of last unique element written

    for (int fast = 1; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow]) {
            slow++;
            nums[slow] = nums[fast]; // write new unique value
        }
    }
    return slow + 1; // length of deduplicated prefix
}
// [1,1,2,3,3]: slow=0(1), fast scans:
//   fast=1: 1==1 → skip
//   fast=2: 2≠1 → slow=1, nums[1]=2 → [1,2,2,3,3]
//   fast=3: 3≠2 → slow=2, nums[2]=3 → [1,2,3,3,3]
//   fast=4: 3==3 → skip
//   return 3 ✓

// Allow at most 2 duplicates (LeetCode 80):
public int removeDuplicatesII(int[] nums) {
    int slow = 0;
    for (int fast = 0; fast < nums.length; fast++) {
        // Write if fewer than 2 elements written OR new element differs from two ago
        if (slow < 2 || nums[fast] != nums[slow - 2]) {
            nums[slow++] = nums[fast];
        }
    }
    return slow;
}
```

### 6.3 Dutch National Flag — 3-way partition

```java
// Sort array containing only 0s, 1s, 2s (LeetCode 75 — Sort Colors).
// Three pointers: low (boundary for 0s), mid (current), high (boundary for 2s).
// Time: O(n)  Space: O(1)
public void sortColors(int[] nums) {
    int low = 0, mid = 0, high = nums.length - 1;

    while (mid <= high) {
        switch (nums[mid]) {
            case 0:
                swap(nums, low++, mid++); // 0: swap to front, advance both
                break;
            case 1:
                mid++;                    // 1: already in middle zone, just advance
                break;
            case 2:
                swap(nums, mid, high--);  // 2: swap to back, retreat high
                // DON'T advance mid — swapped value needs inspection
                break;
        }
    }
}

private void swap(int[] a, int i, int j) {
    int t = a[i]; a[i] = a[j]; a[j] = t;
}
// [2,0,2,1,1,0]: low=0,mid=0,high=5
//   nums[0]=2: swap(0,5)→[0,0,2,1,1,2], high=4. mid stays at 0
//   nums[0]=0: swap(0,0)→same, low=1,mid=1
//   nums[1]=0: swap(1,1)→same, low=2,mid=2
//   nums[2]=2: swap(2,4)→[0,0,1,1,2,2], high=3. mid stays at 2
//   nums[2]=1: mid=3
//   nums[3]=1: mid=4 > high=3 → done
//   Result: [0,0,1,1,2,2] ✓
```

## 7. Sliding Window (Two Pointers Variant)

### 7.1 Longest substring without repeating characters — LeetCode 3

```java
// Variable-size window: expand right, shrink left when duplicate found.
// Time: O(n)  Space: O(1) (alphabet size bounded)
public int lengthOfLongestSubstring(String s) {
    int[] lastIndex = new int[128]; // char → last seen index
    Arrays.fill(lastIndex, -1);
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        int c = s.charAt(right);
        if (lastIndex[c] >= left) {
            // Duplicate in current window: move left past the previous occurrence
            left = lastIndex[c] + 1;
        }
        lastIndex[c] = right;
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
```

### 7.2 Minimum size subarray sum — LeetCode 209

```java
// Shortest contiguous subarray with sum ≥ target.
// Time: O(n)  Space: O(1)
public int minSubArrayLen(int target, int[] nums) {
    int left = 0, sum = 0, minLen = Integer.MAX_VALUE;

    for (int right = 0; right < nums.length; right++) {
        sum += nums[right];

        while (sum >= target) {           // window satisfies condition
            minLen = Math.min(minLen, right - left + 1);
            sum -= nums[left++];          // shrink from left
        }
    }
    return minLen == Integer.MAX_VALUE ? 0 : minLen;
}
```

### 7.3 Fixed-size sliding window — max sum subarray of size k

```java
// Maximum sum of any contiguous subarray of exactly size k.
// Time: O(n)  Space: O(1)
public int maxSumSubarray(int[] nums, int k) {
    int windowSum = 0;
    for (int i = 0; i < k; i++) windowSum += nums[i]; // first window

    int maxSum = windowSum;
    for (int right = k; right < nums.length; right++) {
        windowSum += nums[right] - nums[right - k]; // slide: add right, drop left
        maxSum = Math.max(maxSum, windowSum);
    }
    return maxSum;
}
```

## 8. Sorted Array Merge (Two Pointers on Two Arrays)

```java
// Merge two sorted arrays into one sorted array.
// Time: O(m+n)  Space: O(m+n)
public int[] mergeSorted(int[] a, int[] b) {
    int[] result = new int[a.length + b.length];
    int i = 0, j = 0, k = 0;

    while (i < a.length && j < b.length) {
        if (a[i] <= b[j]) result[k++] = a[i++];
        else               result[k++] = b[j++];
    }
    while (i < a.length) result[k++] = a[i++];
    while (j < b.length) result[k++] = b[j++];
    return result;
}

// Merge in-place: merge sorted B into end of A (LeetCode 88)
// A has extra space at the end. Fill from right to avoid overwriting.
public void merge(int[] nums1, int m, int[] nums2, int n) {
    int i = m - 1, j = n - 1, k = m + n - 1; // three pointers from the right

    while (i >= 0 && j >= 0) {
        if (nums1[i] >= nums2[j]) nums1[k--] = nums1[i--];
        else                      nums1[k--] = nums2[j--];
    }
    while (j >= 0) nums1[k--] = nums2[j--]; // remaining B elements
}

// Intersection of two sorted arrays:
// Advance the pointer pointing to the smaller element.
public int[] intersection(int[] a, int[] b) {
    List<Integer> result = new ArrayList<>();
    int i = 0, j = 0;

    while (i < a.length && j < b.length) {
        if      (a[i] == b[j]) { result.add(a[i]); i++; j++; }
        else if (a[i] <  b[j]) i++;
        else                   j++;
    }
    return result.stream().mapToInt(Integer::intValue).toArray();
}
```

## 9. Summary: Two Pointer Patterns

```
Pattern                  Pointers              Move rule                  Problems
─────────────────────────────────────────────────────────────────────────────────────
Converging (sorted arr)  l→  ←r               sum<target: l++            Two Sum II
                                               sum>target: r--            Three Sum
                                               sum==target: record, both  Container Water

Palindrome check         l→  ←r               mismatch: return false     Palindrome
                                               match: both move inward    Valid Palindrome II

Expand from center       ←l  r→               match: expand outward      Longest Palindrome
                                               mismatch: stop, record

Fast/slow (linked list)  slow→  fast→→         always: slow+1, fast+2     Cycle Detection
                                               meeting: found cycle        Find Middle

Write/scan (in-place)    slow(write) fast(scan) new value: write to slow  Remove Duplicates
                                               duplicate: skip fast        Remove Element

3-way partition          low  mid  high        0: swap(low,mid), both++   Dutch Flag
                                               1: mid++                    Sort Colors
                                               2: swap(mid,high), high--

Sliding window           l→    r→              expand right always        Longest Substr
(variable)                                     shrink left when invalid   Min Subarray Sum

Two sorted arrays        i→    j→              advance smaller pointer    Merge Sorted
                                               equal: record, both        Intersection
```

## 10. Interview Questions

**Q: Why does the converging two-pointer approach work on a sorted array for Two Sum?**

A: Sorting gives us directional information: every element to the right of a pointer is larger, every element to the left is smaller. Start with the widest span (leftmost and rightmost elements). If their sum equals the target, we're done. If the sum is too small, the only way to increase it is to move the left pointer right (to a larger value) — moving the right pointer left would only decrease an already-too-small sum. If the sum is too large, we must move the right pointer left. Each step eliminates at least one element from consideration, so the two pointers traverse the array exactly once — O(n) instead of O(n²) for brute force. This works because on a sorted array, increasing left or decreasing right has predictable, monotonic effects on the sum.

---

**Q: Container With Most Water — why is it provably correct to always move the shorter line?**

A: At any state with pointers `left` and `right`, the water is `min(h[left], h[right]) × (right - left)`. If we move the taller line inward, the new height is still capped by the shorter line (which didn't change), and the width decreased — so the area strictly decreases or stays the same. There is no possible gain from moving the taller line. Moving the shorter line, however, might find a taller line inward, potentially increasing the effective height enough to compensate for the reduced width. So moving the shorter line is the only action that can possibly improve the result. Since every step moves a pointer inward and we always make the only potentially-improving move, the algorithm is correct. Formal proof: every pair `(i, j)` where `i < j` is either evaluated directly, or provably can't be the optimal answer when we skip it (because one of its endpoints was the shorter side at a wider span that was already better).

---

**Q: When would you use slow/fast pointers versus converging pointers?**

A: Converging pointers work when you want to reduce a search space by eliminating pairs — both pointers move toward the middle, never crossing. This requires the data to be sorted (or structured) so you know which pointer to move. Slow/fast pointers (Floyd's) detect cycles or find structural midpoints in sequences or linked lists where there is no "sorted" property and no specific target value. The slow pointer advances one step and the fast pointer advances two; if a cycle exists they must eventually meet. The choice comes down to the problem structure: sorted array with a target → converging; linked list with possible cycle → slow/fast; in-place deduplication or partition → write/scan (same direction, different speeds).

---

**Q: How do you extend Two Sum to Three Sum, and what is the time complexity?**

A: For Three Sum (sum = 0), sort the array first — O(n log n). Then iterate over each element `nums[i]` as the fixed first element, reducing the problem to Two Sum on the remaining sorted subarray `nums[i+1..n-1]` with target `-nums[i]`. Each Two Sum is solved with converging two pointers in O(n). Since we fix n possible first elements and run O(n) for each, the total is O(n²). Key details: skip duplicate values of `nums[i]` to avoid duplicate triplets in the result; after finding a valid pair, skip duplicates on both inner pointers before moving them inward. The pattern extends to k-Sum: sort once, fix k-2 elements with nested loops (O(n^(k-2))), solve the innermost pair with two pointers O(n), giving O(n^(k-1)) overall.

**Q: What is the invariant maintained in the Dutch National Flag three-pointer algorithm?**

A: At every step, the array is partitioned into four regions: `[0, low)` contains only 0s, `[low, mid)` contains only 1s, `[mid, high]` is unsorted/unknown, and `(high, n-1]` contains only 2s. The `mid` pointer is the "current element under investigation." When `nums[mid] == 0`: swap with `nums[low]`, increment both `low` and `mid` — the 0 is correctly placed in the 0s region, and the swapped element (known to be a 1, since `[low,mid)` holds 1s) is now at `mid-1` where it belongs. When `nums[mid] == 1`: just increment `mid` — it expands the 1s region. When `nums[mid] == 2`: swap with `nums[high]`, decrement `high` — the 2 is placed at the end, but `mid` does not advance because the element swapped from `high` is unknown and must be examined. The algorithm terminates when `mid > high`, meaning the unknown region is empty and the array is fully partitioned.

## 11. Key Takeaways

- **Sorted array + pair/sum problem** → converging two pointers; sum too small → advance left; sum too large → retreat right; O(n) vs O(n²) brute force
- **Palindrome** → converging from both ends; mismatch immediately returns false; skip non-alphanumeric with inner while loops
- **Container with most water** → always move the shorter line; moving the taller line can never increase area (width decreases while height cap stays the same)
- **Trapping rain water** → process the side with smaller running max; water at that position is `runningMax - height[current]`
- **Linked list cycle** → slow (×1) and fast (×2) pointers; they meet iff a cycle exists; reset one to head to find the entry point
- **In-place deduplication** → slow pointer writes; fast pointer scans; write only when `nums[fast] != nums[slow]`
- **Dutch National Flag** → three pointers; `low`/`mid`/`high` maintain invariant; when swapping 2 to back, do NOT advance `mid`
- **Two Sum on sorted: O(n); unsorted: O(n) with HashMap; Three Sum: O(n²) sort + fix + two pointers**
- **Sliding window is a two-pointer variant** where both pointers move in the same direction; expand right always, contract left when window violates the constraint
