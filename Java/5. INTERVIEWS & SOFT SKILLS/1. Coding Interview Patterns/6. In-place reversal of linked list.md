# In-Place Reversal of Linked List

## Overview

In-place linked list reversal rewires `next` pointers without allocating new nodes or using extra arrays. The fundamental operation — reversing a chain of nodes — appears as a standalone problem and as a building block inside harder problems (palindrome check, reorder list, k-group reversal). Mastering the three-pointer iterative reversal and the sublist boundary technique covers nearly every interview variant.

**The core operation:**

```
Before reversal:   null ← prev   curr → next → ...
                          ↑
                    we track these three pointers

Step: save curr.next, point curr.next back to prev, advance both
After reversal:    ... → next → curr → prev → null
                                        ↑
                               new head (old tail)

Key invariants throughout reversal:
  prev:  the new "head so far" (nodes already reversed)
  curr:  the node currently being redirected
  next:  saved reference to curr's original successor (would be lost after redirect)
```

**Why three pointers, not two:**

```
If we only have curr and prev:
  curr.next = prev;    ← this severs curr's connection to the rest of the list
  curr = curr.next;    ← curr.next is now prev, not the original next node!
  
We must save curr.next BEFORE redirecting, making three pointers mandatory.
```

## 1. Reverse a Linked List — LeetCode 206

### 1.1 Iterative — the canonical template

```java
// Reverse entire linked list in-place.
// Time: O(n)  Space: O(1)
public ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode curr = head;

    while (curr != null) {
        ListNode next = curr.next;  // 1. save next (about to be overwritten)
        curr.next = prev;           // 2. reverse the pointer
        prev = curr;                // 3. advance prev
        curr = next;                // 4. advance curr
    }
    return prev; // prev is the new head (curr is null at loop exit)
}
```

**Trace on `[1 → 2 → 3 → 4 → 5]`:**

```
State: prev=null, curr=1

Step 1: next=2, 1.next=null, prev=1, curr=2   →  null←1  2→3→4→5
Step 2: next=3, 2.next=1,   prev=2, curr=3    →  null←1←2  3→4→5
Step 3: next=4, 3.next=2,   prev=3, curr=4    →  null←1←2←3  4→5
Step 4: next=5, 4.next=3,   prev=4, curr=5    →  null←1←2←3←4  5
Step 5: next=null, 5.next=4, prev=5, curr=null → null←1←2←3←4←5

curr==null → exit. Return prev=5.
Result: 5→4→3→2→1→null ✓
```

### 1.2 Recursive — understanding the call stack

```java
// Recursive reversal — elegant but O(n) stack space.
// Time: O(n)  Space: O(n) call stack
public ListNode reverseListRecursive(ListNode head) {
    // Base case: empty list or single node → already reversed
    if (head == null || head.next == null) return head;

    // Recurse to the end — newHead = last node (becomes new list head)
    ListNode newHead = reverseListRecursive(head.next);

    // On the way back up: reverse the link between head and head.next
    head.next.next = head;  // make head.next point back to head
    head.next = null;        // sever head's original forward pointer

    return newHead; // propagate the new head all the way up
}
```

**Recursive call trace on `[1 → 2 → 3]`:**

```
reverseListRecursive(1):
  reverseListRecursive(2):
    reverseListRecursive(3):
      head=3, head.next=null → return 3   ← base case

    head=2, newHead=3
    head.next.next = head  →  3.next = 2  →  3→2
    head.next = null       →  2.next = null
    return 3

  head=1, newHead=3
  head.next.next = head  →  2.next = 1  →  3→2→1
  head.next = null       →  1.next = null
  return 3

Result: 3→2→1→null ✓

Call stack depth = n → O(n) space. Use iterative for O(1) space in interviews.
```

## 2. Reverse a Sublist — LeetCode 92

### 2.1 Reverse between positions left and right (1-indexed)

```java
// Reverse the nodes from position left to right (inclusive), 1-indexed.
// Time: O(n)  Space: O(1)
//
// Strategy:
//   1. Advance to the node just BEFORE position left (call it "before")
//   2. Run standard reversal for (right - left + 1) steps
//   3. Reconnect: before.next = new sublist head, old sublist head.next = node after sublist
public ListNode reverseBetween(ListNode head, int left, int right) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;

    // Step 1: advance "before" to the node just before position left
    ListNode before = dummy;
    for (int i = 1; i < left; i++) {
        before = before.next;
    }

    // Step 2: "subHead" is the first node of the sublist (at position left)
    //         After reversal it becomes the LAST node of the sublist
    ListNode subHead = before.next;

    // Step 3: reverse (right - left) times (connecting right - left + 1 nodes)
    ListNode prev = null;
    ListNode curr = subHead;
    for (int i = 0; i <= right - left; i++) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    // prev = new head of reversed sublist (was at position right)
    // curr = node at position right + 1 (node after sublist)

    // Step 4: reconnect
    before.next = prev;      // connect prefix to new sublist head
    subHead.next = curr;     // connect old sublist head (now tail) to suffix

    return dummy.next;
}
```

**Trace on `[1→2→3→4→5]`, `left=2`, `right=4`:**

```
dummy→1→2→3→4→5

Step 1: advance before to position 1 (just before position 2)
  before = dummy. i starts at 1, loop runs while i < 2 → one iteration:
  before = dummy.next = node(1)
  Now: before=1, before.next=2

Step 2: subHead = before.next = node(2)
  This is the start of the sublist [2,3,4]

Step 3: reverse 3 nodes (right-left+1 = 3 iterations, i = 0,1,2):
  i=0: next=3, 2.next=null,  prev=2, curr=3   →  null←2  3→4→5
  i=1: next=4, 3.next=2,     prev=3, curr=4   →  null←2←3  4→5
  i=2: next=5, 4.next=3,     prev=4, curr=5   →  null←2←3←4  5

  prev=4 (new sublist head), curr=5 (node after sublist)

Step 4: reconnect
  before.next = prev   →  node(1).next = node(4)
  subHead.next = curr  →  node(2).next = node(5)

Result: dummy→1→4→3→2→5→null ✓
Visualised: [1, 4, 3, 2, 5]
```

**Trace on `[1→2→3→4→5]`, `left=1`, `right=5` (reverse entire list):**

```
Step 1: left=1, loop runs while i < 1 → zero iterations. before = dummy.
Step 2: subHead = node(1).
Step 3: reverse all 5 nodes → prev=5, curr=null
Step 4: dummy.next = 5; node(1).next = null
Result: [5,4,3,2,1] ✓
```

### 2.2 The dummy head pattern

```
Why dummy head?
  Without dummy: reversing from left=1 requires special-casing "before" as null
  With dummy:    before always exists (it's dummy when left=1)
                 dummy.next always points to the (potentially changed) head
                 return dummy.next handles head-changes transparently

Rule: whenever a reversal or removal might change the head node,
      prepend a dummy (sentinel) node.
```

## 3. Reverse Every K Elements — LeetCode 25

### 3.1 Reverse Nodes in k-Group

```java
// Reverse every consecutive group of k nodes.
// If fewer than k nodes remain at the end, leave them as-is.
// Time: O(n)  Space: O(1)
public ListNode reverseKGroup(ListNode head, int k) {
    // Count if k nodes are available starting from head
    ListNode check = head;
    int count = 0;
    while (check != null && count < k) {
        check = check.next;
        count++;
    }
    if (count < k) return head; // fewer than k nodes → leave unchanged

    // Reverse k nodes starting from head
    ListNode prev = null, curr = head;
    for (int i = 0; i < k; i++) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    // After reversal:
    //   prev = new head of this k-group (was the k-th node)
    //   curr = head of the remaining list (k+1-th node)
    //   head = tail of this k-group (was the 1st node, now points to null)

    // Recursively reverse the remaining list and connect
    head.next = reverseKGroup(curr, k);

    return prev; // new head of this segment
}
```

**Trace on `[1→2→3→4→5]`, `k=2`:**

```

  count check: 1,2 → k=2 available ✓
  Reverse [1,2]: prev=2, curr=3, head=1(tail)
  head.next = reverseKGroup([3,4,5], 2)

  Call reverseKGroup([3,4,5], 2):
    count check: 3,4 → k=2 available ✓
    Reverse [3,4]: prev=4, curr=5, head=3(tail)
    head.next = reverseKGroup([5], 2)

    Call reverseKGroup([5], 2):
      count check: 5 → count=1 < k=2 → return node(5) unchanged

    head(3).next = node(5)  →  4→3→5
    return prev=4

  head(1).next = node(4)  →  2→1→4→3→5
  return prev=2

Result: 2→1→4→3→5→null ✓
```

### 3.2 Iterative k-group reversal (no recursion, O(1) stack space)

```java
// Iterative version — avoids O(n/k) recursion depth.
// Time: O(n)  Space: O(1)
public ListNode reverseKGroupIterative(ListNode head, int k) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode groupPrev = dummy; // node just before each k-group

    while (true) {
        // Find the k-th node from groupPrev
        ListNode kth = getKth(groupPrev, k);
        if (kth == null) break; // fewer than k nodes remain

        ListNode groupNext = kth.next; // node after the k-group

        // Reverse the k-group: groupPrev.next ... kth
        ListNode prev = groupNext; // reversed sublist "terminates" at groupNext
        ListNode curr = groupPrev.next;
        while (curr != groupNext) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }
        // prev = kth (new head of reversed group)
        // groupPrev.next still points to original head (now tail)
        ListNode tmp = groupPrev.next; // save old head (new tail) before overwriting
        groupPrev.next = kth;          // connect prefix to reversed group

        groupPrev = tmp; // advance groupPrev to the tail of the reversed group
        // groupPrev.next was already set to groupNext by the reversal
        // (original head's next is now groupNext after reversal terminates at groupNext)
    }
    return dummy.next;
}

// Helper: advance k steps from node; return the k-th node or null
private ListNode getKth(ListNode node, int k) {
    while (node != null && k > 0) {
        node = node.next;
        k--;
    }
    return node;
}
```

## 4. Reverse Alternating K Elements — LeetCode variant

```java
// Reverse every alternate k-group: reverse k, skip k, reverse k, skip k, ...
// Time: O(n)  Space: O(1)
public ListNode reverseAlternatingKGroup(ListNode head, int k) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode prev = dummy;

    while (head != null) {
        // Phase 1: reverse k nodes
        ListNode tail = prev;
        for (int i = 0; i < k; i++) {
            tail = tail.next;
            if (tail == null) return dummy.next; // fewer than k nodes
        }
        ListNode next = tail.next;

        // Reverse [head ... tail]
        ListNode[] reversed = reverseSublist(head, tail);
        prev.next = reversed[0];  // connect prefix to new head
        reversed[1].next = next;  // connect new tail to remainder

        prev = reversed[1]; // advance prev to tail of reversed group
        head = next;         // head is now the start of the next group

        // Phase 2: skip k nodes (leave as-is)
        for (int i = 0; i < k && head != null; i++) {
            prev = head;
            head = head.next;
        }
    }
    return dummy.next;
}

// Helper: reverse nodes from head to tail (inclusive).
// Returns [newHead, newTail] = [old tail, old head]
private ListNode[] reverseSublist(ListNode head, ListNode tail) {
    ListNode prev = tail.next; // reversal terminates at the node AFTER tail
    ListNode curr = head;
    while (prev != tail) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    return new ListNode[]{tail, head}; // tail is new head, head is new tail
}
```

## 5. Rotate List — Using Reversal

```java
// Rotate list right by k places. Alternative to the circular approach.
// Three-reversal technique (like rotating an array):
//   Full reverse → reverse first k → reverse remaining
// Time: O(n)  Space: O(1)
public ListNode rotateRight(ListNode head, int k) {
    if (head == null || head.next == null || k == 0) return head;

    // Find length
    int length = 1;
    ListNode tail = head;
    while (tail.next != null) { tail = tail.next; length++; }

    k = k % length;
    if (k == 0) return head;

    // Find the new tail (length - k - 1 steps from head)
    ListNode newTail = head;
    for (int i = 0; i < length - k - 1; i++) newTail = newTail.next;

    ListNode newHead = newTail.next;
    newTail.next = null;
    tail.next = head; // complete the circle — not needed for reversal approach
    // (shown as circular method; reversal approach equivalent but more complex)
    return newHead;
}
```

## 6. Palindrome Linked List — Reversal as Sub-Step

```java
// Check palindrome using fast/slow + in-place reversal (O(1) space).
// Covered fully in Fast and Slow Pointers guide — shown here as reversal context.
public boolean isPalindrome(ListNode head) {
    if (head == null || head.next == null) return true;

    // Find middle
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // Reverse second half
    ListNode secondHalf = reverseList(slow);
    ListNode copy = secondHalf; // save for restoration

    // Compare
    ListNode first = head;
    boolean result = true;
    while (secondHalf != null) {
        if (first.val != secondHalf.val) { result = false; break; }
        first = first.next;
        secondHalf = secondHalf.next;
    }

    // Restore
    reverseList(copy);
    return result;
}
```

## 7. Reorder List — Reversal + Merge

```java
// LeetCode 143: reorder L0→L1→...→Ln to L0→Ln→L1→Ln-1→...
// Three steps: find middle → reverse second half → merge alternately
// Time: O(n)  Space: O(1)
public void reorderList(ListNode head) {
    if (head == null || head.next == null) return;

    // Step 1: find middle and split
    ListNode slow = head, fast = head, prev = null;
    while (fast != null && fast.next != null) {
        prev = slow;
        slow = slow.next;
        fast = fast.next.next;
    }
    prev.next = null; // sever list at midpoint

    // Step 2: reverse second half
    ListNode second = reverseList(slow);

    // Step 3: merge alternately
    ListNode first = head;
    while (second != null) {
        ListNode tmp1 = first.next;
        ListNode tmp2 = second.next;
        first.next  = second;
        second.next = tmp1;
        first  = tmp1;
        second = tmp2;
    }
}
// [1,2,3,4,5]:
//   Split: [1,2] | [3,4,5]
//   Reverse second: [5,4,3]
//   Merge: 1→5→2→4→3 ✓
```

## 8. Swap Nodes in Pairs — LeetCode 24

```java
// Swap every two adjacent nodes. Special case of k=2 group reversal.
// Time: O(n)  Space: O(1)
public ListNode swapPairs(ListNode head) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode prev = dummy;

    while (prev.next != null && prev.next.next != null) {
        ListNode first  = prev.next;
        ListNode second = prev.next.next;

        // Swap: prev → second → first → rest
        prev.next   = second;
        first.next  = second.next;
        second.next = first;

        prev = first; // advance: first is now the tail of this pair
    }
    return dummy.next;
}
// [1,2,3,4]:
//   prev=dummy, first=1, second=2
//   dummy→2→1→3→4; prev=1
//   prev=1, first=3, second=4
//   dummy→2→1→4→3; prev=3
//   prev.next=null → exit
// Result: [2,1,4,3] ✓
```

## 9. Reverse Nodes in Even Length Groups — LeetCode 2074

```java
// Groups have sizes 1, 2, 3, 4, ... . Reverse each group with even size.
// Time: O(n)  Space: O(1)
public ListNode reverseEvenLengthGroups(ListNode head) {
    ListNode prev = head; // prev always ends at the tail of the last processed group
    int groupSize = 2;    // first group (size 1) always has odd size, start checking from 2

    while (prev.next != null) {
        // Count actual nodes in this group (may be fewer than groupSize at end of list)
        ListNode node = prev;
        int actualSize = 0;
        for (int i = 0; i < groupSize && node.next != null; i++) {
            actualSize++;
            node = node.next;
        }
        // node is now the last node of this group
        // actualSize is the number of nodes in the group

        if (actualSize % 2 == 1) {
            // Odd group: leave as-is, advance prev to end of group
            prev = node;
        } else {
            // Even group: reverse it
            ListNode tail = prev.next; // current group head → will become tail after reversal
            ListNode curr = prev.next;
            ListNode groupPrev = node.next; // node after the group (reversal terminates here)

            for (int i = 0; i < actualSize; i++) {
                ListNode next = curr.next;
                curr.next = groupPrev;
                groupPrev = curr;
                curr = next;
            }
            prev.next = groupPrev; // connect prefix to new group head
            prev = tail;           // tail is now the end of the reversed group
        }
        groupSize++;
    }
    return head;
}
```

## 11. Common Bugs and How to Avoid Them

```java
// BUG 1: Not saving next before redirecting
// WRONG:
curr.next = prev;
curr = curr.next; // curr.next is now prev, not the original next!

// RIGHT:
ListNode next = curr.next; // save FIRST
curr.next = prev;
curr = next;               // use saved reference

// BUG 2: Forgetting the dummy head when reversal may change head node
// WRONG (left=1 case breaks without dummy):
ListNode before = null; // null has no .next to assign

// RIGHT:
ListNode dummy = new ListNode(0);
dummy.next = head;
ListNode before = dummy; // always safe to call before.next

// BUG 3: Off-by-one in sublist advance
// "advance to node at position left" vs "advance to node BEFORE position left"
// We need the node BEFORE so we can rewire its .next after reversal.

// For left=2 (1-indexed): advance 1 step from dummy (loop: i=1; i < left; i++)
// before ends up at position left-1 → before.next is the sublist head

// BUG 4: Forgetting to reconnect the tail after reversal
// After reversing [l..r]:
before.next = prev;        // ← connect prefix to new sublist head
subHead.next = curr;       // ← connect OLD head (new tail) to remainder
// Missing EITHER line leaves part of the list disconnected

// BUG 5: In k-group, not checking if k nodes are available
// Reversing fewer nodes than k gives wrong result.
// Always count k nodes BEFORE reversing:
ListNode kth = getKth(groupPrev, k);
if (kth == null) break; // fewer than k remain → stop
```

## 12. Interview Questions

**Q: Walk through the three-pointer reversal step by step. Why can't it be done with two pointers?**

A: The three pointers are `prev` (the already-reversed portion's head), `curr` (the node currently being redirected), and `next` (a temporary save of `curr.next`). In each step: save `curr.next` before it's overwritten, redirect `curr.next` to point backward at `prev`, advance `prev` to `curr`, and advance `curr` to the saved `next`. Two pointers fail because redirecting `curr.next = prev` permanently overwrites the only reference to the rest of the list. Before that line executes, `curr.next` is the path forward; after it executes, `curr.next` is the path backward. Without first saving `curr.next` in a third variable, the rest of the list is unreachable and effectively leaked. The third pointer isn't about tracking additional state — it's purely a temporary save to survive the mutation.

---

**Q: How does the dummy head pattern simplify reversal problems, and when is it strictly necessary?**

A: The dummy head (sentinel node) placed before the real head unifies the logic for all positions. In Reverse Sublist, we need a "before" node whose `.next` we'll overwrite after reversal. When `left = 1`, the sublist starts at the head and there is no natural "before" node. Without a dummy, we'd need a special case: if `left == 1`, the return value changes (it becomes `prev` instead of `head`), and we'd need an extra conditional. With a dummy, `before` is always the dummy node when `left = 1`, and `dummy.next` after rewiring always gives the correct new head regardless of whether the head changed. The same logic applies in k-group reversal (`groupPrev` starts as dummy), swap pairs, and any problem where the head node itself might be moved or removed. The cost is one extra node allocation — always worth it for the clarity and correctness it provides.

---

**Q: In Reverse Nodes in k-Group, what happens to the last group if it has fewer than k nodes, and why?**

A: The last group is left in its original order. The problem specification requires this: "if the number of nodes is not a multiple of k then left-out nodes at the end should remain as they are." The implementation enforces this by counting k nodes before committing to a reversal. In the recursive version, if fewer than k nodes are available, we return `head` unchanged — the recursion unwinds, and the preceding groups' tails are connected to the unmodified remainder. In the iterative version, `getKth` returns null when fewer than k nodes exist, triggering a `break` that stops processing. If we reversed the partial group, the output would violate the specification. The count-before-reverse discipline is the critical correctness condition for this problem.

---

**Q: How would you reverse only the nodes at even positions in a linked list?**

A: Traverse the list maintaining a counter. For each even-indexed node (2, 4, 6, ...), collect them into a sub-problem or track their boundaries. A cleaner approach: since even-position nodes form an interleaved subsequence, extract them into a separate list, reverse that list, and re-interleave. In O(1) space: use two passes — first pass reverses the values at even positions using a stack (but that's O(n) space). For true O(1) space reversal of positional subsets, we'd need the nodes to be contiguous — non-contiguous reversal in O(1) space requires either value-swapping (if allowed) or a more complex pointer rewiring. The standard interview answer is: collect even-position node values into an array, reverse the array, re-assign values — O(n) time, O(n/2) space. If pointer rewiring of non-contiguous nodes is required, partition the list into two sublists (odd-indexed and even-indexed), reverse the even sublist, and re-merge alternately — O(n) time, O(1) space.

---

**Q: In Reverse Sublist, what are the four pointer assignments needed after the reversal loop, and what breaks if you omit each?**

A: After the reversal loop, `prev` is the new head of the reversed sublist and `curr` is the first node after the sublist. There are two connections to establish. First, `before.next = prev` links the prefix to the reversed sublist's new head. Omitting this means the prefix still points to `subHead` (the old head, now tail), making the reversed portion unreachable from the list head. Second, `subHead.next = curr` links the old sublist head (now its tail) to the suffix. Without this, `subHead.next` remains null (the reversal loop set it to null when `prev` was null at the start), permanently severing the list — all nodes after position `right` are lost. The dummy head handles the prefix connection when `left = 1`, and `subHead` is saved before the loop begins precisely because the loop will redirect `subHead.next` to `null` (when `prev` starts as `null`). All four facts — save `subHead`, save `before`, assign `before.next = prev`, assign `subHead.next = curr` — must be present for correctness.

---

## 13. Key Takeaways

- **Three-pointer template**: `next = curr.next` → `curr.next = prev` → `prev = curr` → `curr = next` — memorise this sequence; every reversal variant uses it
- **Return `prev`, not `curr`**: at loop exit, `curr` is null; `prev` is the new head of the reversed list
- **Dummy head whenever head might change**: sublist reversal at position 1, k-group reversal, swap pairs — dummy unifies "before" to always exist
- **Sublist reconnection requires two assignments**: `before.next = prev` (prefix to new head) AND `subHead.next = curr` (old head to suffix) — omitting either breaks the list
- **k-group: count before reversing**: check that k nodes are available; leave the last partial group untouched
- **Recursive k-group**: `head.next = reverseKGroup(curr, k)` — `head` is the tail of the just-reversed group and connects to the recursively reversed remainder
- **Iterative k-group**: `groupPrev` tracks the node just before each group; after reversal, `groupPrev` advances to the old group head (new group tail)
- **Reversal as sub-step**: palindrome check (reverse second half, compare, restore), reorder list (reverse second half, merge), rotate list (find new tail, sever, reconnect)
- **O(1) space mandates in-place pointer rewiring** — never copy values to an array; rewire `.next` pointers directly
