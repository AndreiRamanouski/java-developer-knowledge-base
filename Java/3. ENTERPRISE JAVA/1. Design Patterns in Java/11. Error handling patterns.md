# Error Handling Patterns

## Overview

Error handling is the art of managing failure gracefully—detecting errors early, recovering when possible, and failing cleanly when recovery isn't an option. Poor error handling leads to silent failures (bugs hide), cascading failures (one error triggers many), unclear error messages (can't debug), and system instability. This guide covers six critical patterns: Fail Fast vs Fail Safe (when to crash immediately vs continue with degraded functionality), Exception Translation (converting low-level exceptions to domain-specific ones), Checked vs Unchecked Exception Strategy (when to force handling vs allow propagation), Null Object Pattern vs Optional (representing absence safely), Try-Success-Failure Pattern (functional error handling without exceptions), and Circuit Breaker Pattern (preventing cascading failures in distributed systems). The key insight: error handling is about trade-offs. Fail Fast catches bugs early but crashes on minor issues. Fail Safe keeps systems running but hides problems. Checked exceptions force handling but create boilerplate. Unchecked exceptions are clean but easy to ignore. Optional is type-safe but verbose. Null Object is convenient but can hide bugs. Understanding error patterns means understanding context—what's right for a library (fail fast, expose all errors) differs from what's right for a user-facing app (fail safe, recover gracefully). This guide shows when to use each pattern, common mistakes (swallowing exceptions, catching Exception, returning null for errors), and how patterns combine (Circuit Breaker with Fail Safe, Exception Translation with domain boundaries, Optional with Null Object for collections).

## 1. Fail Fast vs Fail Safe

### When to Crash vs When to Continue

```java
/**
 * FAIL FAST VS FAIL SAFE
 * 
 * Fail Fast: Detect and report errors immediately
 * Fail Safe: Continue operating despite errors
 */

// ========================================
// FAIL FAST PATTERN
// ========================================

/**
 * Fail fast: Validate inputs and crash on invalid data
 */
class FailFastValidator {
    
    /**
     * Constructor validates all inputs
     */
    public static class BankAccount {
        private final String accountNumber;
        private final String ownerName;
        private final BigDecimal balance;
        
        public BankAccount(String accountNumber, String ownerName, BigDecimal balance) {
            // ✓ FAIL FAST: Validate immediately
            if (accountNumber == null || accountNumber.trim().isEmpty()) {
                throw new IllegalArgumentException("Account number required");
            }
            if (!accountNumber.matches("\\d{10}")) {
                throw new IllegalArgumentException("Account number must be 10 digits");
            }
            if (ownerName == null || ownerName.trim().isEmpty()) {
                throw new IllegalArgumentException("Owner name required");
            }
            if (balance == null) {
                throw new IllegalArgumentException("Balance required");
            }
            if (balance.compareTo(BigDecimal.ZERO) < 0) {
                throw new IllegalArgumentException("Balance cannot be negative");
            }
            
            this.accountNumber = accountNumber;
            this.ownerName = ownerName;
            this.balance = balance;
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ Can't create invalid object
         * ✓ Bugs caught immediately (at construction)
         * ✓ No invalid state possible
         * ✓ Clear error messages
         * ✓ Easy to debug (fail at source)
         */
    }
    
    /**
     * Method validates parameters
     */
    public void transferMoney(BankAccount from, BankAccount to, BigDecimal amount) {
        // ✓ FAIL FAST: Validate preconditions
        Objects.requireNonNull(from, "Source account required");
        Objects.requireNonNull(to, "Destination account required");
        Objects.requireNonNull(amount, "Amount required");
        
        if (amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new IllegalArgumentException("Amount must be positive");
        }
        
        // Proceed with valid inputs
        // ...
    }
}

/**
 * When to use FAIL FAST:
 * 
 * ✓ Development/testing (catch bugs early)
 * ✓ Input validation (prevent invalid data)
 * ✓ Critical operations (money, safety)
 * ✓ Library code (caller's responsibility to fix)
 * ✓ Initialization (better to fail than run broken)
 */

// ========================================
// FAIL SAFE PATTERN
// ========================================

/**
 * Fail safe: Continue despite errors, use defaults
 */
class FailSafeHandler {
    
    /**
     * Parse configuration with defaults
     */
    public static class ConfigLoader {
        private static final int DEFAULT_PORT = 8080;
        private static final int DEFAULT_TIMEOUT = 30;
        private static final String DEFAULT_HOST = "localhost";
        
        public Config loadConfig(Properties properties) {
            // ✓ FAIL SAFE: Use defaults if parsing fails
            int port = parsePort(properties.getProperty("server.port"));
            int timeout = parseTimeout(properties.getProperty("timeout"));
            String host = parseHost(properties.getProperty("host"));
            
            return new Config(host, port, timeout);
        }
        
        private int parsePort(String value) {
            if (value == null || value.trim().isEmpty()) {
                return DEFAULT_PORT;  // Default
            }
            try {
                int port = Integer.parseInt(value);
                if (port < 1 || port > 65535) {
                    System.err.println("Invalid port: " + port + ", using default: " + DEFAULT_PORT);
                    return DEFAULT_PORT;
                }
                return port;
            } catch (NumberFormatException e) {
                System.err.println("Invalid port format: " + value + ", using default: " + DEFAULT_PORT);
                return DEFAULT_PORT;
            }
        }
        
        private int parseTimeout(String value) {
            if (value == null || value.trim().isEmpty()) {
                return DEFAULT_TIMEOUT;
            }
            try {
                int timeout = Integer.parseInt(value);
                return Math.max(timeout, 1);  // At least 1 second
            } catch (NumberFormatException e) {
                System.err.println("Invalid timeout: " + value + ", using default: " + DEFAULT_TIMEOUT);
                return DEFAULT_TIMEOUT;
            }
        }
        
        private String parseHost(String value) {
            if (value == null || value.trim().isEmpty()) {
                return DEFAULT_HOST;
            }
            return value;
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ App keeps running despite bad config
         * ✓ Graceful degradation
         * ✓ User-friendly (doesn't crash)
         * ✓ Reasonable defaults
         * 
         * DRAWBACKS:
         * 
         * ✗ Hides configuration errors
         * ✗ May run with wrong settings
         * ✗ Harder to debug
         */
    }
    
    static class Config {
        final String host;
        final int port;
        final int timeout;
        Config(String host, int port, int timeout) {
            this.host = host;
            this.port = port;
            this.timeout = timeout;
        }
    }
}

/**
 * When to use FAIL SAFE:
 * 
 * ✓ Production systems (availability critical)
 * ✓ User-facing apps (don't crash on user error)
 * ✓ Configuration (use sensible defaults)
 * ✓ Optional features (degrade gracefully)
 * ✓ Network operations (retry, fallback)
 */

// ========================================
// COMBINING FAIL FAST AND FAIL SAFE
// ========================================

/**
 * Use both patterns appropriately
 */
class CombinedApproach {
    
    /**
     * Email service with fail fast validation and fail safe sending
     */
    public static class EmailService {
        private final SmtpConfig config;
        
        public EmailService(SmtpConfig config) {
            // FAIL FAST: Validate config at startup
            Objects.requireNonNull(config, "SMTP config required");
            if (config.host == null || config.host.isEmpty()) {
                throw new IllegalArgumentException("SMTP host required");
            }
            if (config.port < 1 || config.port > 65535) {
                throw new IllegalArgumentException("Invalid SMTP port: " + config.port);
            }
            this.config = config;
        }
        
        public void sendEmail(String to, String subject, String body) {
            // FAIL FAST: Validate inputs
            Objects.requireNonNull(to, "Recipient required");
            Objects.requireNonNull(subject, "Subject required");
            Objects.requireNonNull(body, "Body required");
            
            if (!isValidEmail(to)) {
                throw new IllegalArgumentException("Invalid email: " + to);
            }
            
            // FAIL SAFE: Try to send, log on failure but don't crash
            try {
                sendEmailInternal(to, subject, body);
            } catch (Exception e) {
                // Log error but don't crash the app
                System.err.println("Failed to send email to " + to + ": " + e.getMessage());
                // Could: Queue for retry, send to dead letter queue, etc.
            }
        }
        
        private void sendEmailInternal(String to, String subject, String body) {
            // Actual sending logic
        }
        
        private boolean isValidEmail(String email) {
            return email.contains("@");
        }
        
        /**
         * PATTERN:
         * 
         * FAIL FAST for:
         * • Configuration (startup)
         * • Input validation (caller's error)
         * 
         * FAIL SAFE for:
         * • Network operations (external failure)
         * • Optional features (email is nice-to-have)
         */
    }
    
    static class SmtpConfig {
        String host;
        int port;
    }
}

// ========================================
// REAL-WORLD EXAMPLE: Payment Processing
// ========================================

/**
 * Payment system combining both patterns
 */
class PaymentProcessor {
    private final PaymentGateway gateway;
    private final FraudDetector fraudDetector;
    
    public PaymentProcessor(PaymentGateway gateway, FraudDetector fraudDetector) {
        // FAIL FAST: Dependencies required
        this.gateway = Objects.requireNonNull(gateway);
        this.fraudDetector = Objects.requireNonNull(fraudDetector);
    }
    
    public PaymentResult processPayment(PaymentRequest request) {
        // FAIL FAST: Validate request
        validateRequest(request);
        
        // FAIL SAFE: Continue with fraud check failure
        boolean suspicious = false;
        try {
            suspicious = fraudDetector.isSuspicious(request);
        } catch (Exception e) {
            // Fraud detector down - log and proceed (FAIL SAFE)
            System.err.println("Fraud detection failed: " + e.getMessage());
            // Could: default to manual review, block high-value transactions, etc.
        }
        
        if (suspicious) {
            return PaymentResult.rejected("Fraud detected");
        }
        
        // FAIL FAST: Payment gateway failures are critical
        try {
            return gateway.charge(request);
        } catch (PaymentException e) {
            // Payment failures are NOT fail-safe - must propagate
            throw e;
        }
    }
    
    private void validateRequest(PaymentRequest request) {
        // FAIL FAST: Invalid requests rejected immediately
        Objects.requireNonNull(request, "Payment request required");
        Objects.requireNonNull(request.amount, "Amount required");
        Objects.requireNonNull(request.currency, "Currency required");
        
        if (request.amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new IllegalArgumentException("Amount must be positive");
        }
    }
}

interface PaymentGateway {
    PaymentResult charge(PaymentRequest request);
}

interface FraudDetector {
    boolean isSuspicious(PaymentRequest request);
}

class PaymentRequest {
    BigDecimal amount;
    String currency;
}

class PaymentResult {
    static PaymentResult rejected(String reason) { return new PaymentResult(); }
}

class PaymentException extends RuntimeException {
    PaymentException(String msg) { super(msg); }
}
```

## 2. Exception Translation

### Converting Between Exception Layers

```java
/**
 * EXCEPTION TRANSLATION
 * 
 * Convert low-level exceptions to high-level domain exceptions
 * Preserve abstraction layers
 */

// ========================================
// PROBLEM: LEAKY ABSTRACTION
// ========================================

/**
 * ❌ BAD: Low-level exceptions leak to caller
 */
class BadUserRepository {
    
    public User findById(String id) throws SQLException {
        Connection conn = getConnection();
        PreparedStatement stmt = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
        stmt.setString(1, id);
        ResultSet rs = stmt.executeQuery();
        
        if (rs.next()) {
            return new User(rs.getString("name"));
        }
        return null;
    }
    
    /**
     * PROBLEMS:
     * 
     * ✗ SQLException leaks (implementation detail)
     * ✗ Caller coupled to database
     * ✗ Can't change to REST API without breaking callers
     * ✗ Violates abstraction
     */
    
    private Connection getConnection() throws SQLException { return null; }
}

// ========================================
// SOLUTION: EXCEPTION TRANSLATION
// ========================================

/**
 * ✓ GOOD: Translate to domain exceptions
 */
class GoodUserRepository {
    
    public User findById(String id) {
        try {
            Connection conn = getConnection();
            PreparedStatement stmt = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
            stmt.setString(1, id);
            ResultSet rs = stmt.executeQuery();
            
            if (rs.next()) {
                return new User(rs.getString("name"));
            }
            return null;
            
        } catch (SQLException e) {
            // ✓ TRANSLATE: SQLException → DataAccessException
            throw new DataAccessException("Failed to find user: " + id, e);
        }
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ Caller not coupled to database
     * ✓ Can change implementation (REST, file, etc.)
     * ✓ Domain-specific exception
     * ✓ Preserves abstraction
     */
    
    private Connection getConnection() throws SQLException { return null; }
}

/**
 * Domain exception hierarchy
 */
class DataAccessException extends RuntimeException {
    public DataAccessException(String message, Throwable cause) {
        super(message, cause);
    }
}

class UserNotFoundException extends DataAccessException {
    public UserNotFoundException(String userId) {
        super("User not found: " + userId, null);
    }
}

class DuplicateUserException extends DataAccessException {
    public DuplicateUserException(String email) {
        super("User already exists: " + email, null);
    }
}

// ========================================
// EXCEPTION TRANSLATION AT BOUNDARIES
// ========================================

/**
 * Translate at architectural boundaries
 */
class LayeredArchitecture {
    
    /**
     * LAYER 1: Data Access (Repository)
     * Translates: SQLException → DataAccessException
     */
    static class UserRepository {
        
        public User save(User user) {
            try {
                // Database operation
                saveToDB(user);
                return user;
                
            } catch (SQLException e) {
                if (isDuplicateKeyError(e)) {
                    throw new DuplicateUserException(user.getEmail());
                }
                throw new DataAccessException("Failed to save user", e);
            }
        }
        
        private void saveToDB(User user) throws SQLException {}
        private boolean isDuplicateKeyError(SQLException e) { return false; }
    }
    
    /**
     * LAYER 2: Business Logic (Service)
     * Translates: DataAccessException → BusinessException
     */
    static class UserService {
        private final UserRepository repository;
        
        public UserService(UserRepository repository) {
            this.repository = repository;
        }
        
        public User registerUser(String email, String password) {
            try {
                User user = new User(email);
                return repository.save(user);
                
            } catch (DuplicateUserException e) {
                // Translate to business exception
                throw new RegistrationException("Email already registered: " + email, e);
            } catch (DataAccessException e) {
                // Translate generic data exception
                throw new BusinessException("Registration failed", e);
            }
        }
    }
    
    /**
     * LAYER 3: Presentation (Controller)
     * Translates: BusinessException → HTTP Response
     */
    static class UserController {
        private final UserService userService;
        
        public UserController(UserService userService) {
            this.userService = userService;
        }
        
        public HttpResponse register(String email, String password) {
            try {
                User user = userService.registerUser(email, password);
                return HttpResponse.ok(user);
                
            } catch (RegistrationException e) {
                // Translate to HTTP 400 Bad Request
                return HttpResponse.badRequest(e.getMessage());
                
            } catch (BusinessException e) {
                // Translate to HTTP 500 Internal Server Error
                return HttpResponse.serverError("Registration failed");
            }
        }
    }
}

class BusinessException extends RuntimeException {
    public BusinessException(String message, Throwable cause) {
        super(message, cause);
    }
}

class RegistrationException extends BusinessException {
    public RegistrationException(String message, Throwable cause) {
        super(message, cause);
    }
}

class HttpResponse {
    static HttpResponse ok(Object body) { return new HttpResponse(); }
    static HttpResponse badRequest(String message) { return new HttpResponse(); }
    static HttpResponse serverError(String message) { return new HttpResponse(); }
}

// ========================================
// BEST PRACTICES
// ========================================

/**
 * Exception translation best practices
 */
class ExceptionTranslationBestPractices {
    
    /**
     * 1. PRESERVE CAUSE (chain exceptions)
     */
    public void preserveCause() {
        try {
            riskyOperation();
        } catch (LowLevelException e) {
            // ✓ GOOD: Preserve original exception
            throw new HighLevelException("Operation failed", e);
            
            // ❌ BAD: Lose original exception
            // throw new HighLevelException("Operation failed");
        }
    }
    
    /**
     * 2. ADD CONTEXT
     */
    public void addContext(String userId) {
        try {
            deleteUser(userId);
        } catch (SQLException e) {
            // ✓ GOOD: Add contextual information
            throw new DataAccessException(
                "Failed to delete user: " + userId + 
                " (table: users, operation: DELETE)", 
                e
            );
        }
    }
    
    /**
     * 3. TRANSLATE AT BOUNDARIES
     */
    // Repository layer: SQLException → DataAccessException
    // Service layer: DataAccessException → BusinessException
    // Controller layer: BusinessException → HTTP Response
    
    /**
     * 4. DON'T OVER-TRANSLATE
     */
    public void dontOverTranslate() {
        try {
            process();
        } catch (IllegalArgumentException e) {
            // ❌ BAD: Unnecessary translation
            // throw new ProcessingException("Invalid argument", e);
            
            // ✓ GOOD: IllegalArgumentException is already appropriate
            throw e;
        }
    }
    
    private void riskyOperation() throws LowLevelException {}
    private void deleteUser(String userId) throws SQLException {}
    private void process() {}
}

class LowLevelException extends Exception {
    LowLevelException(String msg) { super(msg); }
}

class HighLevelException extends RuntimeException {
    HighLevelException(String msg, Throwable cause) { super(msg, cause); }
}
```

## 3. Checked vs Unchecked Exception Strategy

### When to Force Handling vs Allow Propagation

```java
/**
 * CHECKED VS UNCHECKED EXCEPTIONS
 * 
 * Checked: Caller must handle (compile-time enforcement)
 * Unchecked: Caller can ignore (runtime exceptions)
 */

// ========================================
// CHECKED EXCEPTIONS
// ========================================

/**
 * Checked exceptions: Expected, recoverable conditions
 */
class CheckedExceptionExamples {
    
    /**
     * File operations: IOException (checked)
     * Reason: File might not exist, disk might be full
     */
    public String readFile(String path) throws IOException {
        // Caller MUST handle IOException
        return Files.readString(Paths.get(path));
    }
    
    /**
     * Network operations: SocketException (checked)
     * Reason: Network might be down, timeout might occur
     */
    public void connectToServer(String host, int port) throws IOException {
        // Caller MUST handle IOException
        Socket socket = new Socket(host, port);
        // ...
    }
    
    /**
     * Parsing operations: ParseException (checked)
     * Reason: Input might be malformed
     */
    public Date parseDate(String dateStr) throws ParseException {
        // Caller MUST handle ParseException
        SimpleDateFormat format = new SimpleDateFormat("yyyy-MM-dd");
        return format.parse(dateStr);
    }
    
    /**
     * WHEN TO USE CHECKED EXCEPTIONS:
     * 
     * ✓ Recoverable conditions (caller can handle)
     * ✓ Expected failures (part of normal flow)
     * ✓ External dependencies (file, network, DB)
     * ✓ Input validation (user data might be invalid)
     */
}

// ========================================
// UNCHECKED EXCEPTIONS
// ========================================

/**
 * Unchecked exceptions: Programming errors, unexpected conditions
 */
class UncheckedExceptionExamples {
    
    /**
     * Null pointer: NullPointerException (unchecked)
     * Reason: Programming error (should validate)
     */
    public void processUser(User user) {
        // ❌ If user is null, NPE thrown
        String name = user.getName();  // Programming error if null
    }
    
    /**
     * Invalid arguments: IllegalArgumentException (unchecked)
     * Reason: Caller's responsibility to provide valid input
     */
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            throw new IllegalArgumentException("Invalid age: " + age);
        }
        this.age = age;
    }
    private int age;
    
    /**
     * Invalid state: IllegalStateException (unchecked)
     * Reason: Object used incorrectly
     */
    public void withdraw(double amount) {
        if (closed) {
            throw new IllegalStateException("Account is closed");
        }
        // ...
    }
    private boolean closed;
    
    /**
     * WHEN TO USE UNCHECKED EXCEPTIONS:
     * 
     * ✓ Programming errors (bugs in code)
     * ✓ Precondition violations (invalid arguments)
     * ✓ Unrecoverable errors (OutOfMemoryError)
     * ✓ Framework-level errors (configuration)
     */
}

// ========================================
// MODERN JAVA: PREFER UNCHECKED
// ========================================

/**
 * Modern trend: Prefer unchecked exceptions
 */
class ModernExceptionStrategy {
    
    /**
     * OLD STYLE: Checked exceptions everywhere
     */
    class OldStyle {
        public void processOrder(Order order) 
                throws ValidationException,      // Checked
                       InventoryException,        // Checked
                       PaymentException,          // Checked
                       ShippingException {        // Checked
            
            validateOrder(order);    // throws ValidationException
            reserveInventory(order); // throws InventoryException
            processPayment(order);   // throws PaymentException
            arrangeShipping(order);  // throws ShippingException
        }
        
        /**
         * PROBLEMS:
         * 
         * ✗ Method signature cluttered
         * ✗ Caller forced to handle all exceptions
         * ✗ Boilerplate try-catch everywhere
         * ✗ Hard to refactor (changing exceptions breaks callers)
         */
        
        private void validateOrder(Order order) throws ValidationException {}
        private void reserveInventory(Order order) throws InventoryException {}
        private void processPayment(Order order) throws PaymentException {}
        private void arrangeShipping(Order order) throws ShippingException {}
    }
    
    /**
     * NEW STYLE: Unchecked exceptions
     */
    class NewStyle {
        public void processOrder(Order order) {
            // No throws clause needed
            validateOrder(order);
            reserveInventory(order);
            processPayment(order);
            arrangeShipping(order);
        }
        
        /**
         * Exceptions are unchecked
         */
        private void validateOrder(Order order) {
            if (!order.isValid()) {
                throw new OrderValidationException("Invalid order");
            }
        }
        
        private void reserveInventory(Order order) {
            if (!hasInventory(order)) {
                throw new InsufficientInventoryException("Out of stock");
            }
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ Clean method signatures
         * ✓ Caller handles if needed (not forced)
         * ✓ Easy to refactor
         * ✓ Less boilerplate
         */
        
        private boolean hasInventory(Order order) { return true; }
    }
    
    /**
     * GUIDELINE:
     * 
     * Use UNCHECKED for:
     * • Application exceptions (domain errors)
     * • Most business logic failures
     * • Let caller decide whether to handle
     * 
     * Use CHECKED for:
     * • Library/API public interfaces (document expected failures)
     * • Truly recoverable conditions
     * • When forcing handling adds value
     */
}

// Exception classes
class ValidationException extends Exception {}
class InventoryException extends Exception {}
class ShippingException extends Exception {}
class OrderValidationException extends RuntimeException {
    OrderValidationException(String msg) { super(msg); }
}
class InsufficientInventoryException extends RuntimeException {
    InsufficientInventoryException(String msg) { super(msg); }
}
```

## 4. Null Object Pattern vs Optional

### Representing Absence Safely

```java
/**
 * NULL OBJECT PATTERN VS OPTIONAL
 * 
 * Two ways to represent "no value"
 */

// ========================================
// PROBLEM: NULL REFERENCES
// ========================================

/**
 * ❌ Traditional null handling (error-prone)
 */
class TraditionalNullHandling {
    
    public void processUser(User user) {
        if (user != null) {  // Easy to forget null check
            String name = user.getName();
            if (name != null) {  // More null checks
                System.out.println(name.toUpperCase());
            }
        }
        
        /**
         * PROBLEMS:
         * 
         * ✗ Easy to forget null checks → NullPointerException
         * ✗ Verbose (if-checks everywhere)
         * ✗ null has no meaning (error? missing? uninitialized?)
         */
    }
}

// ========================================
// SOLUTION 1: NULL OBJECT PATTERN
// ========================================

/**
 * Null Object: Real object with "do nothing" behavior
 */
class NullObjectPattern {
    
    /**
     * Logger interface
     */
    interface Logger {
        void log(String message);
        void error(String message);
    }
    
    /**
     * Real logger
     */
    static class ConsoleLogger implements Logger {
        @Override
        public void log(String message) {
            System.out.println("[LOG] " + message);
        }
        
        @Override
        public void error(String message) {
            System.err.println("[ERROR] " + message);
        }
    }
    
    /**
     * Null logger (does nothing)
     */
    static class NullLogger implements Logger {
        @Override
        public void log(String message) {
            // Do nothing
        }
        
        @Override
        public void error(String message) {
            // Do nothing
        }
    }
    
    /**
     * Usage: No null checks needed!
     */
    static class Service {
        private final Logger logger;
        
        public Service(Logger logger) {
            // Use NullLogger if logging disabled
            this.logger = logger != null ? logger : new NullLogger();
        }
        
        public void doWork() {
            logger.log("Starting work");  // ✓ Safe, no null check
            // Do work...
            logger.log("Work complete");  // ✓ Safe, no null check
        }
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ No null checks needed
     * ✓ Polymorphic (Logger interface)
     * ✓ Clean code (no if-statements)
     * 
     * DRAWBACKS:
     * 
     * ✗ Hides absence (can't tell if logging disabled)
     * ✗ Extra class needed (NullLogger)
     * ✗ Behavior might be unexpected
     */
}

// ========================================
// SOLUTION 2: OPTIONAL
// ========================================

/**
 * Optional: Explicit "maybe present, maybe not"
 */
class OptionalPattern {
    
    /**
     * Repository returning Optional
     */
    static class UserRepository {
        
        public Optional<User> findById(String id) {
            // Simulate database lookup
            if (id.equals("123")) {
                return Optional.of(new User("John"));
            }
            return Optional.empty();  // ✓ Explicit: not found
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ Explicit in signature (may return nothing)
         * ✓ Compiler forces handling
         * ✓ No NullPointerException
         */
    }
    
    /**
     * Using Optional
     */
    static class UserService {
        private final UserRepository repository;
        
        public UserService(UserRepository repository) {
            this.repository = repository;
        }
        
        /**
         * Pattern 1: orElse (provide default)
         */
        public String getUserName(String id) {
            return repository.findById(id)
                .map(User::getName)
                .orElse("Unknown");  // Default value
        }
        
        /**
         * Pattern 2: orElseGet (lazy default)
         */
        public User getUser(String id) {
            return repository.findById(id)
                .orElseGet(() -> createGuestUser());  // Lazy computation
        }
        
        /**
         * Pattern 3: orElseThrow (fail if absent)
         */
        public User getUserOrFail(String id) {
            return repository.findById(id)
                .orElseThrow(() -> new UserNotFoundException(id));
        }
        
        /**
         * Pattern 4: ifPresent (action if present)
         */
        public void processUser(String id) {
            repository.findById(id)
                .ifPresent(user -> System.out.println("Found: " + user.getName()));
        }
        
        /**
         * Pattern 5: ifPresentOrElse (both cases)
         */
        public void processOrLog(String id) {
            repository.findById(id)
                .ifPresentOrElse(
                    user -> System.out.println("Found: " + user.getName()),
                    () -> System.out.println("User not found")
                );
        }
        
        private User createGuestUser() {
            return new User("Guest");
        }
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ Type-safe (can't forget to check)
     * ✓ Explicit in API (signals "may be absent")
     * ✓ Functional operations (map, filter, flatMap)
     * ✓ No NullPointerException
     * 
     * DRAWBACKS:
     * 
     * ✗ Verbose (more code than null)
     * ✗ Can't use in fields (not Serializable)
     * ✗ Performance overhead (wrapper object)
     */
}

// NULL OBJECT VS OPTIONAL: WHEN TO USE
// ========================================

/**
 * Choosing between Null Object and Optional
 */
class ChoosingTheRight Approach {
    
    /**
     * USE NULL OBJECT WHEN:
     * 
     * ✓ Behavior-focused (do something or do nothing)
     * ✓ Polymorphic (interface with implementations)
     * ✓ No need to distinguish absence
     * 
     * Examples:
     * • Logging (log or don't log)
     * • Validation (validate or skip)
     * • Strategy pattern (execute or skip)
     */
    
    interface Validator {
        boolean validate(String input);
    }
    
    class RealValidator implements Validator {
        @Override
        public boolean validate(String input) {
            return input != null && input.length() > 5;
        }
    }
    
    class NullValidator implements Validator {
        @Override
        public boolean validate(String input) {
            return true;  // Always valid (no validation)
        }
    }
    
    /**
     * USE OPTIONAL WHEN:
     * 
     * ✓ Value-focused (has value or doesn't)
     * ✓ Return types (method may not find result)
     * ✓ Need to distinguish absence
     * 
     * Examples:
     * • Repository queries (may not find entity)
     * • Parsing (may fail)
     * • Configuration (may not be set)
     */
    
    Optional<String> getConfigValue(String key) {
        // May or may not have value
        return Optional.ofNullable(System.getProperty(key));
    }
    
    Optional<User> findUser(String id) {
        // May or may not find user
        return Optional.empty();
    }
    
    /**
     * DON'T USE OPTIONAL FOR:
     * 
     * ✗ Fields (use null or Null Object)
     * ✗ Method parameters (use overloading or null)
     * ✗ Collections (use empty collection, not Optional<List>)
     */
    
    // ❌ BAD: Optional field
    class BadExample {
        private Optional<String> name;  // ✗ Don't do this
    }
    
    // ✓ GOOD: Nullable field
    class GoodExample {
        private String name;  // ✓ Can be null
    }
    
    // ❌ BAD: Optional parameter
    void processUser(Optional<String> name) {  // ✗ Don't do this
    }
    
    // ✓ GOOD: Overloading
    void processUser(String name) {}
    void processUser() { processUser("Default"); }
    
    // ❌ BAD: Optional collection
    Optional<List<User>> getUsers() {  // ✗ Don't do this
        return Optional.empty();
    }
    
    // ✓ GOOD: Empty collection
    List<User> getUsers() {
        return Collections.emptyList();  // ✓ Use empty collection
    }
}
```

## 5. Try-Success-Failure Pattern

### Functional Error Handling

```java
/**
 * TRY-SUCCESS-FAILURE PATTERN
 * 
 * Functional approach: Result type that's either Success or Failure
 * Alternative to exceptions
 */

// ========================================
// RESULT TYPE
// ========================================

/**
 * Result<T> can be Success<T> or Failure
 */
sealed interface Result<T> permits Success, Failure {
    
    boolean isSuccess();
    boolean isFailure();
    
    T getOrThrow();
    T getOrElse(T defaultValue);
    T getOrElse(Supplier<T> defaultSupplier);
    
    <U> Result<U> map(Function<T, U> mapper);
    <U> Result<U> flatMap(Function<T, Result<U>> mapper);
    
    Result<T> onSuccess(Consumer<T> action);
    Result<T> onFailure(Consumer<Throwable> action);
}

/**
 * Success case
 */
record Success<T>(T value) implements Result<T> {
    
    @Override
    public boolean isSuccess() {
        return true;
    }
    
    @Override
    public boolean isFailure() {
        return false;
    }
    
    @Override
    public T getOrThrow() {
        return value;
    }
    
    @Override
    public T getOrElse(T defaultValue) {
        return value;
    }
    
    @Override
    public T getOrElse(Supplier<T> defaultSupplier) {
        return value;
    }
    
    @Override
    public <U> Result<U> map(Function<T, U> mapper) {
        try {
            return new Success<>(mapper.apply(value));
        } catch (Exception e) {
            return new Failure<>(e);
        }
    }
    
    @Override
    public <U> Result<U> flatMap(Function<T, Result<U>> mapper) {
        try {
            return mapper.apply(value);
        } catch (Exception e) {
            return new Failure<>(e);
        }
    }
    
    @Override
    public Result<T> onSuccess(Consumer<T> action) {
        action.accept(value);
        return this;
    }
    
    @Override
    public Result<T> onFailure(Consumer<Throwable> action) {
        return this;  // Do nothing on success
    }
}

/**
 * Failure case
 */
record Failure<T>(Throwable error) implements Result<T> {
    
    @Override
    public boolean isSuccess() {
        return false;
    }
    
    @Override
    public boolean isFailure() {
        return true;
    }
    
    @Override
    public T getOrThrow() {
        if (error instanceof RuntimeException) {
            throw (RuntimeException) error;
        }
        throw new RuntimeException(error);
    }
    
    @Override
    public T getOrElse(T defaultValue) {
        return defaultValue;
    }
    
    @Override
    public T getOrElse(Supplier<T> defaultSupplier) {
        return defaultSupplier.get();
    }
    
    @Override
    @SuppressWarnings("unchecked")
    public <U> Result<U> map(Function<T, U> mapper) {
        return (Result<U>) this;  // Propagate failure
    }
    
    @Override
    @SuppressWarnings("unchecked")
    public <U> Result<U> flatMap(Function<T, Result<U>> mapper) {
        return (Result<U>) this;  // Propagate failure
    }
    
    @Override
    public Result<T> onSuccess(Consumer<T> action) {
        return this;  // Do nothing on failure
    }
    
    @Override
    public Result<T> onFailure(Consumer<Throwable> action) {
        action.accept(error);
        return this;
    }
}

// ========================================
// USING RESULT TYPE
// ========================================

            return new Failure<>(e);

        this.repository = repository;
    }

            .map(name -> "Processed: " + name);

    public void processWithLogging(String id) {

    public Result<String> updateUserName(String id, String newName) {

                return user;
            })
            .flatMap(repository::save)
            .map(user -> "Updated user: " + user.getName());
    }
}
```

## 6. Circuit Breaker Pattern

### Preventing Cascading Failures

```java
/**
 * CIRCUIT BREAKER PATTERN
 * 
 * Prevent repeated calls to failing service
 * Three states: Closed, Open, Half-Open
 */

// ========================================
// CIRCUIT BREAKER IMPLEMENTATION
// ========================================

/**
 * Simple Circuit Breaker
 */
class CircuitBreaker {
    
    /**
     * Circuit states
     */
    enum State {
        CLOSED,    // Normal operation
        OPEN,      // Failing, reject calls
        HALF_OPEN  // Testing if service recovered
    }
    
    private State state = State.CLOSED;
    private int failureCount = 0;
    private long lastFailureTime = 0;
    
    private final int failureThreshold;
    private final long timeout;
    
    public CircuitBreaker(int failureThreshold, long timeoutMillis) {
        this.failureThreshold = failureThreshold;
        this.timeout = timeoutMillis;
    }
    
    /**
     * Execute operation with circuit breaker
     */
    public <T> T execute(Supplier<T> operation) {
        if (state == State.OPEN) {
            // Check if timeout elapsed
            if (System.currentTimeMillis() - lastFailureTime > timeout) {
                // Try half-open
                state = State.HALF_OPEN;
                System.out.println("Circuit HALF_OPEN, trying recovery");
            } else {
                // Still open, fail fast
                throw new CircuitBreakerOpenException("Circuit breaker is OPEN");
            }
        }
        
        try {
            T result = operation.get();
            onSuccess();
            return result;
            
        } catch (Exception e) {
            onFailure();
            throw e;
        }
    }
    
    private void onSuccess() {
        if (state == State.HALF_OPEN) {
            System.out.println("Circuit CLOSED, service recovered");
            state = State.CLOSED;
        }
        failureCount = 0;
    }
    
    private void onFailure() {
        failureCount++;
        lastFailureTime = System.currentTimeMillis();
        
        if (state == State.HALF_OPEN || failureCount >= failureThreshold) {
            System.out.println("Circuit OPEN, too many failures");
            state = State.OPEN;
        }
    }
    
    public State getState() {
        return state;
    }
}

/**
 * Circuit Breaker Exception
 */
class CircuitBreakerOpenException extends RuntimeException {
    public CircuitBreakerOpenException(String message) {
        super(message);
    }
}

// ========================================
// USING CIRCUIT BREAKER
// ========================================

/**
 * Service with circuit breaker
 */
class PaymentService {
    private final PaymentGateway gateway;
    private final CircuitBreaker circuitBreaker;
    
    public PaymentService(PaymentGateway gateway) {
        this.gateway = gateway;
        this.circuitBreaker = new CircuitBreaker(
            5,          // Open after 5 failures
            60_000      // Try again after 60 seconds
        );
    }
    
    /**
     * Process payment with circuit breaker
     */
    public PaymentResult processPayment(PaymentRequest request) {
        try {
            return circuitBreaker.execute(() -> {
                // Call external payment gateway
                return gateway.charge(request);
            });
            
        } catch (CircuitBreakerOpenException e) {
            // Circuit open - fail fast
            System.err.println("Payment service unavailable: " + e.getMessage());
            return PaymentResult.rejected("Service temporarily unavailable");
            
        } catch (Exception e) {
            // Other errors
            System.err.println("Payment failed: " + e.getMessage());
            return PaymentResult.rejected("Payment processing failed");
        }
    }
}
```

## Summary

```java
/**
 * ERROR HANDLING PATTERNS SUMMARY
 */

// ========================================
// QUICK REFERENCE
// ========================================

/**
 * 1. FAIL FAST VS FAIL SAFE
 */
// Fail Fast: Input validation, development, critical ops
if (amount < 0) throw new IllegalArgumentException("Negative amount");

// Fail Safe: Production, user-facing, optional features
int port = parseInt(config).orElse(DEFAULT_PORT);

/**
 * 2. EXCEPTION TRANSLATION
 */
// At boundaries: SQLException → DataAccessException
try {
    db.query(sql);
} catch (SQLException e) {
    throw new DataAccessException("Query failed", e);
}

/**
 * 3. CHECKED VS UNCHECKED
 */
// Modern: Prefer unchecked (cleaner code)
throw new OrderProcessingException("Failed");  // RuntimeException

// Checked: For truly recoverable conditions
public void readFile(String path) throws IOException

/**
 * 4. NULL OBJECT VS OPTIONAL
 */
// Null Object: Behavior (do something or nothing)
Logger logger = enabled ? new ConsoleLogger() : new NullLogger();

// Optional: Values (has value or doesn't)
Optional<User> user = repository.findById(id);

/**
 * 5. TRY-SUCCESS-FAILURE
 */
// Functional error handling
Result<User> result = repository.findById(id);
String name = result.map(User::getName).getOrElse("Unknown");

/**
 * 6. CIRCUIT BREAKER
 */
// Prevent cascading failures
circuitBreaker.execute(() -> callExternalService());

// ========================================
// DECISION GUIDE
// ========================================

/**
 * WHEN TO FAIL FAST:
 * ✓ Invalid input (IllegalArgumentException)
 * ✓ Preconditions (Objects.requireNonNull)
 * ✓ Development/testing (catch bugs early)
 * 
 * WHEN TO FAIL SAFE:
 * ✓ Production (availability critical)
 * ✓ Optional features (degrade gracefully)
 * ✓ User errors (don't crash app)
 * 
 * 
 * WHEN TO TRANSLATE EXCEPTIONS:
 * ✓ At architectural boundaries
 * ✓ Repository layer (SQL → Domain)
 * ✓ Service layer (Domain → Business)
 * ✓ Controller layer (Business → HTTP)
 * 
 * 
 * WHEN TO USE UNCHECKED:
 * ✓ Application exceptions (default)
 * ✓ Programming errors
 * ✓ Non-recoverable conditions
 * 
 * WHEN TO USE CHECKED:
 * ✓ Library public API
 * ✓ Truly recoverable (file I/O)
 * ✓ Expected failures
 * 
 * 
 * NULL OBJECT:
 * ✓ Polymorphic behavior
 * ✓ No need to distinguish absence
 * ✓ Example: NullLogger, NullValidator
 * 
 * OPTIONAL:
 * ✓ Return values (may not find)
 * ✓ Need to distinguish absence
 * ✓ Don't use: fields, parameters, collections
 * 
 * 
 * CIRCUIT BREAKER:
 * ✓ External services
 * ✓ Network calls
 * ✓ Slow operations
 * ✓ Microservices
 */

// ========================================
// BEST PRACTICES
// ========================================

/**
 * ALWAYS:
 * ✓ Validate inputs (fail fast)
 * ✓ Preserve exception cause
 * ✓ Add context to errors
 * ✓ Log exceptions appropriately
 * 
 * NEVER:
 * ✗ Swallow exceptions (empty catch)
 * ✗ Catch Exception/Throwable
 * ✗ Return null for errors
 * ✗ Use exceptions for control flow
 * 
 * CONSIDER:
 * ⚠ Exception translation at boundaries
 * ⚠ Circuit breaker for external calls
 * ⚠ Optional for optional values
 * ⚠ Result type for functional style
 */
```

**Key Takeaways:**

```
Error Handling Hierarchy:
1. Fail Fast (input validation, critical ops)
2. Exception Translation (at boundaries)
3. Circuit Breaker (external services)
4. Fail Safe (production, user-facing)

Modern Java:
• Prefer unchecked exceptions
• Use Optional for return values
• Translate at architectural boundaries
• Circuit breaker for network calls

The Golden Rule:
Fail fast on programmer errors
Fail safe on external failures
```
