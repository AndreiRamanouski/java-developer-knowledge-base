# Functional Design Patterns

## Overview

Functional patterns solve the fundamental problem of managing complexity through immutability, pure functions, and composability—how to write predictable, testable code that's easy to reason about without side effects, null pointer exceptions, or mutable state bugs. Instead of imperative code with mutations and exceptions (null checks everywhere, defensive copying, unpredictable state changes), functional patterns provide declarative solutions: Monads wrap values in a context (Optional eliminates null, Stream enables pipelines, CompletableFuture handles async), Function Composition builds complex operations from simple ones (andThen, compose), Partial Application and Currying create specialized functions, Immutable Objects prevent defensive copying and race conditions, Persistent Data Structures enable efficient immutable updates (structural sharing), and the Loan Pattern ensures resource cleanup (try-with-resources guarantees close()). These patterns are fundamental to modern Java—Optional replaces null, Streams enable functional pipelines, CompletableFuture powers reactive programming, immutable objects are thread-safe by default, and try-with-resources is standard for resource management. Understanding functional patterns means understanding how to write code that's concurrent by default, easier to test (pure functions), and less prone to bugs. This guide covers Monads (Optional, Stream, CompletableFuture as monadic containers), Function Composition (building pipelines, method references), Partial Application/Currying (creating specialized functions), Immutable Objects (final fields, defensive copying, Records), Persistent Data Structures (structural sharing, copy-on-write), and Loan Pattern (try-with-resources, custom resource management).

## 1. Monad Pattern

### Optional, Stream, CompletableFuture as Monads

```java
/**
 * MONAD PATTERN
 * 
 * Container that wraps a value
 * Provides methods to transform wrapped value
 * Maintains context through transformations
 */

// ========================================
// THE NULL PROBLEM
// ========================================
class TraditionalNullHandling {
    public String getUpperCaseCity(User user) {
        if (user != null) {
            Address address = user.getAddress();
            if (address != null) {
                String city = address.getCity();
                if (city != null) {
                    return city.toUpperCase();
                }
            }
        }
        return "UNKNOWN";
    }
    
    /**
     * PROBLEMS:
     * ✗ Nested null checks
     * ✗ Hard to read
     * ✗ Easy to forget null check
     * ✗ NullPointerException risk
     */
}

// ========================================
// OPTIONAL MONAD
// ========================================
class OptionalMonad {
    
    // Replaces null with Optional
    public String getUpperCaseCity(User user) {
        return Optional.ofNullable(user)
            .map(User::getAddress)        // User → Address
            .map(Address::getCity)        // Address → String
            .map(String::toUpperCase)     // String → String
            .orElse("UNKNOWN");           // Default value
    }
    
    /**
     * MONAD OPERATIONS:
     * 
     * of/ofNullable - Wrap value (unit/return)
     * map           - Transform value
     * flatMap       - Transform to another Optional (bind)
     * orElse        - Extract with default
     * 
     * 
     * BENEFITS:
     * 
     * ✓ No null checks
     * ✓ Chainable
     * ✓ Explicit absence
     * ✓ Type-safe
     */
    
    // More examples
    public void optionalExamples() {
        // Creating Optional
        Optional<String> present = Optional.of("Hello");
        Optional<String> nullable = Optional.ofNullable(null);
        Optional<String> empty = Optional.empty();
        
        // map - Transform value
        Optional<Integer> length = present.map(String::length);  // Optional[5]
        
        // flatMap - Transform to Optional (avoid Optional<Optional<T>>)
        Optional<String> name = Optional.of("john")
            .flatMap(this::findUserById)
            .map(User::getName);
        
        // filter - Keep if predicate matches
        Optional<String> longName = present.filter(s -> s.length() > 10);
        
        // orElse - Default value
        String result = empty.orElse("default");
        
        // orElseGet - Lazy default (computed only if empty)
        String result2 = empty.orElseGet(() -> expensiveComputation());
        
        // orElseThrow - Throw exception if empty
        String result3 = empty.orElseThrow(() -> 
            new IllegalStateException("Value required"));
        
        // ifPresent - Execute if present
        present.ifPresent(System.out::println);
        
        // ifPresentOrElse (Java 9+)
        present.ifPresentOrElse(
            System.out::println,
            () -> System.out.println("Empty")
        );
    }
    
    private Optional<User> findUserById(String id) {
        // Database lookup
        return Optional.empty();
    }
    
    private String expensiveComputation() {
        return "computed";
    }
}

// STREAM MONAD
// ========================================
class StreamMonad {
    
    /**
     * Stream is a monad over collections
     * Represents sequence of values
     * Lazy evaluation (operations only run on terminal operation)
     */
    
    public void streamExamples() {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
        
        // map - Transform each element
        List<String> strings = numbers.stream()
            .map(n -> "Number: " + n)
            .collect(Collectors.toList());
        
        // flatMap - Transform to stream and flatten
        List<String> words = Arrays.asList("Hello World", "Java Stream");
        List<String> allWords = words.stream()
            .flatMap(line -> Arrays.stream(line.split(" ")))
            .collect(Collectors.toList());
        // ["Hello", "World", "Java", "Stream"]
        
        // filter - Keep elements matching predicate
        List<Integer> evens = numbers.stream()
            .filter(n -> n % 2 == 0)
            .collect(Collectors.toList());
        
        // reduce - Combine elements
        int sum = numbers.stream()
            .reduce(0, Integer::sum);  // 15
        
        // Chain operations
        int result = numbers.stream()
            .filter(n -> n > 2)       // [3, 4, 5]
            .map(n -> n * 2)          // [6, 8, 10]
            .reduce(0, Integer::sum); // 24
    }
    
    // Real-world: Process orders
    public double calculateTotalRevenue(List<Order> orders) {
        return orders.stream()
            .filter(order -> order.getStatus() == Status.COMPLETED)
            .flatMap(order -> order.getItems().stream())
            .mapToDouble(item -> item.getPrice() * item.getQuantity())
            .sum();
    }
}

class CompletableFutureMonad {
    
    /**
     * CompletableFuture is a monad over asynchronous computations
     * Represents value that may not be available yet
     */
    
    public void completableFutureExamples() {
        // Create CompletableFuture
        CompletableFuture<String> future = 
            CompletableFuture.supplyAsync(() -> "Hello");
        
        // thenApply - map (transform result)
        CompletableFuture<Integer> length = future
            .thenApply(String::length);
        
        // thenCompose - flatMap (chain async operations)
        CompletableFuture<User> userFuture = 
            CompletableFuture.supplyAsync(() -> "userId")
            .thenCompose(this::fetchUserAsync);  // Avoids CompletableFuture<CompletableFuture<User>>
        
        // thenAccept - consume result
        future.thenAccept(System.out::println);
        
        // exceptionally - handle errors
        CompletableFuture<String> withFallback = future
            .exceptionally(ex -> "Error: " + ex.getMessage());
        
        // thenCombine - combine two futures
        CompletableFuture<String> future1 = 
            CompletableFuture.supplyAsync(() -> "Hello");
        CompletableFuture<String> future2 = 
            CompletableFuture.supplyAsync(() -> "World");
        
        CompletableFuture<String> combined = future1.thenCombine(
            future2,
            (s1, s2) -> s1 + " " + s2
        );
    }
    
    // Real-world: Async API calls
    public CompletableFuture<UserProfile> getUserProfile(String userId) {
        return fetchUserAsync(userId)
            .thenCompose(user -> 
                fetchOrdersAsync(userId)
                    .thenApply(orders -> new UserProfile(user, orders))
            );
    }
    
    private CompletableFuture<User> fetchUserAsync(String userId) {
        return CompletableFuture.supplyAsync(() -> {
            // API call
            return new User(userId, null, null);
        });
    }
    
    private CompletableFuture<List<Order>> fetchOrdersAsync(String userId) {
        return CompletableFuture.supplyAsync(() -> {
            // API call
            return List.of();
        });
    }
}

 * All monads must satisfy three laws:
 * 
 * 1. LEFT IDENTITY
 *    Optional.of(a).flatMap(f) ≡ f(a)
 * 
 * 2. RIGHT IDENTITY
 *    m.flatMap(Optional::of) ≡ m

 *    m.flatMap(f).flatMap(g) ≡ m.flatMap(x -> f(x).flatMap(g))

 *

 */
```

## 2. Function Composition

### Building Complex Operations from Simple Functions

```java
/**
 * FUNCTION COMPOSITION
 * 
 * Combine simple functions to create complex operations
 * f(g(x)) = (f ∘ g)(x)
 */

import java.util.function.*;

// ========================================
// BASIC FUNCTION COMPOSITION
// ========================================
class FunctionCompositionBasics {
    
    public void basicComposition() {
        // Simple functions
        Function<String, String> trim = String::trim;
        Function<String, String> lowercase = String::toLowerCase;
        Function<String, String> removeSpaces = s -> s.replaceAll("\\s+", "");
        
        // Compose with andThen (f.andThen(g) = g(f(x)))
        Function<String, String> normalize = trim
            .andThen(lowercase)
            .andThen(removeSpaces);
        
        String result = normalize.apply("  Hello World  ");
        // Result: "helloworld"
        
        // Compose with compose (f.compose(g) = f(g(x)))
        Function<String, Integer> parseToInt = Integer::parseInt;
        Function<Integer, Integer> square = x -> x * x;
        
        Function<String, Integer> parseAndSquare = square.compose(parseToInt);
        // parseAndSquare("5") = square(parseInt("5")) = 25
    }
    
    // andThen vs compose
    public void differenceExample() {
        Function<Integer, Integer> addOne = x -> x + 1;
        Function<Integer, Integer> multiplyTwo = x -> x * 2;
        
        // andThen: left to right
        Function<Integer, Integer> f1 = addOne.andThen(multiplyTwo);
        System.out.println(f1.apply(3));  // (3 + 1) * 2 = 8
        
        // compose: right to left
        Function<Integer, Integer> f2 = addOne.compose(multiplyTwo);
        System.out.println(f2.apply(3));  // (3 * 2) + 1 = 7
    }
}

// ========================================
// PREDICATE COMPOSITION
// ========================================
class PredicateComposition {
    
    public void predicateExamples() {
        Predicate<Integer> isPositive = x -> x > 0;
        Predicate<Integer> isEven = x -> x % 2 == 0;
        Predicate<Integer> lessThan100 = x -> x < 100;
        
        // and - Both must be true
        Predicate<Integer> positiveEven = isPositive.and(isEven);
        System.out.println(positiveEven.test(4));   // true
        System.out.println(positiveEven.test(-4));  // false
        
        // or - At least one must be true
        Predicate<Integer> positiveOrEven = isPositive.or(isEven);
        System.out.println(positiveOrEven.test(-4));  // true (even)
        System.out.println(positiveOrEven.test(3));   // true (positive)
        
        // negate - Logical NOT
        Predicate<Integer> notPositive = isPositive.negate();
        System.out.println(notPositive.test(-5));  // true
        
        // Complex composition
        Predicate<Integer> complexPredicate = isPositive
            .and(isEven)
            .and(lessThan100);
        
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 50, 200);
        List<Integer> filtered = numbers.stream()
            .filter(complexPredicate)
            .collect(Collectors.toList());
        // [2, 4, 50]
    }
}

// REAL-WORLD: VALIDATION PIPELINE
// ========================================
class ValidationPipeline {
    
    // Individual validators
    private static final Predicate<String> notEmpty = 
        s -> s != null && !s.isEmpty();
    
    private static final Predicate<String> minLength = 
        s -> s.length() >= 8;
    
    private static final Predicate<String> hasDigit = 
        s -> s.matches(".*\\d.*");
    
    private static final Predicate<String> hasUpperCase = 
        s -> s.matches(".*[A-Z].*");
    
    private static final Predicate<String> hasLowerCase = 
        s -> s.matches(".*[a-z].*");
    
    // Compose validators
    private static final Predicate<String> isValidPassword = notEmpty
        .and(minLength)
        .and(hasDigit)
        .and(hasUpperCase)
        .and(hasLowerCase);
    
    public boolean validatePassword(String password) {
        return isValidPassword.test(password);
    }
    
    // With detailed error messages
    public List<String> getPasswordErrors(String password) {
        List<String> errors = new ArrayList<>();
        
        if (!notEmpty.test(password)) {
            errors.add("Password cannot be empty");
        }
        if (!minLength.test(password)) {
            errors.add("Password must be at least 8 characters");
        }
        if (!hasDigit.test(password)) {
            errors.add("Password must contain a digit");
        }
        if (!hasUpperCase.test(password)) {
            errors.add("Password must contain uppercase letter");
        }
        if (!hasLowerCase.test(password)) {
            errors.add("Password must contain lowercase letter");
        }
        
        return errors;
    }
}

// ========================================
// REAL-WORLD: DATA TRANSFORMATION PIPELINE
// ========================================
class DataTransformationPipeline {
    
    public List<String> processUsers(List<User> users) {
        // Compose transformation functions
        Function<User, String> getName = User::getName;
        Function<String, String> trim = String::trim;
        Function<String, String> uppercase = String::toUpperCase;
        Function<String, String> addPrefix = name -> "USER: " + name;
        
        // Build complete transformation
        Function<User, String> transformation = getName
            .andThen(trim)
            .andThen(uppercase)
            .andThen(addPrefix);
        
        return users.stream()
            .map(transformation)
            .collect(Collectors.toList());
    }
    
    // Price calculation pipeline
    public double calculateFinalPrice(Product product, Customer customer) {
        Function<Product, Double> getBasePrice = Product::getPrice;
        Function<Double, Double> applyTax = price -> price * 1.2;
        Function<Double, Double> applyDiscount = 
            customer.isPremium() ? price -> price * 0.9 : Function.identity();
        Function<Double, Double> applyShipping = price -> price + 5.0;
        
        Function<Product, Double> pricePipeline = getBasePrice
            .andThen(applyTax)
            .andThen(applyDiscount)
            .andThen(applyShipping);
        
        return pricePipeline.apply(product);
    }
}

class Product {
    double price;
    double getPrice() { return price; }
}
class Customer {
    boolean premium;
    boolean isPremium() { return premium; }
}
```

## 3. Partial Application and Currying

### Creating Specialized Functions

```java
/**
 * PARTIAL APPLICATION & CURRYING
 * 
 * Partial Application: Fix some arguments, create new function
 * Currying: Transform multi-argument function into chain of single-argument functions
 */

// ========================================
// PARTIAL APPLICATION
// ========================================
class PartialApplication {
    
    // Generic partial application helper
    static <A, B, R> Function<B, R> partial(
            BiFunction<A, B, R> function, 
            A fixedArg) {
        return b -> function.apply(fixedArg, b);
    }
    
    public void partialExample() {
        // Original function (multiply two numbers)
        BiFunction<Integer, Integer, Integer> multiply = (a, b) -> a * b;
        
        // Partially apply first argument
        Function<Integer, Integer> multiplyBy10 = partial(multiply, 10);
        Function<Integer, Integer> multiplyBy5 = partial(multiply, 5);
        
        System.out.println(multiplyBy10.apply(3));  // 30
        System.out.println(multiplyBy5.apply(3));   // 15
    }
    
    // Real-world: Database query builder
    static class QueryBuilder {
        BiFunction<String, Map<String, Object>, List<User>> executeQuery = 
            (sql, params) -> {
                // Execute SQL with parameters
                return List.of();
            };
        
        // Partially apply SQL query
        Function<Map<String, Object>, List<User>> findUsersByAge = 
            partial(executeQuery, "SELECT * FROM users WHERE age = :age");
        
        Function<Map<String, Object>, List<User>> findUsersByCity = 
            partial(executeQuery, "SELECT * FROM users WHERE city = :city");
        
        public void example() {
            Map<String, Object> params = Map.of("age", 25);
            List<User> users = findUsersByAge.apply(params);
        }
    }
}

// ========================================
// CURRYING
// ========================================
class Currying {
    
    // Manual currying
    Function<Integer, Function<Integer, Integer>> curriedAdd = 
        a -> b -> a + b;
    
    public void curriedExample() {
        // Use curried function
        Function<Integer, Integer> add5 = curriedAdd.apply(5);
        System.out.println(add5.apply(3));  // 8
        
        // Or directly
        System.out.println(curriedAdd.apply(5).apply(3));  // 8
    }
    
    // Curry 3-argument function
    Function<Integer, Function<Integer, Function<Integer, Integer>>> curriedMultiply =
        a -> b -> c -> a * b * c;
    
    public void threeLevelCurrying() {
        Function<Integer, Function<Integer, Integer>> multiplyBy2 = 
            curriedMultiply.apply(2);
        
        Function<Integer, Integer> multiplyBy2And3 = 
            multiplyBy2.apply(3);
        
        System.out.println(multiplyBy2And3.apply(4));  // 24 (2 * 3 * 4)
    }
    
    // Generic curry helper
    static <A, B, C> Function<A, Function<B, C>> curry(BiFunction<A, B, C> function) {
        return a -> b -> function.apply(a, b);
    }
    
    public void genericCurryExample() {
        BiFunction<String, Integer, String> repeat = 
            (str, times) -> str.repeat(times);
        
        Function<String, Function<Integer, String>> curriedRepeat = curry(repeat);
        
        Function<Integer, String> repeatHello = curriedRepeat.apply("Hello");
        System.out.println(repeatHello.apply(3));  // HelloHelloHello
    }
}

class ConfigurableLogger {
    
    // Base logging function
    TriConsumer<String, String, String> log = (level, category, message) -> {

    };
    
    // Partially apply level
    BiConsumer<String, String> error = (category, message) ->

        log.accept("INFO", category, message);
    
    // Partially apply level and category
    Consumer<String> databaseError = message ->

    Consumer<String> apiInfo = message -> 
        log.accept("INFO", "API", message);

        error.accept("API", "Connection failed");
        // [ERROR][API] Connection failed
        
        databaseError.accept("Query timeout");
        // [ERROR][DATABASE] Query timeout

        // [INFO][API] Request received
    }
```

## 4. Immutable Objects Pattern

### Thread-Safe, Predictable Objects

```java
/**
 * IMMUTABLE OBJECTS PATTERN
 * 
 * Objects whose state cannot change after construction
 * Thread-safe, cacheable, predictable
 */

// ========================================
// IMMUTABLE CLASS (CLASSIC APPROACH)
// ========================================
final class ImmutablePerson {
    private final String name;
    private final int age;
    private final List<String> hobbies;
    
    public ImmutablePerson(String name, int age, List<String> hobbies) {
        this.name = name;
        this.age = age;
        // Defensive copy!
        this.hobbies = List.copyOf(hobbies);
    }
    
    public String getName() { return name; }
    public int getAge() { return age; }
    
    // Return unmodifiable view
    public List<String> getHobbies() {
        return hobbies;  // Already unmodifiable from List.copyOf
    }
    
    // Modification creates new instance
    public ImmutablePerson withAge(int newAge) {
        return new ImmutablePerson(this.name, newAge, this.hobbies);
    }
    
    public ImmutablePerson addHobby(String hobby) {
        List<String> newHobbies = new ArrayList<>(this.hobbies);
        newHobbies.add(hobby);
        return new ImmutablePerson(this.name, this.age, newHobbies);
    }
    
    /**
     * IMMUTABILITY CHECKLIST:
     * 
     * ✓ final class (prevent subclassing)
     * ✓ All fields private final
     * ✓ No setters
     * ✓ Defensive copy in constructor
     * ✓ Return unmodifiable collections
     * ✓ "with" methods return new instance
     */
}

// ========================================
// RECORD (JAVA 14+) - BEST APPROACH
// ========================================
record Person(String name, int age, List<String> hobbies) {
    
    // Compact constructor for validation
    public Person {
        if (name == null || name.isEmpty()) {
            throw new IllegalArgumentException("Name required");
        }
        if (age < 0) {
            throw new IllegalArgumentException("Age must be positive");
        }
        // Defensive copy
        hobbies = List.copyOf(hobbies);
    }
    
    // "with" methods
    public Person withAge(int newAge) {
        return new Person(name, newAge, hobbies);
    }
    
    public Person addHobby(String hobby) {
        List<String> newHobbies = new ArrayList<>(hobbies);
        newHobbies.add(hobby);
        return new Person(name, newAge, newHobbies);
    }
    
    /**
     * Records provide:
     * ✓ final class automatically
     * ✓ private final fields automatically
     * ✓ Constructor, getters, equals, hashCode, toString
     * ✓ Much less boilerplate!
     */
}

// ========================================
// BENEFITS OF IMMUTABILITY
// ========================================
class ImmutabilityBenefits {
    
    // 1. Thread-safe (no synchronization needed)
    public void threadSafeExample() {
        ImmutablePerson person = new ImmutablePerson("John", 30, List.of("reading"));
        
        // Safe to share across threads
        ExecutorService executor = Executors.newFixedThreadPool(10);
        for (int i = 0; i < 10; i++) {
            executor.submit(() -> {
                System.out.println(person.getName());  // No race condition!
            });
        }
        executor.shutdown();
    }
    
    // 2. Cacheable (safe to reuse)
    private static final Map<String, ImmutablePerson> cache = new ConcurrentHashMap<>();
    
    public ImmutablePerson getCachedPerson(String name) {
        return cache.computeIfAbsent(name, 
            n -> new ImmutablePerson(n, 0, List.of()));
    }
    
    // 3. No defensive copying needed
    public void noDefensiveCopy(ImmutablePerson person) {
        // Safe to store reference directly
        this.person = person;  // No need to copy!
    }
    private ImmutablePerson person;
    
    // 4. Predictable (no surprising mutations)
    public void predictableExample() {
        ImmutablePerson p1 = new ImmutablePerson("John", 30, List.of());
        ImmutablePerson p2 = p1.withAge(31);  // Returns NEW instance
        
        System.out.println(p1.getAge());  // 30 (unchanged!)
        System.out.println(p2.getAge());  // 31
    }
}

final class ImmutableUser {

    public static class Builder {
        private final String username;  // Required
        private final String email;     // Required

        private String lastName = "";
        private int age = 0;
        
        public Builder(String username, String email) {
            this.username = username;
            this.email = email;
```

## 5. Persistent Data Structures

### Efficient Immutable Updates with Structural Sharing

```java
/**
 * PERSISTENT DATA STRUCTURES
 * 
 * Immutable data structures that share structure
 * Efficient updates (O(log n) instead of O(n))
 */

// ========================================
// PROBLEM: NAIVE IMMUTABLE LISTS
// ========================================
class NaiveImmutableList {
    
    public List<Integer> add(List<Integer> list, Integer element) {
        List<Integer> newList = new ArrayList<>(list);  // Copy entire list!
        newList.add(element);
        return List.copyOf(newList);
    }
    
    /**
     * PROBLEM:
     * ✗ O(n) time for each modification
     * ✗ O(n) space for each modification
     * ✗ 1000 modifications = 1000 full copies!
     */
}

// ========================================
// PERSISTENT LIST (STRUCTURAL SHARING)
// ========================================

/**
 * Linked list where nodes are shared between versions
 * 
 * Original: 1 -> 2 -> 3 -> null
 * After adding 0:
 * New:      0 -> 1 -> 2 -> 3 -> null
 *                ↑
 *           Shared tail!
 */
class PersistentList<T> {
    private final Node<T> head;
    private final int size;
    
    private static class Node<T> {
        final T value;
        final Node<T> next;
        
        Node(T value, Node<T> next) {
            this.value = value;
            this.next = next;
        }
    }
    
    // Empty list
    public PersistentList() {
        this.head = null;
        this.size = 0;
    }
    
    private PersistentList(Node<T> head, int size) {
        this.head = head;
        this.size = size;
    }
    
    // Add to front (O(1) - shares tail!)
    public PersistentList<T> add(T element) {
        Node<T> newHead = new Node<>(element, head);
        return new PersistentList<>(newHead, size + 1);
    }
    
    // Get head (O(1))
    public T head() {
        if (head == null) throw new NoSuchElementException();
        return head.value;
    }
    
    // Get tail (O(1) - shares structure!)
    public PersistentList<T> tail() {
        if (head == null) throw new NoSuchElementException();
        return new PersistentList<>(head.next, size - 1);
    }
    
    public int size() {
        return size;
    }
    
    public boolean isEmpty() {
        return head == null;
    }
}

/**
 * USAGE:
 * 
 * PersistentList<Integer> list1 = new PersistentList<>();
 * PersistentList<Integer> list2 = list1.add(1);  // [1]
 * PersistentList<Integer> list3 = list2.add(2);  // [2, 1]
 * PersistentList<Integer> list4 = list2.add(3);  // [3, 1]
 * 
 * All lists share nodes!
 * list2, list3, list4 all share the node containing 1
 * 
 * 
 * BENEFITS:
 * 
 * ✓ O(1) add to front
 * ✓ O(1) space per modification
 * ✓ All versions accessible
 * ✓ Thread-safe (immutable)
 */

// ========================================
// PERSISTENT VECTOR (INDEXED ACCESS)
// ========================================

/**
 * Tree-based structure for efficient indexed access
 * Used by Clojure, Scala
 * 
 * 32-way branching tree
 * O(log32 n) ≈ O(1) for practical sizes
 */
class SimplePersistentVector<T> {
    private static final int BRANCHING_FACTOR = 32;
    
    private final Object[] root;
    private final int size;
    private final int depth;
    
    public SimplePersistentVector() {
        this.root = new Object[BRANCHING_FACTOR];
        this.size = 0;
        this.depth = 0;
    }
    
    private SimplePersistentVector(Object[] root, int size, int depth) {
        this.root = root;
        this.size = size;
        this.depth = depth;
    }
    
    // Simplified add (full implementation more complex)
    public SimplePersistentVector<T> add(T element) {
        // Create new root with shared structure
        Object[] newRoot = root.clone();
        // Insert element (simplified)
        return new SimplePersistentVector<>(newRoot, size + 1, depth);
    }
    
    /**
     * ACTUAL COMPLEXITY:
     * 
     * add:  O(log32 n) ≈ O(1) practical
     * get:  O(log32 n) ≈ O(1) practical
     * set:  O(log32 n) ≈ O(1) practical
     * 
     * Space per modification: O(log32 n) nodes copied
     * 
     * For 1 million elements:
     * log32(1,000,000) ≈ 4 levels
     * Only 4 nodes copied per modification!
     */
}

// COPY-ON-WRITE (JAVA CONCURRENCY)
// ========================================
class CopyOnWriteExample {
    
    /**
     * CopyOnWriteArrayList - Thread-safe persistent list
     * 
     * Every modification creates new copy
     * Reads require no locking
     * Good for read-heavy workloads
     */
    public void copyOnWriteExample() {
        CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
        
        list.add("a");  // Creates new internal array
        list.add("b");  // Creates new internal array
        
        // Iterators see snapshot (no ConcurrentModificationException)
        Iterator<String> it = list.iterator();
        list.add("c");  // Iterator still sees ["a", "b"]
        
        while (it.hasNext()) {
            System.out.println(it.next());  // a, b
        }
    }
    
    /**
     * WHEN TO USE:
     * 
     * ✓ Read-heavy workloads (90%+ reads)
     * ✓ Small to medium lists
     * ✓ Thread-safety required
     * 
     * ✗ Write-heavy workloads (O(n) per write)
     * ✗ Large lists (expensive copies)
     */
}

// ========================================
// REAL-WORLD: UNDO/REDO WITH PERSISTENT DATA
// ========================================
class TextEditor {
    private PersistentList<String> history;
    private String currentText;
    
    public TextEditor() {
        this.history = new PersistentList<>();
        this.currentText = "";
    }
    
    private TextEditor(PersistentList<String> history, String currentText) {
        this.history = history;
        this.currentText = currentText;
    }
    
    // Edit text (saves to history)
    public TextEditor edit(String newText) {
        PersistentList<String> newHistory = history.add(currentText);
        return new TextEditor(newHistory, newText);
    }
    
    // Undo (restore previous state)
    public TextEditor undo() {
        if (history.isEmpty()) {
            return this;
        }
        String previousText = history.head();
        PersistentList<String> previousHistory = history.tail();
        return new TextEditor(previousHistory, previousText);
    }
    
    public String getText() {
        return currentText;
    }
}

/**
 * USAGE:
 * 
 * TextEditor editor1 = new TextEditor();
 * TextEditor editor2 = editor1.edit("Hello");
 * TextEditor editor3 = editor2.edit("Hello World");
 * TextEditor editor4 = editor3.undo();  // Back to "Hello"
 * 
 * All versions share history nodes!
 * Efficient undo/redo
 */
```

## 6. Loan Pattern (Try-With-Resources)

### Automatic Resource Management

```java
/**
 * LOAN PATTERN
 * 
 * "Loans" resource to code block
 * Guarantees cleanup (even if exception thrown)
 */

// ========================================
// PROBLEM: MANUAL RESOURCE MANAGEMENT
// ========================================
class ManualResourceManagement {
    
    public void badExample() {
        FileInputStream fis = null;
        try {
            fis = new FileInputStream("file.txt");
            // Read file...
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            if (fis != null) {
                try {
                    fis.close();  // ❌ Can throw exception!
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
    
    /**
     * PROBLEMS:
     * ✗ Verbose
     * ✗ Easy to forget close()
     * ✗ Exception handling complex
     * ✗ Resource leaks
     */
}

// ========================================
// TRY-WITH-RESOURCES (JAVA 7+)
// ========================================
class TryWithResourcesExample {
    
    public void readFile() {
        try (FileInputStream fis = new FileInputStream("file.txt")) {
            // Read file...
            int data = fis.read();
        } catch (IOException e) {
            e.printStackTrace();
        }
        // fis.close() called automatically!
    }
    
    // Multiple resources
    public void multipleResources() {
        try (FileInputStream fis = new FileInputStream("input.txt");
             FileOutputStream fos = new FileOutputStream("output.txt")) {
            
            // Copy file
            byte[] buffer = new byte[1024];
            int bytesRead;
            while ((bytesRead = fis.read(buffer)) != -1) {
                fos.write(buffer, 0, bytesRead);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        // Both fis and fos closed automatically (reverse order)
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ Automatic cleanup
     * ✓ Exception-safe
     * ✓ Less boilerplate
     * ✓ Resources closed in reverse order
     */
}

// ========================================
// AUTOCLOSEABLE INTERFACE
// ========================================

/**
 * Any class implementing AutoCloseable can be used with try-with-resources
 */
class DatabaseConnection implements AutoCloseable {
    private Connection connection;
    
    public DatabaseConnection(String url) throws SQLException {
        this.connection = DriverManager.getConnection(url);
        System.out.println("Connection opened");
    }
    
    public void executeQuery(String sql) throws SQLException {
        try (Statement stmt = connection.createStatement()) {
            stmt.execute(sql);
        }
    }
    
    @Override
    public void close() throws SQLException {
        if (connection != null && !connection.isClosed()) {
            connection.close();
            System.out.println("Connection closed");
        }
    }
}

/**
 * USAGE:
 * 
 * try (DatabaseConnection db = new DatabaseConnection("jdbc:...")) {
 *     db.executeQuery("SELECT * FROM users");
 * }
 * // db.close() called automatically
 */

// ========================================
// CUSTOM LOAN PATTERN
// ========================================

/**
 * For resources that don't implement AutoCloseable
 * Or for more control over resource management
 */
class LoanPattern {
    
    // Generic loan method
    public static <T, R> R withResource(
            Supplier<T> resourceFactory,
            Function<T, R> operation,
            Consumer<T> cleanup) {
        
        T resource = resourceFactory.get();
        try {
            return operation.apply(resource);
        } finally {
            cleanup.accept(resource);
        }
    }
    
    // Specialized lock loan
    public static <R> R withLock(Lock lock, Supplier<R> operation) {
        lock.lock();
        try {
            return operation.get();
        } finally {
            lock.unlock();
        }
    }
    
    // Timer loan (measures execution time)
    public static <R> R timed(Supplier<R> operation) {
        long start = System.nanoTime();
        try {
            return operation.get();
        } finally {
            long elapsed = System.nanoTime() - start;
            System.out.println("Elapsed: " + elapsed / 1_000_000 + "ms");
        }
    }
}

/**
 * USAGE:
 * 
 * // Lock example
 * Lock lock = new ReentrantLock();
 * String result = LoanPattern.withLock(lock, () -> {
 *     // Critical section
 *     return "computed value";
 * });
 * 
 * // Timer example
 * int result = LoanPattern.timed(() -> {
 *     // Long computation
 *     return 42;
 * });
 * // Output: Elapsed: 1234ms
 */

// ========================================
// REAL-WORLD: TRANSACTION MANAGEMENT
// ========================================
class TransactionManager {
    
    public <R> R inTransaction(Function<Connection, R> operation) {
        try (Connection conn = getConnection()) {
            try {
                conn.setAutoCommit(false);
                
                R result = operation.apply(conn);
                
                conn.commit();
                return result;
                
            } catch (Exception e) {
                conn.rollback();
                throw new RuntimeException("Transaction failed", e);
            }
        } catch (SQLException e) {
            throw new RuntimeException("Connection failed", e);
        }
    }
    
    private Connection getConnection() throws SQLException {
        return DriverManager.getConnection("jdbc:...");
    }
}

/**
 * USAGE:
 * 
 * TransactionManager tm = new TransactionManager();
 * 
 * User user = tm.inTransaction(conn -> {
 *     // All database operations in transaction
 *     insertUser(conn, "John");
 *     insertOrder(conn, "ORD-123");
 *     return findUser(conn, "John");
 * });
 * // Automatic commit or rollback
 */

// ========================================
// SPRING @TRANSACTIONAL (DECLARATIVE LOAN)
// ========================================

/**
 * Spring uses aspect-oriented programming (AOP)
 * to implement loan pattern declaratively
 */
class SpringTransactional {
    
    @Transactional
    public void createOrder(Order order) {
        // Transaction started automatically
        orderRepository.save(order);
        inventoryService.decrementStock(order.getProductId());
        // Transaction committed automatically
        // Or rolled back if exception thrown
    }
    
    /**
     * SPRING HANDLES:
     * 
     * 1. Begin transaction
     * 2. Execute method
     * 3. Commit on success
     * 4. Rollback on exception
     * 5. Close connection
     * 
     * All automatic!
     */
}
```

## Summary

### Quick Reference

**Functional Patterns:**

```
1. Monad: Wrap values in context
   - Optional (handle null)
   - Stream (collections)
   - CompletableFuture (async)
   - Laws: left identity, right identity, associativity

2. Function Composition: Build complex from simple
   - andThen (left to right)
   - compose (right to left)
   - Predicate composition (and, or, negate)

3. Partial Application/Currying: Specialize functions
   - Partial: Fix some arguments
   - Currying: Chain of single-argument functions
   - Use: Configuration, specialized functions

4. Immutable Objects: Thread-safe, predictable
   - final class, final fields
   - No setters, "with" methods
   - Records (Java 14+) - best approach

5. Persistent Data Structures: Efficient immutable updates
   - Structural sharing
   - O(log n) instead of O(n)
   - Undo/redo, time-travel debugging

6. Loan Pattern: Automatic resource cleanup
   - try-with-resources (AutoCloseable)
   - Custom loan methods
   - Transaction management
```

**Pattern Comparison:**

```
┌─────────────────┬──────────────────────┬──────────────────┐
│ Pattern         │ Problem Solved       │ Key Benefit      │
├─────────────────┼──────────────────────┼──────────────────┤
│ Monad           │ Null, exceptions     │ Safe chaining    │
│ Composition     │ Complex operations   │ Reusability      │
│ Currying        │ Function reuse       │ Specialization   │
│ Immutable       │ Shared mutable state │ Thread-safe      │
│ Persistent DS   │ Expensive copies     │ Efficient immut. │
│ Loan Pattern    │ Resource leaks       │ Auto cleanup     │
└─────────────────┴──────────────────────┴──────────────────┘
```

**Decision Guide:**

```
Need to handle null safely?
→ Optional monad

Need to transform collections?
→ Stream monad

Need async computation?
→ CompletableFuture monad

Need to compose operations?
→ Function composition (andThen, compose)

Need specialized functions?
→ Partial application or currying

Need thread-safe objects?
→ Immutable objects (Records)

Need efficient immutable updates?
→ Persistent data structures

Need automatic resource cleanup?
→ Loan pattern (try-with-resources)
```

**Common Mistakes:**

```java
// MISTAKE 1: Not using Optional
String city = user.getAddress().getCity();  // ❌ NPE!
// FIX: Optional chaining

// MISTAKE 2: Mutating "immutable" collections
List<String> list = List.of("a", "b");
List<String> copy = list;  // ❌ Both point to same list!
// FIX: Understand immutability

// MISTAKE 3: Forgetting to close resources
FileInputStream fis = new FileInputStream("file.txt");
// ... use fis
// ❌ Never closed!
// FIX: try-with-resources

// MISTAKE 4: Defensive copy missing
public List<String> getHobbies() {
    return hobbies;  // ❌ Caller can modify!
}
// FIX: Return unmodifiable view

// MISTAKE 5: Blocking in Stream operations
list.stream()
    .map(id -> blockingApiCall(id))  // ❌ Sequential blocking!
    .collect(Collectors.toList());
// FIX: CompletableFuture for async
```

```
Currying:      Function specialization
```
