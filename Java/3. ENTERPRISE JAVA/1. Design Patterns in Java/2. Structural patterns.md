# Structural Design Patterns

## Overview

Structural patterns solve the fundamental problem of object composition—how objects and classes are assembled to form larger structures while keeping the system flexible and efficient. Instead of rigid inheritance hierarchies (inflexible, brittle, deep coupling), structural patterns provide flexible composition mechanisms: Adapter translates incompatible interfaces (integrate third-party libraries, legacy code), Decorator adds responsibilities dynamically (Java I/O streams stack behaviors: BufferedReader wraps FileReader), Proxy controls access to objects (lazy loading, security, caching, JDK dynamic proxies), Composite treats individual objects and compositions uniformly (file systems: files and folders), Bridge separates abstraction from implementation (remote controls work with any device), and Facade simplifies complex subsystems (single interface to complex libraries). These patterns are everywhere in Java—the entire java.io package is built on Decorator (InputStream → BufferedInputStream → DataInputStream), Spring's AOP uses dynamic proxies for cross-cutting concerns, Swing uses Composite for UI components, and JDBC's DriverManager is a Facade over multiple database drivers. Understanding structural patterns means understanding how to build flexible, maintainable systems that adapt to change without massive rewrites. This guide covers Adapter (class vs object adapter, real integrations), Decorator (I/O streams, dynamic behavior stacking), Proxy (static, JDK dynamic, CGLIB proxies, use cases), Composite (tree structures, file system implementation), Bridge (decoupling abstraction from implementation), and Facade (simplifying complex APIs, real subsystem examples).

## 1. Adapter Pattern

### Translating Incompatible Interfaces

```java
/**
 * ADAPTER PATTERN
 * 
 * Converts interface of a class into another interface clients expect
 * Allows classes to work together that couldn't otherwise due to incompatible interfaces
 */

// ========================================
// PROBLEM: INCOMPATIBLE INTERFACES
// ========================================

// Your system expects this interface
interface MediaPlayer {
    void play(String audioType, String fileName);
}

// Third-party library provides this (incompatible!)
class AdvancedMediaPlayer {
    public void playVlc(String fileName) {
        System.out.println("Playing VLC file: " + fileName);
    }
    
    public void playMp4(String fileName) {
        System.out.println("Playing MP4 file: " + fileName);
    }
}

/**
 * PROBLEM:
 * - Your code expects MediaPlayer.play()
 * - Third-party provides playVlc(), playMp4()
 * - Cannot change either interface
 * - Need to integrate them
 */

// ========================================
// SOLUTION: OBJECT ADAPTER
// ========================================
class MediaAdapter implements MediaPlayer {
    private AdvancedMediaPlayer advancedPlayer;
    
    public MediaAdapter(String audioType) {
        if (audioType.equalsIgnoreCase("vlc")) {
            advancedPlayer = new AdvancedMediaPlayer();
        } else if (audioType.equalsIgnoreCase("mp4")) {
            advancedPlayer = new AdvancedMediaPlayer();
        }
    }
    
    @Override
    public void play(String audioType, String fileName) {
        if (audioType.equalsIgnoreCase("vlc")) {
            advancedPlayer.playVlc(fileName);
        } else if (audioType.equalsIgnoreCase("mp4")) {
            advancedPlayer.playMp4(fileName);
        }
    }
}

// Your existing player
class AudioPlayer implements MediaPlayer {
    private MediaAdapter mediaAdapter;
    
    @Override
    public void play(String audioType, String fileName) {
        // Built-in support for mp3
        if (audioType.equalsIgnoreCase("mp3")) {
            System.out.println("Playing MP3 file: " + fileName);
        }
        // Use adapter for other formats
        else if (audioType.equalsIgnoreCase("vlc") || 
                 audioType.equalsIgnoreCase("mp4")) {
            mediaAdapter = new MediaAdapter(audioType);
            mediaAdapter.play(audioType, fileName);
        } else {
            System.out.println("Invalid media format: " + audioType);
        }
    }
}

/**
 * USAGE:
 * 
 * MediaPlayer player = new AudioPlayer();
 * player.play("mp3", "song.mp3");     // Built-in
 * player.play("mp4", "video.mp4");    // Via adapter
 * player.play("vlc", "movie.vlc");    // Via adapter
 * 
 * 
 * BENEFITS:
 * 
 * ✓ No modification to existing code
 * ✓ Third-party library unchanged
 * ✓ Single Responsibility (adaptation separate)
 * ✓ Open/Closed Principle (extend without modifying)
 */

// ========================================
// CLASS ADAPTER (Multiple Inheritance - Not in Java)
// ========================================
/**
 * CLASS ADAPTER:
 * 
 * In languages with multiple inheritance (C++):
 * 
 * class MediaAdapter : public MediaPlayer, public AdvancedMediaPlayer {
 *     void play(string audioType, string fileName) override {
 *         if (audioType == "vlc") playVlc(fileName);
 *         else if (audioType == "mp4") playMp4(fileName);
 *     }
 * };
 * 
 * Java doesn't support multiple inheritance of classes
 * → Use Object Adapter (composition)
 * 
 * 
 * COMPARISON:
 * 
 * Object Adapter (Java):
 * ✓ Composition (has-a)
 * ✓ Can adapt entire hierarchy
 * ✓ More flexible
 * 
 * Class Adapter (C++):
 * - Inheritance (is-a)
 * - Single adaptee only
 * - Slightly more efficient
 */

// ========================================
// REAL-WORLD: INTEGRATING PAYMENT SYSTEMS
// ========================================

// Your payment interface
interface PaymentProcessor {
    boolean processPayment(String orderId, double amount);
    String getTransactionId();
}

// PayPal API (third-party)
class PayPalAPI {
    public void makePayment(String email, double amount) {
        System.out.println("PayPal payment: $" + amount + " to " + email);
    }
    
    public String getPayPalTransactionId() {
        return "PP-" + System.currentTimeMillis();
    }
}

// Stripe API (third-party)
class StripeAPI {
    public boolean charge(int cents, String token) {
        System.out.println("Stripe charge: " + cents + " cents");
        return true;
    }
    
    public String getChargeId() {
        return "ch_" + System.currentTimeMillis();
    }
}

// Adapter for PayPal
class PayPalAdapter implements PaymentProcessor {
    private PayPalAPI paypal;
    private String lastTransactionId;
    
    public PayPalAdapter(PayPalAPI paypal) {
        this.paypal = paypal;
    }
    
    @Override
    public boolean processPayment(String orderId, double amount) {
        // Adapt interface
        paypal.makePayment("customer@email.com", amount);
        lastTransactionId = paypal.getPayPalTransactionId();
        return true;
    }
    
    @Override
    public String getTransactionId() {
        return lastTransactionId;
    }
}

// Adapter for Stripe
class StripeAdapter implements PaymentProcessor {
    private StripeAPI stripe;
    private String lastTransactionId;
    
    public StripeAdapter(StripeAPI stripe) {
        this.stripe = stripe;
    }
    
    @Override
    public boolean processPayment(String orderId, double amount) {
        // Convert dollars to cents
        int cents = (int) (amount * 100);
        boolean success = stripe.charge(cents, "tok_visa");
        if (success) {
            lastTransactionId = stripe.getChargeId();
        }
        return success;
    }
    
    @Override
    public String getTransactionId() {
        return lastTransactionId;
    }
}

// Client code
class PaymentService {
    private PaymentProcessor processor;
    
    public PaymentService(PaymentProcessor processor) {
        this.processor = processor;
    }
    
    public void checkout(String orderId, double amount) {
        if (processor.processPayment(orderId, amount)) {
            System.out.println("Payment successful! TX: " + 
                processor.getTransactionId());
        }
    }
}

/**
 * USAGE:
 * 
 * // Use PayPal
 * PaymentProcessor paypalProcessor = new PayPalAdapter(new PayPalAPI());
 * PaymentService service = new PaymentService(paypalProcessor);
 * service.checkout("ORDER-123", 99.99);
 * 
 * // Switch to Stripe
 * PaymentProcessor stripeProcessor = new StripeAdapter(new StripeAPI());
 * service = new PaymentService(stripeProcessor);
 * service.checkout("ORDER-124", 149.99);
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Easy to switch payment providers
 * ✓ Client code unchanged
 * ✓ Third-party APIs unchanged
 * ✓ Can support multiple providers
 * 
 * 
 * WHEN TO USE ADAPTER:
 * 
 * ✓ Legacy code integration
 * ✓ Third-party library integration
 * ✓ Incompatible interface problem
 * ✓ Want to reuse existing class
 * ✓ Need to work with multiple incompatible interfaces
 */

// ========================================
// JAVA STANDARD LIBRARY EXAMPLES
// ========================================
/**
 * ARRAYS.ASLIST():
 * 
 * String[] array = {"a", "b", "c"};
 * List<String> list = Arrays.asList(array);
 * 
 * Adapts array to List interface
 * 
 * 
 * INPUTSTREAMREADER:
 *
```

## 2. Decorator Pattern

### Adding Responsibilities Dynamically

```java
/**
 * DECORATOR PATTERN
 * 
 * Attaches additional responsibilities to object dynamically
 * Provides flexible alternative to subclassing
 */

// ========================================
// PROBLEM: EXPLOSION OF SUBCLASSES
// ========================================
/**
 * Coffee shop with options:
 * - Coffee
 * - Coffee with milk
 * - Coffee with sugar
 * - Coffee with milk and sugar
 * - Coffee with whipped cream
 * - Coffee with milk and whipped cream
 * - Coffee with sugar and whipped cream
 * - Coffee with milk, sugar, and whipped cream
 * 
 * Subclass approach: 8 classes!
 * With more options: Exponential growth!
 * 
 * class Coffee {}
 * class CoffeeWithMilk extends Coffee {}
 * class CoffeeWithSugar extends Coffee {}
 * class CoffeeWithMilkAndSugar extends Coffee {}
 * ...
 * 
 * PROBLEMS:
 * ✗ Class explosion
 * ✗ Inflexible
 * ✗ Hard to add new options
 */

// ========================================
// SOLUTION: DECORATOR PATTERN
// ========================================

// Component interface
interface Coffee {
    double getCost();
    String getDescription();
}

// Concrete component
class SimpleCoffee implements Coffee {
    @Override
    public double getCost() {
        return 2.0;
    }
    
    @Override
    public String getDescription() {
        return "Simple coffee";
    }
}

// Base decorator
abstract class CoffeeDecorator implements Coffee {
    protected Coffee decoratedCoffee;
    
    public CoffeeDecorator(Coffee coffee) {
        this.decoratedCoffee = coffee;
    }
    
    @Override
    public double getCost() {
        return decoratedCoffee.getCost();
    }
    
    @Override
    public String getDescription() {
        return decoratedCoffee.getDescription();
    }
}

// Concrete decorators
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }
    
    @Override
    public double getCost() {
        return super.getCost() + 0.5;
    }
    
    @Override
    public String getDescription() {
        return super.getDescription() + ", milk";
    }
}

class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) {
        super(coffee);
    }
    
    @Override
    public double getCost() {
        return super.getCost() + 0.2;
    }
    
    @Override
    public String getDescription() {
        return super.getDescription() + ", sugar";
    }
}

class WhippedCreamDecorator extends CoffeeDecorator {
    public WhippedCreamDecorator(Coffee coffee) {
        super(coffee);
    }
    
    @Override
    public double getCost() {
        return super.getCost() + 0.7;
    }
    
    @Override
    public String getDescription() {
        return super.getDescription() + ", whipped cream";
    }
}

/**
 * USAGE:
 * 
 * // Simple coffee
 * Coffee coffee = new SimpleCoffee();
 * System.out.println(coffee.getDescription() + " $" + coffee.getCost());
 * // Output: Simple coffee $2.0
 * 
 * // Coffee with milk
 * coffee = new MilkDecorator(new SimpleCoffee());
 * System.out.println(coffee.getDescription() + " $" + coffee.getCost());
 * // Output: Simple coffee, milk $2.5
 * 
 * // Coffee with milk and sugar
 * coffee = new SugarDecorator(new MilkDecorator(new SimpleCoffee()));
 * System.out.println(coffee.getDescription() + " $" + coffee.getCost());
 * // Output: Simple coffee, milk, sugar $2.7
 * 
 * // Coffee with everything
 * coffee = new WhippedCreamDecorator(
 *     new SugarDecorator(
 *         new MilkDecorator(new SimpleCoffee())));
 * System.out.println(coffee.getDescription() + " $" + coffee.getCost());
 * // Output: Simple coffee, milk, sugar, whipped cream $3.4
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Flexible (any combination of decorators)
 * ✓ Open/Closed (add decorators without modifying existing)
 * ✓ Single Responsibility (each decorator one responsibility)
 * ✓ Compose at runtime
 * ✓ No class explosion
 */

// ========================================
// JAVA I/O STREAMS (CLASSIC EXAMPLE)
// ========================================
/**
 * JAVA I/O DECORATOR STACK:
 * 
 * // Basic component
 * InputStream is = new FileInputStream("file.txt");
 * 
 * // Add buffering
 * is = new BufferedInputStream(is);
 * 
 * // Add data reading capabilities
 * DataInputStream dis = new DataInputStream(is);
 * 
 * // All at once:
 * DataInputStream dis = new DataInputStream(
 *     new BufferedInputStream(
 *         new FileInputStream("file.txt")));
 * 
 * 
 * HIERARCHY:
 * 
 * InputStream (Component)
 * ├── FileInputStream (ConcreteComponent)
 * ├── ByteArrayInputStream (ConcreteComponent)
 * └── FilterInputStream (Decorator)
 *     ├── BufferedInputStream (ConcreteDecorator)
 *     ├── DataInputStream (ConcreteDecorator)
 *     └── PushbackInputStream (ConcreteDecorator)
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Stack behaviors (buffering, data reading, etc.)
 * ✓ Any combination possible
 * ✓ Uniform interface (InputStream)
 * ✓ Add new decorators without modifying existing
 */

// ========================================
// REAL-WORLD: TEXT FORMATTING
// ========================================

interface Text {
    String getContent();
}

class PlainText implements Text {
    private String content;
    
    public PlainText(String content) {
        this.content = content;
    }
    
    @Override
    public String getContent() {
        return content;
    }
}

abstract class TextDecorator implements Text {
    protected Text decoratedText;
    
    public TextDecorator(Text text) {
        this.decoratedText = text;
    }
}

class BoldDecorator extends TextDecorator {
    public BoldDecorator(Text text) {
        super(text);
    }
    
    @Override
    public String getContent() {
        return "<b>" + decoratedText.getContent() + "</b>";
    }
}

class ItalicDecorator extends TextDecorator {
    public ItalicDecorator(Text text) {
        super(text);
    }
    
    @Override
    public String getContent() {
        return "<i>" + decoratedText.getContent() + "</i>";
    }
}

class UnderlineDecorator extends TextDecorator {
    public UnderlineDecorator(Text text) {
        super(text);
    }
    
    @Override
    public String getContent() {
        return "<u>" + decoratedText.getContent() + "</u>";
    }
}

/**
 * USAGE:
 * 
 * Text text = new PlainText("Hello World");
 * 
 * // Bold
 * text = new BoldDecorator(text);
 * System.out.println(text.getContent());
 * // Output: <b>Hello World</b>
 * 
 * // Bold and italic
 * text = new ItalicDecorator(new BoldDecorator(new PlainText("Hello")));
 * System.out.println(text.getContent());
 * // Output: <i><b>Hello</b></i>
 * 
 * // All formatting
 * text = new UnderlineDecorator(
 *     new ItalicDecorator(
 *         new BoldDecorator(new PlainText("Important"))));
 * System.out.println(text.getContent());
 * // Output: <u><i><b>Important</b></i></u>
 * 
 * 
 * WHEN TO USE DECORATOR:
 * 
 * ✓ Add responsibilities without subclassing
 * ✓ Responsibilities can be withdrawn
 * ✓ Behavior needs to be combined dynamically
 * ✓ Extension by subclassing impractical
 * ✓ Need multiple independent extensions
 */
```

## 3. Proxy Pattern

### Controlling Access to Objects

```java
/**
 * PROXY PATTERN
 * 
 * Provides placeholder for another object
 * Controls access to original object
 */

import java.lang.reflect.*;
import net.sf.cglib.proxy.*;

// ========================================
// STATIC PROXY (MANUAL)
// ========================================

interface Image {
    void display();
}

// Real object (expensive to create)
class RealImage implements Image {
    private String filename;
    
    public RealImage(String filename) {
        this.filename = filename;
        loadFromDisk();
    }
    
    private void loadFromDisk() {
        System.out.println("Loading " + filename + " from disk...");
        // Simulate expensive operation
        try {
            Thread.sleep(2000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    @Override
    public void display() {
        System.out.println("Displaying " + filename);
    }
}

// Proxy (lazy loading)
class ImageProxy implements Image {
    private String filename;
    private RealImage realImage;
    
    public ImageProxy(String filename) {
        this.filename = filename;
    }
    
    @Override
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(filename);  // Lazy load!
        }
        realImage.display();
    }
}

/**
 * USAGE:
 * 
 * Image image1 = new ImageProxy("photo1.jpg");
 * Image image2 = new ImageProxy("photo2.jpg");
 * 
 * // Images not loaded yet!
 * System.out.println("Images created");
 * 
 * // First display triggers loading
 * image1.display();  // Loads photo1.jpg (2 seconds)
 * // Output: Loading photo1.jpg from disk...
 * //         Displaying photo1.jpg
 * 
 * // Second display uses cached image
 * image1.display();  // Instant!
 * // Output: Displaying photo1.jpg
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Lazy initialization (load on demand)
 * ✓ Saves resources
 * ✓ Transparent to client
 * 
 * 
 * TYPES OF PROXIES:
 * 
 * 1. Virtual Proxy: Lazy loading (above example)
 * 2. Protection Proxy: Access control
 * 3. Remote Proxy: Remote object access
 * 4. Caching Proxy: Cache results
 * 5. Logging Proxy: Log method calls
 */

// ========================================
// PROTECTION PROXY (ACCESS CONTROL)
// ========================================

interface BankAccount {
    void deposit(double amount);
    void withdraw(double amount);
    double getBalance();
}

class RealBankAccount implements BankAccount {
    private double balance;
    
    @Override
    public void deposit(double amount) {
        balance += amount;
        System.out.println("Deposited: $" + amount);
    }
    
    @Override
    public void withdraw(double amount) {
        if (balance >= amount) {
            balance -= amount;
            System.out.println("Withdrawn: $" + amount);
        } else {
            System.out.println("Insufficient funds");
        }
    }
    
    @Override
    public double getBalance() {
        return balance;
    }
}

class BankAccountProxy implements BankAccount {
    private RealBankAccount realAccount;
    private String userRole;
    
    public BankAccountProxy(String userRole) {
        this.userRole = userRole;
        this.realAccount = new RealBankAccount();
    }
    
    @Override
    public void deposit(double amount) {
        realAccount.deposit(amount);
    }
    
    @Override
    public void withdraw(double amount) {
        if (userRole.equals("OWNER")) {
            realAccount.withdraw(amount);
        } else {
            System.out.println("Access denied: Only owner can withdraw");
        }
    }
    
    @Override
    public double getBalance() {
        if (userRole.equals("OWNER") || userRole.equals("AUDITOR")) {
            return realAccount.getBalance();
        } else {
            System.out.println("Access denied: Cannot view balance");
            return 0;
        }
    }
}

/**
 * USAGE:
 * 
 * // Owner - full access
 * BankAccount ownerAccount = new BankAccountProxy("OWNER");
 * ownerAccount.deposit(1000);
 * ownerAccount.withdraw(500);
 * System.out.println("Balance: $" + ownerAccount.getBalance());
 * 
 * // Guest - restricted access
 * BankAccount guestAccount = new BankAccountProxy("GUEST");
 * guestAccount.deposit(1000);
 * guestAccount.withdraw(500);  // Denied!
 * guestAccount.getBalance();    // Denied!
 */

// ========================================
// JDK DYNAMIC PROXY
// ========================================

interface Calculator {
    int add(int a, int b);
    int subtract(int a, int b);
}

class CalculatorImpl implements Calculator {
    @Override
    public int add(int a, int b) {
        return a + b;
    }
    
    @Override
    public int subtract(int a, int b) {
        return a - b;
    }
}

class LoggingHandler implements InvocationHandler {
    private Object target;
    
    public LoggingHandler(Object target) {
        this.target = target;
    }
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) 
            throws Throwable {
        System.out.println("Before: " + method.getName() + 
            " with args " + Arrays.toString(args));
        
        Object result = method.invoke(target, args);
        
        System.out.println("After: " + method.getName() + 
            " returned " + result);
        
        return result;
    }
}

/**
 * USAGE:
 * 
 * Calculator realCalculator = new CalculatorImpl();
 * 
 * Calculator proxy = (Calculator) Proxy.newProxyInstance(
 *     Calculator.class.getClassLoader(),
 *     new Class<?>[] { Calculator.class },
 *     new LoggingHandler(realCalculator)
 * );
 * 
 * int result = proxy.add(5, 3);
 * // Output: Before: add with args [5, 3]
 * //         After: add returned 8
 * 
 * 
 * BENEFITS OF DYNAMIC PROXY:
 * 
 * ✓ No manual proxy classes
 * ✓ Works for any interface
 * ✓ Cross-cutting concerns (logging, security, caching)
 * ✓ Used by Spring AOP
 * 
 * 
 * LIMITATIONS:
 * 
 * ✗ Requires interface
 * ✗ Cannot proxy classes
 * ✗ Slight performance overhead
 */

// ========================================
// CGLIB PROXY (CLASS PROXYING)
// ========================================

class Service {
    public void doSomething() {
        System.out.println("Doing something...");
    }
    
    public String getName() {
        return "Service";
    }
}

class TimingInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, 
                          MethodProxy proxy) throws Throwable {
        long start = System.nanoTime();
        
        Object result = proxy.invokeSuper(obj, args);
        
        long elapsed = System.nanoTime() - start;
        System.out.println(method.getName() + " took " + 
            elapsed / 1_000_000.0 + "ms");
        
        return result;
    }
}

/**
 * USAGE (conceptual - requires CGLIB library):
 * 
 * Enhancer enhancer = new Enhancer();
 * enhancer.setSuperclass(Service.class);
 * enhancer.setCallback(new TimingInterceptor());
 * 
 * Service proxy = (Service) enhancer.create();
 * proxy.doSomething();
 * // Output: Doing something...
 * //         doSomething took 0.123ms
 * 
 * 
 * CGLIB vs JDK PROXY:
 * 
 * JDK Dynamic Proxy:
 * ✓ Standard Java
 * ✓ Interface-based
 * ✗ Requires interface
 * 
 * CGLIB Proxy:
 * ✓ Can proxy classes
 * ✓ No interface required
 * ✗ External library
 * ✗ Cannot proxy final classes/methods
 * 
 * 
 * SPRING USAGE:
 * 
 * Spring AOP uses:
 * - JDK proxy if target implements interface
 * - CGLIB proxy if target is class only
 */

// ========================================
// CACHING PROXY
// ========================================

interface DataService {
    String getData(String key);
}

class ExpensiveDataService implements DataService {
    @Override
    public String getData(String key) {
        System.out.println("Fetching data for: " + key);
        // Simulate expensive operation
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        return "Data for " + key;
    }
}

class CachingDataServiceProxy implements DataService {
    private DataService realService;
    private Map<String, String> cache = new HashMap<>();
    
    public CachingDataServiceProxy(DataService realService) {
        this.realService = realService;
    }
    
    @Override
    public String getData(String key) {
        if (cache.containsKey(key)) {
            System.out.println("Returning cached data for: " + key);
            return cache.get(key);
        }
        
        String data = realService.getData(key);
        cache.put(key, data);
        return data;
    }
}

/**
 * USAGE:
 * 
 * DataService service = new CachingDataServiceProxy(
 *     new ExpensiveDataService());
 * 
 * service.getData("key1");  // Fetches (1 second)
 * service.getData("key1");  // Cached (instant!)
 * service.getData("key2");  // Fetches (1 second)
 * service.getData("key2");  // Cached (instant!)
 * 
 * 
 * WHEN TO USE PROXY:
 * 
 * ✓ Lazy loading (virtual proxy)
 * ✓ Access control (protection proxy)
 * ✓ Remote access (remote proxy)
 * ✓ Caching (caching proxy)
 * ✓ Logging/monitoring (logging proxy)
 * ✓ Thread safety (synchronization proxy)
 */
```

## 4. Composite Pattern

### Treating Objects and Compositions Uniformly

```java
/**
 * COMPOSITE PATTERN
 * 
 * Composes objects into tree structures
 * Treats individual objects and compositions uniformly
 */

// ========================================
// FILE SYSTEM EXAMPLE
// ========================================

// Component interface
interface FileSystemComponent {
    String getName();
    int getSize();
    void print(String indent);
}

// Leaf - File
class File implements FileSystemComponent {
    private String name;
    private int size;
    
    public File(String name, int size) {
        this.name = name;
        this.size = size;
    }
    
    @Override
    public String getName() {
        return name;
    }
    
    @Override
    public int getSize() {
        return size;
    }
    
    @Override
    public void print(String indent) {
        System.out.println(indent + "📄 " + name + " (" + size + " KB)");
    }
}

// Composite - Directory
class Directory implements FileSystemComponent {
    private String name;
    private List<FileSystemComponent> children = new ArrayList<>();
    
    public Directory(String name) {
        this.name = name;
    }
    
    public void add(FileSystemComponent component) {
        children.add(component);
    }
    
    public void remove(FileSystemComponent component) {
        children.remove(component);
    }
    
    @Override
    public String getName() {
        return name;
    }
    
    @Override
    public int getSize() {
        // Size = sum of all children
        return children.stream()
            .mapToInt(FileSystemComponent::getSize)
            .sum();
    }
    
    @Override
    public void print(String indent) {
        System.out.println(indent + "📁 " + name + " (" + getSize() + " KB)");
        for (FileSystemComponent child : children) {
            child.print(indent + "  ");
        }
    }
}

/**
 * USAGE:
 * 
 * // Build file system tree
 * Directory root = new Directory("root");
 * 
 * File file1 = new File("file1.txt", 100);
 * File file2 = new File("file2.txt", 200);
 * root.add(file1);
 * root.add(file2);
 * 
 * Directory documents = new Directory("documents");
 * File doc1 = new File("report.pdf", 500);
 * File doc2 = new File("presentation.pptx", 1000);
 * documents.add(doc1);
 * documents.add(doc2);
 * root.add(documents);
 * 
 * Directory photos = new Directory("photos");
 * File photo1 = new File("vacation.jpg", 2000);
 * File photo2 = new File("family.jpg", 1500);
 * photos.add(photo1);
 * photos.add(photo2);
 * root.add(photos);
 * 
 * // Print structure
 * root.print("");
 * 
 * // Output:
 * // 📁 root (5300 KB)
 * //   📄 file1.txt (100 KB)
 * //   📄 file2.txt (200 KB)
 * //   📁 documents (1500 KB)
 * //     📄 report.pdf (500 KB)
 * //     📄 presentation.pptx (1000 KB)
 * //   📁 photos (3500 KB)
 * //     📄 vacation.jpg (2000 KB)
 * //     📄 family.jpg (1500 KB)
 * 
 * 
 * KEY INSIGHT:
 * 
 * Client treats File and Directory uniformly:
 * - Both implement FileSystemComponent
 * - getSize() works on both (file size vs directory total)
 * - print() works on both (leaf vs recursive)
 * - No type checking needed!
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Uniform treatment of objects and compositions
 * ✓ Easy to add new component types
 * ✓ Client code simplified
 * ✓ Tree structures naturally represented
 */

// ========================================
// REAL-WORLD: ORGANIZATION HIERARCHY
// ========================================

interface Employee {
    String getName();
    String getRole();
    double getSalary();
    void print(String indent);
}

// Leaf - Individual Contributor
class IndividualContributor implements Employee {
    private String name;
    private String role;
    private double salary;
    
    public IndividualContributor(String name, String role, double salary) {
        this.name = name;
        this.role = role;
        this.salary = salary;
    }
    
    @Override
    public String getName() {
        return name;
    }
    
    @Override
    public String getRole() {
        return role;
    }
    
    @Override
    public double getSalary() {
        return salary;
    }
    
    @Override
    public void print(String indent) {
        System.out.println(indent + name + " - " + role + 
            " ($" + String.format("%.0f", salary) + ")");
    }
}

// Composite - Manager
class Manager implements Employee {
    private String name;
    private String role;
    private double salary;
    private List<Employee> subordinates = new ArrayList<>();
    
    public Manager(String name, String role, double salary) {
        this.name = name;
        this.role = role;
        this.salary = salary;
    }
    
    public void addSubordinate(Employee employee) {
        subordinates.add(employee);
    }
    
    public void removeSubordinate(Employee employee) {
        subordinates.remove(employee);
    }
    
    @Override
    public String getName() {
        return name;
    }
    
    @Override
    public String getRole() {
        return role;
    }
    
    @Override
    public double getSalary() {
        // Total salary cost (manager + all subordinates)
        return salary + subordinates.stream()
            .mapToDouble(Employee::getSalary)
            .sum();
    }
    
    @Override
    public void print(String indent) {
        System.out.println(indent + name + " - " + role + 
            " ($" + String.format("%.0f", salary) + 
            ", Team: $" + String.format("%.0f", getSalary()) + ")");
        for (Employee subordinate : subordinates) {
            subordinate.print(indent + "  ");
        }
    }
}

/**
 * USAGE:
 * 
 * // CEO
 * Manager ceo = new Manager("Alice", "CEO", 200000);
 * 
 * // Engineering VP
 * Manager vpEng = new Manager("Bob", "VP Engineering", 150000);
 * vpEng.addSubordinate(new IndividualContributor("Charlie", "Senior Dev", 120000));
 * vpEng.addSubordinate(new IndividualContributor("David", "Junior Dev", 80000));
 * ceo.addSubordinate(vpEng);
 * 
 * // Marketing VP
 * Manager vpMarketing = new Manager("Eve", "VP Marketing", 140000);
 * vpMarketing.addSubordinate(new IndividualContributor("Frank", "Marketing Manager", 100000));
 * ceo.addSubordinate(vpMarketing);
 * 
 * // Print organization
 * ceo.print("");
 * 
 * // Output:
 * // Alice - CEO ($200000, Team: $790000)
 * //   Bob - VP Engineering ($150000, Team: $350000)
 * //     Charlie - Senior Dev ($120000)
 * //     David - Junior Dev ($80000)
 * //   Eve - VP Marketing ($140000, Team: $240000)
 * //     Frank - Marketing Manager ($100000)
 * 
 * System.out.println("Total payroll: $" + ceo.getSalary());
 * // Output: Total payroll: $790000
 * 
 * 
 * WHEN TO USE COMPOSITE:
 * 
 * ✓ Part-whole hierarchies (trees)
 * ✓ Treat individual objects and compositions uniformly
 * ✓ Recursive structures
 * ✓ Don't want client to distinguish leaf vs composite
 * 
 * EXAMPLES:
 * - File systems (files and directories)
 * - Organization charts (employees and managers)
 * - UI components (widgets and containers)
 * - Graphics (shapes and groups)
 */
```

## 5. Bridge Pattern

### Separating Abstraction from Implementation

```java
/**
 * BRIDGE PATTERN
 * 
 * Decouples abstraction from implementation
 * Both can vary independently
 */

// ========================================
// PROBLEM: EXPLOSION OF COMBINATIONS
// ========================================
/**
 * Remote controls for different devices:
 * - BasicRemote, AdvancedRemote
 * - TV, Radio, Stereo
 * 
 * Without Bridge: 2 × 3 = 6 classes
 * - BasicTVRemote, BasicRadioRemote, BasicStereoRemote
 * - AdvancedTVRemote, AdvancedRadioRemote, AdvancedStereoRemote
 * 
 * Adding new remote or device: More classes!
 */

// ========================================
// SOLUTION: BRIDGE PATTERN
// ========================================

// Implementation interface (Device)
interface Device {
    void turnOn();
    void turnOff();
    void setVolume(int volume);
    int getVolume();
    void setChannel(int channel);
    int getChannel();
}

// Concrete implementation - TV
class TV implements Device {
    private boolean on = false;
    private int volume = 30;
    private int channel = 1;
    
    @Override
    public void turnOn() {
        on = true;
        System.out.println("TV is ON");
    }
    
    @Override
    public void turnOff() {
        on = false;
        System.out.println("TV is OFF");
    }
    
    @Override
    public void setVolume(int volume) {
        this.volume = volume;
        System.out.println("TV volume: " + volume);
    }
    
    @Override
    public int getVolume() {
        return volume;
    }
    
    @Override
    public void setChannel(int channel) {
        this.channel = channel;
        System.out.println("TV channel: " + channel);
    }
    
    @Override
    public int getChannel() {
        return channel;
    }
}

// Concrete implementation - Radio
class Radio implements Device {
    private boolean on = false;
    private int volume = 20;
    private int channel = 90; // FM frequency
    
    @Override
    public void turnOn() {
        on = true;
        System.out.println("Radio is ON");
    }
    
    @Override
    public void turnOff() {
        on = false;
        System.out.println("Radio is OFF");
    }
    
    @Override
    public void setVolume(int volume) {
        this.volume = volume;
        System.out.println("Radio volume: " + volume);
    }
    
    @Override
    public int getVolume() {
        return volume;
    }
    
    @Override
    public void setChannel(int channel) {
        this.channel = channel;
        System.out.println("Radio frequency: " + channel + " FM");
    }
    
    @Override
    public int getChannel() {
        return channel;
    }
}

// Abstraction (Remote)
abstract class Remote {
    protected Device device;
    
    public Remote(Device device) {
        this.device = device;
    }
    
    public void togglePower() {
        // Implementation depends on device
        device.turnOn();
    }
    
    public void volumeUp() {
        device.setVolume(device.getVolume() + 10);
    }
    
    public void volumeDown() {
        device.setVolume(device.getVolume() - 10);
    }
    
    public void channelUp() {
        device.setChannel(device.getChannel() + 1);
    }
    
    public void channelDown() {
        device.setChannel(device.getChannel() - 1);
    }
}

// Refined abstraction - Basic Remote
class BasicRemote extends Remote {
    public BasicRemote(Device device) {
        super(device);
    }
}

// Refined abstraction - Advanced Remote
class AdvancedRemote extends Remote {
    public AdvancedRemote(Device device) {
        super(device);
    }
    
    public void mute() {
        device.setVolume(0);
        System.out.println("Device muted");
    }
    
    public void setChannel(int channel) {
        device.setChannel(channel);
    }
}

/**
 * USAGE:
 * 
 * // Basic remote with TV
 * Device tv = new TV();
 * Remote basicRemote = new BasicRemote(tv);
 * basicRemote.togglePower();
 * basicRemote.volumeUp();
 * basicRemote.channelUp();
 * 
 * // Advanced remote with Radio
 * Device radio = new Radio();
 * AdvancedRemote advancedRemote = new AdvancedRemote(radio);
 * advancedRemote.togglePower();
 * advancedRemote.setChannel(105);
 * advancedRemote.mute();
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Separate abstraction from implementation
 * ✓ Both can vary independently
 * ✓ 2 remotes × 3 devices = 2 + 3 = 5 classes (not 6!)
 * ✓ Add new remote: +1 class
 * ✓ Add new device: +1 class
 * ✓ Runtime binding (can change device)
 * 
 * 
 * BRIDGE vs ADAPTER:
 * 
 * Bridge:
 * - Design upfront
 * - Abstraction and implementation vary independently
 * - Both hierarchies designed together
 * 
 * Adapter:
 * - Applied to existing code
 * - Makes incompatible interfaces work together
 * - Fix for legacy/third-party code
 */

// ========================================
// REAL-WORLD: DRAWING API
// ========================================

// Implementation
interface Renderer {
    void renderCircle(int radius);
    void renderRectangle(int width, int height);
}

// Concrete implementation - Vector
class VectorRenderer implements Renderer {
    @Override
    public void renderCircle(int radius) {
        System.out.println("Drawing circle (vector) with radius " + radius);
    }
    
    @Override
    public void renderRectangle(int width, int height) {
        System.out.println("Drawing rectangle (vector) " + width + "x" + height);
    }
}

// Concrete implementation - Raster
class RasterRenderer implements Renderer {
    @Override
    public void renderCircle(int radius) {
        System.out.println("Drawing circle (raster) with radius " + radius);
    }
    
    @Override
    public void renderRectangle(int width, int height) {
        System.out.println("Drawing rectangle (raster) " + width + "x" + height);
    }
}

// Abstraction
abstract class Shape {
    protected Renderer renderer;
    
    public Shape(Renderer renderer) {
        this.renderer = renderer;
    }
    
    abstract void draw();
}

// Refined abstraction
class Circle extends Shape {
    private int radius;
    
    public Circle(Renderer renderer, int radius) {
        super(renderer);
        this.radius = radius;
    }
    
    @Override
    void draw() {
        renderer.renderCircle(radius);
    }
}

class Rectangle extends Shape {
    private int width, height;
    
    public Rectangle(Renderer renderer, int width, int height) {
        super(renderer);
        this.width = width;
        this.height = height;
    }
    
    @Override
    void draw() {
        renderer.renderRectangle(width, height);
    }
}

/**
 * USAGE:
 * 
 * // Vector rendering
 * Renderer vector = new VectorRenderer();
 * Shape circle = new Circle(vector, 10);
 * circle.draw();
 * // Output: Drawing circle (vector) with radius 10
 * 
 * // Raster rendering
 * Renderer raster = new RasterRenderer();
 * Shape rectangle = new Rectangle(raster, 20, 30);
 * rectangle.draw();
 * // Output: Drawing rectangle (raster) 20x30
 * 
 * // Can switch renderer at runtime
 * circle = new Circle(raster, 10);
 * circle.draw();
 * // Output: Drawing circle (raster) with radius 10
 * 
 * 
 * WHEN TO USE BRIDGE:
 * 
 * ✓ Avoid permanent binding between abstraction and implementation
 * ✓ Both abstraction and implementation should be extensible
 * ✓ Changes in implementation shouldn't affect clients
 * ✓ Want to share implementation among multiple objects
 * ✓ Platform independence (different platforms as implementations)
 */
```

## 6. Facade Pattern

### Simplifying Complex Subsystems

```java
/**
 * FACADE PATTERN
 * 
 * Provides unified interface to set of interfaces in subsystem
 * Defines higher-level interface that makes subsystem easier to use
 */

// ========================================
// COMPLEX SUBSYSTEM
// ========================================

// CPU subsystem
class CPU {
    public void freeze() {
        System.out.println("CPU: Freezing...");
    }
    
    public void jump(long position) {
        System.out.println("CPU: Jumping to " + position);
    }
    
    public void execute() {
        System.out.println("CPU: Executing...");
    }
}

// Memory subsystem
class Memory {
    public void load(long position, byte[] data) {
        System.out.println("Memory: Loading data at " + position);
    }
}

// HardDrive subsystem
class HardDrive {
    public byte[] read(long lba, int size) {
        System.out.println("HardDrive: Reading " + size + " bytes from " + lba);
        return new byte[size];
    }
}

// ========================================
// WITHOUT FACADE: CLIENT COMPLEXITY
// ========================================
class ComputerWithoutFacade {
    public void start() {
        // Client must know all subsystem details!
        CPU cpu = new CPU();
        Memory memory = new Memory();
        HardDrive hardDrive = new HardDrive();
        
        cpu.freeze();
        memory.load(0, hardDrive.read(0, 1024));
        cpu.jump(0);
        cpu.execute();
    }
}

// ========================================
// WITH FACADE: SIMPLIFIED INTERFACE
// ========================================
class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private HardDrive hardDrive;
    
    public ComputerFacade() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.hardDrive = new HardDrive();
    }
    
    public void start() {
        // Facade handles complexity
        cpu.freeze();
        memory.load(0, hardDrive.read(0, 1024));
        cpu.jump(0);
        cpu.execute();
    }
    
    public void shutdown() {
        System.out.println("Computer: Shutting down...");
        // Complex shutdown sequence hidden
    }
}

/**
 * USAGE:
 * 
 * // Without facade
 * ComputerWithoutFacade computer1 = new ComputerWithoutFacade();
 * computer1.start();  // Client must know all steps!
 * 
 * // With facade
 * ComputerFacade computer2 = new ComputerFacade();
 * computer2.start();  // Simple!
 * computer2.shutdown();
 * 
 * 
 * BENEFITS:
 * 
 * ✓ Simplifies complex subsystems
 * ✓ Loose coupling (client doesn't depend on subsystem)
 * ✓ Easy to use
 * ✓ Can still access subsystem directly if needed
 */

// ========================================
// REAL-WORLD: HOME THEATER
// ========================================

// Subsystems
class Amplifier {
    public void on() { System.out.println("Amplifier ON"); }
    public void setVolume(int level) { 
        System.out.println("Amplifier volume: " + level); 
    }
    public void off() { System.out.println("Amplifier OFF"); }
}

class DVDPlayer {
    public void on() { System.out.println("DVD Player ON"); }
    public void play(String movie) { 
        System.out.println("Playing: " + movie); 
    }
    public void stop() { System.out.println("DVD stopped"); }
    public void off() { System.out.println("DVD Player OFF"); }
}

class Projector {
    public void on() { System.out.println("Projector ON"); }
    public void wideScreenMode() { 
        System.out.println("Projector: Wide screen mode"); 
    }
    public void off() { System.out.println("Projector OFF"); }
}

class Lights {
    public void dim(int level) { 
        System.out.println("Lights dimmed to " + level + "%"); 
    }
}

// Facade
class HomeTheaterFacade {
    private Amplifier amp;
    private DVDPlayer dvd;
    private Projector projector;
    private Lights lights;
    
    public HomeTheaterFacade(Amplifier amp, DVDPlayer dvd, 
                            Projector projector, Lights lights) {
        this.amp = amp;
        this.dvd = dvd;
        this.projector = projector;
        this.lights = lights;
    }
    
    public void watchMovie(String movie) {
        System.out.println("\nGet ready to watch a movie...\n");
        lights.dim(10);
        projector.on();
        projector.wideScreenMode();
        amp.on();
        amp.setVolume(50);
        dvd.on();
        dvd.play(movie);
    }
    
    public void endMovie() {
        System.out.println("\nShutting down movie theater...\n");
        dvd.stop();
        dvd.off();
        amp.off();
        projector.off();
        lights.dim(100);
    }
}

/**
 * USAGE:
 * 
 * // Setup
 * HomeTheaterFacade homeTheater = new HomeTheaterFacade(
 *     new Amplifier(),
 *     new DVDPlayer(),
 *     new Projector(),
 *     new Lights()
 * );
 * 
 * // Simple interface!
 * homeTheater.watchMovie("Inception");
 * // ... watch movie ...
 * homeTheater.endMovie();
 * 
 * 
 * WITHOUT FACADE:
 * 
 * Lights lights = new Lights();
 * Projector projector = new Projector();
 * Amplifier amp = new Amplifier();
 * DVDPlayer dvd = new DVDPlayer();
 * 
 * lights.dim(10);
 * projector.on();
 * projector.wideScreenMode();
 * amp.on();
 * amp.setVolume(50);
 * dvd.on();
 * dvd.play("Inception");
 * // ... and then all in reverse to shut down!
 * 
 * Much more complex!
 */

// JAVA EXAMPLES
// ========================================
/**
 * JDBC:
 * 
 * DriverManager is a Facade over multiple database drivers
 * 
 * Connection conn = DriverManager.getConnection(url);
 * Statement stmt = conn.createStatement();

 * 
 * Hides complexity of:
 * - Driver registration
 * - Connection pooling
 * - Protocol handling
 * 
 * 
 * SPRING:
 *

 *

 * 
 * Simplifies:
 * - Connection management
 * - Statement creation
 * - ResultSet handling
 * - Exception translation
 * 
 * 
 * WHEN TO USE FACADE:
 * 
 * ✓ Complex subsystem with many classes
 * ✓ Want simple interface to subsystem
 * ✓ Layer dependencies (facade for each layer)
 * ✓ Decouple client from subsystem
 * ✓ Subsystem difficult to use
 */
```

## Summary

### Quick Reference

**Structural Patterns:**

```
1. Adapter: Convert interface
   - Object adapter (composition) - Java standard
   - Class adapter (inheritance) - Not in Java
   - Use: Third-party integration, legacy code

2. Decorator: Add behavior dynamically
   - Wrap objects to extend functionality
   - Java I/O streams (classic example)
   - Use: Flexible behavior addition

3. Proxy: Control access
   - Virtual (lazy loading)
   - Protection (access control)
   - Remote (remote objects)
   - Caching (cache results)
   - Dynamic (JDK, CGLIB)

4. Composite: Tree structures
   - Treat leaf and composite uniformly
   - Recursive structures
   - Use: File systems, organizations, UI

5. Bridge: Decouple abstraction from implementation
   - Both can vary independently
   - Use: Platform independence, multiple variations

6. Facade: Simplify complex subsystems
   - Higher-level interface
   - Use: Complex libraries, layered architecture
```

**Pattern Comparison:**

```
┌─────────────┬────────────────────┬──────────────────────────┐
│ Pattern     │ Intent             │ Key Characteristic       │
├─────────────┼────────────────────┼──────────────────────────┤
│ Adapter     │ Convert interface  │ Make incompatible work   │
│ Decorator   │ Add responsibilities│ Wrap to extend          │
│ Proxy       │ Control access     │ Placeholder/surrogate    │
│ Composite   │ Part-whole tree    │ Uniform treatment        │
│ Bridge      │ Decouple abstract  │ Independent hierarchies  │
│ Facade      │ Simplify subsystem │ Unified interface        │
└─────────────┴────────────────────┴──────────────────────────┘
```

**Decision Guide:**

```
Need to integrate incompatible interface?
→ Adapter

Need to add responsibilities dynamically?
→ Decorator

Need to control access to object?
→ Proxy

Need to represent part-whole hierarchy?
→ Composite

Need independent abstraction/implementation?
→ Bridge

Need to simplify complex subsystem?
→ Facade
```

**Real-World Examples:**

```
Adapter:
- Arrays.asList() (array to List)
- InputStreamReader (byte to char stream)
- Payment gateway adapters

Decorator:
- Java I/O streams (BufferedInputStream, etc.)
- Servlet filters
- Text formatting

Proxy:
- Spring AOP (dynamic proxies)
- Hibernate lazy loading
- RMI (remote proxy)

Composite:
- File systems (files/directories)
- Swing UI components
- Organization charts

Bridge:
- JDBC drivers (abstraction: JDBC, impl: vendors)
- Graphics rendering
- Device drivers

Facade:
- DriverManager (JDBC)
- JdbcTemplate (Spring)
- Complex library wrappers
```

---


**Key insights:**

- **Adapter translates** incompatible interfaces
- **Decorator adds** behavior dynamically (I/O streams!)
- **Proxy controls** access (lazy, security, caching, dynamic)
- **Composite treats** objects and compositions uniformly
- **Bridge separates** abstraction from implementation
- **Facade simplifies** complex subsystems
