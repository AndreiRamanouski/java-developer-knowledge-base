# Resource Management Patterns

## Overview

Resource management is critical in Java—improper handling leads to resource leaks (unclosed files, connections), memory leaks (objects never garbage collected), and performance degradation. Resources are anything that needs explicit cleanup: file handles, database connections, network sockets, native memory, thread pools, locks. Java's garbage collector handles memory, but it doesn't handle external resources—you must close them explicitly. This guide covers six essential patterns: Try-with-resources (Java 7+, the standard way—automatic cleanup for AutoCloseable resources), Dispose pattern (manual cleanup with explicit dispose() method), Object Pool pattern (reuse expensive resources like connections), Resource Leak Prevention (common mistakes and detection), Finalizers and Cleaners (why they're dangerous and how to avoid them), and Weak Reference Cleanup (memory-sensitive caches that release under pressure). The key insight: Resource cleanup is deterministic (happens when you say) vs garbage collection which is non-deterministic (happens when GC runs). You can't rely on finalize() or GC to close resources—by the time GC runs, you've exhausted file handles or connections. Try-with-resources solves 90% of resource management—it's the Java way. Object pools solve the other 10%—when creating/destroying resources is expensive. Finalizers are almost always wrong—they're unpredictable, slow, and dangerous. This guide shows patterns that work (try-with-resources, explicit dispose, pooling) and anti-patterns to avoid (finalizers, ignoring exceptions, missing cleanup). Understanding resource management means understanding the difference between memory (GC handles) and resources (you handle).

## 1. Try-With-Resources

### The Standard Way (Java 7+)

```java
/**
 * TRY-WITH-RESOURCES
 * 
 * Automatic resource management
 * Resources automatically closed at end of block
 * Requires AutoCloseable interface
 */

// ========================================
// ❌ OLD WAY: Manual Resource Management
// ========================================

/**
 * Pre-Java 7 approach (error-prone!)
 */
class OldWay {
    
    public void readFile(String path) {
        FileInputStream fis = null;
        try {
            fis = new FileInputStream(path);
            int data = fis.read();
            // Process data...
            
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // ❌ Manual cleanup
            if (fis != null) {
                try {
                    fis.close();
                } catch (IOException e) {
                    // What to do with this exception?
                    e.printStackTrace();
                }
            }
        }
    }
    
    /**
     * PROBLEMS:
     * 
     * ✗ Verbose (lots of boilerplate)
     * ✗ Easy to forget close()
     * ✗ Exception handling complex
     * ✗ close() can throw exception (nested try-catch!)
     * ✗ Resource leak if forget to close
     */
}

// ========================================
// ✓ NEW WAY: Try-With-Resources
// ========================================

/**
 * Java 7+ approach (clean and safe!)
 */
class NewWay {
    
    public void readFile(String path) {
        try (FileInputStream fis = new FileInputStream(path)) {
            int data = fis.read();
            // Process data...
        } catch (IOException e) {
            e.printStackTrace();
        }
        // ✓ fis.close() called automatically!
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ Automatic cleanup (can't forget)
     * ✓ Less boilerplate
     * ✓ Exception handling cleaner
     * ✓ Multiple resources supported
     * ✓ Suppressed exceptions handled correctly
     */
}

// ========================================
// MULTIPLE RESOURCES
// ========================================

/**
 * Multiple resources in one try-with-resources
 */
class MultipleResources {
    
    public void copyFile(String source, String dest) {
        // Both resources closed automatically (in reverse order)
        try (FileInputStream in = new FileInputStream(source);
             FileOutputStream out = new FileOutputStream(dest)) {
            
            byte[] buffer = new byte[1024];
            int bytesRead;
            while ((bytesRead = in.read(buffer)) != -1) {
                out.write(buffer, 0, bytesRead);
            }
            
        } catch (IOException e) {
            System.err.println("Error copying file: " + e.getMessage());
        }
        // Resources closed in reverse order: out.close(), then in.close()
    }
}

// REAL-WORLD EXAMPLE: Transaction Management
// ========================================

/**
 * Database transaction with automatic rollback
 */
class TransactionManager implements AutoCloseable {
    private final Connection connection;
    private boolean committed = false;
    
    public TransactionManager(DataSource dataSource) throws SQLException {
        this.connection = dataSource.getConnection();
        this.connection.setAutoCommit(false);
        System.out.println("Transaction started");
    }
    
    public void execute(String sql) throws SQLException {
        try (Statement stmt = connection.createStatement()) {
            stmt.execute(sql);
        }
    }
    
    public void commit() throws SQLException {
        connection.commit();
        committed = true;
        System.out.println("Transaction committed");
    }
    
    @Override
    public void close() throws SQLException {
        try {
            if (!committed) {
                connection.rollback();
                System.out.println("Transaction rolled back");
            }
        } finally {
            connection.close();
            System.out.println("Connection closed");
        }
    }
}

/**
 * Usage
 */
class TransactionExample {
    
    public void transferMoney(DataSource dataSource, 
                              String fromAccount, 
                              String toAccount, 
                              double amount) {
        try (TransactionManager tx = new TransactionManager(dataSource)) {
            // Debit from account
            tx.execute("UPDATE accounts SET balance = balance - " + amount + 
                      " WHERE id = '" + fromAccount + "'");
            
            // Credit to account
            tx.execute("UPDATE accounts SET balance = balance + " + amount + 
                      " WHERE id = '" + toAccount + "'");
            
            // Commit if all succeeded
            tx.commit();
            
        } catch (SQLException e) {
            System.err.println("Transfer failed: " + e.getMessage());
            // Transaction automatically rolled back on exception!
        }
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ Automatic rollback if exception thrown
     * ✓ Connection always closed
     * ✓ Clean transaction boundary
     * ✓ No manual cleanup needed
     */
}
```

## 2. Dispose Pattern

### Manual Resource Management

```java
/**
 * DISPOSE PATTERN
 * 
 * Explicit cleanup via dispose() method
 * Used when try-with-resources not suitable
 */

// ========================================
// BASIC DISPOSE PATTERN
// ========================================

/**
 * Object with expensive resources
 */
class ImageEditor {
    private BufferedImage image;
    private Graphics2D graphics;
    private boolean disposed = false;
    
    public ImageEditor(int width, int height) {
        this.image = new BufferedImage(width, height, BufferedImage.TYPE_INT_RGB);
        this.graphics = image.createGraphics();
        // Configure graphics...
    }
    
    public void drawRectangle(int x, int y, int width, int height) {
        checkDisposed();
        graphics.drawRect(x, y, width, height);
    }
    
    public BufferedImage getImage() {
        checkDisposed();
        return image;
    }
    
    /**
     * Explicit dispose method
     */
    public void dispose() {
        if (!disposed) {
            if (graphics != null) {
                graphics.dispose();
                graphics = null;
            }
            image = null;
            disposed = true;
        }
    }
    
    /**
     * Check if already disposed
     */
    private void checkDisposed() {
        if (disposed) {
            throw new IllegalStateException("ImageEditor has been disposed");
        }
    }
    
    /**
     * USAGE:
     * 
     * ImageEditor editor = new ImageEditor(800, 600);
     * try {
     *     editor.drawRectangle(10, 10, 100, 100);
     *     BufferedImage result = editor.getImage();
     * } finally {
     *     editor.dispose();  // Manual cleanup
     * }
     */
}

// ========================================
// DISPOSE + AUTOCLOSEABLE
// ========================================

/**
 * Combine Dispose pattern with AutoCloseable
 * Best of both worlds!
 */
class ResourceManager implements AutoCloseable {
    private final List<Closeable> resources = new ArrayList<>();
    private boolean disposed = false;
    
    public void registerResource(Closeable resource) {
        checkDisposed();
        resources.add(resource);
    }
    
    public void dispose() {
        if (!disposed) {
            // Close all resources (in reverse order)
            List<Closeable> reversed = new ArrayList<>(resources);
            Collections.reverse(reversed);
            
            for (Closeable resource : reversed) {
                try {
                    resource.close();
                } catch (IOException e) {
                    // Log but continue closing others
                    System.err.println("Error closing resource: " + e.getMessage());
                }
            }
            
            resources.clear();
            disposed = true;
        }
    }
    
    @Override
    public void close() {
        dispose();  // AutoCloseable delegates to dispose()
    }
    
    private void checkDisposed() {
        if (disposed) {
            throw new IllegalStateException("ResourceManager disposed");
        }
    }
}

/**
 * Usage - works with both patterns
 */
class ResourceExample {
    
    // Manual dispose
    public void manualDispose() {
        ResourceManager manager = new ResourceManager();
        try {
            manager.registerResource(new FileInputStream("file.txt"));
            manager.registerResource(new FileOutputStream("output.txt"));
            // Use resources...
        } finally {
            manager.dispose();
        }
    }
    
    // Try-with-resources
    public void autoDispose() {
        try (ResourceManager manager = new ResourceManager()) {
            manager.registerResource(new FileInputStream("file.txt"));
            manager.registerResource(new FileOutputStream("output.txt"));
            // Use resources...
        }
        // manager.close() → dispose() called automatically
    }
}

// ========================================
// REAL-WORLD: HTTP Client
// ========================================

/**
 * HTTP client with connection pooling
 */
class HttpClient implements AutoCloseable {
    private final ConnectionPool connectionPool;
    private final ExecutorService executorService;
    private boolean closed = false;
    
    public HttpClient(int maxConnections, int threadPoolSize) {
        this.connectionPool = new ConnectionPool(maxConnections);
        this.executorService = Executors.newFixedThreadPool(threadPoolSize);
    }
    
    public CompletableFuture<HttpResponse> sendAsync(HttpRequest request) {
        checkClosed();
        return CompletableFuture.supplyAsync(
            () -> send(request),
            executorService
        );
    }
    
    public HttpResponse send(HttpRequest request) {
        checkClosed();
        Connection conn = connectionPool.acquire();
        try {
            return conn.execute(request);
        } finally {
            connectionPool.release(conn);
        }
    }
    
    @Override
    public void close() {
        if (!closed) {
            // Shutdown executor (don't accept new tasks)
            executorService.shutdown();
            
            try {
                // Wait for existing tasks to complete (with timeout)
                if (!executorService.awaitTermination(10, TimeUnit.SECONDS)) {
                    executorService.shutdownNow();
                }
            } catch (InterruptedException e) {
                executorService.shutdownNow();
                Thread.currentThread().interrupt();
            }
            
            // Close all connections
            connectionPool.closeAll();
            
            closed = true;
        }
    }
    
    private void checkClosed() {
        if (closed) {
            throw new IllegalStateException("HttpClient is closed");
        }
    }
}

// Helper classes
class ConnectionPool {
    ConnectionPool(int max) {}
    Connection acquire() { return new Connection(); }
    void release(Connection conn) {}
    void closeAll() {}
}
class Connection {
    HttpResponse execute(HttpRequest request) { return new HttpResponse(); }
}
class HttpRequest {}
class HttpResponse {}
```

## 3. Object Pool Pattern

### Reusing Expensive Resources

```java
/**
 * OBJECT POOL PATTERN
 * 
 * Reuse expensive objects instead of creating new ones
 * Common for database connections, threads
 */

// ========================================
// GENERIC OBJECT POOL
// ========================================

/**
 * Generic object pool implementation
 */
class ObjectPool<T> {
    private final BlockingQueue<T> available;
    private final Set<T> inUse;
    private final ObjectFactory<T> factory;
    private final int maxSize;
    
    public ObjectPool(ObjectFactory<T> factory, int initialSize, int maxSize) {
        this.factory = factory;
        this.maxSize = maxSize;
        this.available = new LinkedBlockingQueue<>();
        this.inUse = ConcurrentHashMap.newKeySet();
        
        // Pre-create initial objects
        for (int i = 0; i < initialSize; i++) {
            available.offer(factory.create());
        }
    }
    
    /**
     * Borrow object from pool
     */
    public T acquire() throws InterruptedException {
        T obj = available.poll();
        
        if (obj == null) {
            // Pool empty - create new if under max
            if (inUse.size() < maxSize) {
                obj = factory.create();
            } else {
                // Wait for available object
                obj = available.take();
            }
        }
        
        inUse.add(obj);
        return obj;
    }
    
    /**
     * Return object to pool
     */
    public void release(T obj) {
        if (inUse.remove(obj)) {
            if (factory.validate(obj)) {
                factory.reset(obj);
                available.offer(obj);
            } else {
                // Object invalid - destroy and create new
                factory.destroy(obj);
                available.offer(factory.create());
            }
        }
    }
    
    /**
     * Close pool and destroy all objects
     */
    public void close() {
        // Destroy all available objects
        available.forEach(factory::destroy);
        available.clear();
        
        // Destroy all in-use objects
        inUse.forEach(factory::destroy);
        inUse.clear();
    }
}

/**
 * Factory interface for pool
 */
interface ObjectFactory<T> {
    T create();
    void reset(T obj);
    boolean validate(T obj);
    void destroy(T obj);
}

// ========================================
// REAL-WORLD: CONNECTION POOL
// ========================================

/**
 * Database connection pool
 */
class ConnectionPoolExample {
    
    static class DatabaseConnectionFactory implements ObjectFactory<Connection> {
        private final String url;
        private final String username;
        private final String password;
        
        public DatabaseConnectionFactory(String url, String username, String password) {
            this.url = url;
            this.username = username;
            this.password = password;
        }
        
        @Override
        public Connection create() {
            try {
                return DriverManager.getConnection(url, username, password);
            } catch (SQLException e) {
                throw new RuntimeException("Failed to create connection", e);
            }
        }
        
        @Override
        public void reset(Connection conn) {
            // Reset connection state (rollback transactions, etc.)
            try {
                if (!conn.getAutoCommit()) {
                    conn.rollback();
                    conn.setAutoCommit(true);
                }
            } catch (SQLException e) {
                // Log error
            }
        }
        
        @Override
        public boolean validate(Connection conn) {
            try {
                return conn != null && !conn.isClosed() && conn.isValid(5);
            } catch (SQLException e) {
                return false;
            }
        }
        
        @Override
        public void destroy(Connection conn) {
            try {
                if (conn != null && !conn.isClosed()) {
                    conn.close();
                }
            } catch (SQLException e) {
                // Log error
            }
        }
    }
}

/**
 * Usage
 */
class ConnectionPoolUsage {
    private final ObjectPool<Connection> pool;
    
    public ConnectionPoolUsage(String url, String username, String password) {
        ObjectFactory<Connection> factory = 
            new ConnectionPoolExample.DatabaseConnectionFactory(url, username, password);
        this.pool = new ObjectPool<>(factory, 5, 20);
    }
    
    public List<User> getUsers() {
        Connection conn = null;
        try {
            conn = pool.acquire();
            try (Statement stmt = conn.createStatement();
                 ResultSet rs = stmt.executeQuery("SELECT * FROM users")) {
                
                List<User> users = new ArrayList<>();
                while (rs.next()) {
                    users.add(new User(rs.getString("name")));
                }
                return users;
            }
        } catch (SQLException | InterruptedException e) {
            throw new RuntimeException("Failed to get users", e);
        } finally {
            if (conn != null) {
                pool.release(conn);
            }
        }
    }
    
    public void close() {
        pool.close();
    }
}
```

## 4. Resource Leak Prevention

### Common Mistakes and Detection

```java
/**
 * RESOURCE LEAK PREVENTION
 * 
 * Common mistakes that cause resource leaks
 * How to detect and prevent them
 */

// ========================================
// COMMON LEAK #1: Forgetting to Close
// ========================================

/**
 * ❌ LEAK: Resource not closed
 */
class Leak1_ForgettingClose {
    
    public void badReadFile(String path) throws IOException {
        FileInputStream fis = new FileInputStream(path);
        int data = fis.read();
        // ❌ Never closed! Resource leak!
    }
    
    /**
     * CONSEQUENCE:
     * • File descriptor leak
     * • After ~1000 calls: "Too many open files"
     * • Application crash
     */
}

/**
 * ✓ FIX: Use try-with-resources
 */
class Fix1_UseAutoClose {
    
    public void goodReadFile(String path) throws IOException {
        try (FileInputStream fis = new FileInputStream(path)) {
            int data = fis.read();
        }
        // ✓ Automatically closed
    }
}

// ========================================
// COMMON LEAK #2: Exception Before Close
// ========================================

/**
 * ❌ LEAK: Exception thrown before close
 */
class Leak2_ExceptionBeforeClose {
    
    public void badProcess(String path) throws IOException {
        FileInputStream fis = new FileInputStream(path);
        
        // Exception thrown here
        if (someCondition()) {
            throw new IOException("Error!");  // ❌ fis never closed!
        }
        
        fis.close();  // Never reached if exception thrown
    }
    
    private boolean someCondition() { return true; }
}

/**
 * ✓ FIX: Try-with-resources or try-finally
 */
class Fix2_TryFinally {
    
    public void goodProcess(String path) throws IOException {
        try (FileInputStream fis = new FileInputStream(path)) {
            if (someCondition()) {
                throw new IOException("Error!");
            }
        }
        // ✓ fis closed even if exception thrown
    }
    
    private boolean someCondition() { return true; }
}

// ========================================
// COMMON LEAK #3: Incomplete Cleanup
// ========================================

/**
 * ❌ LEAK: Only partial cleanup
 */
class Leak3_IncompleteCleanup {
    private FileInputStream input;
    private FileOutputStream output;
    
    public void badCopy() throws IOException {
        input = new FileInputStream("source.txt");
        output = new FileOutputStream("dest.txt");
        
        byte[] buffer = new byte[1024];
        int bytesRead = input.read(buffer);
        output.write(buffer, 0, bytesRead);
        
        input.close();
        // ❌ What if exception thrown here?
        output.close();  // Never reached! Leak!
    }
}

/**
 * ✓ FIX: Close in finally or use try-with-resources
 */
class Fix3_CompleteCleanup {
    
    public void goodCopy() throws IOException {
        try (FileInputStream input = new FileInputStream("source.txt");
             FileOutputStream output = new FileOutputStream("dest.txt")) {
            
            byte[] buffer = new byte[1024];
            int bytesRead = input.read(buffer);
            output.write(buffer, 0, bytesRead);
        }
        // ✓ Both closed, even if exception thrown
    }
}

// ========================================
// COMMON LEAK #4: Collection Holding References
// ========================================

/**
 * ❌ LEAK: Collection prevents garbage collection
 */
class Leak4_CollectionReferences {
    private static final Map<String, byte[]> cache = new HashMap<>();
    
    public void addToCache(String key, byte[] data) {
        cache.put(key, data);
        // ❌ Data never removed! Memory leak!
        // Cache grows forever until OutOfMemoryError
    }
}

/**
 * ✓ FIX: Use bounded cache or weak references
 */
class Fix4_BoundedCache {
    private static final Map<String, byte[]> cache = 
        new LinkedHashMap<String, byte[]>(16, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<String, byte[]> eldest) {
                return size() > 1000;  // Max 1000 entries
            }
        };
    
    public void addToCache(String key, byte[] data) {
        cache.put(key, data);
        // ✓ Old entries automatically removed
    }
}

// ========================================
// COMMON LEAK #5: Listener Not Unregistered
// ========================================

/**
 * ❌ LEAK: Event listener never removed
 */
class Leak5_ListenerLeak {
    
    static class EventSource {
        private final List<EventListener> listeners = new ArrayList<>();
        
        public void addListener(EventListener listener) {
            listeners.add(listener);
        }
        
        // ❌ No removeListener method!
    }
    
    interface EventListener {
        void onEvent(String event);
    }
    
    public void useEventSource(EventSource source) {
        EventListener listener = event -> System.out.println(event);
        source.addListener(listener);
        
        // ❌ Listener never removed!
        // EventSource keeps reference to listener forever
        // Listener (and its enclosing object) can't be GC'd
    }
}

/**
 * ✓ FIX: Provide unregister method
 */
class Fix5_UnregisterListener {
    
    static class EventSource {
        private final List<EventListener> listeners = new ArrayList<>();
        
        public void addListener(EventListener listener) {
            listeners.add(listener);
        }
        
        public void removeListener(EventListener listener) {
            listeners.remove(listener);
        }
    }
    
    interface EventListener {
        void onEvent(String event);
    }
    
    public void useEventSource(EventSource source) {
        EventListener listener = event -> System.out.println(event);
        
        try {
            source.addListener(listener);
            // Use source...
        } finally {
            source.removeListener(listener);  // ✓ Cleanup
        }
    }
}
```

## 5. Finalizer and Cleaner (AVOID!)

### Why They're Dangerous

```java
/**
 * FINALIZERS AND CLEANERS
 * 
 * finalize() and Cleaner API
 * Almost always the wrong choice
 */

// ========================================
// ❌ FINALIZER (DEPRECATED, DON'T USE!)
// ========================================

/**
 * Old way: finalize() method (deprecated since Java 9)
 */
class BadFinalizerExample {
    private FileInputStream stream;
    
    public BadFinalizerExample(String path) throws IOException {
        this.stream = new FileInputStream(path);
    }
    
    @Override
    protected void finalize() throws Throwable {
        try {
            if (stream != null) {
                stream.close();  // ❌ BAD! Don't rely on finalize!
            }
        } finally {
            super.finalize();
        }
    }
    
    /**
     * PROBLEMS WITH FINALIZERS:
     * 
     * 1. UNPREDICTABLE
     *    • finalize() called by GC (non-deterministic)
     *    • May never be called!
     *    • May be called hours/days later
     *    • Can't rely on timing
     * 
     * 2. PERFORMANCE
     *    • Finalization is slow (10-100x slower GC)
     *    • Objects with finalizers live longer
     *    • Take up more memory
     * 
     * 3. EXCEPTIONS
     *    • Exceptions in finalize() are ignored!
     *    • No way to handle errors
     * 
     * 4. RESURRECTION
     *    • finalize() can resurrect objects
     *    • Breaks GC assumptions
     * 
     * 5. SECURITY
     *    • Finalize attacks possible
     *    • Can access partially constructed objects
     */
}

// ========================================
// ❌ CLEANER (JAVA 9+, STILL AVOID!)
// ========================================

/**
 * Replacement for finalize(): Cleaner API
 * Still problematic, but less so
 */
class CleanerExample implements AutoCloseable {
    private static final Cleaner cleaner = Cleaner.create();
    
    private final State state;
    private final Cleaner.Cleanable cleanable;
    
    // State class must NOT reference outer class!
    private static class State implements Runnable {
        private FileInputStream stream;
        
        State(String path) throws IOException {
            this.stream = new FileInputStream(path);
        }
        
        @Override
        public void run() {
            // Cleanup action (executed by cleaner thread)
            try {
                if (stream != null) {
                    stream.close();
                }
            } catch (IOException e) {
                // Handle exception
            }
        }
    }
    
    public CleanerExample(String path) throws IOException {
        this.state = new State(path);
        this.cleanable = cleaner.register(this, state);
    }
    
    @Override
    public void close() {
        cleanable.clean();  // Explicit cleanup
    }
    
    /**
     * PROBLEMS WITH CLEANERS:
     * 
     * 1. Still non-deterministic (same as finalizers)
     * 2. Complex to implement correctly
     * 3. Easy to create memory leaks (if State references outer class)
     * 4. Still performance overhead
     * 
     * 
     * WHEN TO USE CLEANER:
     * 
     * • Safety net only (backup to explicit close())
     * • Native resources (JNI)
     * • Never as primary cleanup mechanism
     */
}

// ========================================
// ✓ BETTER: EXPLICIT CLEANUP
// ========================================

/**
 * Proper resource management: try-with-resources
 */
class ProperCleanupExample implements AutoCloseable {
    private final FileInputStream stream;
    
    public ProperCleanupExample(String path) throws IOException {
        this.stream = new FileInputStream(path);
    }
    
    @Override
    public void close() throws IOException {
        stream.close();
    }
    
    /**
     * USAGE:
     * 
     * try (ProperCleanupExample resource = new ProperCleanupExample("file.txt")) {
     *     // Use resource
     * }
     * // ✓ close() called immediately and deterministically!
     */
}

// ========================================
// CLEANER AS SAFETY NET (RARE)
// ========================================

/**
 * Only acceptable use: as safety net
 */
class CleanerAsSafetyNet implements AutoCloseable {
    private static final Cleaner cleaner = Cleaner.create();
    
    private final State state;
    private final Cleaner.Cleanable cleanable;
    
    private static class State implements Runnable {
        private FileInputStream stream;
        
        State(String path) throws IOException {
            this.stream = new FileInputStream(path);
        }
        
        @Override
        public void run() {
            // Safety net cleanup
            try {
                if (stream != null) {
                    System.err.println("WARNING: Resource not closed properly!");
                    stream.close();
                }
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
    
    public CleanerAsSafetyNet(String path) throws IOException {
        this.state = new State(path);
        this.cleanable = cleaner.register(this, state);
    }
    
    @Override
    public void close() throws IOException {
        cleanable.clean();  // Primary cleanup (explicit)
        state.stream = null;  // Mark as cleaned
    }
    
    /**
     * DESIGN:
     * 
     * 1. Primary: Explicit close() via try-with-resources
     * 2. Backup: Cleaner runs if close() not called
     * 3. Warning: Log if cleaner runs (indicates bug)
     * 
     * This catches programming errors in tests/development
     */
}

/**
 * SUMMARY: FINALIZERS AND CLEANERS
 * 
 * finalize():
 * ❌ Deprecated (Java 9+)
 * ❌ Don't use ever
 * 
 * Cleaner:
 * ⚠️  Use only as safety net
 * ✓  Better than finalize() but still problematic
 * 
 * Proper approach:
 * ✓  Try-with-resources
 * ✓  Explicit close()
 * ✓  AutoCloseable interface
 */
```

## 6. Weak Reference Cleanup

### Memory-Sensitive Caches

```java
/**
 * WEAK REFERENCES
 * 
 * References that don't prevent garbage collection
 * Useful for caches that should release under memory pressure
 */

// ========================================
// REFERENCE TYPES IN JAVA
// ========================================

/**
 * Java has 4 types of references:
 * 
 * 1. STRONG (default)
 *    • Regular references
 *    • Object not GC'd while strongly reachable
 * 
 * 2. SOFT
 *    • GC'd when memory needed
 *    • Good for memory-sensitive caches
 * 
 * 3. WEAK
 *    • GC'd in next GC cycle
 *    • Good for canonical mappings
 * 
 * 4. PHANTOM
 *    • Already GC'd, used for cleanup
 *    • Rarely used directly
 */

// ========================================
// WEAK REFERENCE BASICS
// ========================================

/**
 * Basic weak reference usage
 */
class WeakReferenceBasics {
    
    public void demonstrateWeakReference() {
        // Create object with strong reference
        Object strongRef = new Object();
        
        // Create weak reference
        WeakReference<Object> weakRef = new WeakReference<>(strongRef);
        
        System.out.println("Strong ref: " + strongRef);
        System.out.println("Weak ref: " + weakRef.get());
        
        // Remove strong reference
        strongRef = null;
        
        // Force GC
        System.gc();
        
        // Weak reference now returns null
        System.out.println("After GC: " + weakRef.get());  // null
    }
}

// ========================================
// WEAKHASHMAP
// ========================================

/**
 * WeakHashMap - Map with weak keys
 */
class WeakHashMapExample {
    
    /**
     * Keys are weak references
     * Entry removed when key is GC'd
     */
    public void demonstrateWeakHashMap() {
        Map<Object, String> map = new WeakHashMap<>();
        
        Object key1 = new Object();
        Object key2 = new Object();
        
        map.put(key1, "Value 1");
        map.put(key2, "Value 2");
        
        System.out.println("Size: " + map.size());  // 2
        
        // Remove strong reference to key1
        key1 = null;
        
        // Force GC
        System.gc();
        
        // key1 entry removed automatically
        System.out.println("Size after GC: " + map.size());  // 1
    }
    
    /**
     * Use case: Canonical mapping
     * 
     * Example: String interning, class metadata
     */
    private static final Map<String, WeakReference<ExpensiveObject>> cache = 
        new WeakHashMap<>();
    
    public ExpensiveObject getOrCreate(String key) {
        WeakReference<ExpensiveObject> ref = cache.get(key);
        ExpensiveObject obj = (ref != null) ? ref.get() : null;
        
        if (obj == null) {
            obj = new ExpensiveObject(key);
            cache.put(key, new WeakReference<>(obj));
        }
        
        return obj;
    }
}

// ========================================
// SOFT REFERENCE CACHE
// ========================================

/**
 * Soft references for memory-sensitive cache
 */
class SoftReferenceCache<K, V> {
    private final Map<K, SoftReference<V>> cache = new ConcurrentHashMap<>();
    private final ReferenceQueue<V> queue = new ReferenceQueue<>();
    
    /**
     * Put value in cache
     */
    public void put(K key, V value) {
        // Clean up GC'd entries first
        cleanUp();
        
        // Store with soft reference
        cache.put(key, new SoftReference<>(value, queue));
    }
    
    /**
     * Get value from cache
     */
    public Optional<V> get(K key) {
        cleanUp();
        
        SoftReference<V> ref = cache.get(key);
        if (ref != null) {
            V value = ref.get();
            if (value != null) {
                return Optional.of(value);
            } else {
                // Reference GC'd, remove entry
                cache.remove(key);
            }
        }
        
        return Optional.empty();
    }
    
    /**
     * Remove GC'd entries
     */
    private void cleanUp() {
        Reference<? extends V> ref;
        while ((ref = queue.poll()) != null) {
            // Find and remove corresponding cache entry
            cache.values().removeIf(r -> r == ref);
        }
    }
    
    /**
     * Clear cache
     */
    public void clear() {
        cache.clear();
    }
    
    /**
     * BENEFITS:
     * 
     * ✓ Cache releases entries under memory pressure
     * ✓ Automatic cleanup (no manual eviction needed)
     * ✓ No OutOfMemoryError from cache
     * 
     * 
     * TRADEOFFS:
     * 
     * ✗ Non-deterministic eviction
     * ✗ Can't control eviction order
     * ✗ Soft refs kept longer than weak refs
     */
}

// ========================================
// REAL-WORLD: IMAGE CACHE
// ========================================

/**
 * Image cache using soft references
 */
class ImageCache {
    private final Map<String, SoftReference<BufferedImage>> cache = 
        new ConcurrentHashMap<>();
    
    public BufferedImage getImage(String path) throws IOException {
        // Check cache
        SoftReference<BufferedImage> ref = cache.get(path);
        if (ref != null) {
            BufferedImage image = ref.get();
            if (image != null) {
                return image;  // Cache hit
            }
        }
        
        // Cache miss - load image
        BufferedImage image = ImageIO.read(new File(path));
        cache.put(path, new SoftReference<>(image));
        return image;
    }
    
    /**
     * BEHAVIOR:
     * 
     * • Images stay in cache while memory available
     * • GC releases images when memory needed
     * • Transparent to caller (always returns image)
     * • No manual cache size management
     */
}

// ========================================
// WHEN TO USE WEAK/SOFT REFERENCES
// ========================================

/**
 * WEAK REFERENCES:
 * 
 * ✓ Use for:
 * • Canonical mappings (WeakHashMap)
 * • Reverse lookups
 * • Metadata that follows object lifetime
 * 
 * ✗ Don't use for:
 * • General caching (use soft references)
 * • Objects you want to keep
 * 
 * 
 * SOFT REFERENCES:
 * 
 * ✓ Use for:
 * • Memory-sensitive caches
 * • Large objects (images, parsed documents)
 * • Derived data (can be recomputed)
 * 
 * ✗ Don't use for:
 * • Critical data (may be GC'd)
 * • Small objects (overhead not worth it)
 * 
 * 
 * STRONG REFERENCES (DEFAULT):
 * 
 * ✓ Use for:
 * • Everything else!
 * • Most use cases
 * • Critical data
 */

class ExpensiveObject {
    ExpensiveObject(String key) {}
}
```
