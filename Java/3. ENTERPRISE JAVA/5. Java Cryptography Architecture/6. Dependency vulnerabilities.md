# Dependency Vulnerabilities

## Overview

Dependency vulnerabilities are a critical attack vector in modern applications—third-party libraries can introduce security flaws affecting millions of applications. This guide covers six essential dependency security concepts: OWASP Top 10 (most critical web application risks—injection, broken authentication, sensitive data exposure, XML external entities, broken access control, security misconfiguration, XSS, insecure deserialization, insufficient logging, using components with known vulnerabilities), Dependency Scanning Tools (automated vulnerability detection—OWASP Dependency-Check scans JARs against CVE database, Snyk monitors continuously, GitHub Dependabot creates PRs for updates, Maven/Gradle plugins in CI/CD, fail builds on high-severity CVEs), CVE Tracking (vulnerability database—Common Vulnerabilities and Exposures unique identifiers, National Vulnerability Database provides CVSS scores, track CVE-2021-44228 Log4Shell example, severity: Critical/High/Medium/Low, monitor security advisories), Keeping Dependencies Updated (maintenance strategy—semantic versioning guides updates, test thoroughly before updating, automated dependency bots help, balance security vs stability, security patches take priority, document known vulnerabilities), Security Policies in Build (enforce in CI/CD—Maven Enforcer Plugin fails on vulnerable versions, Gradle security plugins scan dependencies, block builds on CVSS > 7.0, require dependency review for PRs, automated security gates), and Security Audits (comprehensive assessment—regular dependency audits quarterly, penetration testing by professionals, vulnerability disclosure programs, bug bounties, third-party security reviews, document findings and remediation). The key insight: 80% of code is third-party dependencies—one vulnerable library can compromise entire application, transitive dependencies multiply risk exponentially, Log4Shell affected millions of applications worldwide. Common mistakes: ignoring security warnings (they matter!), outdated dependencies (years old), no scanning in CI/CD (vulnerabilities deploy to production), blind updates (test first!), not checking transitive dependencies (hidden vulnerabilities), assuming libraries are secure (verify!). The relationship between concepts: Dependency added → Scanner detects CVE → CVSS score assessed → Policy enforces action → Update applied → Audit validates → Build secured. Real-world security strategy: automated scanning in CI/CD (every build), Dependabot PRs for updates, fail builds on Critical/High CVEs, weekly dependency reviews, quarterly security audits, subscribe to security advisories, test updates in staging first, document exceptions with approval.

## 2. Dependency Scanning Tools

### Automated Vulnerability Detection

```xml
<!-- ======================================== -->
<!-- MAVEN: OWASP DEPENDENCY-CHECK PLUGIN    -->
<!-- ======================================== -->

<!--
Add to pom.xml:
-->
<build>
    <plugins>
        <plugin>
            <groupId>org.owasp</groupId>
            <artifactId>dependency-check-maven</artifactId>
            <version>8.4.3</version>
            <configuration>
                <!-- Fail build on CVSS >= 7 (High/Critical) -->
                <failBuildOnCVSS>7</failBuildOnCVSS>
                
                <!-- Suppression file for false positives -->
                <suppressionFiles>
                    <suppressionFile>dependency-check-suppressions.xml</suppressionFile>
                </suppressionFiles>
                
                <!-- Report formats -->
                <formats>
                    <format>HTML</format>
                    <format>JSON</format>
                </formats>
                
                <!-- Skip test dependencies -->
                <skipTestScope>true</skipTestScope>
                
                <!-- Enable experimental analyzers -->
                <assemblyAnalyzerEnabled>true</assemblyAnalyzerEnabled>
            </configuration>
            <executions>
                <execution>
                    <goals>
                        <goal>check</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

<!--
Run manually:
```

## 4. Keeping Dependencies Updated

### Safe Update Strategies

```java
// ========================================
// SEMANTIC VERSIONING
// ========================================

/**
 * SEMANTIC VERSIONING: MAJOR.MINOR.PATCH
 * 
 * Example: 2.5.3
 * - MAJOR (2): Breaking changes
 * - MINOR (5): New features (backward compatible)
 * - PATCH (3): Bug fixes
 * 
 * Update Strategy:
 * - PATCH: Update immediately (bug fixes, security)
 * - MINOR: Update regularly (new features, safe)
 * - MAJOR: Plan carefully (breaking changes)
 * 
 * Version Ranges (Maven/Gradle):
 * - [1.0.0]: Exact version (avoid)
 * - [1.0.0,2.0.0): 1.x versions
 * - [1.0.0,): 1.0.0 or higher (risky!)
```

## 5. Security Policies in Builds

### Enforcing Security in CI/CD

```xml
<!-- ======================================== -->
<!-- MAVEN ENFORCER PLUGIN                    -->
<!-- ======================================== -->

<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-enforcer-plugin</artifactId>
    <version>3.4.1</version>
    <executions>
        <execution>
            <id>enforce-security</id>
            <goals>
                <goal>enforce</goal>
            </goals>
            <configuration>
                <rules>
                    <!-- Require Maven version -->
                    <requireMavenVersion>
                        <version>[3.6.0,)</version>
                    </requireMavenVersion>
                    
                    <!-- Require Java version -->
                    <requireJavaVersion>
                        <version>[17,)</version>
                    </requireJavaVersion>
                    
                    <!-- Ban vulnerable dependencies -->
                    <bannedDependencies>
                        <excludes>
                            <!-- Ban old Log4j versions -->
                            <exclude>org.apache.logging.log4j:log4j-core:[,2.17.0)</exclude>
                            
                            <!-- Ban vulnerable Struts -->
                            <exclude>org.apache.struts:struts2-core:[,2.5.26)</exclude>
                            
                            <!-- Ban old Spring versions -->
                            <exclude>org.springframework:spring-core:[,5.3.18)</exclude>
                            
                            <!-- Ban vulnerable Jackson -->
                            <exclude>com.fasterxml.jackson.core:jackson-databind:[,2.12.6.1)</exclude>
                            
                            <!-- Ban Commons Collections with gadgets -->
                            <exclude>commons-collections:commons-collections:[,3.2.2)</exclude>
                        </excludes>
                        <message>
                            Security violation: Banned dependency detected!
                            This dependency has known vulnerabilities.
                            Please update to a secure version.
                        </message>
                    </bannedDependencies>
                    
                    <!-- Require dependency convergence -->
                    <dependencyConvergence/>
                    
                    <!-- No snapshots in release -->
                    <requireReleaseDeps>
                        <onlyWhenRelease>true</onlyWhenRelease>
                    </requireReleaseDeps>
                </rules>
                <fail>true</fail>
            </configuration>
        </execution>
    </executions>
</plugin>
```

```groovy
    formats = ['HTML', 'JSON', 'JUNIT']
    
    // Suppress false positives
    suppressionFile = 'dependency-check-suppressions.xml'
}

// Custom task to check security
task checkSecurity {
    dependsOn 'dependencyCheckAnalyze'
    
    doLast {
        def reportFile = file('build/reports/dependency-check-report.json')
        if (reportFile.exists()) {
            def json = new JsonSlurper().parse(reportFile)
            def highSeverity = json.dependencies.findAll { dep ->
                dep.vulnerabilities?.any { vuln ->
                    vuln.cvssv3?.baseScore >= 7.0
                }
            }
            
            if (!highSeverity.isEmpty()) {
                throw new GradleException(
                    "High severity vulnerabilities found: ${highSeverity.size()}"
                )
            }
        }
    }
}

// Run on build
build.dependsOn checkSecurity
```
