# Secure Coding Practices

## Overview

Secure coding is the difference between a protected application and a compromised system—essential for preventing the OWASP Top 10 vulnerabilities that account for most security breaches. This guide covers seven critical security practices: Input Validation (first line of defense—whitelist over blacklist, validate length/format/range, sanitize all user input, reject invalid data early, defense in depth), SQL Injection Prevention (most common attack—ALWAYS use PreparedStatement, NEVER concatenate SQL, parameterized queries prevent 100% of SQL injection, stored procedures with parameters, ORM frameworks help but aren't foolproof), XSS Prevention (code execution in browsers—escape output context-aware, use Content Security Policy, sanitize HTML input, HttpOnly cookies, validate on server not just client), CSRF Protection (state-changing request forgery—synchronizer token pattern, SameSite cookies, double-submit cookies, verify Origin/Referer headers, Spring Security CSRF enabled by default), Authentication and Authorization (who you are vs what you can do—multi-factor authentication, session management, principle of least privilege, role-based access control, OAuth2/OpenID Connect for modern apps), Secure Password Storage (protecting credentials—NEVER plain text or plain hash, ALWAYS use BCrypt/Argon2/PBKDF2, salts prevent rainbow tables, work factors slow brute force, pepper adds server-side secret), and Secrets Management (protecting sensitive data—never commit secrets to Git, use environment variables or vaults, rotate secrets regularly, encrypt at rest, audit access, least privilege for services). The key insight: security is about defense in depth—multiple layers of protection because no single control is perfect, assume breach mentality, validate everything. Common mistakes: trusting user input (always assume malicious), storing passwords in plain text (catastrophic breach), SQL concatenation (trivial injection), missing CSRF tokens (state manipulation), weak session management (session hijacking), hardcoded secrets (exposed in Git), insufficient authorization checks (privilege escalation), client-side only validation (easily bypassed). The relationship between concepts: input validation prevents malicious data → parameterized queries stop SQL injection → output encoding prevents XSS → CSRF tokens protect state → authentication proves identity → authorization controls access → secure storage protects credentials → secrets management protects keys. Real-world security strategy: validate all inputs with whitelists, use PreparedStatement exclusively, encode all outputs, enable CSRF protection, implement proper authentication (JWT/session), use role-based authorization, store passwords with BCrypt (work factor 12+), manage secrets in vault (HashiCorp Vault/AWS Secrets Manager), follow OWASP guidelines, conduct security reviews, penetration testing, security training for developers.

## 1. Input Validation

### First Line of Defense

```java
/**
 * INPUT VALIDATION: Prevent malicious data entry
 * 
 * Principles:
 * - Whitelist over blacklist
 * - Validate on server (client validation is UX, not security)
 * - Reject invalid input (don't try to "fix" it)
 * - Validate early (fail fast)
 * - Defense in depth (validate at multiple layers)
 */

import javax.validation.constraints.*;
import org.springframework.validation.annotation.Validated;

// ========================================
// VALIDATION ANNOTATIONS (Bean Validation)
// ========================================

@Validated
public class UserRegistrationRequest {
    
    @NotBlank(message = "Username is required")
    @Size(min = 3, max = 20, message = "Username must be 3-20 characters")

    private String username;

    @Email(message = "Invalid email format")
    private String email;

    @Pattern(
        regexp = "^(?=.*[a-z])(?=.*[A-Z])(?=.*\\d)(?=.*[@$!%*?&])[A-Za-z\\d@$!%*?&]+$",
        message = "Password must contain uppercase, lowercase, digit, and special character"
    )
    private String password;
    
    @Min(value = 18, message = "Must be at least 18 years old")
    @Max(value = 120, message = "Invalid age")
    private Integer age;
    
    @Pattern(regexp = "^\\+?[1-9]\\d{1,14}$", message = "Invalid phone number")
    private String phoneNumber;
    
    // Getters/Setters
}

    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};

public class SafeHtmlValidator implements ConstraintValidator<SafeHtml, String> {
    
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null) {
            return true;  // Use @NotNull separately
        }
        
        // Check for dangerous patterns
        String dangerous = "(?i)<script|javascript:|onerror=|onload=|<iframe|<object|<embed";
        return !value.matches(".*(" + dangerous + ").*");
    }
}

// WHITELIST VALIDATION (Preferred)
// ========================================

public class WhitelistValidation {
    
    /**
     * ✅ GOOD: Whitelist - only allow known good
     */
    public boolean isValidUsername(String username) {
        // Only allow alphanumeric, underscore, dash
        return username != null && username.matches("^[a-zA-Z0-9_-]{3,20}$");
    }
    
    public boolean isValidEmail(String email) {
        // Simple email pattern (or use @Email annotation)
        return email != null && email.matches("^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}$");
    }
    
    /**
     * ❌ BAD: Blacklist - try to block all bad patterns
     */
    public boolean isValidUsernameBlacklist(String username) {
        // Trying to block bad characters - always incomplete!
        if (username.contains("<")) return false;
        if (username.contains(">")) return false;
        if (username.contains("'")) return false;
        if (username.contains("\"")) return false;
        if (username.contains(";")) return false;
        // What about: `, |, &, $, %, etc.? Impossible to block everything!
        
        return true;  // ❌ INSECURE - allows many dangerous characters
    }
}

// VALIDATION TYPES
// ========================================

public class ValidationExamples {
    
    // Length validation
    public void validateLength(String input, int min, int max) {
        if (input == null || input.length() < min || input.length() > max) {
            throw new ValidationException("Invalid length");
        }
    }
    
    // Format validation
    public void validateFormat(String input, String regex) {
        if (input == null || !input.matches(regex)) {
            throw new ValidationException("Invalid format");
        }
    }
    
    // Range validation
    public void validateRange(int value, int min, int max) {
        if (value < min || value > max) {
            throw new ValidationException("Value out of range");
        }
    }
    
    // Enum validation (type-safe)
    public enum UserRole {
        USER, ADMIN, MODERATOR
    }
    
    public void validateRole(String role) {
        try {
            UserRole.valueOf(role);  // Throws if invalid
        } catch (IllegalArgumentException e) {
            throw new ValidationException("Invalid role");
        }
    }
    
    // File upload validation
    public void validateFileUpload(MultipartFile file) {
        // Check file is not empty
        if (file.isEmpty()) {
            throw new ValidationException("File is empty");
        }
        
        // Check file size (e.g., max 5MB)
        if (file.getSize() > 5 * 1024 * 1024) {
            throw new ValidationException("File too large");
        }
        
        // Check file type (whitelist)
        String contentType = file.getContentType();
        List<String> allowedTypes = Arrays.asList("image/jpeg", "image/png", "application/pdf");
        if (!allowedTypes.contains(contentType)) {
            throw new ValidationException("Invalid file type");
        }
        
        // Check file extension (defense in depth)
        String filename = file.getOriginalFilename();
        if (filename == null || !filename.matches(".*\\.(jpg|jpeg|png|pdf)$")) {
            throw new ValidationException("Invalid file extension");
        }
    }
}

// ========================================
// PATH TRAVERSAL PREVENTION
// ========================================

public class PathTraversalPrevention {
    
    /**
     * ❌ VULNERABLE: Path traversal attack
     */
    @GetMapping("/files/{filename}")
    public ResponseEntity<Resource> downloadFileVulnerable(@PathVariable String filename) {
        // Attacker sends: ../../etc/passwd
        Path path = Paths.get("/app/uploads/" + filename);  // ❌ DANGEROUS!
        Resource resource = new FileSystemResource(path.toFile());
        return ResponseEntity.ok(resource);
    }
    
    /**
     * ✅ SECURE: Validate and sanitize filename
     */
    @GetMapping("/files/{filename}")
    public ResponseEntity<Resource> downloadFileSecure(@PathVariable String filename) {
        // Validate filename (whitelist)
        if (!filename.matches("^[a-zA-Z0-9_.-]+$")) {
            throw new ValidationException("Invalid filename");
        }
        
        // Resolve path and check it's within allowed directory
        Path uploadDir = Paths.get("/app/uploads").toAbsolutePath().normalize();
        Path filePath = uploadDir.resolve(filename).normalize();
        
        // Check path is still within upload directory (prevent ../)
        if (!filePath.startsWith(uploadDir)) {
            throw new SecurityException("Path traversal attempt detected");
        }
        
        Resource resource = new FileSystemResource(filePath.toFile());
        return ResponseEntity.ok(resource);
    }
}
```

## 2. SQL Injection Prevention

### Parameterized Queries

```java
/**
 * SQL INJECTION: Most common and dangerous attack
 * 
 * Attack example:
 * username: admin' OR '1'='1' --
 * Query: SELECT * FROM users WHERE username='admin' OR '1'='1' --' AND password='...'
 * Result: Returns all users, bypassing authentication!
 * 
 * Prevention: ALWAYS use parameterized queries (PreparedStatement)
 */

import java.sql.*;

// ========================================
// ❌ VULNERABLE CODE (String Concatenation)
// ========================================

public class VulnerableSqlExample {
    
    /**
     * ❌ CRITICAL VULNERABILITY: SQL Injection
     */
    public User loginVulnerable(String username, String password) throws SQLException {
        Connection conn = dataSource.getConnection();
        
        // ❌ NEVER DO THIS!
        String sql = "SELECT * FROM users WHERE username='" + username + "' AND password='" + password + "'";
        
        Statement stmt = conn.createStatement();
        ResultSet rs = stmt.executeQuery(sql);
        
        // Attacker input:
        // username: admin' OR '1'='1' --
        // password: anything
        // 
        // Resulting SQL:
        // SELECT * FROM users WHERE username='admin' OR '1'='1' --' AND password='anything'
        //                                              ↑ always true    ↑ commented out
        // 
        // Result: Returns admin user without knowing password!
        
        if (rs.next()) {
            return extractUser(rs);
        }
        return null;
    }
    
    /**
     * ❌ CRITICAL: Data deletion
     */
    public void deleteUserVulnerable(String userId) throws SQLException {
        Connection conn = dataSource.getConnection();
        
        // ❌ NEVER DO THIS!
        String sql = "DELETE FROM users WHERE id=" + userId;
        Statement stmt = conn.createStatement();
        stmt.executeUpdate(sql);
        
        // Attacker input:
        // userId: 1 OR 1=1
        // 
        // Resulting SQL:
        // DELETE FROM users WHERE id=1 OR 1=1
        //                             ↑ deletes ALL users!
    }
}

// ========================================
// ✅ SECURE CODE (PreparedStatement)
// ========================================

public class SecureSqlExample {
    
    /**
     * ✅ SECURE: PreparedStatement with parameters
     */
    public User loginSecure(String username, String password) throws SQLException {
        Connection conn = dataSource.getConnection();
        
        // ✅ Use PreparedStatement with ? placeholders
        String sql = "SELECT * FROM users WHERE username = ? AND password = ?";
        PreparedStatement pstmt = conn.prepareStatement(sql);
        
        // Set parameters (automatically escaped)
        pstmt.setString(1, username);
        pstmt.setString(2, password);
        
        ResultSet rs = pstmt.executeQuery();
        
        // Attacker input:
        // username: admin' OR '1'='1' --
        // password: anything
        // 
        // PreparedStatement treats entire input as string literal:
        // SELECT * FROM users WHERE username='admin'' OR ''1''=''1'' --' AND password='anything'
        //                                       ↑ escaped single quote
        // 
        // Result: No user found (injection prevented!)
        
        if (rs.next()) {
            return extractUser(rs);
        }
        return null;
    }
    
    /**
     * ✅ SECURE: DELETE with PreparedStatement
     */
    public void deleteUserSecure(String userId) throws SQLException {
        Connection conn = dataSource.getConnection();
        
        String sql = "DELETE FROM users WHERE id = ?";
        PreparedStatement pstmt = conn.prepareStatement(sql);
        pstmt.setString(1, userId);
        pstmt.executeUpdate();
        
        // Input "1 OR 1=1" is treated as literal string, not SQL
    }
    
    /**
     * ✅ SECURE: Dynamic WHERE clause
     */
    public List<User> searchUsersSecure(String searchTerm, String role) throws SQLException {
        StringBuilder sql = new StringBuilder("SELECT * FROM users WHERE 1=1");
        List<Object> params = new ArrayList<>();
        
        if (searchTerm != null && !searchTerm.isEmpty()) {
            sql.append(" AND (username LIKE ? OR email LIKE ?)");
            params.add("%" + searchTerm + "%");
            params.add("%" + searchTerm + "%");
        }
        
        if (role != null && !role.isEmpty()) {
            sql.append(" AND role = ?");
            params.add(role);
        }
        
        PreparedStatement pstmt = conn.prepareStatement(sql.toString());
        for (int i = 0; i < params.size(); i++) {
            pstmt.setObject(i + 1, params.get(i));
        }
        
        ResultSet rs = pstmt.executeQuery();
        return extractUsers(rs);
    }
}

// JPA/HIBERNATE (Parameterized)
// ========================================

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    List<User> findByRoleAndActiveTrue(String role);
    
    // ✅ SECURE: @Query with named parameters
    @Query("SELECT u FROM User u WHERE u.username = :username AND u.email = :email")

                                          @Param("email") String email);
    
    // ✅ SECURE: Native query with parameters
    @Query(value = "SELECT * FROM users WHERE role = ?1 AND created_date > ?2", 
           nativeQuery = true)
    List<User> findByRoleAfterDate(String role, LocalDateTime date);
    
    // ❌ VULNERABLE: String concatenation in @Query

        Connection conn = dataSource.getConnection();
        
        // Call stored procedure with parameters
        String sql = "{call update_user_status(?, ?)}";

        cstmt.setInt(1, userId);

        // END;
    }
}
```

## 3. XSS Prevention

### Cross-Site Scripting Protection

```java
/**
 * XSS (Cross-Site Scripting): Injecting malicious JavaScript
 * 
 * Attack example:
 * User posts comment: <script>alert(document.cookie)</script>
 * When other users view the page, the script executes
 * Result: Session hijacking, phishing, data theft
 * 
 * Prevention: Encode output, validate input, CSP headers
 */

import org.owasp.encoder.Encode;
import org.springframework.web.util.HtmlUtils;

// ========================================
// ❌ VULNERABLE CODE
// ========================================

@Controller
public class VulnerableXssController {
    
    /**
     * ❌ VULNERABLE: Directly outputting user input
     */
    @GetMapping("/search")
    public String searchVulnerable(@RequestParam String query, Model model) {
        model.addAttribute("query", query);  // ❌ Not escaped!
        return "search";
    }
    
    // search.html (Thymeleaf)
    // ❌ VULNERABLE:
    // <div>You searched for: <span th:utext="${query}"></span></div>
    //                                     ↑ utext = unescaped!
    
    // Attacker input:
    // query: <script>alert('XSS')</script>
    // 
    // Rendered HTML:
    // <div>You searched for: <script>alert('XSS')</script></div>
    //                         ↑ Script executes!
}

// ========================================
// ✅ SECURE CODE (Output Encoding)
// ========================================

@Controller
public class SecureXssController {
    
    /**
     * ✅ SECURE: Escape output
     */
    @GetMapping("/search")
    public String searchSecure(@RequestParam String query, Model model) {
        // Option 1: Manual HTML encoding
        String safeQuery = HtmlUtils.htmlEscape(query);
        model.addAttribute("query", safeQuery);
        
        // Option 2: Let Thymeleaf auto-escape (default)
        model.addAttribute("rawQuery", query);
        
        return "search";
    }
    
    // search.html (Thymeleaf)
    // ✅ SECURE: Auto-escaped by default
    // <div>You searched for: <span th:text="${rawQuery}"></span></div>
    //                                   ↑ th:text auto-escapes!
    
    // Attacker input:
    // query: <script>alert('XSS')</script>
    // 
    // Rendered HTML:
    // <div>You searched for: &lt;script&gt;alert('XSS')&lt;/script&gt;</div>
    //                         ↑ Displayed as text, not executed!
}

// ========================================
// CONTEXT-AWARE ENCODING
// ========================================

public class ContextAwareEncoding {
    
    /**
     * Different contexts require different encoding!
     */
    
    // HTML Context
    public String encodeForHtml(String input) {
        return Encode.forHtml(input);
        // < becomes &lt;
        // > becomes &gt;
        // & becomes &amp;
        // " becomes &quot;
    }
    
    // HTML Attribute Context
    public String encodeForHtmlAttribute(String input) {
        return Encode.forHtmlAttribute(input);
        // More strict than HTML content
        // Encodes quotes, spaces, etc.
    }
    
    // JavaScript Context
    public String encodeForJavaScript(String input) {
        return Encode.forJavaScript(input);
        // \' becomes \\\'
        // \" becomes \\\"
        // < becomes \x3c
    }
    
    // URL Context
    public String encodeForUrl(String input) {
        return Encode.forUriComponent(input);
        // Space becomes %20
        // & becomes %26
    }
    
    // CSS Context
    public String encodeForCss(String input) {
        return Encode.forCssString(input);
        // Prevents CSS injection
    }
    
    // Example usage:
    public void example() {
        String userInput = "<script>alert('XSS')</script>";
        
        // HTML: <div>Input: encoded_value</div>
        String html = "<div>Input: " + encodeForHtml(userInput) + "</div>";
        
        // Attribute: <input value="encoded_value">
        String attr = "<input value=\"" + encodeForHtmlAttribute(userInput) + "\">";
        
        // JavaScript: var data = 'encoded_value';
        String js = "var data = '" + encodeForJavaScript(userInput) + "';";
        
        // URL: /search?q=encoded_value
        String url = "/search?q=" + encodeForUrl(userInput);
    }
}

// ========================================
// CONTENT SECURITY POLICY (CSP)
// ========================================

@Configuration
public class SecurityHeadersConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .headers()
                .contentSecurityPolicy("default-src 'self'; " +
                                     "script-src 'self' https://trusted-cdn.com; " +
                                     "style-src 'self' 'unsafe-inline'; " +
                                     "img-src 'self' data: https:; " +
                                     "font-src 'self'; " +
                                     "connect-src 'self'; " +
                                     "frame-ancestors 'none';")
                .and()
            .xssProtection()
                .block(true)
                .and()
            .httpStrictTransportSecurity()
                .includeSubDomains(true)
                .maxAgeInSeconds(31536000);
        
        return http.build();
    }
    
    // CSP prevents:
    // - Inline scripts (blocks <script>alert('XSS')</script>)
    // - External scripts from untrusted domains
    // - eval() and new Function()
    // - Inline event handlers (onclick="...")
}

// INPUT SANITIZATION (HTML)
// ========================================

import org.jsoup.Jsoup;
import org.jsoup.safety.Safelist;

public class HtmlSanitization {
    
    /**
     * ✅ Sanitize rich text input (e.g., blog posts, comments with formatting)
     */
    public String sanitizeHtml(String input) {
        // Allow only safe tags
        Safelist safelist = Safelist.basicWithImages();
        
        // Allowed: <p>, <b>, <i>, <u>, <a>, <img>, <ul>, <ol>, <li>
        // Removed: <script>, <iframe>, <object>, onclick, etc.
        
        String clean = Jsoup.clean(input, safelist);
        return clean;
    }
    
    /**
     * Example
     */
    public void example() {
        String malicious = "<p>Hello <script>alert('XSS')</script></p><img src=x onerror=alert(1)>";
        
        String safe = sanitizeHtml(malicious);
        // Result: "<p>Hello </p><img src=\"x\" />"
        //          ↑ script removed    ↑ onerror removed
    }
    
    /**
     * Custom whitelist
     */
    public String sanitizeComments(String input) {
        Safelist safelist = new Safelist();
        safelist.addTags("p", "br", "b", "i", "u", "a");
        safelist.addAttributes("a", "href");
        safelist.addProtocols("a", "href", "http", "https");  // Only http/https links
        
        return Jsoup.clean(input, safelist);
    }
}
```

## 4. CSRF Protection

### Cross-Site Request Forgery Prevention

```java
/**
 * CSRF: Attacker tricks authenticated user into making unwanted requests
 * 
 * Attack scenario:
 * 1. User logs into bank.com (gets session cookie)
 * 2. User visits evil.com (while still logged in)
 * 3. evil.com has: <img src="https://bank.com/transfer?to=attacker&amount=1000">
 * 4. Browser automatically sends bank.com cookies
 * 5. Money transferred without user's knowledge!
 * 
 * Prevention: CSRF tokens, SameSite cookies
 */

import org.springframework.security.web.csrf.CsrfToken;

// ========================================
// SPRING SECURITY CSRF (Enabled by Default)
// ========================================

@Configuration
@EnableWebSecurity
public class CsrfSecurityConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf()
```

```java
    @Value("${jwt.expiration:86400000}")  // 24 hours
    private long jwtExpiration;
    
    public String generateToken(Authentication authentication) {
        UserDetails userDetails = (UserDetails) authentication.getPrincipal();
        
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + jwtExpiration);
        
        return Jwts.builder()
            .setSubject(userDetails.getUsername())
            .setIssuedAt(now)
            .setExpiration(expiryDate)
            .signWith(SignatureAlgorithm.HS512, jwtSecret)
            .compact();
    }
    
    public String getUsernameFromToken(String token) {
        Claims claims = Jwts.parser()
            .setSigningKey(jwtSecret)
            .parseClaimsJws(token)
            .getBody();
        
        return claims.getSubject();
    }
    
    public boolean validateToken(String token) {
        try {
            Jwts.parser().setSigningKey(jwtSecret).parseClaimsJws(token);
            return true;
        } catch (JwtException | IllegalArgumentException e) {
            return false;
        }
    }
}

@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    
    @Autowired
    private JwtTokenProvider tokenProvider;
    
    @Autowired
    private CustomUserDetailsService userDetailsService;
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) throws ServletException, IOException {
        
        try {
            String jwt = getJwtFromRequest(request);
            
            if (jwt != null && tokenProvider.validateToken(jwt)) {
                String username = tokenProvider.getUsernameFromToken(jwt);
                UserDetails userDetails = userDetailsService.loadUserByUsername(username);
                
                UsernamePasswordAuthenticationToken authentication = 
                    new UsernamePasswordAuthenticationToken(
                        userDetails, null, userDetails.getAuthorities()
                    );
                
                SecurityContextHolder.getContext().setAuthentication(authentication);
            }
        } catch (Exception e) {
            logger.error("Could not set user authentication", e);
        }
        
        filterChain.doFilter(request, response);
    }
    
    private String getJwtFromRequest(HttpServletRequest request) {
        String bearerToken = request.getHeader("Authorization");
        if (bearerToken != null && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);
        }
        return null;
    }
}

    }
```

```java
     * ✅ Verify password (login)

    public boolean verifyPassword(String plainPassword, String hashedPassword) {
        return passwordEncoder.matches(plainPassword, hashedPassword);

     * Example

    public void example() {
        String password = "MySecureP@ssw0rd";

    }

        boolean hasSpecial = password.matches(".*[@$!%*?&].*");
```

```java
        return secretValue;
```
