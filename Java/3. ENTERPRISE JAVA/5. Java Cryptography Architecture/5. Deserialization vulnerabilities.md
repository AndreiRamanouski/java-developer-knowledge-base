# Deserialization Vulnerabilities

## Overview

Deserialization vulnerabilities are among the most dangerous security flaws in Java applications—enabling attackers to execute arbitrary code remotely. This guide covers five critical deserialization security concepts: Gadget Chains (chaining existing classes for exploitation—attacker crafts malicious serialized object, deserialization triggers chain of method calls, uses legitimate library classes as "gadgets", results in arbitrary code execution, famous examples: Commons Collections, Spring Framework), ObjectInputStream Risks (inherent dangers—automatically instantiates objects, executes code during deserialization, readObject() method runs attacker-controlled code, no validation by default, accepts any Serializable class), SerialKiller and Filters (defensive mechanisms—ObjectInputFilter validates classes before deserialization, SerialKiller blocks known dangerous classes, whitelist safe classes, blacklist dangerous ones, Java 9+ has built-in filtering), Alternatives to Java Serialization (safer approaches—JSON doesn't execute code during parsing, Protocol Buffers schema-based validation, MessagePack binary JSON alternative, avoid Java serialization for untrusted data, use language-agnostic formats), and Safe Deserialization Practices (defense strategies—never deserialize untrusted data if possible, always use ObjectInputFilter, validate all inputs, use allowlist not blocklist, prefer alternatives like JSON, sign serialized data, implement defense in depth). The key insight: Java deserialization = remote code execution risk—serialized data can contain instructions to execute arbitrary code, no built-in protection until Java 9, affects most major applications (Jenkins, WebLogic, JBoss), listed as #8 in OWASP Top 10 (Insecure Deserialization). Common mistakes: deserializing untrusted data (HTTP requests, files, network), no input filtering (accepts any class), using vulnerable libraries (Commons Collections), trusting serialized data (it's data, not code—wrong!), not using ObjectInputFilter, putting Serializable on everything. The relationship between concepts: Untrusted serialized data → ObjectInputStream.readObject() → Gadget chain triggered → Arbitrary code execution → System compromise. Defense: ObjectInputFilter blocks → Safe alternatives used → Data validated → No code execution. Real-world defense strategy: avoid Java serialization entirely for external data, use JSON/Protocol Buffers instead, if must deserialize: use ObjectInputFilter with strict allowlist, update libraries (remove gadgets), validate after deserialization, sign data for integrity, monitor for exploitation attempts.

## 3. SerialKiller and Filters

### Defensive Mechanisms

```java
/**
 * OBJECTINPUTFILTER: Java 9+ built-in protection
 * SERIALKILLER: Third-party library for older Java versions
 * 
 * Both provide filtering to block dangerous classes
 */

import java.io.*;

// ========================================
// OBJECTINPUTFILTER (Java 9+)
// ========================================

public class ObjectInputFilterExample {
    
    /**
     * ✅ SECURE: Use ObjectInputFilter to validate classes
     */
    public Object secureDeserialize(byte[] data) throws Exception {
        ObjectInputStream ois = new ObjectInputStream(new ByteArrayInputStream(data));
        
        // Set filter BEFORE reading
        ois.setObjectInputFilter(filterInfo -> {
            Class<?> clazz = filterInfo.serialClass();
            
            // Allowlist approach (preferred)
            if (clazz == null) {
                return ObjectInputFilter.Status.UNDECIDED;
            }
            
            // Allow only specific safe classes
            if (clazz == String.class ||
                clazz == Integer.class ||
                clazz == Long.class ||
                clazz == Boolean.class ||
                clazz == Double.class ||
                clazz.getName().startsWith("com.myapp.model.")) {
                return ObjectInputFilter.Status.ALLOWED;
            }
            
            // Reject everything else
            System.out.println("Rejected class: " + clazz.getName());
            return ObjectInputFilter.Status.REJECTED;
        });
        
        try {
            Object obj = ois.readObject();
            ois.close();
            return obj;
            
        } catch (InvalidClassException e) {
            System.err.println("Deserialization blocked: " + e.getMessage());
            throw new SecurityException("Invalid class in serialized data", e);
        }
    }
    
    /**
     * ✅ SECURE: Comprehensive filter with limits
     */
    public ObjectInputFilter createSecureFilter() {
        return ObjectInputFilter.Config.createFilter(
            // Allowlist format: class1;class2;!blockedClass;maxdepth=5;maxbytes=1000000
            "java.lang.String;" +
            "java.lang.Integer;" +
            "java.util.ArrayList;" +
            "com.myapp.model.*;" +  // Allow package
            "!org.apache.commons.collections.*;" +  // Block dangerous package
            "maxdepth=5;" +          // Maximum object graph depth
            "maxarray=100;" +        // Maximum array size
            "maxbytes=1000000"       // Maximum stream size (1MB)
        );
    }
    
    /**
     * ✅ SECURE: Apply filter globally (JVM-wide)
     */
    public void configureGlobalFilter() {
        ObjectInputFilter globalFilter = ObjectInputFilter.Config.createFilter(
            "java.lang.*;" +
            "java.util.*;" +
            "com.myapp.model.*;" +
            "!*;"+  // Reject everything not explicitly allowed
            "maxdepth=5;" +
            "maxbytes=5000000"
        );
        
        ObjectInputFilter.Config.setSerialFilter(globalFilter);
        
        System.out.println("Global deserialization filter enabled");
    }
    
    /**
     * JVM argument to set global filter:
     * -Djdk.serialFilter=java.lang.*;java.util.*;com.myapp.model.*;!*;maxdepth=5
     */
}

// CUSTOM OBJECTINPUTSTREAM
// ========================================

public class SecureObjectInputStream extends ObjectInputStream {
    
    private final Set<String> allowedClasses;
    
    public SecureObjectInputStream(InputStream in, Set<String> allowedClasses) 
            throws IOException {
        super(in);
        this.allowedClasses = allowedClasses;
    }
    
    @Override
    protected Class<?> resolveClass(ObjectStreamClass desc) 
            throws IOException, ClassNotFoundException {
        
        String className = desc.getName();
        
        // Check if class is allowed
        boolean allowed = false;
        for (String allowedPrefix : allowedClasses) {
            if (className.startsWith(allowedPrefix)) {
                allowed = true;
                break;
            }
        }
        
        if (!allowed) {
            throw new InvalidClassException(
                "Unauthorized deserialization attempt: " + className
            );
        }
        
        return super.resolveClass(desc);
    }
}

// Usage:

    Set<String> allowedClasses = Set.of(
        "java.lang.",
        "java.util.",
        "com.myapp.model."
    );
    
    SecureObjectInputStream ois = new SecureObjectInputStream(
        new ByteArrayInputStream(data),
        allowedClasses
    );
    
    Object obj = ois.readObject();
    ois.close();
    
    return obj;
}
```

## 4. Alternatives to Java Serialization

### Safer Data Interchange Formats

```java
/**
 * ALTERNATIVES: Use formats that don't execute code
 * 
 * Safe alternatives:
 * - JSON (Jackson, Gson)
 * - Protocol Buffers
 * - XML (with secure parsers)
 * - MessagePack
 * - Avro
 * 
 * Why safer?
 * - No automatic code execution
 * - Schema validation
 * - Language-agnostic
 * - Better performance
 */

// ========================================
// JSON (Jackson)
// ========================================

import com.fasterxml.jackson.databind.ObjectMapper;

public class JsonSerialization {
    
    private final ObjectMapper objectMapper = new ObjectMapper();
    
    /**
     * ✅ SAFE: JSON serialization
     */
    public String serializeToJson(Object obj) throws Exception {
        return objectMapper.writeValueAsString(obj);
    }
    
    /**
     * ✅ SAFE: JSON deserialization
     */
    public <T> T deserializeFromJson(String json, Class<T> clazz) throws Exception {
        return objectMapper.readValue(json, clazz);
    }
    
    /**
     * Why JSON is safer:
     * - No code execution during parsing
     * - Only creates objects of specified class
     * - No gadget chains possible
     * - Human-readable
     * - Language-agnostic
     */
    
    /**
     * Example usage:
     */
    public void example() throws Exception {
        User user = new User("john", "john@example.com");
        
        // Serialize to JSON
        String json = serializeToJson(user);
        System.out.println(json);
        // {"username":"john","email":"john@example.com"}
        
        // Deserialize from JSON
        User deserialized = deserializeFromJson(json, User.class);
        System.out.println(deserialized.getUsername());
        // john
    }
    
    /**
     * ⚠️ Jackson security notes:
     * - Disable default typing (enableDefaultTyping)
     * - Use @JsonTypeInfo carefully
     * - Update to latest version
     * - Validate input size
     */
    public ObjectMapper createSecureObjectMapper() {
        ObjectMapper mapper = new ObjectMapper();
        
        // Disable polymorphic type handling (can be exploited)
        // mapper.enableDefaultTyping(); // DON'T DO THIS!
        
        // Configure for security
        mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);
        
        return mapper;
    }
}

import javax.xml.bind.*;

public class XmlSerialization {
    
    /**
     * ✅ SAFE: JAXB serialization
     */
    public String serializeToXml(Object obj) throws Exception {
        JAXBContext context = JAXBContext.newInstance(obj.getClass());
        Marshaller marshaller = context.createMarshaller();
        marshaller.setProperty(Marshaller.JAXB_FORMATTED_OUTPUT, true);
        
        StringWriter writer = new StringWriter();
        marshaller.marshal(obj, writer);
        
        return writer.toString();
    }
    
    /**
     * ✅ SAFE: JAXB deserialization
     */
    public <T> T deserializeFromXml(String xml, Class<T> clazz) throws Exception {
        JAXBContext context = JAXBContext.newInstance(clazz);
        Unmarshaller unmarshaller = context.createUnmarshaller();
        
        StringReader reader = new StringReader(xml);
        return (T) unmarshaller.unmarshal(reader);
    }
    
    /**
     * ⚠️ XML security notes:
     * - Disable external entities (XXE prevention)
     * - Disable DTD processing
     * - Use schema validation
     */
    public Unmarshaller createSecureUnmarshaller(Class<?> clazz) throws Exception {
        JAXBContext context = JAXBContext.newInstance(clazz);
        Unmarshaller unmarshaller = context.createUnmarshaller();
        
        // Disable external entities
        unmarshaller.setProperty("com.sun.xml.bind.disableXmlSecurity", false);
        
        return unmarshaller;
    }
}
```

```java
     * ✅ Serialize and sign

    public byte[] serializeAndSign(Object obj) throws Exception {
        // Serialize object
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(baos);
        oos.writeObject(obj);
        oos.close();
        
        byte[] serialized = baos.toByteArray();
        
        // Compute HMAC signature
        Mac mac = Mac.getInstance("HmacSHA256");
        mac.init(hmacKey);
        byte[] signature = mac.doFinal(serialized);
        
        // Combine signature + serialized data
        ByteArrayOutputStream combined = new ByteArrayOutputStream();
        combined.write(signature);
        combined.write(serialized);
        
        return combined.toByteArray();
    }

    public Object verifyAndDeserialize(byte[] signedData) throws Exception {
```

```java
1. GADGET CHAINS
   • Chain legitimate classes for exploitation
   • Commons Collections, Spring, Groovy gadgets
   • Attacker crafts malicious serialized object
   • Deserialization triggers chain
   • Result: Arbitrary code execution
   
   Example: LazyMap → ChainedTransformer → Runtime.exec()

2. OBJECTINPUTSTREAM RISKS
   • Automatically instantiates objects
   • Executes readObject() methods
   • No validation by default
   • Accepts any Serializable class
   
   Attack vectors: HTTP requests, files, network, cookies, JMX, RMI

3. SERIALKILLER AND FILTERS
   • ObjectInputFilter (Java 9+)
   • SerialKiller (Java 8)
   • Allowlist safe classes
   • Blocklist dangerous classes
   • Set limits (depth, size, arrays)
   
   Code:
   ois.setObjectInputFilter(filterInfo -> {
       if (allowed.contains(filterInfo.serialClass())) {
           return Status.ALLOWED;
       }
       return Status.REJECTED;
   });

4. ALTERNATIVES TO JAVA SERIALIZATION
   • JSON (Jackson, Gson) - Most common
   • Protocol Buffers - Best for microservices
   • XML (JAXB) - If needed
   • MessagePack - Binary JSON
   
   Why safer: No code execution!

   • Avoid Java serialization (use JSON)
   • If must deserialize: Use ObjectInputFilter
   • Validate after deserialization
   • Sign serialized data (HMAC)
   • Monitor and alert
   • Update dependencies
```
