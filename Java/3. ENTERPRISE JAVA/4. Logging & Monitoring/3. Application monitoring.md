# Application Monitoring

## Overview

Application monitoring transforms invisible application behavior into observable metrics—essential for understanding system health, detecting issues before users do, and making data-driven decisions. This guide covers seven critical monitoring concepts: JMX and MBeans (Java Management Extensions—native Java monitoring, exposes application metrics via MBeans, accessible through JConsole/VisualVM, platform for building monitoring), Metrics with Micrometer (modern facade for metrics—vendor-neutral API, supports Prometheus/Graphite/DataDog/etc, dimensional metrics with tags, counters/gauges/timers/distribution summaries), Health Checks (application readiness indicators—liveness checks if app is alive, readiness checks if app can serve traffic, critical for Kubernetes/load balancers, Spring Boot Actuator provides framework), Custom Metrics (business-specific measurements—track domain events like orders/payments/signups, measure business SLAs, instrument critical code paths, balance detail with cardinality), Prometheus Integration (time-series metrics database—pull-based scraping, powerful query language PromQL, automatic service discovery, stores metrics with labels), Grafana Dashboards (visualization layer—query multiple data sources, create real-time dashboards, templating for dynamic dashboards, built-in alerting), and Alerting Strategies (intelligent notification system—alert on symptoms not causes, use runbooks for responses, rate limit notifications, escalation policies, on-call rotation). The key insight: monitoring is about _actionable insights_, not just data—every metric should answer "what action should I take?" Common mistakes: too many metrics (high cardinality explosion), alerting on everything (alert fatigue), no context (metric without tags), vanity metrics (track but don't act on), monitoring without alerting (data hoarding). The relationship between concepts: JMX provides raw data → Micrometer abstracts collection → Health checks indicate status → Custom metrics track business → Prometheus stores time-series → Grafana visualizes trends → Alerting triggers actions. Real-world monitoring strategy: expose JMX for JVM metrics, use Micrometer for application metrics, implement health checks for orchestrators, create custom metrics for business KPIs, scrape with Prometheus, visualize in Grafana, alert on SLA violations.

### Java Management Extensions

```java
/**
 * JMX: Java Management Extensions
 * 
 * Standard way to expose management information
 * Accessible via JConsole, VisualVM, or programmatically
 */

import javax.management.*;
import java.lang.management.*;

// ========================================
// STANDARD MBEANS (Simple Interface)
// ========================================

/**
 * MBean interface (must end with "MBean")
 */
public interface CacheStatisticsMBean {
    // Attributes (getters/setters)
    long getHitCount();
    long getMissCount();
    double getHitRate();
    long getSize();
    
    // Operations (methods)
    void clear();
    void evictExpired();
}

/**
 * MBean implementation
 */
public class CacheStatistics implements CacheStatisticsMBean {
    
    private final Cache cache;
    
    public CacheStatistics(Cache cache) {
        this.cache = cache;
    }
    
    @Override
    public long getHitCount() {
        return cache.getStats().getHits();
    }
    
    @Override
    public long getMissCount() {
        return cache.getStats().getMisses();
    }
    
    @Override
    public double getHitRate() {
        long hits = getHitCount();
        long total = hits + getMissCount();
        return total == 0 ? 0.0 : (double) hits / total;
    }
    
    @Override
    public long getSize() {
        return cache.size();
    }
    
    @Override
    public void clear() {
        cache.clear();
    }
    
    @Override
    public void evictExpired() {
        cache.evictExpired();
    }
}

// ========================================
// REGISTERING MBEANS
// ========================================

public class MonitoringSetup {
    
    public void registerMBeans() throws Exception {
        MBeanServer mbs = ManagementFactory.getPlatformMBeanServer();
        
        // Register cache statistics
        CacheStatistics cacheStats = new CacheStatistics(cache);
        ObjectName name = new ObjectName("com.example:type=Cache,name=UserCache");
        mbs.registerMBean(cacheStats, name);
        
        // View in JConsole:
        // MBeans → com.example → Cache → UserCache
        // See attributes: HitCount, MissCount, HitRate, Size
        // Invoke operations: clear(), evictExpired()
    }
}

public class ApplicationMetrics implements DynamicMBean {
    
    private final Map<String, Object> attributes = new ConcurrentHashMap<>();
    
    public ApplicationMetrics() {
        attributes.put("RequestCount", 0L);
        attributes.put("ErrorCount", 0L);
        attributes.put("AverageResponseTime", 0.0);
    }
    
    public void incrementRequests() {
        attributes.compute("RequestCount", (k, v) -> ((Long) v) + 1);
    }
    
    public void incrementErrors() {
        attributes.compute("ErrorCount", (k, v) -> ((Long) v) + 1);
    }
    
    @Override
    public Object getAttribute(String attribute) {
        return attributes.get(attribute);
    }
    
    @Override
    public void setAttribute(Attribute attribute) {
        attributes.put(attribute.getName(), attribute.getValue());
    }
    
    @Override
    public AttributeList getAttributes(String[] attributes) {
        AttributeList list = new AttributeList();
        for (String attr : attributes) {
            list.add(new Attribute(attr, this.attributes.get(attr)));
        }
        return list;
    }
    
    @Override
    public AttributeList setAttributes(AttributeList attributes) {
        return attributes;
    }
    
    @Override
    public Object invoke(String actionName, Object[] params, String[] signature) {
        switch (actionName) {
            case "reset":
                attributes.replaceAll((k, v) -> 0L);
                return "Reset successful";
            default:
                throw new UnsupportedOperationException();
        }
    }
    
    @Override
    public MBeanInfo getMBeanInfo() {
        // Define attributes
        MBeanAttributeInfo[] attrs = new MBeanAttributeInfo[] {
            new MBeanAttributeInfo("RequestCount", "long", "Total requests", true, false, false),
            new MBeanAttributeInfo("ErrorCount", "long", "Total errors", true, false, false),
            new MBeanAttributeInfo("AverageResponseTime", "double", "Avg response time", true, false, false)
        };
        
        // Define operations
        MBeanOperationInfo[] ops = new MBeanOperationInfo[] {
            new MBeanOperationInfo("reset", "Reset all counters", null, "void", MBeanOperationInfo.ACTION)
        };
        
        return new MBeanInfo(
            this.getClass().getName(),
            "Application Metrics",
            attrs,
            null,  // constructors
            ops,
            null   // notifications
        );
    }
}
```

## 2. Metrics (Micrometer)

### Modern Metrics Facade

```java
/**
 * MICROMETER: Vendor-neutral metrics facade
 * 
 * Like SLF4J for metrics - one API, multiple backends
 * Supports: Prometheus, Graphite, DataDog, New Relic, etc.
 */

import io.micrometer.core.instrument.*;
import io.micrometer.prometheus.PrometheusConfig;
import io.micrometer.prometheus.PrometheusMeterRegistry;

// ========================================
// BASIC METRICS TYPES
// ========================================

public class MetricsExamples {
    
    private final MeterRegistry registry;
    
    public MetricsExamples(MeterRegistry registry) {
        this.registry = registry;
    }
    
    // ========================================
    // COUNTER: Monotonically increasing value
    // ========================================
    
    private final Counter requestCounter = Counter.builder("http.requests")
            .description("Total HTTP requests")
            .tag("method", "GET")
            .tag("endpoint", "/api/users")
            .register(registry);
    
    public void recordRequest() {
        requestCounter.increment();
        // Or increment by amount
        requestCounter.increment(5);
    }
    
    // ========================================
    // GAUGE: Current value (can go up or down)
    // ========================================
    
    private final AtomicInteger queueSize = new AtomicInteger(0);
    
    public void setupGauge() {
        Gauge.builder("queue.size", queueSize, AtomicInteger::get)
             .description("Current queue size")
             .tag("queue", "orders")
             .register(registry);
    }
    
    public void addToQueue() {
        queueSize.incrementAndGet();
    }
    
    // ========================================
    // TIMER: Measures duration and count
    // ========================================
    
    private final Timer requestTimer = Timer.builder("http.request.duration")
            .description("HTTP request duration")
            .tag("method", "POST")
            .tag("endpoint", "/api/orders")
            .register(registry);
    
    public void processRequest() {
        // Method 1: Record manually
        long start = System.nanoTime();
        try {
            doWork();
        } finally {
            requestTimer.record(System.nanoTime() - start, TimeUnit.NANOSECONDS);
        }
        
        // Method 2: Using sample
        Timer.Sample sample = Timer.start(registry);
        doWork();
        sample.stop(requestTimer);
        
        // Method 3: Using recordCallable
        String result = requestTimer.recordCallable(() -> doWorkAndReturn());
        
        // Method 4: Wrap with annotation (Spring)
        // @Timed("http.request.duration")
    }
    
    // ========================================
    // DISTRIBUTION SUMMARY: Track distribution
    // ========================================
    
    private final DistributionSummary responseSizeSummary = 
        DistributionSummary.builder("http.response.size")
            .description("HTTP response size in bytes")
            .baseUnit("bytes")
            .minimumExpectedValue(1.0)
            .maximumExpectedValue(1000000.0)
            .register(registry);
    
    public void recordResponseSize(long size) {
        responseSizeSummary.record(size);
    }
    
    // ========================================
    // LONG TASK TIMER: Track in-progress tasks
    // ========================================
    
    private final LongTaskTimer batchProcessingTimer = 
        LongTaskTimer.builder("batch.processing")
            .description("Long-running batch processing")
            .register(registry);
    
    public void processBatch() {
        LongTaskTimer.Sample sample = batchProcessingTimer.start();
        try {
            // Long-running work
            processLargeBatch();
        } finally {
            sample.stop();
        }
    }
    
    private void doWork() {}
    private String doWorkAndReturn() { return "result"; }
    private void processLargeBatch() {}
}

// TAGS AND DIMENSIONS

public class TaggedMetrics {
    
    private final MeterRegistry registry;
    
    public TaggedMetrics(MeterRegistry registry) {
        this.registry = registry;
    }
    
    public void recordApiRequest(String method, String endpoint, int statusCode) {
        // Tags create different time series
        Counter.builder("api.requests")
               .tag("method", method)           // GET, POST, PUT, DELETE
               .tag("endpoint", endpoint)       // /api/users, /api/orders
               .tag("status", String.valueOf(statusCode))  // 200, 400, 500
               .register(registry)
               .increment();
        
        // This creates separate counters for:
        // api.requests{method="GET", endpoint="/api/users", status="200"}
        // api.requests{method="POST", endpoint="/api/orders", status="201"}
        // api.requests{method="GET", endpoint="/api/users", status="404"}
        // etc.
    }
    
    // CARDINALITY WARNING!
    public void badTagging() {
        // ✗ BAD: High cardinality (unique per request)
        Counter.builder("requests")
               .tag("userId", userId)          // Millions of users!
               .tag("requestId", requestId)    // Unique per request!
               .register(registry)
               .increment();
        
        // This creates MILLIONS of time series
        // Overwhelms monitoring system
        
        // ✓ GOOD: Low cardinality
        Counter.builder("requests")
               .tag("userType", getUserType(userId))  // "free", "premium", "enterprise"
               .tag("region", getRegion(userId))      // "us-east", "eu-west", "ap-south"
               .register(registry)
               .increment();
    }
    
    private String getUserType(String userId) { return "premium"; }
    private String getRegion(String userId) { return "us-east"; }
}

@Service
public class OrderService {
    
    private final Counter ordersCreated;
    private final Timer orderProcessingTime;
    
    public OrderService(MeterRegistry registry) {
        this.ordersCreated = Counter.builder("orders.created")
                                    .description("Total orders created")
                                    .register(registry);
        
        this.orderProcessingTime = Timer.builder("orders.processing.time")
                                        .description("Order processing duration")
                                        .register(registry);
    }
    
    @Timed("orders.create")  // Automatic timing
    public Order createOrder(OrderRequest request) {
        return orderProcessingTime.record(() -> {
            Order order = processOrder(request);
            ordersCreated.increment();
            return order;
        });
    }
    
    private Order processOrder(OrderRequest request) {
        return new Order();
    }
}
```

## 3. Health Checks

### Application Health Indicators

```java
/**
 * HEALTH CHECKS: Indicate application status
 * 
 * Liveness: Is the app alive? (kill if not)
 * Readiness: Can the app serve traffic? (remove from load balancer if not)
 */

import org.springframework.boot.actuate.health.*;
import org.springframework.stereotype.Component;

// ========================================
// CUSTOM HEALTH INDICATOR
// ========================================

@Component
public class DatabaseHealthIndicator implements HealthIndicator {
    
    private final DataSource dataSource;
    
    public DatabaseHealthIndicator(DataSource dataSource) {
        this.dataSource = dataSource;
    }
    
    @Override
    public Health health() {
        try (Connection conn = dataSource.getConnection()) {
            // Check if database is accessible
            if (conn.isValid(2)) {  // 2 second timeout
                return Health.up()
                            .withDetail("database", "PostgreSQL")
                            .withDetail("version", conn.getMetaData().getDatabaseProductVersion())
                            .build();
            }
        } catch (Exception e) {
            return Health.down()
                        .withDetail("error", e.getMessage())
                        .withException(e)
                        .build();
        }
        
        return Health.down().build();
    }
}

// EXTERNAL SERVICE HEALTH CHECK
// ========================================

@Component
public class PaymentServiceHealthIndicator implements HealthIndicator {
    
    private final RestTemplate restTemplate;
    private final String paymentServiceUrl;
    
    @Override
    public Health health() {
        try {
            ResponseEntity<String> response = restTemplate.getForEntity(
                paymentServiceUrl + "/health", 
                String.class
            );
            
            if (response.getStatusCode().is2xxSuccessful()) {
                return Health.up()
                            .withDetail("service", "payment-service")
                            .withDetail("responseTime", getResponseTime() + "ms")
                            .build();
            }
            
            return Health.down()
                        .withDetail("status", response.getStatusCode())
                        .build();
                        
        } catch (Exception e) {
            return Health.down()
                        .withDetail("error", e.getMessage())
                        .withException(e)
                        .build();
        }
    }
    
    private long getResponseTime() {
        return 150;  // Simplified
    }
}

// DISK SPACE HEALTH CHECK
// ========================================

@Component
public class DiskSpaceHealthIndicator implements HealthIndicator {
    
    private final long thresholdBytes = 10 * 1024 * 1024 * 1024L;  // 10 GB
    
    @Override
    public Health health() {
        File root = new File("/");
        long freeSpace = root.getFreeSpace();
        long totalSpace = root.getTotalSpace();
        long usedSpace = totalSpace - freeSpace;
        double usedPercent = (usedSpace * 100.0) / totalSpace;
        
        Health.Builder builder = freeSpace >= thresholdBytes 
            ? Health.up() 
            : Health.down();
        
        return builder
                .withDetail("free", formatBytes(freeSpace))
                .withDetail("total", formatBytes(totalSpace))
                .withDetail("used", formatBytes(usedSpace))
                .withDetail("usedPercent", String.format("%.2f%%", usedPercent))
                .withDetail("threshold", formatBytes(thresholdBytes))
                .build();
    }
    
    private String formatBytes(long bytes) {
        return String.format("%.2f GB", bytes / (1024.0 * 1024 * 1024));
    }
}

@Component
public class ApplicationHealthIndicator extends CompositeHealthContributor {

            DatabaseHealthIndicator database,

            DiskSpaceHealthIndicator diskSpace) {

        indicators.put("paymentService", paymentService);

        return indicators.entrySet().stream()
                .map(e -> NamedContributor.of(e.getKey(), e.getValue()))
                .iterator();
    }
}
```

## 4. Custom Metrics

### Business-Specific Measurements

```java
/**
 * CUSTOM METRICS: Track domain-specific events
 * 
 * Beyond technical metrics (CPU, memory), track:
 * - Business events (orders, signups, payments)
 * - SLA metrics (response time, error rate)
 * - Domain-specific KPIs
 */

import io.micrometer.core.instrument.*;

@Service
public class OrderMetrics {
    
    private final Counter ordersCreated;
    private final Counter ordersCompleted;
    private final Counter ordersFailed;
    private final Timer orderProcessingTime;
    private final DistributionSummary orderValue;
    private final Gauge pendingOrders;
    
    public OrderMetrics(MeterRegistry registry) {
        // Count orders by status
        this.ordersCreated = Counter.builder("orders.created")
                .description("Total orders created")
                .tag("type", "all")
                .register(registry);
        
        this.ordersCompleted = Counter.builder("orders.completed")
                .description("Orders completed successfully")
                .register(registry);
        
        this.ordersFailed = Counter.builder("orders.failed")
                .description("Orders that failed")
                .register(registry);
        
        // Track processing time
        this.orderProcessingTime = Timer.builder("orders.processing.duration")
                .description("Time to process an order")
                .publishPercentiles(0.5, 0.95, 0.99)  // p50, p95, p99
                .publishPercentileHistogram()
                .register(registry);
        
        // Track order values
        this.orderValue = DistributionSummary.builder("orders.value")
                .description("Order value in dollars")
                .baseUnit("USD")
                .publishPercentiles(0.5, 0.95, 0.99)
                .register(registry);
        
        // Track pending orders (gauge)
        AtomicInteger pending = new AtomicInteger(0);
        this.pendingOrders = Gauge.builder("orders.pending", pending, AtomicInteger::get)
                .description("Number of orders pending")
                .register(registry);
    }
    
    public void recordOrderCreated(Order order) {
        ordersCreated.increment();
        orderValue.record(order.getTotal());
    }
    
    public void recordOrderCompleted(Order order, long processingTimeMs) {
        ordersCompleted.increment();
        orderProcessingTime.record(processingTimeMs, TimeUnit.MILLISECONDS);
    }
    
    public void recordOrderFailed(Order order, String reason) {
        Counter.builder("orders.failed")
               .tag("reason", reason)  // "payment_declined", "out_of_stock", etc.
               .register(registry)
               .increment();
    }
}

@Service
public class SlaMetrics {
    
    private final Timer responseTime;
    private final Counter slaViolations;
    
    public SlaMetrics(MeterRegistry registry) {
        this.responseTime = Timer.builder("sla.response.time")
                .description("API response time SLA tracking")
                .publishPercentiles(0.95, 0.99)
                .sla(Duration.ofMillis(100), Duration.ofMillis(500), Duration.ofSeconds(1))
                .register(registry);
        
        this.slaViolations = Counter.builder("sla.violations")
                .description("SLA violations")
                .register(registry);
    }
    
    public void recordResponse(long durationMs) {
        responseTime.record(durationMs, TimeUnit.MILLISECONDS);
        
        // SLA: 95% of requests < 500ms
        if (durationMs > 500) {
            slaViolations.increment();
        }
    }
}

// ========================================
// NAMING CONVENTIONS
// ========================================

/**
 * Metric naming best practices:
 * 
 * Format: <namespace>.<name>.<unit>
 * 
 * Good names:
 * - orders.created.total
 * - http.request.duration.seconds
 * - cache.hit.ratio
 * - database.query.duration.milliseconds
 * - queue.size.current
 * 
 * Bad names:
 * - OrdersCreated           (not lowercase)
 * - http-request-time       (use dots not dashes)
 * - requestDuration         (no namespace)
 * - time                    (too generic)
 * 
 * Units:
 * - Time: seconds, milliseconds, nanoseconds
 * - Size: bytes, kilobytes, megabytes
 * - Rate: per_second, per_minute
 * - Ratio: ratio (0-1), percent (0-100)
 */
```

```java
/**
 * ALERT ON SYMPTOMS, NOT CAUSES:
 * 
 * ✓ GOOD: "High error rate" (symptom)
 * ✗ BAD: "Database connection failed" (cause)
 * 
 * Users care about symptoms. Causes are for investigation.
 * 
 * 
 * ALERT LEVELS:
 * 
 * CRITICAL: Immediate action required
 * - Service completely down
 * - High error rate (>5%)
 * - Data loss risk
 * → Page on-call engineer
 * 
 * WARNING: Needs attention soon
 * - Slow response time
 * - High memory usage
 * - Approaching limits
 * → Slack notification
 * 
 * INFO: Informational only
 * - Deployment completed
 * - Scheduled maintenance
 * → Email notification
 * 
 * 
 * ALERT FATIGUE PREVENTION:
 * 
 * 1. Set appropriate thresholds
 *    - Too sensitive → Too many alerts
 *    - Not sensitive enough → Miss issues
 * 
 * 2. Use "for" duration
 *    for: 5m  # Alert only if condition persists
 * 
 * 3. Group related alerts
 *    - Don't alert on every instance
 *    - Alert on aggregated metrics
 * 
 * 4. Rate limiting
 *    - Max 1 alert per 15 minutes for same issue
 * 
 * 5. Maintenance windows
 *    - Silence alerts during deployments

 *

 * 
 * Every alert should have a runbook:
 * - What does this alert mean?
 * - What is the impact?

 * Example runbook link in alert:

 * 
 * 
 * ON-CALL ROTATION:
 * 
 * - Primary: Receives all critical alerts
 * - Secondary: Escalation if primary doesn't respond
 * - Rotate weekly to prevent burnout
 * - Handoff meeting each rotation
 * 
 * 
 * ESCALATION POLICY:
 * 
 * Tier 1: On-call engineer (15 min response time)
```

```java
   • Java Management Extensions

   Example:
   MBeanServer mbs = ManagementFactory.getPlatformMBeanServer();

   • Vendor-neutral metrics facade
   • Counter: Monotonic (requests, errors)
   • Gauge: Current value (queue size, temperature)
   • Timer: Duration + count (response time)
   • DistributionSummary: Track distribution (payload size)
   
   Example:
   Counter.builder("orders.created")
          .tag("type", "premium")
          .register(registry)
          .increment();

3. HEALTH CHECKS
   • Liveness: Is app alive? (restart if down)
   • Readiness: Can serve traffic? (remove from LB if down)
   • Custom indicators for dependencies
   • Kubernetes probe integration
   
   Example:
   @Component
   class DatabaseHealth implements HealthIndicator {
     Health health() { /* check database */ }
   }

4. CUSTOM METRICS
   • Business KPIs (orders, signups, revenue)
   • SLA metrics (response time < 500ms)
   • Domain events (payment success rate)
   • Watch cardinality! (avoid unique IDs in tags)
   
   Naming: namespace.name.unit
   Example: orders.created.total

5. PROMETHEUS INTEGRATION
   • Pull-based scraping (/metrics endpoint)
   • Time-series storage
   • PromQL query language
   • Service discovery (Kubernetes)
   
   Query example:
   rate(http_requests_total[5m])

6. GRAFANA DASHBOARDS
   • Visualization layer
   • Multiple data sources
   • Variables for filtering
   • Panel types: Graph, Gauge, Stat, Table
   
   Layout:
   Top: KPIs (request rate, error rate, p95)
   Middle: Service metrics
   Bottom: Infrastructure

7. ALERTING STRATEGIES
   • Alert on symptoms (high error rate)
   • Not causes (database down)
   • Severity: Critical, Warning, Info
   • Include runbooks
   • Prevent alert fatigue
```
