# Effective Logging Strategies

## Overview

Effective logging is the difference between diagnosing production issues in minutes versus days—logs are your primary insight into what happens in production, but only if done correctly. This guide covers seven critical logging strategies: What to Log and What Not to Log (log state changes/errors/business events, don't log every method call/sensitive data/noise—signal-to-noise ratio is key), Log Levels (TRACE for detailed flow, DEBUG for development details, INFO for business events, WARN for recoverable issues, ERROR for failures—each level has specific purpose and audience), Parameterized Logging (use placeholders {} instead of string concatenation—defers formatting until level is enabled, prevents unnecessary object creation, 10-100x performance improvement), Avoiding String Concatenation (logger.info("User: " + name) always creates string even if INFO disabled—waste CPU and memory, use parameterized messages instead), Logging Exceptions Properly (include exception object with context, don't just log message, don't swallow exceptions, add business context to error logs), Performance Considerations (logging can consume 1-5% of application time—check level for expensive operations, use async loggers for high throughput, avoid logging in hot loops, measure impact), and PII and Security (never log passwords/tokens/SSNs/credit cards, sanitize user input, comply with GDPR/CCPA, use data masking for sensitive fields). The key insight: logging is about _information_, not _data_—every log should answer "why would I need to know this?" Common mistakes: logging everything (creates noise), logging nothing (can't debug), wrong log levels (ERROR for warnings), logging sensitive data (security breach), string concatenation (performance), forgetting context (can't correlate), not logging exceptions (lose stack trace). The relationship between concepts: what to log determines verbosity → log levels control output → parameterized logging improves performance → exception logging captures failures → security practices protect data. Real-world logging strategy: log business events at INFO, debug details at DEBUG, use MDC for correlation, sanitize sensitive data, enable async for high load, review logs regularly to reduce noise.

**Key Principle**:

```
Good logging answers these questions:
1. What happened? (the event)
2. When did it happen? (timestamp)
3. Where did it happen? (class/method/line)
4. Who was involved? (user/request ID)
5. Why did it happen? (context/cause)
6. What was the outcome? (success/failure)

If a log doesn't help answer these, don't log it.
```

### The Art of Meaningful Logging

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class LoggingExamples {
    
    private static final Logger logger = LoggerFactory.getLogger(LoggingExamples.class);
    
    // ========================================
    // ✓ WHAT TO LOG: Business Events
    // ========================================
    
    public void goodLogging_BusinessEvents() {
        // User actions
        logger.info("User logged in: username={}, sessionId={}", username, sessionId);
        logger.info("Order created: orderId={}, userId={}, total=${}", orderId, userId, total);
        logger.info("Payment processed: orderId={}, amount=${}, method={}", orderId, amount, method);
        
        // System events
        logger.info("Application started successfully in {}ms", startupTime);
        logger.info("Database connection pool initialized: size={}", poolSize);
        logger.info("Cache refreshed: entries={}, duration={}ms", count, duration);
        
        // Integration points
        logger.info("External API called: service={}, endpoint={}, responseTime={}ms", 
                    service, endpoint, responseTime);
        logger.info("Message sent to queue: topic={}, messageId={}", topic, messageId);
    }
    
    // ========================================
    // ✓ WHAT TO LOG: State Changes
    // ========================================
    
    public void goodLogging_StateChanges() {
        logger.info("Order status changed: orderId={}, from={}, to={}", orderId, oldStatus, newStatus);
        logger.info("User account activated: userId={}", userId);
        logger.info("Configuration reloaded: changed={}, total={}", changedCount, totalCount);
        logger.info("Background job started: jobId={}, type={}", jobId, jobType);
        logger.info("Background job completed: jobId={}, duration={}ms, status={}", 
                    jobId, duration, status);
    }
    
    // ========================================
    // ✓ WHAT TO LOG: Errors and Warnings
    // ========================================
    
    public void goodLogging_ErrorsAndWarnings() {
        // Errors
        logger.error("Failed to process order: orderId={}, reason={}", orderId, reason, exception);
        logger.error("Database query failed: query={}, params={}", query, params, exception);
        logger.error("External service unavailable: service={}, timeout={}ms", service, timeout);
        
        // Warnings
        logger.warn("Retry attempt {}/{} for operation: {}", attempt, maxAttempts, operation);
        logger.warn("Cache miss rate high: {}%, threshold={}%", missRate, threshold);
        logger.warn("Deprecated API endpoint called: {}, use {} instead", oldEndpoint, newEndpoint);
        logger.warn("Response time exceeded SLA: {}ms, SLA={}ms, endpoint={}", 
                    responseTime, slaTime, endpoint);
    }
    
    // ========================================
    // ✓ WHAT TO LOG: Security Events
    // ========================================
    
    public void goodLogging_SecurityEvents() {
        logger.warn("Failed login attempt: username={}, ip={}, reason={}", username, ip, reason);
        logger.warn("Multiple failed login attempts: username={}, count={}, lastAttempt={}", 
                    username, count, timestamp);
        logger.info("User logged out: username={}, sessionDuration={}ms", username, duration);
        logger.error("Unauthorized access attempt: username={}, resource={}, ip={}", 
                     username, resource, ip);
        logger.warn("Password reset requested: username={}, ip={}", username, ip);
    }
    
    // ========================================
    // ✓ WHAT TO LOG: Performance Metrics
    // ========================================
    
    public void goodLogging_PerformanceMetrics() {
        logger.info("Request processed: endpoint={}, method={}, duration={}ms, status={}", 
                    endpoint, method, duration, status);
        logger.info("Database query executed: query={}, rows={}, duration={}ms", 
                    queryType, rowCount, duration);
        logger.warn("Slow query detected: query={}, duration={}ms, threshold={}ms", 
                    queryType, duration, threshold);
    }
    
    // ========================================
    // ✗ WHAT NOT TO LOG: Noise
    // ========================================
    
    public void badLogging_Noise() {
        // ✗ BAD: Every method entry/exit
        logger.debug("Entering calculateTotal method");
        logger.debug("Exiting calculateTotal method");
        
        // ✗ BAD: Variable assignments (too verbose)
        logger.debug("Setting variable x to 5");
        logger.debug("Incrementing counter");
        
        // ✗ BAD: Obvious information
        logger.info("Starting loop");
        logger.info("Checking if user is null");
        
        // ✗ BAD: Every iteration in loop
        for (int i = 0; i < 10000; i++) {
            logger.debug("Processing item {}", i);  // Creates 10,000 log lines!
        }
    }
    
    // ========================================
    // ✗ WHAT NOT TO LOG: Sensitive Data
    // ========================================
    
    public void badLogging_SensitiveData() {
        // ✗ NEVER LOG THESE:
        logger.info("User password: {}", password);  // ← Security breach!
        logger.info("Credit card: {}", creditCard);  // ← PCI violation!
        logger.info("SSN: {}", ssn);                // ← Privacy violation!
        logger.info("API key: {}", apiKey);         // ← Security breach!
        logger.info("JWT token: {}", jwt);          // ← Security breach!
        logger.info("Session cookie: {}", cookie);  // ← Security breach!
    }
    
    // ========================================
    // ✗ WHAT NOT TO LOG: Redundant Information
    // ========================================
    
    public void badLogging_Redundant() {
        // ✗ BAD: Already in stack trace
        try {
            riskyOperation();
        } catch (Exception e) {
            logger.error("Exception occurred: {}", e.getClass().getName(), e);
            // Redundant! Exception already includes this
        }
        
        // ✗ BAD: Framework already logs this
        logger.info("HTTP request received: {}", request.getRequestURI());
        // Framework access logs already capture this
    }
    
    // ========================================
    // ✗ WHAT NOT TO LOG: Large Objects
    // ========================================
    
    public void badLogging_LargeObjects() {
        // ✗ BAD: Entire object dump
        logger.debug("Request body: {}", largeObject);  // Could be megabytes!
        
        // ✓ GOOD: Key fields only
        logger.debug("Request received: userId={}, action={}", 
                     request.getUserId(), request.getAction());
    }
}

 * Ask these questions:
 * 
 * 1. Will this help me debug a production issue?
 *    NO → Don't log
 *    YES → Continue
 * 
 * 2. Is this a business event or state change?
 *    YES → Log at INFO
 *    NO → Continue
 * 
 * 3. Is this an error or exception?
 *    YES → Log at ERROR with exception
 *    NO → Continue
 * 
 * 4. Is this a recoverable warning?
 *    YES → Log at WARN
 *    NO → Continue
 * 
 * 5. Is this useful only during development?
 *    YES → Log at DEBUG
 *    NO → Don't log
 * 
 * 6. Does it contain sensitive data?
 *    YES → Don't log OR sanitize first
 *    NO → Continue
 * 
 * 7. Will this create noise (high volume)?
 *    YES → Don't log OR use sampling
 *    NO → Log it
 */

// ========================================
// LOGGING CHECKLIST
// ========================================

/**
 * For each log statement, verify:
 * 
 * ✓ Answers a diagnostic question
 * ✓ Appropriate level (INFO/DEBUG/ERROR)
 * ✓ Includes context (IDs, values)
 * ✓ No sensitive data
 * ✓ Won't create excessive volume
 * ✓ Uses parameterized messages
 * ✓ Exceptions include stack trace
 * ✓ Not already logged elsewhere
 */
```

## 2. Log Levels (TRACE, DEBUG, INFO, WARN, ERROR)

### Using the Right Level

```java
public class LogLevelExamples {
    
    private static final Logger logger = LoggerFactory.getLogger(LogLevelExamples.class);
    
    // ========================================
    // TRACE: Detailed Execution Flow
    // 
    // Use for: Very detailed debugging
    // Volume: VERY HIGH
    // Production: OFF
    // ========================================
    
    public void traceLevel() {
        logger.trace("→ Entering calculateDiscount method");
        logger.trace("  Input: orderTotal={}, customerType={}", total, type);
        
        if (type.equals("PREMIUM")) {
            logger.trace("  Customer is premium, applying 10% discount");
        }
        
        logger.trace("  Calculated discount: {}", discount);
        logger.trace("← Exiting calculateDiscount method, result={}", finalTotal);
        
        // When to use:
        // - Tracing execution path
        // - Understanding complex logic flow
        // - Rarely needed, even in development
    }
    
    // ========================================
    // DEBUG: Development Information
    // 
    // Use for: Debugging during development
    // Volume: HIGH
    // Production: OFF (enable for troubleshooting)
    // ========================================
    
    public void debugLevel() {
        logger.debug("Processing order: orderId={}", orderId);
        logger.debug("User preferences loaded: {}", preferences);
        logger.debug("Applying discount: type={}, amount={}", discountType, amount);
        logger.debug("Inventory check: productId={}, available={}", productId, available);
        logger.debug("Tax calculated: amount={}, rate={}", taxAmount, taxRate);
        logger.debug("Final total: {}", finalTotal);
        
        // When to use:
        // - Variable values during processing
        // - Intermediate calculation results
        // - Detailed business logic flow
        // - Helpful for developers debugging
    }
    
    // ========================================
    // INFO: Important Business Events
    // 
    // Use for: Normal operations, significant events
    // Volume: MEDIUM
    // Production: ON (default)
    // ========================================
    
    public void infoLevel() {
        // User actions
        logger.info("User registered: userId={}, email={}", userId, email);
        logger.info("Order placed: orderId={}, userId={}, total=${}", orderId, userId, total);
        logger.info("Payment received: orderId={}, amount=${}, method={}", orderId, amount, method);
        
        // System events
        logger.info("Application started in {}ms", startupTime);
        logger.info("Scheduled job executed: jobName={}, duration={}ms", jobName, duration);
        logger.info("Cache refreshed: entries={}", entryCount);
        
        // Integration events
        logger.info("Email sent: to={}, subject={}", email, subject);
        logger.info("External API call: service={}, endpoint={}, status={}", 
                    service, endpoint, status);
        
        // When to use:
        // - Business events worth tracking
        // - State changes
        // - Milestones in processing
        // - Things you'd want in production logs
    }
    
    // ========================================
    // WARN: Potential Issues (Recoverable)
    // 
    // Use for: Unexpected situations that are handled
    // Volume: LOW
    // Production: ON
    // ========================================
    
    public void warnLevel() {
        // Retry scenarios
        logger.warn("API call failed, retry {}/{}: service={}, error={}", 
                    attempt, maxAttempts, service, error);
        
        // Degraded functionality
        logger.warn("Cache unavailable, falling back to database: reason={}", reason);
        logger.warn("Using default configuration, custom config not found: {}", configFile);
        
        // Performance issues
        logger.warn("Query exceeded threshold: duration={}ms, threshold={}ms, query={}", 
                    duration, threshold, query);
        logger.warn("Queue size high: current={}, threshold={}", queueSize, threshold);
        
        // Deprecated usage
        logger.warn("Deprecated method called: {} by {}, use {} instead", 
                    method, caller, replacement);
        
        // Data issues
        logger.warn("Invalid data received, using default: field={}, value={}, default={}", 
                    field, value, defaultValue);
        
        // When to use:
        // - Something unexpected but handled
        // - Degraded operations
        // - Performance degradation
        // - Things that might become errors
        // - Deprecated feature usage
    }
    
    // ========================================
    // ERROR: Errors Requiring Attention
    // 
    // Use for: Failures, exceptions, errors
    // Volume: VERY LOW (ideally!)
    // Production: ON (always)
    // ========================================
    
    public void errorLevel() {
        // Exceptions (ALWAYS include exception object!)
        try {
            processPayment();
        } catch (PaymentException e) {
            logger.error("Payment processing failed: orderId={}, amount={}, reason={}", 
                         orderId, amount, e.getMessage(), e);
        }
        
        // System failures
        logger.error("Database connection failed: host={}, attempt={}", host, attempt, exception);
        logger.error("Unable to send email: to={}, subject={}", email, subject, exception);
        logger.error("External service timeout: service={}, timeout={}ms", service, timeout);
        
        // Data integrity issues
        logger.error("Data validation failed: record={}, errors={}", recordId, errors);
        logger.error("Duplicate key violation: table={}, key={}", table, key, exception);
        
        // Security issues
        logger.error("Authentication failed: username={}, ip={}, reason={}", 
                     username, ip, reason);
        logger.error("Authorization denied: userId={}, resource={}, action={}", 
                     userId, resource, action);
        
        // Configuration errors
        logger.error("Required configuration missing: key={}", configKey);
        logger.error("Invalid configuration value: key={}, value={}", key, value);
        
        // When to use:
        // - Exceptions and failures
        // - Operations that couldn't complete
        // - Data corruption
        // - System errors
        // - Things requiring immediate attention
    }
}
```

## 3. Parameterized Logging

### Efficient Message Formatting

```java
public class ParameterizedLoggingExamples {
    
    private static final Logger logger = LoggerFactory.getLogger(ParameterizedLoggingExamples.class);
    
    // ========================================
    // ✗ BAD: String Concatenation
    // ========================================
    
    public void badConcatenation() {
        String user = "john";
        int age = 30;
        double salary = 50000.0;
        
        // ✗ ALWAYS executes, even if DEBUG disabled!
        logger.debug("User: " + user + ", Age: " + age + ", Salary: " + salary);
        
        // Creates 6 temporary String objects:
        // 1. "User: "
        // 2. "User: john"
        // 3. "User: john, Age: "
        // 4. "User: john, Age: 30"
        // 5. "User: john, Age: 30, Salary: "
        // 6. "User: john, Age: 30, Salary: 50000.0"
        
        // All this happens EVEN IF DEBUG IS DISABLED!
        // Wastes CPU and creates garbage
    }
    
    // ========================================
    // ✓ GOOD: Parameterized Messages
    // ========================================
    
    public void goodParameterized() {
        String user = "john";
        int age = 30;
        double salary = 50000.0;
        
        // ✓ Only formats if DEBUG enabled
        logger.debug("User: {}, Age: {}, Salary: {}", user, age, salary);
        
        // If DEBUG disabled:
        // - No string concatenation
        // - No object creation
        // - Just a method call that returns immediately
        
        // If DEBUG enabled:
        // - Parameters passed as array
        // - Formatted only once
        // - Much more efficient
    }
    
    // ========================================
    // PARAMETERIZED SYNTAX
    // ========================================
    
    public void parameterizedSyntax() {
        // Single parameter
        logger.info("User logged in: {}", username);
        
        // Multiple parameters
        logger.info("Order created: id={}, user={}, total={}", orderId, userId, total);
        
        // Many parameters (uses varargs)
        logger.info("Details: {}, {}, {}, {}, {}", p1, p2, p3, p4, p5);
        
        // Last parameter is exception (special handling)
        logger.error("Failed to process: userId={}", userId, exception);
        // Exception gets full stack trace, not toString()
    }
    
    // ========================================
    // EXPENSIVE OPERATIONS
    // ========================================
    
    public void expensiveOperations() {
        // ✗ BAD: Method always executes
        logger.debug("Result: {}", expensiveComputation());
        // expensiveComputation() runs even if DEBUG disabled!
        
        // ✓ GOOD: Check level first (for very expensive)
        if (logger.isDebugEnabled()) {
            logger.debug("Result: {}", expensiveComputation());
        }
        
        // ✓ BETTER: Lambda (SLF4J 2.0+, Log4j2)
        logger.debug("Result: {}", () -> expensiveComputation());
        // Only executes if DEBUG enabled
    }
    
    // ========================================
    // OBJECT toString()
    // ========================================
    
    public void objectToString() {
        User user = new User("john", 30);
        
        // ✗ BAD: toString() always called
        logger.debug("User: {}", user.toString());
        
        // ✓ GOOD: toString() only if DEBUG enabled
        logger.debug("User: {}", user);
        // SLF4J calls toString() during formatting
        
        // ✓ BETTER: Log specific fields
        logger.debug("User: name={}, age={}", user.getName(), user.getAge());
        // More explicit, better performance
    }
    
    // ========================================
    // ARRAY PARAMETERS
    // ========================================
    
    public void arrayParameters() {
        String[] items = {"apple", "banana", "orange"};
        
        // Arrays are automatically formatted
        logger.debug("Items: {}", items);
        // Output: Items: [apple, banana, orange]
        
        // For custom formatting
        logger.debug("Items: {}", Arrays.toString(items));
    }
    
    // ========================================
    // PERFORMANCE COMPARISON
    // ========================================
    
    public void performanceComparison() {
        // Benchmark: 1,000,000 log calls with DEBUG disabled
        
        // String concatenation: ~500ms
        for (int i = 0; i < 1_000_000; i++) {
            logger.debug("Value: " + i);  // ✗ Slow!
        }
        
        // Parameterized: ~50ms
        for (int i = 0; i < 1_000_000; i++) {
            logger.debug("Value: {}", i);  // ✓ Fast!
        }
        
        // Parameterized is 10x faster!
    }
    
    private String expensiveComputation() {
        // Simulate expensive operation
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        return "result";
    }
}

// ========================================
// BEST PRACTICES
// ========================================

/**
 * DO:
 * ✓ Use parameterized messages: logger.info("User: {}", user)
 * ✓ Check level for expensive ops: if (logger.isDebugEnabled())
 * ✓ Use lambdas for lazy evaluation (SLF4J 2.0+)
 * ✓ Let SLF4J call toString() automatically
 * 
 * DON'T:
 * ✗ Use string concatenation: "User: " + user
 * ✗ Call toString() explicitly: user.toString()
 * ✗ Call expensive methods in parameters (unless level-checked)
 * ✗ Format manually: String.format("User: %s", user)
 */
```

## 4. Logging Exceptions Properly

### Capturing Full Error Context

```java
public class ExceptionLoggingExamples {
    
    private static final Logger logger = LoggerFactory.getLogger(ExceptionLoggingExamples.class);
    
    // ========================================
    // ✗ BAD: Exception Logging Mistakes
    // ========================================
    
    public void badExceptionLogging() {
        try {
            processPayment();
        } catch (Exception e) {
            // ✗ BAD: Loses stack trace
            logger.error("Error occurred: " + e.getMessage());
            
            // ✗ BAD: toString() truncates stack
            logger.error("Error: {}", e.toString());
            
            // ✗ BAD: getMessage() loses exception type and stack
            logger.error("Error: {}", e.getMessage());
            
            // ✗ BAD: Empty catch (swallows exception)
            // No log at all!
            
            // ✗ BAD: Only stack trace, no context
            logger.error("Error", e);
            // What was being processed? What data?
        }
    }
    
    // ========================================
    // ✓ GOOD: Exception Logging with Context
    // ========================================
    
    public void goodExceptionLogging() {
        try {
            paymentService.processPayment(orderId, amount);
        } catch (PaymentException e) {
            // ✓ Include business context AND exception
            logger.error("Payment processing failed: orderId={}, amount=${}, reason={}", 
                         orderId, amount, e.getMessage(), e);
            
            // Last parameter is exception (gets full stack trace)
            // Previous parameters provide business context
        }
    }
    
    // ========================================
    // EXCEPTION LOGGING PATTERNS
    // ========================================
    
    public void exceptionLoggingPatterns() {
        // Pattern 1: Log and rethrow
        try {
            criticalOperation();
        } catch (Exception e) {
            logger.error("Critical operation failed: context={}", context, e);
            throw e;  // Let caller handle
        }
        
        // Pattern 2: Log and wrap
        try {
            databaseOperation();
        } catch (SQLException e) {
            logger.error("Database operation failed: query={}", query, e);
            throw new DataAccessException("Database error", e);
        }
        
        // Pattern 3: Log and recover
        try {
            sendEmail();
        } catch (EmailException e) {
            logger.warn("Email send failed, will retry: to={}, attempt={}", 
                        email, attempt, e);
            retryQueue.add(emailTask);
        }
        
        // Pattern 4: Log and use fallback
        User user;
        try {
            user = userService.getUser(userId);
        } catch (Exception e) {
            logger.warn("Failed to load user, using cached: userId={}", userId, e);
            user = cache.get(userId);
        }
    }
    
    // ========================================
    // MULTIPLE EXCEPTIONS
    // ========================================
    
    public void multipleExceptions() {
        try {
            complexOperation();
        } catch (ValidationException e) {
            logger.error("Validation failed: errors={}", e.getErrors(), e);
        } catch (BusinessRuleException e) {
            logger.error("Business rule violated: rule={}, reason={}", 
                         e.getRule(), e.getReason(), e);
        } catch (DatabaseException e) {
            logger.error("Database error: operation={}, table={}", 
                         e.getOperation(), e.getTable(), e);
        } catch (Exception e) {
            // Catch-all for unexpected
            logger.error("Unexpected error in operation: context={}", context, e);
        }
    }
    
    // ========================================
    // SUPPRESSED EXCEPTIONS (Java 7+)
    // ========================================
    
    public void suppressedExceptions() {
        try (Connection conn = getConnection();
             Statement stmt = conn.createStatement()) {
            
            stmt.execute(query);
            
        } catch (SQLException e) {
            // Log main exception
            logger.error("Database operation failed: query={}", query, e);
            
            // Check for suppressed exceptions (from close())
            Throwable[] suppressed = e.getSuppressed();
            if (suppressed.length > 0) {
                logger.error("Suppressed exceptions during cleanup: count={}", 
                             suppressed.length);
                for (Throwable t : suppressed) {
                    logger.error("Suppressed: {}", t.getMessage(), t);
                }
            }
        }
    }
    
    // ========================================
    // ROOT CAUSE LOGGING
    // ========================================
    
    public void rootCauseLogging() {
        try {
            complexOperation();
        } catch (Exception e) {
            // Get root cause
            Throwable rootCause = getRootCause(e);
            
            logger.error("Operation failed: immediate={}, root={}, context={}", 
                         e.getClass().getSimpleName(),
                         rootCause.getClass().getSimpleName(),
                         context,
                         e);  // Full exception for stack trace
        }
    }
    
    private Throwable getRootCause(Throwable e) {
        Throwable cause = e;
        while (cause.getCause() != null) {
            cause = cause.getCause();
        }
        return cause;
    }
    
    // ========================================
    // ASYNC EXCEPTIONS
    // ========================================
    
    public void asyncExceptions() {
        CompletableFuture.supplyAsync(() -> riskyOperation())
            .exceptionally(ex -> {
                // Log in async context
                logger.error("Async operation failed: taskId={}, reason={}", 
                             taskId, ex.getMessage(), ex);
                return fallbackValue;
            });
    }
}

// EXCEPTION LOGGING RULES
// ========================================

/**
 * RULES:
 * 
 * 1. ALWAYS include exception object as last parameter
 *    logger.error("Failed: context={}", value, exception);
 * 
 * 2. ALWAYS add business context
 *    Not just: logger.error("Error", e);
 *    But: logger.error("Failed to process order: id={}", orderId, e);
 * 
 * 3. LOG ONCE per exception
 *    Don't log at every catch level
 *    Log at the level that handles it
 * 
 * 4. LOG AND RETHROW or LOG AND HANDLE
 *    Not both in same method
 * 
 * 5. USE appropriate level
 *    ERROR: Unexpected failures
 *    WARN: Expected but recoverable errors
 * 
 * 6. INCLUDE relevant data
 *    User ID, order ID, request ID, etc.
 *    (But sanitize sensitive data first!)
 */
```

```java
public class PerformanceConsiderations {
    
    private static final Logger logger = LoggerFactory.getLogger(PerformanceConsiderations.class);
    
    // ========================================
    // HOT PATH LOGGING
    // ========================================
    
    public void hotPathLogging() {
        // ✗ BAD: Logging in hot loop
        for (int i = 0; i < 1_000_000; i++) {
            logger.debug("Processing item {}", i);  // 1M log calls!
        }
        
        // ✓ GOOD: Sample logging
        for (int i = 0; i < 1_000_000; i++) {
            if (i % 10000 == 0) {  // Log every 10,000
                logger.debug("Progress: {}/{}", i, 1_000_000);
            }
        }
        
        // ✓ BETTER: Summary logging
        for (int i = 0; i < 1_000_000; i++) {
            processItem(i);
        }
        logger.info("Batch complete: processed={}, duration={}ms", 
                    1_000_000, duration);
    }
    
    // ========================================
    // LEVEL CHECKING
    // ========================================
    
    public void levelChecking() {
        // For simple messages, NO need to check
        logger.debug("Processing user {}", userId);  // SLF4J handles efficiently
        
        // For EXPENSIVE operations, CHECK level
        if (logger.isDebugEnabled()) {
            String expensiveDebugInfo = buildComplexDiagnostics();
            logger.debug("Diagnostics: {}", expensiveDebugInfo);
        }
        
        // Examples of "expensive":
        // - Large object serialization
        // - Complex string building
        // - Database queries for logging
        // - API calls for logging
    }
    
    // ========================================
    // LAZY EVALUATION
    // ========================================
    
    public void lazyEvaluation() {
        // SLF4J 2.0+ and Log4j2 support lambdas
        logger.debug("Result: {}", () -> computeExpensiveResult());
        
        // Lambda only executes if DEBUG enabled
        // No need for isDebugEnabled() check
    }
    
    // ========================================
    // ASYNC LOGGING
    // ========================================
    
    public void asyncLogging() {
        // Logback: AsyncAppender
        // Log4j2: AsyncLogger
        
        // Benefits:
        // - Logging doesn't block application thread
        // - I/O happens in background
        // - 5-10x throughput improvement
        
        // Trade-offs:
        // - Logs might be lost on crash (buffered)
        // - Increased memory (buffer)
        // - Potential reordering
        
        // When to use:
        // - High throughput (>1000 logs/sec)
        // - Latency-sensitive operations
        // - I/O-bound logging
    }
    
    // ========================================
    // LARGE OBJECT LOGGING
    // ========================================
    
    public void largeObjectLogging() {
        List<String> largeList = getLargeList();  // 10,000 items
        
        // ✗ BAD: Logs entire list
        logger.debug("Items: {}", largeList);  // Huge log line!
        
        // ✓ GOOD: Log summary
        logger.debug("Items: count={}, first={}, last={}", 
                     largeList.size(), 
                     largeList.get(0), 
                     largeList.get(largeList.size() - 1));
        
        // ✓ GOOD: Truncate
        logger.debug("Items (first 10): {}", largeList.subList(0, 10));
    }
    
    // ========================================
    // CONDITIONAL COMPILATION
    // ========================================
    
    public void conditionalCompilation() {
        // For absolute minimum overhead, use final boolean
        private static final boolean DEBUG_ENABLED = false;
        
        if (DEBUG_ENABLED) {
            logger.debug("Debug info: {}", expensiveOperation());
        }
        
        // JVM can optimize away entire if block (dead code elimination)
        // Zero overhead in production
    }
    
    // ========================================
    // MEASURING LOGGING OVERHEAD
    // ========================================
    
    public void measureLoggingOverhead() {
        // Typical overhead:
        // - Parameterized, level disabled: ~10 nanoseconds
        // - Parameterized, level enabled (sync): ~1-5 microseconds
        // - Parameterized, level enabled (async): ~100 nanoseconds
        // - String concatenation (disabled): ~100 nanoseconds
        
        // For 1M calls:
        // - Parameterized disabled: 10ms
        // - String concat disabled: 100ms
        // - Sync logging enabled: 1-5 seconds
        // - Async logging enabled: 100ms
    }
    
    // ========================================
    // PERFORMANCE BEST PRACTICES
    // ========================================
    
    public void performanceBestPractices() {
        // ✓ Use parameterized messages
        logger.debug("User: {}", user);
        
        // ✓ Check level for expensive operations
        if (logger.isDebugEnabled()) {
            logger.debug("Expensive: {}", expensive());
        }
        
        // ✓ Use async logging for high throughput
        // (configured in logback.xml/log4j2.xml)
        
        // ✓ Sample high-frequency logs
        if (counter++ % 1000 == 0) {

        }
        
        // ✓ Log summaries, not details
        logger.info("Batch processed: items={}, duration={}ms, errors={}",

        // ✗ Avoid logging in tight loops
        // ✗ Avoid expensive operations in log parameters
        // ✗ Avoid large object dumps
    }
```

## 6. PII and Security in Logs

### Protecting Sensitive Data

```java
public class SecureLogging {
    
    private static final Logger logger = LoggerFactory.getLogger(SecureLogging.class);
    
    // ========================================
    // ✗ NEVER LOG THESE
    // ========================================
    
    public void neverLogThese() {
        // ✗ Passwords
        logger.info("Login attempt: username={}, password={}", username, password);  // BREACH!
        
        // ✗ Credit cards
        logger.info("Payment: card={}", creditCardNumber);  // PCI VIOLATION!
        
        // ✗ SSN/National IDs
        logger.info("User registered: ssn={}", ssn);  // PRIVACY VIOLATION!
        
        // ✗ API keys/secrets
        logger.info("Calling API: key={}", apiKey);  // SECURITY BREACH!
        
        // ✗ Tokens/JWTs
        logger.info("Auth token: {}", jwtToken);  // SECURITY BREACH!
        
        // ✗ Session IDs
        logger.info("Session: {}", sessionId);  // Can lead to hijacking
        
        // ✗ Full user details
        logger.info("User data: {}", user.toJson());  // May contain PII
    }
    
    // ========================================
    // DATA MASKING
    // ========================================
    
    public void dataMasking() {
        String creditCard = "1234-5678-9012-3456";
        String ssn = "123-45-6789";
        String email = "john.doe@example.com";
        
        // ✓ Mask credit card
        logger.info("Payment processed: card={}", maskCreditCard(creditCard));
        // Output: card=****-****-****-3456
        
        // ✓ Mask SSN
        logger.info("User registered: ssn={}", maskSSN(ssn));
        // Output: ssn=***-**-6789
        
        // ✓ Mask email
        logger.info("Email sent: to={}", maskEmail(email));
        // Output: to=j***@example.com
    }
    
    private String maskCreditCard(String card) {
        if (card == null || card.length() < 4) return "****";
        return "****-****-****-" + card.substring(card.length() - 4);
    }
    
    private String maskSSN(String ssn) {
        if (ssn == null || ssn.length() < 4) return "***";
        return "***-**-" + ssn.substring(ssn.length() - 4);
    }
    
    private String maskEmail(String email) {
        if (email == null || !email.contains("@")) return "***";
        String[] parts = email.split("@");
        return parts[0].charAt(0) + "***@" + parts[1];
    }
    
    // ========================================
    // SAFE LOGGING PATTERNS
    // ========================================
    
    public void safeLoggingPatterns() {
        // ✓ Log IDs, not full objects
        logger.info("User login: userId={}", user.getId());
        // Not: logger.info("User login: {}", user);
        
        // ✓ Log action, not data
        logger.info("Payment processed: orderId={}, amount=${}", orderId, amount);
        // Not: logger.info("Payment: {}", paymentDetails);
        
        // ✓ Log masked/hashed values
        logger.info("Password change: userId={}, hash={}", 
                    userId, hashPassword(newPassword));
        
        // ✓ Log IP address (usually OK, but check regulations)
        logger.info("Login from: userId={}, ip={}", userId, ipAddress);
    }
    
    // ========================================
    // GDPR COMPLIANCE
    // ========================================
    
    public void gdprCompliance() {
        // Under GDPR, personal data in logs must be:
        // 1. Minimized (only necessary data)
        // 2. Protected (encrypted at rest)
        // 3. Deletable (can purge user's logs)
        
        // ✓ GOOD: Pseudonymize user data
        String pseudoId = hashUserId(userId);
        logger.info("Action performed: pseudoId={}, action={}", pseudoId, action);
        
        // ✓ GOOD: Don't log personal details
        logger.info("User profile updated: userId={}, fields={}", userId, fieldCount);
        // Not: logger.info("Profile: name={}, email={}, phone={}", ...)
        
        // ✓ GOOD: Use separate audit log for sensitive operations
        auditLogger.info("Sensitive operation: userId={}, operation={}, timestamp={}", 
                         userId, operation, timestamp);
        // Audit logs have different retention and access controls
    }
    
    // ========================================
    // SANITIZING USER INPUT
    // ========================================
    
    public void sanitizeUserInput() {
        String userInput = request.getParameter("search");
        
        // ✗ BAD: Log raw user input
        logger.info("Search query: {}", userInput);  // Can contain sensitive data!
        
        // ✓ GOOD: Sanitize first
        String sanitized = sanitize(userInput);
        logger.info("Search query: {}", sanitized);
        
        // ✓ GOOD: Log length/type, not content
        logger.info("Search query received: length={}, type={}", 
                    userInput.length(), getQueryType(userInput));
    }
    
    private String sanitize(String input) {
        if (input == null) return null;
        // Remove potential PII patterns
        return input.replaceAll("\\d{3}-\\d{2}-\\d{4}", "***-**-****")  // SSN
                    .replaceAll("\\d{4}-\\d{4}-\\d{4}-\\d{4}", "****-****-****-****");  // CC
    }
    
    // ========================================
    // LOG ENCRYPTION
    // ========================================
    
    public void logEncryption() {
        // For highly sensitive data that MUST be logged:
        
        // 1. Encrypt log files at rest
        // 2. Use encrypted transport (TLS) for log shipping
        // 3. Limit access to encrypted logs
        // 4. Regular log rotation and secure deletion
        
        // Example: AWS CloudWatch Logs with KMS encryption
        // Example: Splunk with SSL/TLS transport
    }
    
    // ========================================
    // SECURITY BEST PRACTICES
    // ========================================
    
    public void securityBestPractices() {
        // ✓ DO:
        // - Log security events (login, logout, failures)
        // - Log authorization failures
        // - Mask sensitive data
        // - Use separate audit logs
        // - Encrypt logs at rest
        // - Limit log access
        // - Regular log review
        // - Define retention policy
        
        // ✗ DON'T:
        // - Log passwords, tokens, keys
        // - Log credit cards, SSNs
        // - Log full user objects
        // - Log raw user input without sanitization
        // - Keep logs indefinitely
        // - Give everyone log access
    }
    
    private String hashPassword(String password) {
        // Hash for logging (not for storage!)
        return Integer.toHexString(password.hashCode());
    }
    
    private String hashUserId(String userId) {
        // Consistent pseudonymization
        return Integer.toHexString(userId.hashCode());
    }
    
    private String getQueryType(String query) {
        if (query.startsWith("SELECT")) return "SQL_SELECT";
        if (query.contains("@")) return "EMAIL_SEARCH";
        return "TEXT_SEARCH";
    }
}
```

```java
   • Log summaries, not details
```
