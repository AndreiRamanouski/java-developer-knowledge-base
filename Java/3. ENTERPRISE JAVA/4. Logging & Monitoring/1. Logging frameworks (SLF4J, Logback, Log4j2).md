# Logging Frameworks

## Overview

Logging is essential for understanding application behavior, debugging issues, and monitoring production systems—proper logging transforms invisible failures into traceable events. This guide covers seven critical logging concepts: SLF4J Abstraction Layer (Simple Logging Facade for Java—provides unified API, decouples code from implementation, allows switching loggers without code changes, most Java libraries log through SLF4J), Logback Configuration (SLF4J's native implementation—XML/Groovy config, appenders for console/file/syslog, encoders for formatting, rolling policies for file rotation, filters for advanced control), Log4j2 Async Logging (high-performance successor to Log4j—async loggers use LMAX Disruptor, 10x faster than synchronous, configurable via XML/JSON/YAML, lambda support for lazy evaluation), Performance Comparison (Log4j2 > Logback > java.util.logging in throughput, async logging crucial for high-load systems, logging overhead typically 1-5% of application time), Structured Logging (JSON format for machine parsing—enables log aggregation in ELK/Splunk, includes metadata/context, queryable logs, essential for microservices), Log Levels (TRACE for detailed debugging, DEBUG for development, INFO for important events, WARN for recoverable issues, ERROR for failures—production typically INFO level), and MDC (Mapped Diagnostic Context—thread-local map for adding context to logs, attach requestId/userId/traceId to all logs in request, propagate context across threads, crucial for distributed tracing). The key insight: logging is both art and science—too little and you can't debug production, too much and logs become noise or impact performance. Common mistakes: logging in hot paths without checking level, not using parameterized messages (causes unnecessary string concatenation), logging sensitive data (passwords/tokens), forgetting to clean up MDC, using wrong log level (ERROR for recoverable warnings). The relationship between concepts: SLF4J provides API → Logback/Log4j2 implement → async logging improves performance → structured logging enables analysis → MDC adds context → proper levels control verbosity. Real-world logging strategy: use SLF4J everywhere, configure Logback/Log4j2 for production, enable async for high throughput, output JSON for log aggregation, use MDC for request tracking, set INFO level in production with dynamic adjustment for debugging.

## 1. SLF4J Abstraction Layer

### The Universal Logging API

```java
/**
 * SLF4J: Simple Logging Facade for Java
 * 
 * Provides abstraction over logging implementations
 * Most popular: Logback, Log4j2, java.util.logging
 */

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserService {
    
    // ========================================
    // BASIC USAGE
    // ========================================
    
    // Logger instance (one per class)
    private static final Logger logger = LoggerFactory.getLogger(UserService.class);
    
    public void createUser(String username) {
        // Different log levels
        logger.trace("Entering createUser method");
        logger.debug("Creating user with username: {}", username);
        logger.info("User created successfully: {}", username);
        logger.warn("User creation took longer than expected");
        logger.error("Failed to create user: {}", username);
    }
    
    // ========================================
    // PARAMETERIZED LOGGING
    // ========================================
    
    public void parameterizedLogging() {
        String user = "john";
        int age = 30;
        
        // ❌ BAD: String concatenation (always executed)
        logger.debug("User: " + user + ", Age: " + age);
        // Creates string even if DEBUG disabled!
        
        // ✓ GOOD: Parameterized (deferred)
        logger.debug("User: {}, Age: {}", user, age);
        // Only formats if DEBUG enabled
        
        // Multiple parameters
        logger.info("User {} logged in from {} at {}", user, "192.168.1.1", new Date());
    }
    
    // ========================================
    // EXCEPTION LOGGING
    // ========================================
    
    public void exceptionLogging() {
        try {
            // Some operation
            processData();
        } catch (Exception e) {
            // ❌ BAD: Converts to string
            logger.error("Error occurred: " + e);
            
            // ✓ GOOD: Pass exception directly
            logger.error("Error processing data", e);
            
            // With context
            logger.error("Error processing data for user: {}", "john", e);
        }
    }
    
    // ========================================
    // CONDITIONAL LOGGING
    // ========================================
    
    public void conditionalLogging() {
        // For expensive operations
        if (logger.isDebugEnabled()) {
            String expensive = computeExpensiveDebugInfo();
            logger.debug("Debug info: {}", expensive);
        }
        
        // Not needed for simple parameters (SLF4J handles)
        logger.debug("Simple message: {}", "value");  // Good enough
    }
    
    // ========================================
    // LAMBDA LOGGING (SLF4J 2.0+)
    // ========================================
    
    public void lambdaLogging() {
        // Lazy evaluation with lambda
        logger.debug("Expensive computation: {}", 
            () -> performExpensiveComputation());
        
        // Only executes if DEBUG enabled
    }
    
    // ========================================
    // MARKERS (for filtering)
    // ========================================
    
    import org.slf4j.Marker;
    import org.slf4j.MarkerFactory;
    
    private static final Marker SECURITY = MarkerFactory.getMarker("SECURITY");
    private static final Marker AUDIT = MarkerFactory.getMarker("AUDIT");
    
    public void markerLogging() {
        // Tagged logging
        logger.info(SECURITY, "User authentication failed: {}", "john");
        logger.info(AUDIT, "User modified record: {}", "record123");
        
        // Can filter logs by marker in configuration
    }
    
    private void processData() throws Exception {
        // Dummy method
    }
    
    private String computeExpensiveDebugInfo() {
        return "expensive";
    }
    
    private String performExpensiveComputation() {
        return "result";
    }
}

// ========================================
// WHY USE SLF4J?
// ========================================

/**
 * BENEFITS:
 * 
 * 1. DECOUPLING
 *    - Code doesn't depend on specific logger
 *    - Can switch Logback → Log4j2 without code change
 * 
 * 2. LIBRARY COMPATIBILITY
 *    - Most libraries use SLF4J
 *    - Unified logging across dependencies
 * 
 * 3. PERFORMANCE
 *    - Parameterized messages avoid string concat
 *    - Lazy evaluation with lambdas
 * 
 * 4. FLEXIBILITY
 *    - Choose implementation at runtime
 *    - Different loggers for different environments
 */

// ========================================
// SLF4J BRIDGES
// ========================================

/**
 * Redirect other logging frameworks to SLF4J:
 * 
 * Log4j → SLF4J:
 * <dependency>
 *     <groupId>org.slf4j</groupId>
 *     <artifactId>log4j-over-slf4j</artifactId>
 * </dependency>
 * 
 * JUL → SLF4J:
 * <dependency>
 *     <groupId>org.slf4j</groupId>
 *     <artifactId>jul-to-slf4j</artifactId>
 * </dependency>
 * 
 * Commons Logging → SLF4J:
 * <dependency>
 *     <groupId>org.slf4j</groupId>
 *     <artifactId>jcl-over-slf4j</artifactId>
 * </dependency>
 * 
 * All logs go through SLF4J → single configuration
 */

// ========================================
// SLF4J BEST PRACTICES
// ========================================

/**
 * DO:
 * ✓ Use parameterized messages: logger.info("User: {}", user)
 * ✓ Log exceptions with context: logger.error("Failed", e)
 * ✓ Use appropriate log level
 * ✓ One logger per class
 * ✓ Use static final logger
 * 
 * DON'T:
 * ✗ String concatenation: "User: " + user
 * ✗ toString() in parameters: logger.info("User: {}", user.toString())
 * ✗ Logging sensitive data: passwords, tokens
 * ✗ Excessive logging in hot paths
 * ✗ Using System.out.println
 */
```

## 2. Logback Configuration

### Configuring Logback

```xml
<!-- ========================================
     LOGBACK CONFIGURATION
     
     File: src/main/resources/logback.xml
     ======================================== -->

<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    
    <!-- ========================================
         BASIC CONSOLE APPENDER
         ======================================== -->
    
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>
                %d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n
            </pattern>
        </encoder>
    </appender>
    
    <!-- ========================================
         FILE APPENDER
         ======================================== -->
    
    <appender name="FILE" class="ch.qos.logback.core.FileAppender">
        <file>logs/application.log</file>
        <append>true</append>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    
    <!-- ========================================
         ROLLING FILE APPENDER
         
         Rotates logs by size and time
         ======================================== -->
    
    <appender name="ROLLING" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/application.log</file>
        
        <!-- Rolling policy: Size and time based -->
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <!-- Daily rollover -->
            <fileNamePattern>logs/application-%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            
            <!-- Each file max 10MB -->
            <maxFileSize>10MB</maxFileSize>
            
            <!-- Keep 30 days -->
            <maxHistory>30</maxHistory>
            
            <!-- Total size cap -->
            <totalSizeCap>1GB</totalSizeCap>
        </rollingPolicy>
        
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    
    <!-- ========================================
         ASYNC APPENDER
         
         Wraps another appender for async writing
         ======================================== -->
    
    <appender name="ASYNC_FILE" class="ch.qos.logback.classic.AsyncAppender">
        <!-- Reference to actual appender -->
        <appender-ref ref="ROLLING" />
        
        <!-- Queue size -->
        <queueSize>512</queueSize>
        
        <!-- Never block (discard on overflow) -->
        <neverBlock>true</neverBlock>
        
        <!-- Discard threshold -->
        <discardingThreshold>0</discardingThreshold>
    </appender>
    
    <!-- ========================================
         FILTERS
         ======================================== -->
    
    <appender name="ERROR_FILE" class="ch.qos.logback.core.FileAppender">
        <file>logs/error.log</file>
        
        <!-- Only ERROR and above -->
        <filter class="ch.qos.logback.classic.filter.ThresholdFilter">
            <level>ERROR</level>
        </filter>
        
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    
    <!-- ========================================
         PATTERN SYNTAX
         ======================================== -->
    
    <!--
    %d{pattern}     - Date/time (ISO8601, yyyy-MM-dd HH:mm:ss.SSS)
    %thread         - Thread name
    %-5level        - Log level (left-aligned, 5 chars)
    %logger{length} - Logger name (abbreviated)
    %msg            - Log message
    %n              - Newline
    %ex             - Exception stack trace
    %mdc{key}       - MDC value
    %X{key}         - Same as %mdc{key}
    -->
    
    <!-- Example patterns: -->
    <appender name="DETAILED" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>
                %d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} [%X{requestId}] - %msg%n
            </pattern>
        </encoder>
    </appender>
    
    <!-- ========================================
         LOGGERS
         
         Configure log levels per package
         ======================================== -->
    
    <!-- Root logger (default) -->
    <root level="INFO">
        <appender-ref ref="CONSOLE" />
        <appender-ref ref="ASYNC_FILE" />
    </root>
    
    <!-- Package-specific loggers -->
    <logger name="com.example.service" level="DEBUG" />
    <logger name="com.example.repository" level="DEBUG" />
    
    <!-- Third-party libraries (less verbose) -->
    <logger name="org.springframework" level="INFO" />
    <logger name="org.hibernate" level="WARN" />
    
    <!-- Disable specific logger -->
    <logger name="com.noisy.library" level="OFF" />
    
    <!-- Logger with specific appender -->
    <logger name="com.example.audit" level="INFO" additivity="false">
        <appender-ref ref="AUDIT_FILE" />
    </logger>
    
    <!-- ========================================
         ENVIRONMENT-SPECIFIC CONFIGURATION
         ======================================== -->
    
    <!-- Spring Boot profiles -->
    <springProfile name="dev">
        <root level="DEBUG">
            <appender-ref ref="CONSOLE" />
        </root>
    </springProfile>
    
    <springProfile name="prod">
        <root level="INFO">
            <appender-ref ref="ASYNC_FILE" />
        </root>
    </springProfile>
    
    <!-- ========================================
         CONDITIONAL CONFIGURATION
         ======================================== -->
    
    <if condition='property("LOG_TO_FILE").equals("true")'>
        <then>
            <appender-ref ref="FILE" />
        </then>
    </if>
    
</configuration>
```

```xml
     ADVANCED LOGBACK CONFIGURATION

    </appender>
    
    <!-- ========================================

            <level>ERROR</level>
            <onMatch>ACCEPT</onMatch>
            <onMismatch>DENY</onMismatch>
        </filter>
        
        <layout class="ch.qos.logback.classic.html.HTMLLayout"/>
    </appender>
    
    <root level="INFO">
        <appender-ref ref="CONSOLE" />
        <appender-ref ref="ROLLING" />
        <appender-ref ref="EMAIL" />
    </root>
    
</configuration>
```

## 3. Log4j2 Async Logging

### High-Performance Logging

```xml
<!-- ========================================
     LOG4J2 CONFIGURATION
     
     File: src/main/resources/log4j2.xml
     ======================================== -->

<?xml version="1.0" encoding="UTF-8"?>
<Configuration status="WARN">
    
    <Properties>
        <Property name="LOG_PATTERN">
            %d{yyyy-MM-dd HH:mm:ss.SSS} [%t] %-5level %logger{36} - %msg%n
        </Property>
    </Properties>
    
    <Appenders>
        <!-- ========================================
             CONSOLE APPENDER
             ======================================== -->
        
        <Console name="Console" target="SYSTEM_OUT">
            <PatternLayout pattern="${LOG_PATTERN}"/>
        </Console>
        
        <!-- ========================================
             ROLLING FILE APPENDER
             ======================================== -->
        
        <RollingFile name="RollingFile"
                     fileName="logs/application.log"
                     filePattern="logs/application-%d{yyyy-MM-dd}-%i.log.gz">
            
            <PatternLayout pattern="${LOG_PATTERN}"/>
            
            <Policies>
                <!-- Rollover daily -->
                <TimeBasedTriggeringPolicy/>
                
                <!-- Or when file reaches 10MB -->
                <SizeBasedTriggeringPolicy size="10MB"/>
            </Policies>
            
            <!-- Keep max 30 files -->
            <DefaultRolloverStrategy max="30"/>
        </RollingFile>
        
        <!-- ========================================
             ASYNC APPENDER (Wrapper)
             ======================================== -->
        
        <Async name="AsyncFile">
            <AppenderRef ref="RollingFile"/>
        </Async>
        
    </Appenders>
    
    <Loggers>
        <!-- Root logger -->
        <Root level="info">
            <AppenderRef ref="Console"/>
            <AppenderRef ref="AsyncFile"/>
        </Root>
        
        <!-- Package-specific -->
        <Logger name="com.example.service" level="debug" additivity="false">
            <AppenderRef ref="Console"/>
        </Logger>
    </Loggers>
    
</Configuration>
```

```java
// LOG4J2: LAMBDA SUPPORT

import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;

public class Log4j2Example {
    
    private static final Logger logger = LogManager.getLogger(Log4j2Example.class);
    
    public void lambdaLogging() {
        // Lazy evaluation with lambda
        logger.debug("Expensive: {}", () -> expensiveComputation());
        
        // Only executes if DEBUG enabled
        // No need for isDebugEnabled() check
    }
    
    // ========================================
    // LOG4J2: GARBAGE-FREE LOGGING
    // ========================================
    
    public void garbageFree() {
        // Log4j2 can log without creating garbage
        // Reuses buffers
        
        // Enable with system property:
        // -Dlog4j2.enable.threadlocals=true
        
        logger.info("Message: {}", "value");
        // Creates no garbage!
    }
    
    private String expensiveComputation() {
        return "result";
    }
}
```

```java
public class LogLevelsGuide {

    // ========================================
    
    public void debugExample() {
        logger.debug("Processing order: {}", orderId);
        logger.debug("User preferences: {}", preferences);
        logger.debug("Calculated total: ${}", total);
        logger.debug("Applied discount: {}%", discount);
        
        // Production: Usually OFF (enable for troubleshooting)
        // Development: ON
        // Volume: HIGH
    }
    
    // ========================================
    // INFO: Important business events
    // Use for: Normal operation, significant events
    // ========================================
    
    public void infoExample() {

        logger.info("Order created: {}", orderId);
        logger.info("Payment processed: ${}", amount);

        // Volume: MEDIUM

    // ========================================

        logger.warn("Using cached data due to service unavailable");
        logger.warn("Deprecated method called: {}", methodName);
        logger.warn("Configuration value missing, using default: {}", defaultValue);
        logger.warn("Response time exceeded threshold: {}ms", responseTime);
        
        // Production: ON
        // Development: ON
        // Volume: LOW
    }
    
    // ========================================
    // ERROR: Errors that need attention
    // Use for: Exceptions, failed operations
    // ========================================
    
    public void errorExample() {

        logger.error("Database connection failed", exception);
        logger.error("Authentication failed for user: {}", username);
        logger.error("Unable to send notification email", exception);

        // Volume: VERY LOW (hopefully!)
    }
```

```java
import org.springframework.web.servlet.HandlerInterceptor;

public class MDCInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // Add request ID
        String requestId = request.getHeader("X-Request-ID");
        if (requestId == null) {
            requestId = UUID.randomUUID().toString();
        }
        MDC.put("requestId", requestId);
        
        // Add user from security context
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth != null) {
            MDC.put("userId", auth.getName());
        }
        
        return true;
    }
    
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, 
                                Object handler, Exception ex) {
        MDC.clear();
    }
}

public class MDCAsyncExample {
    
    private static final Logger logger = LoggerFactory.getLogger(MDCAsyncExample.class);
    
    public void asyncWithMDC() {
        // Set MDC in main thread
        MDC.put("requestId", "req-123");
        
        // Copy MDC to async thread
        Map<String, String> contextMap = MDC.getCopyOfContextMap();
        
        CompletableFuture.runAsync(() -> {
            try {
                // Restore MDC in async thread
                if (contextMap != null) {
                    MDC.setContextMap(contextMap);
                }
                
                logger.info("Async operation");
                // MDC context preserved!
                
            } finally {
                MDC.clear();
            }
        });
    }
}

// ========================================
// THREAD POOL EXECUTOR WITH MDC
// ========================================

public class MDCThreadPoolExecutor extends ThreadPoolExecutor {
    
    @Override
    public void execute(Runnable command) {
        // Capture MDC from submitting thread
        Map<String, String> contextMap = MDC.getCopyOfContextMap();
        
        // Wrap command to restore MDC
        super.execute(() -> {
            try {
                if (contextMap != null) {
                    MDC.setContextMap(contextMap);
                }
                command.run();
            } finally {
                MDC.clear();
            }
        });
    }
}
```

```java
// ========================================
// LOGGING FRAMEWORKS SUMMARY
// ========================================

1. SLF4J ABSTRACTION
   • Unified API for logging
   • Decouple code from implementation
   • Parameterized messages
   • Bridges from other frameworks
   
   Usage: LoggerFactory.getLogger(MyClass.class)
   Log: logger.info("User: {}", username)

2. LOGBACK CONFIGURATION
   • Native SLF4J implementation
   • XML configuration (logback.xml)
   • Appenders: Console, File, Rolling, Async
   • Filters and patterns
   • Environment-specific config
   
   Rolling: SizeAndTimeBasedRollingPolicy
   Async: AsyncAppender (wrap other appenders)

3. LOG4J2 ASYNC LOGGING
   • High-performance successor to Log4j
   • Async loggers (LMAX Disruptor)
   • 10x faster than sync
   • Garbage-free mode
   • Lambda support
   
   Enable: AsyncLoggerContextSelector
   Performance: 2M messages/second

4. PERFORMANCE COMPARISON
   • Log4j2 > Logback > JUL
   • Async 5-10x faster than sync
   • Log4j2 can be garbage-free
   • Parameterized messages avoid concat
   
   Winner: Log4j2 async (fastest)
   Most popular: Logback (SLF4J native)

5. STRUCTURED LOGGING (JSON)
   • Machine-parseable format
   • For ELK, Splunk, CloudWatch
   • Includes metadata automatically
   • Queryable logs
   
   Library: logstash-logback-encoder
   Output: {"@timestamp":"...", "message":"...", ...}

6. LOG LEVELS
   • TRACE: Very detailed (execution flow)
   • DEBUG: Detailed info (debugging)
   • INFO: Important events (default prod)
   • WARN: Potential issues (recoverable)
   • ERROR: Errors/exceptions (needs attention)
   
   Production: INFO level
   Troubleshooting: DEBUG (temporarily)

7. MDC (Mapped Diagnostic Context)
   • Thread-local context map
   • Add requestId, userId, etc.
   • Automatic in all logs
   • Must copy for async
   
   Usage: MDC.put("requestId", id)
   Cleanup: MDC.clear() in finally
```
