# JUnit 5 Advanced Features

## Overview

JUnit 5 (Jupiter) introduced powerful features that go beyond basic @Test methods. These advanced features make tests more expressive, maintainable, and comprehensive. This guide covers seven essential capabilities: @ParameterizedTest (test same logic with multiple inputs—CSV, Method source, ValueSource, Enum, etc.), @RepeatedTest (run tests multiple times to catch intermittent issues), @TestFactory (dynamically generate tests at runtime), @Nested tests (organize related tests hierarchically with shared setup), Test Lifecycle (@BeforeAll, @BeforeEach, @AfterEach, @AfterAll—controlling setup/teardown), Conditional Execution (run tests based on OS, Java version, environment variables), and Test Ordering (control test execution order when needed). The key insight: JUnit 5 is designed for modern Java—lambdas, streams, and a more expressive API. @ParameterizedTest eliminates duplication by running the same test with different data. @TestFactory creates tests dynamically (useful for data-driven testing). @Nested organizes tests logically (like UserServiceTests > RegistrationTests > EmailValidation). Conditional execution ensures tests run only in appropriate environments. Test ordering is usually bad (tests should be independent) but necessary for integration tests or database migrations. Understanding these features means understanding when to use them: @ParameterizedTest for multiple inputs, @RepeatedTest for flaky tests, @TestFactory for complex scenarios, @Nested for organization, lifecycle hooks carefully (prefer inline setup), conditional execution for platform-specific tests, ordering rarely (document why if used).


**Maven Dependency**:

```xml
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```

## 1. @ParameterizedTest

### Testing with Multiple Inputs

```java
/**
 * @ParameterizedTest
 * 
 * Run same test with different parameters
 * Avoid duplicating test logic
 */

// ========================================
// @ValueSource: Single parameter
// ========================================

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.*;

class ValueSourceTests {
    
    /**
     * Test with string values
     */
    @ParameterizedTest
    @ValueSource(strings = {
        "user@example.com",
        "test@gmail.com",
        "admin@company.org"
    })
    void testValidEmails(String email) {
        assertTrue(EmailValidator.isValid(email));
    }
    
    /**
     * Test with int values
     */
    @ParameterizedTest
    @ValueSource(ints = {1, 2, 5, 10, 100})
    void testPositiveNumbers(int number) {
        assertTrue(number > 0);
    }
    
    /**
     * Test with long values
     */
    @ParameterizedTest
    @ValueSource(longs = {1L, 10L, 100L, 1000L})
    void testLongValues(long value) {
        assertTrue(value > 0);
    }
    
    /**
     * Test with double values
     */
    @ParameterizedTest
    @ValueSource(doubles = {0.1, 0.5, 0.9, 1.0})
    void testProbabilities(double probability) {
        assertTrue(probability >= 0.0 && probability <= 1.0);
    }
    
    /**
     * WHEN TO USE @ValueSource:
     * 
     * ✓ Single parameter tests
     * ✓ Simple primitive/String values
     * ✓ Testing validation, bounds, edge cases
     */
}

            )
        );
    }

     * External method source (different class)

        assertNotNull(data);
    }
    
    /**
     * WHEN TO USE @MethodSource:
     * 
     * ✓ Complex objects (not just primitives/strings)

     * ✓ Programmatically generated test cases
     * ✓ Need logic to build test data
     */
}

class EnumSourceTests {
    
    /**
     * Test all enum values
     */
    @ParameterizedTest
    @EnumSource(DayOfWeek.class)
    void testAllDaysOfWeek(DayOfWeek day) {
        assertNotNull(day);
        assertTrue(DayOfWeek.values().length == 7);
    }
    
    /**
     * Test specific enum values
     */
    @ParameterizedTest
    @EnumSource(value = DayOfWeek.class, names = {"SATURDAY", "SUNDAY"})
    void testWeekendDays(DayOfWeek day) {
        assertTrue(isWeekend(day));
    }
    
    /**
     * Exclude specific values
     */
    @ParameterizedTest
    @EnumSource(value = DayOfWeek.class, mode = EnumSource.Mode.EXCLUDE, 
                names = {"SATURDAY", "SUNDAY"})
    void testWeekdays(DayOfWeek day) {
        assertFalse(isWeekend(day));
    }
    
    /**
     * Pattern matching
     */
    @ParameterizedTest
    @EnumSource(value = OrderStatus.class, mode = EnumSource.Mode.MATCH_ANY,
                names = "^(PENDING|CONFIRMED)$")
    void testCancellableStatuses(OrderStatus status) {
        assertTrue(status.isCancellable());
    }
    
    private boolean isWeekend(DayOfWeek day) {
        return day == DayOfWeek.SATURDAY || day == DayOfWeek.SUNDAY;
    }
    
    /**
     * WHEN TO USE @EnumSource:
     * 
     * ✓ Testing all possible enum values
     * ✓ Ensuring enum handling is complete
     * ✓ Testing subsets of enum values
     */
}

// Helper classes
interface EmailValidator {
    static boolean isValid(String email) {
        return email != null && email.contains("@") && email.contains(".");
    }
}

class Calculator {
    static int add(int a, int b) { return a + b; }
}

class User {
    private String name;
    private String email;
    
    User(String name, String email) {
        this.name = name;
        this.email = email;
    }
    
    String getName() { return name; }
    String getEmail() { return email; }
}

interface UserValidator {
    static boolean isValid(User user) {
        return user != null && user.getName() != null && user.getEmail() != null;
    }
    
    static ValidationResult validate(User user) {
        return ValidationResult.valid();
    }
}

class ValidationResult {
    static ValidationResult valid() { return new ValidationResult(); }
    static ValidationResult invalid(String msg) { return new ValidationResult(); }
}

class NameConverter {
    static String toCode(String name) { return name.toUpperCase(); }
}

class Order {
    private List<OrderItem> items;
    
    Order(List<OrderItem> items) {
        this.items = items;
    }
    
    BigDecimal calculateTotal() {
        return items.stream()
            .map(item -> item.price.multiply(BigDecimal.valueOf(item.quantity)))
            .reduce(BigDecimal.ZERO, BigDecimal::add);
    }
}

class OrderItem {
    String name;
    BigDecimal price;
    int quantity;
    
    OrderItem(String name, BigDecimal price, int quantity) {
        this.name = name;
        this.price = price;
        this.quantity = quantity;
    }
}

enum OrderStatus {
    PENDING, CONFIRMED, SHIPPED, DELIVERED;
    
    boolean isCancellable() {
        return this == PENDING || this == CONFIRMED;
    }
}
```

## 2. @RepeatedTest

### Running Tests Multiple Times

```java
/**
 * @RepeatedTest
 * 
 * Run test multiple times
 * Useful for flaky tests, concurrency, random data
 */

import org.junit.jupiter.api.*;

class RepeatedTestExamples {
    
    /**
     * Basic repeated test
     */
    @RepeatedTest(5)
    void testRepeatedly() {
        // Runs 5 times
        int random = (int) (Math.random() * 100);
        assertTrue(random >= 0 && random < 100);
    }
    
    /**
     * Custom display name
     */
    @RepeatedTest(value = 3, name = "Iteration {currentRepetition} of {totalRepetitions}")
    void testWithCustomName() {
        // Output:
        // Iteration 1 of 3 ✓
        // Iteration 2 of 3 ✓
        // Iteration 3 of 3 ✓
    }
    
    /**
     * Access repetition info
     */
    @RepeatedTest(10)
    void testWithRepetitionInfo(RepetitionInfo repetitionInfo) {
        int current = repetitionInfo.getCurrentRepetition();
        int total = repetitionInfo.getTotalRepetitions();
        
        System.out.println("Running " + current + " of " + total);
        assertTrue(current > 0 && current <= total);
    }
    
    /**
     * REAL-WORLD: Test thread safety
     */
    private final AtomicInteger counter = new AtomicInteger(0);
    
    @RepeatedTest(1000)
    void testThreadSafeIncrement() throws InterruptedException {
        // Spawn threads that increment counter
        Thread t1 = new Thread(() -> counter.incrementAndGet());
        Thread t2 = new Thread(() -> counter.incrementAndGet());
        
        t1.start();
        t2.start();
        t1.join();
        t2.join();
        
        // If not thread-safe, this would fail eventually
    }
    
    /**
     * REAL-WORLD: Test with random data
     */
    @RepeatedTest(100)
    void testPasswordValidation() {
        String password = generateRandomPassword();
        
        if (password.length() >= 8 && containsDigit(password)) {
            assertTrue(PasswordValidator.isValid(password));
        }
    }
    
    /**
     * REAL-WORLD: Stress test
     */
    @RepeatedTest(50)
    void stressTestCache() {
        Cache<String, String> cache = new LRUCache<>(100);
        
        // Add many entries
        for (int i = 0; i < 1000; i++) {
            cache.put("key" + i, "value" + i);
        }
        
        // Verify cache respects max size
        assertTrue(cache.size() <= 100);
    }
    
    /**
     * WHEN TO USE @RepeatedTest:
     * 
     * ✓ Flaky tests (intermittent failures)
     * ✓ Thread safety testing
     * ✓ Random data testing
     * ✓ Stress testing
     * ✓ Performance testing (with timing)
     * 
     * ✗ Don't use as "makes test more reliable"
     *   (Fix the flakiness instead!)
     */
    
    private String generateRandomPassword() {
        return "Password123";
    }
    
    private boolean containsDigit(String s) {
        return s.matches(".*\\d.*");
    }
}

interface PasswordValidator {
    static boolean isValid(String password) {
        return password.length() >= 8;
    }
}

interface Cache<K, V> {
    void put(K key, V value);
    V get(K key);
    int size();
}

class LRUCache<K, V> implements Cache<K, V> {
    LRUCache(int capacity) {}
    public void put(K key, V value) {}
    public V get(K key) { return null; }
    public int size() { return 0; }
}
```

## 3. @TestFactory (Dynamic Tests)

### Generating Tests at Runtime

```java
/**
 * @TestFactory
 * 
 * Create tests dynamically at runtime
 * Return DynamicTest, DynamicContainer, or Stream thereof
 */

import org.junit.jupiter.api.*;

class TestFactoryExamples {
    
    /**
     * Basic dynamic test
     */
    @TestFactory
    Stream<DynamicTest> dynamicTestsFromStream() {
        return Stream.of("apple", "banana", "orange")
            .map(fruit -> DynamicTest.dynamicTest(
                "Test: " + fruit,
                () -> assertTrue(fruit.length() > 3)
            ));
    }
    
    /**
     * Dynamic tests from collection
     */
    @TestFactory
    Collection<DynamicTest> dynamicTestsFromCollection() {
        return Arrays.asList(
            DynamicTest.dynamicTest("2 + 2 = 4",
                () -> assertEquals(4, Calculator.add(2, 2))),
            
            DynamicTest.dynamicTest("5 + 5 = 10",
                () -> assertEquals(10, Calculator.add(5, 5))),
            
            DynamicTest.dynamicTest("10 + 10 = 20",
                () -> assertEquals(20, Calculator.add(10, 10)))
        );
    }
    
    /**
     * REAL-WORLD: Test data from external source
     */
    @TestFactory
    Stream<DynamicTest> testEmailsFromDatabase() {
        // Imagine loading from database
        List<String> emails = loadEmailsFromDatabase();
        
        return emails.stream()
            .map(email -> DynamicTest.dynamicTest(
                "Testing email: " + email,
                () -> assertTrue(EmailValidator.isValid(email))
            ));
    }
    
    /**
     * REAL-WORLD: Test files in directory
     */
    @TestFactory
    Stream<DynamicTest> testConfigurationFiles() throws IOException {
        Path configDir = Paths.get("config");
        
        return Files.list(configDir)
            .filter(path -> path.toString().endsWith(".json"))
            .map(path -> DynamicTest.dynamicTest(
                "Validate " + path.getFileName(),
                () -> {
                    String json = Files.readString(path);
                    assertTrue(isValidJson(json), 
                              "Invalid JSON in " + path.getFileName());
                }
            ));
    }
    
    /**
     * REAL-WORLD: Generate tests from test data class
     */
    @TestFactory
    Stream<DynamicTest> testCalculatorOperations() {
        record TestCase(int a, int b, String operation, int expected) {}
        
        List<TestCase> testCases = Arrays.asList(
            new TestCase(2, 3, "add", 5),
            new TestCase(10, 5, "subtract", 5),
            new TestCase(4, 3, "multiply", 12),
            new TestCase(10, 2, "divide", 5)
        );
        
        return testCases.stream()
            .map(tc -> DynamicTest.dynamicTest(
                tc.a + " " + tc.operation + " " + tc.b + " = " + tc.expected,
                () -> {
                    int actual = switch (tc.operation) {
                        case "add" -> tc.a + tc.b;
                        case "subtract" -> tc.a - tc.b;
                        case "multiply" -> tc.a * tc.b;
                        case "divide" -> tc.a / tc.b;
                        default -> throw new IllegalArgumentException();
                    };
                    assertEquals(tc.expected, actual);
                }
            ));
    }
    
    /**
     * DYNAMIC CONTAINER: Nested dynamic tests
     */
    @TestFactory
    Stream<DynamicContainer> dynamicContainersFromStream() {
        return Stream.of("apple", "banana")
            .map(fruit -> DynamicContainer.dynamicContainer(
                "Tests for " + fruit,
                Stream.of(
                    DynamicTest.dynamicTest("Length test",
                        () -> assertTrue(fruit.length() > 3)),
                    DynamicTest.dynamicTest("Lowercase test",
                        () -> assertEquals(fruit, fruit.toLowerCase()))
                )
            ));
    }
    
    /**
     * WHEN TO USE @TestFactory:
     * 
     * ✓ Test data from external source (database, file, API)
     * ✓ Tests generated from model/configuration
     * ✓ Unknown number of tests at compile time
     * ✓ Complex test case generation logic
     * 
     * ✗ Don't use if @ParameterizedTest works
     *   (@ParameterizedTest is simpler for most cases)
     */
    
    private List<String> loadEmailsFromDatabase() {
        return Arrays.asList("user1@example.com", "user2@example.com");
    }
    
    private boolean isValidJson(String json) {
        return json.startsWith("{") && json.endsWith("}");
    }
}
```

## 4. @Nested Tests

### Organizing Related Tests

```java
/**
 * @Nested
 * 
 * Organize related tests hierarchically
 * Each @Nested class can have its own lifecycle
 */

@DisplayName("User Service Tests")
class UserServiceTests {
    
    private UserRepository repository;
    private EmailService emailService;
    private UserService userService;
    
    @BeforeEach
    void setUp() {
        repository = new InMemoryUserRepository();
        emailService = mock(EmailService.class);
        userService = new UserService(repository, emailService);
    }
    
    /**
     * NESTED: Registration tests
     */
    @Nested
    @DisplayName("User Registration")
    class RegistrationTests {
        
        @Test
        @DisplayName("should register valid user")
        void shouldRegisterValidUser() {
            User user = new User("john@example.com", "John", "Doe");
            
            userService.register(user);
            
            assertTrue(repository.findByEmail("john@example.com").isPresent());
            verify(emailService).sendWelcomeEmail("john@example.com");
        }
        
        @Test
        @DisplayName("should reject duplicate email")
        void shouldRejectDuplicateEmail() {
            User user1 = new User("john@example.com", "John", "Doe");
            userService.register(user1);
            
            User user2 = new User("john@example.com", "Jane", "Smith");
            
            assertThrows(DuplicateUserException.class, 
                        () -> userService.register(user2));
        }
        
        @Nested
        @DisplayName("Email Validation")
        class EmailValidationTests {
            
            @Test
            @DisplayName("should reject null email")
            void shouldRejectNullEmail() {
                User user = new User(null, "John", "Doe");
                
                assertThrows(IllegalArgumentException.class,
                           () -> userService.register(user));
            }
            
            @Test
            @DisplayName("should reject invalid email format")
            void shouldRejectInvalidEmail() {
                User user = new User("invalid-email", "John", "Doe");
                
                assertThrows(IllegalArgumentException.class,
                           () -> userService.register(user));
            }
        }
    }
    
    /**
     * NESTED: Authentication tests
     */
    @Nested
    @DisplayName("User Authentication")
    class AuthenticationTests {
        
        private User existingUser;
        
        @BeforeEach
        void setUpAuth() {
            // Additional setup for auth tests
            existingUser = new User("existing@example.com", "Existing", "User");
            repository.save(existingUser);
        }
        
        @Test
        @DisplayName("should authenticate with correct credentials")
        void shouldAuthenticateCorrectCredentials() {
            boolean result = userService.authenticate(
                "existing@example.com", 
                "password123"
            );
            
            assertTrue(result);
        }
        
        @Test
        @DisplayName("should reject wrong password")
        void shouldRejectWrongPassword() {
            boolean result = userService.authenticate(
                "existing@example.com", 
                "wrong-password"
            );
            
            assertFalse(result);
        }
        
        @Test
        @DisplayName("should reject non-existent user")
        void shouldRejectNonExistentUser() {
            boolean result = userService.authenticate(
                "nonexistent@example.com", 
                "password123"
            );
            
            assertFalse(result);
        }
    }
    
    /**
     * NESTED: Profile update tests
     */
    @Nested
    @DisplayName("Profile Updates")
    class ProfileUpdateTests {
        
        private User user;
        
        @BeforeEach
        void setUpProfile() {
            user = new User("user@example.com", "User", "Name");
            repository.save(user);
        }
        
        @Test
        @DisplayName("should update user name")
        void shouldUpdateName() {
            userService.updateName("user@example.com", "New Name");
            
            User updated = repository.findByEmail("user@example.com")
                .orElseThrow();
            assertEquals("New Name", updated.getFirstName());
        }
        
        @Test
        @DisplayName("should send notification on profile update")
        void shouldSendNotification() {
            userService.updateName("user@example.com", "New Name");
            
            verify(emailService).sendProfileUpdateNotification(
                eq("user@example.com")
            );
        }
    }
    
    /**
     * BENEFITS OF @Nested:
     * 
     * ✓ Logical organization (related tests grouped)
     * ✓ Hierarchical structure (easier to navigate)
     * ✓ Shared setup at each level (@BeforeEach)
     * ✓ Clear test output (nested display names)
     * ✓ Better IDE support (collapsible tree)
     */
    
    /**
     * OUTPUT STRUCTURE:
     * 
     * User Service Tests
     * ├─ User Registration
     * │  ├─ should register valid user ✓
     * │  ├─ should reject duplicate email ✓
     * │  └─ Email Validation
     * │     ├─ should reject null email ✓
     * │     └─ should reject invalid email format ✓
     * ├─ User Authentication
     * │  ├─ should authenticate with correct credentials ✓
     * │  ├─ should reject wrong password ✓
     * │  └─ should reject non-existent user ✓

     *    ├─ should update user name ✓
     *    └─ should send notification on profile update ✓

class InMemoryUserRepository implements UserRepository {
    private final Map<String, User> users = new HashMap<>();
    
    @Override
    public void save(User user) {
        users.put(user.getEmail(), user);
    }
    
    @Override
    public Optional<User> findByEmail(String email) {
        return Optional.ofNullable(users.get(email));
    }
    
    @Override
    public List<User> findAll() {
        return new ArrayList<>(users.values());
    }
}

class UserService {
    private final UserRepository repository;
    private final EmailService emailService;
    
    UserService(UserRepository repository, EmailService emailService) {
        this.repository = repository;
        this.emailService = emailService;
    }
    
    void register(User user) {
        if (user.getEmail() == null || !user.getEmail().contains("@")) {
            throw new IllegalArgumentException("Invalid email");
        }
        if (repository.findByEmail(user.getEmail()).isPresent()) {
            throw new DuplicateUserException("User exists");
        }
        repository.save(user);
        emailService.sendWelcomeEmail(user.getEmail());
    }
    
    boolean authenticate(String email, String password) {
        return repository.findByEmail(email).isPresent();
    }
    
    void updateName(String email, String newName) {
        User user = repository.findByEmail(email).orElseThrow();
        user.setFirstName(newName);
        repository.save(user);
        emailService.sendProfileUpdateNotification(email);
    }
}

interface EmailService {
    void sendWelcomeEmail(String email);
    void sendProfileUpdateNotification(String email);
}

    DuplicateUserException(String msg) { super(msg); }
```

```java
/**
 * TEST LIFECYCLE

class TestLifecycleExamples {
    
    /**
     * LIFECYCLE ORDER:
     * 
     * @BeforeAll      (once before all tests)
     * ├─ Test 1
     * │  ├─ @BeforeEach
     * │  ├─ @Test
     * │  └─ @AfterEach
     * ├─ Test 2
     * │  ├─ @BeforeEach
     * │  ├─ @Test
     * │  └─ @AfterEach
     * └─ Test 3
     *    ├─ @BeforeEach
     *    ├─ @Test
     *    └─ @AfterEach
     * @AfterAll       (once after all tests)
     */
    
    private static DatabaseConnection connection;
    private UserRepository repository;
    
    /**
     * Runs ONCE before all tests (must be static)
     */
    @BeforeAll
    static void setUpClass() {
        System.out.println("@BeforeAll - Setting up database connection");
        connection = new DatabaseConnection("test-db");
    }
    
    /**
     * Runs BEFORE EACH test
     */
    @BeforeEach
    void setUp() {
        System.out.println("  @BeforeEach - Creating repository");
        repository = new UserRepository(connection);
        repository.clearAll(); // Clean state
    }
    
    /**
     * Runs AFTER EACH test
     */
    @AfterEach
    void tearDown() {
        System.out.println("  @AfterEach - Cleaning up");
        // Clean up resources
    }
    
    /**
     * Runs ONCE after all tests (must be static)
     */
    @AfterAll
    static void tearDownClass() {
        System.out.println("@AfterAll - Closing database connection");
        if (connection != null) {
            connection.close();
        }
    }
    
    @Test
    void testOne() {
        System.out.println("    Test 1");
        assertTrue(true);
    }
    
    @Test
    void testTwo() {
        System.out.println("    Test 2");
        assertTrue(true);
    }
    
    /**
     * OUTPUT:
     * 
     * @BeforeAll - Setting up database connection
     *   @BeforeEach - Creating repository
     *     Test 1
     *   @AfterEach - Cleaning up
     *   @BeforeEach - Creating repository
     *     Test 2
     *   @AfterEach - Cleaning up
     * @AfterAll - Closing database connection
     */
}

    private static EmbeddedDatabase database;

    private static List<User> sharedUsers = new ArrayList<>();
```

```java
class TestOrderingExamples {
    
    /**
     * ALPHABETICAL ORDER
     */
    @TestMethodOrder(MethodOrderer.MethodName.class)
    class AlphabeticalTests {
        @Test
        void test_A() { System.out.println("A"); }
        
        @Test
        void test_B() { System.out.println("B"); }
        
        @Test
        void test_C() { System.out.println("C"); }
        
        // Output: A, B, C
    }
    
    /**
     * EXPLICIT ORDER
     */
    @TestMethodOrder(MethodOrderer.OrderAnnotation.class)
    class OrderedTests {
        
        @Test
        @Order(1)
        void firstTest() {
            System.out.println("First");
        }
        
        @Test
        @Order(2)
        void secondTest() {
            System.out.println("Second");
        }
        
        @Test
        @Order(3)
        void thirdTest() {
            System.out.println("Third");
        }
        
        // Output: First, Second, Third
    }
    
    /**
     * RANDOM ORDER (good for finding dependencies!)
     */
    @TestMethodOrder(MethodOrderer.Random.class)
    class RandomTests {
        @Test
        void test1() { System.out.println("1"); }
        
        @Test
        void test2() { System.out.println("2"); }
        
        @Test
        void test3() { System.out.println("3"); }
        
        // Output: Random order each run
    }
    
    /**
     * WHEN TO USE ORDERING:
     * 
     * ✓ Integration tests with dependencies
     *   (e.g., create → update → delete)
     * ✓ Database migration tests
     * ✓ Workflow tests (step 1, 2, 3)
     * 
     * ✗ Unit tests (should be independent!)
     * ✗ As a workaround for test dependencies
     *   (Fix the dependency instead!)
     */
    
    /**
     * EXAMPLE: Database integration test
     */
    @TestMethodOrder(MethodOrderer.OrderAnnotation.class)
    class DatabaseIntegrationTest {
        
        private static String userId;
        
        @Test
        @Order(1)
        void shouldCreateUser() {
            User user = new User("test@example.com", "Test", "User");
            userId = database.insert(user);
            assertNotNull(userId);
        }
        
        @Test
        @Order(2)
        void shouldFindUser() {
            User found = database.findById(userId);
            assertNotNull(found);
            assertEquals("test@example.com", found.getEmail());
        }
        
        @Test
        @Order(3)
        void shouldUpdateUser() {
            database.updateEmail(userId, "new@example.com");
            User updated = database.findById(userId);
            assertEquals("new@example.com", updated.getEmail());
        }
        
        @Test
        @Order(4)
        void shouldDeleteUser() {
            database.delete(userId);
            User deleted = database.findById(userId);
            assertNull(deleted);
        }
    }
}
```

**Key Takeaways:**

```
The Modern Testing Toolkit:

@ParameterizedTest → Multiple inputs, one test
  Sources: @ValueSource, @CsvSource, @MethodSource, @EnumSource

@Nested → Logical organization
  Shared setup at each level

Lifecycle → Setup/teardown
  @BeforeAll (once), @BeforeEach (each)

Conditional → Platform-specific
  @EnabledOnOs, @EnabledOnJre, @EnabledIf

The Golden Rule:
Tests should be independent!
Use ordering only when absolutely necessary.
```
