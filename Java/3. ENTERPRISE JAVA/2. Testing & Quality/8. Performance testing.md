# Performance Testing

## Overview

Performance testing measures how fast code runs under various conditions—it answers "how long does this take?" and "can it handle the load?". Unlike functional testing (correctness), performance testing measures speed, throughput, latency, resource usage, and scalability. This guide covers five essential tools and techniques: JMH (Java Microbenchmark Harness) for microbenchmarks (measuring individual method performance—nanoseconds to milliseconds, eliminates JVM warmup effects, provides statistical analysis), JMeter for load testing (simulating many users, HTTP/HTTPS requests, database queries, measures throughput and response times), Gatling for complex scenarios (Scala-based DSL, realistic user simulations, distributed testing, beautiful reports), Profiling in Tests (finding bottlenecks, memory leaks, CPU hotspots using tools like JProfiler/YourKit/VisualVM), and CI Performance Regression Detection (automated performance tests in pipeline, comparing against baselines, failing builds on regressions). The key insight: performance testing complements functional testing—functional tests verify behavior, performance tests verify speed/scalability. Different tools for different scales: JMH for method-level (microseconds), JMeter/Gatling for system-level (concurrent users), profilers for investigation (where is time spent?). Understanding performance testing means understanding measurement challenges: JVM warmup (JIT compilation makes early runs slower), garbage collection (pauses affect timing), system noise (other processes interfere), measurement overhead (timing code affects results). JMH solves microbenchmark challenges through warmup iterations, fork isolation, blackholes (preventing dead code elimination), state management. The workflow: identify bottleneck → write benchmark → optimize → measure improvement → repeat. Common mistakes: premature optimization (optimizing before measuring), ignoring JVM warmup, testing in unrealistic conditions, not having baseline measurements, over-optimizing microbenchmarks while missing system bottlenecks. This guide shows when performance matters (high-throughput APIs, real-time systems, batch processing, mobile apps with limited resources) and when to defer it (early development, rarely-used features, already fast enough). The relationship to other practices: performance testing finds slow code, profiling finds why it's slow, optimization makes it faster, regression detection maintains speed over time.

## 1. JMH (Java Microbenchmark Harness)

### Microbenchmarking Individual Methods

```java
/**
 * JMH: Java Microbenchmark Harness
 * 
 * Measures performance of small code units (methods)
 * Handles JVM warmup, JIT compilation, GC
 */

import org.openjdk.jmh.annotations.*;
import org.openjdk.jmh.runner.*;
import org.openjdk.jmh.runner.options.*;

import java.util.*;
import java.util.concurrent.TimeUnit;

// ========================================
// BASIC JMH BENCHMARK
// ========================================

@BenchmarkMode(Mode.AverageTime)  // Measure average time per operation
@OutputTimeUnit(TimeUnit.NANOSECONDS)  // Report in nanoseconds
@State(Scope.Benchmark)  // Shared state across all threads
@Fork(1)  // Run in 1 forked JVM
@Warmup(iterations = 3, time = 1)  // 3 warmup iterations, 1 second each
@Measurement(iterations = 5, time = 1)  // 5 measurement iterations, 1 second each
public class StringConcatenationBenchmark {
    
    private static final int SIZE = 1000;
    
    /**
     * Benchmark: String concatenation with +
     */
    @Benchmark
    public String stringConcatWithPlus() {
        String result = "";
        for (int i = 0; i < SIZE; i++) {
            result = result + i;  // Creates new String each time
        }
        return result;
    }
    
    /**
     * Benchmark: StringBuilder (mutable)
     */
    @Benchmark
    public String stringConcatWithBuilder() {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < SIZE; i++) {
            sb.append(i);  // Modifies existing buffer
        }
        return sb.toString();
    }
    
    /**
     * Run benchmarks
     */
    public static void main(String[] args) throws Exception {
        Options opt = new OptionsBuilder()
            .include(StringConcatenationBenchmark.class.getSimpleName())
            .build();
        
        new Runner(opt).run();
        
        /**
         * SAMPLE OUTPUT:
         * 
         * Benchmark                                    Mode  Cnt      Score      Error  Units
         * stringConcatWithPlus                         avgt    5  25435.123 ± 1234.567  ns/op
         * stringConcatWithBuilder                      avgt    5    723.456 ±   45.678  ns/op
         * 
         * Result: StringBuilder is ~35x faster!
         */
    }
}

// ========================================
// BENCHMARK MODES
// ========================================

@State(Scope.Benchmark)
public class BenchmarkModes {
    
    private List<Integer> list = new ArrayList<>();
    
    @Setup
    public void setup() {
        for (int i = 0; i < 10000; i++) {
            list.add(i);
        }
    }
    
    /**
     * Mode.AverageTime: Average time per operation
     * Best for: Comparing implementations
     */
    @Benchmark
    @BenchmarkMode(Mode.AverageTime)
    @OutputTimeUnit(TimeUnit.MICROSECONDS)
    public int averageTime() {
        return list.stream().mapToInt(Integer::intValue).sum();
    }
    
    /**
     * Mode.Throughput: Operations per second
     * Best for: Measuring system capacity
     */
    @Benchmark
    @BenchmarkMode(Mode.Throughput)
    @OutputTimeUnit(TimeUnit.SECONDS)
    public int throughput() {
        return list.stream().mapToInt(Integer::intValue).sum();
    }
    
    /**
     * Mode.SampleTime: Distribution of times
     * Best for: Understanding variance (p50, p99, p999)
     */
    @Benchmark
    @BenchmarkMode(Mode.SampleTime)
    @OutputTimeUnit(TimeUnit.MICROSECONDS)
    public int sampleTime() {
        return list.stream().mapToInt(Integer::intValue).sum();
    }
    
    /**
     * Mode.SingleShotTime: Single execution (cold start)
     * Best for: Startup time, initialization
     */
    @Benchmark
    @BenchmarkMode(Mode.SingleShotTime)
    @OutputTimeUnit(TimeUnit.MILLISECONDS)
    @Warmup(iterations = 0)  // No warmup for cold start
    @Measurement(iterations = 1)
    public int singleShot() {
        return list.stream().mapToInt(Integer::intValue).sum();
    }
}

/**
 * STATE SCOPES:
 * - Scope.Benchmark: Shared across all threads
 * - Scope.Thread: One instance per thread
 * - Scope.Group: One instance per thread group
 */

@State(Scope.Benchmark)
public class SharedState {
    private List<String> sharedList = new ArrayList<>();
    
    @Setup
    public void setup() {
        for (int i = 0; i < 1000; i++) {
            sharedList.add("Item" + i);
        }
    }
}

@State(Scope.Thread)
public class ThreadLocalState {
    private Random random = new Random();
    
    @Setup
    public void setup() {
        // Each thread gets its own Random instance
        random = new Random();
    }
}

@BenchmarkMode(Mode.Throughput)
@OutputTimeUnit(TimeUnit.SECONDS)
public class StateExample {
    
    @Benchmark
    public int withSharedState(SharedState state) {
        // All threads share same list
        return state.sharedList.size();
    }
    
    @Benchmark
    public int withThreadState(ThreadLocalState state) {
        // Each thread has own Random
        return state.random.nextInt(100);
    }
}

// ========================================
// AVOIDING DEAD CODE ELIMINATION
// ========================================

@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.NANOSECONDS)
@State(Scope.Benchmark)
public class DeadCodeElimination {
    
    /**
     * BAD: JVM may eliminate dead code
     */
    @Benchmark
    public void bad() {
        int result = compute();  // JVM sees result is unused
        // JVM may optimize away entire computation!
    }
    
    /**
     * GOOD: Return value (JMH uses it)
     */
    @Benchmark
    public int good() {
        return compute();  // JMH consumes return value
    }
    
    /**
     * GOOD: Use Blackhole
     */
    @Benchmark
    public void withBlackhole(Blackhole blackhole) {
        int result = compute();
        blackhole.consume(result);  // Prevents elimination
    }
    
    private int compute() {
        int sum = 0;
        for (int i = 0; i < 100; i++) {
            sum += i;
        }
        return sum;
    }
}

// ========================================
// REAL-WORLD EXAMPLE: COMPARING ALGORITHMS
// ========================================

@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MICROSECONDS)
@State(Scope.Benchmark)
@Fork(1)
@Warmup(iterations = 3, time = 1)
@Measurement(iterations = 5, time = 1)
public class AlgorithmComparison {
    
    @Param({"100", "1000", "10000"})
    private int size;
    
    private List<Integer> data;
    
    @Setup
    public void setup() {
        data = new ArrayList<>();
        Random random = new Random(42);
        for (int i = 0; i < size; i++) {
            data.add(random.nextInt(1000));
        }
    }
    
    /**
     * Linear search
     */
    @Benchmark
    public int linearSearch() {
        int target = 500;
        for (int i = 0; i < data.size(); i++) {
            if (data.get(i) == target) {
                return i;
            }
        }
        return -1;
    }
    
    /**
     * Binary search (requires sorted data)
     */
    @Benchmark
    public int binarySearch() {
        List<Integer> sorted = new ArrayList<>(data);
        Collections.sort(sorted);
        return Collections.binarySearch(sorted, 500);
    }
    
    /**
     * RESULTS (example):
     * 
     * Benchmark              (size)  Mode  Cnt    Score    Error  Units
     * linearSearch              100  avgt    5    0.456 ±  0.012  us/op
     * linearSearch             1000  avgt    5    4.123 ±  0.234  us/op
     * linearSearch            10000  avgt    5   42.567 ±  2.345  us/op
     * binarySearch              100  avgt    5    1.234 ±  0.056  us/op
     * binarySearch             1000  avgt    5    2.456 ±  0.123  us/op
     * binarySearch            10000  avgt    5    3.789 ±  0.234  us/op
     * 
     * Linear: O(n) - time grows linearly
     * Binary: O(log n) - time grows logarithmically
     */
}

// ========================================
// JMH BEST PRACTICES
// ========================================

/**
 * BEST PRACTICES:
 * 
 * 1. ALWAYS use @Fork(1) or higher
 *    - Isolates benchmarks in separate JVM
 *    - Prevents interference
 * 
 * 2. ALWAYS use warmup iterations
 *    - JIT compilation needs time
 *    - First runs are slow
 * 
 * 3. Return or consume results
 *    - Use Blackhole.consume()
 *    - Or return value
 *    - Prevents dead code elimination
 * 
 * 4. Use @Param for different inputs
 *    - Test with various sizes
 *    - See how performance scales
 * 
 * 5. Measure what matters
 *    - Don't optimize microseconds if system bottleneck is elsewhere
 *    - Profile first, optimize second
 * 
 * 6. Run multiple times
 *    - Performance varies
 *    - Look at error margins
 * 
 * 7. Test on target hardware
 *    - Different CPUs perform differently
 *    - Run on production-like environment
 */
```

```java
// ========================================

import org.apache.jmeter.engine.StandardJMeterEngine;
import org.apache.jmeter.protocol.http.sampler.HTTPSamplerProxy;
import org.apache.jmeter.threads.ThreadGroup;
import org.apache.jmeter.testelement.TestPlan;
import org.apache.jorphan.collections.HashTree;

public class JMeterProgrammatic {
    
    public void runLoadTest() {
        // Create JMeter engine
        StandardJMeterEngine jmeter = new StandardJMeterEngine();
        
        // Create test plan
        TestPlan testPlan = new TestPlan("API Load Test");
        
        // Create thread group (users)
        ThreadGroup threadGroup = new ThreadGroup();
        threadGroup.setNumThreads(100);     // 100 concurrent users
        threadGroup.setRampUp(10);          // Ramp up over 10 seconds
        threadGroup.setLoops(10);           // 10 requests per user
        
        // Create HTTP sampler
        HTTPSamplerProxy sampler = new HTTPSamplerProxy();
        sampler.setDomain("localhost");
        sampler.setPort(8080);
        sampler.setPath("/api/users");
        sampler.setMethod("GET");
        
        // Build test hierarchy
        HashTree testPlanTree = new HashTree();
        testPlanTree.add(testPlan);
        
        HashTree threadGroupTree = testPlanTree.add(testPlan, threadGroup);
        threadGroupTree.add(sampler);
        
        // Run test
        jmeter.configure(testPlanTree);
        jmeter.run();
    }
}

// ========================================
```

```scala
// ========================================
// BASIC GATLING SIMULATION
// ========================================

import io.gatling.core.Predef._
import io.gatling.http.Predef._
import scala.concurrent.duration._

class BasicSimulation extends Simulation {
  
  // HTTP protocol configuration
  val httpProtocol = http
    .baseUrl("http://localhost:8080")
    .acceptHeader("application/json")
    .userAgentHeader("Gatling")
  
  // Scenario: User browsing products
  val scn = scenario("Browse Products")
    .exec(
      http("Homepage")
        .get("/")
        .check(status.is(200))
    )
    .pause(1.second, 2.seconds)  // Think time
    .exec(
      http("Product List")
        .get("/api/products")
        .check(status.is(200))
        .check(jsonPath("$[*].id").findAll.saveAs("productIds"))
    )
    .pause(2.seconds)
    .exec(
      http("Product Detail")
        .get("/api/products/${productIds.random()}")
        .check(status.is(200))
    )
  
  // Load injection: 100 users over 10 seconds
  setUp(
    scn.inject(
      rampUsers(100) during (10.seconds)
    )
  ).protocols(httpProtocol)
}
```

## 4. Profiling in Tests

### Finding Performance Bottlenecks

```java
/**
 * PROFILING: Finding WHERE time is spent
 * 
 * Benchmarking shows HOW SLOW
 * Profiling shows WHY SLOW
 */

// ========================================
// CPU PROFILING
// ========================================

/**
 * CPU profilers show:
 * - Which methods consume most CPU time
 * - Call hierarchy (stack traces)
 * - Hot spots
 * 
 * Tools:
 * - JProfiler (commercial)
 * - YourKit (commercial)
 * - VisualVM (free)
 * - Java Flight Recorder (built-in)
 * - Async-profiler (free, production-ready)
 */

// Example: Method consuming CPU
public class SlowProcessor {
    
    public void process(List<String> items) {
        for (String item : items) {
            // Profiler will show this method is hot spot
            String processed = slowTransform(item);
            save(processed);
        }
    }
    
    private String slowTransform(String item) {
        // Inefficient string operations
        String result = "";
        for (int i = 0; i < item.length(); i++) {
            result = result + item.charAt(i) + "-";  // Bad!
        }
        return result;
    }
    
    /**
     * PROFILER OUTPUT:
     * 
     * Method                        Time    %
     * ------------------------------------ 
     * slowTransform                 2.5s   89%  ← HOT SPOT!
     * process                       2.8s   100%
     * save                          0.3s   11%
     * 
     * Action: Optimize slowTransform
     */
    
    private String optimizedTransform(String item) {
        StringBuilder sb = new StringBuilder(item.length() * 2);
        for (int i = 0; i < item.length(); i++) {
            sb.append(item.charAt(i)).append('-');
        }
        return sb.toString();
    }
    
    private void save(String s) {
        // Database save
    }
}

/**
 * Memory profilers show:
 * - Object allocation
 * - Memory leaks
 * - GC behavior
 * - Heap dumps

public class MemoryLeakExample {
    
    // Memory leak: never cleared
    private static List<byte[]> cache = new ArrayList<>();
    
    public void processData() {
        // Allocates 1MB each time
        byte[] data = new byte[1024 * 1024];
        
        // Leak: adds to static list, never removed
        cache.add(data);
        
        /**
         * PROFILER SHOWS:
         * 
         * Objects                    Count    Size
         * ----------------------------------------
         * byte[]                     1000     1000MB  ← LEAK!
         * ArrayList                  1        32KB
```

```java
public class BaselineComparison {

     * Store baseline measurements

    private static final Map<String, Double> BASELINES = Map.of(
        "stringConcat", 50.0,      // microseconds
        "jsonParsing", 200.0,

    void shouldNotRegress() throws Exception {
        // Run benchmarks
        Map<String, Double> current = runBenchmarks();
        
        // Compare against baseline
        for (Map.Entry<String, Double> entry : current.entrySet()) {
            String operation = entry.getKey();
            double currentTime = entry.getValue();
            double baseline = BASELINES.get(operation);

                fail(String.format(
                    "Performance regression in %s: %.2fμs > %.2fμs (baseline: %.2fμs)",
                    operation, currentTime, threshold, baseline

        }
```

**Key Takeaways:**

```
The Tools:
JMH → Microbenchmarks (method-level, μs)
JMeter → Load testing (API-level, concurrent users)
Gatling → Complex scenarios (realistic workflows)
Profilers → Investigation (find bottlenecks)
CI Tests → Regression detection (automation)

When to Use:
"Which algorithm is faster?" → JMH
"Can API handle 1000 users?" → JMeter/Gatling
"Why is this slow?" → Profiler
"Prevent slowdowns?" → CI regression tests

JMH Pattern:
@Benchmark
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MICROSECONDS)
@Warmup(iterations = 3)
@Measurement(iterations = 5)

Performance Workflow:
1. Measure (baseline)
2. Profile (find hot spots)
3. Optimize (improve code)
4. Verify (measure again)
5. Automate (CI regression tests)

Critical Metrics:
Latency: p50, p95, p99
Throughput: req/sec
Resources: CPU, memory
Errors: error rate
```
