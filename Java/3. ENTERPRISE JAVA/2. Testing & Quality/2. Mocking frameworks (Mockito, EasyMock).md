# Mocking Frameworks (Mockito & EasyMock)

## Overview

Mocking frameworks create test doubles—fake objects that replace real dependencies in tests. Mockito is the de facto standard for Java mocking (simple, powerful, widely used), while EasyMock is an older alternative (record-replay model, less popular today). This guide focuses on Mockito with EasyMock comparisons. Mocking enables testing in isolation—test one class without its dependencies (database, network, external services). The key insight: mocking is about isolation, not avoiding real code. Mock external dependencies (slow, unreliable), not your own domain logic. There are three test double types: Mock (verifies interactions—"was method called?"), Spy (partial mock—real object with some methods stubbed), Stub (returns canned data—"what should method return?"). Understanding the difference prevents over-mocking and brittle tests. Mockito provides powerful features: @Mock creates mocks, @InjectMocks injects them automatically, when().thenReturn() stubs behavior, argument matchers (any(), eq()) match parameters flexibly, verify() checks interactions, MockedStatic mocks static methods (Mockito 3.4+). The danger: over-mocking leads to tests that verify implementation details instead of behavior—brittle tests that break on refactoring. Best practices: mock at architectural boundaries (repositories, external services), don't mock value objects (User, Order), prefer real objects when fast, test behavior not implementation. This guide shows when mocking helps (isolating business logic) and when it hurts (testing everything in isolation loses integration issues).

## 1. Mock vs Spy vs Stub

### Understanding Test Doubles

```java
/**
 * MOCK vs SPY vs STUB
 * 
 * Three types of test doubles with different purposes
 */

import org.mockito.*;
import static org.mockito.Mockito.*;
import org.junit.jupiter.api.*;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.junit.jupiter.MockitoExtension;

// ========================================
// PRODUCTION CODE
// ========================================

interface EmailService {
    void sendEmail(String to, String subject, String body);
    boolean isConnected();
}

interface UserRepository {
    void save(User user);
    Optional<User> findByEmail(String email);
}

class User {
    private final String email;
    private final String name;
    
    User(String email, String name) {
        this.email = email;
        this.name = name;
    }
    
    public String getEmail() { return email; }
    public String getName() { return name; }
}

class UserService {
    private final UserRepository repository;
    private final EmailService emailService;
    
    UserService(UserRepository repository, EmailService emailService) {
        this.repository = repository;
        this.emailService = emailService;
    }
    
    public void registerUser(String email, String name) {
        // Check if user exists
        if (repository.findByEmail(email).isPresent()) {
            throw new IllegalArgumentException("User already exists");
        }
        
        // Save user
        User user = new User(email, name);
        repository.save(user);
        
        // Send welcome email
        emailService.sendEmail(email, "Welcome!", "Welcome to our service");
    }
}

// ========================================
// 1. STUB: Returns pre-programmed responses
// ========================================

@ExtendWith(MockitoExtension.class)
class StubExample {
    
    /**
     * STUB provides canned answers
     * No behavior verification
     */
    @Test
    void testWithStub() {
        // Create stub using Mockito
        UserRepository stubRepo = mock(UserRepository.class);
        EmailService stubEmail = mock(EmailService.class);
        
        // Configure stub behavior
        when(stubRepo.findByEmail("test@example.com")).thenReturn(Optional.empty());
        when(stubEmail.isConnected()).thenReturn(true);
        
        // Use stubs
        UserService service = new UserService(stubRepo, stubEmail);
        service.registerUser("test@example.com", "Test User");
        
        // ✓ Test passes (no verification of interactions)
    }
    
    /**
     * CHARACTERISTICS:
     * 
     * ✓ Provides return values
     * ✓ No interaction verification
     * ✓ Simple and focused
     * ✓ Used for queries (read operations)
     * 
     * WHEN TO USE:
     * - Need specific return values
     * - Don't care about interactions
     * - Testing query operations
     */
}

// ========================================
// 2. MOCK: Verifies interactions
// ========================================

@ExtendWith(MockitoExtension.class)
class MockExample {
    
    /**
     * MOCK verifies behavior was called
     */
    @Test
    void testWithMock() {
        // Create mocks
        UserRepository mockRepo = mock(UserRepository.class);
        EmailService mockEmail = mock(EmailService.class);
        
        // Stub findByEmail (still need this for test to work)
        when(mockRepo.findByEmail("test@example.com")).thenReturn(Optional.empty());
        
        // Execute
        UserService service = new UserService(mockRepo, mockEmail);
        service.registerUser("test@example.com", "Test User");
        
        // ✓ VERIFY: Check interactions happened
        verify(mockRepo).save(any(User.class));
        verify(mockEmail).sendEmail(
            eq("test@example.com"),
            eq("Welcome!"),
            eq("Welcome to our service")
        );
    }
    
    /**
     * CHARACTERISTICS:
     * 
     * ✓ Verifies method calls
     * ✓ Checks call arguments
     * ✓ Checks call count
     * ✓ Used for commands (write operations)
     * 
     * WHEN TO USE:
     * - Verify method was called
     * - Verify correct arguments
     * - Testing command operations
     * - Testing side effects
     */
}

// ========================================
// 3. SPY: Partial mock (real + verification)
// ========================================

@ExtendWith(MockitoExtension.class)
class SpyExample {
    
    /**
     * Real implementation to spy on
     */
    static class RealEmailService implements EmailService {
        private List<String> sentEmails = new ArrayList<>();
        
        @Override
        public void sendEmail(String to, String subject, String body) {
            sentEmails.add(to + ": " + subject);
            // Real implementation (no actual sending in test)
        }
        
        @Override
        public boolean isConnected() {
            return true;
        }
        
        public List<String> getSentEmails() {
            return sentEmails;
        }
    }
    
    @Test
    void testWithSpy() {
        UserRepository mockRepo = mock(UserRepository.class);
        when(mockRepo.findByEmail(any())).thenReturn(Optional.empty());
        
        // Create spy of real object
        RealEmailService emailService = new RealEmailService();
        EmailService spyEmail = spy(emailService);
        
        // Use spy
        UserService service = new UserService(mockRepo, spyEmail);
        service.registerUser("test@example.com", "Test User");
        
        // ✓ Can verify calls (like mock)
        verify(spyEmail).sendEmail(
            eq("test@example.com"),
            eq("Welcome!"),
            anyString()
        );
        
        // ✓ Can check real state
        assertEquals(1, emailService.getSentEmails().size());
    }
    
    /**
     * Override specific methods on spy
     */
    @Test
    void testSpyWithOverride() {
        RealEmailService emailService = new RealEmailService();
        EmailService spyEmail = spy(emailService);
        
        // Override isConnected() only
        when(spyEmail.isConnected()).thenReturn(false);
        
        // sendEmail() still uses real implementation
        spyEmail.sendEmail("test@example.com", "Test", "Body");
        
        // Verify override
        assertFalse(spyEmail.isConnected());
        
        // Verify real method was called
        assertEquals(1, emailService.getSentEmails().size());
    }
    
    /**
     * CHARACTERISTICS:
     * 
     * ✓ Wraps real object
     * ✓ Can override specific methods
     * ✓ Calls real methods by default
     * ✓ Can verify interactions
     * 
     * WHEN TO USE:
     * - Test mostly real object
     * - Override one method, keep others real
     * - Legacy code (hard to inject dependencies)
     * 
     * ⚠ WARNING: Usually a code smell
     * - Prefer full mocks or real objects
     * - Spies indicate poor design (too coupled)
     */
}

// ========================================
// COMPARISON TABLE
// ========================================

/**
 * ┌──────────────┬──────────────┬──────────────┬──────────────┐
 * │ Feature      │ Stub         │ Mock         │ Spy          │
 * ├──────────────┼──────────────┼──────────────┼──────────────┤
 * │ Returns data │ Yes          │ Yes          │ Yes (real)   │
 * │ Verifies     │ No           │ Yes          │ Yes          │
 * │ Real object  │ No           │ No           │ Yes          │
 * │ Override     │ All methods  │ All methods  │ Selective    │
 * ├──────────────┼──────────────┼──────────────┼──────────────┤
 * │ Use case     │ Query ops    │ Command ops  │ Partial mock │
 * │ Purpose      │ Provide data │ Verify calls │ Real + verify│
 * └──────────────┴──────────────┴──────────────┴──────────────┘
 */
```

```java
class InjectMocksStrategies {
    
    /**
     * STRATEGY 1: Constructor injection (preferred)
     */
    static class ServiceWithConstructor {
        private final UserRepository repository;
        
        ServiceWithConstructor(UserRepository repository) {
            this.repository = repository;
        }
    }
    
    /**
     * STRATEGY 2: Setter injection
     */
    static class ServiceWithSetter {
        private UserRepository repository;
        
        public void setRepository(UserRepository repository) {
            this.repository = repository;
        }
    }
    
    /**
     * STRATEGY 3: Field injection
     */
    static class ServiceWithField {
        @SuppressWarnings("unused")
        private UserRepository repository;
    }
    
    /**
     * PREFERENCE ORDER:
     * 
     * 1. Constructor (most explicit)
     * 2. Setter (flexible)
     * 3. Field (reflection, not ideal)
     */
}
```

## 3. when-thenReturn Stubbing

### Configuring Mock Behavior

```java
/**
 * STUBBING: when-thenReturn
 * 
 * Configure what mocks return
 */

@ExtendWith(MockitoExtension.class)
class StubbingExamples {
    
    @Mock
    private UserRepository repository;
    
    @Mock
    private EmailService emailService;
    
    // ========================================
    // BASIC STUBBING
    // ========================================
    
    @Test
    void basicStubbing() {
        // Simple return value
        when(repository.findByEmail("test@example.com"))
            .thenReturn(Optional.of(new User("test@example.com", "Test")));
        
        // Use stubbed method
        Optional<User> user = repository.findByEmail("test@example.com");
        assertTrue(user.isPresent());
        assertEquals("Test", user.get().getName());
    }
    
    // ========================================
    // MULTIPLE RETURNS
    // ========================================
    
    @Test
    void multipleReturns() {
        // Return different values on successive calls
        when(emailService.isConnected())
            .thenReturn(false)  // First call
            .thenReturn(true)   // Second call
            .thenReturn(true);  // Third+ calls
        
        assertFalse(emailService.isConnected());  // false
        assertTrue(emailService.isConnected());   // true
        assertTrue(emailService.isConnected());   // true
    }
    
    // ========================================
    // THROWING EXCEPTIONS
    // ========================================
    
    @Test
    void throwingExceptions() {
        // Throw exception when called
        when(repository.save(any(User.class)))
            .thenThrow(new DatabaseException("Connection failed"));
        
        assertThrows(DatabaseException.class, () -> {
            repository.save(new User("test@example.com", "Test"));
        });
    }
    
    @Test
    void throwingOnVoidMethod() {
        // For void methods, use doThrow
        doThrow(new NetworkException("Connection timeout"))
            .when(emailService).sendEmail(anyString(), anyString(), anyString());
        
        assertThrows(NetworkException.class, () -> {
            emailService.sendEmail("test@example.com", "Subject", "Body");
        });
    }
    
    // ========================================
    // ANSWER (CUSTOM LOGIC)
    // ========================================
    
    @Test
    void answerWithCustomLogic() {
        // Execute custom logic when called
        when(repository.findByEmail(anyString()))
            .thenAnswer(invocation -> {
                String email = invocation.getArgument(0);
                if (email.contains("admin")) {
                    return Optional.of(new User(email, "Admin"));
                }
                return Optional.empty();
            });
        
        assertTrue(repository.findByEmail("admin@example.com").isPresent());
        assertFalse(repository.findByEmail("user@example.com").isPresent());
    }
    
    // ========================================
    // CALL REAL METHOD
    // ========================================
    
    @Test
    void callRealMethod() {
        // For concrete class with partial mocking
        UserService service = mock(UserService.class);
        
        // Call real method
        when(service.registerUser(anyString(), anyString(), anyString()))
            .thenCallRealMethod();
        
        // Only useful with spy or partial mocks
    }
    
    // ========================================
    // CHAINING (FLUENT)
    // ========================================
    
    @Test
    void chainingStubs() {
        // Chain multiple configurations
        when(repository.findByEmail("test@example.com"))
            .thenReturn(Optional.of(new User("test@example.com", "Test")));
        
        when(repository.findByEmail("admin@example.com"))
            .thenReturn(Optional.of(new User("admin@example.com", "Admin")));
        
        // Both stubs active
        assertTrue(repository.findByEmail("test@example.com").isPresent());
        assertTrue(repository.findByEmail("admin@example.com").isPresent());
    }
    
    // ========================================
    // STUBBING VOID METHODS
    // ========================================
    
    @Test
    void stubbingVoidMethods() {
        // doNothing (default for void methods)
        doNothing().when(emailService).sendEmail(anyString(), anyString(), anyString());
        
        // doThrow
        doThrow(new RuntimeException()).when(emailService)
            .sendEmail(eq("bad@example.com"), anyString(), anyString());
        
        // doAnswer
        doAnswer(invocation -> {
            System.out.println("Email sent to: " + invocation.getArgument(0));
            return null;
        }).when(emailService).sendEmail(anyString(), anyString(), anyString());
    }
    
    // ========================================
    // CONSECUTIVE CALLS
    // ========================================
    
    @Test
    void consecutiveCalls() {
        // Alternative syntax for multiple returns
        when(emailService.isConnected())
            .thenReturn(false, true, true);
        
        assertFalse(emailService.isConnected());
        assertTrue(emailService.isConnected());
        assertTrue(emailService.isConnected());
    }
}

class DatabaseException extends RuntimeException {
    DatabaseException(String msg) { super(msg); }
}

class NetworkException extends RuntimeException {
    NetworkException(String msg) { super(msg); }
}
```

## 4. Argument Matchers

### Flexible Parameter Matching

```java
/**
 * ARGUMENT MATCHERS
 * 
 * Match parameters flexibly in stubs and verification
 */

import static org.mockito.ArgumentMatchers.*;

@ExtendWith(MockitoExtension.class)
class ArgumentMatchersExamples {
    
    @Mock
    private UserRepository repository;
    
    @Mock
    private EmailService emailService;
    
    // ========================================
    // COMMON MATCHERS
    // ========================================
    
    @Test
    void commonMatchers() {
        // any() - any object (including null)
        when(repository.findByEmail(any())).thenReturn(Optional.empty());
        
        // anyString() - any string (null not included)
        when(repository.findByEmail(anyString())).thenReturn(Optional.empty());
        
        // anyInt(), anyLong(), anyDouble(), etc.
        when(repository.findById(anyInt())).thenReturn(Optional.empty());
        
        // eq() - exact match
        when(repository.findByEmail(eq("test@example.com")))
            .thenReturn(Optional.of(new User("test@example.com", "Test")));
        
        // Use mocks
        repository.findByEmail("any-email@example.com");
        repository.findByEmail("test@example.com");
    }
    
    // ========================================
    // MIXING MATCHERS AND VALUES
    // ========================================
    
    @Test
    void mixingMatchersAndValues() {
        // ❌ WRONG: Can't mix matchers and raw values
        // when(emailService.sendEmail("test@example.com", anyString(), anyString()))
        
        // ✓ CORRECT: Use eq() for exact values when mixing
        when(emailService.sendEmail(eq("test@example.com"), anyString(), anyString()))
            .thenReturn(true);
        
        // Rule: If you use one matcher, use matchers for ALL arguments
    }
    
    // ========================================
    // COLLECTION MATCHERS
    // ========================================
    
    @Test
    void collectionMatchers() {
        // anyList(), anySet(), anyMap(), anyCollection()
        when(repository.saveAll(anyList())).thenReturn(3);
        
        // Verify with collection matchers
        List<User> users = Arrays.asList(
            new User("user1@example.com", "User1"),
            new User("user2@example.com", "User2")
        );
        repository.saveAll(users);
        
        verify(repository).saveAll(anyList());
    }
    
    // ========================================
    // NULL HANDLING
    // ========================================
    
    @Test
    void nullHandling() {
        // isNull() - matches null
        when(repository.findByEmail(isNull())).thenReturn(Optional.empty());
        
        // notNull() - matches non-null
        when(repository.findByEmail(notNull())).thenReturn(Optional.of(new User("", "")));
        
        // nullable() - matches null or non-null (same as any())
        when(repository.findByEmail(nullable(String.class))).thenReturn(Optional.empty());
    }
    
    // ========================================
    // STRING MATCHERS
    // ========================================
    
    @Test
    void stringMatchers() {
        // contains()
        when(repository.findByEmail(contains("@example.com")))
            .thenReturn(Optional.of(new User("user@example.com", "User")));
        
        // startsWith()
        when(repository.findByEmail(startsWith("admin")))
            .thenReturn(Optional.of(new User("admin@example.com", "Admin")));
        
        // endsWith()
        when(repository.findByEmail(endsWith(".com")))
            .thenReturn(Optional.of(new User("user@example.com", "User")));
        
        // matches() - regex
        when(repository.findByEmail(matches(".*@gmail\\.com")))
            .thenReturn(Optional.of(new User("user@gmail.com", "Gmail User")));
        
        assertTrue(repository.findByEmail("admin@example.com").isPresent());
        assertTrue(repository.findByEmail("user@gmail.com").isPresent());
    }
    
    // ========================================
    // CUSTOM MATCHERS
    // ========================================
    
    @Test
    void customMatchers() {
        // argThat() - custom predicate
        when(repository.findByEmail(argThat(email -> 
            email != null && email.length() > 5
        ))).thenReturn(Optional.of(new User("long@example.com", "User")));
        
        // Long email matches
        assertTrue(repository.findByEmail("longeremail@example.com").isPresent());
        
        // Short email doesn't match
        assertFalse(repository.findByEmail("a@b").isPresent());
    }
    
    @Test
    void customMatcherWithType() {
        // Match User objects with custom logic
        when(repository.save(argThat(user -> 
            user != null && user.getEmail().endsWith("@example.com")
        ))).thenReturn(new User("saved@example.com", "Saved"));
        
        User validUser = new User("user@example.com", "Valid");
        User savedUser = repository.save(validUser);
        assertNotNull(savedUser);
    }
    
    // ========================================
    // VERIFICATION WITH MATCHERS
    // ========================================
    
    @Test
    void verificationWithMatchers() {
        // Verify method called with any arguments
        emailService.sendEmail("test@example.com", "Subject", "Body");
        
        verify(emailService).sendEmail(anyString(), anyString(), anyString());
        
        // Verify with specific matcher
        verify(emailService).sendEmail(
            contains("@example.com"),
            eq("Subject"),
            anyString()
        );
    }
    
    // ========================================
    // CAPTURING ARGUMENTS
    // ========================================
    
    @Test
    void capturingArguments() {
        // Capture argument for later inspection
        ArgumentCaptor<User> userCaptor = ArgumentCaptor.forClass(User.class);
        
        repository.save(new User("test@example.com", "Test User"));
        
        verify(repository).save(userCaptor.capture());
        
        // Inspect captured value
        User capturedUser = userCaptor.getValue();
        assertEquals("test@example.com", capturedUser.getEmail());
        assertEquals("Test User", capturedUser.getName());
    }
    
    @Test
    void capturingMultipleArguments() {
        ArgumentCaptor<String> emailCaptor = ArgumentCaptor.forClass(String.class);
        
        // Multiple calls
        emailService.sendEmail("user1@example.com", "Subject1", "Body1");
        emailService.sendEmail("user2@example.com", "Subject2", "Body2");
        
        verify(emailService, times(2)).sendEmail(
            emailCaptor.capture(),
            anyString(),
            anyString()
        );
        
        // Get all captured values
        List<String> capturedEmails = emailCaptor.getAllValues();
        assertEquals(2, capturedEmails.size());
        assertEquals("user1@example.com", capturedEmails.get(0));
        assertEquals("user2@example.com", capturedEmails.get(1));
    }
}

interface UserRepository {
```

```java
/**
 * MOCKING STATIC METHODS
 * 
 * New in Mockito 3.4+
 * Requires mockito-inline dependency

import org.mockito.MockedStatic;

class StaticMockingExamples {
    
    /**
     * Class with static methods
     */
    static class DateUtils {
        public static LocalDate today() {
            return LocalDate.now();
        }
        
        public static boolean isWeekend(LocalDate date) {
            DayOfWeek day = date.getDayOfWeek();
            return day == DayOfWeek.SATURDAY || day == DayOfWeek.SUNDAY;
        }
    }
    
    // ========================================
    // MOCKING STATIC METHOD
    // ========================================
    
    @Test
    void mockStaticMethod() {
        // Mock static method
        try (MockedStatic<DateUtils> mockedStatic = mockStatic(DateUtils.class)) {
            // Stub static method
            mockedStatic.when(DateUtils::today).thenReturn(LocalDate.of(2024, 1, 1));
            
            // Use stubbed static method
            LocalDate date = DateUtils.today();
            assertEquals(LocalDate.of(2024, 1, 1), date);
            
            // Verify static method called
            mockedStatic.verify(DateUtils::today);
        }
        // Mock automatically closed after try block
    }
    
    // ========================================
    // MOCKING WITH ARGUMENTS
    // ========================================
    
    @Test
    void mockStaticMethodWithArguments() {
        try (MockedStatic<DateUtils> mockedStatic = mockStatic(DateUtils.class)) {
            // Stub with argument matching
            mockedStatic.when(() -> DateUtils.isWeekend(any(LocalDate.class)))
                       .thenReturn(true);
            
            // All dates are weekends!
            assertTrue(DateUtils.isWeekend(LocalDate.of(2024, 1, 15)));  // Monday
            assertTrue(DateUtils.isWeekend(LocalDate.of(2024, 1, 20)));  // Saturday
            
            // Verify
            mockedStatic.verify(() -> DateUtils.isWeekend(any(LocalDate.class)), times(2));
        }
    }
    
    // ========================================
    // PARTIAL MOCKING
    // ========================================
    
    @Test
    void partialStaticMocking() {
        try (MockedStatic<DateUtils> mockedStatic = mockStatic(DateUtils.class)) {
            // Mock only one method, call real for others
            mockedStatic.when(DateUtils::today).thenReturn(LocalDate.of(2024, 1, 6)); // Saturday
            
            // isWeekend() uses real implementation
            mockedStatic.when(() -> DateUtils.isWeekend(any(LocalDate.class)))
                       .thenCallRealMethod();
            
            LocalDate today = DateUtils.today();
            assertTrue(DateUtils.isWeekend(today));  // Real method says Saturday is weekend
        }
    }
    
    // ========================================
    // REAL-WORLD EXAMPLE: UUID
    // ========================================
    
    @Test
    void mockUUIDGeneration() {
        try (MockedStatic<UUID> mockedStatic = mockStatic(UUID.class)) {
            UUID fixedUuid = UUID.fromString("123e4567-e89b-12d3-a456-426614174000");
            
            mockedStatic.when(UUID::randomUUID).thenReturn(fixedUuid);
            
            // Predictable UUID in test
            UUID generated = UUID.randomUUID();
            assertEquals(fixedUuid, generated);
        }
    }
    
    // ========================================
    // REAL-WORLD EXAMPLE: LocalDateTime.now()
    // ========================================
    
    @Test
    void mockCurrentTime() {
        try (MockedStatic<LocalDateTime> mockedStatic = mockStatic(LocalDateTime.class)) {
            LocalDateTime fixedTime = LocalDateTime.of(2024, 1, 1, 12, 0);
            
            mockedStatic.when(LocalDateTime::now).thenReturn(fixedTime);
            
            // Predictable time in test
            LocalDateTime now = LocalDateTime.now();
            assertEquals(fixedTime, now);
        }
    }
    
    // ========================================
    // BEST PRACTICES
    // ========================================
    
    /**
     * ⚠ CAUTION WITH STATIC MOCKING:
     * 
     * ✗ Don't mock JDK classes in production code
     *   (LocalDateTime, UUID, etc. - use Clock, supplier pattern instead)
     * 
     * ✗ Don't mock utility classes you control
     *   (Better to inject them as dependencies)
     * 
     * ✓ OK for legacy code (can't change to dependency injection)
     * ✓ OK for third-party static methods (can't control)
     * ✓ OK for testing time-dependent code
     * 
     * BETTER DESIGN:
     * - Inject Clock for time
     * - Inject Supplier<UUID> for UUIDs
     * - Make utility classes instantiable and injectable
     */
```

## 7. Best Practices and Anti-Patterns

### Do's and Don'ts

```java
/**
 * BEST PRACTICES AND ANTI-PATTERNS
 */

class BestPracticesAndAntiPatterns {
    
    // ========================================
    // ✓ BEST PRACTICE 1: Mock at boundaries
    // ========================================
    
    /**
     * ✓ GOOD: Mock external dependencies
     */
    @Test
    void mockAtBoundaries() {
        // Mock repository (database boundary)
        UserRepository repository = mock(UserRepository.class);
        
        // Mock email service (external service boundary)
        EmailService emailService = mock(EmailService.class);
        
        // Test service layer
        UserService service = new UserService(repository, emailService);
        
        /**
         * MOCK:
         * ✓ Database access (repository)
         * ✓ External APIs (payment gateway, email)
         * ✓ File system, network I/O
         * ✓ Time/date (Clock)
         * ✓ Random number generation
         */
    }
    
    /**
     * ❌ BAD: Mock domain objects
     */
    @Test
    void dontMockDomainObjects() {
        // ❌ DON'T mock value objects
        User mockUser = mock(User.class);
        Order mockOrder = mock(Order.class);
        
        // ✓ Use real objects instead
        User realUser = new User("test@example.com", "Test");
        Order realOrder = new Order(realUser, Arrays.asList());
        
        /**
         * DON'T MOCK:
         * ✗ Value objects (User, Order, Money)
         * ✗ DTOs (data transfer objects)
         * ✗ Domain entities
         * ✗ Your own business logic
         */
    }
    
    // ========================================
    // ✓ BEST PRACTICE 2: Test behavior, not implementation
    // ========================================
    
    /**
     * ❌ BAD: Testing implementation details
     */
    @Test
    void testingImplementation() {
        UserRepository repository = mock(UserRepository.class);
        UserService service = new UserService(repository);
        
        when(repository.findByEmail("test@example.com")).thenReturn(Optional.empty());
        
        service.registerUser("test@example.com", "Test");
        
        // ❌ Brittle: Verifies HOW, not WHAT
        verify(repository).findByEmail("test@example.com");  // Implementation detail
        verify(repository).save(any(User.class));            // Implementation detail
    }
    
    /**
     * ✓ GOOD: Testing behavior
     */
    @Test
    void testingBehavior() {
        UserRepository repository = mock(UserRepository.class);
        EmailService emailService = mock(EmailService.class);
        UserService service = new UserService(repository, emailService);
        
        when(repository.findByEmail(anyString())).thenReturn(Optional.empty());
        
        service.registerUser("test@example.com", "Test");
        
        // ✓ Tests behavior: "User should receive welcome email"
        verify(emailService).sendEmail(
            eq("test@example.com"),
            eq("Welcome!"),
            anyString()
        );
        
        /**
         * FOCUS ON:
         * ✓ Observable behavior (side effects)
         * ✓ Return values
         * ✓ State changes
         * ✗ Internal method calls
         * ✗ Call order (unless it matters)
         */
    }
    
    // ========================================
    // ✓ BEST PRACTICE 3: Minimal verification
    // ========================================
    
    /**
     * ❌ BAD: Over-verification
     */
    @Test
    void overVerification() {
        UserRepository repository = mock(UserRepository.class);
        EmailService emailService = mock(EmailService.class);
        UserService service = new UserService(repository, emailService);
        
        when(repository.findByEmail(anyString())).thenReturn(Optional.empty());
        
        service.registerUser("test@example.com", "Test");
        
        // ❌ Verifying everything makes test brittle
        verify(repository).findByEmail("test@example.com");
        verify(repository).save(argThat(u -> u.getEmail().equals("test@example.com")));
        verify(emailService).sendEmail("test@example.com", "Welcome!", anyString());
        verifyNoMoreInteractions(repository, emailService);
    }
    
    /**
     * ✓ GOOD: Verify what matters
     */
    @Test
    void minimalVerification() {
        UserRepository repository = mock(UserRepository.class);
        EmailService emailService = mock(EmailService.class);
        UserService service = new UserService(repository, emailService);
        
        when(repository.findByEmail(anyString())).thenReturn(Optional.empty());
        
        service.registerUser("test@example.com", "Test");
        
        // ✓ Verify only the critical behavior
        verify(emailService).sendEmail(
            eq("test@example.com"),
            anyString(),
            anyString()
        );
        
        // Don't verify every internal call
    }
    
    // ========================================
    // ✓ BEST PRACTICE 4: Use real objects when possible
    // ========================================
    
    /**
     * ❌ BAD: Mocking everything
     */
    @Test
    void mockingEverything() {
        User mockUser = mock(User.class);
        when(mockUser.getEmail()).thenReturn("test@example.com");
        when(mockUser.getName()).thenReturn("Test");
        
        // Why mock a simple value object?
    }
    
    /**
     * ✓ GOOD: Use real objects
     */
    @Test
    void usingRealObjects() {
        User realUser = new User("test@example.com", "Test");
        
        // Simple, clear, fast
        assertEquals("test@example.com", realUser.getEmail());
    }
    
    // ========================================
    // ❌ ANTI-PATTERN 1: Mocking types you don't own
    // ========================================
    
    /**
     * ❌ BAD: Mocking third-party library
     */
    @Test
    void mockingThirdParty() {
        // ❌ Don't mock HttpClient, ObjectMapper, etc.
        HttpClient mockClient = mock(HttpClient.class);
        
        /**
         * PROBLEMS:
         * ✗ Don't know implementation details
         * ✗ Might mock incorrectly
         * ✗ Tests don't reflect real behavior
         */
    }
    
    /**
     * ✓ GOOD: Wrap third-party in your interface
     */
    interface HttpClientWrapper {
        String get(String url);
    }
    
    @Test
    void wrappingThirdParty() {
        // ✓ Mock your wrapper
        HttpClientWrapper wrapper = mock(HttpClientWrapper.class);
        when(wrapper.get("https://api.example.com")).thenReturn("response");
        
        // Use wrapper in production code
    }
    
    // ========================================
    // ❌ ANTI-PATTERN 2: Returning mocks
    // ========================================
    
    /**
     * ❌ BAD: Mock returning mock
     */
    @Test
    void mockReturningMock() {
        UserRepository repository = mock(UserRepository.class);
        User mockUser = mock(User.class);
        
        when(repository.findByEmail(anyString())).thenReturn(Optional.of(mockUser));
        when(mockUser.getName()).thenReturn("Test");
        
        // ❌ Mocks all the way down!
    }
    
    /**
     * ✓ GOOD: Mock returns real object
     */
    @Test
    void mockReturningReal() {
        UserRepository repository = mock(UserRepository.class);
        User realUser = new User("test@example.com", "Test");
        
        when(repository.findByEmail(anyString())).thenReturn(Optional.of(realUser));
        
        // ✓ Clear boundary: mock at repository, real objects in domain
    }
    
    // ========================================
    // ❌ ANTI-PATTERN 3: Too many mocks
    // ========================================
    
    /**
     * ❌ BAD: Service with 10 dependencies
     */
    class ServiceWithTooManyDependencies {
        @Mock private Repository1 repo1;
        @Mock private Repository2 repo2;
        @Mock private Service1 service1;
        @Mock private Service2 service2;
        @Mock private Service3 service3;
        @Mock private EmailService emailService;
        @Mock private SmsService smsService;
        @Mock private PaymentService paymentService;
        @Mock private NotificationService notificationService;
        @Mock private AuditService auditService;
        
        /**
         * PROBLEM: Too many mocks = design smell
         * 
         * ✓ Solution: Break up the service (SRP violation)
         */
    }
    
    // ========================================
    // MOCKITO VS EASYMOCK
    // ========================================
    
    /**
     * MOCKITO (recommended)
     */
    @Test
    void mockitoExample() {
        UserRepository repository = mock(UserRepository.class);
        
        // Stub
        when(repository.findByEmail("test@example.com"))
            .thenReturn(Optional.of(new User("test@example.com", "Test")));
        
        // Use
        Optional<User> user = repository.findByEmail("test@example.com");
        
        // Verify
        verify(repository).findByEmail("test@example.com");
    }
    
    /**
     * EASYMOCK (older, less popular)
     */
    @Test
    void easyMockExample() {
        // Record-replay model
        UserRepository repository = EasyMock.createMock(UserRepository.class);
        
        // RECORD expectations
        EasyMock.expect(repository.findByEmail("test@example.com"))
                .andReturn(Optional.of(new User("test@example.com", "Test")));
        
        // REPLAY
        EasyMock.replay(repository);
        
        // Use
        Optional<User> user = repository.findByEmail("test@example.com");
        
        // Verify
        EasyMock.verify(repository);
        
        /**
         * WHY MOCKITO PREFERRED:
         * ✓ Simpler (no record-replay)
         * ✓ More flexible
         * ✓ Better error messages
         * ✓ More popular (better support)
         */
    }
}
```

```java
// ========================================

 * 1. Mock vs Spy vs Stub

// Stub: Returns data, no verification
when(repo.findById("123")).thenReturn(user);

verify(emailService).sendEmail(eq("user@example.com"), anyString(), anyString());

EmailService spy = spy(new RealEmailService());

/**
 * 2. Annotations
 */
@Mock private UserRepository repository;
@InjectMocks private UserService userService;

/**
 * 3. Stubbing
 */
when(method()).thenReturn(value);
when(method()).thenThrow(exception);
doThrow(exception).when(mock).voidMethod();

/**
 * 4. Argument Matchers
 */
any(), anyString(), anyInt()
eq("exact"), contains("substring")
argThat(predicate), isNull(), notNull()

/**
 * 5. Verification
 */
verify(mock).method();
verify(mock, times(3)).method();
verify(mock, never()).method();
verifyNoInteractions(mock);

/**
 * 6. Static Mocking
 */
try (MockedStatic<ClassName> mocked = mockStatic(ClassName.class)) {

// ========================================

 * ✗ Value objects (User, Order, Money)

 * ✗ Simple utility classes
```

**Key Takeaways:**

```
The Mocking Hierarchy:

Mock: Verifies interactions (commands)
Stub: Returns data (queries)
Spy: Real + verification (rare)

Mock at boundaries:
✓ Repository (database)
✓ External APIs
✓ File system, network
✗ Value objects
✗ Your domain logic

Test behavior, not implementation:
✓ "User receives welcome email"
✗ "repository.save() called with User"

The Golden Rule:
Mock external dependencies, not your own code.
Use real objects when fast and simple.
```
