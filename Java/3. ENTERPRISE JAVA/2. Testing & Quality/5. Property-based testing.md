# Property-Based Testing

## Overview

Property-based testing verifies universal properties of code instead of specific examples—rather than testing "add(2, 3) = 5", test "add(a, b) = add(b, a) for all integers a, b". Traditional example-based tests check specific cases (explicit inputs/outputs), while property-based tests check general rules across thousands of randomly generated inputs. This guide covers six essential concepts: JUnit QuickCheck (early property testing for Java, verbose but established), jqwik Framework (modern, JUnit 5-native, recommended for new projects—better generators, shrinking, API), Generating Test Data (automatic generation of random inputs—numbers, strings, lists, custom objects), Shrinking Failures (when test fails with complex input, framework automatically finds minimal failing case), Use Cases (testing algorithms, parsers, encoders, data structures, mathematical properties), and Complementing Example-Based Tests (property tests find edge cases, example tests document behavior—use both). The key insight: property-based testing finds bugs example-based testing misses—edge cases, boundary conditions, unexpected combinations. Instead of thinking "what inputs should I test?", think "what should always be true?". Properties are invariants: reversibility (encode then decode = original), idempotence (f(f(x)) = f(x)), commutativity (f(a,b) = f(b,a)), structural preservation (sort doesn't change length), error handling (invalid input throws exception). Understanding property-based testing means understanding the paradigm shift: from "does this work for these cases?" to "does this work for all cases?". Good properties are universal (true for all valid inputs), checkable (can be verified programmatically), and meaningful (catch real bugs). The framework generates hundreds/thousands of random inputs, runs property checks, and when failure found, shrinks to minimal failing case. This guide shows when property testing shines (complex algorithms, parsers, data transformations) and when example-based tests are better (specific business rules, exact calculations, regression tests). The combination is powerful: properties verify general correctness, examples document specific behavior.

## 1. Introduction: Property-Based vs Example-Based Testing

### The Paradigm Shift

```java
/**
 * EXAMPLE-BASED TESTING (Traditional)
 * 
 * Test specific inputs and outputs
 */
class ExampleBasedTesting {
    
    @Test
    void shouldAddTwoPositiveNumbers() {
        assertEquals(5, add(2, 3));
    }
    
    @Test
    void shouldAddNegativeNumbers() {
        assertEquals(-5, add(-2, -3));
    }
    
    @Test
    void shouldAddZero() {
        assertEquals(5, add(5, 0));
    }
    
    /**
     * LIMITATIONS:
     * ✗ Only tests specific cases
     * ✗ Might miss edge cases
     * ✗ Requires thinking of all scenarios
     * ✗ Can't test all possible inputs
     */
}

/**
 * PROPERTY-BASED TESTING
 * 
 * Test universal properties across many random inputs
 */
class PropertyBasedTesting {
    
    @Property
    void additionIsCommutative(@ForAll int a, @ForAll int b) {
        // Tests with HUNDREDS of random integer pairs
        assertEquals(add(a, b), add(b, a));
    }
    
    @Property
    void addingZeroDoesNothing(@ForAll int n) {
        // Tests with HUNDREDS of random integers
        assertEquals(n, add(n, 0));
        assertEquals(n, add(0, n));
    }
    
    @Property
    void additionIsAssociative(@ForAll int a, @ForAll int b, @ForAll int c) {
        // Tests with HUNDREDS of random integer triples
        assertEquals(add(add(a, b), c), add(a, add(b, c)));
    }
    
    /**
     * BENEFITS:
     * ✓ Tests thousands of cases automatically
     * ✓ Finds edge cases you didn't think of
     * ✓ Tests mathematical properties
     * ✓ Catches subtle bugs
     */
}

// ========================================
// REAL-WORLD COMPARISON
// ========================================

/**
 * Example: Testing a reverse function
 */
class ReverseFunction {
    
    public static <T> List<T> reverse(List<T> list) {
        List<T> result = new ArrayList<>(list);
        Collections.reverse(result);
        return result;
    }
}

/**
 * EXAMPLE-BASED: Test specific cases
 */
class ReverseExampleTest {
    
    @Test
    void shouldReverseThreeElements() {
        List<Integer> input = List.of(1, 2, 3);
        List<Integer> expected = List.of(3, 2, 1);
        assertEquals(expected, ReverseFunction.reverse(input));
    }
    
    @Test
    void shouldReverseEmptyList() {
        List<Integer> input = List.of();
        List<Integer> expected = List.of();
        assertEquals(expected, ReverseFunction.reverse(input));
    }
    
    @Test
    void shouldReverseSingleElement() {
        List<Integer> input = List.of(42);
        List<Integer> expected = List.of(42);
        assertEquals(expected, ReverseFunction.reverse(input));
    }
    
    // What about 2 elements? 100 elements? Duplicates? Nulls?
}

/**
 * PROPERTY-BASED: Test universal properties
 */
class ReversePropertyTest {
    
    @Property
    void reversingTwiceGivesOriginal(@ForAll List<Integer> list) {
        // Reversing twice should give original list
        List<Integer> reversed = ReverseFunction.reverse(list);
        List<Integer> reversedTwice = ReverseFunction.reverse(reversed);
        assertEquals(list, reversedTwice);
    }
    
    @Property
    void reversedListHasSameLength(@ForAll List<Integer> list) {
        // Reverse doesn't change length
        assertEquals(list.size(), ReverseFunction.reverse(list).size());
    }
    
    @Property
    void reversedListHasSameElements(@ForAll List<Integer> list) {
        // Reverse doesn't change elements (just order)
        List<Integer> reversed = ReverseFunction.reverse(list);
        assertTrue(list.containsAll(reversed));
        assertTrue(reversed.containsAll(list));
    }
    
    @Property
    void firstElementBecomesLast(@ForAll List<@IntRange(min = 1) Integer> nonEmptyList) {
        Assume.that(!nonEmptyList.isEmpty());
        
        Integer first = nonEmptyList.get(0);
        List<Integer> reversed = ReverseFunction.reverse(nonEmptyList);
        Integer last = reversed.get(reversed.size() - 1);
        
        assertEquals(first, last);
    }
    
    /**
     * These properties automatically test:
     * - Empty lists
     * - Single element
     * - Many elements
     * - Duplicates
     * - All sorts of combinations
     * 
     * Framework generates and tests hundreds of cases!
     */
}

// ========================================
// KEY CONCEPTS
// ========================================

/**
 * PROPERTY: A statement that should always be true
 * 
 * Good properties:
 * ✓ Universal (true for all valid inputs)
 * ✓ Checkable (can verify programmatically)
 * ✓ Meaningful (catches real bugs)
 * 
 * Common property patterns:
 * • Reversibility: encode(decode(x)) = x
 * • Idempotence: sort(sort(x)) = sort(x)
 * • Commutativity: f(a,b) = f(b,a)
 * • Invariants: operation preserves properties
 * • Round-trip: serialize then deserialize = original
 */

private static int add(int a, int b) {
    return a + b;
}
```

## 2. jqwik Framework

### Modern Property-Based Testing for JUnit 5

```java
/**
 * JQWIK: Recommended framework for Java property testing
 * 
 * JUnit 5-native, excellent generators, automatic shrinking

import net.jqwik.api.*;
import net.jqwik.api.constraints.*;

// ========================================
// BASIC PROPERTY TESTS
// ========================================

class BasicJqwikTests {
    
    /**
     * Simple property test
     */
    @Property
    void absoluteValueIsNeverNegative(@ForAll int n) {
        int abs = Math.abs(n);
        assertTrue(abs >= 0);
    }
    
    /**
     * Multiple parameters
     */
    @Property
    void stringConcatenationLength(@ForAll String s1, @ForAll String s2) {
        String concatenated = s1 + s2;
        assertEquals(s1.length() + s2.length(), concatenated.length());
    }
    
    /**
     * Configure number of tries
     */
    @Property(tries = 1000)  // Run 1000 times (default is 1000)
    void multiplicationByZeroIsZero(@ForAll int n) {
        assertEquals(0, n * 0);
    }
    
    /**
     * Seed for reproducibility
     */
    @Property(seed = "42")  // Same random values every run
    void deterministicTest(@ForAll int n) {
        assertTrue(n >= Integer.MIN_VALUE);

}

        assertTrue(sqrt <= 100);

// ASSUMPTIONS AND PRECONDITIONS
// ========================================

class AssumptionsAndPreconditions {
    
    /**
     * Filter inputs with assumptions
     */
    @Property
    void divisionByNonZero(@ForAll int a, @ForAll int b) {
        Assume.that(b != 0);  // Skip test if b is zero
        
        int result = a / b;
        assertTrue(result * b + (a % b) == a);
    }
    
    /**
     * Multiple assumptions
     */
    @Property
    void sortedListOperations(@ForAll List<Integer> list) {
        Assume.that(list.size() >= 2);
        Assume.that(isSorted(list));
        
        // Now we know list is sorted and has at least 2 elements
        assertTrue(list.get(0) <= list.get(list.size() - 1));
    }
    
    private boolean isSorted(List<Integer> list) {
        for (int i = 0; i < list.size() - 1; i++) {
            if (list.get(i) > list.get(i + 1)) {
                return false;
            }
        }
        return true;
    }
}

// ========================================

class ComplexDataGeneration {
    
    /**
     * Generate trees
     */
    @Property
    void binaryTreeProperties(@ForAll("binaryTrees") BinaryTree tree) {
        assertTrue(tree.size() >= 0);
        if (tree.size() > 0) {
            assertNotNull(tree.value());
        }
    }
    
    @Provide
    Arbitrary<BinaryTree> binaryTrees() {
        return Arbitraries.lazy(
            () -> Arbitraries.oneOf(
                Arbitraries.just(BinaryTree.empty()),
                Combinators.combine(
                    Arbitraries.integers(),
                    binaryTrees(),
                    binaryTrees()
                ).as(BinaryTree::new)
            ),
            BinaryTree::new
        );
    }
    
    /**
     * Generate email addresses
     */
    @Property
    void emailProcessing(@ForAll("emails") String email) {
        assertTrue(email.matches("^[^@]+@[^@]+\\.[a-z]+$"));
    }
    
    @Provide
    Arbitrary<String> emails() {
        Arbitrary<String> localPart = Arbitraries.strings()
            .withCharRange('a', 'z')
            .ofMinLength(1)
            .ofMaxLength(20);
        
        Arbitrary<String> domain = Arbitraries.strings()
            .withCharRange('a', 'z')
            .ofMinLength(1)
            .ofMaxLength(20);
        
        Arbitrary<String> tld = Arbitraries.of("com", "org", "net");
        
        return Combinators.combine(localPart, domain, tld)
            .as((local, dom, t) -> local + "@" + dom + "." + t);
    }
}

class BinaryTree {
    private Integer value;
    private BinaryTree left, right;
    
    BinaryTree(Integer value, BinaryTree left, BinaryTree right) {
        this.value = value;
        this.left = left;
        this.right = right;
    }
    
    static BinaryTree empty() {
        return new BinaryTree(null, null, null);
    }
    
    Integer value() { return value; }
    int size() {
        if (value == null) return 0;
        return 1 + (left != null ? left.size() : 0) + (right != null ? right.size() : 0);
    }
```

## 3. Shrinking: Finding Minimal Failing Cases

### Automatic Test Case Minimization

```java
/**
 * SHRINKING
 * 
 * When property test fails, jqwik automatically finds
 * the SMALLEST input that causes failure
 */

// ========================================
// HOW SHRINKING WORKS
// ========================================

class ShrinkingExamples {
    
    /**
     * Example: Buggy sort function
     */
    static List<Integer> buggySort(List<Integer> list) {
        if (list.size() <= 1) return new ArrayList<>(list);
        
        List<Integer> result = new ArrayList<>(list);
        // Bug: doesn't sort correctly for lists with duplicates
        Collections.sort(result);
        if (result.size() > 2 && result.get(0).equals(result.get(1))) {
            // Intentional bug for demonstration
            Collections.swap(result, 0, 1);
        }
        return result;
    }
    
    /**
     * Property test that will fail
     */
    @Property
    void sortedListIsActuallySorted(@ForAll List<Integer> list) {
        List<Integer> sorted = buggySort(list);
        
        // Check if sorted
        for (int i = 0; i < sorted.size() - 1; i++) {
            assertTrue(sorted.get(i) <= sorted.get(i + 1),
                "List not sorted at index " + i + ": " + sorted);
        }
    }
    
    /**
     * WHAT HAPPENS:
     * 
     * 1. jqwik generates random list, e.g. [5, 3, 8, 1, 3, 9, 2, 3]
     * 2. Test fails on this list
     * 3. jqwik SHRINKS to minimal failing case
     * 4. Reports: Test failed with [1, 1] (smallest failing input)
     * 
     * WITHOUT SHRINKING:
     * "Test failed with [5, 3, 8, 1, 3, 9, 2, 3]" ← Hard to debug
     * 
     * WITH SHRINKING:
     * "Test failed with [1, 1]" ← Easy to see the bug!
     */
}

// ========================================
// SHRINKING STRATEGIES
// ========================================

class ShrinkingStrategies {
    
    /**
     * Numbers shrink towards zero
     */
    @Property
    void numberShrinking(@ForAll int n) {
        // If test fails with n=537, shrinks to smallest failing number
        // e.g., if bug is "fails for n > 10", shrinks to n=11
        assertTrue(n <= 1000000, "Failed with: " + n);
    }
    
    /**
     * Lists shrink towards empty
     */
    @Property
    void listShrinking(@ForAll List<Integer> list) {
        // Shrinks both:
        // 1. Size of list (fewer elements)
        // 2. Values in list (smaller numbers)
        
        // If fails with [100, 200, 300, 400], might shrink to [1, 2]
        assertTrue(list.size() < 1000, "Failed with: " + list);
    }
    
    /**
     * Strings shrink towards empty
     */
    @Property
    void stringShrinking(@ForAll String s) {
        // Shrinks to shorter strings
        // If fails with "abcdefghijklmnop", might shrink to "aa"
        assertTrue(s.length() < 100, "Failed with: " + s);
    }
    
    /**
     * Objects shrink component-wise
     */
    @Property
    void objectShrinking(@ForAll("users") User user) {
        // Shrinks each field independently
        // Name → shorter
        // Age → smaller
        // Email → shorter
        assertTrue(user.getAge() < 200, "Failed with: " + user);
    }
    
    @Provide
    Arbitrary<User> users() {
        return Combinators.combine(
            Arbitraries.strings().alpha().ofMinLength(1),
            Arbitraries.strings().alpha().ofMinLength(1),
            Arbitraries.integers().between(0, 150)
        ).as((e, n, a) -> new User(e + "@example.com", n, a));
    }
}

// ========================================
// SHRINKING DEMONSTRATION
// ========================================

class ShrinkingDemonstration {
    
    /**
     * Bug: Function fails when sum of list exceeds 100
     */
    static int processNumbers(List<Integer> numbers) {
        int sum = numbers.stream().mapToInt(Integer::intValue).sum();
        if (sum > 100) {
            throw new IllegalArgumentException("Sum too large: " + sum);
        }
        return sum;
    }
    
    @Property
    void shouldProcessAnyNumbers(@ForAll List<@IntRange(min = 0, max = 50) Integer> numbers) {
        // Test fails when sum > 100
        // Example: [25, 30, 28, 22] → sum = 105
        
        int result = processNumbers(numbers);
        
        // jqwik will shrink to minimal failing case
        // Final failure: [1, 1, 1, ..., 1] (101 times)
        // Or: [50, 50, 1] (sum = 101)
        // Or: [51, 50] if max was higher
        
        assertTrue(result <= 100);
    }
    
    /**
     * OUTPUT (after shrinking):
     * 
     * Property [shouldProcessAnyNumbers] failed with sample
     * Input: [[50, 50, 1]]
     * 
     * This is the MINIMAL failing case!
     * Original failure might have been [42, 17, 28, 31, 5]
     */
}

// ========================================
// CONTROLLING SHRINKING
// ========================================

class ControllingShrinking {
    
    /**
     * Disable shrinking
     */
    @Property(shrinking = ShrinkingMode.OFF)
    void noShrinking(@ForAll String s) {
        // Reports original failing input, no shrinking
        assertTrue(s.length() < 10);
    }
    
    /**
     * Custom shrinking
     */
    @Property
    void customShrinking(@ForAll("customInts") int n) {
        assertTrue(n < 100);
    }
    
    @Provide
    Arbitrary<Integer> customInts() {
        return Arbitraries.integers()
            .between(0, 1000)
            .shrinkTowards(50);  // Shrink towards 50, not 0
    }
}
```

## 5. Real-World Use Cases

### When Property-Based Testing Shines

```java
/**
 * REAL-WORLD USE CASES FOR PROPERTY-BASED TESTING
 */

// ========================================
// USE CASE 1: DATA SERIALIZATION
// ========================================

class JsonSerialization {
    
    /**
     * Property: Serializing then deserializing gives original
     */
    @Property
    void jsonRoundTrip(@ForAll("users") User user) {
        // Serialize
        String json = toJson(user);
        
        // Deserialize
        User deserialized = fromJson(json, User.class);
        
        // Should be equal
        assertEquals(user, deserialized);
    }
    
    @Provide
    Arbitrary<User> users() {
        return Combinators.combine(
            Arbitraries.strings().alpha().ofMinLength(1).map(s -> s + "@example.com"),
            Arbitraries.strings().alpha().ofMinLength(1),
            Arbitraries.integers().between(0, 120)
        ).as(User::new);
    }
    
    /**
     * BENEFIT: Tests ALL edge cases automatically:
     * - Empty strings
     * - Special characters
     * - Large numbers
     * - Unicode
     * - etc.
     */
}

// ========================================
// USE CASE 2: ENCODING/DECODING
// ========================================

class Base64Encoding {
    
    /**
     * Property: encode then decode = original
     */
    @Property
    void base64RoundTrip(@ForAll byte[] data) {
        String encoded = Base64.getEncoder().encodeToString(data);
        byte[] decoded = Base64.getDecoder().decode(encoded);
        
        assertArrayEquals(data, decoded);
    }
    
    /**
     * Property: encoded string is valid Base64
     */
    @Property
    void encodedIsValidBase64(@ForAll byte[] data) {
        String encoded = Base64.getEncoder().encodeToString(data);
        
        // Valid Base64: only contains A-Z, a-z, 0-9, +, /, =
        assertTrue(encoded.matches("^[A-Za-z0-9+/]*={0,2}$"));
    }
    
    /**
     * Property: encoding is deterministic
     */
    @Property
    void encodingIsDeterministic(@ForAll byte[] data) {
        String encoded1 = Base64.getEncoder().encodeToString(data);
        String encoded2 = Base64.getEncoder().encodeToString(data);
        
        assertEquals(encoded1, encoded2);
    }
}

// ========================================
// USE CASE 3: SORTING ALGORITHMS
// ========================================

class SortingAlgorithms {
    
    /**
     * Property: sorted list is actually sorted
     */
    @Property
    void sortProducesOrderedList(@ForAll List<Integer> list) {
        List<Integer> sorted = new ArrayList<>(list);
        Collections.sort(sorted);
        
        for (int i = 0; i < sorted.size() - 1; i++) {
            assertTrue(sorted.get(i) <= sorted.get(i + 1));
        }
    }
    
    /**
     * Property: sorting preserves length
     */
    @Property
    void sortPreservesLength(@ForAll List<Integer> list) {
        List<Integer> sorted = new ArrayList<>(list);
        Collections.sort(sorted);
        
        assertEquals(list.size(), sorted.size());
    }
    
    /**
     * Property: sorting preserves elements
     */
    @Property
    void sortPreservesElements(@ForAll List<Integer> list) {
        List<Integer> sorted = new ArrayList<>(list);
        Collections.sort(sorted);
        
        // Same elements, just reordered
        assertTrue(list.containsAll(sorted));
        assertTrue(sorted.containsAll(list));
    }
    
    /**
     * Property: sorting is idempotent
     */
    @Property
    void sortingIsIdempotent(@ForAll List<Integer> list) {
        List<Integer> sorted1 = new ArrayList<>(list);
        Collections.sort(sorted1);
        
        List<Integer> sorted2 = new ArrayList<>(sorted1);
        Collections.sort(sorted2);
        
        assertEquals(sorted1, sorted2);
    }

class UrlParsing {

     * Property: parsing then serializing gives original
     */
    @Property

        // Parse URL
        URL parsed = new URL(url);
        
        // Serialize back to string
        String serialized = parsed.toString();
        
        // Should be equivalent
        assertEquals(normalize(url), normalize(serialized));
    }
    
    @Provide
    Arbitrary<String> validUrls() {
        Arbitrary<String> protocols = Arbitraries.of("http", "https");
        Arbitrary<String> domains = Arbitraries.strings()
            .withCharRange('a', 'z')
            .ofMinLength(1)
            .ofMaxLength(20);
        Arbitrary<String> tlds = Arbitraries.of("com", "org", "net");
        
        return Combinators.combine(protocols, domains, tlds)
            .as((p, d, t) -> p + "://" + d + "." + t);
    }
    
    private String normalize(String url) {
        // Remove trailing slashes, etc.
        return url.replaceAll("/$", "");
    }
```

```java
class PasswordValidator {
    
    public static boolean isValid(String password) {
        if (password == null || password.length() < 8) {

        boolean hasLower = password.chars().anyMatch(Character::isLowerCase);

    }
}
```

**Key Takeaways:**

```
The Paradigm Shift:
Example-based: "Does this work for THESE cases?"
Property-based: "Does this work for ALL cases?"

Common Properties:
• Reversibility: encode(decode(x)) = x
• Idempotence: f(f(x)) = f(x)
• Commutativity: f(a,b) = f(b,a)
• Invariants: sort preserves length
• Round-trip: serialize then deserialize

jqwik Pattern:
@Property
void propertyName(@ForAll Type param) {
    // Test universal property
    // Runs hundreds of times with random inputs
}

Shrinking Magic:
Test fails with: [5, 3, 8, 1, 3, 9, 2, 3]
Shrinks to:      [1, 1]
→ Easy to debug!

When to Use:
Property-based → Algorithms, parsers, encoders
Example-based → Business rules, specific cases
BOTH → Critical functionality
```
