# Integration Testing Strategies

## Overview

Integration tests verify that multiple components work together correctly—they test interactions, not just isolated units. Unlike unit tests (fast, mocked), integration tests use real components (databases, message queues, external services) to catch configuration issues, integration bugs, and contract mismatches. This guide covers six essential strategies: Spring Boot Test Slices (focused integration tests—@WebMvcTest for controllers, @DataJpaTest for repositories, @SpringBootTest for full context), Testcontainers (Docker containers for dependencies—real databases, message queues, external services in tests), Database Testing (H2 in-memory for speed vs Testcontainers with real databases for accuracy), REST API Testing (MockMvc for controller testing, RestAssured for full integration), Message Queue Testing (Kafka, RabbitMQ with Testcontainers), and Test Data Management (fixtures, builders, SQL scripts, proper cleanup). The key insight: integration tests trade speed for confidence—slower than unit tests but catch real-world issues. Good integration tests are focused (test specific layers, not everything), isolated (clean state between tests), realistic (use real components when possible), and maintainable (good test data management). Understanding these strategies means understanding trade-offs: @WebMvcTest is fast but mocks services, @SpringBootTest is slow but tests full integration; H2 is fast but behavior differs from production, Testcontainers is slower but uses real databases; MockMvc is lightweight but server-side only, RestAssured tests full HTTP but requires running application. This guide shows when to use each approach, common mistakes (testing too much at once, poor data cleanup, flaky tests from timing), and how strategies combine (@DataJpaTest + Testcontainers = reliable repository tests, @WebMvcTest + MockMvc = fast controller tests, @SpringBootTest + RestAssured + Testcontainers = comprehensive integration tests).

## 1. Spring Boot Test Slices

### Focused Integration Testing

```java
/**
 * SPRING BOOT TEST SLICES
 * 
 * Test specific layers without loading entire application context
 * Faster and more focused than @SpringBootTest
 */

import org.springframework.boot.test.context.*;
import org.springframework.test.web.servlet.*;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.mock.mockito.MockBean;

// ========================================
// PRODUCTION CODE
// ========================================

@Entity
class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String email;
    private String name;
    
    // Constructors, getters, setters
    public User() {}
    public User(String email, String name) {
        this.email = email;
        this.name = name;
    }
    
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getEmail() { return email; }
    public String getName() { return name; }
}

@Repository
interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);
}

@Service
class UserService {
    private final UserRepository repository;
    
    public UserService(UserRepository repository) {
        this.repository = repository;
    }
    
    public User createUser(String email, String name) {
        if (repository.findByEmail(email).isPresent()) {
            throw new IllegalArgumentException("User already exists");
        }
        User user = new User(email, name);
        return repository.save(user);
    }
    
    public Optional<User> findByEmail(String email) {
        return repository.findByEmail(email);
    }
}

@RestController
@RequestMapping("/api/users")
class UserController {
    private final UserService userService;
    
    public UserController(UserService userService) {
        this.userService = userService;
    }
    
    @PostMapping
    public ResponseEntity<User> createUser(@RequestBody CreateUserRequest request) {
        User user = userService.createUser(request.email(), request.name());
        return ResponseEntity.status(HttpStatus.CREATED).body(user);
    }
    
    @GetMapping("/{email}")
    public ResponseEntity<User> getUser(@PathVariable String email) {
        return userService.findByEmail(email)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }
}

record CreateUserRequest(String email, String name) {}

// ========================================
// 1. @WebMvcTest - Controller Layer Testing
// ========================================

/**
 * @WebMvcTest: Test controllers with mocked services
 * 
 * Loads:
 * ✓ @Controller, @RestController
 * ✓ @ControllerAdvice
 * ✓ MockMvc
 * ✓ WebMvcConfigurer
 * 
 * Does NOT load:
 * ✗ @Service, @Repository
 * ✗ Database
 * ✗ Full application context
 */
@WebMvcTest(UserController.class)
class UserControllerTest {
    
    @Autowired
    private MockMvc mockMvc;
    
    @MockBean  // Mock the service layer
    private UserService userService;
    
    @Test
    void shouldCreateUser() throws Exception {
        // Arrange
        User user = new User("test@example.com", "Test User");
        user.setId(1L);
        
        when(userService.createUser("test@example.com", "Test User"))
            .thenReturn(user);
        
        // Act & Assert
        mockMvc.perform(post("/api/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""
                    {
                        "email": "test@example.com",
                        "name": "Test User"
                    }
                    """))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.id").value(1))
            .andExpect(jsonPath("$.email").value("test@example.com"))
            .andExpect(jsonPath("$.name").value("Test User"));
        
        verify(userService).createUser("test@example.com", "Test User");
    }
    
    @Test
    void shouldReturn404WhenUserNotFound() throws Exception {
        // Arrange
        when(userService.findByEmail("notfound@example.com"))
            .thenReturn(Optional.empty());
        
        // Act & Assert
        mockMvc.perform(get("/api/users/notfound@example.com"))
            .andExpect(status().isNotFound());
    }
    
    /**
     * BENEFITS:
     * ✓ Fast (no database)
     * ✓ Focused (only controller layer)
     * ✓ Tests request/response handling
     * ✓ Tests validation
     * 
     * USE WHEN:
     * • Testing controller logic
     * • Testing request mapping
     * • Testing error handling
     * • Don't need real service implementation
     */
}

// ========================================
// 2. @DataJpaTest - Repository Layer Testing
// ========================================

/**
 * @DataJpaTest: Test JPA repositories with in-memory database
 * 
 * Loads:
 * ✓ @Repository
 * ✓ EntityManager
 * ✓ In-memory database (H2 by default)
 * ✓ @Transactional (rolled back after each test)
 * 
 * Does NOT load:
 * ✗ @Service, @Controller
 * ✗ Full application context
 */
@DataJpaTest
class UserRepositoryTest {
    
    @Autowired
    private UserRepository userRepository;
    
    @Autowired
    private TestEntityManager entityManager;
    
    @Test
    void shouldSaveAndFindUser() {
        // Arrange
        User user = new User("test@example.com", "Test User");
        
        // Act
        User saved = userRepository.save(user);
        entityManager.flush();  // Force SQL execution
        
        // Assert
        assertNotNull(saved.getId());
        
        Optional<User> found = userRepository.findByEmail("test@example.com");
        assertTrue(found.isPresent());
        assertEquals("Test User", found.get().getName());
    }
    
    @Test
    void shouldReturnEmptyWhenUserNotFound() {
        // Act
        Optional<User> found = userRepository.findByEmail("notfound@example.com");
        
        // Assert
        assertTrue(found.isEmpty());
    }
    
    @Test
    void shouldFindUserByEmail() {
        // Arrange - use TestEntityManager for setup
        User user = new User("test@example.com", "Test User");
        entityManager.persist(user);
        entityManager.flush();
        
        // Act
        Optional<User> found = userRepository.findByEmail("test@example.com");
        
        // Assert
        assertTrue(found.isPresent());
        assertEquals("Test User", found.get().getName());
    }
    
    /**
     * BENEFITS:
     * ✓ Fast (in-memory database)
     * ✓ Transactional (auto rollback)
     * ✓ Tests real SQL queries
     * ✓ TestEntityManager for setup
     * 
     * USE WHEN:
     * • Testing custom queries
     * • Testing repository methods
     * • Verifying database interactions
     */
}

// ========================================
// 3. @SpringBootTest - Full Application Testing
// ========================================

/**
 * @SpringBootTest: Load full application context
 * 
 * Loads:
 * ✓ Full application context
 * ✓ All beans (@Service, @Repository, @Controller)
 * ✓ Configuration
 * ✓ Can start embedded server
 * 
 * Slower but comprehensive
 */
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class UserIntegrationTest {
    
    @LocalServerPort
    private int port;
    
    @Autowired
    private TestRestTemplate restTemplate;
    
    @Autowired
    private UserRepository userRepository;
    
    @BeforeEach
    void setUp() {
        userRepository.deleteAll();  // Clean slate
    }
    
    @Test
    void shouldCreateAndRetrieveUser() {
        // Arrange
        CreateUserRequest request = new CreateUserRequest(
            "test@example.com", 
            "Test User"
        );
        
        // Act - Create user
        ResponseEntity<User> createResponse = restTemplate.postForEntity(
            "/api/users",
            request,
            User.class
        );
        
        // Assert creation
        assertEquals(HttpStatus.CREATED, createResponse.getStatusCode());
        assertNotNull(createResponse.getBody());
        assertNotNull(createResponse.getBody().getId());
        
        // Act - Retrieve user
        ResponseEntity<User> getResponse = restTemplate.getForEntity(
            "/api/users/test@example.com",
            User.class
        );
        
        // Assert retrieval
        assertEquals(HttpStatus.OK, getResponse.getStatusCode());
        assertEquals("Test User", getResponse.getBody().getName());
    }
    
    @Test
    void shouldReturn404WhenUserNotFound() {
        // Act
        ResponseEntity<User> response = restTemplate.getForEntity(
            "/api/users/notfound@example.com",
            User.class
        );
        
        // Assert
        assertEquals(HttpStatus.NOT_FOUND, response.getStatusCode());
    }
    
    /**
     * BENEFITS:
     * ✓ Tests full integration
     * ✓ Real HTTP requests
     * ✓ All beans loaded
     * ✓ Tests complete workflows
     * 
     * DRAWBACKS:
     * ✗ Slower (full context)
     * ✗ More complex setup
     * 
     * USE WHEN:
     * • Testing complete workflows
     • Testing bean wiring
     • Integration tests
     • E2E tests
     */
}
```

## 2. Testcontainers for Dependencies

### Docker Containers in Tests

```java
/**
 * TESTCONTAINERS
 * 
 * Use real Docker containers for dependencies in tests
 * More realistic than mocks or in-memory alternatives
 */

import org.testcontainers.containers.*;
import org.testcontainers.junit.jupiter.*;

// ========================================
// BASIC TESTCONTAINERS USAGE
// ========================================

@Testcontainers  // JUnit 5 extension
class TestcontainersBasics {
    
    /**
     * PostgreSQL container
     */
    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
        .withDatabaseName("testdb")
        .withUsername("test")
        .withPassword("test");
    
    @Test
    void shouldConnectToPostgres() {
        // Container started automatically
        String jdbcUrl = postgres.getJdbcUrl();
        assertNotNull(jdbcUrl);
        assertTrue(postgres.isRunning());
    }
    
    /**
     * LIFECYCLE:
     * 
     * @BeforeAll  → Container starts
     * Tests run   → Container available
     * @AfterAll   → Container stops
     * 
     * "static" = shared across all tests in class
     * "non-static" = new container per test
     */
}

// ========================================
// SPRING BOOT + TESTCONTAINERS
// ========================================

/**
 * Integrate Testcontainers with Spring Boot
 */
@SpringBootTest
@Testcontainers
class UserRepositoryWithTestcontainersTest {
    
    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
        .withDatabaseName("testdb")
        .withUsername("test")
        .withPassword("test");
    
    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        // Override Spring properties with container values
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }
    
    @Autowired
    private UserRepository userRepository;
    
    @Test
    void shouldPersistToRealPostgres() {
        // Using REAL PostgreSQL, not H2!
        User user = new User("test@example.com", "Test User");
        User saved = userRepository.save(user);
        
        assertNotNull(saved.getId());
        
        Optional<User> found = userRepository.findByEmail("test@example.com");
        assertTrue(found.isPresent());
    }
    
    /**
     * BENEFITS:
     * ✓ Real database (not H2)
     * ✓ Tests PostgreSQL-specific features
     * ✓ Catches SQL dialect issues
     * ✓ Same DB as production
     */
}

        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }
}

/**
```

```java
 * WHEN TO USE H2:
 * ✓ Unit-style repository tests
 * ✓ Fast CI/CD pipeline
 * ✓ Simple queries (no DB-specific features)
 * ✓ Developer machine (no Docker)
 * 
 * WHEN TO USE TESTCONTAINERS:
 * ✓ Complex queries
 * ✓ Database-specific features (JSON, arrays)
 * ✓ Migration testing
 * ✓ Pre-deployment verification
 * ✓ Want same DB as production
 * 
 * HYBRID APPROACH:
 * • H2 for fast feedback (developer machine)
 * • Testcontainers in CI/CD
 * • Use @ActiveProfiles to switch
 */
```

## 4. REST API Testing

### MockMvc vs RestAssured

```java
/**
 * REST API TESTING
 * 
 * MockMvc: Server-side testing (no HTTP)
 * RestAssured: Client-side testing (real HTTP)
 */

// ========================================
// MOCKMVC - SERVER-SIDE TESTING
// ========================================

/**
 * Test controllers without starting server
 */
@WebMvcTest(UserController.class)
class UserControllerMockMvcTest {
    
    @Autowired
    private MockMvc mockMvc;
    
    @MockBean
    private UserService userService;
    
    @Test
    void shouldCreateUser() throws Exception {
        // Arrange
        User user = new User("test@example.com", "Test User");
        user.setId(1L);
        
        when(userService.createUser(anyString(), anyString())).thenReturn(user);
        
        // Act & Assert
        mockMvc.perform(post("/api/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""
                    {
                        "email": "test@example.com",
                        "name": "Test User"
                    }
                    """))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.id").value(1))
            .andExpect(jsonPath("$.email").value("test@example.com"))
            .andExpect(jsonPath("$.name").value("Test User"));
    }
    
    @Test
    void shouldValidateRequestBody() throws Exception {
        // Test validation
        mockMvc.perform(post("/api/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("{}"))  // Empty body
            .andExpect(status().isBadRequest());
    }
    
    @Test
    void shouldHandleExceptions() throws Exception {
        // Test error handling
        when(userService.createUser(anyString(), anyString()))
            .thenThrow(new IllegalArgumentException("User exists"));
        
        mockMvc.perform(post("/api/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""
                    {
                        "email": "existing@example.com",
                        "name": "Test"
                    }
                    """))
            .andExpect(status().isBadRequest())
            .andExpect(content().string(containsString("User exists")));
    }
    
    /**
     * BENEFITS:
     * ✓ Fast (no server startup)
     * ✓ Tests request/response mapping
     * ✓ Tests validation
     * ✓ Tests error handling
     * 
     * LIMITATIONS:
     * ✗ Server-side only
     * ✗ No real HTTP
     * ✗ No serialization issues
     */
}

// RESTASSURED - CLIENT-SIDE TESTING
// ========================================

/**
 * Test with real HTTP requests
 */
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class UserControllerRestAssuredTest {
    
    @LocalServerPort
    private int port;
    
    @Autowired
    private UserRepository userRepository;
    
    @BeforeEach
    void setUp() {
        RestAssured.port = port;
        userRepository.deleteAll();
    }
    
    @Test
    void shouldCreateUser() {
        // Arrange
        String requestBody = """
            {
                "email": "test@example.com",
                "name": "Test User"
            }
            """;
        
        // Act & Assert
        given()
            .contentType(ContentType.JSON)
            .body(requestBody)
        .when()
            .post("/api/users")
        .then()
            .statusCode(201)
            .body("email", equalTo("test@example.com"))
            .body("name", equalTo("Test User"))
            .body("id", notNullValue());
    }
    
    @Test
    void shouldGetUser() {
        // Arrange - create user first
        User user = userRepository.save(new User("test@example.com", "Test"));
        
        // Act & Assert
        given()
            .pathParam("email", "test@example.com")
        .when()
            .get("/api/users/{email}")
        .then()
            .statusCode(200)
            .body("email", equalTo("test@example.com"))
            .body("name", equalTo("Test"));
    }
    
    @Test
    void shouldReturn404WhenUserNotFound() {
        given()
            .pathParam("email", "notfound@example.com")
        .when()
            .get("/api/users/{email}")
        .then()
            .statusCode(404);
```

```java
/**
 * Custom cleanup utility
 */
@Component
class DatabaseCleaner {
    
    @PersistenceContext
    private EntityManager entityManager;
    
    @Transactional
    public void cleanDatabase() {
        // Disable foreign key checks
        entityManager.createNativeQuery("SET REFERENTIAL_INTEGRITY FALSE").executeUpdate();
        
        // Get all table names
        List<String> tableNames = entityManager.getMetamodel().getEntities().stream()
            .map(entityType -> entityType.getName())
            .collect(Collectors.toList());
        
        // Truncate all tables
        for (String tableName : tableNames) {
            entityManager.createNativeQuery("TRUNCATE TABLE " + tableName).executeUpdate();
        }
        
        // Re-enable foreign key checks
        entityManager.createNativeQuery("SET REFERENTIAL_INTEGRITY TRUE").executeUpdate();
    }
}
```

**Key Takeaways:**

```
Test Slice Hierarchy:
@WebMvcTest    → Controllers (fast, mocked)
@DataJpaTest   → Repositories (H2, isolated)
@SpringBootTest → Full app (slow, comprehensive)

The Trade-Off:
Fast → H2, @WebMvcTest, Embedded
Realistic → Testcontainers, @SpringBootTest, Real

Testcontainers Pattern:
@Container
static PostgreSQLContainer postgres = ...

@DynamicPropertySource
static void props(DynamicPropertyRegistry registry) {
    registry.add("spring.datasource.url", postgres::getJdbcUrl);
}

Test Data Management:
1. Builders (flexible)
2. Fixtures (reusable)
3. SQL Scripts (complex data)
4. @BeforeEach cleanup (fresh state)
5. @Transactional (auto rollback)
```
