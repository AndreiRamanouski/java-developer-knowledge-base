# GraalVM and Native Images

## Overview

GraalVM represents a paradigm shift in Java deployment—instead of running on the JVM with Just-In-Time (JIT) compilation, GraalVM can compile Java applications ahead-of-time (AOT) into native executables that run without a JVM. This transformation delivers dramatic improvements: startup time drops from seconds to milliseconds, memory footprint shrinks by 5-10x, and applications become standalone binaries with no JVM dependency. However, native compilation imposes strict constraints—the closed-world assumption means no dynamic class loading, reflection requires explicit configuration, and frameworks must be designed for AOT compatibility. This guide covers how GraalVM's native image works (AOT compilation, static analysis, tree shaking), understanding constraints (no dynamic classloading, serialization limitations), configuring reflection and resources (JSON config files, tracing agent), measuring performance improvements (startup time, memory, throughput), GraalVM's polyglot capabilities (running JavaScript, Python, Ruby alongside Java), and real-world use cases (microservices, serverless functions, CLI tools, containers). Whether you're building cloud-native applications that need instant startup or creating command-line tools that feel native, GraalVM's native images offer a compelling alternative to traditional JVM deployment—if you can work within its constraints.

## 1. Ahead-of-Time Compilation

### From JIT to AOT

```java
/**
 * UNDERSTANDING AOT COMPILATION
 */

package com.example;

public class AotExplained {
    
    /**
     * Traditional JVM execution
     */
    public static void traditionalJvm() {
        /**
         * TRADITIONAL JVM (JIT):
         * 
         * 1. javac → .class files (bytecode)
         * 2. java command starts JVM
         * 3. JVM loads classes
         * 4. Interpreter executes bytecode
         * 5. HotSpot identifies hot paths
         * 6. JIT compiles to native code
         * 7. Optimized execution
         * 
         * 
         * PROBLEMS:
         * 
         * Slow startup:
         * - Load JVM
         * - Load classes
         * - Interpret bytecode
         * - Wait for JIT warmup
         * → Seconds to start
         * 
         * Large memory:
         * - JVM itself (~50-100MB)
         * - JIT compiler
         * - Code cache
         * - Class metadata
         * → 100-500MB baseline
         * 
         * Distribution:
         * - Requires JVM installed
         * - .jar files not standalone
         * - Version compatibility issues
         */
    }
    
    /**
     * GraalVM Native Image (AOT)
     */
    public static void nativeImage() {
        /**
         * GRAALVM NATIVE IMAGE (AOT):
         * 
         * 1. javac → .class files
         * 2. native-image analyzes ALL code
         * 3. Static analysis finds reachable code
         * 4. Compiles to native machine code
         * 5. Links with substrate VM
         * 6. Produces standalone executable
         * 
         * 
         * BENEFITS:
         * 
         * Fast startup:
         * - No JVM loading
         * - No class loading
         * - No JIT warmup
         * → Milliseconds to start
         * 
         * Small memory:
         * - No JVM overhead
         * - No JIT compiler
         * - Only reachable code
         * → 10-50MB for simple apps
         * 
         * Standalone:
         * - Single executable
         * - No JVM required
         * - OS-specific binary
         * 
         * 
         * BUILD PROCESS:
         * 
         * native-image \
         *   -jar myapp.jar \
         *   -H:Name=myapp \
         *   --no-fallback
         * 
         * Produces: myapp (native executable)
         * 
         * 
         * ARCHITECTURE:
         * 
         * Native Image =
         *   Compiled Java code
         *   + Substrate VM (minimal runtime)
         *   + Required native libraries
         */
    }
    
    /**
     * How static analysis works
     */
    public static void staticAnalysis() {
        /**
         * STATIC ANALYSIS:
         * 
         * 1. Start from entry points:
         *    - main() method
         *    - @Inject constructors
         *    - Framework hooks
         * 
         * 2. Build call graph:
         *    - Follow all method calls
         *    - Track field accesses
         *    - Find all reachable code
         * 
         * 3. Tree shaking:
         *    - Remove unreachable code
         *    - Dead code elimination
         *    - Unused classes dropped
         * 
         * 4. Heap snapshot:
         *    - Initialize static fields
         *    - Create constant objects
         *    - Embed in executable
         * 
         * 5. Compile:
         *    - All reachable methods
         *    - AOT to machine code
         *    - No dynamic compilation
         * 
         * 
         * RESULT:
         * 
         * Only code that CAN be called
         * is included in binary.
         * 
         * Unreachable code = removed
         * → Smaller binaries!
         */
    }
    
    /**
     * Substrate VM
     */
    public static void substrateVm() {
        /**
         * SUBSTRATE VM:
         * 
         * Minimal runtime for native images
         * 
         * Includes:
         * ✓ Memory management (GC)
         * ✓ Thread scheduling
         * ✓ Exception handling
         * ✓ Synchronization
         * 
         * Excludes:
         * ✗ Class loader
         * ✗ JIT compiler
         * ✗ Dynamic class loading
         * ✗ Many reflection features
         * 
         * 
         * GC OPTIONS:
         * 
         * Serial GC (default):
         * - Single-threaded
         * - Low overhead
         * - Good for small heaps
         * 
         * G1 GC:
         * - Multi-threaded
         * - Better for larger heaps
         * - Flag: --gc=G1
         * 
         * 
         * Much smaller than full JVM!
         */
    }
}
```

## 2. Native Image Constraints

### The Closed-World Assumption

```java
/**
 * NATIVE IMAGE CONSTRAINTS
 */

package com.example;

import java.lang.reflect.*;

public class NativeImageConstraints {
    
    /**
     * Constraint 1: No dynamic class loading
     */
    public static void noDynamicClassLoading() {
        /**
         * DYNAMIC CLASS LOADING FAILS:
         * 
         * // This works in JVM
         * Class<?> clazz = Class.forName("com.example.Plugin");
         * 
         * // This FAILS in native image!
         * // Class not found at build time
         * 
         * 
         * CLOSED-WORLD ASSUMPTION:
         * 
         * All code must be known at build time
         * Can't load classes at runtime
         * No URLClassLoader, no custom loaders
         * 
         * 
         * WORKAROUNDS:
         * 
         * 1. Include all classes at build
         * 2. Use explicit configuration
         * 3. Redesign to avoid dynamic loading
         */
    }
    
    /**
     * Constraint 2: Reflection requires configuration
     */
    public static void reflectionConstraints() throws Exception {
        /**
         * REFLECTION IN NATIVE IMAGE:
         * 
         * Code:
         * Class<?> clazz = Class.forName("com.example.User");
         * Object user = clazz.getDeclaredConstructor().newInstance();
         * 
         * Build time:
         * ✗ Fails without configuration!
         * 
         * Need: reflect-config.json
         * [
         *   {
         *     "name": "com.example.User",
         *     "allDeclaredConstructors": true,
         *     "allDeclaredMethods": true,
         *     "allDeclaredFields": true
         *   }
         * ]
         * 
         * 
         * WHY:
         * 
         * Static analysis can't see:
         * - Reflection calls
         * - What classes will be loaded
         * - What methods will be called
         * 
         * Must tell it explicitly!
         */
        
        // This needs configuration
        Class<?> userClass = Class.forName("com.example.User");
        Constructor<?> ctor = userClass.getDeclaredConstructor();
        Object instance = ctor.newInstance();
        
        Method setName = userClass.getMethod("setName", String.class);
        setName.invoke(instance, "Alice");
    }
    
    /**
     * Constraint 3: Resources need configuration
     */
    public static void resourceConstraints() {
        /**
         * RESOURCES IN NATIVE IMAGE:
         * 
         * Code:
         * InputStream stream = 
         *   getClass().getResourceAsStream("/config.properties");
         * 
         * Build time:
         * ✗ Resource not included!
         * 
         * Need: resource-config.json
         * {
         *   "resources": {
         *     "includes": [
         *       {"pattern": ".*\\.properties$"},
         *       {"pattern": "META-INF/.*"}
         *     ]
         *   }
         * }
         * 
         * 
         * IMPORTANT:
         * 
         * Resources embedded in binary
         * Must list explicitly or with patterns
         */
    }
    
    /**
     * Constraint 4: JNI requires configuration
     */
    public static void jniConstraints() {
        /**
         * JNI IN NATIVE IMAGE:
         * 
         * Native methods need configuration
         * 
         * jni-config.json:
         * [
         *   {
         *     "name": "com.example.NativeLib",
         *     "methods": [
         *       {"name": "nativeMethod", "parameterTypes": ["int"]}
         *     ]
         *   }
         * ]
         * 
         * Also need to link native library
         */
    }
    
    /**
     * Constraint 5: Serialization limitations
     */
    public static void serializationConstraints() {
        /**
         * SERIALIZATION:
         * 
         * Java serialization works but:
         * 
         * serialization-config.json:
         * [
         *   {
         *     "name": "com.example.User",
         *     "allDeclaredConstructors": true
         *   }
         * ]
         * 
         * Better: Use alternatives
         * - JSON (Jackson, Gson)
         * - Protocol Buffers
         * - MessagePack
         * 
         * These work better with AOT
         */
    }
    
    /**
     * Constraint 6: Proxy classes
     */
    public static void proxyConstraints() {
        /**
         * DYNAMIC PROXIES:
         * 
         * Proxy.newProxyInstance() needs config
         * 
         * proxy-config.json:
         * [
         *   {
         *     "interfaces": [
         *       "com.example.UserService",
         *       "java.io.Serializable"
         *     ]
         *   }
         * ]
         * 
         * Used by:
         * - JPA/Hibernate proxies
         * - Spring AOP
         * - Mockito
         * 
         * Framework support required!
         */
    }
    
    /**
     * Constraint 7: Thread limits
     */
    public static void threadConstraints() {
        /**
         * THREADING:
         * 
         * Most threading works:
         * ✓ Thread creation
         * ✓ Synchronization
         * ✓ ExecutorService
         * ✓ Virtual threads (Java 21+)
         * 
         * Some limitations:
         * ✗ ThreadGroup manipulation
         * ✗ Some Thread methods
         * 
         * Generally works well!
         */
    }
    
    /**
     * Constraint 8: No dynamic code generation
     */
    public static void noCodeGeneration() {
        /**
         * CODE GENERATION:
         * 
         * These DON'T work:
         * ✗ ByteBuddy at runtime
         * ✗ CGlib at runtime
         * ✗ ASM at runtime
         * ✗ Groovy scripts at runtime
         * 
         * Workarounds:
         * ✓ Generate code at build time
         * ✓ Use build-time plugins
         * ✓ Pre-compile everything
         * 
         * AOT = Everything ahead of time!
         */
    }
}
```

## 3. Reflection Configuration

### Making Reflection Work

```java
/**
 * EXAMPLE APPLICATION
 */

package com.example;

// User class with reflection
public class User {
    private String name;
    private int age;
    
    public User() {}
    
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

// Code using reflection
public class ReflectionExample {
    public static void main(String[] args) throws Exception {
        // Create via reflection
        Class<?> userClass = Class.forName("com.example.User");
        Object user = userClass.getDeclaredConstructor().newInstance();
        
        // Set field via reflection
        Method setName = userClass.getMethod("setName", String.class);
        setName.invoke(user, "Alice");
        
        Method setAge = userClass.getMethod("setAge", int.class);
        setAge.invoke(user, 30);
        
        // Get fields via reflection
        Method getName = userClass.getMethod("getName");
        Method getAge = userClass.getMethod("getAge");
        
        System.out.println("Name: " + getName.invoke(user));
        System.out.println("Age: " + getAge.invoke(user));
    }
}
```

```java
/**
 * PROGRAMMATIC CONFIGURATION
 */

import org.graalvm.nativeimage.hosted.Feature;
import org.graalvm.nativeimage.hosted.RuntimeReflection;

public class ReflectionFeature implements Feature {
    
    @Override
    public void beforeAnalysis(BeforeAnalysisAccess access) {
        try {
            // Register class for reflection
            Class<?> userClass = Class.forName("com.example.User");
            
            RuntimeReflection.register(userClass);
            RuntimeReflection.registerForReflectiveInstantiation(userClass);
            
            // Register all methods
            for (Method method : userClass.getDeclaredMethods()) {
                RuntimeReflection.register(method);
            }
            
            // Register all fields
            for (Field field : userClass.getDeclaredFields()) {
                RuntimeReflection.register(field);
            }
            
        } catch (ClassNotFoundException e) {
            throw new RuntimeException(e);
        }
    }
}
```

## 5. Polyglot Capabilities

### Running Multiple Languages on GraalVM

```java
/**
 * GRAALVM POLYGLOT
 */

package com.example;

import org.graalvm.polyglot.*;

public class PolyglotExample {
    
    /**
     * Running JavaScript
     */
    public static void runJavaScript() {
        System.out.println("=== JavaScript ===");
        
        try (Context context = Context.create("js")) {
            // Execute JavaScript
            Value result = context.eval("js", "2 + 2");
            System.out.println("2 + 2 = " + result.asInt());
            
            // Define function
            context.eval("js", """
                function greet(name) {
                    return 'Hello, ' + name + '!';
                }
            """);
            
            // Call function
            Value greet = context.getBindings("js").getMember("greet");
            String greeting = greet.execute("Alice").asString();
            System.out.println(greeting);
        }
        
        /**
         * JAVASCRIPT ON GRAALVM:
         * 
         * ✓ Full ES2021+ support
         * ✓ Fast execution (JIT compiled)
         * ✓ Node.js compatibility
         * ✓ NPM modules work
         */
    }
    
    /**
     * Running Python
     */
    public static void runPython() {
        System.out.println("\n=== Python ===");
        
        try (Context context = Context.create("python")) {
            // Execute Python
            Value result = context.eval("python", """
                def factorial(n):
                    if n <= 1:
                        return 1
                    return n * factorial(n - 1)
                
                factorial(5)
            """);
            
            System.out.println("5! = " + result.asInt());
        }
        
        /**
         * PYTHON ON GRAALVM:
         * 
         * ✓ Python 3.8+ compatibility
         * ✓ Most standard library
         * ✗ Some C extensions don't work
         */
    }
    
    /**
     * Running Ruby
     */
    public static void runRuby() {
        System.out.println("\n=== Ruby ===");
        
        try (Context context = Context.create("ruby")) {
            Value result = context.eval("ruby", """
                class Person
                  def initialize(name)
                    @name = name
                  end
                  
                  def greet
                    "Hello, I'm #{@name}"
                  end
                end
                
                person = Person.new("Bob")
                person.greet
            """);
            
            System.out.println(result.asString());
        }
        
        /**
         * RUBY ON GRAALVM:
         * 
         * ✓ Ruby 3.0+ compatibility
         * ✓ Rails works
         * ✓ Good performance
         */
    }
    
    /**
     * Language interop
     */
    public static void languageInterop() {
        System.out.println("\n=== Language Interop ===");
        
        try (Context context = Context.newBuilder()
                .allowAllAccess(true)
                .build()) {
            
            // Define JavaScript function
            context.eval("js", """
                function processArray(arr) {
                    return arr.map(x => x * 2);
                }
            """);
            
            // Call from Java
            Value jsFunction = context.getBindings("js").getMember("processArray");
            Value jsArray = context.eval("js", "[1, 2, 3, 4, 5]");
            Value result = jsFunction.execute(jsArray);
            
            System.out.println("Result: " + result);
            
            /**
             * POLYGLOT INTEROP:
             * 
             * ✓ Share objects between languages
             * ✓ Call functions across languages
             * ✓ Type conversion automatic
             * ✓ High performance
             */
        }
    }
    
    /**
     * Sharing Java objects
     */
    public static void sharingJavaObjects() {
        System.out.println("\n=== Sharing Java Objects ===");
        
        try (Context context = Context.newBuilder()
                .allowAllAccess(true)
                .build()) {
            
            // Create Java object
            java.util.List<String> javaList = new java.util.ArrayList<>();
            javaList.add("Alice");
            javaList.add("Bob");
            
            // Share with JavaScript
            context.getBindings("js").putMember("javaList", javaList);
            
            // Use in JavaScript
            context.eval("js", """
                javaList.add("Charlie");
                console.log("From JS: " + javaList.size());
            """);
            
            System.out.println("From Java: " + javaList);
            
            /**
             * SHARED OBJECTS:
             * 
             * Same object in both languages!
             * Changes visible everywhere
             * High-performance sharing
             */
        }
    }
    
    public static void main(String[] args) {
        runJavaScript();
        runPython();
        runRuby();
        languageInterop();
        sharingJavaObjects();
    }
}
```
