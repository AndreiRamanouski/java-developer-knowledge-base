# Scala Interoperability

## Overview

Scala's interoperability with Java is fundamental to its design—Scala compiles to JVM bytecode and can seamlessly call Java code and be called from Java. However, Scala's advanced features (case classes, traits, implicits, higher-kinded types, functional patterns) create interop challenges that don't exist with simpler JVM languages. This guide covers the complete Scala-Java interop story: how primitive and object types map between languages, accessing Scala case classes from Java (they're just classes with generated methods), understanding how Scala traits compile to Java interfaces (with potential default methods), converting between Scala and Java collections (using JavaConverters), dealing with implicits (they're invisible to Java), and adopting Scala's functional patterns in Java code (Option vs null, Either for error handling, immutable collections). Unlike Kotlin which was designed for Java interop, Scala prioritizes language features over Java compatibility, so interop requires more explicit conversions and understanding of compilation details. Whether you're calling Scala libraries from Java, exposing Scala APIs to Java consumers, or maintaining a mixed codebase, this guide provides the patterns and pitfalls to navigate the boundary.

## 1. Java-Scala Type Mapping

### How Types Convert Between Languages

```scala
/**
 * SCALA TYPES VISIBLE TO JAVA
 */

package com.example

// Primitive types
class ScalaPrimitives {
  val intVal: Int = 42
  val longVal: Long = 42L
  val doubleVal: Double = 3.14
  val booleanVal: Boolean = true
  val charVal: Char = 'A'
  
  def processInt(x: Int): Int = x * 2
  def processString(s: String): String = s.toUpperCase
}

// Object types
class ScalaObjectTypes {
  val string: String = "Hello"
  val list: List[Int] = List(1, 2, 3)
  val option: Option[String] = Some("value")
  val either: Either[String, Int] = Right(42)
}

// Unit type
class ScalaUnit {
  def doSomething(): Unit = {
    println("Doing something")
  }
  
  def returnNothing(): Unit = ()
}

// Nothing and Null
class ScalaSpecialTypes {
  def throwException(): Nothing = {
    throw new RuntimeException("Error")
  }
  
  def returnNull: String = null
}
```

```java
/**
 * JAVA CALLING SCALA TYPES
 */

package com.example;

import scala.Option;
import scala.Some;
import scala.collection.immutable.List;

public class JavaCallingScalaTypes {
    
    /**
     * Primitive type mapping
     */
    public static void primitiveTypes() {
        System.out.println("=== Primitive Types ===");
        
        ScalaPrimitives obj = new ScalaPrimitives();
        
        // Scala primitives → Java primitives
        int i = obj.intVal();        // Int → int
        long l = obj.longVal();      // Long → long
        double d = obj.doubleVal();  // Double → double
        boolean b = obj.booleanVal(); // Boolean → boolean
        char c = obj.charVal();      // Char → char
        
        // Methods work naturally
        int result = obj.processInt(10);
        String upper = obj.processString("hello");
        
        System.out.println("Int: " + i + ", Result: " + result);
        
        /**
         * SCALA → JAVA PRIMITIVES:
         * 
         * Scala Type → Java Type
         * Int        → int
         * Long       → long
         * Double     → double
         * Float      → float
         * Boolean    → boolean
         * Char       → char
         * Byte       → byte
         * Short      → short
         * 
         * Direct mapping, no wrapper!
         */
    }
    
    /**
     * Object type mapping
     */
    public static void objectTypes() {
        System.out.println("\n=== Object Types ===");
        
        ScalaObjectTypes obj = new ScalaObjectTypes();
        
        // String is the same
        String str = obj.string();
        
        // Scala collections are Scala types in Java
        scala.collection.immutable.List<Object> list = obj.list();
        
        // Option is Scala type in Java
        scala.Option<String> option = obj.option();
        
        // Either is Scala type in Java
        scala.util.Either<String, Object> either = obj.either();
        
        /**
         * SCALA → JAVA OBJECTS:
         * 
         * String → String (same)
         * List → scala.collection.immutable.List
         * Option → scala.Option
         * Either → scala.util.Either
         * 
         * Scala types remain Scala types!
         * Need explicit conversion to Java types
         */
    }
    
    /**
     * Unit type
     */
    public static void unitType() {
        System.out.println("\n=== Unit Type ===");
        
        ScalaUnit obj = new ScalaUnit();
        
        // Unit → void in Java
        obj.doSomething();
        
        /**
         * UNIT → JAVA:
         * 
         * Scala Unit → Java void
         * 
         * In signatures only!
         * Scala Unit is actually an object
         */
    }
    
    /**
     * Special types
     */
    public static void specialTypes() {
        /**
         * SPECIAL TYPES:
         * 
         * Nothing:
         * - Bottom type (subtype of everything)
         * - Methods that never return
         * - throw, infinite loop
         * - Java sees return type but method never returns
         * 
         * Null:
         * - Type of null
         * - Subtype of all reference types
         * - Java just sees null
         * 
         * Any:
         * - Top type (supertype of everything)
         * - Java sees Object
         * 
         * AnyVal:
         * - Parent of value types
         * - Java sees primitive or wrapper
         * 
         * AnyRef:
         * - Parent of reference types
         * - Java sees Object
         */
    }
    
    public static void main(String[] args) {
        primitiveTypes();
        objectTypes();
        unitType();
    }
}
```

```scala
    val result = javaPrimitives.add(5, 10)

     * 
     * Java can return null
     * Scala should handle it:
     * 
     * // Bad: Assume non-null
     * val str: String = javaMethod()
     * 
     * // Good: Option
     * val str: Option[String] = Option(javaMethod())
     * 
     * // Or check explicitly
     * val result = javaMethod() match {
     *   case null => "default"
     *   case s => s.toUpperCase
     * }
     */
  }
}
```

## 2. Case Classes in Java

### Accessing Scala's Data Classes

```scala
/**
 * SCALA CASE CLASSES
 */

package com.example

// Simple case class
case class Person(name: String, age: Int)

// Case class with default parameters
case class User(
  id: String,
  email: String,
  verified: Boolean = false
)

// Case class with methods
case class Point(x: Double, y: Double) {
  def distance(other: Point): Double = {
    math.sqrt(
      math.pow(other.x - x, 2) + math.pow(other.y - y, 2)
    )
  }
}

// Case class with companion object
case class Product(id: String, name: String, price: Double)

object Product {
  def create(id: String, name: String): Product = {
    Product(id, name, 0.0)
  }
}
```

```java
/**
 * JAVA CALLING CASE CLASSES
 */

package com.example;

public class JavaCallingCaseClasses {
    
    /**
     * Creating case class instances
     */
    public static void creatingInstances() {
        System.out.println("=== Creating Case Classes ===");
        
        // Constructor is public
        Person person = new Person("Alice", 30);
        
        // Default parameters - must provide in Java
        User user1 = new User("123", "alice@example.com", true);
        
        // No overloaded constructor for defaults!
        // Must use apply() method
        User user2 = User.apply("456", "bob@example.com", false);
        
        System.out.println("Person: " + person);
        System.out.println("User: " + user1);
        
        /**
         * CASE CLASS CONSTRUCTION:
         * 
         * From Java:
         * - new CaseClass(args) works
         * - Default parameters not available
         * - Use apply() for defaults (verbose)
         */
    }
    
    /**
     * Accessing fields
     */
    public static void accessingFields() {
        System.out.println("\n=== Accessing Fields ===");
        
        Person person = new Person("Bob", 25);
        
        // Fields are public final
        String name = person.name();
        int age = person.age();
        
        // No setters - immutable!
        // person.age() = 26;  // Error
        
        System.out.println("Name: " + name + ", Age: " + age);
        
        /**
         * CASE CLASS FIELDS:
         * 
         * Scala val → public final field
         * Accessed via method: person.name()
         * 
         * Immutable - no setters!
         */
    }
    
    /**
     * Generated methods
     */
    public static void generatedMethods() {
        System.out.println("\n=== Generated Methods ===");
        
        Person p1 = new Person("Alice", 30);
        Person p2 = new Person("Alice", 30);
        Person p3 = new Person("Bob", 25);
        
        // toString
        String str = p1.toString();
        System.out.println("toString: " + str);
        
        // equals
        boolean equals1 = p1.equals(p2);  // true
        boolean equals2 = p1.equals(p3);  // false
        System.out.println("equals: " + equals1 + ", " + equals2);
        
        // hashCode
        int hash1 = p1.hashCode();
        int hash2 = p2.hashCode();
        System.out.println("hashCode: " + hash1 + " == " + hash2);
        
        /**
         * GENERATED METHODS:
         * 
         * Case classes automatically get:
         * ✓ toString() - nice string representation
         * ✓ equals() - structural equality
         * ✓ hashCode() - consistent with equals
         * ✓ copy() - immutable updates
         * 
         * Just like Java records!
         */
    }
    
    /**
     * Copy method
     */
    public static void copyMethod() {
        System.out.println("\n=== Copy Method ===");
        
        Person original = new Person("Alice", 30);
        
        // copy() method available but awkward in Java
        // Must use named parameters via Scala syntax
        
        // This doesn't work in Java:
        // Person updated = original.copy(age = 31);
        
        // Must create new instance instead
        Person updated = new Person(original.name(), 31);
        
        System.out.println("Original: " + original);
        System.out.println("Updated: " + updated);
        
        /**
         * COPY METHOD:
         * 
         * Scala copy() uses named parameters
         * Not accessible from Java easily
         * 
         * Workaround: Create new instance
         */
    }
    
    /**
     * Pattern matching methods
     */
    public static void patternMatchingMethods() {
        System.out.println("\n=== Pattern Matching ===");
        
        Person person = new Person("Charlie", 35);
        
        // unapply exists but not useful in Java
        // It's for Scala pattern matching
        
        // _1(), _2() methods for tuples
        // Not generated for case classes
        
        /**
         * PATTERN MATCHING:
         * 
         * Case classes generate:
         * - unapply() method
         * - For Scala pattern matching
         * - Not useful from Java
         */
    }
    
    /**
     * Companion object
     */
    public static void companionObject() {
        System.out.println("\n=== Companion Object ===");
        
        // Companion object → MODULE$
        Product product1 = Product.apply("P1", "Widget", 19.99);
        
        // Custom factory method
        Product product2 = Product.create("P2", "Gadget");
        
        System.out.println("Product 1: " + product1);
        System.out.println("Product 2: " + product2);
        
        /**
         * COMPANION OBJECT:
         * 
         * Companion methods available as static
         * - apply() for construction
         * - Custom methods
         * 
         * Access via ClassName.method()
         */
    }
    
    public static void main(String[] args) {
        creatingInstances();
        accessingFields();
        generatedMethods();
        copyMethod();
        companionObject();
    }
}
```

## 3. Traits vs Interfaces

### Understanding Scala's Trait System

```scala
/**
 * SCALA TRAITS
 */

package com.example

// Simple trait (like interface)
trait Greeter {
  def greet(name: String): String
}

// Trait with default implementation
trait Logger {
  def log(message: String): Unit = {
    println(s"LOG: $message")
  }
  
  def error(message: String): Unit = {
    println(s"ERROR: $message")
  }
}

// Trait with abstract and concrete members
trait Repository {
  // Abstract
  def save(data: String): Unit
  def load(id: String): String
  
  // Concrete
  def saveAll(items: List[String]): Unit = {
    items.foreach(save)
  }
}

// Trait with fields
trait HasName {
  val name: String
  
  def printName(): Unit = {
    println(s"Name: $name")
  }
}

// Multiple traits
trait Readable {
  def read(): String
}

trait Writable {
  def write(data: String): Unit
}

class FileHandler extends Readable with Writable with Logger {
  override def read(): String = "file contents"
  
  override def write(data: String): Unit = {
    log(s"Writing: $data")
  }
}
```

```java
/**
 * JAVA CALLING SCALA TRAITS
 */

package com.example;

public class JavaCallingTraits {
    
    /**
     * Traits as interfaces
     */
    public static void traitsAsInterfaces() {
        System.out.println("=== Traits as Interfaces ===");
        
        // Can implement Scala trait from Java
        class MyGreeter implements Greeter {
            @Override
            public String greet(String name) {
                return "Hello, " + name;
            }
        }
        
        Greeter greeter = new MyGreeter();
        String greeting = greeter.greet("Alice");
        System.out.println(greeting);
        
        /**
         * SIMPLE TRAITS → INTERFACES:
         * 
         * Traits with only abstract methods
         * → Java interfaces
         * 
         * Can implement from Java
         */
    }
    
    /**
     * Traits with default implementations
     */
    public static void traitsWithDefaults() {
        System.out.println("\n=== Traits with Defaults ===");
        
        // Can use default implementations
        class MyLogger implements Logger {
            // Inherits log() and error()
        }
        
        Logger logger = new MyLogger();
        logger.log("Hello");
        logger.error("Oops");
        
        // Can override if needed
        class CustomLogger implements Logger {
            @Override
            public void log(String message) {
                System.out.println("CUSTOM: " + message);
            }
        }
        
        /**
         * TRAITS WITH DEFAULTS → INTERFACE + DEFAULT METHODS:
         * 
         * Scala trait default → Java default method
         * Works in Java 8+
         */
    }
    
    /**
     * Traits with fields
     */
    public static void traitsWithFields() {
        /**
         * TRAITS WITH FIELDS:
         * 
         * Scala trait with val/var
         * → Interface + getter/setter methods
         * 
         * From Java:
         * - Must implement abstract field
         * - Accessed via method
         * 
         * Example:
         * trait HasName {
         *   val name: String
         * }
         * 
         * Java sees:
         * String name();
         */
    }
    
    /**
     * Multiple traits
     */
    public static void multipleTraits() {
        System.out.println("\n=== Multiple Traits ===");
        
        FileHandler handler = new FileHandler();
        
        // Can use as any trait type
        Readable readable = handler;
        Writable writable = handler;
        Logger logger = handler;
        
        // Call methods
        String content = readable.read();
        writable.write("data");
        logger.log("Done");
        
        System.out.println("Content: " + content);
        
        /**
         * MULTIPLE TRAITS:
         * 
         * Scala: extends Trait1 with Trait2 with Trait3
         * Java: implements Interface1, Interface2, Interface3
         * 
         * Works naturally!
         */
    }
    
    /**
     * Limitations
     */
    public static void limitations() {
        /**
         * LIMITATIONS:
         * 
         * 1. Can't create Scala trait from Java:
         *    - Scala traits can have state
         *    - Compilation is complex
         *    - Java can only implement
         * 
         * 2. Self-types don't work:
         *    trait T { this: OtherTrait => }
         *    Java can't express this
         * 
         * 3. Path-dependent types:
         *    Advanced Scala feature
         *    Not accessible from Java
         * 
         * 4. Constructor parameters in traits:
         *    trait T(val x: Int)
         *    Scala 3 feature, Java can't use
         */
    }
    
    public static void main(String[] args) {
        traitsAsInterfaces();
        traitsWithDefaults();
        multipleTraits();
    }
}
```

## 4. Scala Collections in Java

### Converting Between Collection Systems

```scala
/**
 * SCALA COLLECTIONS
 */

package com.example

import scala.jdk.CollectionConverters._

object ScalaCollections {
  
  // Scala collections
  val scalaList: List[String] = List("a", "b", "c")
  val scalaSet: Set[Int] = Set(1, 2, 3)
  val scalaMap: Map[String, Int] = Map("a" -> 1, "b" -> 2)
  
  // Converting to Java
  def toJavaCollections(): Unit = {
    println("=== Scala → Java ===")
    
    // List
    val javaList: java.util.List[String] = scalaList.asJava
    
    // Set
    val javaSet: java.util.Set[Int] = scalaSet.asJava
    
    // Map
    val javaMap: java.util.Map[String, Int] = scalaMap.asJava
    
    println(s"Java List: $javaList")
    println(s"Java Set: $javaSet")
    println(s"Java Map: $javaMap")
  }
  
  // Converting from Java
  def fromJavaCollections(
    javaList: java.util.List[String],
    javaSet: java.util.Set[Int],
    javaMap: java.util.Map[String, Int]
  ): Unit = {
    println("\n=== Java → Scala ===")
    
    // To Scala
    val scalaList: List[String] = javaList.asScala.toList
    val scalaSet: Set[Int] = javaSet.asScala.toSet
    val scalaMap: Map[String, Int] = javaMap.asScala.toMap
    
    println(s"Scala List: $scalaList")
    println(s"Scala Set: $scalaSet")
    println(s"Scala Map: $scalaMap")
  }
  
  // Mutable collections
  def mutableCollections(): Unit = {
    import scala.collection.mutable
    
    val mutableList = mutable.ListBuffer(1, 2, 3)
    val javaList: java.util.List[Int] = mutableList.asJava
    
    // Changes are visible!
    javaList.add(4)
    println(s"After add: $mutableList")  // [1, 2, 3, 4]
  }
}
```

```java
/**
 * JAVA CALLING SCALA COLLECTIONS
 */

package com.example;

import scala.collection.immutable.List;
import scala.collection.immutable.Set;
import scala.collection.immutable.Map;
import scala.jdk.javaapi.CollectionConverters;

public class JavaCallingScalaCollections {
    
    /**
     * Using Scala collections from Java
     */
    public static void usingScalaCollections() {
        System.out.println("=== Using Scala Collections ===");
        
        // Scala List from Java - awkward!
        List<String> scalaList = List.of("a", "b", "c");
        
        // Iterate
        scala.collection.Iterator<String> iter = scalaList.iterator();
        while (iter.hasNext()) {
            System.out.println(iter.next());
        }
        
        // Size
        int size = scalaList.size();
        System.out.println("Size: " + size);
        
        /**
         * SCALA COLLECTIONS FROM JAVA:
         * 
         * Awkward to use directly
         * Different API than Java collections
         * Should convert instead
         */
    }
    
    /**
     * Converting Scala → Java
     */
    public static void convertingToJava() {
        System.out.println("\n=== Converting Scala → Java ===");
        
        // Scala list
        List<String> scalaList = List.of("a", "b", "c");
        
        // Convert to Java
        java.util.List<String> javaList = 
            CollectionConverters.asJava(scalaList);
        
        // Now can use Java API
        javaList.forEach(System.out::println);
        int size = javaList.size();
        boolean contains = javaList.contains("b");
        
        System.out.println("Size: " + size + ", Contains 'b': " + contains);
        
        /**
         * CONVERSION:
         * 
         * Use CollectionConverters.asJava()
         * Creates Java view
         * Much more natural!
         */
    }
    
    /**
     * Converting Java → Scala
     */
    public static void convertingToScala() {
        System.out.println("\n=== Converting Java → Scala ===");
        
        // Java list
        java.util.List<String> javaList = 
            java.util.Arrays.asList("x", "y", "z");
        
        // Convert to Scala
        scala.collection.immutable.List<String> scalaList = 
            CollectionConverters.asScala(javaList).toList();
        
        System.out.println("Scala list: " + scalaList);
        
        /**
         * CONVERSION:
         * 
         * Use CollectionConverters.asScala()
         * Then .toList() to get immutable
         */
    }
    
    /**
     * Best practices
     */
    public static void bestPractices() {
        /**
         * BEST PRACTICES:
         * 
         * 1. Convert at boundary:
         *    - Scala API → return Java collections
         *    - Java calling Scala → convert immediately
         * 
         * 2. Use Java collections in mixed code:
         *    - More familiar to Java devs
         *    - Better IDE support
         *    - Easier debugging
         * 
         * 3. Document conversion needs:
         *    - Make it clear what needs conversion
         *    - Provide conversion helpers
         * 
         * 4. Watch for mutability:
         *    - Scala collections immutable by default
         *    - Java collections mutable
         *    - Conversions create views (changes visible!)
         */
    }
    
    public static void main(String[] args) {
        usingScalaCollections();
        convertingToJava();
        convertingToScala();
    }
}
```

## 5. Implicits and Java

### The Invisible Parameters

```scala
/**
 * SCALA IMPLICITS
 */

package com.example

// Implicit parameter
class Formatter {
  def format(value: Int)(implicit prefix: String): String = {
    s"$prefix$value"
  }
}

object Formatter {
  implicit val defaultPrefix: String = "Value: "
}

// Implicit conversion
object Implicits {
  implicit def intToString(x: Int): String = x.toString
  
  implicit class RichInt(val x: Int) extends AnyVal {
    def times(f: => Unit): Unit = {
      (1 to x).foreach(_ => f)
    }
  }
}

// Type class pattern
trait Show[T] {
  def show(value: T): String
}

object Show {
  implicit val intShow: Show[Int] = new Show[Int] {
    def show(value: Int) = s"Int($value)"
  }
  
  implicit val stringShow: Show[String] = new Show[String] {
    def show(value: String) = s"String($value)"
  }
  
  def show[T](value: T)(implicit s: Show[T]): String = {
    s.show(value)
  }
}
```

```java
/**
 * JAVA AND IMPLICITS
 */

package com.example;

public class JavaAndImplicits {
    
    /**
     * Implicit parameters
     */
    public static void implicitParameters() {
        System.out.println("=== Implicit Parameters ===");
        
        Formatter formatter = new Formatter();
        
        // Java must pass implicitly explicitly!
        String result = formatter.format(42, Formatter.defaultPrefix());
        
        System.out.println("Result: " + result);
        
        /**
         * IMPLICIT PARAMETERS FROM JAVA:
         * 
         * Scala:
         * def format(x: Int)(implicit prefix: String)
         * 
         * Java sees:
         * String format(int x, String prefix)
         * 
         * Must pass explicitly!
         * No implicit resolution in Java
         */
    }
    
    /**
     * Implicit conversions
     */
    public static void implicitConversions() {
        /**
         * IMPLICIT CONVERSIONS FROM JAVA:
         * 
         * Scala implicit conversions don't work in Java!
         * 
         * Scala:
         * implicit def intToString(x: Int): String
         * 
         * In Scala can:
         * val s: String = 42  // Auto-converts
         * 
         * In Java:
         * String s = 42;  // Error!
         * 
         * Must call conversion manually:
         * String s = Implicits.intToString(42);
         */
    }
    
    /**
     * Implicit classes
     */
    public static void implicitClasses() {
        /**
         * IMPLICIT CLASSES FROM JAVA:
         * 
         * Scala:
         * implicit class RichInt(val x: Int) {
         *   def times(f: => Unit): Unit
         * }
         * 
         * In Scala can:
         * 5.times { println("Hello") }
         * 
         * In Java:
         * - Extension methods don't work
         * - Must create wrapper manually
         * 
         * RichInt wrapper = new RichInt(5);
         * wrapper.times(() -> System.out.println("Hello"));
         */
    }
    
    /**
     * Type classes
     */
    public static void typeClasses() {
        System.out.println("\n=== Type Classes ===");
        
        // Type class instances exist
        Show<Integer> intShow = Show.intShow();
        Show<String> stringShow = Show.stringShow();
        
        // Must pass explicitly
        String result1 = Show.show(42, intShow);
        String result2 = Show.show("Hello", stringShow);
        
        System.out.println(result1);
        System.out.println(result2);
        
        /**
         * TYPE CLASSES FROM JAVA:
         * 
         * Can access instances
         * Must pass explicitly
         * No implicit resolution
         */
    }
    
    /**
     * Summary
     */
    public static void summary() {
        /**
         * IMPLICITS FROM JAVA:
         * 
         * All implicit features invisible/manual:
         * ✗ Implicit parameters → must pass
         * ✗ Implicit conversions → must call
         * ✗ Implicit classes → manual wrapper
         * ✗ Type classes → manual instances
         * 
         * Java sees:
         * - Extra parameters
         * - Regular methods
         * - Regular classes
         * 
         * No magic, just explicit!
         */
    }
    
    public static void main(String[] args) {
        implicitParameters();
        typeClasses();
    }
}
```

## 6. Functional Programming Patterns

### Adopting Scala's FP in Java

```scala
/**
 * SCALA FUNCTIONAL PATTERNS
 */

package com.example

object FunctionalPatterns {
  
  // Option instead of null
  def findUser(id: String): Option[String] = {
    if (id == "123") Some("Alice")
    else None
  }
  
  // Either for error handling
  def divide(a: Int, b: Int): Either[String, Int] = {
    if (b == 0) Left("Division by zero")
    else Right(a / b)
  }
  
  // Try for exceptions
  def parseNumber(s: String): scala.util.Try[Int] = {
    scala.util.Try(s.toInt)
  }
  
  // Immutable data
  case class User(name: String, age: Int) {
    def withAge(newAge: Int): User = copy(age = newAge)
  }
  
  // Higher-order functions
  def process[A, B](value: A, f: A => B): B = f(value)
  
  def retry[A](times: Int)(f: => A): A = {
    var attempt = 0
    var lastException: Throwable = null
    
    while (attempt < times) {
      try {
        return f
      } catch {
        case e: Throwable =>
          lastException = e
          attempt += 1
      }
    }
    
    throw lastException
  }
}
```

```java
/**
 * JAVA USING SCALA FUNCTIONAL PATTERNS
 */

package com.example;

import scala.Option;
import scala.Some;
import scala.None$;
import scala.util.Either;
import scala.util.Left;
import scala.util.Right;

import java.util.Optional;

public class JavaUsingScalaFP {
    
    /**
     * Option vs null
     */
    public static void optionPattern() {
        System.out.println("=== Option Pattern ===");
        
        // Scala Option
        Option<String> user1 = FunctionalPatterns.findUser("123");
        Option<String> user2 = FunctionalPatterns.findUser("456");
        
        // Check if present
        if (user1.isDefined()) {
            System.out.println("User: " + user1.get());
        }
        
        if (user2.isEmpty()) {
            System.out.println("User not found");
        }
        
        // Convert to Java Optional
        Optional<String> optional1 = user1.isEmpty() 
            ? Optional.empty() 
            : Optional.of(user1.get());
        
        optional1.ifPresent(u -> System.out.println("Found: " + u));
        
        /**
         * SCALA OPTION FROM JAVA:
         * 
         * Methods:
         * - isDefined() → has value
         * - isEmpty() → no value
         * - get() → retrieve value (throws if empty!)
         * 
         * Better: Convert to Java Optional
         */
    }
    
    /**
     * Either for error handling
     */
    public static void eitherPattern() {
        System.out.println("\n=== Either Pattern ===");
        
        Either<String, Integer> result1 = FunctionalPatterns.divide(10, 2);
        Either<String, Integer> result2 = FunctionalPatterns.divide(10, 0);
        
        // Check which side
        if (result1.isRight()) {
            System.out.println("Success: " + result1.right().get());
        }
        
        if (result2.isLeft()) {
            System.out.println("Error: " + result2.left().get());
        }
        
        /**
         * SCALA EITHER FROM JAVA:
         * 
         * Methods:
         * - isLeft() → error
         * - isRight() → success
         * - left().get() → get error
         * - right().get() → get value
         * 
         * Convention: Left = error, Right = success
         */
    }
    
    /**
     * Try for exceptions
     */
    public static void tryPattern() {
        System.out.println("\n=== Try Pattern ===");
        
        scala.util.Try<Integer> try1 = FunctionalPatterns.parseNumber("42");
        scala.util.Try<Integer> try2 = FunctionalPatterns.parseNumber("abc");
        
        // Check success
        if (try1.isSuccess()) {
            System.out.println("Parsed: " + try1.get());
        }
        
        if (try2.isFailure()) {
            System.out.println("Parse error: " + try2.failed().get());
        }
        
        /**
         * SCALA TRY FROM JAVA:
         * 
         * Methods:
         * - isSuccess() → succeeded
         * - isFailure() → failed
         * - get() → get value (throws if failed!)
         * - failed().get() → get exception
         * 
         * Like Either but for exceptions
         */
    }
    
    /**
     * Immutable updates
     */
    public static void immutableUpdates() {
        System.out.println("\n=== Immutable Updates ===");
        
        FunctionalPatterns.User user = new FunctionalPatterns.User("Alice", 30);
        
        // Immutable update
        FunctionalPatterns.User updated = user.withAge(31);
        
        System.out.println("Original: " + user);
        System.out.println("Updated: " + updated);
        
        /**
         * IMMUTABILITY:
         * 
         * Scala favors immutability
         * Updates create new instances
         * 
         * Java should adopt:
         * - Use records (immutable)
         * - Wither methods (Java 14+)
         * - Avoid mutation
         */
    }
    
    /**
     * Higher-order functions
     */
    public static void higherOrderFunctions() {
        System.out.println("\n=== Higher-Order Functions ===");
        
        // Pass function to Scala
        Integer result = FunctionalPatterns.process(
            42,
            x -> x * 2
        );
        
        System.out.println("Result: " + result);
        
        // Scala functions work with Java lambdas!
        
        /**
         * HIGHER-ORDER FUNCTIONS:
         * 
         * Scala functions → Java Function
         * Works with lambdas
         * Natural interop!
         */
    }
    
    /**
     * Converting patterns to Java
     */
    public static void convertingToJava() {
        /**
         * SCALA → JAVA PATTERNS:
         * 
         * Option → Optional
         * Option<T> opt → Optional.ofNullable(opt.getOrElse(null))
         * 
         * Either → Custom result type
         * sealed interface Result<L, R> {
         *   record Left<L, R>(L value) implements Result<L, R> {}
         *   record Right<L, R>(R value) implements Result<L, R> {}
         * }
         * 
         * Try → Optional or CompletableFuture
         * 
         * Immutable → records + withers
         */
    }
    
    public static void main(String[] args) {
        optionPattern();
        eitherPattern();
        tryPattern();
        immutableUpdates();
        higherOrderFunctions();
    }
}
```

## 7. Best Practices

### Writing Scala for Java Interop

```scala
/**
 * BEST PRACTICES FOR JAVA INTEROP
 */

package com.example

object BestPractices {
  
  // 1. Avoid implicit parameters in public API
  // BAD: Java must pass explicitly
  def badFormat(x: Int)(implicit prefix: String): String = s"$prefix$x"
  
  // GOOD: Make optional parameter explicit
  def goodFormat(x: Int, prefix: String = "Value: "): String = s"$prefix$x"
  
  
  // 2. Use Java collections for public API
  import scala.jdk.CollectionConverters._
  
  // BAD: Returns Scala List
  def badGetItems: List[String] = List("a", "b", "c")
  
  // GOOD: Returns Java List
  def goodGetItems: java.util.List[String] = List("a", "b", "c").asJava
  
  
  // 3. Convert Option to Java Optional or nullable
  // BAD: Returns Scala Option
  def badFindUser(id: String): Option[String] = None
  
  // GOOD: Returns Java Optional
  def goodFindUser(id: String): java.util.Optional[String] = {
    badFindUser(id) match {
      case Some(value) => java.util.Optional.of(value)
      case None => java.util.Optional.empty()
    }
  }
  
  // OR: Return nullable with annotation
  def findUserNullable(id: String): String = null
  
  
  // 4. Use @throws for exceptions
  @throws[java.io.IOException]
  def readFile(path: String): String = {
    throw new java.io.IOException("Not implemented")
  }
  
  
  // 5. Provide Java-friendly factory methods
  case class User(id: String, email: String, verified: Boolean = false)
  
  object User {
    // Java-friendly factory
    def create(id: String, email: String): User = {
      User(id, email, verified = false)
    }
    
    def createVerified(id: String, email: String): User = {
      User(id, email, verified = true)
    }
  }
  
  
  // 6. Avoid advanced Scala features in public API
  // BAD: Path-dependent types
  trait BadContainer {
    type Element
    def get: Element
  }
  
  // GOOD: Generic types
  trait GoodContainer[T] {
    def get: T
  }
  
  
  // 7. Use @varargs for variable arguments
  class Logger {
    @scala.annotation.varargs
    def log(messages: String*): Unit = {
      messages.foreach(println)
    }
  }
  
  
  // 8. Document nullability
  class Repository {
    // Returns null if not found
    def findById(id: String): String = null
    
    // Never returns null
    def getDefault: String = "default"
  }
}
```

## Summary

### Quick Reference

**Type Mapping:**

```
Scala → Java:
Int → int
Long → long
String → String
Unit → void
List[T] → scala.collection.immutable.List<T>
Option[T] → scala.Option<T>

Case class → regular class with:
- public final fields (via methods)
- equals/hashCode/toString
- copy() method

Trait → interface (+ default methods)
```

**Collections:**

```scala
// Scala → Java
import scala.jdk.CollectionConverters._
scalaList.asJava

// Java → Scala
import scala.jdk.javaapi.CollectionConverters
CollectionConverters.asScala(javaList)
```

**Functional Patterns:**

```scala
// Option
Option[T] → isDefined(), isEmpty(), get()
Convert to Java Optional

// Either
Either[L, R] → isLeft(), isRight(), left().get(), right().get()

// Try
Try[T] → isSuccess(), isFailure(), get(), failed().get()
```

---



**Key insights:**

- **Less seamless than Kotlin**: Scala prioritizes features over Java compat
- **Explicit conversions**: Collections need manual conversion
- **Implicits invisible**: Java must pass implicitly explicitly
- **Case classes work**: But copy() awkward in Java
- **FP patterns**: Option, Either available but verbose

**The interop challenge:**

```
Scala to Java:
- Case classes → regular classes (OK)
- Traits → interfaces (OK)
- Collections → Scala types (need conversion)
- Option/Either → Scala types (verbose)
- Implicits → invisible (must pass explicitly)

Java to Scala:
- Primitives → work naturally
- Collections → need conversion
- Null → handle with Option
- Exceptions → checked become unchecked

Best practices:
✓ Return Java collections from public API
✓ Convert Option to Optional
✓ Avoid implicits in public API
✓ Use @varargs, @throws
✓ Document nullability
✓ Provide Java-friendly factories

Result: Workable but requires explicit effort!
```

**Scala vs Kotlin Interop:**

- Kotlin: Designed for seamless Java interop
- Scala: Powerful features, interop secondary
- Kotlin: Properties map naturally
- Scala: Need collection conversions
- Kotlin: Null safety enforced
- Scala: Option is explicit choice
