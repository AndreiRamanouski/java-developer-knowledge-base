# Kotlin Interoperability

## Overview

Kotlin was designed from day one to be 100% interoperable with Java, allowing gradual migration, mixed codebases, and seamless integration. This interoperability is Kotlin's killer feature—you can introduce Kotlin into existing Java projects without rewriting everything, call Java libraries from Kotlin effortlessly, and expose Kotlin code to Java consumers naturally. However, perfect interop requires understanding how Kotlin features map to Java bytecode, handling null safety boundaries, managing visibility of extension functions and companion objects, and integrating Kotlin coroutines with Java code. This guide covers the complete interoperability story: calling Kotlin from Java (properties, data classes, objects, extension functions), calling Java from Kotlin (platform types, null safety, checked exceptions), comparing Kotlin data classes with Java records, handling null safety at the boundary with `@Nullable`/`@NotNull` annotations, understanding extension function visibility, working with companion objects as static members, and integrating Kotlin coroutines in mixed projects. Whether you're adding Kotlin to a Java project or exposing Kotlin APIs to Java consumers, this guide provides the patterns and best practices for seamless interop.

## 1. Calling Kotlin from Java

### Java Consuming Kotlin Code

```kotlin
// Kotlin code
// File: Person.kt
package com.example

/**
 * KOTLIN CODE THAT JAVA WILL CALL
 */

// Simple class
class Person(val name: String, var age: Int) {
    fun greet() = "Hello, I'm $name"

        age++
    }

    val id: String,

    const val APP_NAME = "MyApp"
    val version = "1.0"
    
    fun initialize() {
        println("Config initialized")
    }
}

// Companion object

    companion object {
        fun connect(url: String): Database {
            println("Connecting to $url")
            return Database()
        }
        
        const val DEFAULT_TIMEOUT = 30
    }
    
    fun query(sql: String) = "Results"
}

// Extension function
fun String.reverse(): String = this.reversed()

// Top-level function
fun sayHello(name: String) = "Hello, $name!"

// Top-level property
```

```java
/**
 * JAVA CODE CALLING KOTLIN
 * 
 * How Java sees Kotlin code
 */

package com.example;

public class JavaCallingKotlin {
    
    /**
     * Calling Kotlin classes
     */
    public static void callingKotlinClasses() {
        System.out.println("=== Calling Kotlin Classes ===");
        
        // Simple class
        Person person = new Person("Alice", 30);
        
        // Properties become getters/setters
        String name = person.getName();      // val -> getter
        int age = person.getAge();           // var -> getter
        person.setAge(31);                   // var -> setter
        
        // Methods work normally
        String greeting = person.greet();
        person.celebrateBirthday();
        
        System.out.println(name + " is now " + person.getAge());
        
        /**
         * KOTLIN → JAVA MAPPING:
         * 
         * val property → getter only
         * var property → getter + setter
         * fun method   → normal method
         */
    }
    
    /**
     * Calling Kotlin data classes
     */
    public static void callingDataClasses() {
        System.out.println("\n=== Calling Data Classes ===");
        
        // Constructor with default parameters
        User user1 = new User("123", "alice@example.com", true);
        
        // Default parameter - must pass explicitly in Java
        User user2 = new User("456", "bob@example.com", false);
        
        // Properties
        String id = user1.getId();
        String email = user1.getEmail();
        boolean verified = user1.getVerified();
        
        // Generated methods
        String toString = user1.toString();
        int hashCode = user1.hashCode();
        boolean equals = user1.equals(user2);
        
        // copy() method
        User user3 = user1.copy("789", "charlie@example.com", false);
        
        System.out.println("User: " + user1);
        
        /**
         * DATA CLASS → JAVA:
         * 
         * Properties:
         * - Getters for all properties
         * - No setters (immutable by default)
         * 
         * Generated methods:
         * - toString()
         * - hashCode()
         * - equals()
         * - copy() with all parameters
         * 
         * Note: copy() is verbose in Java!
         */
    }
    
    /**
     * Calling Kotlin objects (singletons)
     */
    public static void callingObjects() {
        System.out.println("\n=== Calling Objects ===");
        
        // Object accessed via INSTANCE
        Config config = Config.INSTANCE;
        
        // Properties
        String appName = Config.APP_NAME;  // const val -> static final
        String version = Config.INSTANCE.getVersion();  // val -> getter
        
        // Methods
        Config.INSTANCE.initialize();
        
        // More natural way - static import
        // import static com.example.Config.INSTANCE;
        // INSTANCE.initialize();
        
        System.out.println("App: " + appName + " v" + version);
        
        /**
         * OBJECT → JAVA:
         * 
         * - Singleton via INSTANCE field
         * - const val → static final
         * - val → instance getter
         * - fun → instance method
         * 
         * Note: Always use INSTANCE!
         */
    }
    
    /**
     * Calling companion objects
     */
    public static void callingCompanionObjects() {
        System.out.println("\n=== Calling Companion Objects ===");
        
        // Companion object members are static
        Database db = Database.connect("jdbc:mysql://localhost");
        
        // const val in companion → static final
        int timeout = Database.DEFAULT_TIMEOUT;
        
        // Regular methods
        String results = db.query("SELECT * FROM users");
        
        System.out.println("Query: " + results + " (timeout: " + timeout + ")");
        
        /**
         * COMPANION OBJECT → JAVA:
         * 
         * Members become static:
         * - @JvmStatic fun → static method
         * - const val → static final field
         * - val → getter (via Companion)
         * 
         * Natural Java-like API!
         */
    }
    
    /**
     * Calling extension functions
     */
    public static void callingExtensionFunctions() {
        System.out.println("\n=== Calling Extension Functions ===");
        
        // Extension function becomes static method
        String reversed = PersonKt.reverse("Hello");
        
        // Must use file name class (PersonKt for Person.kt)
        // Pass receiver as first parameter
        
        System.out.println("Reversed: " + reversed);
        
        /**
         * EXTENSION FUNCTION → JAVA:
         * 
         * fun String.reverse() becomes:
         * static String reverse(String receiver)
         * 
         * File: Person.kt → class: PersonKt
         * Can customize with @file:JvmName
         */
    }
    
    /**
     * Calling top-level functions
     */
    public static void callingTopLevelFunctions() {
        System.out.println("\n=== Calling Top-Level Functions ===");
        
        // Top-level function becomes static method
        String greeting = PersonKt.sayHello("Bob");
        
        // Top-level property becomes static getter
        String version = PersonKt.getAPP_VERSION();
        
        System.out.println(greeting + " (v" + version + ")");
        
        /**
         * TOP-LEVEL → JAVA:
         * 
         * Top-level declarations in File.kt:
         * → static members in FileKt class
         * 
         * Can customize class name:
         * @file:JvmName("Utils")
         */
    }
    
    public static void main(String[] args) {
        callingKotlinClasses();
        callingDataClasses();
        callingObjects();
        callingCompanionObjects();
        callingExtensionFunctions();
        callingTopLevelFunctions();
    }
}
```

### Kotlin Consuming Java Code

```java
        if (path == null) {
            throw new IOException("Path cannot be null");

    }
    
    public void writeFile(String path, String content) 
            throws IOException, SecurityException {
        // Simulate writing
        System.out.println("Writing to " + path);

}

// Class with nullability
public class Repository {

            System.out.println("Saving: " + item);

}
```

```kotlin
/**
 * KOTLIN CODE CALLING JAVA
 * 
 * How Kotlin sees Java code
 */

package com.example

import java.io.IOException

object KotlinCallingJava {
    
    /**
     * Calling Java classes
     */
    fun callingJavaClasses() {
        println("=== Calling Java Classes ===")
        
        // Constructor
        val customer = Customer("Alice", 30)
        
        // Properties (getters/setters become properties)
        val name = customer.name      // getName()
        customer.age = 31             // setAge()
        
        // Methods
        customer.celebrate()
        
        println("$name is now ${customer.age}")
        
        /**
         * JAVA → KOTLIN MAPPING:
         * 
         * getter/setter → property
         * method → method
         * 
         * Feels natural in Kotlin!
         */
    }
    
    /**
     * Overloaded methods
     */
    fun overloadedMethods() {
        println("\n=== Overloaded Methods ===")
        
        val calc = Calculator()
        
        // Kotlin infers types
        val sum1 = calc.add(1, 2)          // int
        val sum2 = calc.add(1.5, 2.5)      // double
        val sum3 = calc.add("Hello", "!")  // String
        
        println("Sums: $sum1, $sum2, $sum3")
        
        /**
         * OVERLOADING:
         * 
         * Works naturally
         * Type inference helps
         */
    }
    
    /**
     * Checked exceptions
     */
    fun checkedExceptions() {
        println("\n=== Checked Exceptions ===")
        
        val manager = FileManager()
        
        // Checked exceptions become unchecked in Kotlin!
        try {
            val content = manager.readFile("file.txt")
            println("Read: $content")
        } catch (e: IOException) {
            println("Error: ${e.message}")
        }
        
        // No need to declare throws
        // Can catch if you want
        
        /**
         * CHECKED EXCEPTIONS:
         * 
         * Java: Must catch or declare
         * Kotlin: All exceptions unchecked
         * 
         * Can still catch if needed
         */
    }
    
    /**
     * Platform types (nullability)
     */
    fun platformTypes() {
        println("\n=== Platform Types ===")
        
        val repo = Repository()
        
        // findById returns String! (platform type)
        // ! means: "comes from Java, may or may not be null"
        
        // Option 1: Assume non-null (dangerous!)
        val item1: String = repo.findById("123")
        println("Item 1: $item1")
        
        // Option 2: Nullable type (safe)
        val item2: String? = repo.findById("456")
        println("Item 2: ${item2 ?: "Not found"}")
        
        // Option 3: Use safe call
        val item3 = repo.findById("789")?.uppercase()
        println("Item 3: $item3")
        
        /**
         * PLATFORM TYPES:
         * 
         * Type! means:
         * - Comes from Java
         * - Nullability unknown
         * - Your responsibility!
         * 
         * Best practice: Treat as nullable
         */
    }
    
    /**
     * Static members
     */
    fun staticMembers() {
        println("\n=== Static Members ===")
        
        // Static method
        val formatted = Utils.format("Hello %s", "World")
        
        // Static field
        val version = Utils.VERSION
        
        println("$formatted (v$version)")
        
        /**
         * STATIC MEMBERS:
         * 
         * Accessed via class name
         * Just like Java
         */
    }
    
    /**
     * SAM conversions
     */
    fun samConversions() {
        println("\n=== SAM Conversions ===")
        
        // Java functional interface
        val runnable = Runnable {
            println("Running!")
        }
        runnable.run()
        
        // Comparator
        val numbers = listOf(3, 1, 4, 1, 5, 9)
        val sorted = numbers.sortedWith(Comparator { a, b -> a - b })
        println("Sorted: $sorted")
        
        /**
         * SAM CONVERSION:
         * 
         * Java functional interface → Kotlin lambda
         * Automatic conversion
         * Very convenient!
         */
    }
    
    @JvmStatic
    fun main(args: Array<String>) {
        callingJavaClasses()
        overloadedMethods()
        checkedExceptions()
        platformTypes()
        staticMembers()
        samConversions()
    }
}
```

## 3. Data Classes vs Records

### Comparison and Interop

```kotlin
/**
 * KOTLIN DATA CLASS
 */

data class KotlinPerson(
    val name: String,
    val age: Int,
    val email: String
) {
    // Can have body, custom methods
    fun description() = "$name ($age) - $email"
    
    // Can have init block
    init {
        require(age >= 0) { "Age must be positive" }
    }
}
```

```java
/**
 * JAVA RECORD
 */

public record JavaPerson(
    String name,
    int age,
    String email
) {
    // Can have compact constructor
    public JavaPerson {
        if (age < 0) {
            throw new IllegalArgumentException("Age must be positive");
        }
    }
    
    // Can have custom methods
    public String description() {
        return name + " (" + age + ") - " + email;
    }
}
```

```kotlin
/**
 * COMPARISON
 */

object DataClassVsRecord {
    
    fun comparison() {
        println("=== Data Class vs Record ===")
        
        // Kotlin data class
        val kotlinPerson = KotlinPerson("Alice", 30, "alice@example.com")
        
        // Java record
        val javaPerson = JavaPerson("Bob", 25, "bob@example.com")
        
        // Both have similar features
        println("Kotlin: $kotlinPerson")
        println("Java: $javaPerson")
        
        // Both immutable
        // kotlinPerson.age = 31  // Error - val
        // javaPerson.age(31)      // Error - no setter
        
        /**
         * SIMILARITIES:
         * 
         * ✓ Immutable by default
         * ✓ Automatic toString()
         * ✓ Automatic hashCode()
         * ✓ Automatic equals()
         * ✓ Can have custom methods
         * ✓ Constructor validation
         */
    }
    
    fun differences() {
        /**
         * DIFFERENCES:
         * 
         * KOTLIN DATA CLASS:
         * ✓ copy() method
         * ✓ componentN() destructuring
         * ✓ Can have var properties (mutable)
         * ✓ Can inherit from classes
         * ✓ Can be abstract/sealed/inner
         * ✓ More flexible
         * 
         * JAVA RECORD:
         * ✓ Final by default
         * ✓ Cannot inherit from classes
         * ✓ Cannot have var (always immutable)
         * ✓ More restricted
         * ✓ Enforces immutability
         * 
         * 
         * INTEROP:
         * 
         * Kotlin can use Java records naturally
         * Java can use Kotlin data classes (but verbose copy())
         */
    }
    
    fun usingRecordsInKotlin() {
        println("\n=== Using Records in Kotlin ===")
        
        val person = JavaPerson("Charlie", 35, "charlie@example.com")
        
        // Access components
        val name = person.name()
        val age = person.age()
        val email = person.email()
        
        println("$name is $age years old")
        
        // Records don't have copy in Kotlin!
        // Must create new instance
        val updated = JavaPerson(person.name(), 36, person.email())
        
        /**
         * RECORDS IN KOTLIN:
         * 
         * ✓ Methods become functions
         * ✓ No property syntax (use methods)
         * ✗ No copy() - must construct new
         * ✗ No destructuring
         */
    }
    
    fun usingDataClassInJava() {
        // See Java example below
    }
}
```

## 4. Null Safety Interop

### Handling the Nullability Boundary

```kotlin
/**
 * NULL SAFETY INTEROP
 * 
 * Managing nullability between Java and Kotlin
 */

package com.example

import org.jetbrains.annotations.NotNull
import org.jetbrains.annotations.Nullable

object NullSafetyInterop {
    
    /**
     * Platform types - the challenge
     */
    fun platformTypes() {
        /**
         * PLATFORM TYPES:
         * 
         * When Kotlin calls Java:
         * - Java doesn't have nullability info
         * - Kotlin creates "platform type" (Type!)
         * - Can be treated as nullable or non-null
         * 
         * Problem:
         * - Unsafe!
         * - Can cause NullPointerException
         * - No compile-time guarantee
         * 
         * 
         * Example:
         * 
         * Java:
         * public String getName() { ... }
         * 
         * Kotlin sees:
         * fun getName(): String!
         * 
         * Can use as:
         * val name: String = getName()   // Crash if null!
         * val name: String? = getName()  // Safe
         */
    }
    
    /**
     * Solution: Annotations
     */
    fun annotationSolution() {
        /**
         * NULLABILITY ANNOTATIONS:
         * 
         * Kotlin understands:
         * - @Nullable → String? in Kotlin
         * - @NotNull → String in Kotlin
         * 
         * Common annotation libraries:
         * - JetBrains: org.jetbrains.annotations
         * - JSR-305: javax.annotation
         * - Android: androidx.annotation
         * - Spring: org.springframework.lang
         * - Eclipse: org.eclipse.jdt.annotation
         * 
         * Configure in build.gradle:
         * compileKotlin {
         *   freeCompilerArgs += ["-Xjsr305=strict"]
         * }
         */
    }
}
```

```java
/**
 * JAVA CODE WITH NULLABILITY ANNOTATIONS
 */

package com.example;

import org.jetbrains.annotations.NotNull;
import org.jetbrains.annotations.Nullable;

public class AnnotatedJavaCode {
    
    // Never returns null
    @NotNull
    public String getRequired() {
        return "Always present";
    }
    
    // May return null
    @Nullable
    public String getOptional() {
        return Math.random() > 0.5 ? "Present" : null;
    }
    
    // Never accepts null
    public void processRequired(@NotNull String value) {
        System.out.println(value.toUpperCase());
    }
    
    // Can accept null
    public void processOptional(@Nullable String value) {
        if (value != null) {
            System.out.println(value.toUpperCase());
        }
    }
    
    /**
     * ANNOTATIONS EFFECT:
     * 
     * Kotlin sees:
     * - @NotNull → String (non-null)
     * - @Nullable → String? (nullable)
     * 
     * Provides compile-time safety!
     */
}
```

```kotlin
/**
 * KOTLIN CALLING ANNOTATED JAVA
 */

fun callingAnnotatedJava() {
    val api = AnnotatedJavaCode()
    
    // @NotNull return → String
    val required: String = api.getRequired()  // OK
    // val required: String? = api.getRequired()  // Warning
    
    // @Nullable return → String?
    val optional: String? = api.getOptional()  // OK
    // val optional: String = api.getOptional()  // Error!
    
    // @NotNull parameter → requires non-null
    api.processRequired("Hello")  // OK
    // api.processRequired(null)  // Error!
    
    // @Nullable parameter → accepts null
    api.processOptional("Hello")  // OK
    api.processOptional(null)     // OK
    
    /**
     * RESULT:
     * 
     * ✓ Compile-time safety
     * ✓ No platform types
     * ✓ Clear contract
     */
}
```

```kotlin
/**
 * KOTLIN CODE EXPOSING TO JAVA
 */

class KotlinApi {

    // Non-null return
    fun getDefaultUser(): User {
        return User("0", "Guest")
    }
    
    // Nullable parameter
    fun save(user: User?) {
        user?.let { println("Saving: $it") }
    }
    
    // Non-null parameter
    fun validate(user: User) {
        println("Validating: $user")
    }
}

data class User(val id: String, val name: String)
```

```java
/**
 * JAVA CALLING KOTLIN CODE

         * Kotlin generates JetBrains annotations:
         * - String? → @Nullable String
         * - String → @NotNull String
         * 
         * Java IDE sees these and warns!
         */
    }
```

## 5. Extension Functions Visibility

### How Java Sees Kotlin Extensions

```kotlin
/**
 * EXTENSION FUNCTIONS IN KOTLIN
 */

package com.example

// Extension function
fun String.isPalindrome(): Boolean {
    return this == this.reversed()
}

// Extension property
val String.wordCount: Int
    get() = this.split("\\s+".toRegex()).size

// Extension on generic type
fun <T> List<T>.secondOrNull(): T? {
    return if (size >= 2) this[1] else null
}

// Extension in class
class StringUtils {
    // Not really an extension - just a method!
    fun String.addPrefix(prefix: String) = "$prefix$this"
}
```

```java
/**
 * JAVA CALLING EXTENSION FUNCTIONS
 */

package com.example;

public class CallingExtensions {
    
    public static void example() {
        // Extension function becomes static method
        // File: Extensions.kt → Class: ExtensionsKt
        
        // String.isPalindrome()
        boolean result = ExtensionsKt.isPalindrome("racecar");
        System.out.println("Is palindrome: " + result);
        
        // String.wordCount
        int count = ExtensionsKt.getWordCount("Hello World");
        System.out.println("Word count: " + count);
        
        // List.secondOrNull()
        List<String> list = Arrays.asList("a", "b", "c");
        String second = ExtensionsKt.secondOrNull(list);
        System.out.println("Second: " + second);
        
        /**
         * EXTENSION → JAVA:
         * 
         * fun Type.extensionName(params) becomes:
         * static ReturnType extensionName(Type receiver, params)
         * 
         * Receiver becomes first parameter!
         */
    }
    
    public static void customFileName() {
        // Can customize file class name
        
        /**
         * In Kotlin:
         * @file:JvmName("StringExtensions")
         * 
         * fun String.isPalindrome() = ...
         * 
         * In Java:
         * StringExtensions.isPalindrome("racecar")
         * 
         * Much better naming!
         */
    }
}
```

## 6. Companion Objects

### Static Members in Kotlin

```kotlin
/**
 * COMPANION OBJECTS
 * 
 * Kotlin's way of doing "static"
```

```kotlin
package com.example

import kotlinx.coroutines.*

class UserService {
    
    // Suspend function - Kotlin only!
    suspend fun fetchUser(id: String): User {
        delay(100)  // Simulate network
        return User(id, "User $id")
    }
    
    // Blocking wrapper for Java
    fun fetchUserBlocking(id: String): User {
        return runBlocking {
            fetchUser(id)
        }
    }
    
    // Async with callback for Java
    fun fetchUserAsync(id: String, callback: (User) -> Unit) {
        GlobalScope.launch {
            val user = fetchUser(id)
            callback(user)
        }
    }
    
    // CompletableFuture for Java
    fun fetchUserFuture(id: String): CompletableFuture<User> {
        val future = CompletableFuture<User>()
        
        GlobalScope.launch {
            try {
                val user = fetchUser(id)
                future.complete(user)
            } catch (e: Exception) {
                future.completeExceptionally(e)
            }
        }
        
        return future
    }
}
```

```java
/**
 * JAVA CALLING COROUTINES

import java.util.concurrent.*;

public class CallingCoroutines {
    
    public static void blockingApproach() {
        System.out.println("=== Blocking Approach ===");
        
        UserService service = new UserService();
        
        // Blocking call - simple but blocks thread
        User user = service.fetchUserBlocking("123");
        System.out.println("User: " + user.getName());
        
        /**
         * PROS:
         * ✓ Simple
         * ✓ Synchronous flow
         * 
         * CONS:
         * ✗ Blocks thread
         * ✗ Not efficient
         * ✗ Can't cancel
         */
    }
    
    public static void callbackApproach() {
        System.out.println("\n=== Callback Approach ===");
        
        UserService service = new UserService();
        
        // Async with callback
        service.fetchUserAsync("456", user -> {
            System.out.println("User: " + user.getName());
        });
        
        System.out.println("Request sent...");
        
        /**
         * PROS:
         * ✓ Non-blocking
         * ✓ Familiar to Java devs
         * 
         * CONS:
         * ✗ Callback hell
         * ✗ Error handling awkward
         */
    }
    
    public static void futureApproach() throws Exception {
        System.out.println("\n=== CompletableFuture Approach ===");
        
        UserService service = new UserService();
        
        // CompletableFuture - best for Java
        CompletableFuture<User> future = service.fetchUserFuture("789");
        
        // Chain operations
        future
            .thenApply(user -> user.getName())
            .thenAccept(name -> System.out.println("User: " + name))
            .exceptionally(e -> {
                System.err.println("Error: " + e.getMessage());
                return null;
            });
        
        // Or block if needed
        User user = future.get();
        System.out.println("Got user: " + user);
        
        /**
         * PROS:
         * ✓ Non-blocking
         * ✓ Composable
         * ✓ Good error handling
         * ✓ Java 8+ standard
         * 
         * CONS:
         * ✗ More verbose than coroutines
         * 
         * BEST APPROACH for Java!
         */
    }
}
```

```kotlin
import java.util.concurrent.CompletableFuture

suspend fun <T> CompletableFuture<T>.await(): T {
    return suspendCancellableCoroutine { continuation ->
        whenComplete { result, exception ->
            when {
                exception != null -> continuation.resumeWithException(exception)
                else -> continuation.resume(result)
            }
        }
    }
}
```

## 8. Best Practices

### Writing Kotlin for Java Consumers

```kotlin
/**
 * BEST PRACTICES FOR JAVA INTEROP
 */

// 1. Use @JvmStatic for companion object members
class Config {
    companion object {
        @JvmStatic
        fun load(path: String): Config = Config()
        
        const val VERSION = "1.0"
        
        @JvmField
        val DEFAULT_TIMEOUT = 30
    }
}

// 2. Use @JvmName for better file names
@file:JvmName("StringUtils")
package com.example.utils

fun String.capitalize(): String = this.replaceFirstChar { it.uppercase() }

// 3. Use @JvmOverloads for default parameters
class User @JvmOverloads constructor(
    val id: String,
    val name: String,
    val email: String = ""
) {
    // Java sees three constructors:
    // User(String id, String name, String email)
    // User(String id, String name)
}

// 4. Avoid nullable types in public API when possible
class Repository {
    // BAD: Forces Java to handle nullability
    fun findUser(id: String): User? = null
    
    // BETTER: Use Optional or throw
    fun findUserOpt(id: String): Optional<User> = Optional.empty()
    
    // OR: Throw exception
    fun getUserOrThrow(id: String): User = 
        throw NoSuchElementException("User not found")
}

// 5. Use @Throws for checked exceptions
class FileManager {
    @Throws(IOException::class)
    fun readFile(path: String): String {
        // Java will see IOException in signature
        throw IOException("File not found")
    }
}

// 6. Avoid extension functions in public API
// Use regular methods or utility classes instead
object StringProcessor {
    @JvmStatic
    fun reverse(input: String): String = input.reversed()
}

// 7. Use primitive types for performance
class Counter {
    // GOOD: Primitive int in bytecode
    var count: Int = 0
    
    // BAD: Integer wrapper (if nullable)
    var nullableCount: Int? = null
}

// 8. Expose suspend functions as CompletableFuture
class AsyncService {
    private suspend fun fetchData(): String {
        delay(100)
        return "Data"
    }
    
    // Expose to Java
    fun fetchDataAsync(): CompletableFuture<String> = 
        GlobalScope.future { fetchData() }
}
```

## Summary

### Quick Reference

**Kotlin → Java:**

```kotlin
// Properties
val name: String        // → getName()
var age: Int            // → getAge() / setAge()

// Object
object Config { }       // → Config.INSTANCE

// Companion
companion object {
    @JvmStatic fun create() // → static method
    const val VERSION       // → static final
    @JvmField val DEFAULT   // → static field
}

// Extension
fun String.reverse()    // → static reverse(String)

// Top-level
fun sayHello()          // → FileKt.sayHello()
```

**Java → Kotlin:**

```java
// Properties
getName()               // → name
setAge()                // → age = value

// Static
static void method()    // → ClassName.method()
static final String X   // → ClassName.X

// Nullability
String                  // → String! (platform type)
@Nullable String        // → String?
@NotNull String         // → String
```

---


**Key insights:**

- **100% interop**: Kotlin designed for seamless Java integration
- **Platform types**: Type! means "from Java, may be null"
- **Annotations**: Use @JvmStatic, @JvmField, @JvmName
- **Null safety**: Annotate Java code or treat as nullable
- **Async bridge**: CompletableFuture connects coroutines and Java

**The interop story:**

```
Kotlin to Java:
- Properties → getters/setters
- Objects → INSTANCE field
- Companion → static members (with @JvmStatic)
- Extensions → static methods
- Nullability → @Nullable/@NotNull annotations

Java to Kotlin:
- Getters/setters → properties
- Static → companion object semantics
- Null → platform type (Type!)
- CompletableFuture → convert to suspend

Best practices:
✓ Use @JvmStatic in companion objects
✓ Annotate nullability in Java
✓ Use @JvmOverloads for defaults
✓ Bridge coroutines with CompletableFuture
✓ Customize names with @JvmName

Result: Seamless mixed projects!
```
