# Groovy in Java Projects

## Overview

Groovy occupies a unique niche in the JVM ecosystem—while it can be used for full application development, its real strength lies in augmenting Java projects with scripting, DSLs, and testing. Groovy compiles to Java bytecode and offers seamless Java interop, but adds powerful features: optional dynamic typing, closures, operator overloading, and an incredibly flexible syntax that makes DSL creation trivial. This makes Groovy ideal for specific use cases within Java projects: runtime scripting (GroovyShell for evaluating user scripts), configuration DSLs (type-safe builders), testing (Spock framework with BDD-style tests), and build automation (Gradle). This guide covers embedding Groovy scripts in Java applications using GroovyShell and GroovyClassLoader, understanding how Groovy's dynamic typing interoperates with Java's static typing, using Groovy closures from Java (they're just interfaces), creating internal DSLs with builders and operator overloading, writing expressive tests with Spock, and understanding Gradle's Groovy-based build scripts. Unlike Kotlin or Scala which aim to replace Java, Groovy is best used as a complement—Java for the core application, Groovy for the flexible, dynamic parts.

```java
     * Using GroovyShell - simple evaluation
     */
    public static void usingGroovyShell() {
        System.out.println("=== GroovyShell ===");
        
        GroovyShell shell = new GroovyShell();
        
        // Evaluate expression
        Object result = shell.evaluate("2 + 2");
        System.out.println("Result: " + result);
        
        // Evaluate script
        String script = """
            def message = 'Hello from script'
            message.toUpperCase()
        """;
        
        Object scriptResult = shell.evaluate(script);
        System.out.println("Script result: " + scriptResult);
        
        /**
         * GROOVYSHELL:
         * 
         * Good for:
         * ✓ One-time script execution
         * ✓ Simple expressions
         * ✓ Quick evaluations
         * 
         * Not for:
         * ✗ Repeated execution (slow)
         * ✗ Complex scripts
         * ✗ Performance-critical code
         */
    }
    
    /**
     * Using Binding - passing variables
     */
    public static void usingBinding() {
        System.out.println("\n=== Binding Variables ===");
        
        Binding binding = new Binding();
        
        // Set variables
        binding.setVariable("name", "Alice");
        binding.setVariable("age", 30);
        
        GroovyShell shell = new GroovyShell(binding);
        
        // Script can access variables
        String script = """
            println "Name: ${name}"
            println "Age: ${age}"
            
            // Can set variables too
            result = "Processed: ${name}"
        """;
        
        shell.evaluate(script);
        
        // Get variables back
        Object result = binding.getVariable("result");
        System.out.println("Result from script: " + result);
        
        /**
         * BINDING:
         * 
         * Pass variables to script
         * Get variables back
         * Two-way communication!
         */
    }
    
    /**
     * Using GroovyClassLoader - compile and reuse
     */
    public static void usingGroovyClassLoader() throws Exception {
        System.out.println("\n=== GroovyClassLoader ===");
        
        GroovyClassLoader loader = new GroovyClassLoader();
        
        // Compile script to class
        String script = """
            class Calculator {
                int add(int a, int b) {
                    a + b
                }
                
                int multiply(int a, int b) {
                    a * b
                }
            }
        """;
        
        Class<?> calculatorClass = loader.parseClass(script);
        
        // Create instance
        Object calculator = calculatorClass.getDeclaredConstructor().newInstance();

        if (calculator instanceof GroovyObject) {
            GroovyObject groovyCalc = (GroovyObject) calculator;
            
            Object sum = groovyCalc.invokeMethod("add", new Object[]{5, 3});
            Object product = groovyCalc.invokeMethod("multiply", new Object[]{5, 3});
            
            System.out.println("Sum: " + sum);
            System.out.println("Product: " + product);
        }
        
        /**
         * GROOVYCLASSLOADER:
         * 
         * ✓ Compile once, use many times
         * ✓ Better performance
         * ✓ Can load from files
         * ✓ Class reuse
         */
    }

         * 
         * shell.evaluate(File)
         * shell.evaluate(Reader)
         * shell.evaluate(String scriptText)
         */
    }
```

## 2. Dynamic Typing Interop

### Groovy's Dynamic Nature with Java

```groovy
/**
 * GROOVY DYNAMIC TYPING
 */

// Dynamic typing
def dynamicVar = "Hello"
dynamicVar = 42
dynamicVar = [1, 2, 3]

// Static typing (optional)
String staticVar = "Hello"
// staticVar = 42  // Compile error

// Duck typing
def processItem(item) {
    item.process()  // Works with anything that has process()
}

// Meta-programming
String.metaClass.shout = { ->
    delegate.toUpperCase() + "!"
}

"hello".shout()  // "HELLO!"
```

```java
/**
 * JAVA CALLING GROOVY DYNAMIC CODE
 */

package com.example;

import groovy.lang.*;

public class DynamicTypingInterop {
    
    /**
     * Groovy's def becomes Object in Java
     */
    public static void defBecomesObject() {
        System.out.println("=== def → Object ===");
        
        String script = """
            def result = 'Hello'
            result
        """;
        
        GroovyShell shell = new GroovyShell();
        Object result = shell.evaluate(script);
        
        // Must cast or use instanceof
        if (result instanceof String) {
            String str = (String) result;
            System.out.println("String: " + str);
        }
        
        /**
         * GROOVY def → JAVA Object:
         * 
         * All dynamic types are Object
         * Must check type at runtime
         * No compile-time safety
         */
    }
    
    /**
     * Calling methods on dynamic objects
     */
    public static void callingDynamicMethods() throws Exception {
        System.out.println("\n=== Dynamic Method Calls ===");
        
        GroovyClassLoader loader = new GroovyClassLoader();
        
        String script = """
            class DynamicClass {
                def processData(data) {
                    "Processed: ${data}"
                }
                
                def methodMissing(String name, args) {
                    "Called missing method: ${name}"
                }
            }
        """;
        
        Class<?> clazz = loader.parseClass(script);
        GroovyObject obj = (GroovyObject) clazz.getDeclaredConstructor().newInstance();
        
        // Call method via GroovyObject
        Object result1 = obj.invokeMethod("processData", new Object[]{"test"});
        System.out.println("Result: " + result1);
        
        // Call non-existent method (methodMissing handles it)
        Object result2 = obj.invokeMethod("nonExistent", new Object[]{"arg"});
        System.out.println("Missing: " + result2);
        
        /**
         * GROOVYOBJECT INTERFACE:
         * 
         * invokeMethod(name, args) - call any method
         * getProperty(name) - get property
         * setProperty(name, value) - set property
         * 
         * Enables duck typing from Java!
         */
    }
    
    /**
     * Properties in Groovy
     */
    public static void groovyProperties() throws Exception {
        System.out.println("\n=== Groovy Properties ===");
        
        GroovyClassLoader loader = new GroovyClassLoader();
        
        String script = """
            class Person {
                String name
                int age
                
                String toString() {
                    "Person(${name}, ${age})"
                }
            }
        """;
        
        Class<?> personClass = loader.parseClass(script);
        GroovyObject person = (GroovyObject) personClass.getDeclaredConstructor().newInstance();
        
        // Set properties
        person.setProperty("name", "Alice");
        person.setProperty("age", 30);
        
        // Get properties
        String name = (String) person.getProperty("name");
        Integer age = (Integer) person.getProperty("age");
        
        System.out.println("Name: " + name + ", Age: " + age);
        System.out.println("Person: " + person);
        
        /**
         * GROOVY PROPERTIES:
         * 
         * Groovy:
         * class Person {
         *   String name  // Property (auto getter/setter)
         * }
         * 
         * Java sees:
         * - getProperty("name") / setProperty("name", val)
         * - Or: getName() / setName() if you know the type
         */
    }
    
    /**
     * Type checking at compile time
     */
    public static void compileTimeTyping() {
        /**
         * GROOVY STATIC TYPING:
         * 
         * Groovy supports @TypeChecked and @CompileStatic
         * 
         * @groovy.transform.CompileStatic
         * class StaticGroovy {
         *   String process(String input) {
         *     input.toUpperCase()
         *   }
         * }
         * 
         * Benefits:
         * ✓ Compile-time type checking
         * ✓ Better performance
         * ✓ IDE support
         * ✓ Safer for Java interop
         * 
         * Use @CompileStatic for Java-facing classes!
         */
    }
    
    public static void main(String[] args) throws Exception {
        defBecomesObject();
        callingDynamicMethods();
        groovyProperties();
    }
}
```

## 3. Groovy Closures

### Powerful Closures with Java Interop

```groovy
/**
 * GROOVY CLOSURES
 */

// Simple closure
def greet = { name -> "Hello, ${name}!" }
println greet("Alice")

// Closure with multiple parameters
def add = { a, b -> a + b }
println add(5, 3)

// Closure with implicit parameter (it)
def shout = { it.toUpperCase() + "!" }
println shout("hello")

// Closure with no parameters
def random = { -> Math.random() }
println random()

// Closures as method parameters
def process(List items, Closure action) {
    items.each { action(it) }
}

process([1, 2, 3], { println it * 2 })

// Closure delegates
class Builder {
    String result = ""
    
    void append(String text) {
        result += text
    }
}

def build(Closure closure) {
    def builder = new Builder()
    closure.delegate = builder
    closure()
    builder.result
}

def text = build {
    append "Hello"
    append " "
    append "World"
}
```

```java
/**
 * JAVA USING GROOVY CLOSURES
 */

package com.example;

import groovy.lang.*;

public class GroovyClosuresInJava {
    
    /**
     * Closures are just Callable/Runnable
     */
    public static void closuresAsInterfaces() throws Exception {
        System.out.println("=== Closures as Interfaces ===");
        
        GroovyShell shell = new GroovyShell();
        
        // Evaluate closure
        String script = "{ name -> \"Hello, ${name}!\" }";
        Closure<?> closure = (Closure<?>) shell.evaluate(script);
        
        // Call closure
        Object result = closure.call("Alice");
        System.out.println("Result: " + result);
        
        /**
         * CLOSURE INTERFACE:
         * 
         * Closure is an interface
         * call(...) method to invoke
         * Varargs support
         */
    }
    
    /**
     * Creating closures from Java
     */
    public static void creatingClosures() {
        System.out.println("\n=== Creating Closures ===");
        
        // Can't create Groovy closures directly from Java
        // But can use Java lambdas with Groovy code
        
        /**
         * FROM JAVA:
         * 
         * Can't create Groovy Closure
         * Use Java lambdas instead
         * Groovy accepts Java lambdas!
         * 
         * Example in Groovy code:
         * def processList(List items, Consumer<String> action)
         * 
         * From Java:
         * processList(list, s -> System.out.println(s));
         */
    }
    
    /**
     * Passing closures to Groovy
     */
    public static void passingClosures() throws Exception {
        System.out.println("\n=== Passing Closures ===");
        
        GroovyClassLoader loader = new GroovyClassLoader();
        
        // Groovy class that accepts closure
        String script = """
            class Processor {
                def process(List items, Closure action) {
                    items.collect { action(it) }
                }
            }
        """;
        
        Class<?> processorClass = loader.parseClass(script);
        Object processor = processorClass.getDeclaredConstructor().newInstance();
        
        // Create closure in Groovy
        GroovyShell shell = new GroovyShell();
        Closure<?> doubler = (Closure<?>) shell.evaluate("{ it * 2 }");
        
        // Call with closure
        java.util.List<Integer> numbers = java.util.Arrays.asList(1, 2, 3, 4, 5);
        
        if (processor instanceof GroovyObject) {
            Object result = ((GroovyObject) processor).invokeMethod(
                "process",
                new Object[]{numbers, doubler}
            );
            System.out.println("Processed: " + result);
        }
    }
    
    /**
     * Real-world example: Builder pattern
     */
    public static void builderExample() throws Exception {
        System.out.println("\n=== Builder Example ===");
        
        GroovyShell shell = new GroovyShell();
        
        String builderScript = """
            class HtmlBuilder {
                def result = new StringBuilder()
                
                void html(Closure content) {
                    result << '<html>'
                    content.delegate = this
                    content()
                    result << '</html>'
                }
                
                void head(Closure content) {
                    result << '<head>'
                    content.delegate = this
                    content()
                    result << '</head>'
                }
                
                void title(String text) {
                    result << "<title>${text}</title>"
                }
                
                void body(Closure content) {
                    result << '<body>'
                    content.delegate = this
                    content()
                    result << '</body>'
                }
                
                void h1(String text) {
                    result << "<h1>${text}</h1>"
                }
                
                void p(String text) {
                    result << "<p>${text}</p>"
                }
                
                String toString() { result.toString() }
            }
            
            def builder = new HtmlBuilder()
            builder.html {
                head {
                    title "My Page"
                }
                body {
                    h1 "Welcome"
                    p "This is content"
                }
            }
            
            builder.toString()
        """;
        
        Object html = shell.evaluate(builderScript);
        System.out.println("HTML:");
        System.out.println(html);
        
        /**
         * BUILDER PATTERN:
         * 
         * Closures enable DSL builders
         * Delegate pattern
         * Very powerful for configuration!
         */
    }
    
    public static void main(String[] args) throws Exception {
        closuresAsInterfaces();
        passingClosures();
        builderExample();
    }
}
```

```groovy
class DatabaseConfig {

class Project {

    config.delegate = proj
    config()
    proj

// Example 3: Test DSL (like Spock)
class Scenario {
    String description
    Closure given
    Closure when
    Closure then
    
    void given(Closure code) { this.given = code }
    void when(Closure code) { this.when = code }
    void then(Closure code) { this.then = code }
    
    void execute() {
        given?.call()
        when?.call()
        then?.call()
    }
}

def scenario(String desc, Closure config) {
    def s = new Scenario(description: desc)
    config.delegate = s
    config()
    s
}

    given {
        println "Setup user data"
    }
    when {
        println "Register user"
    }
    then {
        println "Verify registration"
    }
}
```

```java
package com.example;

import groovy.lang.*;

                config()
                s

                host 'localhost'
                port 8080
                timeout 60
            }
        """;

        // Access configuration
        if (config instanceof GroovyObject) {

            Integer port = (Integer) groovyConfig.getProperty("port");
```

## 5. Testing with Spock

### BDD-Style Testing Framework

```groovy
/**

// build.gradle dependency:
// testImplementation 'org.spockframework:spock-core:2.3-groovy-3.0'

import spock.lang.*

class CalculatorSpec extends Specification {
    
    /**
     * Basic test structure
     */
    def "should add two numbers"() {
        given: "a calculator"
        def calculator = new Calculator()
        
        when: "adding two numbers"
        def result = calculator.add(2, 3)
        
        then: "result should be sum"
        result == 5
    }
    
    /**
     * Data-driven testing
     */
    def "should add numbers correctly"() {
        expect:
        calculator.add(a, b) == result
        
        where:
        a | b | result
        1 | 2 | 3
        5 | 3 | 8
        0 | 0 | 0
        -1| 1 | 0
    }
    
    /**
     * Exception testing
     */
    def "should throw exception for division by zero"() {
        given:
        def calculator = new Calculator()
        
        when:
        calculator.divide(10, 0)
        
        then:
        thrown(ArithmeticException)
    }
    
    /**
     * Mocking
     */
    def "should call repository"() {
        given: "a mock repository"
        def repository = Mock(UserRepository)
        def service = new UserService(repository)
        
        when: "finding user"
        service.findUser("123")
        
        then: "repository is called"

    def "should handle user not found"() {

        when:
        def user = service.findUser("999")
        
        then:
        user == null
    }
    
    /**
     * Setup and cleanup
     */
    def setup() {
        // Before each test
    }
    
    def cleanup() {
        // After each test
    }
    
    def setupSpec() {
        // Before all tests
    }
    
    def cleanupSpec() {
        // After all tests
    }
}
```

```java
         *
```

```java
         *    ✓ Proof of concepts
         *    ✓ One-off scripts

     * When NOT to use Groovy
     */

         * 
         * 1. Core application logic:
         *    ✗ Business logic
         *    ✗ Domain models
         *    ✗ Critical paths
         *    
         *    Why: Dynamic typing less safe
         *    Better: Use Java or Kotlin
         * 
         * 2. Performance-critical code:
         *    ✗ Hot loops
         *    ✗ High-throughput services
         *    ✗ Low-latency requirements
         *    
         *    Why: Dynamic dispatch overhead
         *    Better: Use Java with static typing
         * 
         * 3. Large codebases:
         *    ✗ Entire application in Groovy
         *    ✗ Complex systems
         *    
         *    Why: Harder to maintain, refactor
         *    Better: Java core + Groovy scripts
         * 
         * 4. API contracts:
         *    ✗ Public APIs
         *    ✗ Library interfaces
         *    
         *    Why: Type safety important
         *    Better: Java or Kotlin
         */
    }

    public static void mixedProjects() {
```

**Key insights:**

- **Complement not replacement**: Groovy augments Java
- **Scripting power**: Runtime script evaluation
- **DSL magic**: Closures + delegates = beautiful DSLs
- **Spock testing**: Most readable tests on JVM
- **Gradle dominance**: Industry standard build tool

**The Groovy niche:**

```
Groovy's Sweet Spots:
✓ Build automation (Gradle)
✓ Testing (Spock)
✓ Scripting (user scripts)
✓ DSLs (configuration)
✓ Prototyping (rapid development)

Groovy's Weaknesses:
✗ Core application logic (use Java)
✗ Performance critical (dynamic overhead)
✗ Large codebases (harder to maintain)
✗ Public APIs (type safety)

Perfect Mixed Strategy:
Java: Core application (80%)
Groovy: Flexible parts (20%)
- Tests (Spock)
- Build (Gradle)
- Scripts (user customization)
- DSLs (configuration)

Result: Best of both worlds!
```

**Groovy vs Kotlin vs Scala:**

- Kotlin: Replace Java (full language)
- Scala: Replace Java (functional paradigm)
- Groovy: Augment Java (scripting, DSLs, testing)

**Real-world adoption:**

- Gradle: ~70% of Java projects
- Spock: Growing testing alternative
- Jenkins: Pipeline DSL
- Grails: Full-stack framework
