# Optional and Null Handling

## Overview

Optional is a container object introduced in Java 8 to explicitly represent the absence of a value, addressing the "billion-dollar mistake" of null references. This guide covers creating Optional instances (of, ofNullable, empty), consuming values safely (ifPresent, orElse, orElseGet, orElseThrow), transforming with map/flatMap/filter, the critical rule that Optional should only be used as a return type, common anti-patterns to avoid, and comparison with nullable annotations.

---

## 1. Creating Optional

### Three Ways to Create Optional

```java
/**
 * CREATING OPTIONAL
 * 
 * Different factory methods for different scenarios
 */

import java.util.*;

public class CreatingOptional {
    
    /**
     * Optional.of() - For non-null values
     */
    public static void optionalOf() {
        System.out.println("=== Optional.of() ===");
        
        String value = "Hello";
        Optional<String> optional = Optional.of(value);
        System.out.println("Optional: " + optional);
        
        // Throws NullPointerException if null
        try {
            Optional<String> nullOptional = Optional.of(null);
        } catch (NullPointerException e) {
            System.out.println("Cannot create Optional.of(null): " + e.getClass().getSimpleName());
        }
        
        /**
         * OPTIONAL.OF():
         * 
         * Optional.of(T value)
         * 
         * Use when:
         * - Value is guaranteed non-null
         * - Want to fail fast if null
         * - Explicit contract: value must exist
         * 
         * Throws:
         * - NullPointerException if value is null
         * 
         * Example use cases:
         * - Method always returns non-null
         * - Already validated non-null
         * - Constants or literals
         */
    }
    
    /**
     * Optional.ofNullable() - For potentially null values
     */
    public static void optionalOfNullable() {
        System.out.println("\n=== Optional.ofNullable() ===");
        
        // Non-null value
        String value1 = "Hello";
        Optional<String> optional1 = Optional.ofNullable(value1);
        System.out.println("Non-null: " + optional1);
        
        // Null value
        String value2 = null;
        Optional<String> optional2 = Optional.ofNullable(value2);
        System.out.println("Null: " + optional2);
        
        /**
         * OPTIONAL.OFNULLABLE():
         * 
         * Optional.ofNullable(T value)
         * 
         * Use when:
         * - Value might be null
         * - Converting nullable to Optional
         * - Most common creation method
         * 
         * Returns:
         * - Optional containing value if non-null
         * - Optional.empty() if null
         * 
         * Example use cases:
         * - Method return value might be null
         * - Converting legacy code
         * - Database query results
         * - Map.get() results
         */
    }
    
    /**
     * Optional.empty() - For explicit absence
     */
    public static void optionalEmpty() {
        System.out.println("\n=== Optional.empty() ===");
        
        Optional<String> empty = Optional.empty();
        System.out.println("Empty: " + empty);
        System.out.println("Is present: " + empty.isPresent());
        System.out.println("Is empty: " + empty.isEmpty());  // Java 11+
        
        /**
         * OPTIONAL.EMPTY():
         * 
         * Optional.empty()
         * 
         * Use when:
         * - Explicitly returning "no value"
         * - Clearer than ofNullable(null)
         * - Singleton instance (efficient)
         * 
         * Example use cases:
         * - No result found in search
         * - Invalid input (return empty)
         * - Optional method return
         */
    }
    
    /**
     * Common patterns
     */
    public static void commonPatterns() {
        System.out.println("\n=== Common Creation Patterns ===");
        
        // From Map.get()
        Map<String, String> map = new HashMap<>();
        map.put("key1", "value1");
        
        Optional<String> value = Optional.ofNullable(map.get("key1"));
        System.out.println("Map value: " + value);
        
        Optional<String> missing = Optional.ofNullable(map.get("missing"));
        System.out.println("Missing: " + missing);
        
        // From method that might return null
        String result = mightReturnNull();
        Optional<String> optResult = Optional.ofNullable(result);
        
        // Conditional creation
        boolean condition = true;
        Optional<String> conditional = condition 
            ? Optional.of("value") 
            : Optional.empty();
        
        /**
         * COMMON PATTERNS:
         * 
         * 1. DATABASE/REPOSITORY:
         *    Optional<User> findById(Long id) {
         *      User user = database.query(...);
         *      return Optional.ofNullable(user);
         *    }
         * 
         * 2. CONFIGURATION:
         *    Optional<String> getProperty(String key) {
         *      return Optional.ofNullable(properties.get(key));
         *    }
         * 
         * 3. PARSING:
         *    Optional<Integer> parseInt(String s) {
         *      try {
         *        return Optional.of(Integer.parseInt(s));
         *      } catch (NumberFormatException e) {
         *        return Optional.empty();
         *      }
         *    }
         * 
         * 4. FIRST/LAST:
         *    Optional<T> getFirst(List<T> list) {
         *      return list.isEmpty() 
         *        ? Optional.empty() 
         *        : Optional.of(list.get(0));
         *    }
         */
    }
    
    private static String mightReturnNull() {
        return Math.random() > 0.5 ? "value" : null;
    }
    
    public static void main(String[] args) {
        optionalOf();
        optionalOfNullable();
        optionalEmpty();
        commonPatterns();
    }
}
```

## 2. Consuming Values

### Safe Value Extraction

```java
/**
 * CONSUMING VALUES
 * 
 * Safe ways to work with Optional values
 */

import java.util.*;

public class ConsumingOptional {
    
    /**
     * isPresent() and get() - The basic way (don't use!)
     */
    public static void isPresentAndGet() {
        System.out.println("=== isPresent() and get() ===");
        
        Optional<String> optional = Optional.of("Hello");
        
        // Works but NOT recommended
        if (optional.isPresent()) {
            String value = optional.get();
            System.out.println("Value: " + value);
        }
        
        // Throws NoSuchElementException if empty
        try {
            Optional<String> empty = Optional.empty();
            String value = empty.get();  // DANGER!
        } catch (NoSuchElementException e) {
            System.out.println("Error: " + e.getClass().getSimpleName());
        }
        
        /**
         * ISPRESENT() AND GET():
         * 
         * boolean isPresent()
         * T get()
         * 
         * Problems:
         * - Defeats purpose of Optional
         * - Same as null check: if (x != null)
         * - Easy to forget check
         * - Verbose
         * 
         * ANTI-PATTERN:
         * if (optional.isPresent()) {
         *   doSomething(optional.get());
         * }
         * 
         * Better alternatives:
         * - ifPresent()
         * - orElse()
         * - orElseGet()
         * - map()
         */
    }
    
    /**
     * ifPresent() - Execute if value present
     */
    public static void ifPresent() {
        System.out.println("\n=== ifPresent() ===");
        
        Optional<String> optional = Optional.of("Hello");
        
        // Execute Consumer if present
        optional.ifPresent(value -> 
            System.out.println("Value: " + value)
        );
        
        // No output if empty
        Optional<String> empty = Optional.empty();
        empty.ifPresent(value -> 
            System.out.println("This won't print")
        );
        
        // Method reference
        Optional<String> name = Optional.of("Alice");
        name.ifPresent(System.out::println);
        
        /**
         * IFPRESENT():
         * 
         * void ifPresent(Consumer<T> action)
         * 
         * Use when:
         * - Side effect if value exists
         * - No return value needed
         * - One-way operation
         * 
         * Examples:
         * - Logging
         * - Updating UI
         * - Sending notification
         * - Caching
         * 
         * Better than:
         * if (optional.isPresent()) {
         *   action(optional.get());
         * }
         */
    }
    
    /**
     * ifPresentOrElse() - Execute either way (Java 9+)
     */
    public static void ifPresentOrElse() {
        System.out.println("\n=== ifPresentOrElse() (Java 9+) ===");
        
        /**
         * IFPRESENTORELSE() (Java 9+):
         * 
         * void ifPresentOrElse(Consumer<T> action, Runnable emptyAction)
         * 
         * optional.ifPresentOrElse(
         *   value -> System.out.println("Found: " + value),
         *   () -> System.out.println("Not found")
         * );
         * 
         * Use when:
         * - Need different side effects for present/absent
         * - Logging both cases
         * - Updating different metrics
         * 
         * Before Java 9:
         * if (optional.isPresent()) {
         *   action(optional.get());
         * } else {
         *   emptyAction();
         * }
         */
    }
    
    /**
     * orElse() - Provide default value
     */
    public static void orElse() {
        System.out.println("\n=== orElse() ===");
        
        // Value present - returns value
        Optional<String> optional = Optional.of("Hello");
        String result1 = optional.orElse("Default");
        System.out.println("Result 1: " + result1);  // Hello
        
        // Value absent - returns default
        Optional<String> empty = Optional.empty();
        String result2 = empty.orElse("Default");
        System.out.println("Result 2: " + result2);  // Default
        
        // CAUTION: Default always evaluated
        String result3 = optional.orElse(expensiveDefault());
        System.out.println("Result 3: " + result3);
        
        /**
         * ORELSE():
         * 
         * T orElse(T other)
         * 
         * Use when:
         * - Need default value
         * - Default is constant or cheap
         * - Default has no side effects
         * 
         * WARNING:
         * Default is ALWAYS evaluated!
         * 
         * optional.orElse(new ArrayList<>())
         * Creates ArrayList even if value present!
         * 
         * Examples:
         * - Constants: orElse("Unknown")
         * - Simple values: orElse(0)
         * - Existing objects: orElse(defaultConfig)
         */
    }
    
    private static String expensiveDefault() {
        System.out.println("  Computing expensive default...");
        return "Expensive Default";
    }
    
    /**
     * orElseGet() - Lazy default supplier
     */
    public static void orElseGet() {
        System.out.println("\n=== orElseGet() ===");
        
        // Value present - supplier NOT called
        Optional<String> optional = Optional.of("Hello");
        String result1 = optional.orElseGet(() -> {
            System.out.println("  This won't execute");
            return "Default";
        });
        System.out.println("Result 1: " + result1);  // Hello
        
        // Value absent - supplier called
        Optional<String> empty = Optional.empty();
        String result2 = empty.orElseGet(() -> {
            System.out.println("  Computing default...");
            return "Default";
        });
        System.out.println("Result 2: " + result2);  // Default
        
        // Method reference
        String result3 = empty.orElseGet(ConsumingOptional::expensiveDefault);
        
        /**
         * ORELSEGET():
         * 
         * T orElseGet(Supplier<T> supplier)
         * 
         * Use when:
         * - Default is expensive to compute
         * - Default has side effects
         * - Creating new objects
         * 
         * Lazy evaluation:
         * Supplier only called if value absent
         * 
         * PREFER THIS OVER ORELSE for:
         * - new ArrayList<>()
         * - Database queries
         * - File I/O
         * - Any computation
         * 
         * Performance difference:
         * orElse: Always evaluates
         * orElseGet: Lazy (only if needed)
         */
    }
    
    /**
     * orElseThrow() - Throw exception if empty
     */
    public static void orElseThrow() {
        System.out.println("\n=== orElseThrow() ===");
        
        // Value present - returns value
        Optional<String> optional = Optional.of("Hello");
        String result1 = optional.orElseThrow();  // Java 10+
        System.out.println("Result: " + result1);
        
        // Value absent - throws NoSuchElementException
        try {
            Optional<String> empty = Optional.empty();
            String result2 = empty.orElseThrow();
        } catch (NoSuchElementException e) {
            System.out.println("Threw: " + e.getClass().getSimpleName());
        }
        
        // Custom exception
        try {
            Optional<String> empty = Optional.empty();
            String result3 = empty.orElseThrow(() -> 
                new IllegalStateException("Value required!")
            );
        } catch (IllegalStateException e) {
            System.out.println("Custom exception: " + e.getMessage());
        }
        
        /**
         * ORELSETHROW():
         * 
         * T orElseThrow()  // Java 10+
         * T orElseThrow(Supplier<X> exceptionSupplier)
         * 
         * Use when:
         * - Value is required
         * - Absence is error condition
         * - Want to fail fast
         * 
         * Examples:
         * User user = repository.findById(id)
         *   .orElseThrow(() -> new UserNotFoundException(id));
         * 
         * Config config = getConfig(key)
         *   .orElseThrow(() -> new ConfigException("Missing: " + key));
         * 
         * Better than:
         * if (!optional.isPresent()) {
         *   throw new Exception(...);
         * }
         * return optional.get();
         */
    }
    
    /**
     * Choosing the right method
     */
    public static void choosingTheRightMethod() {
        /**
         * CHOOSING CONSUMPTION METHOD:
         * 
         * SIDE EFFECT ONLY:
         * optional.ifPresent(value -> log(value))
         * 
         * NEED RETURN VALUE:
         * 
         * 1. Constant default:
         *    String name = optional.orElse("Unknown");
         * 
         * 2. Computed default:
         *    List<T> list = optional.orElseGet(ArrayList::new);
         * 
         * 3. Value required:
         *    User user = optional.orElseThrow();
         * 
         * 4. Custom exception:
         *    Config cfg = optional.orElseThrow(
         *      () -> new ConfigException()
         *    );
         * 
         * 
         * DECISION TREE:
         * 
         * Need value or side effect?
         * ├─ Side effect only
         * │  └─ Use ifPresent()
         * └─ Need value
         *    ├─ Have default?
         *    │  ├─ Constant/cheap
         *    │  │  └─ Use orElse()
         *    │  └─ Expensive/computed
         *    │     └─ Use orElseGet()
         *    └─ No default (required)
         *       └─ Use orElseThrow()
         */
    }
    
    public static void main(String[] args) {
        isPresentAndGet();
        ifPresent();
        orElse();
        orElseGet();
        orElseThrow();
    }
}
```

## 3. Transforming Optional

### map, flatMap, and filter

```java
/**
 * TRANSFORMING OPTIONAL
 * 
 * Functional transformations on Optional values
 */

import java.util.*;

public class TransformingOptional {
    
    static class User {
        String name;
        Address address;
        
        User(String name, Address address) {
            this.name = name;
            this.address = address;
        }
        
        String getName() { return name; }
        Address getAddress() { return address; }
    }
    
    static class Address {
        String city;
        
        Address(String city) {
            this.city = city;
        }
        
        String getCity() { return city; }
    }
    
    /**
     * map() - Transform value if present
     */
    public static void mapTransformation() {
        System.out.println("=== map() Transformation ===");
        
        Optional<String> optional = Optional.of("hello");
        
        // Transform to uppercase
        Optional<String> upper = optional.map(String::toUpperCase);
        System.out.println("Uppercase: " + upper);
        
        // Transform to length
        Optional<Integer> length = optional.map(String::length);
        System.out.println("Length: " + length);
        
        // Chain transformations
        Optional<String> transformed = optional
            .map(String::toUpperCase)
            .map(s -> s + "!")
            .map(s -> "Prefix: " + s);
        System.out.println("Chained: " + transformed);
        
        // Empty optional
        Optional<String> empty = Optional.empty();
        Optional<Integer> emptyLength = empty.map(String::length);
        System.out.println("Empty mapped: " + emptyLength);  // empty
        
        /**
         * MAP():
         * 
         * <U> Optional<U> map(Function<T, U> mapper)
         * 
         * Behavior:
         * - If value present: applies mapper, wraps result
         * - If empty: returns empty
         * 
         * Use when:
         * - Simple transformation
         * - Mapper returns non-Optional
         * - Extracting properties
         * - Type conversion
         * 
         * Examples:
         * optional.map(String::length)
         * optional.map(User::getName)
         * optional.map(Integer::parseInt)
         * 
         * Similar to Stream.map()
         * But operates on single value
         */
    }
    
    /**
     * flatMap() - Transform to Optional
     */
    public static void flatMapTransformation() {
        System.out.println("\n=== flatMap() Transformation ===");
        
        User user = new User("Alice", new Address("NYC"));
        Optional<User> optUser = Optional.of(user);
        
        // WRONG: map() returns Optional<Optional<Address>>
        // Optional<Optional<Address>> nested = optUser.map(User::getAddress);
        
        // RIGHT: flatMap() flattens to Optional<Address>
        Optional<Address> address = optUser
            .flatMap(u -> Optional.ofNullable(u.getAddress()));
        System.out.println("Address present: " + address.isPresent());
        
        // Chain flatMaps for nested navigation
        Optional<String> city = optUser
            .flatMap(u -> Optional.ofNullable(u.getAddress()))
            .map(Address::getCity);
        System.out.println("City: " + city);
        
        /**
         * FLATMAP():
         * 
         * <U> Optional<U> flatMap(Function<T, Optional<U>> mapper)
         * 
         * Behavior:
         * - If value present: applies mapper, returns result
         * - If empty: returns empty
         * - Flattens nested Optional
         * 
         * Use when:
         * - Mapper returns Optional
         * - Navigating nested structures
         * - Chaining Optional operations
         * - Avoiding nested Optional
         * 
         * map vs flatMap:
         * 
         * map(f):     T → U → Optional<U>
         * flatMap(f): T → Optional<U> → Optional<U>
         * 
         * Example:
         * Optional<User> user = ...;
         * 
         * // map: Optional<Optional<Address>>
         * user.map(u -> findAddress(u))
         * 
         * // flatMap: Optional<Address>
         * user.flatMap(u -> findAddress(u))
         */
    }
    
    /**
     * filter() - Conditionally keep value
     */
    public static void filterTransformation() {
        System.out.println("\n=== filter() Transformation ===");
        
        Optional<String> optional = Optional.of("Hello");
        
        // Filter keeps value if predicate true
        Optional<String> filtered1 = optional.filter(s -> s.length() > 3);
        System.out.println("Length > 3: " + filtered1);  // present
        
        // Filter returns empty if predicate false
        Optional<String> filtered2 = optional.filter(s -> s.length() > 10);
        System.out.println("Length > 10: " + filtered2);  // empty
        
        // Filter on empty returns empty
        Optional<String> empty = Optional.empty();
        Optional<String> filtered3 = empty.filter(s -> true);
        System.out.println("Empty filtered: " + filtered3);  // empty
        
        // Chain with map
        Optional<String> result = optional
            .filter(s -> s.startsWith("H"))
            .map(String::toUpperCase)
            .filter(s -> s.length() > 3);
        System.out.println("Chained: " + result);
        
        /**
         * FILTER():
         * 
         * Optional<T> filter(Predicate<T> predicate)
         * 
         * Behavior:
         * - If value present AND predicate true: returns same Optional
         * - If value present AND predicate false: returns empty
         * - If empty: returns empty
         * 
         * Use when:
         * - Conditional validation
         * - Range checking
         * - Type checking
         * - Business rules
         * 
         * Examples:
         * optional.filter(s -> !s.isEmpty())
         * optional.filter(n -> n > 0)
         * optional.filter(user -> user.isActive())
         */
    }
    
    /**
     * Combining transformations
     */
    public static void combiningTransformations() {
        System.out.println("\n=== Combining Transformations ===");
        
        Optional<String> input = Optional.of("  42  ");
        
        // Complex pipeline
        Optional<Integer> result = input
            .map(String::trim)           // Remove whitespace
            .filter(s -> !s.isEmpty())   // Ensure not empty
            .map(Integer::parseInt)      // Parse to int
            .filter(n -> n > 0)          // Ensure positive
            .map(n -> n * 2);            // Double it
        
        result.ifPresent(n -> System.out.println("Result: " + n));
        
        // Safe navigation
        Optional<User> user = Optional.of(
            new User("Bob", new Address("SF"))
        );
        
        Optional<String> city = user
            .map(User::getAddress)
            .map(Address::getCity)
            .map(String::toUpperCase);
        
        System.out.println("City: " + city.orElse("Unknown"));
        
        /**
         * TRANSFORMATION PATTERNS:
         * 
         * 1. SAFE NAVIGATION:
         *    user.map(User::getAddress)
         *        .map(Address::getCity)
         * 
         * 2. VALIDATION CHAIN:
         *    input.filter(s -> !s.isEmpty())
         *         .filter(s -> s.length() < 100)
         *         .map(s -> process(s))
         * 
         * 3. PARSE AND VALIDATE:
         *    input.map(String::trim)
         *         .filter(s -> s.matches("\\d+"))
         *         .map(Integer::parseInt)
         * 
         * 4. FIND AND TRANSFORM:
         *    findUser(id)
         *      .filter(User::isActive)
         *      .map(User::getEmail)
         * 
         * Similar to Stream operations
         * But for single Optional value
         */
    }
    
    /**
     * or() - Alternative Optional (Java 9+)
     */
    public static void orAlternative() {
        /**
         * OR() (Java 9+):
         * 
         * Optional<T> or(Supplier<Optional<T>> supplier)
         * 
         * Returns current if present, else supplier's Optional
         * 
         * Example:
         * Optional<String> result = primary
         *   .or(() -> secondary)
         *   .or(() -> tertiary);
         * 
         * Use for:
         * - Fallback chain
         * - Multiple sources
         * - Lazy alternatives
         * 
         * vs orElse:
         * orElse returns T
         * or returns Optional<T>
         */
    }
    
    /**
     * stream() - Convert to Stream (Java 9+)
     */
    public static void streamConversion() {
        /**
         * STREAM() (Java 9+):
         * 
         * Stream<T> stream()
         * 
         * Returns:
         * - Stream with one element if present
         * - Empty stream if empty
         * 
         * Example:
         * List<String> results = users.stream()
         *   .map(this::findAddress)      // Stream<Optional<Address>>
         *   .flatMap(Optional::stream)    // Stream<Address>
         *   .collect(toList());
         * 
         * Before Java 9:
         * .filter(Optional::isPresent)
         * .map(Optional::get)
         */
    }
    
    public static void main(String[] args) {
        mapTransformation();
        flatMapTransformation();
        filterTransformation();
        combiningTransformations();
    }
}
```

## 4. Optional as Return Type Only

### Proper Usage Guidelines

```java
/**
 * OPTIONAL AS RETURN TYPE ONLY
 * 
 * Where to use and not use Optional
 */

import java.util.*;

public class OptionalReturnType {
    
    /**
     * CORRECT: Optional as return type
     */
    public static class CorrectUsage {
        private Map<String, User> users = new HashMap<>();
        
        // ✓ GOOD: Return Optional for search
        public Optional<User> findUserById(String id) {
            return Optional.ofNullable(users.get(id));
        }
        
        // ✓ GOOD: Computation may not produce result
        public Optional<Integer> parsePositiveInt(String s) {
            try {
                int value = Integer.parseInt(s);
                return value > 0 
                    ? Optional.of(value) 
                    : Optional.empty();
            } catch (NumberFormatException e) {
                return Optional.empty();
            }
        }
        
        // ✓ GOOD: May not find first element
        public Optional<String> getFirstNonEmpty(List<String> list) {
            return list.stream()
                .filter(s -> !s.isEmpty())
                .findFirst();
        }
        
        /**
         * RETURN TYPE BENEFITS:
         * 
         * 1. EXPLICIT CONTRACT:
         *    Method signature says "might not return value"
         *    Caller forced to handle absence
         * 
         * 2. NO NULL CHECKS:
         *    User user = findUser(id).orElse(defaultUser);
         *    
         *    vs
         *    
         *    User user = findUser(id);
         *    if (user == null) {
         *      user = defaultUser;
         *    }
         * 
         * 3. COMPOSABLE:
         *    findUser(id)
         *      .map(User::getEmail)
         *      .filter(email -> email.contains("@"))
         *      .ifPresent(this::sendEmail);
         */
    }
    
    /**
     * INCORRECT: Optional as parameter
     */
    public static class IncorrectParameterUsage {
        
        // ✗ BAD: Optional parameter
        public void processUser(Optional<User> user) {
            user.ifPresent(u -> System.out.println(u.getName()));
        }
        
        // ✓ GOOD: Overloading instead
        public void processUser(User user) {
            if (user != null) {
                System.out.println(user.getName());
            }
        }
        
        public void processUser() {
            // Default behavior
        }
        
        /**
         * WHY NOT OPTIONAL PARAMETERS:
         * 
         * 1. CALLER BURDEN:
         *    processUser(Optional.of(user))     // Verbose
         *    processUser(Optional.ofNullable(user))  // Even worse
         *    vs
         *    processUser(user)  // Simple
         * 
         * 2. NULL STILL POSSIBLE:
         *    processUser(null)  // Still compiles!
         *    Now have to check:
         *    - Is parameter null?
         *    - Is Optional empty?
         * 
         * 3. ALTERNATIVES BETTER:
         *    - Overloading
         *    - Null parameter (with null check)
         *    - Builder pattern
         *    - @Nullable annotation
         */
    }
    
    /**
     * INCORRECT: Optional as field
     */
    public static class IncorrectFieldUsage {
        
        // ✗ BAD: Optional field
        private Optional<String> name;
        
        // ✓ GOOD: Nullable field with Optional getter
        private String nameField;
        
        public Optional<String> getName() {
            return Optional.ofNullable(nameField);
        }
        
        /**
         * WHY NOT OPTIONAL FIELDS:
         * 
         * 1. SERIALIZATION:
         *    Optional not Serializable
         *    Breaks Java serialization
         *    JSON libraries handle poorly
         * 
         * 2. MEMORY OVERHEAD:
         *    Optional is object wrapper
         *    16 bytes + value
         *    For millions of objects: significant
         * 
         * 3. COMPLEXITY:
         *    this.name = Optional.of(value)
         *    vs
         *    this.name = value
         * 
         * 4. NULL POSSIBLE ANYWAY:
         *    Optional<String> name = null;
         *    Still need null check!
         * 
         * BETTER APPROACH:
         * - Nullable field
         * - Optional getter
         * - Initialize in constructor if required
         */
    }
    
    /**
     * INCORRECT: Optional in collections
     */
    public static void incorrectCollectionUsage() {
        // ✗ BAD: Collection of Optional
        List<Optional<String>> badList = new ArrayList<>();
        badList.add(Optional.of("a"));
        badList.add(Optional.empty());
        
        // ✓ GOOD: Filter out nulls
        List<String> goodList = new ArrayList<>();
        goodList.add("a");
        // Don't add null
        
        // Or stream filter
        List<String> nonNulls = badList.stream()
            .flatMap(Optional::stream)  // Java 9+
            .collect(java.util.stream.Collectors.toList());
        
        /**
         * WHY NOT OPTIONAL IN COLLECTIONS:
         * 
         * 1. EMPTY COLLECTION SUFFICES:
         *    List<String> - empty means no values
         *    List<Optional<String>> - redundant
         * 
         * 2. FILTER INSTEAD:
         *    list.stream()
         *      .filter(Objects::nonNull)
         *      .collect(toList());
         * 
         * 3. MEMORY WASTE:
         *    Each Optional is extra object
         * 
         * 4. COMPLEXITY:
         *    for (Optional<String> opt : list) {
         *      opt.ifPresent(s -> ...);
         *    }
         *    vs
         *    for (String s : list) {
         *      if (s != null) ...;
         *    }
         */
    }

        Optional<String> longest = names.stream()
            .reduce((a, b) -> a.length() > b.length() ? a : b);
        
        // ✓ GOOD: Max/min returns Optional
        Optional<String> max = names.stream()
            .max(String::compareTo);
        
        /**
         * STREAM OPERATIONS RETURNING OPTIONAL:
         * 
         * - findFirst()
         * - findAny()
         * - min(comparator)
         * - max(comparator)
         * - reduce(binaryOperator)
         * 
         * Makes sense because:
         * - Stream might be empty
         * - Operation might not produce result
         * - Caller must handle absence
         */
    }
```

## 5. Optional Anti-patterns

### Common Mistakes to Avoid

```java
/**
 * OPTIONAL ANTI-PATTERNS
 * 
 * Common misuses and how to fix them
 */

import java.util.*;

public class OptionalAntipatterns {
    
    /**
     * Anti-pattern 1: Using get() without check
     */
    public static void antipatternGet() {
        System.out.println("=== Anti-pattern: get() ===");
        
        // ✗ BAD: get() without check
        Optional<String> optional = Optional.empty();
        try {
            String value = optional.get();  // NoSuchElementException!
        } catch (NoSuchElementException e) {
            System.out.println("Error: " + e.getClass().getSimpleName());
        }
        
        // ✗ BAD: isPresent() + get()
        if (optional.isPresent()) {
            String value = optional.get();
            System.out.println(value);
        }
        
        // ✓ GOOD: Use orElse/orElseGet/orElseThrow
        String value1 = optional.orElse("default");
        String value2 = optional.orElseGet(() -> "default");
        
        /**
         * WHY BAD:
         * 
         * get() throws NoSuchElementException
         * Same danger as NullPointerException
         * Defeats purpose of Optional
         * 
         * isPresent() + get() = null check
         * No benefit over: if (x != null)
         * 
         * FIXES:
         * - orElse() for constant default
         * - orElseGet() for computed default
         * - orElseThrow() if required
         * - map/flatMap for transformation
         * - ifPresent() for side effects
         */
    }
    
    /**
     * Anti-pattern 2: Optional.of(null)
     */
    public static void antipatternOfNull() {
        System.out.println("\n=== Anti-pattern: of(null) ===");
        
        // ✗ BAD: Optional.of(null)
        try {
            Optional<String> bad = Optional.of(null);
        } catch (NullPointerException e) {
            System.out.println("Error: of(null) throws NPE");
        }
        
        // ✓ GOOD: Use ofNullable
        String value = null;
        Optional<String> good = Optional.ofNullable(value);
        
        /**
         * WHY BAD:
         * 
         * Optional.of(null) throws NullPointerException
         * Immediate failure
         * 
         * FIX:
         * Always use ofNullable() for potentially null values
         * Only use of() when guaranteed non-null
         */
    }
    
    /**
     * Anti-pattern 3: orElse with side effects
     */
    public static void antipatternOrElseSideEffects() {
        System.out.println("\n=== Anti-pattern: orElse Side Effects ===");
        
        Optional<String> optional = Optional.of("value");
        
        // ✗ BAD: orElse always evaluates
        String result = optional.orElse(createDefault());
        System.out.println("Result: " + result);
        
        // ✓ GOOD: orElseGet lazy evaluation
        String result2 = optional.orElseGet(OptionalAntipatterns::createDefault);
        
        /**
         * WHY BAD:
         * 
         * orElse() ALWAYS evaluates argument
         * Even if value present!
         * 
         * Problems:
         * - Wasted computation
         * - Unwanted side effects
         * - Object creation
         * - Database queries
         * 
         * EXAMPLE:
         * user.orElse(database.query())
         * Queries database even if user present!
         * 
         * FIX:
         * Use orElseGet() for:
         * - New object creation
         * - Method calls
         * - Computations
         * - Anything with side effects
         */
    }
    
    private static String createDefault() {
        System.out.println("  Creating default (should be lazy!)");
        return "default";
    }
    
    /**
     * Anti-pattern 4: Wrapping everything in Optional
     */
    public static void antipatternWrappingEverything() {
        // ✗ BAD: Wrapping method calls
        Optional<Integer> bad = Optional.of(calculateValue());
        
        // ✓ GOOD: Only wrap at API boundary
        public Optional<User> findUser(String id) {
            User user = database.query(id);
            return Optional.ofNullable(user);
        }
        
        /**
         * WHY BAD:
         * 
         * Optional has overhead:
         * - Object allocation
         * - Method calls
         * - Memory
         * 
         * Use Optional only for:
         * - Public API returns
         * - May legitimately not have value
         * - Explicit "optional" contract
         * 
         * DON'T wrap:
         * - Internal methods
         * - Always-present values
         * - Performance-critical code
         * - Private methods
         */
    }
    
    private static int calculateValue() {
        return 42;
    }
    
    /**
     * Anti-pattern 5: Optional.of(optional.get())
     */
    public static void antipatternNestedOptional() {
        Optional<String> optional = Optional.of("value");
        
        // ✗ BAD: Unnecessary wrapping
        Optional<String> bad = Optional.of(optional.get());
        
        // ✓ GOOD: Already have Optional
        Optional<String> good = optional;
        
        // ✗ BAD: Double wrapping
        Optional<Optional<String>> nested = Optional.of(optional);
        
        // ✓ GOOD: Use flatMap to avoid nesting
        Optional<String> flattened = Optional.of(optional)
            .flatMap(opt -> opt);
        
        /**
         * WHY BAD:
         * 
         * Unnecessary complexity
         * Risk of nested Optional<Optional<T>>
         * 
         * FIXES:
         * - Don't rewrap
         * - Use flatMap for Optional-returning functions
         * - Keep it simple
         */
    }
    
    /**
     * Anti-pattern 6: Returning null instead of Optional.empty()
     */
    public static void antipatternReturningNull() {
        // ✗ BAD: Return null from Optional method
        public Optional<User> findUserBad(String id) {
            if (notFound) {
                return null;  // Defeats purpose!
            }
            return Optional.of(user);
        }
        
        // ✓ GOOD: Return Optional.empty()
        public Optional<User> findUserGood(String id) {
            if (notFound) {
                return Optional.empty();
            }
            return Optional.of(user);
        }
        
        /**
         * WHY BAD:
         * 
         * Caller expects Optional
         * Gets null instead
         * NullPointerException still possible!
         * 
         * findUser(id).orElse(default)  // NPE if null!
         * 
         * FIX:
         * Always return Optional.empty()
         * Never return null from Optional method
         */
    }
    
    /**
     * Anti-pattern 7: Optional in equals/hashCode
     */
    public static void antipatternEqualsHashCode() {
        class BadClass {
            Optional<String> name;
            
            @Override
            public boolean equals(Object obj) {
                // ✗ BAD: Optional in equals
                BadClass other = (BadClass) obj;
                return Objects.equals(this.name, other.name);
            }
        }
        
        class GoodClass {
            String name;
            
            @Override
            public boolean equals(Object obj) {
                // ✓ GOOD: Compare actual values
                GoodClass other = (GoodClass) obj;
                return Objects.equals(this.name, other.name);
            }
            
            public Optional<String> getName() {
                return Optional.ofNullable(name);
            }
        }
        
        /**
         * WHY BAD:
         * 
         * Optional not designed for fields
         * Not Serializable
         * equals() behavior surprising
         * 
         * Optional.of("a").equals(Optional.of("a"))  // true
         * But should compare domain objects
         * 
         * FIX:
         * - Store actual values
         * - Return Optional from getters
         * - Compare unwrapped values
         */
    }
    
    public static void main(String[] args) {
        antipatternGet();
        antipatternOfNull();
        antipatternOrElseSideEffects();
    }
}
```

```java
import javax.annotation.Nonnull;
// Or: import org.jetbrains.annotations.Nullable;
// Or: import org.eclipse.jdt.annotation.Nullable;

public class OptionalVsNullable {
    
    /**
     * Optional approach
     */
    public static class OptionalApproach {
        private Map<String, User> users = new HashMap<>();
        
        // Optional return type
        public Optional<User> findUser(String id) {
            return Optional.ofNullable(users.get(id));
        }
        
        // Usage - forced to handle absence
        public void useOptional(String id) {
            findUser(id)
                .map(User::getName)
                .ifPresent(name -> System.out.println("Found: " + name));
        }
        
        /**
         * OPTIONAL ADVANTAGES:
         * 
         * 1. EXPLICIT IN TYPE:
         *    Method signature shows optionality
         *    Optional<User> - clearly may be absent
         * 
         * 2. FORCES HANDLING:
         *    Can't ignore Optional
         *    Must call orElse/ifPresent/etc.
         * 
         * 3. COMPOSABLE:
         *    map/flatMap/filter chains
         *    Functional style
         * 
         * 4. RUNTIME SAFETY:
         *    Type system enforces handling
         *    No accidental NPE from forgetting check
         * 
         * 5. NO TOOL DEPENDENCY:
         *    Part of Java standard library
         *    Works everywhere
         */
    }
    
    /**
     * @Nullable approach
     */
    public static class NullableApproach {
        private Map<String, User> users = new HashMap<>();
        
        // Annotated return type
        @Nullable
        public User findUser(String id) {
            return users.get(id);
        }
        
        // Usage - must remember to check
        public void useNullable(String id) {
            User user = findUser(id);
            if (user != null) {
                System.out.println("Found: " + user.getName());
            }
        }
        
        /**
         * @NULLABLE ADVANTAGES:
         * 
         * 1. ZERO OVERHEAD:
         *    No object allocation
         *    Compile-time only
         *    No runtime cost
         * 
         * 2. TOOL INTEGRATION:
         *    IDEs warn about missing checks
         *    Static analyzers catch issues
         *    IntelliJ, Eclipse, SpotBugs
         * 
         * 3. SIMPLER CODE:
         *    No Optional wrapping/unwrapping
         *    Familiar null checks
         *    Less ceremony
         * 
         * 4. WORKS WITH LEGACY:
         *    Can annotate existing code
         *    No API changes needed
         *    Gradual adoption
         * 
         * 5. FIELDS AND PARAMETERS:
         *    Can annotate anything
         *    Not just return types
         */
    }
```

## Summary

### Quick Reference

**Creating Optional:**

```java
Optional.of(value)              // Non-null (throws NPE if null)
Optional.ofNullable(value)      // Nullable (empty if null)
Optional.empty()                // Explicit empty
```

**Consuming Values:**

```java
optional.ifPresent(val -> ...)              // Side effect
optional.orElse(defaultValue)               // Default (eager)
optional.orElseGet(() -> defaultValue)      // Default (lazy)
optional.orElseThrow()                      // Required value
optional.orElseThrow(() -> new Ex())        // Custom exception
```

**Transforming:**

```java
optional.map(User::getName)                 // Transform if present
optional.flatMap(u -> findAddress(u))       // Flatten nested Optional
optional.filter(s -> s.length() > 5)        // Conditional keep
```

**Method Chaining:**

```java
repository.findById(id)
    .filter(User::isActive)
    .map(User::getEmail)
    .orElseThrow(() -> new UserNotFoundException(id));
```

### Decision Matrix

```
┌──────────────────┬─────────────┬──────────────┐
│ Use Case         │ Optional    │ @Nullable    │
├──────────────────┼─────────────┼──────────────┤
│ Return types     │ ✓ Best      │ ○ OK         │
│ Method params    │ ✗ Don't     │ ✓ Good       │
│ Fields           │ ✗ Don't     │ ✓ Good       │
│ Local variables  │ ✗ Rare      │ ✓ Good       │
│ Public API       │ ✓ Best      │ ○ OK         │
│ Internal methods │ ○ Sometimes │ ✓ Preferred  │
│ Performance code │ ✗ Overhead  │ ✓ Zero cost  │
│ Legacy code      │ ○ Hard      │ ✓ Easy       │
└──────────────────┴─────────────┴──────────────┘
```

### Usage Rules

**✅ DO:**

- Use Optional for return types
- Use ofNullable() for potentially null values
- Use orElseGet() for expensive defaults
- Use map/flatMap for transformations
- Use ifPresent() for side effects
- Keep fields nullable, return Optional from getters

**❌ DON'T:**

- Use Optional as method parameter
- Use Optional as field
- Put Optional in collections
- Call get() without checking
- Use Optional.of(null)
- Use orElse() with side effects
- Return null from Optional method

### Common Patterns

**Safe Navigation:**

```java
user.map(User::getAddress)
    .map(Address::getCity)
    .orElse("Unknown");
```

**Validation Chain:**

```java
input.map(String::trim)
     .filter(s -> !s.isEmpty())
     .map(Integer::parseInt)
     .filter(n -> n > 0)
     .orElseThrow(() -> new InvalidInputException());
```

**Repository Pattern:**

```java
public interface Repository<T, ID> {
    Optional<T> findById(ID id);
    Optional<T> findByEmail(String email);
}
```

**Configuration:**

```java
public Optional<String> getProperty(String key) {
    return Optional.ofNullable(properties.get(key));
}

String value = config.getProperty("db.url")
    .orElse("jdbc:mysql://localhost:3306/default");
```

### Anti-patterns to Avoid

```java
// ❌ isPresent() + get()
if (optional.isPresent()) {
    return optional.get();
}

// ✅ Use orElse/orElseGet
return optional.orElse(defaultValue);

// ❌ orElse with side effects
optional.orElse(createDefault())

// ✅ Use orElseGet
optional.orElseGet(() -> createDefault())

// ❌ Optional parameter
void process(Optional<User> user)

// ✅ Overload or use null
void process(User user)
void process()
```

- New code

**Use @Nullable:**

- Fields

- Local variables
- Performance-critical code
- Legacy code
- Internal methods

- Optional is for **return types**, not fields/parameters
- Always use `orElseGet()` for expensive defaults (lazy)
- Never call `get()` directly (defeats purpose)
- `@Nullable` for fields/params, Optional for returns
- Hybrid approach often best

**Anti-pattern recap:**

```
❌ optional.get()
❌ Optional.of(null)
❌ orElse(expensive())
❌ Optional as parameter
❌ Optional as field
❌ List<Optional<T>>
❌ return null from Optional method
```
