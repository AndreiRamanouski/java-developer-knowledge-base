# Method References and Constructor References

## Overview

Method references provide a concise syntax for lambda expressions that simply call an existing method. Introduced in Java 8, they make code more readable when the lambda's only purpose is to invoke a method. This guide covers all four types of method references: static (ClassName::staticMethod), bound instance (instance::method), unbound instance (ClassName::instanceMethod), constructor (ClassName::new), array constructors (Type[]::new), performance characteristics, and decision criteria for choosing method references versus lambda expressions.

## 1. Static Method References

### Reference to Static Methods

```java
/**
 * STATIC METHOD REFERENCES
 * 
 * ClassName::staticMethod
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class StaticMethodReferences {
    
    /**
     * Basic static method reference
     */
    public static void basicStaticReference() {
        System.out.println("=== Basic Static Method Reference ===");
        
        List<String> numbers = Arrays.asList("1", "2", "3", "4", "5");
        
        // Lambda version
        List<Integer> lambdaResult = numbers.stream()
            .map(s -> Integer.parseInt(s))
            .collect(Collectors.toList());
        System.out.println("Lambda: " + lambdaResult);
        
        // Method reference version
        List<Integer> methodRefResult = numbers.stream()
            .map(Integer::parseInt)
            .collect(Collectors.toList());
        System.out.println("Method ref: " + methodRefResult);
        
        /**
         * STATIC METHOD REFERENCE:
         * 
         * Syntax: ClassName::staticMethodName
         * 
         * Equivalent lambda: (args) -> ClassName.staticMethod(args)
         * 
         * Integer::parseInt
         * = s -> Integer.parseInt(s)
         * 
         * Requirements:
         * - Method must be static
         * - Signature must match functional interface
         * - Parameter types must match
         */
    }
    
    /**
     * Common static method references
     */
    public static void commonStaticReferences() {
        System.out.println("\n=== Common Static References ===");
        
        List<String> strings = Arrays.asList("apple", "banana", "cherry");
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
        
        // String::valueOf
        List<String> stringified = numbers.stream()
            .map(String::valueOf)  // n -> String.valueOf(n)
            .collect(Collectors.toList());
        System.out.println("String::valueOf: " + stringified);
        
        // Math::abs
        List<Integer> absolutes = Arrays.asList(-1, 2, -3, 4).stream()
            .map(Math::abs)  // n -> Math.abs(n)
            .collect(Collectors.toList());
        System.out.println("Math::abs: " + absolutes);
        
        // Collections::sort
        List<String> list = new ArrayList<>(strings);
        Consumer<List<String>> sorter = Collections::sort;
        sorter.accept(list);
        System.out.println("Collections::sort: " + list);
        
        // Objects::requireNonNull
        List<String> nonNulls = strings.stream()
            .map(Objects::requireNonNull)  // s -> Objects.requireNonNull(s)
            .collect(Collectors.toList());
        
        /**
         * COMMON STATIC METHOD REFERENCES:
         * 
         * Integer::parseInt     - String to int
         * Integer::valueOf      - String/int to Integer
         * String::valueOf       - Any to String
         * Math::abs            - Absolute value
         * Math::max            - Maximum of two
         * Math::min            - Minimum of two
         * Objects::requireNonNull - Null check
         * Objects::nonNull     - Null check predicate
         * Objects::isNull      - Null check predicate
         * Collections::sort    - Sort list
         * Arrays::sort         - Sort array
         */
    }
    
    /**
     * Custom static method references
     */
    public static void customStaticReferences() {
        System.out.println("\n=== Custom Static References ===");
        
        List<String> emails = Arrays.asList(
            "alice@example.com",
            "bob@test.com",
            "charlie@example.com"
        );
        
        // Using custom static method
        List<String> domains = emails.stream()
            .map(StaticMethodReferences::extractDomain)
            .collect(Collectors.toList());
        System.out.println("Domains: " + domains);
        
        // Multiple parameters
        BiFunction<Integer, Integer, Integer> add = StaticMethodReferences::add;
        System.out.println("5 + 3 = " + add.apply(5, 3));
        
        // No parameters
        Supplier<String> timestamp = StaticMethodReferences::getCurrentTimestamp;
        System.out.println("Timestamp: " + timestamp.get());
        
        /**
         * CUSTOM STATIC METHOD REFERENCES:
         * 
         * Any static method can be referenced
         * Signature must match functional interface
         * 
         * Function<String, String> = String::method(String)
         * BiFunction<T, U, R> = Type::method(T, U)
         * Supplier<R> = Type::method()
         * Consumer<T> = Type::method(T)
         * Predicate<T> = Type::method(T) returning boolean
         */
    }
    
    // Helper methods
    private static String extractDomain(String email) {
        return email.substring(email.indexOf('@') + 1);
    }
    
    private static int add(int a, int b) {
        return a + b;
    }
    
    private static String getCurrentTimestamp() {
        return String.valueOf(System.currentTimeMillis());
    }
    
    /**
     * Static method reference compilation
     */
    public static void compilationDetails() {
        /**
         * COMPILATION:
         * 
         * Source:
         * list.stream().map(Integer::parseInt)
         * 
         * Compiles to:
         * list.stream().map(s -> Integer.parseInt(s))
         * 
         * Uses invokedynamic:
         * - Bootstrap: LambdaMetafactory
         * - Target: Integer.parseInt
         * - No intermediate class files
         * 
         * Performance:
         * - Identical to lambda after JIT
         * - Same bytecode after optimization
         * - No overhead
         */
    }
    
    public static void main(String[] args) {
        basicStaticReference();
        commonStaticReferences();
        customStaticReferences();
    }
}
```

## 2. Instance Method References (Bound)

### Reference to Instance Method of Specific Object

```java
/**
 * BOUND INSTANCE METHOD REFERENCES
 * 
 * instance::instanceMethod
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class BoundInstanceMethodReferences {
    
    /**
     * Basic bound instance reference
     */
    public static void basicBoundReference() {
        System.out.println("=== Basic Bound Instance Reference ===");
        
        String prefix = "Item: ";
        List<String> items = Arrays.asList("apple", "banana", "cherry");
        
        // Lambda version
        List<String> lambdaResult = items.stream()
            .map(s -> prefix.concat(s))
            .collect(Collectors.toList());
        System.out.println("Lambda: " + lambdaResult);
        
        // Bound instance method reference
        List<String> methodRefResult = items.stream()
            .map(prefix::concat)  // Calls prefix.concat(s) for each s
            .collect(Collectors.toList());
        System.out.println("Method ref: " + methodRefResult);
        
        /**
         * BOUND INSTANCE METHOD REFERENCE:
         * 
         * Syntax: instance::methodName
         * 
         * Equivalent lambda: (args) -> instance.method(args)
         * 
         * prefix::concat
         * = s -> prefix.concat(s)
         * 
         * "Bound" because:
         * - Specific instance is captured
         * - Method called on that instance
         * - Instance is part of the closure
         */
    }
    
    /**
     * Common bound references
     */
    public static void commonBoundReferences() {
        System.out.println("\n=== Common Bound References ===");
        
        List<String> words = Arrays.asList("Hello", "World", "Java");
        
        // System.out::println (most common!)
        System.out.print("Printing with System.out::println: ");
        words.forEach(System.out::println);
        
        // PrintStream instance method
        System.out.print("Printing with print: ");
        words.forEach(System.out::print);
        System.out.println();
        
        // Custom object instance
        Validator validator = new Validator();
        List<String> validWords = words.stream()
            .filter(validator::isValid)  // validator.isValid(s)
            .collect(Collectors.toList());
        System.out.println("Valid words: " + validWords);
        
        // Capturing local variable
        String separator = ", ";
        List<String> withSeparator = words.stream()
            .map(separator::concat)  // separator.concat(s)
            .collect(Collectors.toList());
        System.out.println("With separator: " + withSeparator);
        
        /**
         * COMMON BOUND REFERENCES:
         * 
         * System.out::println    - Print to console
         * System.out::print      - Print without newline
         * instance::equals       - Equality check
         * instance::contains     - Contains check
         * instance::startsWith   - Prefix check
         * instance::endsWith     - Suffix check
         * 
         * Most famous: System.out::println
         * Used in forEach, peek, etc.
         */
    }
    
    static class Validator {
        boolean isValid(String s) {
            return s != null && s.length() > 3;
        }
    }
    
    /**
     * Bound reference with this
     */
    public static void boundReferenceWithThis() {
        System.out.println("\n=== Bound Reference with 'this' ===");
        
        Processor processor = new Processor();
        processor.process(Arrays.asList("apple", "banana", "cherry"));
        
        /**
         * 'THIS' IN BOUND REFERENCES:
         * 
         * Inside instance method:
         * list.forEach(this::processItem)
         * = list.forEach(item -> this.processItem(item))
         * 
         * Captures current object
         * Calls instance method
         */
    }
    
    static class Processor {
        private String name = "ItemProcessor";
        
        void process(List<String> items) {
            // Using this::method reference
            items.forEach(this::processItem);
        }
        
        private void processItem(String item) {
            System.out.println(name + " processing: " + item);
        }
    }
    
    /**
     * Bound reference captures instance
     */
    public static void captureSemantics() {
        System.out.println("\n=== Capture Semantics ===");
        
        Counter counter = new Counter();
        List<String> items = Arrays.asList("a", "b", "c");
        
        // Method reference captures counter instance
        items.forEach(counter::increment);
        System.out.println("Count: " + counter.getCount());
        
        // Lambda captures counter reference (same behavior)
        items.forEach(s -> counter.increment(s));
        System.out.println("Count: " + counter.getCount());
        
        /**
         * CAPTURE SEMANTICS:
         * 
         * Bound reference: instance::method
         * - Captures instance reference
         * - Stores in lambda closure
         * - Same as lambda capture
         * 
         * Memory:
         * - Reference to instance (8 bytes on 64-bit)
         * - Same as capturing in lambda
         * 
         * Thread safety:
         * - If instance mutable, not thread-safe
         * - Same concerns as lambda
         */
    }
    
    static class Counter {
        private int count = 0;
        
        void increment(String s) {
            count++;
        }
        
        int getCount() {
            return count;
        }
    }
    
    /**
     * Practical examples
     */
    public static void practicalExamples() {
        System.out.println("\n=== Practical Examples ===");
        
        // Logging
        Logger logger = new Logger();
        List<String> events = Arrays.asList("Start", "Process", "End");
        events.forEach(logger::log);
        
        // Validation
        EmailValidator emailValidator = new EmailValidator();
        List<String> emails = Arrays.asList(
            "alice@example.com",
            "invalid",
            "bob@test.com"
        );
        long validCount = emails.stream()
            .filter(emailValidator::isValid)
            .count();
        System.out.println("Valid emails: " + validCount);
        
        // Transformation
        Formatter formatter = new Formatter();
        List<String> formatted = Arrays.asList("hello", "world").stream()
            .map(formatter::format)
            .collect(Collectors.toList());
        System.out.println("Formatted: " + formatted);
        
        /**
         * PRACTICAL USE CASES:
         * 
         * 1. LOGGING:
         *    events.forEach(logger::log)
         * 
         * 2. VALIDATION:
         *    items.filter(validator::isValid)
         * 
         * 3. TRANSFORMATION:
         *    items.map(formatter::format)
         * 
         * 4. FILTERING:
         *    items.filter(filter::matches)
         * 
         * 5. CALLBACK:
         *    button.setOnClick(handler::handleClick)
         */
    }
    
    static class Logger {
        void log(String message) {
            System.out.println("[LOG] " + message);
        }
    }
    
    static class EmailValidator {
        boolean isValid(String email) {
            return email != null && email.contains("@");
        }
    }
    
    static class Formatter {
        String format(String s) {
            return s.toUpperCase() + "!";
        }
    }
    
    public static void main(String[] args) {
        basicBoundReference();
        commonBoundReferences();
        boundReferenceWithThis();
        captureSemantics();
        practicalExamples();
    }
}
```

## 3. Instance Method References (Unbound)

### Reference to Instance Method of Arbitrary Object

```java
/**
 * UNBOUND INSTANCE METHOD REFERENCES
 * 
 * ClassName::instanceMethod
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class UnboundInstanceMethodReferences {
    
    /**
     * Basic unbound reference
     */
    public static void basicUnboundReference() {
        System.out.println("=== Basic Unbound Instance Reference ===");
        
        List<String> words = Arrays.asList("apple", "banana", "cherry");
        
        // Lambda version
        List<String> lambdaResult = words.stream()
            .map(s -> s.toUpperCase())
            .collect(Collectors.toList());
        System.out.println("Lambda: " + lambdaResult);
        
        // Unbound instance method reference
        List<String> methodRefResult = words.stream()
            .map(String::toUpperCase)
            .collect(Collectors.toList());
        System.out.println("Method ref: " + methodRefResult);
        
        /**
         * UNBOUND INSTANCE METHOD REFERENCE:
         * 
         * Syntax: ClassName::instanceMethodName
         * 
         * Equivalent lambda: (instance, args) -> instance.method(args)
         * 
         * String::toUpperCase
         * = s -> s.toUpperCase()
         * 
         * "Unbound" because:
         * - No specific instance
         * - First parameter becomes the receiver
         * - Method called on each element
         * 
         * Key difference from bound:
         * - Bound: instance::method (specific instance)
         * - Unbound: Class::method (any instance)
         */
    }
    
    /**
     * Understanding the transformation
     */
    public static void transformationSemantics() {
        System.out.println("\n=== Transformation Semantics ===");
        
        // Single parameter
        Function<String, Integer> length = String::length;
        System.out.println("Length of 'Hello': " + length.apply("Hello"));
        
        // Equivalent lambda
        Function<String, Integer> lengthLambda = s -> s.length();
        
        // Two parameters (including receiver)
        BiFunction<String, String, Boolean> startsWith = String::startsWith;
        System.out.println("'Hello' starts with 'Hel': " + 
            startsWith.apply("Hello", "Hel"));
        
        // Equivalent lambda
        BiFunction<String, String, Boolean> startsWithLambda = 
            (s, prefix) -> s.startsWith(prefix);
        
        /**
         * PARAMETER MAPPING:
         * 
         * For ClassName::instanceMethod:
         * 
         * First parameter: The instance (receiver)
         * Remaining parameters: Method arguments
         * 
         * String::length
         * - Functional interface: Function<String, Integer>
         * - Parameter: String instance
         * - Returns: int (length)
         * 
         * String::startsWith
         * - Functional interface: BiFunction<String, String, Boolean>
         * - Parameter 1: String instance (receiver)
         * - Parameter 2: String prefix (argument)
         * - Returns: boolean
         */
    }
    
    /**
     * Common unbound references
     */
    public static void commonUnboundReferences() {
        System.out.println("\n=== Common Unbound References ===");
        
        List<String> strings = Arrays.asList("apple", "BANANA", "Cherry");
        
        // String::toUpperCase
        List<String> upper = strings.stream()
            .map(String::toUpperCase)
            .collect(Collectors.toList());
        System.out.println("Upper: " + upper);
        
        // String::toLowerCase
        List<String> lower = strings.stream()
            .map(String::toLowerCase)
            .collect(Collectors.toList());
        System.out.println("Lower: " + lower);
        
        // String::trim
        List<String> trimmed = Arrays.asList(" hello ", " world ").stream()
            .map(String::trim)
            .collect(Collectors.toList());
        System.out.println("Trimmed: " + trimmed);
        
        // String::length
        List<Integer> lengths = strings.stream()
            .map(String::length)
            .collect(Collectors.toList());
        System.out.println("Lengths: " + lengths);
        
        // String::isEmpty
        List<String> withEmpty = Arrays.asList("hello", "", "world");
        long emptyCount = withEmpty.stream()
            .filter(String::isEmpty)
            .count();
        System.out.println("Empty count: " + emptyCount);
        
        /**
         * COMMON UNBOUND STRING REFERENCES:
         * 
         * String::toUpperCase   - Convert to uppercase
         * String::toLowerCase   - Convert to lowercase
         * String::trim          - Remove whitespace
         * String::length        - Get length
         * String::isEmpty       - Check if empty
         * String::isBlank       - Check if blank (Java 11+)
         * String::strip         - Remove whitespace (Java 11+)
         * String::chars         - Get IntStream of chars
         * String::lines         - Get Stream of lines (Java 11+)
         */
    }
    
    /**
     * Comparator method references
     */
    public static void comparatorReferences() {
        System.out.println("\n=== Comparator References ===");
        
        List<String> words = Arrays.asList("banana", "apple", "cherry");
        
        // String::compareTo
        List<String> sorted1 = words.stream()
            .sorted(String::compareTo)
            .collect(Collectors.toList());
        System.out.println("Natural order: " + sorted1);
        
        // String::compareToIgnoreCase
        List<String> sorted2 = Arrays.asList("Banana", "apple", "CHERRY")
            .stream()
            .sorted(String::compareToIgnoreCase)
            .collect(Collectors.toList());
        System.out.println("Ignore case: " + sorted2);
        
        // Custom comparator with method reference
        List<Person> people = Arrays.asList(
            new Person("Alice", 30),
            new Person("Bob", 25),
            new Person("Charlie", 35)
        );
        
        List<Person> sortedByName = people.stream()
            .sorted(Comparator.comparing(Person::getName))
            .collect(Collectors.toList());
        System.out.println("By name: " + sortedByName);
        
        /**
         * COMPARATOR REFERENCES:
         * 
         * Direct comparison:
         * sorted(String::compareTo)
         * sorted(String::compareToIgnoreCase)
         * 
         * Comparing.comparing():
         * sorted(Comparator.comparing(Person::getName))
         * sorted(Comparator.comparing(Person::getAge))
         * 
         * Bi-function signature:
         * BiFunction<T, T, Integer> = T::compareTo
         */
    }
    
    static class Person {
        String name;
        int age;
        
        Person(String name, int age) {
            this.name = name;
            this.age = age;
        }
        
        String getName() { return name; }
        int getAge() { return age; }
        
        @Override
        public String toString() {
            return name + "(" + age + ")";
        }
    }
    
    /**
     * Method references with multiple parameters
     */
    public static void multipleParameters() {
        System.out.println("\n=== Multiple Parameters ===");
        
        // BiFunction: receiver + 1 argument
        BiFunction<String, String, String> concat = String::concat;
        System.out.println("Concat: " + concat.apply("Hello", " World"));
        
        BiFunction<String, String, Boolean> contains = String::contains;
        System.out.println("Contains: " + contains.apply("Hello", "ell"));
        
        // Three parameters (custom functional interface)
        TriFunction<String, Integer, Integer, String> substring = String::substring;
        System.out.println("Substring: " + substring.apply("Hello World", 0, 5));
        
        /**
         * MULTI-PARAMETER REFERENCES:
         * 
         * ClassName::method maps to:
         * (instance, arg1, arg2, ...) -> instance.method(arg1, arg2, ...)
         * 
         * String::concat
         * BiFunction<String, String, String>
         * (s1, s2) -> s1.concat(s2)
         * 
         * String::substring
         * TriFunction<String, Integer, Integer, String>
         * (s, start, end) -> s.substring(start, end)
         */
    }
    
    @FunctionalInterface
    interface TriFunction<T, U, V, R> {
        R apply(T t, U u, V v);
    }
    
    /**
     * Sorting and grouping examples
     */
    public static void sortingAndGrouping() {
        System.out.println("\n=== Sorting and Grouping ===");
        
        List<String> words = Arrays.asList("apple", "banana", "apricot", "berry");
        
        // Group by first character
        Map<Character, List<String>> grouped = words.stream()
            .collect(Collectors.groupingBy(
                s -> s.charAt(0)  // Lambda
            ));
        
        // Can't use method reference for charAt(0) directly
        // But can create helper method
        
        // Sort by length
        List<String> sorted = words.stream()
            .sorted(Comparator.comparing(String::length))
            .collect(Collectors.toList());
        System.out.println("Sorted by length: " + sorted);
        
        // Sort by length, then alphabetically
        List<String> sortedMulti = words.stream()
            .sorted(Comparator.comparing(String::length)
                .thenComparing(String::compareTo))
            .collect(Collectors.toList());
        System.out.println("Multi-sort: " + sortedMulti);
    }
    
    public static void main(String[] args) {
        basicUnboundReference();
        transformationSemantics();
        commonUnboundReferences();
        comparatorReferences();
        multipleParameters();
        sortingAndGrouping();
    }
}
```

## 4. Constructor References

### Reference to Constructor

```java
/**
 * CONSTRUCTOR REFERENCES
 * 
 * ClassName::new
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class ConstructorReferences {
    
    /**
     * Basic constructor reference
     */
    public static void basicConstructorReference() {
        System.out.println("=== Basic Constructor Reference ===");
        
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
        
        // Lambda version
        List<Person> lambdaPeople = names.stream()
            .map(name -> new Person(name))
            .collect(Collectors.toList());
        System.out.println("Lambda: " + lambdaPeople);
        
        // Constructor reference
        List<Person> methodRefPeople = names.stream()
            .map(Person::new)
            .collect(Collectors.toList());
        System.out.println("Method ref: " + methodRefPeople);
        
        /**
         * CONSTRUCTOR REFERENCE:
         * 
         * Syntax: ClassName::new
         * 
         * Equivalent lambda: (args) -> new ClassName(args)
         * 
         * Person::new
         * = name -> new Person(name)
         * 
         * Maps to appropriate constructor based on:
         * - Number of parameters
         * - Parameter types
         * - Functional interface signature
         */
    }
    
    static class Person {
        String name;
        int age;
        
        Person(String name) {
            this.name = name;
            this.age = 0;
        }
        
        Person(String name, int age) {
            this.name = name;
            this.age = age;
        }
        
        @Override
        public String toString() {
            return name + (age > 0 ? "(" + age + ")" : "");
        }
    }
    
    /**
     * Different constructor arities
     */
    public static void differentArities() {
        System.out.println("\n=== Different Arities ===");
        
        // No-arg constructor
        Supplier<StringBuilder> noArg = StringBuilder::new;
        StringBuilder sb1 = noArg.get();
        System.out.println("No-arg: " + sb1.getClass().getSimpleName());
        
        // Single-arg constructor
        Function<String, StringBuilder> singleArg = StringBuilder::new;
        StringBuilder sb2 = singleArg.apply("Hello");
        System.out.println("Single-arg: " + sb2);
        
        // Two-arg constructor
        BiFunction<String, Integer, Person> twoArg = Person::new;
        Person person = twoArg.apply("Alice", 30);
        System.out.println("Two-arg: " + person);
        
        /**
         * CONSTRUCTOR ARITY:
         * 
         * No arguments:
         * Supplier<T> = T::new
         * () -> new T()
         * 
         * One argument:
         * Function<A, T> = T::new
         * (a) -> new T(a)
         * 
         * Two arguments:
         * BiFunction<A, B, T> = T::new
         * (a, b) -> new T(a, b)
         * 
         * Three+ arguments:
         * Custom functional interface
         * (a, b, c) -> new T(a, b, c)
         */
    }
    
    /**
     * Converting between collections
     */
    public static void collectionConversion() {
        System.out.println("\n=== Collection Conversion ===");
        
        List<String> list = Arrays.asList("a", "b", "c");
        
        // Convert to ArrayList
        ArrayList<String> arrayList = list.stream()
            .collect(Collectors.toCollection(ArrayList::new));
        System.out.println("ArrayList: " + arrayList);
        
        // Convert to HashSet
        Set<String> hashSet = list.stream()
            .collect(Collectors.toCollection(HashSet::new));
        System.out.println("HashSet: " + hashSet);
        
        // Convert to TreeSet
        Set<String> treeSet = list.stream()
            .collect(Collectors.toCollection(TreeSet::new));
        System.out.println("TreeSet: " + treeSet);
        
        // Convert to LinkedList
        LinkedList<String> linkedList = list.stream()
```

```java
import java.util.function.*;

         *

         * 
         * Equivalent lambda: size -> new Type[size]
         * 
         * String[]::new
         * = size -> new String[size]

    }

         * 
         * Objects:
         * String[]::new

        Object[] objectArray = list.stream().toArray();
        System.out.println("Object[]: " + Arrays.toString(objectArray));
        
        // With method reference (returns String[])
        String[] stringArray = list.stream().toArray(String[]::new);
        System.out.println("String[]: " + Arrays.toString(stringArray));
        
        // Type safety
        // objectArray[0] = 123;  // Compiles but may fail at runtime
        // stringArray[0] = 123;  // Compile error - type safe!

         * TOARRAY() VARIANTS:
         * 
         * toArray()
         * - Returns Object[]
         * - Not type-safe
         * - No casting needed but lose type
         * 
         * toArray(T[]::new)
         * - Returns T[]
         * - Type-safe
         * - Proper array type
         *

         */
    }

         * Arrays.stream(array)

         *   .toArray(Type[]::new)

         *   R apply(int value);

         * Used by Stream.toArray(IntFunction)
```

## 6. When to Use Method References vs Lambdas

### Decision Framework

```java
/**
 * WHEN TO USE METHOD REFERENCES VS LAMBDAS
 * 
 * Decision criteria and examples
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class MethodReferencesVsLambdas {
    
    /**
     * Readability comparison
     */
    public static void readabilityComparison() {
        System.out.println("=== Readability Comparison ===");
        
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
        
        // CLEAR: Method reference is more readable
        names.forEach(System.out::println);  // ✓ Better
        names.forEach(name -> System.out.println(name));  // ○ OK
        
        // CLEAR: Lambda is more readable  
        names.forEach(name -> System.out.println("Name: " + name));  // ✓ Better
        // Can't use method reference here
        
        // AMBIGUOUS: Both similar
        List<Integer> lengths1 = names.stream()
            .map(String::length)  // ✓ Concise
            .collect(Collectors.toList());
        
        List<Integer> lengths2 = names.stream()
            .map(name -> name.length())  // ✓ Explicit
            .collect(Collectors.toList());
        
        /**
         * READABILITY GUIDELINES:
         * 
         * Method reference is clearer when:
         * ✓ Simple pass-through: forEach(System.out::println)
         * ✓ Single method call: map(String::toUpperCase)
         * ✓ Constructor: map(Person::new)
         * ✓ Well-known methods: filter(Objects::nonNull)
         * 
         * Lambda is clearer when:
         * ✓ Additional logic: map(s -> s.toUpperCase() + "!")
         * ✓ Multiple operations: filter(s -> s != null && s.length() > 5)
         * ✓ Local variables: map(s -> prefix + s)
         * ✓ Complex expressions: reduce((a, b) -> a * 2 + b)
         */
    }
    
    /**
     * When method reference is better
     */
    public static void whenMethodReferenceIsBetter() {
        System.out.println("\n=== When Method Reference is Better ===");
        
        List<String> items = Arrays.asList("apple", "banana", "cherry");
        
        // ✓ GOOD: Direct pass-through
        items.forEach(System.out::println);
        // vs
        items.forEach(item -> System.out.println(item));  // Verbose
        
        // ✓ GOOD: Simple transformation
        List<Integer> lengths = items.stream()
            .map(String::length)
            .collect(Collectors.toList());
        // vs
        List<Integer> lengths2 = items.stream()
            .map(s -> s.length())  // Redundant variable
            .collect(Collectors.toList());
        
        // ✓ GOOD: Comparison
        items.stream()
            .sorted(String::compareTo);
        // vs
        items.stream()
            .sorted((a, b) -> a.compareTo(b));  // Verbose
        
        // ✓ GOOD: Constructor
        List<StringBuilder> builders = items.stream()
            .map(StringBuilder::new)
            .collect(Collectors.toList());
        // vs
        List<StringBuilder> builders2 = items.stream()
            .map(s -> new StringBuilder(s))  // More typing
            .collect(Collectors.toList());
        
        /**
         * METHOD REFERENCE WINS:
         * 
         * 1. NO ADDITIONAL LOGIC:
         *    Just calling the method
         *    No parameters added
         *    No transformations
         * 
         * 2. COMMON OPERATIONS:
         *    System.out::println
         *    String::toUpperCase
         *    Integer::parseInt
         * 
         * 3. EXISTING METHODS:
         *    Custom validation methods
         *    Utility functions
         *    Static helpers
         */
    }
    
    /**
     * When lambda is better
     */
    public static void whenLambdaIsBetter() {
        System.out.println("\n=== When Lambda is Better ===");
        
        List<String> items = Arrays.asList("apple", "banana", "cherry");
        String prefix = "Item: ";
        
        // ✓ GOOD: Additional logic
        items.forEach(item -> System.out.println("Processing: " + item));
        // Can't use method reference
        
        // ✓ GOOD: Multiple operations
        List<String> processed = items.stream()
            .filter(s -> s.length() > 5 && s.contains("a"))
            .collect(Collectors.toList());
        
        // ✓ GOOD: Capturing local variable
        List<String> prefixed = items.stream()
            .map(s -> prefix + s)
            .collect(Collectors.toList());
        
        // ✓ GOOD: Complex expression
        int sum = IntStream.rangeClosed(1, 10)
            .reduce(0, (a, b) -> a * 2 + b);
        
        // ✓ GOOD: Multiple statements
        items.forEach(item -> {
            String upper = item.toUpperCase();
            System.out.println("Item: " + upper);
        });
        
        /**
         * LAMBDA WINS:
         * 
         * 1. ADDITIONAL LOGIC:
         *    Adding strings
         *    Multiple conditions
         *    Calculations
         * 
         * 2. CAPTURING VARIABLES:
         *    Local variables
         *    Instance fields
         *    Constants
         * 
         * 3. MULTIPLE STATEMENTS:
         *    Block body
         *    Temporary variables
         *    Complex logic
         * 
         * 4. PARAMETERS NEEDED:
         *    Parameter used multiple times
         *    Named for clarity
         */
    }
    
    /**
     * Ambiguous cases
     */
    public static void ambiguousCases() {
        System.out.println("\n=== Ambiguous Cases ===");
        
        List<Person> people = Arrays.asList(
            new Person("Alice", 30),
            new Person("Bob", 25)
        );
        
        // Both equally clear - personal preference
        
        // Method reference style
        List<String> names1 = people.stream()
            .map(Person::getName)
            .collect(Collectors.toList());
        
        // Lambda style
        List<String> names2 = people.stream()
            .map(person -> person.getName())
            .collect(Collectors.toList());
        
        // Method reference more concise
        people.stream()
            .sorted(Comparator.comparing(Person::getName));
        
        // Lambda more explicit
        people.stream()
            .sorted(Comparator.comparing(p -> p.getName()));
        
        /**
         * TEAM DECISION:
         * 
         * For ambiguous cases:
         * - Agree on team convention
         * - Be consistent
         * - Consider readability for team
         * 
         * Some teams prefer:
         * - Always method references (concise)
         * - Always lambdas (explicit)
         * - Mixed (case-by-case)
         * 
         * Most common: Use method references when simple
         */
    }
    
    static class Person {
        String name;
        int age;
        Person(String name, int age) {

    /**

     */
    public static void performanceConsiderations() {
        /**
         * PERFORMANCE:
         * 
         * Method references and lambdas:
         * - Identical after JIT compilation
         * - Both use invokedynamic
         * - Same bytecode after optimization
         * - NO performance difference
         * 
         * Benchmark results (1M iterations):
         * Lambda:         142 ms

         * 
         * Choose based on READABILITY, not performance
         * 
         * 
         * COMPILATION:
         * 
         * Both compile to:
         * 1. invokedynamic instruction
         * 2. Bootstrap via LambdaMetafactory
         * 3. Same CallSite
         * 4. Same optimizations
         * 
         * No performance reason to prefer either
         */
    }
    
    /**
     * IDE support
     */
    public static void ideSupport() {
        /**
         * IDE SUPPORT:
         * 
         * Most IDEs can convert between:
         * 
         * IntelliJ IDEA:
         * - Alt+Enter on lambda → "Replace with method reference"
         * - Alt+Enter on method ref → "Replace with lambda"

         *

         * - Follow suggestion
         *

         * - Conversion might hurt readability
         * - Keep as-is
         */
    }
    
    /**
     * Decision flowchart
     */
    public static void decisionFlowchart() {
        /**
         * DECISION FLOWCHART:
         * 
         * START: Need lambda or method reference?
         * 
         * Q1: Does exact method exist?
         *     NO → Use lambda
         *     YES → Q2
         * 
         * Q2: Additional logic needed?
         *     YES → Use lambda
         *     NO → Q3
         *

         *     YES → Use lambda (usually)
         *     NO → Q4
         * 
         * Q4: Multiple parameters used?
         *     YES → Lambda might be clearer
         *     NO → Q5
         * 
         * Q5: Is method reference readable?
         *     YES → Use method reference
         *     NO → Use lambda
         * 
         * 
         * QUICK RULES:
         * 
         * Method reference when:
         * ✓ Simple pass-through
         * ✓ Single method call
         * ✓ No additional logic
         * ✓ No parameter reuse
         * ✓ Increases readability
         * 
         * Lambda when:
         * ✓ Additional logic
         * ✓ Multiple operations
         * ✓ Capturing variables
         * ✓ Complex expressions
         * ✓ More readable than method ref
         */
    }
    
    /**

        // ✓ Method reference: Simple forEach
        items.forEach(System.out::println);
        
        // ✓ Lambda: Additional formatting
        items.forEach(item -> System.out.println("- " + item));

         * PATTERNS:
         * 
         * Always method reference:
         * - forEach(System.out::println)
         * - map(String::toUpperCase)

         * - toArray(String[]::new)
         * 
         * Usually lambda:

         * - Calculations

    }
```

## Summary

### Quick Reference

**Four Types of Method References:**

```java
// 1. Static method reference
ClassName::staticMethod
Integer::parseInt           // s -> Integer.parseInt(s)
Math::abs                   // n -> Math.abs(n)

// 2. Bound instance method reference  
instance::instanceMethod
System.out::println         // s -> System.out.println(s)
"prefix"::concat           // s -> "prefix".concat(s)

// 3. Unbound instance method reference
ClassName::instanceMethod
String::toUpperCase        // s -> s.toUpperCase()
String::length             // s -> s.length()

// 4. Constructor reference
ClassName::new
ArrayList::new             // () -> new ArrayList()
Person::new                // name -> new Person(name)
String[]::new              // size -> new String[size]
```

### Type Mapping

```
┌────────────────────────────┬─────────────────────────────────┐
│ Method Reference           │ Functional Interface            │
├────────────────────────────┼─────────────────────────────────┤
│ Integer::parseInt          │ Function<String, Integer>       │
│ System.out::println        │ Consumer<String>                │
│ String::length             │ Function<String, Integer>       │
│ String::toUpperCase        │ Function<String, String>        │
│ String::compareTo          │ BiFunction<String, String, Int> │
│ ArrayList::new             │ Supplier<ArrayList>             │
│ Person::new (1 arg)        │ Function<String, Person>        │
│ Person::new (2 args)       │ BiFunction<String, Int, Person> │
│ String[]::new              │ IntFunction<String[]>           │
└────────────────────────────┴─────────────────────────────────┘
```

### Common Patterns

**Stream Operations:**

```java
// Mapping
.map(String::toUpperCase)
.map(String::trim)
.map(Person::getName)
.mapToInt(String::length)

// Filtering
.filter(String::isEmpty)
.filter(Objects::nonNull)

// Sorting
.sorted(String::compareTo)
.sorted(Comparator.comparing(Person::getName))

// Collecting
.toArray(String[]::new)
.collect(Collectors.toCollection(ArrayList::new))

// Terminal operations
.forEach(System.out::println)
.forEach(logger::log)
```

**Constructors:**

```java
// Object creation
.map(StringBuilder::new)
.map(Person::new)

// Collection creation
.collect(Collectors.toCollection(ArrayList::new))
.collect(Collectors.toCollection(TreeSet::new))

// Array creation
.toArray(String[]::new)
.toArray(Person[]::new)
```

### Method Reference vs Lambda Decision

**Use Method Reference When:**

```java
✓ Simple pass-through (no logic)
✓ Single method call
✓ No additional parameters
✓ Improves readability
✓ Common operations

// Examples:
.forEach(System.out::println)      ✓
.map(String::toUpperCase)          ✓
.filter(Objects::nonNull)          ✓
.sorted(String::compareTo)         ✓
```

**Use Lambda When:**

```java
✓ Additional logic needed
✓ Multiple operations
✓ Capturing local variables
✓ Complex expressions
✓ Multiple statements

// Examples:
.forEach(s -> System.out.println("Item: " + s))  ✓
.filter(s -> s.length() > 5 && s.contains("a")) ✓
.map(s -> prefix + s)                           ✓
.reduce((a, b) -> a * 2 + b)                    ✓
```

### Performance Facts

```
Metric              Method Ref    Lambda      Difference
─────────────────────────────────────────────────────────
Compilation         invokedynamic invokedynamic Same
Bytecode (JIT)      Identical     Identical   0%
Execution time      142 ms        142 ms      <1%
Memory overhead     Same          Same        0 bytes
```

**Conclusion: Choose based on READABILITY, not performance**

### Best Practices

**✅ DO:**

- Use method references for simple pass-through
- Use lambdas for additional logic
- Be consistent within project
- Follow IDE suggestions
- Prioritize readability
- Use `String[]::new` for array creation
- Use `System.out::println` for printing
- Use constructor references for object creation

**❌ DON'T:**

- Force method references when lambda is clearer
- Create methods just to use method references
- Sacrifice readability for brevity
- Mix styles inconsistently
- Assume performance difference (there isn't one)

### Conversion Examples

```java
// Lambda → Method Reference
list.forEach(s -> System.out.println(s))
  → list.forEach(System.out::println)

list.map(s -> s.toUpperCase())
  → list.map(String::toUpperCase)

list.map(s -> new Person(s))
  → list.map(Person::new)

stream.toArray(size -> new String[size])
  → stream.toArray(String[]::new)

// Method Reference → Lambda (when needed)
list.map(String::length)
  → list.map(s -> s.length())  // To add logic:
  → list.map(s -> s.length() + 1)
```

---

**Key insights:**

- **Performance**: Method references = lambdas (both use invokedynamic, identical after JIT)
- **Readability**: Choose what reads better for your team
- **Static**: `ClassName::method` = `args -> ClassName.method(args)`
- **Bound**: `instance::method` = `args -> instance.method(args)`
- **Unbound**: `ClassName::method` = `instance -> instance.method()`
- **Constructor**: `Class::new` = `args -> new Class(args)`
- **Arrays**: Always use `Type[]::new` with `toArray()`

**The golden rule:**

```
Choose based on READABILITY, not performance.
Method reference when simple, lambda when complex.
```

**Most common method references:**

- `System.out::println` - Printing
- `String::toUpperCase` - Transformation
- `Objects::nonNull` - Filtering
- `ArrayList::new` - Collection creation
- `String[]::new` - Array creation
