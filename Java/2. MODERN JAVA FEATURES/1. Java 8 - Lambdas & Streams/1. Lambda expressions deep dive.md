# Lambda Expressions Deep Dive

## Overview

Lambda expressions revolutionized Java programming in Java 8, enabling functional programming paradigms and cleaner, more concise code. Under the hood, lambdas use invokedynamic for efficient implementation. This guide covers functional interfaces, lambda syntax variations, method references, variable capturing mechanics, comparison with anonymous inner classes, serialization concerns, and critical performance characteristics essential for writing efficient modern Java code.

## 1. Functional Interfaces

### Single Abstract Method (SAM) Interfaces

```java
/**
 * FUNCTIONAL INTERFACES
 * 
 * Interfaces with exactly one abstract method
 */

import java.util.function.*;
import java.util.*;

public class FunctionalInterfacesDeepdive {
    
    /**
     * Defining functional interfaces
     */
    @FunctionalInterface
    interface Calculator {
        int calculate(int a, int b);
        
        // Default methods allowed
        default int add(int a, int b) {
            return a + b;
        }
        
        // Static methods allowed
        static int multiply(int a, int b) {
            return a * b;
        }
        
        // Object methods don't count
        @Override
        boolean equals(Object obj);
        
        // Compile error if uncommented - not a functional interface!
        // int anotherAbstractMethod();
    }
    
    /**
     * Using custom functional interface
     */
    public static void customFunctionalInterface() {
        // Lambda expression
        Calculator add = (a, b) -> a + b;
        Calculator multiply = (a, b) -> a * b;
        
        System.out.println("=== Custom Functional Interface ===");
        System.out.println("Add: " + add.calculate(5, 3));
        System.out.println("Multiply: " + multiply.calculate(5, 3));
        
        /**
         * FUNCTIONAL INTERFACE RULES:
         * 
         * Must have:
         * - Exactly ONE abstract method
         * 
         * Can have:
         * - Multiple default methods
         * - Multiple static methods
         * - Methods from Object (equals, hashCode, toString)
         * 
         * @FunctionalInterface annotation:
         * - Optional but recommended
         * - Compiler enforces single abstract method
         * - Documentation for readers
         */
    }
    
    /**
     * Built-in functional interfaces
     */
    public static void builtInFunctionalInterfaces() {
        System.out.println("\n=== Built-in Functional Interfaces ===");
        
        // Function<T, R> - takes T, returns R
        Function<String, Integer> stringLength = s -> s.length();
        System.out.println("Length of 'Hello': " + stringLength.apply("Hello"));
        
        // Predicate<T> - takes T, returns boolean
        Predicate<String> isEmpty = s -> s.isEmpty();
        System.out.println("Is '' empty: " + isEmpty.test(""));
        
        // Consumer<T> - takes T, returns void
        Consumer<String> printer = s -> System.out.println("  " + s);
        printer.accept("Consumer output");
        
        // Supplier<T> - takes nothing, returns T
        Supplier<Double> randomSupplier = () -> Math.random();
        System.out.println("Random: " + randomSupplier.get());
        
        // BiFunction<T, U, R> - takes T and U, returns R
        BiFunction<String, String, String> concat = (a, b) -> a + b;
        System.out.println("Concat: " + concat.apply("Hello", "World"));
        
        // UnaryOperator<T> - Function<T, T>
        UnaryOperator<String> toUpper = s -> s.toUpperCase();
        System.out.println("Upper: " + toUpper.apply("hello"));
        
        // BinaryOperator<T> - BiFunction<T, T, T>
        BinaryOperator<Integer> sum = (a, b) -> a + b;
        System.out.println("Sum: " + sum.apply(5, 3));
        
        /**
         * STANDARD FUNCTIONAL INTERFACES:
         * 
         * BASIC:
         * Function<T, R>        - T → R
         * Predicate<T>          - T → boolean
         * Consumer<T>           - T → void
         * Supplier<T>           - () → T
         * 
         * BI-ARG:
         * BiFunction<T, U, R>   - (T, U) → R
         * BiPredicate<T, U>     - (T, U) → boolean
         * BiConsumer<T, U>      - (T, U) → void
         * 
         * OPERATORS:
         * UnaryOperator<T>      - T → T
         * BinaryOperator<T>     - (T, T) → T
         * 
         * PRIMITIVE SPECIALIZATIONS:
         * IntFunction<R>        - int → R
         * IntPredicate          - int → boolean
         * IntConsumer           - int → void
         * IntSupplier           - () → int
         * IntUnaryOperator      - int → int
         * IntBinaryOperator     - (int, int) → int
         * 
         * Similarly: Long*, Double*
         * 
         * TO-PRIMITIVE:
         * ToIntFunction<T>      - T → int
         * ToLongFunction<T>     - T → long
         * ToDoubleFunction<T>   - T → double
         * 
         * BI TO-PRIMITIVE:
         * ToIntBiFunction<T, U> - (T, U) → int
         */
    }
    
    /**
     * Primitive specializations
     */
    public static void primitiveSpecializations() {
        System.out.println("\n=== Primitive Specializations ===");
        
        // Generic - requires boxing
        Function<Integer, Integer> square = x -> x * x;
        Integer result1 = square.apply(5);  // Boxing: 5 → Integer(5)
        
        // Primitive - no boxing
        IntUnaryOperator squarePrimitive = x -> x * x;
        int result2 = squarePrimitive.applyAsInt(5);  // No boxing!
        
        System.out.println("Square (generic): " + result1);
        System.out.println("Square (primitive): " + result2);
        
        /**
         * PRIMITIVE SPECIALIZATIONS BENEFIT:
         * 
         * Without specialization:
         * - Boxing/unboxing overhead
         * - Extra object allocation
         * - GC pressure
         * 
         * With specialization:
         * - No boxing
         * - Primitive operations
         * - Better performance
         * 
         * WHEN TO USE:
         * 
         * Use primitive specializations when:
         * ✓ Working with primitives
         * ✓ Performance critical
         * ✓ High volume operations
         * ✓ Avoiding GC pressure
         * 
         * Use generic when:
         * ✓ Working with objects
         * ✓ Need flexibility
         * ✓ Performance not critical
         */
    }
    
    /**
     * Composing functions
     */
    public static void composingFunctions() {
        System.out.println("\n=== Composing Functions ===");
        
        Function<String, String> trim = String::trim;
        Function<String, String> toLowerCase = String::toLowerCase;
        Function<String, Integer> length = String::length;
        
        // andThen - apply this, then that
        Function<String, String> trimAndLower = trim.andThen(toLowerCase);
        System.out.println("Trim and lower: '" + trimAndLower.apply("  HELLO  ") + "'");
        
        // compose - apply that, then this
        Function<String, Integer> lengthAfterTrim = length.compose(trim);
        System.out.println("Length after trim: " + lengthAfterTrim.apply("  HELLO  "));
        
        // Chaining
        Function<String, Integer> pipeline = trim
            .andThen(toLowerCase)
            .andThen(length);
        System.out.println("Full pipeline: " + pipeline.apply("  HELLO WORLD  "));
        
        // Predicate composition
        Predicate<String> isLong = s -> s.length() > 5;
        Predicate<String> hasSpace = s -> s.contains(" ");
        
        Predicate<String> isLongAndHasSpace = isLong.and(hasSpace);
        Predicate<String> isLongOrHasSpace = isLong.or(hasSpace);
        Predicate<String> isNotLong = isLong.negate();
        
        System.out.println("\nPredicate composition:");
        System.out.println("'Hello World' is long AND has space: " + 
            isLongAndHasSpace.test("Hello World"));
        System.out.println("'Hi' is long OR has space: " + 
            isLongOrHasSpace.test("Hi"));
        
        /**
         * FUNCTION COMPOSITION:
         * 
         * Function<T, R>:
         * - andThen(after)  - f.andThen(g) = g(f(x))
         * - compose(before) - f.compose(g) = f(g(x))
         * 
         * Predicate<T>:
         * - and(other)   - this AND that
         * - or(other)    - this OR that
         * - negate()     - NOT this
         * 
         * Consumer<T>:
         * - andThen(after) - do this, then that
         */
    }
    
    public static void main(String[] args) {
        customFunctionalInterface();
        builtInFunctionalInterfaces();
        primitiveSpecializations();
        composingFunctions();
    }
}
```

## 2. Lambda Syntax Variations

### All Lambda Forms

```java
/**
 * LAMBDA SYNTAX VARIATIONS
 * 
 * Different ways to write lambda expressions
 */

import java.util.function.*;
import java.util.*;

public class LambdaSyntaxVariations {
    
    /**
     * Basic syntax
     */
    public static void basicSyntax() {
        System.out.println("=== Basic Lambda Syntax ===");
        
        // No parameters, no return
        Runnable noParams = () -> System.out.println("No parameters");
        noParams.run();
        
        // Single parameter, inferred type
        Consumer<String> oneParam = s -> System.out.println("One param: " + s);
        oneParam.accept("Hello");
        
        // Single parameter, explicit type
        Consumer<String> oneParamTyped = (String s) -> System.out.println("Typed: " + s);
        oneParamTyped.accept("World");
        
        // Multiple parameters, inferred types
        BiFunction<Integer, Integer, Integer> twoParams = (a, b) -> a + b;
        System.out.println("Two params: " + twoParams.apply(5, 3));
        
        // Multiple parameters, explicit types
        BiFunction<Integer, Integer, Integer> twoParamsTyped = 
            (Integer a, Integer b) -> a + b;
        System.out.println("Typed params: " + twoParamsTyped.apply(5, 3));
        
        /**
         * PARAMETER SYNTAX:
         * 
         * Zero parameters:
         * () -> expression
         * 
         * One parameter (type inferred):
         * x -> expression
         * (x) -> expression  // Parentheses optional
         * 
         * One parameter (explicit type):
         * (String x) -> expression  // Parentheses required
         * 
         * Multiple parameters:
         * (x, y) -> expression
         * (String x, String y) -> expression
         * 
         * RULES:
         * - Parentheses required for 0 or 2+ parameters
         * - Parentheses optional for 1 parameter (no type)
         * - Parentheses required for 1 parameter (with type)
         * - All parameters typed or all inferred
         */
    }
    
    /**
     * Expression vs block body
     */
    public static void expressionVsBlock() {
        System.out.println("\n=== Expression vs Block Body ===");
        
        // Expression body - implicit return
        Function<Integer, Integer> squareExpr = x -> x * x;
        System.out.println("Square (expression): " + squareExpr.apply(5));
        
        // Block body - explicit return
        Function<Integer, Integer> squareBlock = x -> {
            int result = x * x;
            return result;
        };
        System.out.println("Square (block): " + squareBlock.apply(5));
        
        // Expression body - method call
        Consumer<String> printExpr = System.out::println;
        printExpr.accept("Expression style");
        
        // Block body - multiple statements
        Consumer<String> printBlock = s -> {
            System.out.println("Processing: " + s);
            System.out.println("Length: " + s.length());
        };
        printBlock.accept("Block style");
        
        /**
         * EXPRESSION VS BLOCK:
         * 
         * Expression body:
         * x -> expression
         * - Single expression
         * - Implicit return
         * - No braces
         * - No semicolon
         * 
         * Block body:
         * x -> { statements; }
         * - Multiple statements
         * - Explicit return (if needed)
         * - Braces required
         * - Semicolons required
         * 
         * WHEN TO USE:
         * 
         * Expression:
         * ✓ Simple operations
         * ✓ Single line
         * ✓ Cleaner syntax
         * 
         * Block:
         * ✓ Multiple statements
         * ✓ Local variables
         * ✓ Control flow
         */
    }
    
    /**
     * Type inference
     */
    public static void typeInference() {
        System.out.println("\n=== Type Inference ===");
        
        // Compiler infers types from context
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
        
        // Type inferred from List<String>
        names.forEach(name -> System.out.println("  " + name));
        
        // Explicit types (rarely needed)
        names.forEach((String name) -> System.out.println("  " + name));
        
        // Complex inference
        Map<String, Integer> map = new HashMap<>();
        map.put("Alice", 30);
        map.put("Bob", 25);
        
        // Both K and V inferred
        map.forEach((k, v) -> System.out.println("  " + k + " -> " + v));
        
        /**
         * TYPE INFERENCE:
         * 
         * Compiler determines types from:
         * - Target type (functional interface)
         * - Generic type parameters
         * - Method signature
         * 
         * Example:
         * List<String>.forEach(Consumer<? super String>)
         * 
         * Lambda: name -> ...
         * Inferred: (String name) -> ...
         * 
         * BENEFITS:
         * ✓ Less verbose
         * ✓ Cleaner code
         * ✓ Less typing
         * 
         * WHEN TO USE EXPLICIT TYPES:
         * - Type not obvious
         * - Improve readability
         * - Complex generics
         */
    }
    
    /**
     * Special cases
     */
    public static void specialCases() {
        System.out.println("\n=== Special Cases ===");
        
        // Underscore for unused parameters (Java 9+)
        BiFunction<String, Integer, String> ignoreSecond = (s, _) -> s;
        System.out.println("Ignore second: " + ignoreSecond.apply("Hello", 42));
        
        // Var parameters (Java 11+)
        BiFunction<String, String, String> concat = (var a, var b) -> a + b;
        System.out.println("Var params: " + concat.apply("Hello", "World"));
        
        // Parentheses required for single parameter with var
        Function<String, Integer> length = (var s) -> s.length();
        System.out.println("Single var: " + length.apply("Test"));
        
        /**
         * SPECIAL SYNTAX:
         * 
         * Underscore (Java 9+):
         * (x, _) -> x
         * - Marks unused parameter
         * - Documentation
         * - Warning suppression
         * 
         * Var (Java 11+):
         * (var x, var y) -> x + y
         * - Local variable syntax
         * - All or none
         * - Annotations allowed: (@NonNull var x)
         * 
         * LIMITATIONS:
         * - Cannot mix var and explicit types
         * - Cannot use _ as parameter name (Java 9+)
         * - var requires parentheses even for single parameter
         */
    }
    
    /**
     * Common mistakes
     */
    public static void commonMistakes() {
        /**
         * COMMON LAMBDA SYNTAX MISTAKES:
         * 
         * 1. MISSING PARENTHESES:
         *    ✗ x, y -> x + y          // Error
         *    ✓ (x, y) -> x + y        // Correct
         * 
         * 2. MIXING TYPED AND INFERRED:
         *    ✗ (String x, y) -> x + y // Error
         *    ✓ (String x, String y) -> x + y
         *    ✓ (x, y) -> x + y
         * 
         * 3. MISSING BRACES FOR MULTIPLE STATEMENTS:
         *    ✗ x -> System.out.println(x); return x;  // Error
         *    ✓ x -> { System.out.println(x); return x; }
         * 
         * 4. MISSING RETURN IN BLOCK:
         *    ✗ x -> { x * x }         // Error (non-void)
         *    ✓ x -> { return x * x; }
         *    ✓ x -> x * x
         * 
         * 5. RETURN IN EXPRESSION BODY:
         *    ✗ x -> return x * x      // Error
         *    ✓ x -> x * x
         * 
         * 6. SEMICOLON IN EXPRESSION:
         *    ✗ x -> x * x;            // Error
         *    ✓ x -> x * x
         * 
         * 7. WRONG ARROW:
         *    ✗ x => x * x             // Error (JavaScript syntax)
         *    ✓ x -> x * x
         */
    }
    
    public static void main(String[] args) {
        basicSyntax();
        expressionVsBlock();
        typeInference();
        specialCases();
    }
}
```

## 3. Method References

### Four Types of Method References

```java
/**
 * METHOD REFERENCES
 * 
 * Shorthand for lambda expressions
 */

import java.util.*;
import java.util.function.*;

public class MethodReferences {
    
    /**
     * Static method references
     */
    public static void staticMethodReferences() {
        System.out.println("=== Static Method References ===");
        
        // Lambda version
        Function<String, Integer> lambda = s -> Integer.parseInt(s);
        System.out.println("Lambda: " + lambda.apply("123"));
        
        // Method reference version
        Function<String, Integer> methodRef = Integer::parseInt;
        System.out.println("Method ref: " + methodRef.apply("456"));
        
        // More examples
        BiFunction<Double, Double, Double> max = Math::max;
        System.out.println("Max: " + max.apply(3.14, 2.71));
        
        Supplier<Double> random = Math::random;
        System.out.println("Random: " + random.get());
        
        /**
         * STATIC METHOD REFERENCE:
         * 
         * Syntax: ClassName::staticMethod
         * 
         * Lambda equivalent:
         * (args) -> ClassName.staticMethod(args)
         * 
         * Examples:
         * Integer::parseInt     = s -> Integer.parseInt(s)
         * Math::max            = (a, b) -> Math.max(a, b)
         * Collections::sort    = list -> Collections.sort(list)
         */
    }
    
    /**
     * Instance method references (bound)
     */
    public static void instanceMethodReferencesBound() {
        System.out.println("\n=== Instance Method References (Bound) ===");
        
        String str = "Hello World";
        
        // Lambda version
        Supplier<String> lambda = () -> str.toUpperCase();
        System.out.println("Lambda: " + lambda.get());
        
        // Method reference version (bound to str)
        Supplier<String> methodRef = str::toUpperCase;
        System.out.println("Method ref: " + methodRef.get());
        
        // More examples
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
        
        // Bound to names instance
        Supplier<Integer> getSize = names::size;
        System.out.println("Size: " + getSize.get());
        
        /**
         * BOUND INSTANCE METHOD REFERENCE:
         * 
         * Syntax: instance::instanceMethod
         * 
         * Lambda equivalent:
         * (args) -> instance.instanceMethod(args)
         * 
         * Bound = specific instance captured
         * 
         * Examples:
         * str::toUpperCase     = () -> str.toUpperCase()
         * list::add           = item -> list.add(item)
         * System.out::println = msg -> System.out.println(msg)
         */
    }
    
    /**
     * Instance method references (unbound)
     */
    public static void instanceMethodReferencesUnbound() {
        System.out.println("\n=== Instance Method References (Unbound) ===");
        
        // Lambda version
        Function<String, String> lambda = s -> s.toUpperCase();
        System.out.println("Lambda: " + lambda.apply("hello"));
        
        // Method reference version (unbound)
        Function<String, String> methodRef = String::toUpperCase;
        System.out.println("Method ref: " + methodRef.apply("world"));
        
        // First parameter becomes receiver
        BiFunction<String, String, Boolean> startsWith = String::startsWith;
        System.out.println("'Hello' starts with 'He': " + 
            startsWith.apply("Hello", "He"));
        
        // Comparator example
        List<String> names = Arrays.asList("Charlie", "Alice", "Bob");
        names.sort(String::compareToIgnoreCase);
        System.out.println("Sorted: " + names);
        
        /**
         * UNBOUND INSTANCE METHOD REFERENCE:
         * 
         * Syntax: ClassName::instanceMethod
         * 
         * Lambda equivalent:
         * (instance, args) -> instance.instanceMethod(args)
         * 
         * Unbound = no specific instance
         * First parameter IS the instance
         * 
         * Examples:
         * String::toUpperCase       = s -> s.toUpperCase()
         * String::length           = s -> s.length()
         * String::compareToIgnoreCase = (a, b) -> a.compareToIgnoreCase(b)
         */
    }
    
    /**
     * Constructor references
     */
    public static void constructorReferences() {
        System.out.println("\n=== Constructor References ===");
        
        // No-arg constructor
        Supplier<ArrayList<String>> listSupplier = ArrayList::new;
        List<String> list1 = listSupplier.get();
        System.out.println("Empty list: " + list1);
        
        // Constructor with parameter
        Function<Integer, ArrayList<String>> sizedListSupplier = ArrayList::new;
        List<String> list2 = sizedListSupplier.get(10);
        System.out.println("Sized list capacity: " + list2.size());
        
        // Array constructor
        IntFunction<String[]> arrayCreator = String[]::new;
        String[] array = arrayCreator.apply(5);
        System.out.println("Array length: " + array.length);
        
        // Custom class
        class Person {
            String name;
            int age;
            
            Person(String name, int age) {
                this.name = name;
                this.age = age;
            }
            
            @Override
            public String toString() {
                return name + "(" + age + ")";
            }
        }
        
        BiFunction<String, Integer, Person> personCreator = Person::new;
        Person p = personCreator.apply("Alice", 30);
        System.out.println("Person: " + p);
        
        /**
         * CONSTRUCTOR REFERENCE:
         * 
         * Syntax: ClassName::new
         * 
         * Lambda equivalent:
         * (args) -> new ClassName(args)
         * 
         * Matches constructor by signature
         * 
         * Examples:
         * ArrayList::new           = () -> new ArrayList<>()
         * ArrayList::new           = size -> new ArrayList<>(size)
         * String[]::new           = size -> new String[size]
         * Person::new             = (name, age) -> new Person(name, age)
         */
    }
    
    /**
     * When to use method references
     */
    public static void whenToUseMethodReferences() {
        /**
         * USE METHOD REFERENCES WHEN:
         * 
         * ✓ Method already exists
         * ✓ Lambda just calls method
         * ✓ No additional logic
         * ✓ Improves readability
         * 
         * Example:
         * names.forEach(System.out::println)
         * vs
         * names.forEach(name -> System.out.println(name))
         * 
         * 
         * USE LAMBDAS WHEN:
         * 
         * ✓ Additional logic needed
         * ✓ Multiple statements
         * ✓ Reordering parameters
         * ✓ Method reference unclear
         * 
         * Example:
         * names.forEach(name -> System.out.println("Name: " + name))
         * 
         * 
         * READABILITY:
         * 
         * More readable:
         * list.sort(String::compareToIgnoreCase)
         * 
         * Less readable:
         * list.sort((a, b) -> a.compareToIgnoreCase(b))
         * 
         * But sometimes lambda is clearer:
         * numbers.forEach(n -> process(n * 2))
         * vs
         * // No simple method reference
         */
    }
    
    /**
     * Method reference vs lambda performance
     */
    public static void methodReferencePerformance() {
        /**
         * PERFORMANCE:
         * 
         * Method references and lambdas have
         * identical performance characteristics
         * 
         * Both use invokedynamic
         * Both optimize to same bytecode
         * 
         * Choose based on readability,
         * not performance
         * 
         * 
         * COMPILATION:
         * 
         * Method reference:
         * list.forEach(System.out::println)
         * 
         * Lambda:
         * list.forEach(x -> System.out.println(x))
         * 
         * Both compile to:
         * invokedynamic ...
         * 
         * Same target MethodHandle
         * Identical runtime behavior
         */
    }
    
    public static void main(String[] args) {
        staticMethodReferences();
        instanceMethodReferencesBound();
        instanceMethodReferencesUnbound();
        constructorReferences();
    }
}
```

## 4. Capturing Variables (Effectively Final)

### Variable Capture Mechanics

```java
/**
 * CAPTURING VARIABLES
 * 
 * How lambdas capture variables from enclosing scope
 */

import java.util.*;
import java.util.function.*;

public class CapturingVariables {
    
    /**
     * Effectively final requirement
     */
    public static void effectivelyFinalRequirement() {
        System.out.println("=== Effectively Final ===");
        
        // Final variable - can capture
        final String prefix = "Hello";
        Consumer<String> printer1 = s -> System.out.println(prefix + " " + s);
        printer1.accept("World");
        
        // Effectively final - can capture
        String suffix = "!";  // Never modified
        Consumer<String> printer2 = s -> System.out.println(s + suffix);
        printer2.accept("Test");
        
        // Not effectively final - CANNOT capture
        String mutable = "Initial";
        // This would fail:
        // Consumer<String> printer3 = s -> System.out.println(mutable + s);
        // mutable = "Changed";  // Error: variable is not effectively final
        
        /**
         * EFFECTIVELY FINAL:
         * 
         * Definition:
         * Variable never assigned after initialization
         * 
         * final int x = 5;           // Final
         * int y = 10;                // Effectively final (if never reassigned)
         * int z = 15; z = 20;        // Not effectively final
         * 
         * 
         * WHY REQUIRED:
         * 
         * Lambdas can outlive scope
         * Must capture value, not variable
         * 
         * Example:
         * Runnable r;
         * {
         *   int value = 42;
         *   r = () -> System.out.println(value);
         * }  // value goes out of scope
         * r.run();  // But lambda still has it!
         * 
         * 
         * COMPILER CHECK:
         * 
         * If variable could change:
         * - Lambda behavior unpredictable
         * - Thread safety issues
         * - Compiler error prevents bugs
         */
    }
    
    /**
     * What can be captured
     */
    public static void whatCanBeCaptured() {
        System.out.println("\n=== What Can Be Captured ===");
        
        // Local variables (effectively final)
        String local = "local";
        Runnable r1 = () -> System.out.println("Local: " + local);
        r1.run();
        
        // Parameters (always effectively final)
        Consumer<String> showParam = param -> {
            Runnable r = () -> System.out.println("Param: " + param);
            r.run();
        };
        showParam.accept("parameter");
        
        // Instance fields (always accessible)
        CaptureExample example = new CaptureExample();
        example.instanceFieldCapture();
        
        // Static fields (always accessible)
        CaptureExample.staticFieldCapture();
        
        /**
         * CAPTURE RULES:
         * 
         * CAN capture:
         * ✓ Effectively final local variables
         * ✓ Parameters (always effectively final)
         * ✓ Instance fields (via 'this')
         * ✓ Static fields
         * 
         * CANNOT capture:
         * ✗ Non-final local variables
         * ✗ Non-final parameters (conceptually - they are final)
         * 
         * 
         * 'this' REFERENCE:
         * 
         * Lambda captures 'this' from enclosing class
         * Not from lambda itself (lambda has no 'this')
         * 
         * class Example {
         *   void method() {
         *     Runnable r = () -> {
         *       this.doSomething();  // 'this' = Example instance
         *     };
         *   }
         * }
         */
    }
    
    static class CaptureExample {
        private String instanceField = "instance field";
        private static String staticField = "static field";
        
        public void instanceFieldCapture() {
            // Can always access instance fields
            Runnable r = () -> System.out.println("Instance field: " + instanceField);
            r.run();
            
            // 'this' reference
            Runnable r2 = () -> System.out.println("Via this: " + this.instanceField);
            r2.run();
        }
        
        public static void staticFieldCapture() {
            // Can always access static fields
            Runnable r = () -> System.out.println("Static field: " + staticField);
            r.run();
        }
    }
    
    /**
     * Workarounds for mutable variables
     */
    public static void mutableVariableWorkarounds() {
        System.out.println("\n=== Mutable Variable Workarounds ===");
        
        // Problem: Want to modify variable in lambda
        // int counter = 0;
        // list.forEach(s -> counter++);  // ERROR!
        
        // Workaround 1: Use array (mutable container)
        int[] counter = {0};
        List<String> list1 = Arrays.asList("a", "b", "c");
        list1.forEach(s -> counter[0]++);
        System.out.println("Array counter: " + counter[0]);
        
        // Workaround 2: Use AtomicInteger
        java.util.concurrent.atomic.AtomicInteger atomicCounter = 
            new java.util.concurrent.atomic.AtomicInteger(0);
        List<String> list2 = Arrays.asList("a", "b", "c");
        list2.forEach(s -> atomicCounter.incrementAndGet());
        System.out.println("Atomic counter: " + atomicCounter.get());
        
        // Workaround 3: Use mutable object
        class Counter {
            int value = 0;
        }
        Counter objCounter = new Counter();
        List<String> list3 = Arrays.asList("a", "b", "c");
        list3.forEach(s -> objCounter.value++);
        System.out.println("Object counter: " + objCounter.value);
        
        /**
         * WORKAROUNDS:
         * 
         * 1. ARRAY:
         *    int[] x = {0};
         *    lambda -> x[0]++;
         *    
         *    Pros: Simple
         *    Cons: Ugly, not thread-safe
         * 
         * 2. ATOMIC:
         *    AtomicInteger x = new AtomicInteger(0);
         *    lambda -> x.incrementAndGet();
         *    
         *    Pros: Thread-safe
         *    Cons: Overhead, overkill for single-threaded
         * 
         * 3. MUTABLE OBJECT:
         *    class Holder { int value; }
         *    Holder x = new Holder();
         *    lambda -> x.value++;
         *    
         *    Pros: Clean, flexible
         *    Cons: Extra class, not thread-safe
         * 
         * BEST PRACTICE:
         * 
         * Avoid mutation in lambdas
         * Use Stream operations instead:
         * long count = list.stream().count();
         */
    }
    
    /**
     * Capturing in loops (common mistake)

    public static void capturingInLoops() {
        System.out.println("\n=== Capturing in Loops ===");
        
        List<Runnable> runnables = new ArrayList<>();

        for (int i = 0; i < 3; i++) {
            final int captured = i;  // Effectively final
            runnables.add(() -> System.out.println("  Value: " + captured));
        }

        runnables.forEach(Runnable::run);

         * LOOP CAPTURE:
         * 
         * Each iteration creates new variable
         * Lambda captures that iteration's value
         * 
         * for (int i = 0; i < 3; i++) {
         *   final int captured = i;  // New variable each iteration
         *   lambda -> use captured   // Captures current value
         * }
         * 
         * Prints: 0, 1, 2 (as expected)

         * 
         * JAVASCRIPT COMPARISON:
         * 
         * JavaScript (var - before ES6):
         * for (var i = 0; i < 3; i++) {
         *   callbacks.push(() => console.log(i));
         * }
         * Prints: 3, 3, 3 (all reference same variable!)
         * 
         * Java doesn't have this problem
         * Each loop iteration = new scope
         */
    }

        /**

         * Runnable r = () -> System.out.println("Hello");
         * 
         * - Singleton instance
         * - No captured state
         * - Very efficient
         * 
         *

         * But mutable objects can change:
         * List<String> list = new ArrayList<>();
         * Runnable r = () -> list.add("x");  // Not thread-safe!
         * 
         * Reference is final, content is not
         */
    }
```

```java
         * 
         * Intersection type cast:
         * (FunctionalInterface & Serializable) lambda
         * 
         * Example:
         * Runnable r = (Runnable & Serializable) () -> { };
         *

         * - FunctionalInterface (for lambda)
         * - Serializable (for serialization)

         * }
```

```java
import java.util.*;
import java.util.function.*;

public class LambdaPerformance {
    
    /**
     * Creation overhead
     */
    public static void creationOverhead() {

        long start = System.nanoTime();

            Runnable r = () -> {};  // Reuses same instance
        }
        long nonCapturingTime = System.nanoTime() - start;
        
        // Capturing lambda (new instance each time)
        String msg = "Hello";
        start = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            Runnable r = () -> System.out.println(msg);  // New instance
        }
        long capturingTime = System.nanoTime() - start;
        
        // Anonymous class
        start = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            Runnable r = new Runnable() {
                public void run() {}
            };
        }
        long anonymousTime = System.nanoTime() - start;
        
        System.out.println("=== Creation Overhead (" + iterations + " iterations) ===");
        System.out.println("Non-capturing lambda: " + nonCapturingTime / 1_000_000 + " ms");
        System.out.println("Capturing lambda:     " + capturingTime / 1_000_000 + " ms");
        System.out.println("Anonymous class:      " + anonymousTime / 1_000_000 + " ms");
        
        /**
         * CREATION PERFORMANCE:
         * 
         * Non-capturing lambda:
         * - Singleton instance
         * - Cached by JVM
         * - Essentially free (~1ns)
         * 
         * Capturing lambda:
         * - New instance per creation
         * - Factory method call
         * - ~10-20ns per instance
         * 
         * Anonymous class:
         * - New instance per creation
         * - Constructor call
         * - ~15-30ns per instance
         * 
         * Typical results:
         * Non-capturing: 10 ms (cached)
         * Capturing:     150 ms (allocation)
         * Anonymous:     200 ms (allocation + init)
         */
    }
    
    /**
     * Invocation overhead
     */
    public static void invocationOverhead() {
        int iterations = 100_000_000;
        
        // Direct method call
        long start = System.nanoTime();
        int sum1 = 0;
        for (int i = 0; i < iterations; i++) {
            sum1 += directMethod(i);
        }
        long directTime = System.nanoTime() - start;
        
        // Lambda invocation
        IntUnaryOperator lambda = x -> x * 2;
        start = System.nanoTime();
        int sum2 = 0;
        for (int i = 0; i < iterations; i++) {
            sum2 += lambda.applyAsInt(i);
        }
        long lambdaTime = System.nanoTime() - start;
        
        // Anonymous class invocation
        IntUnaryOperator anonymous = new IntUnaryOperator() {
            public int applyAsInt(int x) { return x * 2; }
        };
        start = System.nanoTime();
        int sum3 = 0;
        for (int i = 0; i < iterations; i++) {
            sum3 += anonymous.applyAsInt(i);
        }
        long anonymousTime = System.nanoTime() - start;
        
        System.out.println("\n=== Invocation Overhead (" + iterations + " iterations) ===");
        System.out.println("Direct call:     " + directTime / 1_000_000 + " ms");
        System.out.println("Lambda:          " + lambdaTime / 1_000_000 + " ms");
        System.out.println("Anonymous class: " + anonymousTime / 1_000_000 + " ms");
        System.out.println("Sums: " + sum1 + ", " + sum2 + ", " + sum3);
        
        /**
         * INVOCATION PERFORMANCE:
         * 
         * After JIT warmup:
         * - Direct ≈ Lambda ≈ Anonymous
         * - All inline to same code
         * - No measurable difference
         * 
         * Cold/interpreted:
         * - Direct: fastest
         * - Lambda: medium
         * - Anonymous: slowest
         *

         * Lambda:    52 ms (4% slower)

    /**
     * Primitive specializations performance
     */
    public static void primitiveSpecializationsPerformance() {
        int iterations = 100_000_000;
        
        // Generic (boxing)
        Function<Integer, Integer> generic = x -> x * 2;
        long start = System.nanoTime();
        int sum1 = 0;
        for (int i = 0; i < iterations; i++) {
            sum1 += generic.apply(i);  // Boxing: i → Integer
        }
        long genericTime = System.nanoTime() - start;
        
        // Primitive specialization (no boxing)
        IntUnaryOperator primitive = x -> x * 2;
        start = System.nanoTime();
        int sum2 = 0;
        for (int i = 0; i < iterations; i++) {
            sum2 += primitive.applyAsInt(i);  // No boxing!
        }
        long primitiveTime = System.nanoTime() - start;
        
        System.out.println("\n=== Primitive Specializations (" + iterations + " iterations) ===");
        System.out.println("Generic (boxing):    " + genericTime / 1_000_000 + " ms");
        System.out.println("Primitive (no box):  " + primitiveTime / 1_000_000 + " ms");
        System.out.println("Speedup:             " + 
            String.format("%.1fx", genericTime / (double) primitiveTime));
        
        /**
         * PRIMITIVE SPECIALIZATION BENEFIT:
         * 
         * Generic version:
         * - Boxing on every call
         * - Integer allocation
         * - GC pressure
         * - 2-5x slower
         * 
         * Primitive version:
         * - No allocation
         * - Direct primitive ops
         * - No GC impact
         * - Fast
         * 
         * Typical results:
         * Generic:   2000 ms (boxing overhead)
         * Primitive:  800 ms (2.5x faster)
         * 
         * ALWAYS use primitive specializations
         * for performance-critical code!
         */
    }

    public static void performanceBestPractices() {
        /**
         * PERFORMANCE BEST PRACTICES:
         * 
         * 1. PREFER NON-CAPTURING:

         *    ✗ () -> variable

         * 2. USE PRIMITIVE SPECIALIZATIONS:
```

## Summary

### Quick Reference

**Lambda Syntax:**

```java
// No parameters
() -> expression
() -> { statements; }

// One parameter
x -> expression
(x) -> expression
(Type x) -> expression

// Multiple parameters
(x, y) -> expression
(Type x, Type y) -> { statements; }
```

**Method References:**

```java
// Static method
ClassName::staticMethod

// Instance method (bound)
instance::instanceMethod

// Instance method (unbound)
ClassName::instanceMethod

// Constructor
ClassName::new
Array::new
```

**Built-in Functional Interfaces:**

```
Function<T, R>     - T → R
Predicate<T>       - T → boolean
Consumer<T>        - T → void
Supplier<T>        - () → T
BiFunction<T,U,R>  - (T, U) → R
UnaryOperator<T>   - T → T
BinaryOperator<T>  - (T, T) → T
```

### Lambda vs Anonymous Class

```
┌────────────────────┬──────────────┬───────────────────┐
│ Feature            │ Lambda       │ Anonymous Class   │
├────────────────────┼──────────────┼───────────────────┤
│ Syntax             │ Concise      │ Verbose           │
│ 'this' reference   │ Enclosing    │ Own instance      │
│ .class file        │ No           │ Yes (Class$1)     │
│ Implementation     │ invokedynamic│ new + constructor │
│ Own fields/methods │ No           │ Yes               │
│ Variable shadowing │ No           │ Yes               │
│ Performance        │ Same         │ Same (after JIT)  │
│ Memory (cold)      │ Less         │ More              │
└────────────────────┴──────────────┴───────────────────┘
```

### Performance Summary

```
CREATION (10M iterations):
Non-capturing lambda:    10 ms  (singleton)
Capturing lambda:       150 ms  (new instance)
Anonymous class:        200 ms  (new + init)

INVOCATION (100M iterations, after warmup):
Direct call:             50 ms  (baseline)
Lambda:                  52 ms  (4% slower)
Anonymous class:         53 ms  (6% slower)
→ Difference negligible!

PRIMITIVE SPECIALIZATION (100M):
Generic (boxing):      2000 ms
Primitive (no box):     800 ms
→ 2.5x speedup!
```

### Best Practices Summary

**✅ DO:**

- Use lambdas for functional interfaces
- Prefer method references when clear
- Use primitive specializations (IntPredicate, etc.)
- Make variables effectively final
- Reuse lambda instances in loops
- Let JIT warm up before measuring
- Use non-capturing when possible

**❌ DON'T:**

- Modify captured variables
- Create lambdas in hot loops
- Use generic types for primitives
- Serialize lambdas unless necessary
- Shadow variable names
- Micro-optimize without measuring
- Use lambdas with multiple abstract methods
