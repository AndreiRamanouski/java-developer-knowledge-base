# Default and Static Methods in Interfaces

## Overview

Java 8 introduced default and static methods in interfaces, fundamentally changing how interfaces work. Default methods provide concrete implementations in interfaces, enabling multiple inheritance of behavior and allowing interfaces to evolve without breaking existing implementations. Static methods in interfaces provide utility functions associated with the interface contract. This guide covers default method syntax and semantics, multiple inheritance of behavior through interfaces, diamond problem resolution rules, static method usage, practical interface evolution strategies, and the defender methods pattern for backward compatibility.

## 1. Default Method Syntax

### Writing Default Methods in Interfaces

```java
/**
 * DEFAULT METHOD SYNTAX
 * 
 * Default methods provide concrete implementations in interfaces
 */

import java.util.*;

public class DefaultMethodSyntax {
    
    /**
     * Basic default method
     */
    interface Greeting {
        // Abstract method (must be implemented)
        String getName();
        
        // Default method (can be overridden, has implementation)
        default void greet() {
            System.out.println("Hello, " + getName() + "!");
        }
        
        // Another default method
        default void greetFormal() {
            System.out.println("Good day, " + getName() + ".");
        }
    }
    
    static class Person implements Greeting {
        private String name;
        
        Person(String name) {
            this.name = name;
        }
        
        @Override
        public String getName() {
            return name;
        }
        
        // Can use default methods without overriding
    }
    
    public static void basicDefaultMethod() {
        System.out.println("=== Basic Default Method ===");
        
        Person person = new Person("Alice");
        person.greet();         // Uses default implementation
        person.greetFormal();   // Uses default implementation
        
        /**
         * DEFAULT METHOD RULES:
         * 
         * Syntax:
         * default ReturnType methodName(params) {
         *   // implementation
         * }
         * 
         * Characteristics:
         * - Must have 'default' keyword
         * - Must have method body
         * - Can be overridden by implementing class
         * - Can call other interface methods
         * - Cannot override Object methods (equals, hashCode, toString)
         * 
         * Access:
         * - public by default (interfaces)
         * - Cannot be private (before Java 9)
         * - Cannot be protected
         * - Cannot be final
         */
    }
    
    /**
     * Overriding default methods
     */
    interface Vehicle {
        default void start() {
            System.out.println("Vehicle starting...");
        }
        
        default void stop() {
            System.out.println("Vehicle stopping...");
        }
    }
    
    static class Car implements Vehicle {
        @Override
        public void start() {
            System.out.println("Car engine starting...");
        }
        
        // stop() uses default implementation
    }
    
    public static void overridingDefaults() {
        System.out.println("\n=== Overriding Default Methods ===");
        
        Car car = new Car();
        car.start();  // Overridden
        car.stop();   // Default
        
        /**
         * OVERRIDING RULES:
         * 
         * 1. Optional override:
         *    Can use default or override
         * 
         * 2. Complete override:
         *    public void method() {
         *      // new implementation
         *    }
         * 
         * 3. Call super:
         *    InterfaceName.super.method()
         * 
         * 4. Make abstract:
         *    Can redeclare as abstract to force implementation
         */
    }
    
    /**
     * Calling default methods from implementing class
     */
    interface Calculator {
        default int add(int a, int b) {
            return a + b;
        }
        
        default int multiply(int a, int b) {
            return a * b;
        }
    }
    
    static class AdvancedCalculator implements Calculator {
        @Override
        public int add(int a, int b) {
            System.out.println("Adding " + a + " + " + b);
            return Calculator.super.add(a, b);  // Call default
        }
        
        // Can use multiply() default as-is
    }
    
    public static void callingDefaultMethods() {
        System.out.println("\n=== Calling Default Methods ===");
        
        AdvancedCalculator calc = new AdvancedCalculator();
        System.out.println("Result: " + calc.add(5, 3));
        System.out.println("Result: " + calc.multiply(5, 3));
        
        /**
         * CALLING SUPER DEFAULT:
         * 
         * Syntax:
         * InterfaceName.super.methodName()
         * 
         * Use when:
         * - Want to enhance default behavior
         * - Need to delegate to default
         * - Resolving multiple inheritance
         * 
         * Note:
         * Cannot use super.method() (ambiguous)
         * Must specify interface name
         */
    }
    
    /**
     * Default methods calling abstract methods
     */
    interface Validator {
        // Abstract method
        boolean isValid();
        
        // Default method using abstract method
        default void validate() {
            if (!isValid()) {
                throw new IllegalStateException("Invalid state");
            }
        }
        
        default String getStatus() {
            return isValid() ? "Valid" : "Invalid";
        }
    }
    
    static class User implements Validator {
        private String name;
        
        User(String name) {
            this.name = name;
        }
        
        @Override
        public boolean isValid() {
            return name != null && !name.isEmpty();
        }
    }
    
    public static void defaultMethodsUsingAbstract() {
        System.out.println("\n=== Default Methods Using Abstract ===");
        
        User validUser = new User("Alice");
        System.out.println("Status: " + validUser.getStatus());
        validUser.validate();  // OK
        
        User invalidUser = new User("");
        System.out.println("Status: " + invalidUser.getStatus());
        try {
            invalidUser.validate();
        } catch (IllegalStateException e) {
            System.out.println("Validation failed: " + e.getMessage());
        }
        
        /**
         * DEFAULT METHODS AND ABSTRACT:
         * 
         * Default methods can:
         * - Call abstract methods
         * - Provide template behavior
         * - Use abstract as hooks
         * 
         * Pattern:
         * Abstract method: "What to do"
         * Default method: "How to do it"
         * 
         * Similar to template method pattern
         */
    }
    
    public static void main(String[] args) {
        basicDefaultMethod();
        overridingDefaults();
        callingDefaultMethods();
        defaultMethodsUsingAbstract();
    }
}
```

## 2. Multiple Inheritance of Behavior

### Implementing Multiple Interfaces with Defaults

```java
/**
 * MULTIPLE INHERITANCE OF BEHAVIOR
 * 
 * Classes can inherit behavior from multiple interfaces
 */

import java.util.*;

public class MultipleInheritanceBehavior {
    
    /**
     * Basic multiple inheritance
     */
    interface Flyable {
        default void fly() {
            System.out.println("Flying...");
        }
    }
    
    interface Swimmable {
        default void swim() {
            System.out.println("Swimming...");
        }
    }
    
    static class Duck implements Flyable, Swimmable {
        // Inherits both fly() and swim()
    }
    
    public static void basicMultipleInheritance() {
        System.out.println("=== Basic Multiple Inheritance ===");
        
        Duck duck = new Duck();
        duck.fly();   // From Flyable
        duck.swim();  // From Swimmable
        
        /**
         * MULTIPLE INHERITANCE:
         * 
         * Java classes can implement multiple interfaces
         * Each interface can provide default methods
         * Class inherits all default methods
         * 
         * Benefits:
         * - Reuse behavior from multiple sources
         * - Mix-in pattern
         * - Flexible composition
         * 
         * vs C++ multiple inheritance:
         * - No state inheritance (interfaces can't have fields)
         * - Clearer conflict resolution
         * - No "deadly diamond of death" for state
         */
    }
    
    /**
     * Combining behaviors
     */
    interface Loggable {
        default void log(String message) {
            System.out.println("[LOG] " + getTimestamp() + ": " + message);
        }
        
        default String getTimestamp() {
            return String.valueOf(System.currentTimeMillis());
        }
    }
    
    interface Auditable {
        default void audit(String action) {
            System.out.println("[AUDIT] " + action + " by " + getUser());
        }
        
        String getUser();
    }
    
    static class Service implements Loggable, Auditable {
        private String user;
        
        Service(String user) {
            this.user = user;
        }
        
        @Override
        public String getUser() {
            return user;
        }
        
        void performAction() {
            log("Starting action");
            audit("Action performed");
        }
    }
    
    public static void combiningBehaviors() {
        System.out.println("\n=== Combining Behaviors ===");
        
        Service service = new Service("Alice");
        service.performAction();
        
        /**
         * BEHAVIOR COMPOSITION:
         * 
         * Mix multiple capabilities:
         * - Loggable: logging behavior
         * - Auditable: audit behavior
         * - Serializable: serialization
         * - Comparable: comparison
         * 
         * Advantages:
         * - Single Responsibility: each interface = one concern
         * - Flexible combination
         * - Easy to add/remove capabilities
         */
    }
    
    /**
     * Interface inheritance chain
     */
    interface Animal {
        default void breathe() {
            System.out.println("Breathing...");
        }
    }
    
    interface Mammal extends Animal {
        default void feedYoung() {
            System.out.println("Feeding young with milk...");
        }
    }
    
    interface Aquatic extends Animal {
        default void swim() {
            System.out.println("Swimming...");
        }
    }
    
    static class Whale implements Mammal, Aquatic {
        // Inherits: breathe(), feedYoung(), swim()
    }
    
    public static void interfaceInheritanceChain() {
        System.out.println("\n=== Interface Inheritance Chain ===");
        
        Whale whale = new Whale();
        whale.breathe();    // From Animal
        whale.feedYoung();  // From Mammal
        whale.swim();       // From Aquatic
        
        /**
         * INTERFACE INHERITANCE:
         * 
         * Interfaces can extend other interfaces
         * Child inherits parent's defaults
         * Can override parent's defaults
         * 
         * Example:
         * interface Child extends Parent {
         *   @Override
         *   default void method() {
         *     // override parent's default
         *   }
         * }
         * 
         * Implementing class gets:
         * - All parent defaults
         * - All child defaults (may override parents)
         */
    }
    
    /**
     * Mix-in pattern
     */
    interface Comparable<T> {
        int compareTo(T other);
        
        default boolean isGreaterThan(T other) {
            return compareTo(other) > 0;
        }
        
        default boolean isLessThan(T other) {
            return compareTo(other) < 0;
        }
        
        default boolean isEqualTo(T other) {
            return compareTo(other) == 0;
        }
    }
    
    static class Score implements Comparable<Score> {
        private int value;
        
        Score(int value) {
            this.value = value;
        }
        
        @Override
        public int compareTo(Score other) {
            return Integer.compare(this.value, other.value);
        }
        
        // Gets isGreaterThan, isLessThan, isEqualTo for free!
    }
    
    public static void mixInPattern() {
        System.out.println("\n=== Mix-in Pattern ===");
        
        Score score1 = new Score(85);
        Score score2 = new Score(90);
        
        System.out.println("85 < 90: " + score1.isLessThan(score2));
        System.out.println("85 > 90: " + score1.isGreaterThan(score2));
        
        /**
         * MIX-IN PATTERN:
         * 
         * Interface provides:
         * - Abstract method (must implement)
         * - Default methods (automatic behavior)
         * 
         * Implementing class:
         * - Implements abstract method
         * - Gets default methods for free
         * 
         * Benefits:
         * - Reduce boilerplate
         * - Consistent behavior
         * - Easy to extend
         */
    }
    
    public static void main(String[] args) {
        basicMultipleInheritance();
        combiningBehaviors();
        interfaceInheritanceChain();
        mixInPattern();
    }
}
```

## 3. Diamond Problem Resolution

### Conflict Resolution Rules

```java
/**
 * DIAMOND PROBLEM RESOLUTION
 * 
 * Rules for resolving conflicts when multiple interfaces
 * have the same default method
 */

import java.util.*;

public class DiamondProblemResolution {
    
    /**
     * Basic diamond problem
     */
    interface A {
        default void method() {
            System.out.println("A's method");
        }
    }
    
    interface B {
        default void method() {
            System.out.println("B's method");
        }
    }
    
    // Won't compile without resolving conflict!
    static class C implements A, B {
        @Override
        public void method() {
            // Must resolve conflict explicitly
            A.super.method();  // Or B.super.method()
        }
    }
    
    public static void basicDiamondProblem() {
        System.out.println("=== Basic Diamond Problem ===");
        
        C c = new C();
        c.method();  // Uses resolved version
        
        /**
         * DIAMOND PROBLEM:
         * 
         * Occurs when:
         * - Two interfaces have same default method
         * - Class implements both interfaces
         * 
         * Won't compile unless:
         * - Class overrides the method
         * - Explicitly chooses one implementation
         * 
         * Resolution:
         * InterfaceA.super.method()  // Call A's version
         * InterfaceB.super.method()  // Call B's version
         * Or provide new implementation
         */
    }
    
    /**
     * Resolution rules
     */
    interface Animal {
        default void move() {
            System.out.println("Animal moving");
        }
    }
    
    interface Mammal extends Animal {
        @Override
        default void move() {
            System.out.println("Mammal walking");
        }
    }
    
    interface Fish extends Animal {
        @Override
        default void move() {
            System.out.println("Fish swimming");
        }
    }
    
    // Conflict: Mammal.move() vs Fish.move()
    static class Platypus implements Mammal, Fish {
        @Override
        public void move() {
            System.out.println("Platypus: can walk or swim");
            // Could call Mammal.super.move() or Fish.super.move()
        }
    }
    
    public static void resolutionRules() {
        System.out.println("\n=== Resolution Rules ===");
        
        Platypus platypus = new Platypus();
        platypus.move();
        
        /**
         * CONFLICT RESOLUTION RULES:
         * 
         * Rule 1: Classes win over interfaces
         * If a class has a method, it wins over default
         * 
         * Rule 2: More specific interface wins
         * If B extends A, B's default wins
         * 
         * Rule 3: Explicit choice required
         * If no rule applies, must override
         * 
         * Example:
         * 
         * interface A { default void m() {} }
         * interface B extends A { @Override default void m() {} }
         * class C implements A, B {}
         * 
         * C uses B.m() (Rule 2: B is more specific)
         */
    }
    
    /**
     * Rule 1: Classes win
     */
    interface Printable {
        default String toString() {
            return "Printable";
        }
    }
    
    static class Document implements Printable {
        @Override
        public String toString() {
            return "Document";  // Class method wins
        }
    }
    
    public static void classesWin() {
        System.out.println("\n=== Rule 1: Classes Win ===");
        
        Document doc = new Document();
        System.out.println(doc.toString());  // Uses class method
        
        /**
         * RULE 1: CLASSES WIN OVER INTERFACES
         * 
         * Class implementation always wins:
         * - Superclass method > default method
         * - Direct class method > default method
         * 
         * Cannot override Object methods with default:
         * - toString(), equals(), hashCode()
         * - These must be in classes
         * 
         * Why: Preserves class hierarchy semantics
         */
    }
    
    /**
     * Rule 2: More specific wins
     */
    interface Vehicle {
        default void start() {
            System.out.println("Vehicle starting");
        }
    }
    
    interface Car extends Vehicle {
        @Override
        default void start() {
            System.out.println("Car starting");
        }
    }
    
    static class Sedan implements Vehicle, Car {
        // No conflict: Car is more specific than Vehicle
        // Uses Car.start()
    }
    
    public static void moreSpecificWins() {
        System.out.println("\n=== Rule 2: More Specific Wins ===");
        
        Sedan sedan = new Sedan();
        sedan.start();  // Uses Car.start()
        
        /**
         * RULE 2: MORE SPECIFIC WINS
         * 
         * If B extends A:
         * - B is more specific
         * - B's default wins
         * 
         * Example:
         * interface A { default void m() {} }
         * interface B extends A { @Override default void m() {} }
         * class C implements A, B {}
         * 
         * C.m() uses B's implementation
         * 
         * No explicit override needed
         */
    }
    
    /**
     * Rule 3: Explicit choice
     */
    interface Logger {
        default void log(String msg) {
            System.out.println("[LOG] " + msg);
        }
    }
    
    interface Auditor {
        default void log(String msg) {
            System.out.println("[AUDIT] " + msg);
        }
    }
    
    static class Service implements Logger, Auditor {
        @Override
        public void log(String msg) {
            // Must choose explicitly
            Logger.super.log(msg);    // Option 1: Use Logger
            // Auditor.super.log(msg); // Option 2: Use Auditor
            // Or provide own implementation
        }
    }
    
    static class AuditedService implements Logger, Auditor {
        @Override
        public void log(String msg) {
            // Can call both!
            Logger.super.log(msg);
            Auditor.super.log(msg);
        }
    }
    
    public static void explicitChoice() {
        System.out.println("\n=== Rule 3: Explicit Choice ===");
        
        Service service = new Service();
        service.log("Event occurred");
        
        System.out.println();
        AuditedService auditedService = new AuditedService();
        auditedService.log("Important event");
        
        /**
         * RULE 3: EXPLICIT CHOICE REQUIRED
         * 
         * When no rule applies:
         * - Must override method
         * - Choose one with InterfaceA.super.method()
         * - Or provide new implementation
         * 
         * Can combine:
         * @Override
         * public void method() {
         *   InterfaceA.super.method();
         *   InterfaceB.super.method();
         *   // Additional logic
         * }
         */
    }
    
    /**
     * Complex diamond
     */
    interface Top {
        default void method() {
            System.out.println("Top");
        }
    }
    
    interface Left extends Top {
        // Inherits method()
    }
    
    interface Right extends Top {
        @Override
        default void method() {
            System.out.println("Right");
        }
    }
    
    static class Bottom implements Left, Right {
        // Uses Right.method() (more specific via override)
    }
    
    public static void complexDiamond() {
        System.out.println("\n=== Complex Diamond ===");
        
        Bottom bottom = new Bottom();
        bottom.method();  // Uses Right.method()
        
        /**
         * COMPLEX DIAMOND:
         * 
         *       Top
         *      /   \
         *   Left   Right
         *      \   /
         *     Bottom
         * 
         * Resolution:
         * - Left inherits Top.method()
         * - Right overrides Top.method()
         * - Bottom implements both
         * - Right is more specific (Rule 2)
         * - Bottom uses Right.method()
         */
    }
    
    public static void main(String[] args) {
        basicDiamondProblem();
        resolutionRules();
        classesWin();
        moreSpecificWins();
        explicitChoice();
        complexDiamond();
    }
}
```

## 4. Static Methods in Interfaces

### Utility Methods in Interfaces

```java
/**
 * STATIC METHODS IN INTERFACES
 * 
 * Interfaces can have static methods with implementations
 */

import java.util.*;
import java.util.function.*;

public class InterfaceStaticMethods {
    
    /**
     * Basic static methods
     */
    interface MathUtils {
        static int add(int a, int b) {
            return a + b;
        }
        
        static int multiply(int a, int b) {
            return a * b;
        }
        
        static double average(int... numbers) {
            return Arrays.stream(numbers).average().orElse(0);
        }
    }
    
    public static void basicStaticMethods() {
        System.out.println("=== Basic Static Methods ===");
        
        int sum = MathUtils.add(5, 3);
        int product = MathUtils.multiply(5, 3);
        double avg = MathUtils.average(1, 2, 3, 4, 5);
        
        System.out.println("Sum: " + sum);
        System.out.println("Product: " + product);
        System.out.println("Average: " + avg);
        
        /**
         * STATIC METHODS IN INTERFACES:
         * 
         * Syntax:
         * static ReturnType methodName(params) {
         *   // implementation
         * }
         * 
         * Characteristics:
         * - Must have implementation
         * - Called via interface name
         * - Not inherited by implementing classes
         * - Cannot be overridden
         * - Can access other static methods
         * - Cannot access instance methods/defaults
         * 
         * Use for:
         * - Utility methods
         * - Factory methods
         * - Helper methods
         */
    }
    
    /**
     * Factory methods
     */
    interface Comparator<T> {
        int compare(T o1, T o2);
        
        // Static factory methods
        static <T extends Comparable<T>> Comparator<T> naturalOrder() {
            return (o1, o2) -> o1.compareTo(o2);
        }
        
        static <T extends Comparable<T>> Comparator<T> reverseOrder() {
            return (o1, o2) -> o2.compareTo(o1);
        }
        
        static <T> Comparator<T> comparing(
            Function<T, ? extends Comparable> keyExtractor
        ) {
            return (o1, o2) -> {
                Comparable k1 = keyExtractor.apply(o1);
                Comparable k2 = keyExtractor.apply(o2);
                return k1.compareTo(k2);
            };
        }
    }
    
    public static void factoryMethods() {
        System.out.println("\n=== Factory Methods ===");
        
        List<String> words = Arrays.asList("banana", "apple", "cherry");
        
        // Using static factory
        words.sort(Comparator.naturalOrder());
        System.out.println("Natural order: " + words);
        
        words.sort(Comparator.reverseOrder());
        System.out.println("Reverse order: " + words);
        
        List<String> byLength = new ArrayList<>(words);
        byLength.sort(Comparator.comparing(String::length));
        System.out.println("By length: " + byLength);
        
        /**
         * FACTORY METHOD PATTERN:
         * 
         * Static methods in interfaces perfect for:
         * - Creating instances
         * - Common configurations
         * - Convenience constructors
         * 
         * Examples in Java API:
         * - Comparator.naturalOrder()
         * - Comparator.comparing()
         * - Stream.of()
         * - Collections.emptyList()
         * 
         * Benefits:
         * - Self-documenting
         * - Type-safe
         * - Fluent API
         */
    }
    
    /**
     * Helper methods for default methods
     */
    interface Calculator {
        static int validateNonNegative(int value) {
            if (value < 0) {
                throw new IllegalArgumentException("Must be non-negative");
            }
            return value;
        }
        
        default int addPositive(int a, int b) {
            return validateNonNegative(a) + validateNonNegative(b);
        }
    }
    
    static class SimpleCalculator implements Calculator {
        // Can use addPositive() which uses validateNonNegative()
    }
    
    public static void helperMethods() {
        System.out.println("\n=== Helper Methods ===");
        
        SimpleCalculator calc = new SimpleCalculator();
        System.out.println("5 + 3 = " + calc.addPositive(5, 3));
        
        try {
            calc.addPositive(-5, 3);
        } catch (IllegalArgumentException e) {
            System.out.println("Error: " + e.getMessage());
        }
        
        /**
         * STATIC HELPERS FOR DEFAULTS:
         * 
         * Pattern:
         * - Private logic in static methods
         * - Default methods use static methods
         * - Implementing classes inherit defaults
         * 
         * Benefits:
         * - Code reuse
         * - Encapsulation
         * - Testing (can test static separately)
         * 
         * Before Java 9:
         * Must use static (no private methods)
         * 
         * Java 9+:
         * Can use private methods instead
         */
    }
    
    /**
     * Static methods not inherited
     */
    interface Parent {
        static void staticMethod() {
            System.out.println("Parent static");
        }
    }
    
    interface Child extends Parent {
        // Does NOT inherit staticMethod()
        // Can define own static with same name (not override)
        static void staticMethod() {
            System.out.println("Child static");
        }
    }
    
    static class Implementation implements Child {
        // Cannot call staticMethod() on instance
    }
    
    public static void staticNotInherited() {
        System.out.println("\n=== Static Not Inherited ===");
        
        Parent.staticMethod();
        Child.staticMethod();
        
        // Implementation.staticMethod();  // Compile error!
        // Must use interface name
        
        /**
         * STATIC METHODS NOT INHERITED:
         * 
         * Static methods:
         * - Not inherited by subinterfaces
         * - Not inherited by implementing classes
         * - Must be called via interface name
         * 
         * vs default methods:
         * - Default methods ARE inherited
         * - Can be called on instances
         * - Can be overridden
         * 
         * Reason:
         * Static = belongs to type
         * Default = belongs to instance
         */
    }
    
    public static void main(String[] args) {
        basicStaticMethods();
        factoryMethods();
        helperMethods();
        staticNotInherited();
    }
}
```

## 5. Interface Evolution

### Evolving Interfaces Without Breaking Code

```java
/**
 * INTERFACE EVOLUTION
 * 
 * How default methods enable backward-compatible evolution
 */

import java.util.*;

public class InterfaceEvolution {
    
    /**
     * Problem before Java 8
     */
    public static void problemBeforeJava8() {
        /**
         * BEFORE JAVA 8:
         * 
         * Cannot add methods to interface without breaking ALL implementations
         * 
         * Example:
         * 
         * // Original interface
         * interface List<E> {
         *   int size();
         *   void add(E element);
         * }
         * 
         * // Many implementations exist
         * class ArrayList<E> implements List<E> { ... }
         * class LinkedList<E> implements List<E> { ... }
         * class CustomList<E> implements List<E> { ... }
         * 
         * // Want to add new method
         * interface List<E> {
         *   int size();
         *   void add(E element);
         *   void sort(Comparator<E> c);  // NEW
         * }
         * 
         * // BREAKS ALL IMPLEMENTATIONS!
         * // ArrayList, LinkedList, CustomList all fail to compile
         * 
         * Solution before Java 8:
         * - Create new interface (ListSortable)
         * - Or add to abstract class
         * - Or don't add the method
         */
    }
    
    /**
     * Solution with default methods
     */
    interface Collection<E> {
        // Original methods
        int size();
        void add(E element);
        
        // New method added in Java 8 - with default!
        default void forEach(java.util.function.Consumer<? super E> action) {
            for (E element : this) {
                action.accept(element);
            }
        }
        
        // Fake iterator for example
        Iterator<E> iterator();
        default Iterable<E> asIterable() {
            return this::iterator;
        }
    }
    
    public static void solutionWithDefaults() {
        System.out.println("\n=== Solution with Default Methods ===");
        
        /**
         * WITH DEFAULT METHODS:
         * 
         * Can add methods without breaking implementations:
         * 
         * interface Collection<E> {
         *   int size();
         *   void add(E element);
         *   
         *   // New method - existing implementations still work!
         *   default void forEach(Consumer<E> action) {
         *     for (E e : this) {
         *       action.accept(e);
         *     }
         *   }
         * }
         * 
         * Existing implementations:
         * - Continue to work (binary compatible)
         * - Get new method automatically
         * - Can override if needed
         * 
         * This is how Java 8 added:
         * - Collection.forEach()
         * - Collection.stream()
         * - List.sort()
         * - Map.computeIfAbsent()
         * - Many others
         * 
         * Without breaking millions of existing classes!
         */
    }
    
    /**
     * Real Java 8 evolution example
     */
    public static void realJava8Evolution() {
        System.out.println("\n=== Real Java 8 Evolution ===");
        
        /**
         * JAVA 8 INTERFACE EVOLUTION:
         * 
         * Collection interface added:
         * 
         * default Stream<E> stream() {
         *   return StreamSupport.stream(spliterator(), false);
         * }
         * 
         * default void forEach(Consumer<? super E> action) {
         *   Objects.requireNonNull(action);
         *   for (T t : this) {
         *     action.accept(t);
         *   }
         * }
         * 
         * List interface added:
         * 
         * default void sort(Comparator<? super E> c) {
         *   Object[] a = this.toArray();
         *   Arrays.sort(a, (Comparator) c);
         *   ListIterator<E> i = this.listIterator();
         *   for (Object e : a) {
         *     i.next();
         *     i.set((E) e);
         *   }
         * }
         * 
         * Map interface added:
         * 
         * default V getOrDefault(Object key, V defaultValue) {
         *   V v = get(key);
         *   return (v != null) ? v : defaultValue;
         * }
         * 
         * default V computeIfAbsent(K key,
         *   Function<? super K, ? extends V> mappingFunction) {
         *   // implementation
         * }
         * 
         * Result:
         * - All existing ArrayList, HashMap, etc. got new methods
         * - No code broke
         * - Backward compatible
         * - Forward compatible
         */
    }
    
    /**
     * Evolution strategy
     */
    interface Version1 {
        void doSomething();
    }
    
    // Later: want to add new capability
    interface Version2 extends Version1 {
        // Don't add abstract method - breaks implementations!
        // void doSomethingElse();

        default void doSomethingElse() {

    static class NewUsage {

        System.out.println("\n=== Evolution Strategy ===");

         *    - Continue to work

         *    - Can override if needed
         *    - Or use default

    }

        /**
```

## 6. Defender Methods Pattern

### Using Default Methods for Backward Compatibility

```java
/**
 * DEFENDER METHODS PATTERN
 * 
 * Pattern for adding functionality while maintaining compatibility
 */

import java.util.*;
import java.util.function.*;

public class DefenderMethodsPattern {
    
    /**
     * Basic defender pattern
     */
    interface Repository<T> {
        // Original contract
        T findById(String id);
        void save(T entity);
        
        // New methods added later as defenders
        default List<T> findAll() {
            throw new UnsupportedOperationException(
                "findAll() not implemented"
            );
        }
        
        default void delete(String id) {
            throw new UnsupportedOperationException(
                "delete() not implemented"
            );
        }
    }
    
    // Old implementation - still works
    static class OldRepository implements Repository<String> {
        @Override
        public String findById(String id) {
            return "Found: " + id;
        }
        
        @Override
        public void save(String entity) {
            System.out.println("Saved: " + entity);
        }
        
        // Doesn't implement findAll() or delete()
        // Uses default (throws exception)
    }
    
    // New implementation - implements new methods
    static class NewRepository implements Repository<String> {
        private Map<String, String> storage = new HashMap<>();
        
        @Override
        public String findById(String id) {
            return storage.get(id);
        }
        
        @Override
        public void save(String entity) {
            storage.put(entity, entity);
        }
        
        @Override
        public List<String> findAll() {
            return new ArrayList<>(storage.values());
        }
        
        @Override
        public void delete(String id) {
            storage.remove(id);
        }
    }
    
    public static void basicDefenderPattern() {
        System.out.println("=== Basic Defender Pattern ===");
        
        // Old implementation works
        OldRepository oldRepo = new OldRepository();
        System.out.println(oldRepo.findById("123"));
        oldRepo.save("data");
        
        // New implementation has full functionality
        NewRepository newRepo = new NewRepository();
        newRepo.save("item1");
        newRepo.save("item2");
        System.out.println("All items: " + newRepo.findAll());
        
        /**
         * DEFENDER METHODS PATTERN:
         * 
         * Purpose:
         * - Add methods to interface
         * - Don't break existing implementations
         * - Encourage gradual migration
         * 
         * Pattern:
         * 1. Add method with default implementation
         * 2. Default throws UnsupportedOperationException
         * 3. Or provides minimal functionality
         * 4. Document that implementations should override
         * 
         * Name:
         * "Defender" because default "defends" against breakage
         * Also called "virtual extension methods"
         */
    }
    
    /**
     * Defender with fallback implementation
     */
    interface Cache<K, V> {
        V get(K key);
        void put(K key, V value);
        
        // Defender with fallback to get/put
        default V computeIfAbsent(K key, Function<K, V> mappingFunction) {
            V value = get(key);
            if (value == null) {
                value = mappingFunction.apply(key);
                if (value != null) {
                    put(key, value);
                }
            }
            return value;
        }
        
        // Defender with iteration fallback
        default void clear() {
            // Basic implementation - implementations can optimize
            throw new UnsupportedOperationException(
                "clear() not supported - override for bulk deletion"
            );
        }
    }
    
    static class SimpleCache<K, V> implements Cache<K, V> {
        private Map<K, V> map = new HashMap<>();
        
        @Override
        public V get(K key) {
            return map.get(key);
        }
        
        @Override
        public void put(K key, V value) {
            map.put(key, value);
        }
        
        // Gets computeIfAbsent() for free!
        // Can override clear() if needed
    }
    
    public static void defenderWithFallback() {
        System.out.println("\n=== Defender with Fallback ===");
        
        SimpleCache<String, String> cache = new SimpleCache<>();
        
        // Uses default computeIfAbsent implementation
        String value = cache.computeIfAbsent("key1", k -> {
            System.out.println("Computing value for " + k);
            return "computed value";
        });
        System.out.println("Value: " + value);
        
        // Second call doesn't compute (cached)
        String value2 = cache.computeIfAbsent("key1", k -> "new value");
        System.out.println("Cached: " + value2);
        
        /**
         * DEFENDER WITH FALLBACK:
         * 
         * Provide actual implementation when possible:
         * 
         * Good:
         * default V computeIfAbsent(...) {
         *   // Working implementation using existing methods
         * }
         * 
         * Acceptable:
         * default void clear() {
         *   throw new UnsupportedOperationException();
         * }
         * 
         * Best:
         * Implementations can optimize by overriding
         */
    }
    
    /**
     * Staged evolution pattern
     */
    interface DataSource {
        // Phase 1: Original
        String getData();
        
        // Phase 2: Added with defender
        default List<String> getBatchData() {
            return Arrays.asList(getData());
        }
        
        // Phase 3: Added with defender
        default Stream<String> streamData() {
            return getBatchData().stream();
        }
    }
    
    // Phase 1 implementation
    static class Phase1DataSource implements DataSource {
        @Override
        public String getData() {
            return "data";
        }
        // Gets getBatchData() and streamData() automatically
    }
    
    // Phase 2 implementation
    static class Phase2DataSource implements DataSource {
        @Override
        public String getData() {
            return "data";
        }
        
        @Override
        public List<String> getBatchData() {
            return Arrays.asList("data1", "data2", "data3");
        }
        // Gets streamData() automatically
    }
    
    // Phase 3 implementation  
    static class Phase3DataSource implements DataSource {
        @Override
        public String getData() {
            return "data";
        }
        
        @Override
        public List<String> getBatchData() {
            return Arrays.asList("data1", "data2", "data3");
        }
        
        @Override
        public Stream<String> streamData() {
            // Optimized implementation
            return Stream.of("data1", "data2", "data3");
        }
    }
    
    public static void stagedEvolution() {
        System.out.println("\n=== Staged Evolution ===");
        
        Phase1DataSource phase1 = new Phase1DataSource();
        System.out.println("Phase 1: " + phase1.streamData().count());
        
        Phase2DataSource phase2 = new Phase2DataSource();
        System.out.println("Phase 2: " + phase2.streamData().count());
        
        Phase3DataSource phase3 = new Phase3DataSource();
        System.out.println("Phase 3: " + phase3.streamData().count());
        
        /**
         * STAGED EVOLUTION:
         * 
         * Interface evolves over time:
         * Phase 1: Original methods
         * Phase 2: Add methods with defenders
         * Phase 3: Add more methods with defenders
         * 
         * Implementations can:
         * - Stay at Phase 1 (use all defaults)
         * - Move to Phase 2 (override some)
         * - Move to Phase 3 (override all)
         * 
         * Benefits:
         * - Gradual migration
         * - No big-bang updates
         * - Implementations evolve at their own pace
         */
    }
    
    /**
     * Defender best practices
     */
    public static void defenderBestPractices() {
        /**
         * DEFENDER METHODS BEST PRACTICES:
         * 
         * 1. PROVIDE WORKING IMPLEMENTATION WHEN POSSIBLE:
         *    
         *    Good:
         *    default List<T> findAll() {
         *      List<T> result = new ArrayList<>();
         *      // Use existing methods to implement
         *      return result;
         *    }
         *    
         *    Acceptable:
         *    default List<T> findAll() {
         *      throw new UnsupportedOperationException(
         *        "Implement findAll() for full functionality"
         *      );
         *    }
         * 
         * 2. DOCUMENT EXPECTATIONS:
         *    
         *    @implSpec
         *    This implementation returns empty list.
         *    Override for actual functionality.
         *    
         *    @throws UnsupportedOperationException
         *    if not overridden by implementation
         * 
         * 3. CONSIDER PERFORMANCE:
         *    
         *    Default should be correct but may be slow
         *    Document that implementations should optimize
         *    
         *    Example:
         *    // Default O(n), override for O(1)
         *    default int size() {
         *      int count = 0;
         *      for (E e : this) count++;
         *      return count;
         *    }
         * 
         * 4. BUILD ON EXISTING METHODS:
         *    
         *    Default methods should use abstract methods
         *    Creates consistent behavior
         *    
         *    default void forEach(Consumer<T> action) {
         *      for (T t : this) {  // Uses iterator()
         *        action.accept(t);
         *      }
         *    }
         * 
         * 5. THROW MEANINGFUL EXCEPTIONS:
         *    
         *    Bad:
         *    throw new UnsupportedOperationException();
         *    
         *    Good:
         *    throw new UnsupportedOperationException(
         *      "delete() not supported by " + 
         *      getClass().getName() +
         *      ". Override to enable deletion."
         *    );
         * 
         * 6. VERSIONING STRATEGY:
         *    
         *    Don't create MyInterface_v2
         *    Evolve MyInterface with defenders
         *    
         *    Deprecated old patterns:
         *    @deprecated Use newMethod() instead
         *    default void oldMethod() {
         *      newMethod();
         *    }
         */
    }
    
    /**
     * Anti-patterns
     */
    public static void antiPatterns() {
        /**
         * DEFENDER ANTI-PATTERNS:
         * 
         * ❌ DON'T: Add abstract method
         *    interface MyInterface {
         *      void newMethod();  // BREAKS ALL IMPLEMENTATIONS!
         *    }
         * 
         * ✅ DO: Add default method
         *    interface MyInterface {
         *      default void newMethod() {
         *        // implementation or throw
         *      }
         *    }
         * 
         * ❌ DON'T: Silent failure
         *    default void delete(T item) {
         *      // Do nothing - silently ignores
         *    }
         * 
         * ✅ DO: Explicit failure or working implementation
         *    default void delete(T item) {
         *      throw new UnsupportedOperationException();
         *    }
         * 
         * ❌ DON'T: Duplicate logic
         *    interface A {
         *      default void method() { /* logic */ }
         *    }
         *    interface B {
         *      default void method() { /* same logic */ }
         *    }
         * 
         * ✅ DO: Extract to common interface
         *    interface Common {
         *      default void method() { /* logic */ }
         *    }
         *    interface A extends Common {}
         *    interface B extends Common {}
         * 
         * ❌ DON'T: Override Object methods
         *    interface MyInterface {
         *      default String toString() {  // ILLEGAL!
         *        return "MyInterface";
         *      }
         *    }
         * 
         * ✅ DO: Use abstract class if needed
         *    abstract class MyAbstract {
         *      @Override
         *      public String toString() {
         *        return "MyAbstract";
         *      }
         *    }
         */
    }
    
    public static void main(String[] args) {
        basicDefenderPattern();
        defenderWithFallback();
        stagedEvolution();
    }
}
```

### Best Practices

**✅ DO:**

- Use default methods to evolve interfaces
- Provide working implementations when possible
- Document default behavior clearly
- Use static methods for utilities/factories
- Call Interface.super.method() to resolve conflicts
- Build defaults on existing abstract methods
- Throw UnsupportedOperationException if no good default

**❌ DON'T:**

- Override Object methods (toString, equals, hashCode)
- Add state to interfaces (still no fields!)
- Use defaults to avoid proper design
- Silent failure in defaults
- Create complex default method hierarchies
- Use when abstract class is more appropriate

### Default vs Abstract Class

```
Use Default Methods when:
✓ No state needed
✓ Multiple inheritance desired
✓ Evolving existing interface
✓ Mix-in capabilities

Use Abstract Class when:
✓ Need state (fields)
✓ Need constructors
✓ Override Object methods
✓ Single inheritance sufficient
```

### Conflict Resolution Quick Guide

```java
// Scenario 1: Class vs Interface
class A { void m() {} }
interface B { default void m() {} }
class C extends A implements B {}
→ A.m() wins (classes win)

// Scenario 2: Parent vs Child Interface
interface A { default void m() {} }
interface B extends A { @Override default void m() {} }
class C implements A, B {}
→ B.m() wins (more specific)

// Scenario 3: Sibling Interfaces
interface A { default void m() {} }
interface B { default void m() {} }
class C implements A, B {}
→ Must override and choose:
  A.super.m() or B.super.m()
```

**Key insights:**

- **Default methods** enabled Java 8 to add methods to Collection/List/Map without breaking millions of classes
- **Three resolution rules**: (1) Classes win (2) More specific wins (3) Explicit choice required
- **Static methods** perfect for factory methods and utilities
- **Defender pattern** allows gradual API evolution
- Can't override Object methods (toString, equals, hashCode)

**The revolution:**

```
Before Java 8:
Adding method to interface = breaking change
→ Cannot evolve interfaces!

After Java 8:
Default methods = backward compatible evolution
→ Java 8 added 100+ methods to core interfaces!
```

**Conflict resolution mastered:**

```
Rule 1: class A wins over interface B
Rule 2: child interface B wins over parent A  
Rule 3: must choose A.super or B.super
```
