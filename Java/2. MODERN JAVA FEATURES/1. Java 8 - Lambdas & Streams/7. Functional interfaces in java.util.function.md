# Functional Interfaces in java.util.function

## Overview

The `java.util.function` package provides a rich set of functional interfaces introduced in Java 8, serving as the foundation for lambda expressions and method references. This guide covers the four core interfaces (Predicate, Function, Consumer, Supplier), their binary variants (BiFunction, BiConsumer, BiPredicate), operator specializations (UnaryOperator, BinaryOperator), primitive specializations for avoiding boxing overhead (IntPredicate, LongFunction, etc.), powerful composition methods (andThen, compose, and, or), and guidelines for creating custom functional interfaces.


---

## 1. Predicate, Function, Consumer, Supplier

### The Core Four Functional Interfaces

```java
/**
 * CORE FUNCTIONAL INTERFACES
 * 
 * Predicate: T -> boolean (test)
 * Function:  T -> R (transform)
 * Consumer:  T -> void (accept)
 * Supplier:  () -> T (provide)
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class CoreFunctionalInterfaces {
    
    /**
     * Predicate<T> - Boolean test
     */
    public static void predicateInterface() {
        System.out.println("=== Predicate<T> ===");
        
        // Basic predicate
        Predicate<String> isEmpty = String::isEmpty;
        System.out.println("'' is empty: " + isEmpty.test(""));
        System.out.println("'hello' is empty: " + isEmpty.test("hello"));
        
        // Lambda predicate
        Predicate<Integer> isPositive = n -> n > 0;
        System.out.println("5 is positive: " + isPositive.test(5));
        System.out.println("-1 is positive: " + isPositive.test(-1));
        
        // With streams
        List<String> words = Arrays.asList("apple", "banana", "", "cherry", "");
        long emptyCount = words.stream()
            .filter(String::isEmpty)  // Predicate
            .count();
        System.out.println("Empty strings: " + emptyCount);
        
        // Complex predicate
        Predicate<String> isLongWord = s -> s.length() > 5;
        List<String> longWords = words.stream()
            .filter(isLongWord)
            .collect(Collectors.toList());
        System.out.println("Long words: " + longWords);
        
        /**
         * PREDICATE<T>:
         * 
         * @FunctionalInterface
         * interface Predicate<T> {
         *   boolean test(T t);
         * }
         * 
         * Purpose: Boolean-valued function
         * 
         * Common uses:
         * - Filtering: stream.filter(predicate)
         * - Validation: if (predicate.test(value))
         * - Searching: list.removeIf(predicate)
         * 
         * Key method:
         * - test(T t) -> boolean
         * 
         * Default methods:
         * - and(Predicate other)
         * - or(Predicate other)
         * - negate()
         */
    }
    
    /**
     * Function<T, R> - Transformation
     */
    public static void functionInterface() {
        System.out.println("\n=== Function<T, R> ===");
        
        // Basic function
        Function<String, Integer> length = String::length;
        System.out.println("Length of 'hello': " + length.apply("hello"));
        
        // Lambda function
        Function<Integer, String> intToStr = n -> "Number: " + n;
        System.out.println(intToStr.apply(42));
        
        // With streams
        List<String> words = Arrays.asList("apple", "banana", "cherry");
        List<Integer> lengths = words.stream()
            .map(String::length)  // Function
            .collect(Collectors.toList());
        System.out.println("Lengths: " + lengths);
        
        // Chaining functions
        Function<String, String> trim = String::trim;
        Function<String, String> upper = String::toUpperCase;
        Function<String, Integer> trimAndLength = trim.andThen(String::length);
        System.out.println("Trim and length: " + trimAndLength.apply("  hello  "));
        
        /**
         * FUNCTION<T, R>:
         * 
         * @FunctionalInterface
         * interface Function<T, R> {
         *   R apply(T t);
         * }
         * 
         * Purpose: Transform T to R
         * 
         * Common uses:
         * - Mapping: stream.map(function)
         * - Extraction: map.computeIfAbsent(key, function)
         * - Conversion: convert(value, function)
         * 
         * Key method:
         * - apply(T t) -> R
         * 
         * Default methods:
         * - andThen(Function after)
         * - compose(Function before)
         * 
         * Static method:
         * - identity() -> t -> t
         */
    }
    
    /**
     * Consumer<T> - Side effect
     */
    public static void consumerInterface() {
        System.out.println("\n=== Consumer<T> ===");
        
        // Basic consumer
        Consumer<String> printer = System.out::println;
        printer.accept("Hello, World!");
        
        // Lambda consumer
        Consumer<String> logger = s -> System.out.println("[LOG] " + s);
        logger.accept("Application started");
        
        // With streams
        List<String> items = Arrays.asList("apple", "banana", "cherry");
        items.forEach(System.out::println);  // Consumer
        
        // Chaining consumers
        Consumer<String> printUpper = s -> System.out.println(s.toUpperCase());
        Consumer<String> printLower = s -> System.out.println(s.toLowerCase());
        Consumer<String> both = printUpper.andThen(printLower);
        both.accept("Hello");
        
        // Stateful consumer
        List<String> results = new ArrayList<>();
        Consumer<String> accumulator = results::add;
        items.forEach(accumulator);
        System.out.println("Accumulated: " + results);
        
        /**
         * CONSUMER<T>:
         * 
         * @FunctionalInterface
         * interface Consumer<T> {
         *   void accept(T t);
         * }
         * 
         * Purpose: Perform action on T
         * 
         * Common uses:
         * - Iteration: forEach(consumer)
         * - Side effects: peek(consumer)
         * - Callbacks: onComplete(consumer)
         * 
         * Key method:
         * - accept(T t) -> void
         * 
         * Default method:
         * - andThen(Consumer after)
         */
    }
    
    /**
     * Supplier<T> - Factory
     */
    public static void supplierInterface() {
        System.out.println("\n=== Supplier<T> ===");
        
        // Basic supplier
        Supplier<String> helloSupplier = () -> "Hello";
        System.out.println(helloSupplier.get());
        
        // Method reference supplier
        Supplier<Double> randomSupplier = Math::random;
        System.out.println("Random: " + randomSupplier.get());
        
        // Constructor reference
        Supplier<ArrayList<String>> listSupplier = ArrayList::new;
        List<String> newList = listSupplier.get();
        System.out.println("New list: " + newList);
        
        // Lazy evaluation
        Optional<String> optional = Optional.empty();
        String result = optional.orElseGet(() -> {
            System.out.println("Computing default...");
            return "Default value";
        });
        System.out.println("Result: " + result);
        
        // With streams
        Stream<String> generated = Stream.generate(() -> "constant")
            .limit(3);
        System.out.println("Generated: " + generated.collect(Collectors.toList()));
        
        /**
         * SUPPLIER<T>:
         * 
         * @FunctionalInterface
         * interface Supplier<T> {
         *   T get();
         * }
         * 
         * Purpose: Provide T (factory)
         * 
         * Common uses:
         * - Lazy evaluation: orElseGet(supplier)
         * - Factories: Supplier<T> factory = T::new
         * - Generators: Stream.generate(supplier)
         * - Initialization: computeIfAbsent(key, k -> supplier.get())
         * 
         * Key method:
         * - get() -> T
         * 
         * No default methods
         */
    }
    
    /**
     * Comparison table
     */
    public static void comparisonTable() {
        /**
         * CORE FOUR COMPARISON:
         * 
         * ┌───────────┬──────────────┬──────────┬────────────────┐
         * │ Interface │ Signature    │ Method   │ Use Case       │
         * ├───────────┼──────────────┼──────────┼────────────────┤
         * │ Predicate │ T → boolean  │ test()   │ Filter/test    │
         * │ Function  │ T → R        │ apply()  │ Transform      │
         * │ Consumer  │ T → void     │ accept() │ Side effect    │
         * │ Supplier  │ () → T       │ get()    │ Provide/create │
         * └───────────┴──────────────┴──────────┴────────────────┘
         * 
         * 
         * MEMORY AID:
         * 
         * Predicate: P for "Pass" (filter test)
         * Function:  F for "From-To" (transform)
         * Consumer:  C for "Consume" (use value)
         * Supplier:  S for "Supply" (provide value)
         * 
         * 
         * PARAMETERS:
         * 
         * 2 params: Predicate, Function, Consumer
         * 1 param:  Supplier
         * 0 params: (none in core four)
         */
    }
    
    public static void main(String[] args) {
        predicateInterface();
        functionInterface();
        consumerInterface();
        supplierInterface();
    }
}
```

## 2. BiFunction, BiConsumer, BiPredicate

### Binary Variants - Two Parameters

```java
/**
 * BINARY FUNCTIONAL INTERFACES
 * 
 * BiFunction:  (T, U) -> R
 * BiConsumer:  (T, U) -> void
 * BiPredicate: (T, U) -> boolean
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class BinaryFunctionalInterfaces {
    
    /**
     * BiFunction<T, U, R> - Two-parameter transformation
     */
    public static void biFunctionInterface() {
        System.out.println("=== BiFunction<T, U, R> ===");
        
        // Basic BiFunction
        BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;
        System.out.println("5 + 3 = " + add.apply(5, 3));
        
        // String operations
        BiFunction<String, String, String> concat = (a, b) -> a + b;
        System.out.println("Concat: " + concat.apply("Hello", "World"));
        
        // Different types
        BiFunction<String, Integer, String> repeat = (s, n) -> s.repeat(n);
        System.out.println("Repeat: " + repeat.apply("ab", 3));
        
        // With Map.merge()
        Map<String, Integer> inventory = new HashMap<>();
        inventory.put("apple", 5);
        inventory.merge("apple", 3, Integer::sum);  // BiFunction
        System.out.println("Inventory: " + inventory);
        
        // Chaining with andThen
        BiFunction<Integer, Integer, Integer> multiply = (a, b) -> a * b;
        BiFunction<Integer, Integer, String> multiplyAndFormat = 
            multiply.andThen(result -> "Result: " + result);
        System.out.println(multiplyAndFormat.apply(4, 5));
        
        /**
         * BIFUNCTION<T, U, R>:
         * 
         * @FunctionalInterface
         * interface BiFunction<T, U, R> {
         *   R apply(T t, U u);
         * }
         * 
         * Purpose: Transform two inputs to output
         * 
         * Common uses:
         * - Map operations: map.merge(key, value, biFunction)
         * - Reduction: reduce(identity, biFunction)
         * - Combining: combine(t, u, biFunction)
         * 
         * Key method:
         * - apply(T t, U u) -> R
         * 
         * Default method:
         * - andThen(Function after)
         * 
         * Note: No compose() (which param would it apply to?)
         */
    }
    
    /**
     * BiConsumer<T, U> - Two-parameter side effect
     */
    public static void biConsumerInterface() {
        System.out.println("\n=== BiConsumer<T, U> ===");
        
        // Basic BiConsumer
        BiConsumer<String, Integer> printPair = (name, age) -> 
            System.out.println(name + " is " + age + " years old");
        printPair.accept("Alice", 30);
        
        // Map forEach
        Map<String, Integer> scores = new HashMap<>();
        scores.put("Alice", 90);
        scores.put("Bob", 85);
        scores.forEach((name, score) -> 
            System.out.println(name + ": " + score)
        );
        
        // Accumulating into map
        Map<String, List<Integer>> groupedScores = new HashMap<>();
        BiConsumer<String, Integer> addScore = (name, score) -> 
            groupedScores.computeIfAbsent(name, k -> new ArrayList<>()).add(score);
        
        addScore.accept("Alice", 90);
        addScore.accept("Alice", 95);
        addScore.accept("Bob", 85);
        System.out.println("Grouped: " + groupedScores);
        
        // Chaining
        BiConsumer<String, Integer> print1 = (s, n) -> 
            System.out.println("First: " + s + ", " + n);
        BiConsumer<String, Integer> print2 = (s, n) -> 
            System.out.println("Second: " + s + ", " + n);
        BiConsumer<String, Integer> both = print1.andThen(print2);
        both.accept("Test", 42);
        
        /**
         * BICONSUMER<T, U>:
         * 
         * @FunctionalInterface
         * interface BiConsumer<T, U> {
         *   void accept(T t, U u);
         * }
         * 
         * Purpose: Action on two inputs
         * 
         * Common uses:
         * - Map iteration: map.forEach(biConsumer)
         * - Pair processing: pairs.forEach(biConsumer)
         * - Callbacks: onEvent(t, u, biConsumer)
         * 
         * Key method:
         * - accept(T t, U u) -> void
         * 
         * Default method:
         * - andThen(BiConsumer after)
         */
    }
    
    /**
     * BiPredicate<T, U> - Two-parameter test
     */
    public static void biPredicateInterface() {
        System.out.println("\n=== BiPredicate<T, U> ===");
        
        // Basic BiPredicate
        BiPredicate<String, Integer> isLengthGreater = (s, n) -> s.length() > n;
        System.out.println("'hello' > 3: " + isLengthGreater.test("hello", 3));
        System.out.println("'hi' > 3: " + isLengthGreater.test("hi", 3));
        
        // Comparison
        BiPredicate<Integer, Integer> isGreater = (a, b) -> a > b;
        System.out.println("5 > 3: " + isGreater.test(5, 3));
        
        // String operations
        BiPredicate<String, String> startsWith = String::startsWith;
        System.out.println("'hello' starts with 'hel': " + 
            startsWith.test("hello", "hel"));
        
        // Combining with and/or
        BiPredicate<Integer, Integer> isEqual = (a, b) -> a.equals(b);
        BiPredicate<Integer, Integer> greaterOrEqual = isGreater.or(isEqual);
        System.out.println("5 >= 5: " + greaterOrEqual.test(5, 5));
        
        // Custom validation
        BiPredicate<String, String> sameIgnoreCase = 
            (a, b) -> a.equalsIgnoreCase(b);
        System.out.println("'Hello' == 'hello' (ignore case): " + 
            sameIgnoreCase.test("Hello", "hello"));
        
        /**
         * BIPREDICATE<T, U>:
         * 
         * @FunctionalInterface
         * interface BiPredicate<T, U> {
         *   boolean test(T t, U u);
         * }
         * 
         * Purpose: Test two inputs
         * 
         * Common uses:
         * - Map filtering: filter by key and value
         * - Pair validation: isValid(t, u, biPredicate)
         * - Comparison: compare(a, b, biPredicate)
         * 
         * Key method:
         * - test(T t, U u) -> boolean
         * 
         * Default methods:
         * - and(BiPredicate other)
         * - or(BiPredicate other)
         * - negate()
         */
    }
    
    /**
     * No BiSupplier
     */
    public static void noBiSupplier() {
        /**
         * WHY NO BISUPPLIER?
         * 
         * Supplier has no input: () -> T
         * 
         * BiSupplier would be: () -> (T, U)
         * 
         * But Java doesn't have tuples!
         * Would need to return Pair<T, U>
         * 
         * Alternative:
         * - Supplier<Pair<T, U>>
         * - Custom functional interface
         * - Record (Java 14+)
         * 
         * Example:
         * record Pair<T, U>(T first, U second) {}
         * Supplier<Pair<String, Integer>> pairSupplier = 
         *   () -> new Pair<>("Alice", 30);
         */
    }
    
    /**
     * Binary vs Unary comparison
     */
    public static void binaryVsUnary() {
        /**
         * BINARY VS UNARY:
         * 
         * ┌────────────┬────────────────┬──────────────────┐
         * │ Purpose    │ One Param      │ Two Params       │
         * ├────────────┼────────────────┼──────────────────┤
         * │ Test       │ Predicate<T>   │ BiPredicate<T,U> │
         * │ Transform  │ Function<T,R>  │ BiFunction<T,U,R>│
         * │ Action     │ Consumer<T>    │ BiConsumer<T,U>  │
         * │ Supply     │ Supplier<T>    │ (none)           │
         * └────────────┴────────────────┴──────────────────┘
         * 
         * When to use binary:
         * - Map operations (key, value)
         * - Pair processing
         * - Two-argument operations
         * 
         * Note: No tri-variants (3 parameters)
         * For 3+ params: Create custom interface
         */
    }
    
    public static void main(String[] args) {
        biFunctionInterface();
        biConsumerInterface();
        biPredicateInterface();
    }
}
```

## 3. UnaryOperator and BinaryOperator

### Specialized Function Variants

```java
/**
 * OPERATOR FUNCTIONAL INTERFACES
 * 
 * UnaryOperator:  T -> T (Function specialization)
 * BinaryOperator: (T, T) -> T (BiFunction specialization)
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class OperatorInterfaces {
    
    /**
     * UnaryOperator<T> - Transform same type
     */
    public static void unaryOperatorInterface() {
        System.out.println("=== UnaryOperator<T> ===");
        
        // Basic UnaryOperator
        UnaryOperator<String> toUpper = String::toUpperCase;
        System.out.println(toUpper.apply("hello"));
        
        // Numeric operation
        UnaryOperator<Integer> square = n -> n * n;
        System.out.println("5² = " + square.apply(5));
        
        // With replaceAll
        List<String> words = new ArrayList<>(Arrays.asList("apple", "banana"));
        words.replaceAll(String::toUpperCase);  // UnaryOperator
        System.out.println("Upper case: " + words);
        
        // Chaining
        UnaryOperator<String> trim = String::trim;
        UnaryOperator<String> upper = String::toUpperCase;
        UnaryOperator<String> trimAndUpper = trim.andThen(upper);
        System.out.println(trimAndUpper.apply("  hello  "));
        
        // Identity
        UnaryOperator<String> identity = UnaryOperator.identity();
        System.out.println("Identity: " + identity.apply("test"));
        
        /**
         * UNARYOPERATOR<T>:
         * 
         * @FunctionalInterface
         * interface UnaryOperator<T> extends Function<T, T> {
         *   // Inherits: T apply(T t);
         * }
         * 
         * Purpose: Transform T to same T
         * 
         * vs Function<T, T>:
         * - Same functionality
         * - More semantic
         * - Clearer intent
         * 
         * Common uses:
         * - List.replaceAll(unaryOperator)
         * - Stream.iterate(seed, unaryOperator)
         * - Transformations: transform(t, unaryOperator)
         * 
         * Static method:
         * - identity() -> t -> t
         * 
         * When to use:
         * Use UnaryOperator when input/output same type
         * Makes intent clearer than Function<T, T>
         */
    }
    
    /**
     * BinaryOperator<T> - Combine same types
     */
    public static void binaryOperatorInterface() {
        System.out.println("\n=== BinaryOperator<T> ===");
        
        // Basic BinaryOperator
        BinaryOperator<Integer> add = (a, b) -> a + b;
        System.out.println("5 + 3 = " + add.apply(5, 3));
        
        // Using method reference
        BinaryOperator<Integer> sum = Integer::sum;
        System.out.println("10 + 20 = " + sum.apply(10, 20));
        
        // String concatenation
        BinaryOperator<String> concat = (a, b) -> a + b;
        System.out.println(concat.apply("Hello ", "World"));
        
        // With reduce
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
        int total = numbers.stream()
            .reduce(0, Integer::sum);  // BinaryOperator
        System.out.println("Sum: " + total);
        
        // Finding max/min
        BinaryOperator<Integer> max = BinaryOperator.maxBy(Integer::compareTo);
        BinaryOperator<Integer> min = BinaryOperator.minBy(Integer::compareTo);
        
        System.out.println("Max of 5 and 3: " + max.apply(5, 3));
        System.out.println("Min of 5 and 3: " + min.apply(5, 3));
        
        // With Map.merge()
        Map<String, Integer> counts = new HashMap<>();
        counts.put("a", 1);
        counts.merge("a", 2, Integer::sum);  // BinaryOperator
        System.out.println("Counts: " + counts);
        
        /**
         * BINARYOPERATOR<T>:
         * 
         * @FunctionalInterface
         * interface BinaryOperator<T> extends BiFunction<T, T, T> {
         *   // Inherits: T apply(T t1, T t2);
         * }
         * 
         * Purpose: Combine two T into one T
         * 
         * vs BiFunction<T, T, T>:
         * - Same functionality
         * - More semantic
         * - Clearer intent
         * 
         * Common uses:
         * - Reduction: stream.reduce(identity, binaryOperator)
         * - Merging: map.merge(key, value, binaryOperator)
         * - Accumulation: accumulate(a, b, binaryOperator)
         * 
         * Static methods:
         * - maxBy(Comparator) -> max element
         * - minBy(Comparator) -> min element
         * 
         * When to use:
         * Use BinaryOperator when:
         * - All types same (T, T) -> T
         * - Combining/merging/reducing
         */
    }
    
    /**
     * Practical examples
     */
    public static void practicalExamples() {
        System.out.println("\n=== Practical Examples ===");
        
        // UnaryOperator: Data transformation pipeline
        List<String> data = new ArrayList<>(
            Arrays.asList("  apple  ", "  BANANA  ", "  Cherry  ")
        );
        
        data.replaceAll(String::trim);
        data.replaceAll(String::toLowerCase);
        System.out.println("Cleaned: " + data);
        
        // BinaryOperator: Aggregation
        List<String> words = Arrays.asList("Hello", " ", "World", "!");
        String sentence = words.stream()
            .reduce("", (a, b) -> a + b);
        System.out.println("Sentence: " + sentence);
        
        // BinaryOperator: Finding extremes
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David");
        Optional<String> longest = names.stream()
            .reduce(BinaryOperator.maxBy(Comparator.comparing(String::length)));
        longest.ifPresent(s -> System.out.println("Longest: " + s));
        
        // UnaryOperator: Iterative generation
        Stream<Integer> powers = Stream.iterate(1, n -> n * 2)
            .limit(10);
        System.out.println("Powers of 2: " + 
            powers.collect(Collectors.toList()));
    }
    
    /**
     * Operator comparison
     */
    public static void operatorComparison() {
        /**
         * OPERATOR COMPARISON:
         * 
         * ┌────────────────┬─────────────┬──────────────────┐
         * │ Operator       │ Signature   │ Extends          │
         * ├────────────────┼─────────────┼──────────────────┤
         * │ UnaryOperator  │ T -> T      │ Function<T,T>    │
         * │ BinaryOperator │ (T,T) -> T  │ BiFunction<T,T,T>│
         * └────────────────┴─────────────┴──────────────────┘
         * 
         * When to use Operator vs Function:
         * 
         * Same types → Operator (semantic clarity)
         * Different types → Function
         * 
         * Example:
         * UnaryOperator<String> vs Function<String, String>
         * → Use UnaryOperator (clearer intent)
         * 
         * Function<String, Integer> vs UnaryOperator
         * → Use Function (different types)
         */
    }
    
    public static void main(String[] args) {
        unaryOperatorInterface();
        binaryOperatorInterface();
        practicalExamples();
    }
}
```

## 4. Primitive Specializations

### Avoiding Boxing Overhead

```java
/**
 * PRIMITIVE SPECIALIZATIONS
 * 
 * IntPredicate, IntFunction, IntConsumer, IntSupplier, etc.
 * LongPredicate, LongFunction, etc.
 * DoublePredicate, DoubleFunction, etc.
 */

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class PrimitiveSpecializations {
    
    /**
     * Int specializations
     */
    public static void intSpecializations() {
        System.out.println("=== Int Specializations ===");
        
        // IntPredicate
        IntPredicate isEven = n -> n % 2 == 0;
        System.out.println("4 is even: " + isEven.test(4));
        System.out.println("5 is even: " + isEven.test(5));
        
        // With IntStream
        long evenCount = IntStream.rangeClosed(1, 10)
            .filter(isEven)  // IntPredicate - no boxing!
            .count();
        System.out.println("Even count: " + evenCount);
        
        // IntFunction<R>
        IntFunction<String> intToString = n -> "Number: " + n;
        System.out.println(intToString.apply(42));
        
        // IntConsumer
        IntConsumer printer = n -> System.out.println("Value: " + n);
        IntStream.rangeClosed(1, 3).forEach(printer);
        
        // IntSupplier
        IntSupplier randomInt = () -> (int) (Math.random() * 100);
        System.out.println("Random: " + randomInt.getAsInt());
        
        // IntUnaryOperator
        IntUnaryOperator square = n -> n * n;
        int[] numbers = {1, 2, 3, 4, 5};
        Arrays.setAll(numbers, square);
        System.out.println("Squared: " + Arrays.toString(numbers));
        
        // IntBinaryOperator
        IntBinaryOperator add = (a, b) -> a + b;
        int sum = IntStream.rangeClosed(1, 5)
            .reduce(0, add);
```

```java
/**
 * FUNCTION COMPOSITION

 * Combining functions to create new functions

import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class FunctionComposition {
    
    /**
     * Function composition - andThen
     */
    public static void andThenComposition() {
        System.out.println("=== andThen() ===");
        
        // Basic andThen
        Function<String, String> trim = String::trim;
        Function<String, String> upper = String::toUpperCase;
        Function<String, String> trimThenUpper = trim.andThen(upper);
        
        System.out.println(trimThenUpper.apply("  hello  "));  // HELLO
        
        // Chaining multiple
        Function<String, Integer> length = String::length;
        Function<String, Integer> pipeline = trim
            .andThen(upper)
            .andThen(length);
        System.out.println("Length: " + pipeline.apply("  hello  "));  // 5
        
        // Consumer andThen
        Consumer<String> printOriginal = s -> 
            System.out.println("Original: " + s);
        Consumer<String> printUpper = s -> 
            System.out.println("Upper: " + s.toUpperCase());
        Consumer<String> both = printOriginal.andThen(printUpper);
        both.accept("hello");
        
        /**
         * ANDTHEN():
         * 
         * Available on:
         * - Function<T, R>
         * - Consumer<T>
         * - UnaryOperator<T>
         * - BiFunction<T, U, R>
         * - BiConsumer<T, U>
         * 
         * Signature:
         * Function<T, R>.andThen(Function<R, V>) -> Function<T, V>
         * 
         * Execution order:
         * f.andThen(g)
         * = first f, then g
         * = g(f(x))
         * 
         * Example:
         * trim.andThen(upper)
         * = upper(trim(x))
         * = first trim, then upper
         */
    }
    
    /**
     * Function composition - compose
     */
    public static void composeComposition() {
        System.out.println("\n=== compose() ===");
        
        // Basic compose
        Function<String, String> trim = String::trim;
        Function<String, String> upper = String::toUpperCase;
        Function<String, String> upperComposeTrim = upper.compose(trim);
        
        System.out.println(upperComposeTrim.apply("  hello  "));  // HELLO
        
        // Compose vs andThen
        Function<Integer, Integer> addOne = n -> n + 1;
        Function<Integer, Integer> multiplyTwo = n -> n * 2;
        
        // andThen: (2 + 1) * 2 = 6
        Function<Integer, Integer> andThenResult = addOne.andThen(multiplyTwo);
        System.out.println("andThen: " + andThenResult.apply(2));  // 6
        
        // compose: (2 * 2) + 1 = 5
        Function<Integer, Integer> composeResult = addOne.compose(multiplyTwo);
        System.out.println("compose: " + composeResult.apply(2));  // 5
        
        /**
         * COMPOSE():
         * 
         * Available on:
         * - Function<T, R>
         * - UnaryOperator<T>
         * 
         * NOT on:
         * - Consumer (no return value)
         * - BiFunction (ambiguous which param)
         * 
         * Signature:
         * Function<T, R>.compose(Function<V, T>) -> Function<V, R>
         * 
         * Execution order:
         * f.compose(g)
         * = first g, then f
         * = f(g(x))
         * 
         * vs andThen:
         * f.andThen(g) = g(f(x))   // f first
         * f.compose(g) = f(g(x))   // g first
         * 
         * Mnemonic:
         * f.andThen(g) = "f AND THEN g"
         * f.compose(g) = "f COMPOSED WITH g first"
         */
    }
    
    /**
     * Predicate composition - and, or, negate
     */
    public static void predicateComposition() {
        System.out.println("\n=== Predicate Composition ===");
        
        Predicate<String> isLong = s -> s.length() > 5;
        Predicate<String> startsWithA = s -> s.startsWith("a");
        
        // and()
        Predicate<String> longAndStartsWithA = isLong.and(startsWithA);
        System.out.println("'application': " + 
            longAndStartsWithA.test("application"));  // true
        System.out.println("'apple': " + 
            longAndStartsWithA.test("apple"));  // false (not long)
        
        // or()
        Predicate<String> longOrStartsWithA = isLong.or(startsWithA);
        System.out.println("'apple': " + 
            longOrStartsWithA.test("apple"));  // true (starts with a)
        
        // negate()
        Predicate<String> isShort = isLong.negate();
        System.out.println("'hi': " + isShort.test("hi"));  // true
        
        // Complex combinations
        Predicate<String> complex = isLong
            .and(startsWithA)
            .or(s -> s.contains("z"));
        
        /**
         * PREDICATE COMPOSITION:
         * 
         * and(Predicate other):
         * - Logical AND
         * - p1.and(p2) = p1(x) && p2(x)
         * - Short-circuits (if p1 false, p2 not evaluated)
         * 
         * or(Predicate other):
         * - Logical OR
         * - p1.or(p2) = p1(x) || p2(x)
         * - Short-circuits (if p1 true, p2 not evaluated)
         * 
         * negate():
         * - Logical NOT
         * - p.negate() = !p(x)
         * 
         * Static helper:
         * Predicate.not(predicate)  // Java 11+
         * = predicate.negate()
         * 
         * Usage with streams:
         * list.stream()
         *   .filter(isLong.and(startsWithA))
         *   .collect(toList());
         */
    }
    
    /**
     * BiPredicate composition
     */
    public static void biPredicateComposition() {
        System.out.println("\n=== BiPredicate Composition ===");
        
        BiPredicate<String, Integer> longerThan = (s, n) -> s.length() > n;
        BiPredicate<String, Integer> shorterThan = (s, n) -> s.length() < n;
        
        // and()
        BiPredicate<String, Integer> inRange = 
            longerThan.and((s, n) -> s.length() < n + 10);
        System.out.println("'hello' in range 3: " + 
            inRange.test("hello", 3));
        
        // or()
        BiPredicate<String, Integer> extremes = 
            longerThan.or(shorterThan);
        
        // negate()
        BiPredicate<String, Integer> notLonger = longerThan.negate();
        
        /**
         * BIPREDICATE COMPOSITION:
         * 
         * Same methods as Predicate:
         * - and(BiPredicate)
         * - or(BiPredicate)
         * - negate()
         * 
         * Works identically to Predicate
         */
    }
    
    /**
     * Practical composition examples
     */
    public static void practicalExamples() {
        System.out.println("\n=== Practical Examples ===");
        
        // Data cleaning pipeline
        Function<String, String> clean = s -> s
            .trim();
        Function<String, String> normalize = s -> s
            .toLowerCase()
            .replaceAll("\\s+", " ");
        Function<String, String> cleanPipeline = clean.andThen(normalize);
        
        System.out.println("Cleaned: '" + 
            cleanPipeline.apply("  Hello   World  ") + "'");
        
        // Validation pipeline
        Predicate<String> notEmpty = s -> !s.isEmpty();
        Predicate<String> hasMinLength = s -> s.length() >= 3;
        Predicate<String> hasMaxLength = s -> s.length() <= 20;
        Predicate<String> isValid = notEmpty
            .and(hasMinLength)
            .and(hasMaxLength);
        
        List<String> inputs = Arrays.asList("ab", "hello", "this is a very long string");
        List<String> valid = inputs.stream()
            .filter(isValid)
            .collect(Collectors.toList());
        System.out.println("Valid: " + valid);
        
        // Transformation chain
        Function<String, String> step1 = String::trim;
        Function<String, String> step2 = String::toLowerCase;
        Function<String, Integer> step3 = String::length;
        Function<Integer, Boolean> step4 = n -> n > 5;
        
        Function<String, Boolean> fullPipeline = step1
            .andThen(step2)
            .andThen(step3)
            .andThen(step4);
        
        System.out.println("'  HELLO  ' pipeline: " + 
            fullPipeline.apply("  HELLO  "));
    }
    
    /**
     * Composition performance
     */
    public static void compositionPerformance() {
        /**
         * COMPOSITION PERFORMANCE:
         * 
         * Composition creates new function object
         * But after JIT warming: zero overhead
         * 
         * f.andThen(g).andThen(h)
         * - Creates 2 composition objects
         * - JIT inlines to: h(g(f(x)))
         * - Same as manual composition
         * 
         * BEST PRACTICES:
         * 
         * 1. Define reusable compositions:
         *    Function<String, String> clean = trim.andThen(upper);
         *    // Reuse clean multiple times
         * 
         * 2. Don't over-compose:
         *    function1.andThen(f2).andThen(f3).andThen(f4)...
         *    Can hurt readability
         * 
         * 3. Name compositions meaningfully:
         *    cleanAndValidate, normalizeAndFormat
         * 
         * 4. Use for common patterns:
         *    - Data cleaning
         *    - Validation chains
         *    - Transformation pipelines
         */
    }
    
    public static void main(String[] args) {
        andThenComposition();
        composeComposition();
        predicateComposition();
        biPredicateComposition();
        practicalExamples();
    }
```

```java
/**
 * CUSTOM FUNCTIONAL INTERFACES
 * 
 * Creating domain-specific functional interfaces
 */

import java.util.function.*;

public class CustomFunctionalInterfaces {
    
    /**
     * Simple custom interface
     */
    @FunctionalInterface
    interface StringProcessor {
        String process(String input);
        
        // Can have default methods
        default StringProcessor andThen(StringProcessor after) {
            return input -> after.process(this.process(input));
        }
        
        // Can have static methods
        static StringProcessor identity() {
            return s -> s;
        }
    }
    
    public static void simpleCustomInterface() {
        System.out.println("=== Simple Custom Interface ===");
        
        StringProcessor toUpper = String::toUpperCase;
        StringProcessor addExclamation = s -> s + "!";
        
        StringProcessor combined = toUpper.andThen(addExclamation);
        System.out.println(combined.process("hello"));  // HELLO!
        
        /**
         * CUSTOM INTERFACE RULES:
         * 
         * 1. Annotate with @FunctionalInterface
         * 2. Exactly ONE abstract method
         * 3. Can have default methods
         * 4. Can have static methods
         * 5. Can extend other interfaces
         * 
         * @FunctionalInterface ensures:
         * - Compile error if not functional
         * - Clear intent for readers
         * - Tool support
         */
    }
    
    /**
     * Custom interface with multiple parameters
     */
    @FunctionalInterface
    interface TriFunction<T, U, V, R> {
        R apply(T t, U u, V v);
        
        // Helper for common case
        static <T> TriFunction<T, T, T, T> summing() {
            return (a, b, c) -> {
                @SuppressWarnings("unchecked")
                T result = (T) Integer.valueOf(
                    ((Integer) a) + ((Integer) b) + ((Integer) c)
                );
                return result;
            };
        }
    }
    
    public static void multipleparameterInterface() {
        System.out.println("\n=== Multiple Parameter Interface ===");
        
        TriFunction<Integer, Integer, Integer, Integer> sum = 
            (a, b, c) -> a + b + c;
        System.out.println("Sum: " + sum.apply(1, 2, 3));
        
        TriFunction<String, String, String, String> concat = 
            (a, b, c) -> a + b + c;
        System.out.println("Concat: " + concat.apply("Hello", " ", "World"));
        
        /**
         * WHEN TO CREATE CUSTOM:
         * 
         * Need 3+ parameters:
         * - TriFunction<T, U, V, R>
         * - QuadFunction<T, U, V, W, R>
         * - Standard library only has Bi*
         * 
         * Domain-specific:
         * - Better names for clarity
         * - StringProcessor vs Function<String, String>
         * - UserValidator vs Predicate<User>
         * 
         * Special requirements:
         * - Checked exceptions
         * - Multiple abstract methods (not functional!)
         * - Custom default methods
         */
    }
    
    /**
     * Domain-specific interfaces
     */
    @FunctionalInterface
    interface UserValidator {
        boolean validate(User user);
        
        default UserValidator and(UserValidator other) {
            return user -> this.validate(user) && other.validate(user);
        }
        
        static UserValidator ageRange(int min, int max) {
            return user -> user.age >= min && user.age <= max;
        }
    }
    
    @FunctionalInterface
    interface PriceCalculator {
        double calculate(double basePrice, int quantity);
        
        static PriceCalculator withDiscount(double discountPercent) {
            return (price, qty) -> price * qty * (1 - discountPercent / 100);
        }
    }
    
    static class User {
        String name;
        int age;
        User(String name, int age) {
            this.name = name;
            this.age = age;
        }
    }
    
    public static void domainSpecificInterfaces() {
        System.out.println("\n=== Domain-Specific Interfaces ===");
        
        // UserValidator
        UserValidator isAdult = user -> user.age >= 18;
        UserValidator hasName = user -> user.name != null && !user.name.isEmpty();
        UserValidator isValid = isAdult.and(hasName);
        
        User user1 = new User("Alice", 25);
        User user2 = new User("", 16);
        
        System.out.println("Alice valid: " + isValid.validate(user1));
        System.out.println("'' valid: " + isValid.validate(user2));
        
        // PriceCalculator
        PriceCalculator regularPrice = (price, qty) -> price * qty;
        PriceCalculator discounted = PriceCalculator.withDiscount(10);
        
        System.out.println("Regular: $" + regularPrice.calculate(100, 3));
        System.out.println("Discounted: $" + discounted.calculate(100, 3));
        
        /**
         * DOMAIN-SPECIFIC BENEFITS:
         * 
         * 1. CLARITY:
         *    UserValidator vs Predicate<User>
         *    More obvious purpose
         * 
         * 2. TYPE SAFETY:
         *    Can't accidentally use wrong validator
         * 
         * 3. DOMAIN METHODS:
         *    ageRange(), withDiscount()
         *    Factory methods specific to domain
         * 
         * 4. DOCUMENTATION:
         *    Interface name documents intent
         *    Better than generic Function
         */
    }
    
    /**
     * Checked exception handling
     */
    @FunctionalInterface
    interface CheckedFunction<T, R> {
        R apply(T t) throws Exception;
        
        static <T, R> Function<T, R> unchecked(CheckedFunction<T, R> function) {
            return t -> {
                try {
                    return function.apply(t);
                } catch (Exception e) {
                    throw new RuntimeException(e);
                }
            };
        }
    }
    
    public static void checkedExceptionHandling() {
        System.out.println("\n=== Checked Exception Handling ===");
        
        // Method that throws checked exception
        CheckedFunction<String, Integer> parseInt = Integer::parseInt;
        
        // Convert to unchecked for use in streams
        Function<String, Integer> uncheckedParseInt = 
            CheckedFunction.unchecked(parseInt);
        
        try {
            Integer result = uncheckedParseInt.apply("42");
            System.out.println("Parsed: " + result);
        } catch (RuntimeException e) {
            System.out.println("Error: " + e.getMessage());
        }
        
        /**
         * CHECKED EXCEPTION PROBLEM:
         * 
         * Standard functional interfaces don't throw:
         * Function<T, R>.apply() doesn't declare throws
         * 
         * Solutions:
         * 
         * 1. Custom interface with throws:
         *    @FunctionalInterface
         *    interface ThrowingFunction<T, R> {
         *      R apply(T t) throws Exception;
         *    }
         * 
         * 2. Wrapper method:
         *    static <T, R> Function<T, R> unchecked(
         *      ThrowingFunction<T, R> f
         *    ) {
         *      return t -> {
         *        try { return f.apply(t); }
         *        catch (Exception e) { 
         *          throw new RuntimeException(e); 
         *        }
         *      };
         *    }
         * 
         * 3. Try-catch inside lambda:
         *    list.stream().map(s -> {
         *      try { return parse(s); }
         *      catch (Exception e) { return null; }
         *    })
         */
    }
    
    /**
     * When to use built-in vs custom
     */
    public static void builtInVsCustom() {
        /**
         * USE BUILT-IN WHEN:
         * 
         * ✓ Standard operations (test, apply, accept, get)
         * ✓ Generic transformations
         * ✓ Stream operations
         * ✓ 0-2 parameters
         * ✓ No checked exceptions
         * 
         * Examples:
         * - Predicate<User> for filtering
         * - Function<String, Integer> for parsing
         * - Consumer<String> for logging
         * 
         * 
         * CREATE CUSTOM WHEN:
         * 
         * ✓ 3+ parameters needed
         * ✓ Domain-specific naming improves clarity
         * ✓ Need checked exceptions
         * ✓ Want domain-specific factory methods
         * ✓ Complex validation/calculation logic
         * 
         * Examples:
         * - TriFunction<T, U, V, R> for 3 params
         * - UserValidator for domain clarity
         * - CheckedFunction for exception handling
         * - PriceCalculator for business logic
         * 
         * 
         * GUIDELINES:
         * 
         * 1. START WITH BUILT-IN:
         *    Use Predicate, Function, etc. first
         * 
         * 2. EXTRACT IF NEEDED:
         *    If code unclear, extract to custom
         * 
         * 3. NAME MEANINGFULLY:
         *    StringProcessor > Function<String, String>
         * 
         * 4. AVOID OVER-ENGINEERING:
         *    Don't create custom for everything
         *    Balance clarity vs simplicity
         */
    }
    
    public static void main(String[] args) {
        simpleCustomInterface();
        multipleparameterInterface();
        domainSpecificInterfaces();
        checkedExceptionHandling();
    }
}
```

## Summary

### Quick Reference

**Core Four Interfaces:**

```java
Predicate<T>      T → boolean    test()    // Filter
Function<T, R>    T → R          apply()   // Transform
Consumer<T>       T → void       accept()  // Action
Supplier<T>       () → T         get()     // Provide
```

**Binary Variants:**

```java
BiPredicate<T, U>   (T, U) → boolean   test()
BiFunction<T, U, R> (T, U) → R         apply()
BiConsumer<T, U>    (T, U) → void      accept()
// No BiSupplier (Java has no tuples)
```

**Operators:**

```java
UnaryOperator<T>    T → T           // extends Function<T, T>
```

### Usage Patterns

**Stream Operations:**

```java
list.stream()
    .filter(Predicate)          // Keep matching
    .map(Function)              // Transform
    .peek(Consumer)             // Side effect
    .forEach(Consumer)          // Terminal action

// Reduction
.reduce(identity, BinaryOperator)

// Specialized
.mapToInt(ToIntFunction)        // Avoid boxing!
.filter(IntPredicate)           // 5x faster
```

**Optional Operations:**

```java
optional.map(Function)          // Transform
optional.flatMap(Function)      // Flatten
optional.filter(Predicate)      // Test
optional.ifPresent(Consumer)    // Action
optional.orElseGet(Supplier)    // Default
```

**Map Operations:**

```java
map.computeIfAbsent(key, Function)      // Create if missing
map.merge(key, value, BinaryOperator)   // Combine values
map.forEach(BiConsumer)                 // Iterate pairs
map.replaceAll(BiFunction)              // Transform values
```

### Best Practices

**✅ DO:**

- Use primitive specializations (IntPredicate, etc.)
- Compose functions for pipelines
- Use built-in interfaces when possible
- Name custom interfaces meaningfully
- Add @FunctionalInterface annotation
- Provide factory methods on custom interfaces

**❌ DON'T:**

- Use Predicate<Integer> (use IntPredicate)
- Create custom interface for 1-2 params
- Forget @FunctionalInterface annotation
- Add multiple abstract methods
- Use checked exceptions in lambdas (wrap them)

### Common Gotchas

**1. Wrong primitive type:**

```java
// ❌ BAD - Boxing overhead
stream.filter(n -> n > 0)  // Predicate<Integer>

// ✅ GOOD - No boxing
intStream.filter(n -> n > 0)  // IntPredicate
```

**2. Checked exceptions:**

```java
// ❌ BAD - Won't compile
list.stream().map(Integer::parseInt)  // NumberFormatException

// ✅ GOOD - Wrap or use custom interface
list.stream().map(s -> {
    try { return Integer.parseInt(s); }
    catch (NumberFormatException e) { return 0; }
})
```

**3. Composition order:**

```java
f.andThen(g)    // f then g = g(f(x))
f.compose(g)    // g then f = f(g(x))
// Different order!
```
