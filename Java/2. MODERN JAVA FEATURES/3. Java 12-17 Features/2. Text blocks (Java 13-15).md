# Text Blocks (Java 13-15)

## Overview

Java 13 introduced text blocks as a preview feature, providing a cleaner way to write multiline strings. Text blocks use triple quotes (`"""`) to delimit strings that span multiple lines, automatically handling indentation and line breaks without requiring escape sequences or concatenation. The compiler intelligently strips incidental indentation (code formatting) while preserving essential indentation (actual content structure). Java 14 added new escape sequences (`\s` for space, `\` for line continuation) and Java 15 finalized the feature. While Java doesn't have built-in string interpolation like some languages, text blocks still dramatically improve readability for SQL queries, JSON templates, HTML content, and any multiline text. This guide covers the triple-quote syntax, how indentation is automatically managed, available escape sequences, workarounds for interpolation, and practical use cases.

## 1. Multiline Strings with """

### Basic Text Block Syntax

```java
/**
 * TEXT BLOCKS
 * 
 * Java 13-15 multiline string syntax
 */

public class TextBlockBasics {
    
    /**
     * Traditional multiline strings
     */
    public static void traditionalMultiline() {
        System.out.println("=== Traditional Multiline ===");
        
        // Before text blocks - painful!
        String html = "<html>\n" +
                      "  <body>\n" +
                      "    <h1>Hello World</h1>\n" +
                      "  </body>\n" +
                      "</html>\n";
        
        System.out.println(html);
        
        String json = "{\n" +
                      "  \"name\": \"John\",\n" +
                      "  \"age\": 30,\n" +
                      "  \"city\": \"New York\"\n" +
                      "}\n";
        
        System.out.println(json);
        
        /**
         * PROBLEMS:
         * 
         * - Explicit \n everywhere
         * - String concatenation with +
         * - Escaped quotes \"
         * - Hard to read
         * - Hard to maintain
         * - Error-prone
         * - Visual structure lost
         */
    }
    
    /**
     * Text blocks - clean and clear
     */
    public static void textBlocks() {
        System.out.println("\n=== Text Blocks ===");
        
        // With text blocks - beautiful!
        String html = """
            <html>
              <body>
                <h1>Hello World</h1>
              </body>
            </html>
            """;
        
        System.out.println(html);
        
        String json = """
            {
              "name": "John",
              "age": 30,
              "city": "New York"
            }
            """;
        
        System.out.println(json);
        
        /**
         * BENEFITS:
         * 
         * ✓ No \n needed
         * ✓ No + concatenation
         * ✓ No \" escaping (usually)
         * ✓ What you see is what you get
         * ✓ Easy to read
         * ✓ Easy to maintain
         * ✓ Visual structure preserved
         */
    }
    
    /**
     * Basic syntax rules
     */
    public static void syntaxRules() {
        System.out.println("\n=== Syntax Rules ===");
        
        // Opening """ must be on own line
        String valid = """
            Content here
            """;
        
        // WRONG - content on same line as opening """
        // String invalid = """Content here
        //     """;
        
        // Closing """ can be on own line or with content
        String option1 = """
            Line 1
            Line 2
            """;  // Ends with newline
        
        String option2 = """
            Line 1
            Line 2""";  // No final newline
        
        System.out.println("Option 1 ends with newline: " + 
            (option1.endsWith("\n")));
        System.out.println("Option 2 ends with newline: " + 
            (option2.endsWith("\n")));
        
        /**
         * SYNTAX RULES:
         * 
         * 1. Opening """:
         *    - Must be followed by line terminator
         *    - Cannot have content on same line
         *    - Must be on own line
         * 
         * 2. Content:
         *    - Starts on next line
         *    - Can span multiple lines
         *    - Preserves formatting
         * 
         * 3. Closing """:
         *    - Can be on own line (includes final \n)
         *    - Can be on last content line (no final \n)
         *    - Determines final newline
         * 
         * Valid:
         * """
         * content
         * """
         * 
         * Invalid:
         * """content
         * """
         */
    }
    
    /**
     * Empty and single-line text blocks
     */
    public static void edgeCases() {
        System.out.println("\n=== Edge Cases ===");
        
        // Empty text block
        String empty = """
            """;
        System.out.println("Empty length: " + empty.length());
        
        // Single line
        String single = """
            Hello World
            """;
        System.out.println("Single: '" + single + "'");
        
        // Just whitespace
        String spaces = """
                
            """;
        System.out.println("Spaces length: " + spaces.length());
        
        /**
         * EDGE CASES:
         * 
         * Empty:
         * """
         * """
         * → ""
         * 
         * Single line:
         * """
         * Hello
         * """
         * → "Hello\n"
         * 
         * Whitespace:
         * """
         *   
         * """
         * → "  \n"
         */
    }
    
    /**
     * Text blocks are just Strings
     */
    public static void justStrings() {
        System.out.println("\n=== Just Strings ===");
        
        String traditional = "Hello\nWorld\n";
        String textBlock = """
            Hello
            World
            """;
        
        // They're equal!
        System.out.println("Equal: " + traditional.equals(textBlock));
        
        // Can use all String methods
        System.out.println("Length: " + textBlock.length());
        System.out.println("Lines: " + textBlock.lines().count());
        System.out.println("Upper: " + textBlock.toUpperCase());
        
        // Can concatenate
        String combined = textBlock + traditional;
        System.out.println("Combined: " + combined);
        
        /**
         * TEXT BLOCKS ARE STRINGS:
         * 
         * - Type: String (not special type)
         * - All String methods work
         * - Can be concatenated
         * - Can be compared with equals()
         * - Stored in string pool
         * - Compile-time constants
         * 
         * Just syntactic sugar for:
         * - Multiline content
         * - Better readability
         * - Automatic indentation
         */
    }
    
    /**
     * Comparison: Before and After
     */
    public static void comparison() {
        System.out.println("\n=== Comparison ===");
        
        // SQL query example
        
        // Before:
        String sqlBefore = "SELECT id, name, email\n" +
                          "FROM users\n" +
                          "WHERE status = 'active'\n" +
                          "  AND created_at > '2023-01-01'\n" +
                          "ORDER BY name\n";
        
        // After:
        String sqlAfter = """
            SELECT id, name, email
            FROM users
            WHERE status = 'active'
              AND created_at > '2023-01-01'
            ORDER BY name
            """;
        
        System.out.println("Equal: " + sqlBefore.equals(sqlAfter));
        
        /**
         * COMPARISON:
         * 
         * Before:
         * - 5 lines of code
         * - 5 \n escapes
         * - 4 + concatenations
         * - Hard to read structure
         * 
         * After:
         * - 5 lines of code (but natural)
         * - 0 \n escapes
         * - 0 + concatenations
         * - Clear structure
         * 
         * Readability: 10x better
         * Maintainability: Much easier
         * Error rate: Much lower
         */
    }
    
    public static void main(String[] args) {
        traditionalMultiline();
        textBlocks();
        syntaxRules();
        edgeCases();
        justStrings();
        comparison();
    }
}
```

## 2. Indentation Handling

### Automatic Indentation Management

```java
/**
 * INDENTATION HANDLING
 * 
 * How text blocks handle indentation
 */

public class IndentationHandling {
    
    /**
     * Incidental vs Essential indentation
     */
    public static void incidentalVsEssential() {
        System.out.println("=== Incidental vs Essential ===");
        
        String code = """
            public class Hello {
                public static void main(String[] args) {
                    System.out.println("Hello World");
                }
            }
            """;
        
        System.out.println(code);
        System.out.println("--- Visual separator ---");
        
        /**
         * INCIDENTAL INDENTATION:
         * 
         * Definition:
         * Indentation added for code formatting
         * (alignment with surrounding Java code)
         * 
         * Automatically stripped by compiler
         * 
         * Example:
         * public void method() {
         *     String s = """
         *         Line 1    ← 8 spaces incidental
         *         Line 2    ← 8 spaces incidental
         *         """;
         * }
         * 
         * Result:
         * "Line 1\nLine 2\n"
         * (incidental 8 spaces removed)
         * 
         * 
         * ESSENTIAL INDENTATION:
         * 
         * Definition:
         * Indentation that's part of content
         * (meaningful structure)
         * 
         * Preserved by compiler
         * 
         * Example:
         * String s = """
         *     Line 1      ← 4 incidental
         *       Line 2    ← 4 incidental + 2 essential
         *         Line 3  ← 4 incidental + 4 essential
         *     """;
         * 
         * Result:
         * "Line 1\n  Line 2\n    Line 3\n"
         * (incidental 4 removed, essential preserved)
         */
    }
    
    /**
     * How indentation is determined
     */
    public static void determinationRules() {
        System.out.println("\n=== Determination Rules ===");
        
        // Closing """ determines base indentation
        String example1 = """
            Line 1
            Line 2
            """;  // Closing at column 12
        
        System.out.println("Example 1: '" + example1 + "'");
        
        String example2 = """
            Line 1
            Line 2
        """;  // Closing at column 8
        
        System.out.println("Example 2: '" + example2 + "'");
        
        String example3 = """
            Line 1
            Line 2
    """;  // Closing at column 4
        
        System.out.println("Example 3: '" + example3 + "'");
        
        /**
         * DETERMINATION ALGORITHM:
         * 
         * 1. Find leftmost non-blank line
         * 2. Count leading whitespace
         * 3. That's incidental indentation
         * 4. Strip from all lines
         * 
         * Example:
         * """
         *     Line 1      ← 4 spaces (leftmost)
         *       Line 2    ← 6 spaces
         * """
         * 
         * Incidental: 4 spaces (minimum)
         * Result:
         * "Line 1\n  Line 2\n"
         * 
         * 
         * CLOSING """ POSITION:
         * 
         * Also determines incidental:
         * """
         * Line 1
         * Line 2
         *     """  ← Closing at 4 spaces
         * 
         * If closing is leftmost:
         * Incidental = closing position
         * 
         * Result:
         * "    Line 1\n    Line 2\n"
         * (content indented relative to closing)
         */
    }
    
    /**
     * Practical examples
     */
    public static void practicalExamples() {
        System.out.println("\n=== Practical Examples ===");
        
        // Example 1: No indentation
        String noIndent = """
Line 1
Line 2
Line 3
""";
        System.out.println("No indent:");
        System.out.println(noIndent);
        
        // Example 2: Preserve structure
        String tree = """
            root
              child1
                grandchild1
              child2
            """;
        System.out.println("Tree:");
        System.out.println(tree);
        
        // Example 3: Code snippet
        String snippet = """
            if (condition) {
                doSomething();
            }
            """;
        System.out.println("Snippet:");
        System.out.println(snippet);
        
        /**
         * RESULTS:
         * 
         * No indent:
         * "Line 1\nLine 2\nLine 3\n"
         * 
         * Tree:
         * "root\n  child1\n    grandchild1\n  child2\n"
         * 
         * Snippet:
         * "if (condition) {\n    doSomething();\n}\n"
         */
    }
    
    /**
     * Controlling indentation
     */
    public static void controllingIndentation() {
        System.out.println("\n=== Controlling Indentation ===");
        
        // Method 1: Position closing """
        String method1 = """
            Content
            More content
            """;  // Closing aligned with opening
        
        String method2 = """
            Content
            More content
        """;  // Closing at column 8
        
        String method3 = """
            Content
            More content
    """;  // Closing at column 4
        
        System.out.println("Method 1 indent: " + 
            (method1.startsWith(" ") ? "yes" : "no"));
        System.out.println("Method 2 indent: " + 
            (method2.startsWith(" ") ? "yes" : "no"));
        System.out.println("Method 3 indent: " + 
            (method3.startsWith(" ") ? "yes" : "no"));
        
        /**
         * CONTROLLING INDENTATION:
         * 
         * 1. Leftmost line:
         *    String s = """
         *    Line 1      ← Start at column 0
         *      Line 2
         *    """;
         *    Result: "Line 1\n  Line 2\n"
         * 
         * 2. Closing position:
         *    String s = """
         *        Line 1
         *        Line 2
         *    """;  ← Closing at column 4
         *    Result: "    Line 1\n    Line 2\n"
         * 
         * 3. Add explicit spaces:
         *    String s = """
         *        Line 1
         *         Line 2
         *        """;
         *    Result: "Line 1\n Line 2\n"
         */
    }
    
    /**
     * Common pitfalls
     */
    public static void commonPitfalls() {
        /**
         * PITFALL 1: Unexpected indentation
         * 
         * void method() {
         *     String s = """
         *         Line 1
         *         Line 2
         *         """;
         * }
         * 
         * Expected: "Line 1\nLine 2\n"
         * Actual: "Line 1\nLine 2\n" ✓
         * 
         * (Incidental 8 spaces stripped)
         * 
         * 
         * PITFALL 2: Tabs vs Spaces
         * 
         * String s = """
         * \tLine 1    ← Tab
         *     Line 2    ← 4 spaces
         * """;
         * 
         * Tabs and spaces mixed!
         * Compiler treats tab as single character
         * Can cause unexpected results
         * 
         * Recommendation: Use spaces only
         * 
         * 
         * PITFALL 3: Closing position
         * 
         * String s = """
         *     Content
         * """;  ← Closing at column 0
         * 
         * Result: "    Content\n"
         * (4 spaces preserved)
         * 
         * If you want no indent:
         * String s = """
         * Content
         * """;
         */
    }
    
    /**
     * String methods for indentation
     */
    public static void stringMethods() {
        System.out.println("\n=== String Methods ===");
        
        String text = """
            Line 1
            Line 2
            """;
        
        // Add indentation
        String indented = text.indent(4);
        System.out.println("Indented:");
        System.out.println(indented);
        
        // Remove indentation
        String dedented = indented.indent(-4);
        System.out.println("Dedented:");
        System.out.println(dedented);
        
        // Strip indentation
        String stripped = text.stripIndent();
        System.out.println("Stripped:");
        System.out.println(stripped);
        
        /**
         * INDENTATION METHODS:
         * 
         * indent(n):
         * - Add n spaces to each line
         * - Negative n removes spaces
         * - Java 12+
         * 
         * stripIndent():
         * - Remove common leading whitespace
         * - Same algorithm as text blocks
         * - Java 13+
         * 
         * Use for:
         * - Adjusting existing strings
         * - Processing external text
         * - Dynamic indentation
         */
    }
    
    public static void main(String[] args) {
        incidentalVsEssential();
        determinationRules();
        practicalExamples();
        controllingIndentation();
        stringMethods();
    }
}
```

## 3. Escape Sequences

### Traditional and New Escape Sequences

```java
/**
 * ESCAPE SEQUENCES
 * 
 * Escape sequences in text blocks
 */

public class EscapeSequences {
    
    /**
     * Traditional escapes still work
     */
    public static void traditionalEscapes() {
        System.out.println("=== Traditional Escapes ===");
        
        String text = """
            Line 1\nLine 2
            Tab\there
            Quote: \"Hello\"
            Backslash: \\
            """;
        
        System.out.println(text);
        
        /**
         * TRADITIONAL ESCAPES:
         * 
         * \n - Newline (usually not needed)
         * \t - Tab
         * \" - Double quote (usually not needed)
         * \' - Single quote
         * \\ - Backslash
         * \r - Carriage return
         * \f - Form feed
         * \b - Backspace
         * 
         * Still work in text blocks
         * But many are unnecessary now
         */
    }
    
    /**
     * New escape sequences (Java 14+)
     */
    public static void newEscapes() {
        System.out.println("\n=== New Escapes ===");
        
        // \s - Escaped space (not stripped)
        String withSpaces = """
            Line 1\s
            Line 2\s\s
            """;
        System.out.println("With \\s: '" + withSpaces + "'");
        
        // \ (backslash at end) - Line continuation
        String continued = """
            This is a very long line \
            that continues here
            """;
        System.out.println("Continued: '" + continued + "'");
        
        /**
         * NEW ESCAPES (JAVA 14):
         * 
         * \s (space):
         * - Represents single space
         * - Not stripped by indentation
         * - Preserves trailing spaces
         * 
         * Use when:
         * - Need trailing spaces
         * - Prevent stripping
         * - Explicit spacing
         * 
         * 
         * \ (line continuation):
         * - Joins lines
         * - No newline inserted
         * - Like string concatenation
         * 
         * Use when:
         * - Very long lines
         * - Avoid wrapping
         * - Format code nicely
         */
    }
    
    /**
     * Escaped space (\s) examples
     */
    public static void escapedSpace() {
        System.out.println("\n=== Escaped Space ===");
        
        // Without \s - trailing spaces stripped
        String noEscape = """
            Line 1   
            Line 2   
            """;
        System.out.println("No escape:");
        System.out.println("'" + noEscape + "'");
        
        // With \s - spaces preserved
        String withEscape = """
            Line 1\s\s\s
            Line 2\s\s\s
            """;
        System.out.println("With escape:");
        System.out.println("'" + withEscape + "'");
        
        // Mixed
        String mixed = """
            Start\smiddle\send
            """;
        System.out.println("Mixed: '" + mixed + "'");
        
        /**
         * \s BEHAVIOR:
         * 
         * Trailing spaces:
         * "Line   "  → "Line" (stripped)
         * "Line\s\s\s" → "Line   " (preserved)
         * 
         * Middle spaces:
         * "Word Word" → "Word Word" (normal)
         * "Word\sWord" → "Word Word" (same)
         * 
         * Use \s when:
         * - Trailing spaces needed
         * - Formatting matters
         * - Testing (exact spaces)
         */
    }
    
    /**
     * Line continuation (\) examples
     */
    public static void lineContinuation() {
        System.out.println("\n=== Line Continuation ===");
        
        // Without continuation
        String withNewline = """
            This is a very long line
            that wraps
            """;
        System.out.println("With newline:");
        System.out.println("'" + withNewline + "'");
        
        // With continuation
        String continued = """
            This is a very long line \
            that continues
            """;
        System.out.println("Continued:");
        System.out.println("'" + continued + "'");
        
        // Multiple continuations
        String multiple = """
            First \
            Second \
            Third
            """;
        System.out.println("Multiple:");
        System.out.println("'" + multiple + "'");
        
        /**
         * LINE CONTINUATION:
         * 
         * Syntax:
         * "Line 1 \
         * Line 2"
         * 
         * Result:
         * "Line 1 Line 2"
         * (joined, no newline)
         * 
         * Use cases:
         * - Long URLs
         * - Long text in code
         * - Keep code width
         * 
         * Example:
         * String url = """
         *     https://example.com/\
         *     very/long/path/\
         *     to/resource
         *     """;
         * 
         * Result:
         * "https://example.com/very/long/path/to/resource\n"
         */
    }
    
    /**
     * Quotes in text blocks
     */
    public static void quotesInTextBlocks() {
        System.out.println("\n=== Quotes ===");
        
        // Double quotes - usually no escape needed
        String json = """
            {
              "name": "John",
              "age": 30
            }
            """;
        System.out.println(json);
        
        // Triple quotes - need escape
        String withTripleQuotes = """
            Text with \"""triple quotes\"""
            """;
        System.out.println(withTripleQuotes);
        
        // Single quotes - never need escape
        String withSingleQuotes = """
            Text with 'single quotes'
            """;
        System.out.println(withSingleQuotes);
        
        /**
         * QUOTES:
         * 
         * Double quotes ("):
         * - Usually don't need escape
         * - Can use freely
         * - Great for JSON, HTML
         * 
         * Triple quotes ("""):
         * - Must escape: \"\"\"
         * - Rare in practice
         * 
         * Single quotes ('):
         * - Never need escape
         * - Use freely
         * 
         * Example (JSON):
         * String json = """
         *   {"key": "value"}
         *   """;
         * No escapes needed!
         */
    }
    
    /**
     * Backslashes
     */
    public static void backslashes() {
        System.out.println("\n=== Backslashes ===");
        
        // Windows paths
        String path = """
            C:\\Users\\John\\Documents
            """;
        System.out.println("Path: " + path);
        
        // Regex
        String regex = """
            \\d+\\.\\d+
            """;
        System.out.println("Regex: " + regex);
        
        /**
         * BACKSLASHES:
         * 
         * Still need doubling:
         * \ → \\
         * 
         * Examples:
         * 
         * Windows path:
         * "C:\\Users\\John"
         * 
         * Regex:
         * "\\d+" (digits)
         * "\\s+" (whitespace)
         * 
         * No change from traditional strings
         */
    }
    
    /**
     * Complete escape reference
     */
    public static void escapeReference() {
        /**
         * ESCAPE SEQUENCES:
         * 
         * Traditional (Java 1.0):
         * \n - Newline
         * \t - Tab
         * \r - Carriage return
         * \f - Form feed
         * \b - Backspace
         * \" - Double quote
         * \' - Single quote
         * \\ - Backslash
         * 
         * New (Java 14):
         * \s - Space (preserved)
         * \  - Line continuation (at line end)
         * 
         * 
         * IN TEXT BLOCKS:
         * 
         * Usually don't need:
         * ✗ \n (natural line breaks)
         * ✗ \" (double quotes OK)
         * ✗ + (no concatenation)
         * 
         * Still need:
         * ✓ \\ (backslash)
         * ✓ \t (tab, if not using actual tab)
         * ✓ \s (preserve trailing space)
         * ✓ \  (line continuation)
         * 
         * 
         * EXAMPLES:
         * 
         * Traditional string:
         * "Line 1\nLine 2\n\"quoted\""
         * 
         * Text block:
         * """
         * Line 1
         * Line 2
         * "quoted"
         * """
         * 
         * Much cleaner!
         */
    }
    
    public static void main(String[] args) {
        traditionalEscapes();
        newEscapes();
        escapedSpace();
        lineContinuation();
        quotesInTextBlocks();
        backslashes();
    }
}
```

## 4. String Interpolation (Not in Java)

### Workarounds for Variable Substitution

```java
/**
 * STRING INTERPOLATION
 * 
 * Java doesn't have built-in interpolation
 * Workarounds and alternatives
 */

public class StringInterpolation {
    
    /**
     * Why Java doesn't have interpolation
     */
    public static void whyNoInterpolation() {
        /**
         * STRING INTERPOLATION:
         * 
         * Other languages have it:
         * 
         * JavaScript:
         * `Hello ${name}, you are ${age} years old`
         * 
         * Python:
         * f"Hello {name}, you are {age} years old"
         * 
         * C#:
         * $"Hello {name}, you are {age} years old"
         * 
         * Kotlin:
         * "Hello $name, you are $age years old"
         * 
         * 
         * JAVA DOESN'T HAVE IT:
         * 
         * Text blocks don't support:
         * """
         * Hello ${name}  ← Not supported!
         * """
         * 
         * Reasons:
         * - Compile-time strings preferred
         * - Security concerns (injection)
         * - Type safety
         * - Complexity
         * 
         * Proposals:
         * - String templates (possible future)
         * - Template expressions
         * - String interpolation syntax
         * 
         * Current status:
         * - Not in Java 17
         * - May come in future
         * - Use workarounds for now
         */
    }
    
    /**
     * Workaround 1: String.format()
     */
    public static void stringFormat() {
        System.out.println("=== String.format() ===");
        
        String name = "Alice";
        int age = 30;
        
        String message = String.format("""
            Hello %s,
            You are %d years old.
            Welcome!
            """, name, age);
        
        System.out.println(message);
        
        // Inline formatting
        String inline = """
            Name: %s
            Age: %d
            Email: %s@example.com
            """.formatted(name, age, name.toLowerCase());
        
        System.out.println(inline);
        
        /**
         * STRING.FORMAT():
         * 
         * Syntax:
         * String.format(template, args...)
         * 
         * Or (Java 15+):
         * template.formatted(args...)
         * 
         * Format specifiers:
         * %s - String
         * %d - Integer
         * %f - Float/Double
         * %b - Boolean
         * %n - Newline (platform-specific)
         * 
         * Example:
         * String msg = """
         *     User: %s
         *     Score: %d
         *     """.formatted(user, score);
         * 
         * Pros:
         * ✓ Built-in
         * ✓ Type-safe
         * ✓ Familiar syntax
         * 
         * Cons:
         * ✗ Positional arguments
         * ✗ Can be verbose
         * ✗ Runtime formatting
         */
    }
    
    /**
     * Workaround 2: String concatenation
     */
    public static void concatenation() {
        System.out.println("\n=== Concatenation ===");
        
        String name = "Bob";
        int age = 25;
        
        String message = """
            Hello """ + name + """
            ,
            You are """ + age + """
             years old.
            """;
        
        System.out.println(message);
        
        /**
         * CONCATENATION:
         * 
         * Can split text blocks:
         * """
         * Text before """ + variable + """
         * text after
         * """
         * 
         * Pros:
         * ✓ Simple
         * ✓ No formatting needed
         * 
         * Cons:
         * ✗ Ugly syntax
         * ✗ Breaks visual structure
         * ✗ Hard to read
         * 
         * Recommendation:
         * Avoid for complex templates
         * OK for simple cases
         */
    }
    
    /**
     * Workaround 3: String.replace()
     */
    public static void stringReplace() {
        System.out.println("\n=== String.replace() ===");
        
        String template = """
            Hello {name},
            You are {age} years old.
            Email: {email}
            """;
        
        String name = "Charlie";
        int age = 35;
        String email = "charlie@example.com";
        
        String message = template
            .replace("{name}", name)
            .replace("{age}", String.valueOf(age))
            .replace("{email}", email);
        
        System.out.println(message);
        
        /**
         * REPLACE PATTERN:
         * 
         * Template with placeholders:
         * """
         * Text with {placeholder}
         * """
         * 
         * Replace:
         * .replace("{placeholder}", value)
         * 
         * Pros:
         * ✓ Clear placeholders
         * ✓ Template reusable
         * ✓ Named parameters
         * 
         * Cons:
         * ✗ Manual chaining
         * ✗ String conversion needed
         * ✗ No type safety
         * 
         * Good for:
         * - Templates
         * - Configuration
         * - Localization
         */
    }
    
    /**
     * Workaround 4: Template libraries
     */
    public static void templateLibraries() {
        /**
         * TEMPLATE LIBRARIES:
         * 
         * External libraries:
         * 
         * 1. Apache Velocity:
         * String template = """
         *     Hello $name,
         *     You are $age years old.
         *     """;
         * VelocityContext context = new VelocityContext();
         * context.put("name", "Alice");
         * context.put("age", 30);
         * String result = Velocity.evaluate(context, template);
         * 
         * 2. Freemarker:
         * String template = """
         *     Hello ${name},
         *     You are ${age} years old.
         *     """;
         * 
         * 3. Mustache:
         * String template = """
         *     Hello {{name}},
         *     You are {{age}} years old.
         *     """;
         * 
         * 4. StringTemplate:
         * StringTemplate st = new StringTemplate("""
         *     Hello <name>,
         *     You are <age> years old.
         *     """);
         * st.add("name", "Alice");
         * st.add("age", 30);
         * 
         * Pros:
         * ✓ Powerful features
         * ✓ Conditional logic
         * ✓ Loops
         * ✓ Filters
         * 
         * Cons:
         * ✗ External dependency
         * ✗ Learning curve
         * ✗ Overhead
         * 
         * When to use:
         * - Complex templates
         * - Web applications
         * - Report generation
         * - Email templates
         */
    }
    
    /**
     * Recommended approach
     */
    public static void recommendedApproach() {
        System.out.println("\n=== Recommended Approach ===");
        
        String name = "David";
        int age = 40;
        double salary = 75000.50;
        
        // For simple cases: formatted()
        String simple = """
            Name: %s
            Age: %d
            """.formatted(name, age);
        System.out.println(simple);
        
        // For complex cases: build method
        String complex = buildEmployeeReport(name, age, salary);
        System.out.println(complex);
        
        /**
         * BEST PRACTICES:
         * 
         * Simple (1-3 variables):
         * Use .formatted()
         * 
         * String msg = """
         *     User: %s
         *     Age: %d
         *     """.formatted(name, age);
         * 
         * 
         * Medium complexity (4-10 variables):
         * Extract to method
         * 
         * String report = buildReport(data);
         * 
         * private String buildReport(Data data) {
         *   return """
         *       Name: %s
         *       Age: %d
         *       ...
         *       """.formatted(data.name(), data.age(), ...);
         * }
         * 
         * 
         * Complex (10+ variables, logic):
         * Use template library
         * 
         * StringTemplate st = loadTemplate("report.st");
         * st.add("data", data);
         * return st.render();
         */
    }
    
    private static String buildEmployeeReport(String name, int age, 
            double salary) {
        return """
            ===== EMPLOYEE REPORT =====
            Name: %s
            Age: %d
            Salary: $%,.2f
            Status: %s
            ===========================
            """.formatted(
                name, 
                age, 
                salary,
                age >= 65 ? "Retired" : "Active"
            );
    }
    
    public static void main(String[] args) {
        stringFormat();
        concatenation();
        stringReplace();
        recommendedApproach();
    }
```

## Summary

### Quick Reference

**Basic Syntax:**

```java
// Opening """ must be on own line
String text = """
    Line 1
    Line 2
    """;
// Result: "Line 1\nLine 2\n"
```

**Indentation:**

```java
// Incidental indentation stripped
String code = """
    public void method() {
        System.out.println("Hello");
    }
    """;
// Result: "public void method() {\n    System.out.println(\"Hello\");\n}\n"
```

**Escape Sequences:**

```java
String text = """
    Line 1\s\s   <- Preserved spaces
    Line 2 \
    continued
    """;
// Result: "Line 1   \nLine 2 continued\n"
```

**Formatting:**

```java
String msg = """
    Name: %s
    Age: %d
    """.formatted(name, age);
```

**Common Use Cases:**

```java
// SQL
String sql = """
    SELECT * FROM users
    WHERE status = 'active'
    ORDER BY name
    """;

// JSON
String json = """
    {
      "name": "Alice",
      "age": 30
    }
    """;

// HTML
String html = """
    <html>
      <body>
        <h1>Title</h1>
      </body>
    </html>
    """;
```

---


**Key insights:**

- **Triple quotes**: Natural multiline strings, no \n needed
- **Indentation**: Automatically strips incidental whitespace
- **Escape sequences**: \s preserves spaces, \ continues lines
- **No interpolation**: Use .formatted() instead
- **Use cases**: Perfect for SQL, JSON, HTML, XML

**The revolution:**

```
Before:
String sql = "SELECT id, name\n" +
             "FROM users\n" +
             "WHERE status = 'active'\n";

After:
String sql = """
    SELECT id, name
    FROM users
    WHERE status = 'active'
    """;

From 3 \n + 2 concatenations
To natural, readable text
```
