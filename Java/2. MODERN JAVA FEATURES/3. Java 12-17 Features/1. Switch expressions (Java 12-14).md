# Switch Expressions (Java 12-14)

## Overview

Java 12-14 transformed the traditional switch statement into a powerful expression form, introducing cleaner syntax and more capabilities. The new arrow syntax (`->`) eliminates fall-through bugs, the `yield` keyword enables returning values from complex blocks, multiple case labels reduce repetition, and exhaustiveness checking ensures all possible values are handled. Switch expressions treat switch as an expression that produces a value, not just a control flow statement. Later Java versions introduced pattern matching in switch (preview), enabling sophisticated type checks and deconstruction. This guide covers the new arrow syntax and its benefits, the yield keyword for complex logic, pattern matching capabilities, how exhaustiveness checking prevents bugs, using multiple labels efficiently, and the fundamental difference between switch statements and switch expressions.

## 1. New Switch Syntax (->)

### Arrow Syntax for Cleaner Switch Code

```java
/**
 * NEW SWITCH SYNTAX
 * 
 * Java 12 introduced arrow syntax for switch
 */

public class NewSwitchSyntax {
    
    /**
     * Traditional switch statement
     */
    public static void traditionalSwitch() {
        System.out.println("=== Traditional Switch ===");
        
        int day = 3;
        String dayName;
        
        // Old style - verbose with breaks
        switch (day) {
            case 1:
                dayName = "Monday";
                break;
            case 2:
                dayName = "Tuesday";
                break;
            case 3:
                dayName = "Wednesday";
                break;
            case 4:
                dayName = "Thursday";
                break;
            case 5:
                dayName = "Friday";
                break;
            case 6:
                dayName = "Saturday";
                break;
            case 7:
                dayName = "Sunday";
                break;
            default:
                dayName = "Invalid";
                break;
        }
        
        System.out.println("Day: " + dayName);
        
        /**
         * TRADITIONAL SWITCH PROBLEMS:
         * 
         * 1. Verbose:
         * - Need break statements
         * - Lots of boilerplate
         * - Hard to read
         * 
         * 2. Fall-through bugs:
         * - Forget break → wrong behavior
         * - Hard to detect
         * - Common source of bugs
         * 
         * 3. Not an expression:
         * - Cannot return value
         * - Must use variable
         * - Cannot use in assignments directly
         * 
         * 4. Scope issues:
         * - Variables visible across cases
         * - Must use blocks for scoping
         */
    }
    
    /**
     * New arrow syntax (Java 12+)
     */
    public static void arrowSyntax() {
        System.out.println("\n=== Arrow Syntax ===");
        
        int day = 3;
        
        // New style - clean with arrows
        String dayName = switch (day) {
            case 1 -> "Monday";
            case 2 -> "Tuesday";
            case 3 -> "Wednesday";
            case 4 -> "Thursday";
            case 5 -> "Friday";
            case 6 -> "Saturday";
            case 7 -> "Sunday";
            default -> "Invalid";
        };
        
        System.out.println("Day: " + dayName);
        
        /**
         * ARROW SYNTAX BENEFITS:
         * 
         * 1. Concise:
         * - No break needed
         * - Less boilerplate
         * - Clearer intent
         * 
         * 2. No fall-through:
         * - Each case isolated
         * - Impossible to forget break
         * - Safer code
         * 
         * 3. Expression form:
         * - Returns value
         * - Direct assignment
         * - More functional style
         * 
         * 4. Better scoping:
         * - Each case has own scope
         * - No variable leakage
         */
    }
    
    /**
     * Arrow syntax with code blocks
     */
    public static void arrowWithBlocks() {
        System.out.println("\n=== Arrow with Blocks ===");
        
        int day = 3;
        
        String message = switch (day) {
            case 1 -> {
                System.out.println("Start of work week");
                yield "Monday";
            }
            case 2, 3, 4 -> {
                System.out.println("Middle of week");
                yield "Midweek";
            }
            case 5 -> {
                System.out.println("End of work week");
                yield "Friday";
            }
            case 6, 7 -> {
                System.out.println("Weekend!");
                yield "Weekend";
            }
            default -> "Invalid";
        };
        
        System.out.println("Message: " + message);
        
        /**
         * BLOCKS WITH YIELD:
         * 
         * When case needs multiple statements:
         * case X -> {
         *   // Multiple statements
         *   yield value;  // Return value
         * }
         * 
         * yield:
         * - Returns value from block
         * - Required in expression switch
         * - Cannot be used in statement switch
         * 
         * vs return:
         * - yield: exits switch, returns value
         * - return: exits method
         */
    }
    
    /**
     * Comparison: Old vs New
     */
    public static void comparison() {
        System.out.println("\n=== Comparison ===");
        
        String fruit = "apple";
        
        // OLD STYLE
        String colorOld;
        switch (fruit) {
            case "apple":
                colorOld = "red";
                break;
            case "banana":
                colorOld = "yellow";
                break;
            case "orange":
                colorOld = "orange";
                break;
            default:
                colorOld = "unknown";
                break;
        }
        System.out.println("Old style: " + colorOld);
        
        // NEW STYLE
        String colorNew = switch (fruit) {
            case "apple" -> "red";
            case "banana" -> "yellow";
            case "orange" -> "orange";
            default -> "unknown";
        };
        System.out.println("New style: " + colorNew);
        
        /**
         * LINE COUNT:
         * Old style: 15 lines
         * New style: 6 lines
         * Reduction: 60%
         * 
         * CLARITY:
         * Old: Imperative, mutation-based
         * New: Declarative, expression-based
         * 
         * SAFETY:
         * Old: Can forget break
         * New: Impossible to fall-through
         */
    }
    
    /**
     * Arrow syntax without values (statement form)
     */
    public static void statementForm() {
        System.out.println("\n=== Statement Form ===");
        
        int day = 3;
        
        // Statement switch with arrows (no value returned)
        switch (day) {
            case 1 -> System.out.println("Monday");
            case 2 -> System.out.println("Tuesday");
            case 3 -> System.out.println("Wednesday");
            case 4 -> System.out.println("Thursday");
            case 5 -> System.out.println("Friday");
            case 6, 7 -> System.out.println("Weekend");
            default -> System.out.println("Invalid");
        }
        
        /**
         * STATEMENT SWITCH:
         * 
         * When you don't need value:
         * - Just execute code
         * - Side effects only
         * - No yield needed
         * 
         * Still benefits from arrow syntax:
         * - No fall-through
         * - Cleaner code
         * - Better scoping
         */
    }
    
    /**
     * Mixing old and new (NOT recommended)
     */
    public static void mixingStyles() {
        /**
         * DON'T MIX:
         * 
         * switch (x) {
         *   case 1:      // Old style
         *     break;
         *   case 2 ->    // New style - COMPILATION ERROR
         *     "value";
         * }
         * 
         * Must use one style consistently
         * Either all colon or all arrow
         * 
         * Recommended: Always use arrow (->)
         */
    }
    
    /**
     * Scoping differences
     */
    public static void scopingDifferences() {
        System.out.println("\n=== Scoping ===");
        
        int value = 2;
        
        // Old style - shared scope
        switch (value) {
            case 1:
                String msg1 = "One";
                System.out.println(msg1);
                break;
            case 2:
                // Can't redeclare msg1 here
                // String msg1 = "Two";  // Compilation error!
                String msg2 = "Two";
                System.out.println(msg2);
                break;
        }
        
        // New style - separate scopes
        switch (value) {
            case 1 -> {
                String msg = "One";  // Separate scope
                System.out.println(msg);
            }
            case 2 -> {
                String msg = "Two";  // Can redeclare - different scope!
                System.out.println(msg);
            }
        }
        
        /**
         * SCOPING:
         * 
         * Old style:
         * - All cases share scope
         * - Cannot redeclare variables
         * - Need blocks to isolate
         * 
         * New style:
         * - Each case has own scope
         * - Can redeclare variables
         * - Cleaner, safer
         */
    }
    
    public static void main(String[] args) {
        traditionalSwitch();
        arrowSyntax();
        arrowWithBlocks();
        comparison();
        statementForm();
        scopingDifferences();
    }
}
```

## 2. yield Keyword

### Returning Values from Switch Blocks

```java
/**
 * YIELD KEYWORD
 * 
 * Introduced in Java 13 for switch expressions
 */

public class YieldKeyword {
    
    /**
     * Basic yield usage
     */
    public static void basicYield() {
        System.out.println("=== Basic yield ===");
        
        int number = 3;
        
        String result = switch (number) {
            case 1 -> {
                System.out.println("Processing one");
                yield "One";
            }
            case 2 -> {
                System.out.println("Processing two");
                yield "Two";
            }
            case 3 -> {
                System.out.println("Processing three");
                yield "Three";
            }
            default -> {
                System.out.println("Unknown number");
                yield "Unknown";
            }
        };
        
        System.out.println("Result: " + result);
        
        /**
         * YIELD:
         * 
         * Purpose:
         * - Return value from switch block
         * - Like return for switch expressions
         * 
         * Syntax:
         * case X -> {
         *   // statements
         *   yield value;
         * }
         * 
         * Required when:
         * - Switch is expression (returns value)
         * - Case has block with multiple statements
         * 
         * Not needed when:
         * - Single expression: case X -> value
         * - Statement switch (no value returned)
         */
    }
    
    /**
     * yield vs return
     */
    public static void yieldVsReturn() {
        System.out.println("\n=== yield vs return ===");
        
        int value = 5;
        
        String result = switch (value) {
            case 1 -> "One";
            case 2 -> {
                // yield exits switch and returns value
                yield "Two";
            }
            case 3 -> {
                if (Math.random() > 0.5) {
                    yield "Three (high)";
                } else {
                    yield "Three (low)";
                }
            }
            default -> {
                // return would exit entire method!
                // return "Default";  // WRONG!
                yield "Default";  // Correct
            }
        };
        
        System.out.println("Result: " + result);
        
        /**
         * YIELD VS RETURN:
         * 
         * yield:
         * - Exits switch expression
         * - Returns value from switch
         * - Method continues
         * 
         * return:
         * - Exits method
         * - Returns from method
         * - Method ends
         * 
         * Example:
         * String getValue(int x) {
         *   String s = switch (x) {
         *     case 1 -> {
         *       yield "One";     // Exits switch
         *     }
         *     default -> {
         *       return "Exit";   // Exits method!
         *     }
         *   };
         *   return s;  // Never reached if default hit
         * }
         */
    }
    
    /**
     * Complex logic with yield
     */
    public static void complexLogic() {
        System.out.println("\n=== Complex Logic ===");
        
        String grade = "B";
        
        double gpa = switch (grade) {
            case "A" -> {
                System.out.println("Excellent!");
                yield 4.0;
            }
            case "B" -> {
                System.out.println("Good!");
                // Can have complex logic
                boolean plus = grade.contains("+");
                yield plus ? 3.3 : 3.0;
            }
            case "C" -> {
                System.out.println("Average");
                boolean plus = grade.contains("+");
                boolean minus = grade.contains("-");
                if (plus) yield 2.3;
                else if (minus) yield 1.7;
                else yield 2.0;
            }
            case "D" -> {
                System.out.println("Passing");
                yield 1.0;
            }
            case "F" -> {
                System.out.println("Failing");
                yield 0.0;
            }
            default -> {
                System.err.println("Invalid grade: " + grade);
                yield 0.0;
            }
        };
        
        System.out.println("GPA: " + gpa);
        
        /**
         * COMPLEX BLOCKS:
         * 
         * Can have:
         * ✓ Multiple statements
         * ✓ Variable declarations
         * ✓ Conditional logic
         * ✓ Loops
         * ✓ Method calls
         * ✓ Try-catch blocks
         * 
         * Must:
         * ✓ End with yield
         * ✓ All paths must yield
         * ✓ Type must match switch type
         */
    }
    
    /**
     * yield in traditional colon case
     */
    public static void yieldInColonCase() {
        System.out.println("\n=== yield in Colon Case ===");
        
        int day = 3;
        
        // Can use yield in traditional switch too!
        String type = switch (day) {
            case 1, 2, 3, 4, 5:
                System.out.println("Weekday");
                yield "Weekday";
            case 6, 7:
                System.out.println("Weekend");
                yield "Weekend";
            default:
                yield "Invalid";
        };
        
        System.out.println("Type: " + type);
        
        /**
         * YIELD IN COLON CASES:
         * 
         * Syntax:
         * case X:
         *   statements;
         *   yield value;
         * 
         * No break needed:
         * - yield replaces break
         * - Returns value and exits
         * 
         * Fall-through still possible:
         * case 1:
         * case 2:  // Falls through
         *   yield "One or Two";
         * 
         * Mixed styles still not allowed:
         * Cannot mix -> and : in same switch
         */
    }
    
    /**
     * All paths must yield
     */
    public static void allPathsMustYield() {
        /**
         * COMPILATION RULES:
         * 
         * Every path must yield:
         * 
         * ✓ VALID:
         * case X -> {
         *   if (condition) {
         *     yield "A";
         *   } else {
         *     yield "B";
         *   }
         * }
         * 
         * ✗ INVALID:
         * case X -> {
         *   if (condition) {
         *     yield "A";
         *   }
         *   // Missing yield in else path!
         * }
         * 
         * Compiler checks:
         * - All branches yield
         * - No missing returns
         * - Prevents bugs
         */
    }
    
    /**
     * yield with exceptions
     */
    public static void yieldWithExceptions() {
        System.out.println("\n=== yield with Exceptions ===");
        
        int value = 5;
        
        String result = switch (value) {
            case 1 -> "One";
            case 2 -> {
                try {
                    // Some risky operation
                    yield "Two";
                } catch (Exception e) {
                    yield "Error";
                }
            }
            case 3 -> {
                if (value < 0) {
                    throw new IllegalArgumentException("Negative");
                }
                yield "Three";
            }
            default -> "Unknown";
        };
        
        System.out.println("Result: " + result);
        
        /**
         * EXCEPTIONS:
         * 
         * Can throw from switch:
         * case X -> throw new Exception();
         * 
         * Can catch in switch:
         * case X -> {
         *   try {
         *     // risky
         *     yield "value";
         *   } catch (Exception e) {
         *     yield "error";
         *   }
         * }
         * 
         * If thrown, propagates to caller:
         * try {
         *   String s = switch (x) {
         *     case 1 -> throw new Exception();
         *     default -> "ok";
         *   };
         * } catch (Exception e) {
         *   // Caught here
         * }
         */
    }
    
    public static void main(String[] args) {
        basicYield();
        yieldVsReturn();
        complexLogic();
        yieldInColonCase();
        yieldWithExceptions();
    }
}
```

## 3. Pattern Matching in Switch (Preview)

### Type Patterns and Deconstruction (Java 17+)

```java
/**
 * PATTERN MATCHING IN SWITCH
 * 
 * Preview feature in Java 17+, finalized in Java 21
 */

public class PatternMatchingSwitch {
    
    /**
     * Traditional instanceof chain
     */
    public static void traditionalInstanceof() {
        System.out.println("=== Traditional instanceof ===");
        
        Object obj = "Hello";
        
        // Old way - verbose
        String result;
        if (obj instanceof String) {
            String s = (String) obj;
            result = "String: " + s.length();
        } else if (obj instanceof Integer) {
            Integer i = (Integer) obj;
            result = "Integer: " + i;
        } else if (obj instanceof Double) {
            Double d = (Double) obj;
            result = "Double: " + d;
        } else {
            result = "Unknown";
        }
        
        System.out.println(result);
        
        /**
         * PROBLEMS:
         * - Verbose
         * - Repetitive casting
         * - Easy to make mistakes
         * - Hard to maintain
         */
    }
    
    /**
     * Pattern matching switch (Java 17+ preview)
     */
    public static void patternSwitch() {
        System.out.println("\n=== Pattern Switch ===");
        
        Object obj = "Hello";
        
        // New way - clean
        String result = switch (obj) {
            case String s -> "String: " + s.length();
            case Integer i -> "Integer: " + i;
            case Double d -> "Double: " + d;
            case null -> "Null";  // Can match null!
            default -> "Unknown";
        };
        
        System.out.println(result);
        
        /**
         * PATTERN MATCHING:
         * 
         * Syntax:
         * case Type variable -> expression
         * 
         * Features:
         * - Type check
         * - Cast automatically
         * - Bind to variable
         * - All in one step
         * 
         * Benefits:
         * - Much cleaner
         * - Less repetition
         * - Type-safe
         * - Compiler helps
         */
    }
    
    /**
     * Null handling
     */
    public static void nullHandling() {
        System.out.println("\n=== Null Handling ===");
        
        Object obj = null;
        
        // Can match null explicitly
        String result = switch (obj) {
            case null -> "It's null!";
            case String s -> "String: " + s;
            case Integer i -> "Integer: " + i;
            default -> "Other";
        };
        
        System.out.println(result);
        
        /**
         * NULL IN SWITCH:
         * 
         * Traditional switch:
         * switch (obj) {
         *   case ...:  // NPE if obj is null!
         * }
         * 
         * Pattern switch:
         * case null -> "handle null";
         * 
         * Benefits:
         * - No NullPointerException
         * - Explicit null handling
         * - Safer code
         * 
         * If no null case:
         * - NPE thrown (like traditional)
         * - Must handle explicitly
         */
    }
    
    /**
     * Guarded patterns (when clause)
     */
    public static void guardedPatterns() {
        System.out.println("\n=== Guarded Patterns ===");
        
        Object obj = "hello";
        
        // Pattern with guard
        String result = switch (obj) {
            case String s when s.length() > 5 -> 
                "Long string: " + s;
            case String s when s.length() > 0 -> 
                "Short string: " + s;
            case String s -> 
                "Empty string";
            case Integer i when i > 0 -> 
                "Positive: " + i;
            case Integer i -> 
                "Non-positive: " + i;
            case null -> "Null";
            default -> "Other";
        };
        
        System.out.println(result);
        
        /**
         * GUARDED PATTERNS:
         * 
         * Syntax:
         * case Type var when condition -> result
         * 
         * Order matters:
         * - More specific first
         * - More general last
         * - Compiler checks ordering
         * 
         * Example:
         * case String s when s.length() > 10 -> ...  // Specific
         * case String s when s.length() > 5 -> ...   // Less specific
         * case String s -> ...                       // Most general
         * 
         * If wrong order:
         * - Compilation error
         * - "Label dominated by preceding case"
         */
    }
    
    /**
     * Sealed types and patterns
     */
    public static void sealedTypes() {
        System.out.println("\n=== Sealed Types ===");
        
        /**
         * SEALED TYPES:
         * 
         * sealed interface Shape permits Circle, Rectangle, Triangle {}
         * final class Circle implements Shape { ... }
         * final class Rectangle implements Shape { ... }
         * final class Triangle implements Shape { ... }
         * 
         * Switch with sealed types:
         * double area(Shape shape) {
         *   return switch (shape) {
         *     case Circle c -> Math.PI * c.radius * c.radius;
         *     case Rectangle r -> r.width * r.height;
         *     case Triangle t -> 0.5 * t.base * t.height;
         *     // No default needed - exhaustive!
         *   };
         * }
         * 
         * Benefits:
         * - Compiler knows all types
         * - No default needed
         * - Adding new type breaks compilation
         * - Forces update of all switches
         * - Safer refactoring
         */
    }
    
    /**
     * Record patterns (Java 19+ preview)
     */
    public static void recordPatterns() {
        /**
         * RECORD PATTERNS:
         * 
         * record Point(int x, int y) {}
         * 
         * Switch with deconstruction:
         * String describe(Object obj) {
         *   return switch (obj) {
         *     case Point(int x, int y) -> 
         *       "Point at (" + x + ", " + y + ")";
         *     case String s -> 
         *       "String: " + s;
         *     default -> 
         *       "Other";
         *   };
         * }
         * 
         * Nested patterns:
         * record Rectangle(Point topLeft, Point bottomRight) {}
         * 
         * switch (shape) {
         *   case Rectangle(Point(int x1, int y1), 
         *                  Point(int x2, int y2)) ->
         *     "Rectangle from " + x1 + "," + y1 + 
         *     " to " + x2 + "," + y2;
         * }
         * 
         * Benefits:
         * - Deconstruct records inline
         * - Access components directly
         * - No getter calls
         * - Very concise
         */
    }
    
    public static void main(String[] args) {
        traditionalInstanceof();
        patternSwitch();
        nullHandling();
        guardedPatterns();
    }
}
```

## 4. Exhaustiveness Checking

### Compiler-Enforced Complete Coverage

```java
/**
 * EXHAUSTIVENESS CHECKING
 * 
 * Compiler ensures all cases are covered
 */

public class ExhaustivenessChecking {
    
    enum Day {
        MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
    }
    
    enum Status {
        PENDING, APPROVED, REJECTED
    }
    
    /**
     * Exhaustiveness with enums
     */
    public static void enumExhaustiveness() {
        System.out.println("=== Enum Exhaustiveness ===");
        
        Status status = Status.APPROVED;
        
        // Expression switch - MUST be exhaustive
        String message = switch (status) {
            case PENDING -> "Waiting";
            case APPROVED -> "Accepted";
            case REJECTED -> "Denied";
            // No default needed - all cases covered!
        };
        
        System.out.println(message);
        
        /**
         * EXHAUSTIVENESS RULES:
         * 
         * For expression switch:
         * - Must cover all possible values
         * - Or have default case
         * 
         * For enums:
         * - Can list all enum constants
         * - No default needed if all covered
         * 
         * Compiler checks:
         * ✓ All enum values present
         * ✓ Or default case exists
         * ✗ Missing case → Compilation error
         * 
         * Benefits:
         * - Cannot forget a case
         * - Adding enum breaks compilation
         * - Forces update of switches
         * - Refactoring safety
         */
    }
    
    /**
     * Non-exhaustive example (error)
     */
    public static void nonExhaustive() {
        /**
         * COMPILATION ERROR:
         * 
         * Status status = Status.APPROVED;
         * 
         * String message = switch (status) {
         *   case PENDING -> "Waiting";
         *   case APPROVED -> "Accepted";
         *   // Missing REJECTED!
         * };
         * 
         * Error: "switch expression does not cover all possible values"
         * 
         * Must either:
         * 1. Add missing case:
         *    case REJECTED -> "Denied";
         * 
         * 2. Add default:
         *    default -> "Unknown";
         */
    }
    
    /**
     * Exhaustiveness with sealed types
     */
    public static void sealedExhaustiveness() {
        System.out.println("\n=== Sealed Exhaustiveness ===");
        
        /**
         * SEALED TYPES:
         * 
         * sealed interface Result permits Success, Failure {}
         * record Success(String value) implements Result {}
         * record Failure(String error) implements Result {}
         * 
         * Exhaustive switch:
         * String process(Result result) {
         *   return switch (result) {
         *     case Success s -> "OK: " + s.value();
         *     case Failure f -> "Error: " + f.error();
         *     // No default needed - all subtypes covered!
         *   };
         * }
         * 
         * Benefits:
         * - Compiler knows all possible types
         * - Adding new type breaks compilation
         * - Forces update everywhere
         * - Prevents forgotten cases
         * 
         * Example refactoring:
         * sealed interface Result 
         *   permits Success, Failure, Pending {}  // Added Pending
         * 
         * All switches break:
         * - Must add case Pending
         * - Or add default
         * - Compiler helps find all switches
         */
    }
    
    /**
     * Exhaustiveness with primitives
     */
    public static void primitiveExhaustiveness() {
        /**
         * PRIMITIVES:
         * 
         * Not exhaustive (infinite values):
         * int x = 5;
         * String s = switch (x) {
         *   case 1 -> "One";
         *   case 2 -> "Two";
         *   // Must have default!
         * };
         * 
         * Error if no default:
         * "switch expression does not cover all possible values"
         * 
         * Must add:
         * default -> "Other";
         * 
         * 
         * BOOLEAN (special case):
         * 
         * Can be exhaustive:
         * boolean flag = true;
         * String s = switch (flag) {
         *   case true -> "Yes";
         *   case false -> "No";
         *   // No default needed!
         * };
         * 
         * Only two values - compiler knows all covered
         */
    }
    
    /**
     * Statement switch vs expression switch
     */
    public static void statementVsExpression() {
        System.out.println("\n=== Statement vs Expression ===");
        
        Day day = Day.MONDAY;
        
        // Statement switch - NOT required to be exhaustive
        switch (day) {
            case MONDAY -> System.out.println("Monday");
            case FRIDAY -> System.out.println("Friday");
            // No default needed - other days ignored
        }
        
        // Expression switch - MUST be exhaustive
        String dayType = switch (day) {
            case SATURDAY, SUNDAY -> "Weekend";
            case MONDAY, TUESDAY, WEDNESDAY, 
                 THURSDAY, FRIDAY -> "Weekday";
            // All covered, no default needed
        };
        
        System.out.println(dayType);
        
        /**
         * RULES:
         * 
         * Statement switch:
         * - Not required to be exhaustive
         * - Missing cases just ignored
         * - No compilation error
         * 
         * Expression switch:
         * - MUST be exhaustive
         * - All cases covered
         * - Or have default
         * 
         * Reasoning:
         * - Expression must return value
         * - Cannot return if case missing
         * - Statement just executes or not
         */
    }
    
    /**
     * Default as catch-all
     */
    public static void defaultCatchAll() {
        System.out.println("\n=== Default Catch-All ===");
        
        Status status = Status.PENDING;
        
        // Using default for exhaustiveness
        String message = switch (status) {
            case PENDING -> "Waiting";
            default -> "Completed";  // Covers APPROVED and REJECTED
        };
        
        System.out.println(message);
        
        /**
         * DEFAULT USAGE:
         * 
         * Pros:
         * ✓ Easy exhaustiveness
         * ✓ Handles future values
         * ✓ Less code
         * 
         * Cons:
         * ✗ Hides missing cases
         * ✗ No refactoring safety
         * ✗ May hide bugs
         * 
         * When to use:
         * - Many values, few special cases
         * - Open-ended types (primitives)
         * - "Other" is valid category
         * 
         * When NOT to use:
         * - Small enums (list all)
         * - Sealed types (list all)
         * - Want refactoring safety
         */
    }
    
    public static void main(String[] args) {
        enumExhaustiveness();
        statementVsExpression();
        defaultCatchAll();
    }
}
```

```java
public class MultipleLabels {
    
    /**
     * Traditional fall-through

    public static void traditionalFallthrough() {

        int day = 3;
        
        // Old way - fall-through
        String type;
        switch (day) {
            case 1:
            case 2:
            case 3:
            case 4:
            case 5:
                type = "Weekday";
                break;
            case 6:
            case 7:
                type = "Weekend";

            default:
                type = "Invalid";
                break;
        }
        
        System.out.println(type);
        
        /**
         * PROBLEMS:
         * - Verbose (many case labels)
         * - Easy to forget break
         * - Hard to read
         */
    }
    
    /**
     * Multiple labels with commas
     */
    public static void multipleLabels() {
        System.out.println("\n=== Multiple Labels ===");
        
        int day = 3;
        
        // New way - comma-separated
        String type = switch (day) {
            case 1, 2, 3, 4, 5 -> "Weekday";
            case 6, 7 -> "Weekend";
            default -> "Invalid";
        };
        
        System.out.println(type);
        
        /**
         * MULTIPLE LABELS:
         * 
         * Syntax:
         * case label1, label2, label3 -> result
         * 
         * Benefits:
         * - Concise
         * - Clear grouping
         * - No fall-through needed
         * - Easy to read
         * 
         * vs fall-through:
         * 8 lines → 1 line
         */
    }
    
    /**
     * Multiple labels with different types
     */
    public static void differentTypes() {
        System.out.println("\n=== Different Types ===");
        
        String grade = "B";
        
        double gpa = switch (grade) {
            case "A", "A+" -> 4.0;
            case "A-" -> 3.7;
            case "B+", "B" -> 3.0;
            case "B-" -> 2.7;
            case "C+", "C" -> 2.0;
            case "C-" -> 1.7;
            case "D" -> 1.0;
            case "F" -> 0.0;
            default -> throw new IllegalArgumentException(
                "Invalid grade: " + grade);
        };
        
        System.out.println("GPA: " + gpa);
        
        /**
         * GROUPING STRATEGY:
         * 
         * Group by:
         * - Same result value
         * - Same logic
         * - Related cases
         * 
         * Example patterns:
         * case 1, 2, 3 -> "Low";
         * case 4, 5, 6 -> "Medium";
         * case 7, 8, 9 -> "High";
         * 
         * Or:
         * case "SUCCESS", "OK", "ACCEPTED" -> handleSuccess();
         * case "ERROR", "FAILURE", "REJECTED" -> handleError();
         */
    }
    
    /**
     * Multiple labels with blocks
     */
    public static void multipleLabelsWithBlocks() {
        System.out.println("\n=== Multiple Labels with Blocks ===");
        
        int month = 4;
        
        int days = switch (month) {
            case 1, 3, 5, 7, 8, 10, 12 -> {
                System.out.println("31-day month");
                yield 31;
            }
            case 4, 6, 9, 11 -> {
                System.out.println("30-day month");
                yield 30;
            }
            case 2 -> {
                System.out.println("February");
                boolean leapYear = false;  // Simplified
                yield leapYear ? 29 : 28;
            }
            default -> {
                System.err.println("Invalid month");
                yield 0;
            }
        };
        
        System.out.println("Days: " + days);
        
        /**
         * BLOCKS WITH MULTIPLE LABELS:
         * 
         * Syntax:
         * case 1, 2, 3 -> {
         *   // statements
         *   yield value;
         * }
         * 
         * Use when:
         * - Need complex logic
         * - Multiple statements
         * - Variables needed
         * - Logging/debugging
         */
    }
    
    /**
     * Enum constants as multiple labels
     */
    public static void enumMultipleLabels() {
        System.out.println("\n=== Enum Multiple Labels ===");
        
        enum Season { SPRING, SUMMER, FALL, WINTER }
        
        Season season = Season.SUMMER;
        
        String description = switch (season) {
            case SPRING, FALL -> "Mild weather";
            case SUMMER -> "Hot weather";
            case WINTER -> "Cold weather";
        };
        
        System.out.println(description);
        
        /**
         * ENUM GROUPING:
         * 
         * Benefits:
         * - Group related constants
         * - Reduce duplication
         * - Clear categorization
         * 
         * Example:
         * enum Status { DRAFT, PENDING, APPROVED, REJECTED, ARCHIVED }
         * 
         * switch (status) {
         *   case DRAFT, PENDING -> "In progress";
         *   case APPROVED -> "Done";
         *   case REJECTED, ARCHIVED -> "Closed";
         * }
         */
    }
    
    /**
     * Limitations
     */
    public static void limitations() {
        /**
         * LIMITATIONS:
         * 
         * Cannot mix types:
         * case 1, "one" -> ...  // ERROR - different types
         * 
         * Cannot mix patterns:
         * case String s, Integer i -> ...  // ERROR
         * 
         * Each label evaluated independently:
         * case x > 0, x < 0 -> ...  // ERROR - not labels
         * 
         * Must be constants:
         * int a = 5;
         * case 1, a -> ...  // ERROR - a not constant
         * 
         * 
         * VALID ALTERNATIVES:
         * 
         * Different cases for different types:
         * case String s -> ...
         * case Integer i -> ...
         * 
         * Guards for conditions:
         * case Integer i when i > 0 -> "Positive";
         * case Integer i when i < 0 -> "Negative";
         * case Integer i -> "Zero";
         */
    }
    
    public static void main(String[] args) {
        traditionalFallthrough();
        multipleLabels();
        differentTypes();
        multipleLabelsWithBlocks();
        enumMultipleLabels();
    }
```

## 6. Statement vs Expression

### Understanding the Fundamental Difference

```java
/**
 * STATEMENT VS EXPRESSION
 * 
 * Key difference in switch usage
 */

public class StatementVsExpression {
    
    /**
     * Switch as statement
     */
    public static void switchStatement() {
        System.out.println("=== Switch Statement ===");
        
        int day = 3;
        
        // Statement - executes code
        switch (day) {
            case 1 -> System.out.println("Monday");
            case 2 -> System.out.println("Tuesday");
            case 3 -> System.out.println("Wednesday");
            case 4 -> System.out.println("Thursday");
            case 5 -> System.out.println("Friday");
            case 6, 7 -> System.out.println("Weekend");
        }
        
        /**
         * STATEMENT CHARACTERISTICS:
         * 
         * - Executes code (side effects)
         * - Doesn't return value
         * - Not assigned to variable
         * - Can be incomplete (not exhaustive)
         * - No yield keyword
         * 
         * Use for:
         * - Side effects (printing, logging)
         * - Method calls
         * - State changes
         * - When no return value needed
         */
    }
    
    /**
     * Switch as expression
     */
    public static void switchExpression() {
        System.out.println("\n=== Switch Expression ===");
        
        int day = 3;
        
        // Expression - produces value
        String dayName = switch (day) {
            case 1 -> "Monday";
            case 2 -> "Tuesday";
            case 3 -> "Wednesday";
            case 4 -> "Thursday";
            case 5 -> "Friday";
            case 6, 7 -> "Weekend";
            default -> "Invalid";
        };
        
        System.out.println(dayName);
        
        /**
         * EXPRESSION CHARACTERISTICS:
         * 
         * - Produces value
         * - Can be assigned
         * - Must be exhaustive
         * - All cases return value
         * - Can use yield in blocks
         * 
         * Use for:
         * - Computing values
         * - Assignments
         * - Method arguments
         * - Return statements
         */
    }
    
    /**
     * Direct usage in expressions
     */
    public static void directUsage() {
        System.out.println("\n=== Direct Usage ===");
        
        int score = 85;
        
        // In method call
        System.out.println(switch (score / 10) {
            case 10, 9 -> "A";
            case 8 -> "B";
            case 7 -> "C";
            case 6 -> "D";
            default -> "F";
        });
        
        // In return statement
        String grade = getGrade(score);
        System.out.println("Grade: " + grade);
        
        // In calculation
        int bonus = 1000 * switch (score / 10) {
            case 10, 9 -> 3;
            case 8 -> 2;
            case 7 -> 1;
            default -> 0;
        };
        System.out.println("Bonus: $" + bonus);
        
        /**
         * EXPRESSION FLEXIBILITY:
         * 
         * Can use anywhere expression expected:
         * - Variable assignment
         * - Method arguments
         * - Return statements
         * - Arithmetic operations
         * - String concatenation
         * - Array initialization
         * 
         * Benefits:
         * - More concise
         * - More functional
         * - Less intermediate variables
         */
    }
    
    private static String getGrade(int score) {
        return switch (score / 10) {
            case 10, 9 -> "A";
            case 8 -> "B";
            case 7 -> "C";
            case 6 -> "D";
            default -> "F";
        };
    }
    
    /**
     * Statement with side effects
     */
    public static void statementSideEffects() {
        System.out.println("\n=== Statement Side Effects ===");
        
        enum Priority { LOW, MEDIUM, HIGH, CRITICAL }
        Priority priority = Priority.HIGH;
        
        // Statement for side effects
        switch (priority) {
            case CRITICAL -> {
                System.out.println("ALERT: Critical issue!");
                sendAlert();
                logCritical();
            }
            case HIGH -> {
                System.out.println("High priority");
                logHigh();
            }
            case MEDIUM -> {
                System.out.println("Medium priority");
            }
            case LOW -> {
                System.out.println("Low priority");
            }
        }
        
        /**
         * SIDE EFFECTS:
         * 
         * When to use statement:
         * - Logging
         * - Alerts
         * - Database updates
         * - File I/O
         * - State changes
         * 
         * Pattern:
         * switch (condition) {
         *   case X -> performAction();
         *   case Y -> performOtherAction();
         * }
         */
    }
    
    private static void sendAlert() {}
    private static void logCritical() {}
    private static void logHigh() {}
    
    /**
     * Expression for value computation
     */
    public static void expressionComputation() {
        System.out.println("\n=== Expression Computation ===");
        
        record Point(int x, int y) {}
        
        Point point = new Point(5, 10);
        
        // Expression for computation
        String quadrant = switch (point) {
            case Point p when p.x() > 0 && p.y() > 0 -> "I";
            case Point p when p.x() < 0 && p.y() > 0 -> "II";
            case Point p when p.x() < 0 && p.y() < 0 -> "III";
            case Point p when p.x() > 0 && p.y() < 0 -> "IV";
            case Point p -> "Origin or axis";
        };
        
        System.out.println("Quadrant: " + quadrant);
        
        /**
         * COMPUTATION PATTERN:
         * 
         * String result = switch (input) {
         *   case A -> compute1();
         *   case B -> compute2();
         *   default -> defaultValue();
         * };
         * 
         * Benefits:
         * - Clear data flow
         * - No mutation
         * - Functional style
         * - Easy to test
         */
    }
    
    /**
     * Combining statement and expression
     */
    public static void combining() {
        System.out.println("\n=== Combining ===");
        
        int value = 5;
        
        // Expression with side effects in blocks
        String result = switch (value) {
            case 1 -> {
                System.out.println("Processing 1");
                yield "One";
            }
            case 2 -> {
                System.out.println("Processing 2");
                yield "Two";
            }
            default -> {
                System.out.println("Processing default");
                yield "Other";
            }
        };
        
        System.out.println("Result: " + result);
        
        /**
         * BEST OF BOTH:
         * 
         * Can have side effects AND return value:
         * String s = switch (x) {
         *   case 1 -> {
         *     log("Processing 1");  // Side effect
         *     yield "One";           // Return value
         *   }
         * };
         * 
         * Pattern:
         * - Side effects in block
         * - yield for value
         * 
         * Use when:
         * - Need logging
         * - Need validation
         * - Complex computation
         * - But also need result
         */
    }
    
    /**
     * Comparison summary
     */
    public static void comparisonSummary() {
        /**
         * STATEMENT VS EXPRESSION:
         * 
         * ┌─────────────────┬───────────────┬──────────────────┐
         * │ Characteristic  │ Statement     │ Expression       │
         * ├─────────────────┼───────────────┼──────────────────┤
         * │ Returns value   │ No            │ Yes              │
         * │ Can assign      │ No            │ Yes              │
         * │ Exhaustiveness  │ Optional      │ Required         │
         * │ Uses yield      │ No            │ In blocks        │
         * │ Use case        │ Side effects  │ Computation      │
         * └─────────────────┴───────────────┴──────────────────┘
         * 
         * CHOOSING:
         * 
         * Use statement when:
         * - Only side effects needed
         * - No value to return
         * - Executing commands
         * - Not all cases matter
         * 
         * Use expression when:
         * - Need value
         * - Assignment/return
         * - Computation
         * - All cases produce value
         * 
         * MODERN STYLE:
         * 
         * Prefer expressions:
         * ✓ More functional
         * ✓ Easier to test
         * ✓ Clearer data flow
         * ✓ Less mutation
         * 
         * Use statements sparingly:
         * ○ Pure side effects
         * ○ Logging/debugging
         * ○ Legacy code integration
         */
    }
    
    public static void main(String[] args) {
        switchStatement();
        switchExpression();
        directUsage();
        statementSideEffects();
        expressionComputation();
        combining();
    }
}
```

## Summary

### Quick Reference

**Arrow Syntax:**

```java
// Old style (colon)
switch (day) {
    case 1:
        result = "Monday";
        break;
    case 2:
        result = "Tuesday";
        break;
}

// New style (arrow)
String result = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    default -> "Other";
};
```

**yield Keyword:**

```java
String result = switch (value) {
    case 1 -> "One";  // Single expression
    case 2 -> {       // Block needs yield
        System.out.println("Two");
        yield "Two";
    }
    default -> "Other";
};
```

**Multiple Labels:**

```java
String type = switch (day) {
    case 1, 2, 3, 4, 5 -> "Weekday";
    case 6, 7 -> "Weekend";
    default -> "Invalid";
};
```

**Pattern Matching (Java 17+):**

```java
String result = switch (obj) {
    case String s -> "String: " + s.length();
    case Integer i -> "Integer: " + i;
    case null -> "Null";
    default -> "Other";
};
```

**Exhaustiveness:**

```java
enum Status { PENDING, APPROVED, REJECTED }

// Must cover all cases
String msg = switch (status) {
    case PENDING -> "Waiting";
    case APPROVED -> "Done";
    case REJECTED -> "Denied";
    // No default needed - exhaustive!
};
```

---


**Key insights:**

- **Arrow syntax**: No fall-through, cleaner code, safer
- **yield**: Return values from blocks, like return for switch
- **Exhaustiveness**: Compiler ensures all cases covered
- **Pattern matching**: Type checks + cast + binding in one
- **Expression**: Can assign, must be exhaustive

**The revolution:**

```
Before Java 12:
- Verbose break statements
- Fall-through bugs common
- Not an expression
- No pattern matching

After Java 12-17:
- Clean arrow syntax
- No fall-through possible
- True expressions
- Pattern matching

From 15 lines to 5 lines
From error-prone to type-safe
```
