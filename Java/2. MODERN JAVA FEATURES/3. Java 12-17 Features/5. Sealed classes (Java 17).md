# Sealed Classes (Java 17)

## Overview

Java 17 finalized sealed classes, a feature that allows classes and interfaces to control which other classes or interfaces can extend or implement them. Using the `permits` clause, a sealed type explicitly declares its permitted subtypes, creating a closed hierarchy. This enables the compiler to verify exhaustiveness in switch expressions and pattern matching, ensuring all possible subtypes are handled. Sealed classes work with records and final classes to create precise type hierarchies, making them ideal for modeling Algebraic Data Types (ADTs) where all variants are known. Abstract sealed classes provide a template for subclasses while maintaining control over inheritance. Sealed interfaces extend these benefits to interface hierarchies. This guide covers the permits clause syntax, how to restrict inheritance effectively, achieving exhaustiveness checking in pattern matching, combining abstract and sealed modifiers, using sealed interfaces, and practical applications in domain modeling, state machines, and type-safe APIs.

## 1. permits Clause

### Explicitly Declaring Permitted Subtypes

```java
/**
 * PERMITS CLAUSE
 * 
 * Controlling which classes can extend/implement
 */

public class PermitsClause {
    
    /**
     * Basic sealed class
     */
    public static void basicSealed() {
        System.out.println("=== Basic Sealed Class ===");
        
        // Sealed class with explicit permits
        sealed class Shape permits Circle, Rectangle, Triangle {
            abstract double area();
        }
        
        // Permitted subclasses
        final class Circle extends Shape {
            private final double radius;
            Circle(double radius) { this.radius = radius; }
            double area() { return Math.PI * radius * radius; }
        }
        
        final class Rectangle extends Shape {
            private final double width, height;
            Rectangle(double width, double height) {
                this.width = width;
                this.height = height;
            }
            double area() { return width * height; }
        }
        
        final class Triangle extends Shape {
            private final double base, height;
            Triangle(double base, double height) {
                this.base = base;
                this.height = height;
            }
            double area() { return 0.5 * base * height; }
        }
        
        Shape circle = new Circle(5.0);
        System.out.println("Circle area: " + circle.area());
        
        /**
         * SEALED CLASS SYNTAX:
         * 
         * sealed class Name permits Subtype1, Subtype2, ... {
         *   // Class body
         * }
         * 
         * Requirements:
         * - Must use 'sealed' modifier
         * - Must have 'permits' clause
         * - All permitted subtypes must:
         *   * Be accessible at compile time
         *   * Extend/implement the sealed type
         *   * Be final, sealed, or non-sealed
         * 
         * Benefits:
         * ✓ Closed hierarchy
         * ✓ Compiler knows all subtypes
         * ✓ Exhaustiveness checking
         * ✓ Better pattern matching
         */
    }
    
    /**
     * Permitted subtypes must choose modifier
     */
    public static void subtypeModifiers() {
        /**
         * SUBTYPE MODIFIERS:
         * 
         * Each permitted subtype must be:
         * 
         * 1. final - Cannot be extended further
         * final class Circle extends Shape { }
         * 
         * 2. sealed - Can have own permitted subtypes
         * sealed class Polygon extends Shape 
         *     permits Triangle, Rectangle { }
         * 
         * 3. non-sealed - Opens hierarchy again
         * non-sealed class Custom extends Shape { }
         * 
         * 
         * EXAMPLES:
         * 
         * sealed class Shape permits Circle, Polygon, Custom { }
         * 
         * final class Circle extends Shape {
         *   // Cannot extend further
         * }
         * 
         * sealed class Polygon extends Shape 
         *     permits Triangle, Rectangle {
         *   // Has own sealed hierarchy
         * }
         * 
         * non-sealed class Custom extends Shape {
         *   // Anyone can extend Custom
         * }
         * 
         * // Later:
         * class MyCustom extends Custom {
         *   // OK - Custom is non-sealed
         * }
         * 
         * class MyCircle extends Circle {
         *   // ERROR - Circle is final
         * }
         */
    }
    
    /**
     * Implicit permits (same file)
     */
    public static void implicitPermits() {
        /**
         * IMPLICIT PERMITS:
         * 
         * If all permitted subtypes in same file,
         * permits clause can be omitted:
         * 
         * sealed class Shape {
         *   // permits clause inferred
         * }
         * 
         * final class Circle extends Shape { }
         * final class Rectangle extends Shape { }
         * 
         * Compiler infers:
         * sealed class Shape permits Circle, Rectangle { }
         * 
         * 
         * REQUIREMENTS:
         * - All subtypes in same source file
         * - Subtypes must be top-level or nested
         * 
         * 
         * EXPLICIT vs IMPLICIT:
         * 
         * Use explicit permits when:
         * - Subtypes in different files
         * - Want to document hierarchy
         * - Hierarchy complex
         * 
         * Use implicit permits when:
         * - All types in same file
         * - Small hierarchy
         * - Keep it simple
         */
    }
    
    /**
     * Sealed class with records
     */
    public static void sealedWithRecords() {
        System.out.println("\n=== Sealed with Records ===");
        
        sealed interface Result permits Success, Failure {}
        
        record Success(String value) implements Result {}
        record Failure(String error) implements Result {}
        
        Result result = new Success("Operation completed");
        
        // Pattern matching - exhaustive!
        String message = switch (result) {
            case Success s -> "Success: " + s.value();
            case Failure f -> "Error: " + f.error();
            // No default needed - compiler knows all types!
        };
        
        System.out.println(message);
        
        /**
         * SEALED + RECORDS:
         * 
         * Perfect combination:
         * sealed interface Type permits Sub1, Sub2 { }
         * record Sub1(...) implements Type { }
         * record Sub2(...) implements Type { }
         * 
         * Benefits:
         * ✓ Records are implicitly final
         * ✓ Concise syntax
         * ✓ Immutable data
         * ✓ Perfect for ADTs
         * 
         * Common pattern:
         * sealed interface Message permits Text, Image { }
         * record Text(String content) implements Message { }
         * record Image(byte[] data) implements Message { }
         * 
         * This is an Algebraic Data Type!
         */
    }
    
    /**
     * Multiple levels of sealing
     */
    public static void multipleLevels() {
        System.out.println("\n=== Multiple Levels ===");
        
        sealed interface Animal permits Mammal, Bird, Fish {}
        
        sealed interface Mammal extends Animal 
            permits Dog, Cat, Whale {}
        
        final class Dog implements Mammal {}
        final class Cat implements Mammal {}
        final class Whale implements Mammal {}
        
        sealed interface Bird extends Animal 
            permits Eagle, Penguin {}
        
        final class Eagle implements Bird {}
        final class Penguin implements Bird {}
        
        final class Fish implements Animal {}
        
        /**
         * HIERARCHICAL SEALING:
         * 
         * sealed interface Top permits A, B { }
         * 
         * sealed interface A extends Top permits A1, A2 { }
         * final class A1 implements A { }
         * final class A2 implements A { }
         * 
         * final class B implements Top { }
         * 
         * Tree structure:
         * Top
         * ├── A (sealed)
         * │   ├── A1 (final)
         * │   └── A2 (final)
         * └── B (final)
         * 
         * Benefits:
         * - Organized hierarchy
         * - Gradual refinement
         * - Clear categories
         */
    }
    
    public static void main(String[] args) {
        basicSealed();
        sealedWithRecords();
        multipleLevels();
    }
}
```

## 2. Restricting Inheritance

### Controlling Class Hierarchies

```java
/**
 * RESTRICTING INHERITANCE
 * 
 * How sealed classes control who can extend
 */

public class RestrictingInheritance {
    
    /**
     * Why restrict inheritance
     */
    public static void whyRestrict() {
        /**
         * WHY RESTRICT INHERITANCE?
         * 
         * Problems with unrestricted inheritance:
         * 
         * 1. Unknown subtypes:
         * public class Shape { }
         * // Anyone can extend Shape!
         * // Cannot know all possible types
         * 
         * 2. No exhaustiveness:
         * Cannot verify all cases handled
         * Must have default in switch
         * 
         * 3. Breaking changes:
         * Adding method to Shape
         * May break unknown subclasses
         * 
         * 4. Security concerns:
         * Sensitive classes extended
         * Behavior modified
         * 
         * 
         * SEALED CLASSES SOLVE THIS:
         * 
         * sealed class Shape permits Circle, Rectangle {
         *   // Only Circle and Rectangle can extend
         *   // Compiler knows all subtypes
         *   // Can verify exhaustiveness
         *   // Safe to evolve Shape
         * }
         * 
         * Benefits:
         * ✓ Predictable hierarchy
         * ✓ Type safety
         * ✓ Better optimization
         * ✓ Clear API boundaries
         */
    }
    
    /**
     * Access control with sealed
     */
    public static void accessControl() {
        System.out.println("=== Access Control ===");
        
        // Package-private sealed class
        sealed class InternalType permits Implementation1, Implementation2 {
            abstract void process();
        }
        
        final class Implementation1 extends InternalType {
            void process() { System.out.println("Implementation 1"); }
        }
        
        final class Implementation2 extends InternalType {
            void process() { System.out.println("Implementation 2"); }
        }
        
        /**
         * ACCESS CONTROL:
         * 
         * Sealed class visibility:
         * - public: Visible outside package
         * - package-private: Only in package
         * - private: Only in enclosing class
         * 
         * Permitted subtypes:
         * Must be accessible where sealed type is
         * 
         * Example:
         * public sealed class Public permits A, B { }
         * public final class A extends Public { }
         * public final class B extends Public { }
         * 
         * Package design:
         * sealed class Internal permits Impl1, Impl2 { }
         * // Outside package cannot extend
         * // Internal implementation hidden
         */
    }
    
    /**
     * Preventing extension with final
     */
    public static void finalVsSealed() {
        /**
         * FINAL VS SEALED:
         * 
         * final class:
         * final class NoExtension { }
         * // Cannot extend at all
         * // Closed hierarchy
         * 
         * sealed class:
         * sealed class Limited permits A, B { }
         * // Can extend, but only A and B
         * // Controlled hierarchy
         * 
         * 
         * WHEN TO USE:
         * 
         * Use final when:
         * - No extension ever needed
         * - Implementation complete
         * - Security sensitive
         * - Utility classes
         * 
         * Example:
         * final class Math { }
         * final class String { }
         * 
         * 
         * Use sealed when:
         * - Limited set of subtypes
         * - Domain modeling
         * - State machines
         * - Type hierarchies
         * 
         * Example:
         * sealed interface Result permits Success, Failure { }
         * sealed class Vehicle permits Car, Truck, Motorcycle { }
         * 
         * 
         * Use neither (open) when:
         * - Extension encouraged
         * - Plugin architecture
         * - Framework extension points
         * 
         * Example:
         * class Plugin { }  // Anyone can extend
         */
    }
    
    /**
     * non-sealed to reopen hierarchy
     */
    public static void nonSealed() {
        System.out.println("\n=== non-sealed ===");
        
        sealed class Base permits Middle, End {}
        
        // Middle is non-sealed - opens hierarchy
        non-sealed class Middle extends Base {
            // Anyone can extend Middle
        }
        
        // End is final - closes hierarchy
        final class End extends Base {
            // Cannot extend End
        }
        
        // This is OK - Middle is non-sealed
        class MyMiddle extends Middle {}
        
        /**
         * NON-SEALED:
         * 
         * Purpose:
         * Break out of sealed hierarchy
         * Allow open extension from that point
         * 
         * Use cases:
         * 
         * 1. Extensibility point:
         * sealed class Framework permits Core, Extension { }
         * final class Core extends Framework { }
         * non-sealed class Extension extends Framework {
         *   // Users can extend Extension
         * }
         * 
         * 2. Default implementation:
         * sealed interface Service permits DefaultImpl, Custom { }
         * non-sealed class DefaultImpl implements Service {
         *   // Users can override default
         * }
         * final class Custom implements Service { }
         * 
         * 3. Migration path:
         * sealed class Legacy permits Old, New { }
         * final class Old extends Legacy { }
         * non-sealed class New extends Legacy {
         *   // Future extension allowed
         * }
         */
    }
    
    /**
     * Sealed classes in libraries
     */
    public static void libraryDesign() {
        /**
         * LIBRARY DESIGN:
         * 
         * API design with sealed classes:
         * 
         * 1. Stable API:
         * public sealed interface Status 
         *     permits Success, Failure, Pending { }
         * public record Success(String msg) implements Status { }
         * public record Failure(String error) implements Status { }
         * public record Pending() implements Status { }
         * 
         * Guarantees:
         * - Users know all possible statuses
         * - Can exhaustively handle all cases
         * - Library can add methods to Status safely
         * 
         * 
         * 2. Internal implementation:
         * public sealed interface Database 
         *     permits PostgresDB, MySQLDB { }
         * 
         * final class PostgresDB implements Database { }
         * final class MySQLDB implements Database { }
         * 
         * Users:
         * - Get Database reference
         * - Cannot create own implementations
         * - Library controls all implementations
         * 
         * 
         * 3. Version compatibility:
         * sealed interface Event permits V1Event, V2Event { }
         * record V1Event(String data) implements Event { }
         * record V2Event(String data, Map<String, String> meta) 
         *     implements Event { }
         * 
         * Adding V3Event:
         * - Update permits clause
         * - Users must handle new case
         * - Compile-time safety
         */
    }
    
    public static void main(String[] args) {
        accessControl();
        nonSealed();
    }
}
```

## 3. Pattern Matching Exhaustiveness

### Compiler-Verified Complete Coverage

```java
/**
 * PATTERN MATCHING EXHAUSTIVENESS
 * 
 * Sealed classes enable exhaustiveness checking
 */

public class PatternMatchingExhaustiveness {
    
    /**
     * Exhaustive switch with sealed types
     */
    public static void exhaustiveSwitch() {
        System.out.println("=== Exhaustive Switch ===");
        
        sealed interface Result permits Success, Failure {}
        record Success(String value) implements Result {}
        record Failure(String error) implements Result {}
        
        Result result = new Success("Data loaded");
        
        // No default needed - exhaustive!
        String message = switch (result) {
            case Success s -> "Success: " + s.value();
            case Failure f -> "Error: " + f.error();
        };
        
        System.out.println(message);
        
        /**
         * EXHAUSTIVENESS:
         * 
         * Compiler knows all possible types:
         * sealed interface Type permits A, B, C { }
         * 
         * Switch must handle all:
         * switch (type) {
         *   case A a -> ...
         *   case B b -> ...
         *   case C c -> ...
         *   // No default needed!
         * }
         * 
         * If you forget one:
         * switch (type) {
         *   case A a -> ...
         *   case B b -> ...
         *   // Missing C - COMPILATION ERROR!
         * }
         * 
         * Error: "switch expression does not cover all 
         *         possible input values"
         * 
         * Benefits:
         * ✓ Cannot forget a case
         * ✓ Adding type breaks compilation
         * ✓ Forces update everywhere
         * ✓ Type-safe
         */
    }
    
    /**
     * Evolution safety
     */
    public static void evolutionSafety() {
        /**
         * EVOLUTION SAFETY:
         * 
         * Initial version:
         * sealed interface Status permits Success, Failure { }
         * record Success(String msg) implements Status { }
         * record Failure(String error) implements Status { }
         * 
         * Code using it:
         * switch (status) {
         *   case Success s -> handle(s);
         *   case Failure f -> handle(f);
         * }
         * 
         * Add new type:
         * sealed interface Status 
         *     permits Success, Failure, Pending { }
         * record Pending() implements Status { }
         * 
         * All switches break!
         * switch (status) {
         *   case Success s -> handle(s);
         *   case Failure f -> handle(f);
         *   // ERROR: Missing case Pending!
         * }
         * 
         * Must add:
         * case Pending p -> handle(p);
         * 
         * Compiler finds ALL switches
         * Forces update
         * No runtime surprises
         * 
         * This is SAFE evolution:
         * - Break at compile time
         * - Not at runtime
         * - Catch all usages
         */
    }
    
    /**
     * Hierarchical exhaustiveness
     */
    public static void hierarchicalExhaustiveness() {
        System.out.println("\n=== Hierarchical Exhaustiveness ===");
        
        sealed interface Shape permits Circle, Polygon {}
        record Circle(double radius) implements Shape {}
        
        sealed interface Polygon extends Shape 
            permits Triangle, Rectangle {}
        record Triangle(double base, double height) implements Polygon {}
        record Rectangle(double width, double height) implements Polygon {}
        
        Shape shape = new Circle(5.0);
        
        // Must handle all leaf types
        double area = switch (shape) {
            case Circle c -> Math.PI * c.radius() * c.radius();
            case Triangle t -> 0.5 * t.base() * t.height();
            case Rectangle r -> r.width() * r.height();
        };
        
        System.out.println("Area: " + area);
        
        /**
         * HIERARCHICAL CASES:
         * 
         * Can match at any level:
         * 
         * Option 1: Match leaf types
         * switch (shape) {
         *   case Circle c -> ...
         *   case Triangle t -> ...
         *   case Rectangle r -> ...
         * }
         * 
         * Option 2: Match intermediate types
         * switch (shape) {
         *   case Circle c -> ...
         *   case Polygon p -> ...  // Covers Triangle & Rectangle
         * }
         * 
         * Option 3: Mix levels
         * switch (shape) {
         *   case Circle c -> ...
         *   case Triangle t -> ...
         *   case Rectangle r -> ...
         * }
         * 
         * Compiler verifies:
         * All possible types covered
         * At any level of hierarchy
         */
    }
    
    /**
     * Guarded patterns
     */
    public static void guardedPatterns() {
        System.out.println("\n=== Guarded Patterns ===");
        
        sealed interface Number permits SmallNumber, LargeNumber {}
        record SmallNumber(int value) implements Number {}
        record LargeNumber(long value) implements Number {}
        
        Number num = new SmallNumber(42);
        
        // With guards
        String description = switch (num) {
            case SmallNumber n when n.value() > 100 -> 
                "Large small number";
            case SmallNumber n -> 
                "Regular small number";
            case LargeNumber n -> 
                "Large number";
        };
        
        System.out.println(description);
        
        /**
         * GUARDS WITH SEALED TYPES:
         * 
         * Can add conditions:
         * switch (num) {
         *   case SmallNumber n when n.value() < 0 -> "Negative";
         *   case SmallNumber n -> "Positive or zero";
         *   case LargeNumber n -> "Large";
         * }
         * 
         * Order matters:
         * More specific cases first
         * Guards checked in order
         * 
         * Exhaustiveness:
         * Must still cover all types
         * Guards don't affect exhaustiveness
         * Last case for each type should have no guard
         */
    }
    
    /**
     * Default clause optional
     */
    public static void defaultClause() {
        /**
         * DEFAULT CLAUSE:
         * 
         * With sealed types:
         * switch (sealedType) {
         *   case Type1 t1 -> ...
         *   case Type2 t2 -> ...
         *   // No default needed if exhaustive
         * }
         * 
         * Can still add default:
         * switch (sealedType) {
         *   case Type1 t1 -> ...
         *   case Type2 t2 -> ...
         *   default -> throw new AssertionError();
         * }
         * 
         * But if exhaustive, default never reached:
         * switch (sealedType) {
         *   case Type1 t1 -> ...
         *   case Type2 t2 -> ...
         *   default -> throw new AssertionError("Impossible");
         * }
         * 
         * 
         * WHEN TO USE DEFAULT:
         * 
         * Don't use default:
         * - With sealed types
         * - Want compile-time checking
         * - Want evolution safety
         * 
         * Use default:
         * - Open hierarchies
         * - Unknown subtypes
         * - Fallback behavior
         * 
         * 
         * ANTI-PATTERN:
         * 
         * sealed interface Status permits Success, Failure { }
         * 
         * // BAD - loses exhaustiveness checking
         * switch (status) {
         *   case Success s -> ...
         *   default -> ...  // Hides Failure!
         * }
         * 
         * // GOOD - explicit and exhaustive
         * switch (status) {
         *   case Success s -> ...
         *   case Failure f -> ...
         * }
         */
    }
    
    public static void main(String[] args) {
        exhaustiveSwitch();
        hierarchicalExhaustiveness();
        guardedPatterns();
    }
}
```

## 4. Abstract Sealed Classes

### Combining abstract and sealed

```java
/**
 * ABSTRACT SEALED CLASSES
 * 
 * Using abstract with sealed modifier
 */

public class AbstractSealedClasses {
    
    /**
     * Basic abstract sealed class
     */
    public static void basicAbstractSealed() {
        System.out.println("=== Abstract Sealed Class ===");
        
        abstract sealed class Expression 
            permits Constant, Addition, Multiplication {
            abstract int eval();
        }
        
        final class Constant extends Expression {
            private final int value;
            Constant(int value) { this.value = value; }
            int eval() { return value; }
        }
        
        final class Addition extends Expression {
            private final Expression left, right;
            Addition(Expression left, Expression right) {
                this.left = left;
                this.right = right;
            }
            int eval() { return left.eval() + right.eval(); }
        }
        
        final class Multiplication extends Expression {
            private final Expression left, right;
            Multiplication(Expression left, Expression right) {
                this.left = left;
                this.right = right;
            }
            int eval() { return left.eval() * right.eval(); }
        }
        
        // (2 + 3) * 4
        Expression expr = new Multiplication(
            new Addition(new Constant(2), new Constant(3)),
            new Constant(4)
        );
        
        System.out.println("Result: " + expr.eval());
        
        /**
         * ABSTRACT SEALED:
         * 
         * Combines benefits:
         * - abstract: Partial implementation
         * - sealed: Controlled hierarchy
         * 
         * Use for:
         * - Template pattern
         * - Common behavior
         * - Abstract operations
         * - Type hierarchies
         * 
         * Pattern:
         * abstract sealed class Base permits Sub1, Sub2 {
         *   // Common methods
         *   void common() { }
         *   
         *   // Abstract methods
         *   abstract void specific();
         * }
         */
    }
    
    /**
     * Template method pattern
     */
    public static void templateMethod() {
        System.out.println("\n=== Template Method ===");
        
        abstract sealed class DataProcessor 
            permits JsonProcessor, XmlProcessor {
            
            // Template method
            final void process(String input) {
                validate(input);
                Object data = parse(input);
                transform(data);
                output(data);
            }
            
            // Common validation
            void validate(String input) {
                if (input == null || input.isBlank()) {
                    throw new IllegalArgumentException("Invalid input");
                }
            }
            
            // Abstract methods
            abstract Object parse(String input);
            abstract void transform(Object data);
            abstract void output(Object data);
        }
        
        final class JsonProcessor extends DataProcessor {
            Object parse(String input) {
                System.out.println("Parsing JSON");
                return input;  // Simplified
            }
            
            void transform(Object data) {
                System.out.println("Transforming JSON");
            }
            
            void output(Object data) {
                System.out.println("Outputting JSON");
            }
        }
        
        final class XmlProcessor extends DataProcessor {
            Object parse(String input) {
                System.out.println("Parsing XML");
                return input;  // Simplified
            }
            
            void transform(Object data) {
                System.out.println("Transforming XML");
            }
            
            void output(Object data) {
                System.out.println("Outputting XML");
            }
        }
        
        DataProcessor processor = new JsonProcessor();
        processor.process("{ \"data\": \"value\" }");
        
        /**
         * TEMPLATE METHOD:
         * 
         * abstract sealed class defines algorithm:
         * - Final method with steps
         * - Abstract methods for customization
         * - Common behavior shared
         * - Sealed ensures known implementations
         * 
         * Benefits:
         * ✓ Code reuse
         * ✓ Controlled variations
         * ✓ Exhaustive handling
         */
    }
    
    /**
     * Common state and behavior
     */
    public static void commonState() {
        System.out.println("\n=== Common State ===");
        
        abstract sealed class Vehicle permits Car, Truck {
            private final String model;
            private int fuel;
            
            Vehicle(String model, int fuel) {
                this.model = model;
                this.fuel = fuel;
            }
            
            // Common methods
            void refuel(int amount) {
                fuel += amount;
                System.out.println("Refueled " + amount + " units");
            }
            
            int getFuel() {
                return fuel;
            }
            
            String getModel() {
                return model;
            }
            
            // Abstract methods
            abstract int getCapacity();
            abstract void drive();
        }
        
        final class Car extends Vehicle {
            Car(String model) {
                super(model, 50);
            }
            
            int getCapacity() { return 5; }
            
            void drive() {
                System.out.println("Driving car: " + getModel());
            }
        }
        
        final class Truck extends Vehicle {
            Truck(String model) {
                super(model, 100);
            }
            
            int getCapacity() { return 2; }
            
            void drive() {
                System.out.println("Driving truck: " + getModel());
            }
        }
        
        Vehicle car = new Car("Sedan");
        car.refuel(20);
        car.drive();
        
        /**
         * COMMON STATE:
         * 
         * abstract sealed class provides:
         * - Shared fields
         * - Common methods
         * - Abstract operations
         * - Controlled hierarchy
         * 
         * Subclasses:
         * - Inherit state
         * - Reuse behavior
         * - Implement specifics

    }
```

```java
                System.out.println("Status: Running");

        }
        
        Command cmd = new StartCommand();

         * 
         * sealed interface Name permits Type1, Type2 {
         *   // Interface methods
         * }
         * 
         * Implementers must be:
         * - final class
         * - sealed class/interface
         * - non-sealed class
         * - record (implicitly final)
         * 
         * Benefits same as sealed classes:
         * ✓ Known implementations
         * ✓ Exhaustiveness checking
         * ✓ Type-safe

    /**
     * Multiple sealed interfaces
     */
    public static void multipleInterfaces() {
        System.out.println("\n=== Multiple Interfaces ===");

            void draw();
        }

            void resize(double factor);
        }
        
        record Circle(double radius) 
            implements Drawable, Resizable {
            public void draw() {

            }
            public void resize(double factor) {
                System.out.println("Resizing to: " + (radius * factor));
            }
        }
        
        record Rectangle(double width, double height) 
            implements Drawable, Resizable {
            public void draw() {
                System.out.println("Drawing rectangle: " + width + "x" + height);
            }
            public void resize(double factor) {
                System.out.println("Resizing to: " + 
                    (width * factor) + "x" + (height * factor));
            }
        }
        
        Drawable d = new Circle(5.0);
        d.draw();
        
        /**
         * MULTIPLE SEALED INTERFACES:
         * 
         * Type can implement multiple sealed interfaces:
         * sealed interface A permits X, Y { }
         * sealed interface B permits X, Y { }
         * 
         * record X() implements A, B { }
         * record Y() implements A, B { }
         * 
         * All sealed interfaces must permit the type
         * Enables composable abstractions

    }

     * Sealed interface hierarchy
     */
    public static void interfaceHierarchy() {
        System.out.println("\n=== Interface Hierarchy ===");

            permits ClickEvent, KeyEvent {}
        
        sealed interface SystemEvent extends Event 
            permits StartEvent, StopEvent {}

        record KeyEvent(char key) implements UserEvent {}
        record StartEvent() implements SystemEvent {}
        record StopEvent() implements SystemEvent {}
        
        Event event = new ClickEvent(100, 200);

            case ClickEvent e -> "Click at " + e.x() + "," + e.y();
            case KeyEvent e -> "Key: " + e.key();
            case StartEvent e -> "System start";
            case StopEvent e -> "System stop";
        };
        
        System.out.println(description);
        
        /**
         * HIERARCHICAL INTERFACES:
         * 
         * sealed interface Top permits Mid, Leaf { }
         * sealed interface Mid extends Top permits A, B { }
         * record A() implements Mid { }
         * record B() implements Mid { }
         * record Leaf() implements Top { }
         * 
         * Creates tree of interfaces
         * Each level sealed

    }
```

```java
import java.util.*;

public class UseCases {
    
    /**
     * Algebraic Data Types (ADTs)
     */
    public static void algebraicDataTypes() {
        System.out.println("=== Algebraic Data Types ===");
        
        // Option type (like Optional but sealed)
        sealed interface Option<T> permits Some, None {}
        record Some<T>(T value) implements Option<T> {}
        record None<T>() implements Option<T> {}
        
        Option<String> found = new Some<>("Hello");
        Option<String> notFound = new None<>();
        
        String result1 = switch (found) {
            case Some<String> s -> s.value();
            case None<String> n -> "Not found";
        };
        
        String result2 = switch (notFound) {
            case Some<String> s -> s.value();
            case None<String> n -> "Not found";
        };
        
        System.out.println("Found: " + result1);
        System.out.println("Not found: " + result2);
        
        // Result type (Either/Result monad)
        sealed interface Result<T, E> permits Ok, Err {}
        record Ok<T, E>(T value) implements Result<T, E> {}
        record Err<T, E>(E error) implements Result<T, E> {}
        
        Result<Integer, String> success = new Ok<>(42);
        Result<Integer, String> failure = new Err<>("Error occurred");
        
        String msg = switch (success) {
            case Ok<Integer, String> ok -> "Value: " + ok.value();
            case Err<Integer, String> err -> "Error: " + err.error();
        };
        
        System.out.println(msg);
        
        /**
         * ADTS WITH SEALED CLASSES:
         * 
         * Sum types (OR):
         * sealed interface Option permits Some, None { }
         * // Value is Some OR None
         * 
         * Product types (AND):
         * record Some(value) { }
         * // Has value AND type
         * 
         * Combination:
         * sealed interface Result permits Ok, Err { }
         * record Ok(value) { }
         * record Err(error) { }
         * 
         * Benefits:
         * ✓ Type-safe null handling
         * ✓ Error handling
         * ✓ Pattern matching
         * ✓ Exhaustiveness
         */
    }
    
    /**
     * State machines
     */
    public static void stateMachines() {
        System.out.println("\n=== State Machines ===");
        
        sealed interface OrderState 
            permits Pending, Confirmed, Shipped, Delivered, Cancelled {}
        
        record Pending(String orderId) implements OrderState {}
        record Confirmed(String orderId, String paymentId) 
            implements OrderState {}
        record Shipped(String orderId, String trackingNumber) 
            implements OrderState {}
        record Delivered(String orderId, java.time.Instant deliveredAt) 
            implements OrderState {}
        record Cancelled(String orderId, String reason) 
            implements OrderState {}
        
        // State transitions
        class OrderStateMachine {
            OrderState confirm(Pending pending, String paymentId) {
                return new Confirmed(pending.orderId(), paymentId);
            }
            
            OrderState ship(Confirmed confirmed, String trackingNumber) {
                return new Shipped(confirmed.orderId(), trackingNumber);
            }
            
            OrderState deliver(Shipped shipped) {
                return new Delivered(
                    shipped.orderId(), 
                    java.time.Instant.now()
                );
            }
            
            OrderState cancel(OrderState state, String reason) {
                String orderId = switch (state) {
                    case Pending p -> p.orderId();
                    case Confirmed c -> c.orderId();
                    case Shipped s -> s.orderId();
                    case Delivered d -> throw new IllegalStateException(
                        "Cannot cancel delivered order");
                    case Cancelled c -> throw new IllegalStateException(
                        "Already cancelled");
                };
                return new Cancelled(orderId, reason);
            }
        }
        
        OrderStateMachine machine = new OrderStateMachine();
        OrderState state = new Pending("ORDER-123");
        state = machine.confirm((Pending) state, "PAY-456");
        state = machine.ship((Confirmed) state, "TRACK-789");
        
        String status = switch (state) {
            case Pending p -> "Waiting for payment";
            case Confirmed c -> "Payment confirmed: " + c.paymentId();
            case Shipped s -> "In transit: " + s.trackingNumber();
            case Delivered d -> "Delivered at " + d.deliveredAt();
            case Cancelled c -> "Cancelled: " + c.reason();
        };
        
        System.out.println("Status: " + status);
        
        /**
         * STATE MACHINE BENEFITS:
         * 
         * ✓ All states known at compile time
         * ✓ Type-safe transitions
         * ✓ Impossible states impossible
         * ✓ Exhaustive handling
         * 
         * Example states:
         * - Order states
         * - Connection states
         * - UI states
         * - Protocol states
         */
    }
    
    /**
     * Expression trees (AST)
     */
    public static void expressionTrees() {
        System.out.println("\n=== Expression Trees ===");
        
        sealed interface Expr permits Literal, Add, Multiply, Variable {}
        record Literal(int value) implements Expr {}
        record Add(Expr left, Expr right) implements Expr {}
        record Multiply(Expr left, Expr right) implements Expr {}
        record Variable(String name) implements Expr {}
        
        // Evaluator
        class Evaluator {
            int eval(Expr expr, Map<String, Integer> vars) {
                return switch (expr) {
                    case Literal l -> l.value();
                    case Add a -> eval(a.left(), vars) + eval(a.right(), vars);
                    case Multiply m -> eval(m.left(), vars) * eval(m.right(), vars);
                    case Variable v -> vars.get(v.name());
                };
            }
        }
        
        // (x + 2) * 3
        Expr expr = new Multiply(
            new Add(new Variable("x"), new Literal(2)),
            new Literal(3)
        );
        
        Evaluator evaluator = new Evaluator();
        int result = evaluator.eval(expr, Map.of("x", 5));
        System.out.println("Result: " + result);  // (5 + 2) * 3 = 21

         * EXPRESSION TREES:
         * 
         * Use for:

         * - Formula evaluators
```

### Quick Reference

**Basic Sealed Class:**

```java
sealed class Shape permits Circle, Rectangle, Triangle {}

final class Circle extends Shape {}
final class Rectangle extends Shape {}
final class Triangle extends Shape {}
```

**Sealed Interface with Records:**

```java
sealed interface Result permits Success, Failure {}
record Success(String value) implements Result {}
record Failure(String error) implements Result {}
```

**Exhaustive Pattern Matching:**

```java
String msg = switch (result) {
    case Success s -> "OK: " + s.value();
    case Failure f -> "Error: " + f.error();
    // No default needed - exhaustive!
};
```

**Hierarchical Sealing:**

```java
sealed interface Animal permits Mammal, Bird {}
sealed interface Mammal extends Animal permits Dog, Cat {}
final class Dog implements Mammal {}
final class Cat implements Mammal {}
final class Bird implements Animal {}
```

**Abstract Sealed:**

```java
abstract sealed class Base permits Sub1, Sub2 {
    abstract void process();
}

final class Sub1 extends Base {
    void process() { /* ... */ }
}
```

---

**Key insights:**

- **Closed hierarchies**: Only permitted types can extend
- **Exhaustiveness**: Compiler verifies all cases handled
- **Type safety**: No unknown subtypes at runtime
- **Evolution safety**: Adding type breaks all switches
- **Perfect with records**: sealed + records = ADTs

**The revolution:**

```
Before Sealed Classes:
public class Shape {}
// Anyone can extend Shape
// Cannot verify exhaustiveness
// Unknown subtypes possible

After Sealed Classes:
sealed class Shape permits Circle, Rectangle {}
// Only Circle and Rectangle can extend
// Compiler knows all types
// Exhaustive pattern matching

From open to controlled
From runtime to compile-time
```
