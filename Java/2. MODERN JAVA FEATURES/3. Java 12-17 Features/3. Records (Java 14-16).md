# Records (Java 14-16)

## Overview

Java 14 introduced records as a preview feature to eliminate boilerplate when creating simple data carrier classes. Records provide a compact syntax for declaring classes that are transparent holders for shallowly immutable data. The compiler automatically generates the canonical constructor, component accessor methods, `equals()`, `hashCode()`, and `toString()` based on the record's components. Records are implicitly final and cannot be extended, ensuring their immutability contract. Java 15 refined the feature as a second preview, and Java 16 finalized it. This guide covers the compact record syntax and what it replaces, how the canonical constructor works, component accessor methods, the automatically generated methods, validation techniques in compact constructors, when to use records versus traditional classes, and how records can implement interfaces.

## 1. Compact Syntax for Data Carriers

### Eliminating Boilerplate for Simple Data Classes

```java
/**
 * COMPACT SYNTAX
 * 
 * Records provide concise data class syntax
 */

public class CompactSyntax {
    
    /**
     * Traditional class - verbose
     */
    public static final class PersonOld {
        private final String name;
        private final int age;
        private final String email;
        
        public PersonOld(String name, int age, String email) {
            this.name = name;
            this.age = age;
            this.email = email;
        }
        
        public String getName() {
            return name;
        }
        
        public int getAge() {
            return age;
        }
        
        public String getEmail() {
            return email;
        }
        
        @Override
        public boolean equals(Object obj) {
            if (this == obj) return true;
            if (obj == null || getClass() != obj.getClass()) return false;
            PersonOld person = (PersonOld) obj;
            return age == person.age &&
                   name.equals(person.name) &&
                   email.equals(person.email);
        }
        
        @Override
        public int hashCode() {
            int result = name.hashCode();
            result = 31 * result + age;
            result = 31 * result + email.hashCode();
            return result;
        }
        
        @Override
        public String toString() {
            return "PersonOld[" +
                   "name='" + name + '\'' +
                   ", age=" + age +
                   ", email='" + email + '\'' +
                   ']';
        }
    }
    
    /**
     * Record - concise
     */
    public record PersonNew(String name, int age, String email) {}
    
    /**
     * Demonstration
     */
    public static void demonstration() {
        System.out.println("=== Records vs Classes ===");
        
        // Both work the same
        PersonOld oldPerson = new PersonOld("Alice", 30, "alice@example.com");
        PersonNew newPerson = new PersonNew("Alice", 30, "alice@example.com");
        
        System.out.println("Old: " + oldPerson);
        System.out.println("New: " + newPerson);
        
        // Same functionality
        System.out.println("Name: " + newPerson.name());
        System.out.println("Age: " + newPerson.age());
        System.out.println("Email: " + newPerson.email());
        
        /**
         * COMPARISON:
         * 
         * Traditional class:
         * - 50+ lines of code
         * - Boilerplate constructor
         * - Boilerplate getters
         * - Manual equals()
         * - Manual hashCode()
         * - Manual toString()
         * - Easy to make mistakes
         * 
         * Record:
         * - 1 line of code
         * - Everything generated
         * - No boilerplate
         * - Correct by default
         * - Impossible to mess up
         * 
         * Code reduction: 98%!
         */
    }
    
    /**
     * What records are
     */
    public static void whatRecordsAre() {
        /**
         * RECORDS:
         * 
         * Definition:
         * "Transparent carriers for shallowly immutable data"
         * 
         * Characteristics:
         * - Immutable (final fields)
         * - Transparent (public accessors)
         * - Data carriers (no behavior focus)
         * - Value-based (equals by value)
         * 
         * Automatically generated:
         * ✓ Constructor (canonical)
         * ✓ Accessor methods (name(), age(), etc.)
         * ✓ equals() method
         * ✓ hashCode() method
         * ✓ toString() method
         * 
         * Restrictions:
         * - Cannot extend classes
         * - Implicitly final
         * - All fields are final
         * - Cannot declare instance fields
         * 
         * Can:
         * ✓ Implement interfaces
         * ✓ Have static fields/methods
         * ✓ Have instance methods
         * ✓ Be generic
         * ✓ Be nested
         * ✓ Have annotations
         */
    }
    
    /**
     * Basic record examples
     */
    public static void basicExamples() {
        System.out.println("\n=== Basic Examples ===");
        
        // Simple point
        record Point(int x, int y) {}
        Point p = new Point(10, 20);
        System.out.println("Point: " + p);
        System.out.println("X: " + p.x() + ", Y: " + p.y());
        
        // User data
        record User(String username, String email) {}
        User user = new User("alice", "alice@example.com");
        System.out.println("User: " + user);
        
        // Temperature
        record Temperature(double celsius) {
            // Can add methods
            public double fahrenheit() {
                return celsius * 9.0 / 5.0 + 32;
            }
        }
        Temperature temp = new Temperature(25);
        System.out.println("Celsius: " + temp.celsius());
        System.out.println("Fahrenheit: " + temp.fahrenheit());
        
        /**
         * BASIC PATTERN:
         * 
         * record Name(Type1 field1, Type2 field2, ...) {}
         * 
         * This generates:
         * - Constructor: Name(Type1 field1, Type2 field2)
         * - Accessors: field1(), field2()
         * - equals(), hashCode(), toString()
         * 
         * Use for:
         * - DTOs (Data Transfer Objects)
         * - Value objects
         * - Method return types
         * - Data containers
         * - Configuration
         */
    }
    
    public static void main(String[] args) {
        demonstration();
        basicExamples();
    }
}
```

## 2. Canonical Constructor

### The Automatically Generated Constructor

```java
/**
 * CANONICAL CONSTRUCTOR
 * 
 * The main constructor for records
 */

public class CanonicalConstructor {
    
    /**
     * Implicit canonical constructor
     */
    public static void implicitConstructor() {
        System.out.println("=== Implicit Constructor ===");
        
        // Simple record - constructor generated
        record Point(int x, int y) {}
        
        Point p = new Point(10, 20);
        System.out.println(p);
        
        /**
         * IMPLICIT CANONICAL CONSTRUCTOR:
         * 
         * For:
         * record Point(int x, int y) {}
         * 
         * Compiler generates:
         * public Point(int x, int y) {
         *   this.x = x;
         *   this.y = y;
         * }
         * 
         * Automatic:
         * - Public visibility
         * - Parameter for each component
         * - Assigns all fields
         * - No validation
         */
    }
    
    /**
     * Explicit canonical constructor
     */
    public static void explicitConstructor() {
        System.out.println("\n=== Explicit Constructor ===");
        
        // Explicit canonical constructor
        record Range(int min, int max) {
            // Explicit canonical constructor
            public Range(int min, int max) {
                if (min > max) {
                    throw new IllegalArgumentException(
                        "min must be <= max");
                }
                this.min = min;
                this.max = max;
            }
        }
        
        Range valid = new Range(1, 10);
        System.out.println("Valid: " + valid);
        
        try {
            Range invalid = new Range(10, 1);
        } catch (IllegalArgumentException e) {
            System.out.println("Invalid: " + e.getMessage());
        }
        
        /**
         * EXPLICIT CANONICAL CONSTRUCTOR:
         * 
         * Syntax:
         * record Name(Type field) {
         *   public Name(Type field) {
         *     // validation
         *     this.field = field;
         *   }
         * }
         * 
         * Use for:
         * - Validation
         * - Normalization
         * - Defensive copies
         * - Preconditions
         * 
         * Must:
         * - Match record signature
         * - Assign ALL fields
         * - Be public (or omit modifier)
         */
    }
    
    /**
     * Compact constructor (Java 14+)
     */
    public static void compactConstructor() {
        System.out.println("\n=== Compact Constructor ===");
        
        // Compact constructor - cleaner!
        record Range(int min, int max) {
            // Compact constructor
            public Range {
                if (min > max) {
                    throw new IllegalArgumentException(
                        "min must be <= max");
                }
                // No field assignment needed!
            }
        }
        
        Range r = new Range(1, 10);
        System.out.println("Range: " + r);
        
        /**
         * COMPACT CONSTRUCTOR:
         * 
         * Syntax:
         * record Name(Type field) {
         *   public Name {
         *     // validation/normalization
         *     // NO parameter list
         *     // NO field assignments
         *   }
         * }
         * 
         * Compiler transforms to:
         * public Name(Type field) {
         *   // Your code here
         *   this.field = field;  // Auto-added
         * }
         * 
         * Benefits:
         * ✓ Less boilerplate
         * ✓ Can't forget assignments
         * ✓ Clearer intent
         * ✓ Recommended style
         * 
         * Use for:
         * - Validation
         * - Normalization
         * - Null checks
         */
    }
    
    /**
     * Validation examples
     */
    public static void validationExamples() {
        System.out.println("\n=== Validation ===");
        
        // Null check
        record Person(String name, int age) {
            public Person {
                if (name == null) {
                    throw new NullPointerException("name cannot be null");
                }
                if (age < 0 || age > 150) {
                    throw new IllegalArgumentException("age out of range");
                }
            }
        }
        
        Person p = new Person("Alice", 30);
        System.out.println("Valid: " + p);
        
        // Range validation
        record Temperature(double celsius) {
            public Temperature {
                if (celsius < -273.15) {
                    throw new IllegalArgumentException(
                        "Below absolute zero!");
                }
            }
        }
        
        Temperature temp = new Temperature(25);
        System.out.println("Temperature: " + temp);
        
        /**
         * VALIDATION PATTERNS:
         * 
         * 1. Null checks:
         * record Person(String name) {
         *   public Person {
         *     if (name == null) {
         *       throw new NullPointerException();
         *     }
         *   }
         * }
         * 
         * 2. Range validation:
         * record Age(int value) {
         *   public Age {
         *     if (value < 0 || value > 150) {
         *       throw new IllegalArgumentException();
         *     }
         *   }
         * }
         * 
         * 3. Objects.requireNonNull:
         * record Person(String name) {
         *   public Person {
         *     Objects.requireNonNull(name, "name required");
         *   }
         * }
         * 
         * 4. Multiple validations:
         * record User(String name, String email) {
         *   public User {
         *     Objects.requireNonNull(name);
         *     Objects.requireNonNull(email);
         *     if (!email.contains("@")) {
         *       throw new IllegalArgumentException();
         *     }
         *   }
         * }
         */
    }
    
    /**
     * Normalization in constructor
     */
    public static void normalization() {
        System.out.println("\n=== Normalization ===");
        
        // Normalize email to lowercase
        record User(String name, String email) {
            public User {
                email = email.toLowerCase();
            }
        }
        
        User user = new User("Alice", "ALICE@EXAMPLE.COM");
        System.out.println("Email: " + user.email());
        
        // Trim strings
        record Person(String firstName, String lastName) {
            public Person {
                firstName = firstName.trim();
                lastName = lastName.trim();
            }
        }
        
        Person person = new Person("  Alice  ", "  Smith  ");
        System.out.println("Person: " + person);
        
        /**
         * NORMALIZATION:
         * 
         * Can modify parameters before assignment:
         * record Name(String value) {
         *   public Name {
         *     value = value.trim().toLowerCase();
         *   }
         * }
         * 
         * Common normalizations:
         * - trim() strings
         * - toLowerCase() for case-insensitive
         * - Remove whitespace
         * - Format phone numbers
         * - Standardize dates
         * 
         * Note:
         * Parameters are mutable in compact constructor
         * Assignment happens AFTER compact constructor
         */
    }
    
    /**
     * Defensive copying
     */
    public static void defensiveCopying() {
        System.out.println("\n=== Defensive Copying ===");
        
        // Defensive copy of mutable object
        record Wrapper(java.util.List<String> items) {
            public Wrapper {
                items = new java.util.ArrayList<>(items);
            }
        }
        
        java.util.List<String> original = 
            new java.util.ArrayList<>(java.util.List.of("a", "b"));
        Wrapper w = new Wrapper(original);
        
        // Modify original
        original.add("c");
        
        System.out.println("Original: " + original);
        System.out.println("Wrapper: " + w.items());
        
        /**
         * DEFENSIVE COPYING:
         * 
         * For mutable components:
         * record Wrapper(List<String> items) {
         *   public Wrapper {
         *     items = List.copyOf(items);  // Immutable copy
         *     // or
         *     items = new ArrayList<>(items);  // Mutable copy
         *   }
         * }
         * 
         * For arrays:
         * record Data(int[] values) {
         *   public Data {
         *     values = values.clone();
         *   }
         * }
         * 
         * For dates:
         * record Event(Date date) {
         *   public Event {
         *     date = new Date(date.getTime());
         *   }
         * }
         * 
         * Important:
         * Records are "shallowly immutable"
         * Deep immutability needs defensive copying
         */
    }
    
    public static void main(String[] args) {
        implicitConstructor();
        explicitConstructor();
        compactConstructor();
        validationExamples();
        normalization();
        defensiveCopying();
    }
}
```

## 3. Component Accessors

### Automatic Getter Methods

```java
/**
 * COMPONENT ACCESSORS
 * 
 * Automatic accessor methods for record components
 */

public class ComponentAccessors {
    
    /**
     * Default accessors
     */
    public static void defaultAccessors() {
        System.out.println("=== Default Accessors ===");
        
        record Person(String name, int age, String email) {}
        
        Person person = new Person("Alice", 30, "alice@example.com");
        
        // Accessor methods (NOT getters!)
        System.out.println("Name: " + person.name());
        System.out.println("Age: " + person.age());
        System.out.println("Email: " + person.email());
        
        /**
         * DEFAULT ACCESSORS:
         * 
         * For each component, generates:
         * - Public method
         * - Same name as component (NOT getXxx())
         * - Returns component value
         * - No "get" prefix
         * 
         * Example:
         * record Point(int x, int y) {}
         * 
         * Generates:
         * public int x() { return x; }
         * public int y() { return y; }
         * 
         * NOT:
         * public int getX() ...  ✗
         * public int getY() ...  ✗
         * 
         * This is intentional:
         * - Concise names
         * - Like field access
         * - Modern style
         */
    }
    
    /**
     * Custom accessors
     */
    public static void customAccessors() {
        System.out.println("\n=== Custom Accessors ===");
        
        // Override accessor
        record Temperature(double celsius) {
            // Custom accessor
            @Override
            public double celsius() {
                return Math.round(celsius * 10.0) / 10.0;
            }
            
            // Additional method
            public double fahrenheit() {
                return celsius * 9.0 / 5.0 + 32;
            }
        }
        
        Temperature temp = new Temperature(25.67);
        System.out.println("Celsius: " + temp.celsius());  // Rounded
        System.out.println("Fahrenheit: " + temp.fahrenheit());
        
        /**
         * CUSTOM ACCESSORS:
         * 
         * Can override accessor:
         * record Name(Type field) {
         *   @Override
         *   public Type field() {
         *     // custom logic
         *     return field;  // or computed value
         *   }
         * }
         * 
         * Use for:
         * - Computed values
         * - Formatting
         * - Rounding
         * - Defensive copies
         * - Validation
         */
    }
    
    /**
     * Defensive copying in accessor
     */
    public static void defensiveCopyInAccessor() {
        System.out.println("\n=== Defensive Copy in Accessor ===");
        
        // Return defensive copy
        record Data(java.util.List<String> items) {
            @Override
            public java.util.List<String> items() {
                return java.util.List.copyOf(items);  // Immutable copy
            }
        }
        
        Data data = new Data(new java.util.ArrayList<>(
            java.util.List.of("a", "b", "c")));
        
        java.util.List<String> items = data.items();
        System.out.println("Items: " + items);
        
        try {
            items.add("d");  // UnsupportedOperationException
        } catch (UnsupportedOperationException e) {
            System.out.println("Cannot modify: immutable");
        }
        
        /**
         * DEFENSIVE COPYING:
         * 
         * For mutable components:
         * record Data(List<String> items) {
         *   @Override
         *   public List<String> items() {
         *     return List.copyOf(items);
         *   }
         * }
         * 
         * For arrays:
         * record Data(int[] values) {
         *   @Override
         *   public int[] values() {
         *     return values.clone();
         *   }
         * }
         * 
         * Trade-off:
         * - Extra object creation
         * - But ensures immutability
         * - Caller can't modify state
         */
    }
    
    /**
     * Computed values
     */
    public static void computedValues() {
        System.out.println("\n=== Computed Values ===");
        
        record Rectangle(int width, int height) {
            // Accessor returns computed value
            public int area() {
                return width * height;
            }
            
            public int perimeter() {
                return 2 * (width + height);
            }
        }
        
        Rectangle rect = new Rectangle(5, 10);
        System.out.println("Width: " + rect.width());
        System.out.println("Height: " + rect.height());
        System.out.println("Area: " + rect.area());
        System.out.println("Perimeter: " + rect.perimeter());
        
        /**
         * COMPUTED VALUES:
         * 
         * Not component accessors, but additional methods:
         * record Circle(double radius) {
         *   public double area() {
         *     return Math.PI * radius * radius;
         *   }
         *   
         *   public double circumference() {
         *     return 2 * Math.PI * radius;
         *   }
         * }
         * 
         * Benefits:
         * - Encapsulate calculations
         * - No redundant storage
         * - Always consistent
         * - Clear API
         */
    }
    
    /**
     * Accessor naming
     */
    public static void accessorNaming() {
        /**
         * ACCESSOR NAMING:
         * 
         * Records use name(), not getName():
         * record Person(String name) {}
         * person.name()  ✓
         * person.getName()  ✗
         * 
         * Rationale:
         * - Concise
         * - Like field access
         * - Modern languages (Kotlin, Scala)
         * - Less boilerplate
         * 
         * 
         * INTEROP WITH JAVABEANS:
         * 
         * If need JavaBeans naming:
         * record Person(String name) {
         *   // Add JavaBeans-style getter
         *   public String getName() {
         *     return name();
         *   }
         * }
         * 
         * Or use traditional class
         * 
         * 
         * BOOLEAN COMPONENTS:
         * 
         * record Status(boolean active) {}
         * status.active()  ✓
         * 
         * NOT:
         * status.isActive()  ✗
         * status.getActive()  ✗
         * 
         * If need "is" prefix, add method:
         * public boolean isActive() {
         *   return active();
         * }
         */
    }
    
    public static void main(String[] args) {
        defaultAccessors();
        customAccessors();
        defensiveCopyInAccessor();
        computedValues();
    }
}
```

## 4. equals(), hashCode(), toString()

### Automatically Generated Methods

```java
/**
 * AUTO-GENERATED METHODS
 * 
 * Records automatically generate equals, hashCode, toString
 */

import java.util.*;

public class AutoGeneratedMethods {
    
    /**
     * equals() method
     */
    public static void equalsMethod() {
        System.out.println("=== equals() ===");
        
        record Point(int x, int y) {}
        
        Point p1 = new Point(10, 20);
        Point p2 = new Point(10, 20);
        Point p3 = new Point(15, 25);
        
        System.out.println("p1.equals(p2): " + p1.equals(p2));  // true
        System.out.println("p1.equals(p3): " + p1.equals(p3));  // false
        System.out.println("p1 == p2: " + (p1 == p2));          // false
        
        /**
         * EQUALS():
         * 
         * Generated equals():
         * - Compares all components
         * - Component-wise equality
         * - Null-safe
         * - Type-safe
         * 
         * Equivalent to:
         * @Override
         * public boolean equals(Object obj) {
         *   if (this == obj) return true;
         *   if (obj == null || getClass() != obj.getClass()) 
         *     return false;
         *   Point other = (Point) obj;
         *   return x == other.x && y == other.y;
         * }
         * 
         * Compares by VALUE, not reference
         * Two records equal if all components equal
         */
    }
    
    /**
     * hashCode() method
     */
    public static void hashCodeMethod() {
        System.out.println("\n=== hashCode() ===");
        
        record Person(String name, int age) {}
        
        Person p1 = new Person("Alice", 30);
        Person p2 = new Person("Alice", 30);
        Person p3 = new Person("Bob", 25);
        
        System.out.println("p1 hashCode: " + p1.hashCode());
        System.out.println("p2 hashCode: " + p2.hashCode());
        System.out.println("p3 hashCode: " + p3.hashCode());
        System.out.println("p1.hashCode() == p2.hashCode(): " + 
            (p1.hashCode() == p2.hashCode()));
        
        // Use in HashMap
        Map<Person, String> map = new HashMap<>();
        map.put(p1, "Employee 1");
        map.put(p3, "Employee 2");
        
        System.out.println("Map size: " + map.size());
        System.out.println("Get p2: " + map.get(p2));  // Works!
        
        /**
         * HASHCODE():
         * 
         * Generated hashCode():
         * - Based on all components
         * - Consistent with equals()
         * - Properly implemented
         * 
         * Guarantees:
         * - If equals(), same hashCode()
         * - Can use in HashMap, HashSet
         * - Correct hash contract
         * 
         * No need to override
         * Already optimal
         */
    }
    
    /**
     * toString() method
     */
    public static void toStringMethod() {
        System.out.println("\n=== toString() ===");
        
        record Person(String name, int age, String email) {}
        
        Person person = new Person("Alice", 30, "alice@example.com");
        System.out.println(person);
        
        /**
         * TOSTRING():
         * 
         * Generated toString():
         * RecordName[component1=value1, component2=value2, ...]
         * 
         * Example:
         * Person[name=Alice, age=30, email=alice@example.com]
         * 
         * Benefits:
         * - Clear format
         * - All components shown
         * - Easy debugging
         * - Consistent style
         * 
         * Can override if needed:
         * @Override
         * public String toString() {
         *   return name + " (" + age + ")";
         * }
         */
    }
    
    /**
     * Custom equals/hashCode/toString
     */
    public static void customMethods() {
        System.out.println("\n=== Custom Methods ===");
        
        // Custom toString
        record Person(String name, int age) {
            @Override
            public String toString() {
                return name + " (age " + age + ")";
            }
        }
        
        Person person = new Person("Alice", 30);
        System.out.println("Custom: " + person);
        
        /**
         * CUSTOMIZATION:
         * 
         * Can override any generated method:
         * - toString() - most common
         * - equals() - rarely needed
         * - hashCode() - rarely needed
         * 
         * Warning:
         * If override equals(), must override hashCode()
         * Better to use default for both
         * 
         * Use custom toString() for:
         * - User-friendly display
         * - Logging
         * - Specific format
         */
    }
    
    /**
     * Records in collections
     */
    public static void recordsInCollections() {
        System.out.println("\n=== Records in Collections ===");
        
        record Point(int x, int y) {}
        
        // HashSet - uses equals() and hashCode()
        Set<Point> points = new HashSet<>();
        points.add(new Point(1, 2));
        points.add(new Point(1, 2));  // Duplicate!
        points.add(new Point(3, 4));
        
        System.out.println("Set size: " + points.size());  // 2
        System.out.println("Contains (1,2): " + 
            points.contains(new Point(1, 2)));  // true
        
        // HashMap - works perfectly
        Map<Point, String> labels = new HashMap<>();
        labels.put(new Point(0, 0), "Origin");

/**
```

```java
import java.util.Objects;

public class ValidationPatterns {
    
    /**
     * Null validation
     */
    public static void nullValidation() {
        System.out.println("=== Null Validation ===");
        
        // Using Objects.requireNonNull
        record Person(String name, String email) {
            public Person {
                Objects.requireNonNull(name, "name cannot be null");
                Objects.requireNonNull(email, "email cannot be null");
            }
        }
        
        Person valid = new Person("Alice", "alice@example.com");
        System.out.println("Valid: " + valid);
        
        try {
            Person invalid = new Person(null, "email@example.com");
        } catch (NullPointerException e) {
            System.out.println("Null rejected: " + e.getMessage());
        }
        
        /**
         * NULL VALIDATION:
         * 
         * Best practice:
         * Objects.requireNonNull(field, "message")
         * 
         * Alternative:
         * if (field == null) {
         *   throw new NullPointerException("message");
         * }
         * 
         * For multiple fields:
         * public Person {
         *   Objects.requireNonNull(name);
         *   Objects.requireNonNull(email);
         *   Objects.requireNonNull(phone);
         * }
         */
    }
    
    /**
     * Range validation
     */
    public static void rangeValidation() {
        System.out.println("\n=== Range Validation ===");
        
        record Age(int value) {
            public Age {
                if (value < 0 || value > 150) {
                    throw new IllegalArgumentException(
                        "Age must be between 0 and 150, got: " + value);
                }
            }
        }
        
        Age valid = new Age(30);
        System.out.println("Valid: " + valid);
        
        try {
            Age invalid = new Age(200);
        } catch (IllegalArgumentException e) {
            System.out.println("Invalid: " + e.getMessage());
        }
        
        /**
         * RANGE VALIDATION:
         * 
         * Pattern:
         * record Value(int n) {
         *   public Value {
         *     if (n < MIN || n > MAX) {
         *       throw new IllegalArgumentException();
         *     }
         *   }
         * }
         * 
         * For floating point:
         * if (value < 0.0 || value > 1.0) ...
         * if (Double.isNaN(value)) ...
         * if (Double.isInfinite(value)) ...
         */
    }
    
    /**
     * String validation
     */
    public static void stringValidation() {
        System.out.println("\n=== String Validation ===");
        
        record Email(String address) {
            public Email {
                Objects.requireNonNull(address);
                if (address.isBlank()) {
                    throw new IllegalArgumentException("Email cannot be blank");
                }
                if (!address.contains("@")) {
                    throw new IllegalArgumentException("Invalid email format");
                }
            }
        }
        
        Email valid = new Email("user@example.com");
        System.out.println("Valid: " + valid);
        
        try {
            Email invalid = new Email("notanemail");
        } catch (IllegalArgumentException e) {
            System.out.println("Invalid: " + e.getMessage());
        }
        
        /**
         * STRING VALIDATION:
         * 
         * Common checks:
         * - Not null: Objects.requireNonNull()
         * - Not blank: !s.isBlank()
         * - Not empty: !s.isEmpty()
         * - Min length: s.length() >= MIN
         * - Max length: s.length() <= MAX
         * - Pattern: s.matches(regex)
         * - Contains: s.contains(substring)
         * 
         * Example:
         * record Username(String value) {
         *   public Username {
         *     Objects.requireNonNull(value);
         *     if (value.length() < 3 || value.length() > 20) {
         *       throw new IllegalArgumentException(
         *         "Username must be 3-20 characters");
         *     }
         *     if (!value.matches("[a-zA-Z0-9_]+")) {
         *       throw new IllegalArgumentException(
         *         "Username can only contain letters, numbers, underscore");
         *     }
         *   }
         * }
         */
    }
    
    /**
     * Cross-field validation
     */
    public static void crossFieldValidation() {
        System.out.println("\n=== Cross-Field Validation ===");
        
        record DateRange(
                java.time.LocalDate start, 
                java.time.LocalDate end) {
            public DateRange {
                Objects.requireNonNull(start);
                Objects.requireNonNull(end);
                if (start.isAfter(end)) {
                    throw new IllegalArgumentException(

                }
            }
        }
        
        var valid = new DateRange(
            java.time.LocalDate.of(2023, 1, 1),
            java.time.LocalDate.of(2023, 12, 31)
        );
        System.out.println("Valid: " + valid);
        
        /**
         * CROSS-FIELD VALIDATION:
         * 
         * Check relationships between fields:
         * 
         * record Range(int min, int max) {
         *   public Range {
         *     if (min > max) {
         *       throw new IllegalArgumentException();
         *     }
         *   }
         * }
         * 
         * record Rectangle(int width, int height) {
         *   public Rectangle {
         *     if (width <= 0 || height <= 0) {
         *       throw new IllegalArgumentException();
         *     }
         *   }
         * }
         * 
         * record Credentials(String username, String password) {
         *   public Credentials {

         *     }
         *   }
         * }
         */
    }
    
    /**
     * Collection validation
     */
    public static void collectionValidation() {

                Objects.requireNonNull(members);

                    throw new IllegalArgumentException(
                        "Team must have at least one member");

                members = java.util.List.copyOf(members);  // Immutable
            }
        }
        
        Team team = new Team("Alpha",

        System.out.println("Team: " + team);
        
        /**
         * COLLECTION VALIDATION:
         * 
         * Pattern:
         * record Data(List<String> items) {
         *   public Data {
         *     Objects.requireNonNull(items);
         *     if (items.isEmpty()) {
         *       throw new IllegalArgumentException();
         *     }
         *     // Defensive copy
         *     items = List.copyOf(items);
         *   }
         * }
         * 
         * Checks:
         * - Not null
         * - Not empty
         * - Size constraints
         * - No null elements
         * - Make immutable copy
         * 
         * For null elements:
         * if (items.contains(null)) {
         *   throw new IllegalArgumentException();
         * }
         */
    }

                validateEmail(email);
                validateAge(age);

            private static void validateEmail(String email) {

         * Benefits:
         * ✓ Reusable logic

         * ✓ Organized code
         *

         * record Data(String field1, String field2) {
         *   public Data {

         *   }
         *   
         *   private static void validateField1(String f) {
```

```java
/**

public class RecordsVsClasses {
    
    /**
     * Use records when
     */
    public static void useRecordsWhen() {
        /**
         * USE RECORDS FOR:
         * 
         * 1. Data transfer objects (DTOs):
         * record UserDTO(long id, String name, String email) {}
         * 
         * 2. Value objects:
         * record Money(BigDecimal amount, Currency currency) {}
         * record Point(int x, int y) {}
         * 
         * 3. Immutable data:
         * record Configuration(String host, int port, boolean ssl) {}
         * 
         * 4. Method return types:
         * record Result(boolean success, String message) {}
         * 
         * 5. Map entries:
         * record Pair<K, V>(K key, V value) {}
         * 
         * 6. Domain models (simple):
         * record Customer(String id, String name, String email) {}
         * 
         * 
         * CHARACTERISTICS:
         * ✓ Transparent data carriers
         * ✓ No behavior (or minimal)
         * ✓ Immutable
         * ✓ Value semantics
         * ✓ Equals by value
         */
    }
    
    /**
     * Use classes when
     */
    public static void useClassesWhen() {
        /**
         * USE CLASSES FOR:
         * 
         * 1. Need inheritance:
         * class Manager extends Employee { ... }
         * 
         * 2. Mutable state:
         * class ShoppingCart {
         *   private List<Item> items = new ArrayList<>();
         *   public void addItem(Item item) { ... }
         * }
         * 
         * 3. Complex behavior:
         * class OrderProcessor {
         *   public void process(Order order) { ... }
         * }
         * 
         * 4. Encapsulation needed:
         * class BankAccount {
         *   private BigDecimal balance;  // Hidden
         *   public void deposit(BigDecimal amount) { ... }
         * }
         * 
         * 5. Identity-based equality:
         * class User {
         *   private UUID id;
         *   // equals() based on id only
         * }
         * 
         * 6. JavaBeans conventions:
         * class Person {
         *   public String getName() { ... }
         *   public void setName(String name) { ... }
         * }
         * 
         * 
         * CHARACTERISTICS:
         * ✓ Extensible
         * ✓ Mutable (if needed)
         * ✓ Encapsulated
         * ✓ Behavioral
         * ✓ Identity semantics
         */
    }
    
    /**
     * Comparison table
     */
    public static void comparisonTable() {
        /**
         * RECORDS VS CLASSES:
         * 
         * ┌────────────────────┬─────────────┬─────────────┐
         * │ Feature            │ Record      │ Class       │
         * ├────────────────────┼─────────────┼─────────────┤
         * │ Inheritance        │ No          │ Yes         │
         * │ Mutability         │ Immutable   │ Flexible    │
         * │ Fields             │ Final       │ Any         │
         * │ Constructor        │ Auto        │ Manual      │
         * │ Accessors          │ Auto        │ Manual      │
         * │ equals/hashCode    │ Auto        │ Manual      │
         * │ toString           │ Auto        │ Manual      │
         * │ Boilerplate        │ Minimal     │ Extensive   │
         * │ Use case           │ Data        │ Behavior    │
         * └────────────────────┴─────────────┴─────────────┘
         */
    }
    
    /**
     * Migration example
     */
    public static void migration() {
        /**
         * MIGRATING CLASS → RECORD:
         * 
         * Before:
         * public final class Point {
         *   private final int x;
         *   private final int y;
         *   
         *   public Point(int x, int y) {
         *     this.x = x;
         *     this.y = y;
         *   }
         *   
         *   public int getX() { return x; }
         *   public int getY() { return y; }
         *   
         *   @Override
         *   public boolean equals(Object obj) { ... }
         *   
         *   @Override
         *   public int hashCode() { ... }
         *   
         *   @Override
         *   public String toString() { ... }
         * }
         * 
         * After:

         * 
         * BREAKING CHANGES:
         * 
         * Accessors:
         * Before: point.getX()
         * After: point.x()
         * 
         * Solution:
         * Add legacy methods:
         * record Point(int x, int y) {
         *   public int getX() { return x(); }
         *   public int getY() { return y(); }
         * }
         */
    }
    
    /**
     * Hybrid approach
     */
    public static void hybridApproach() {
        /**
         * HYBRID: RECORDS + CLASSES
         * 
         * Use both appropriately:
         * 
         * // Record for data

         *                  BigDecimal total) {}

         * // Class for behavior
         * class OrderProcessor {
         *   public OrderData process(Order order) {

         * }

         * 
         * PATTERN:
         * 
         * Records:
         * - Input/Output
         * - DTOs
         * - Value objects
         * 
         * Classes:
         * - Services
         * - Controllers
         * - Repositories
         * - Processors

         * - Clean separation
         * - Data vs behavior
         * - Immutability where it matters
         */
    }
    
    /**
     * Best practices
     */
    public static void bestPractices() {
        /**
         * BEST PRACTICES:
         * 
         * 1. Default to records for data:
         * ✓ Start with record
         * ✓ Switch to class if needed
         * 
         * 2. Keep records simple:
         * ✓ Minimal methods
         * ✓ No complex logic
         * ✓ Pure data
         * 
         * 3. Validate in constructor:

         * ✓ Need inheritance? Use class

    }
```

```java
/**
 * RECORDS WITH INTERFACES
 * 
 * How records can implement interfaces
 */

import java.util.Comparator;

public class RecordsWithInterfaces {
    
    /**
     * Basic interface implementation
     */
    public static void basicInterface() {
        System.out.println("=== Basic Interface ===");
        
        interface Describable {
            String describe();
        }
        
        record Person(String name, int age) implements Describable {
            @Override
            public String describe() {
                return name + " is " + age + " years old";
            }
        }
        
        Person person = new Person("Alice", 30);
        System.out.println(person.describe());
        
        /**
         * INTERFACE IMPLEMENTATION:
         * 
         * Syntax:
         * record Name(...) implements Interface1, Interface2 {
         *   // Implement methods
         * }
         * 
         * Can implement:
         * - Any interface
         * - Multiple interfaces
         * - Default methods inherited
         * 
         * Cannot:
         * - Extend classes
         * - Be extended
         */
    }
    
    /**
     * Comparable interface
     */
    public static void comparableInterface() {
        System.out.println("\n=== Comparable Interface ===");
        
        record Person(String name, int age) 
                implements Comparable<Person> {
            @Override
            public int compareTo(Person other) {
                return Integer.compare(this.age, other.age);
            }
        }
        
        var people = java.util.List.of(
            new Person("Alice", 30),
            new Person("Bob", 25),
            new Person("Charlie", 35)
        );
        
        var sorted = people.stream()
            .sorted()
            .toList();
        
        System.out.println("Sorted by age:");
        sorted.forEach(System.out::println);
        
        /**
         * COMPARABLE:
         * 
         * Common pattern for records:
         * record Point(int x, int y) implements Comparable<Point> {
         *   @Override
         *   public int compareTo(Point other) {
         *     int cmp = Integer.compare(this.x, other.x);
         *     if (cmp != 0) return cmp;
         *     return Integer.compare(this.y, other.y);
         *   }
         * }
         * 
         * Or use Comparator.comparing:
         * record Person(String name, int age) {
         *   private static final Comparator<Person> BY_AGE =
         *     Comparator.comparing(Person::age);
         * }
         */
    }
    
    /**
     * Multiple interfaces
     */
    public static void multipleInterfaces() {
        System.out.println("\n=== Multiple Interfaces ===");
        
        interface Validatable {
            boolean isValid();
        }
        
        interface Serializable {
            String serialize();
        }
        
        record Email(String address) 
                implements Validatable, Serializable {
            @Override
            public boolean isValid() {
                return address != null && address.contains("@");
            }
            
            @Override
            public String serialize() {
                return address;
            }
        }
        
        Email email = new Email("user@example.com");
        System.out.println("Valid: " + email.isValid());
        System.out.println("Serialized: " + email.serialize());
        
        /**
         * MULTIPLE INTERFACES:
         * 
         * record Name(...) 
         *     implements Interface1, Interface2, Interface3 {
         *   // Implement all methods
         * }
         * 
         * Common combinations:
         * - Comparable + Serializable
         * - Validatable + Describable
         * - Custom domain interfaces
         */
    }
    
    /**
     * Sealed interfaces with records
     */
    public static void sealedInterfaces() {
        System.out.println("\n=== Sealed Interfaces ===");
        
        sealed interface Result permits Success, Failure {}
        record Success(String value) implements Result {}
        record Failure(String error) implements Result {}
        
        Result result = new Success("Operation completed");
        
        // Pattern matching
        String message = switch (result) {
            case Success s -> "Success: " + s.value();
            case Failure f -> "Failure: " + f.error();
        };
        
        System.out.println(message);
        
        /**
         * SEALED INTERFACES + RECORDS:
         * 
         * Perfect combination:
         * sealed interface Shape permits Circle, Rectangle {}
         * record Circle(double radius) implements Shape {}
         * record Rectangle(double width, double height) 
         *     implements Shape {}
         * 
         * Benefits:
         * ✓ Exhaustive pattern matching
         * ✓ Type safety
         * ✓ Clean ADTs (Algebraic Data Types)
         * ✓ Compiler helps
         * 
         * Use for:
         * - Domain modeling
         * - State machines
         * - Result types
         * - Event hierarchies
         */
    }
    
    /**
     * Functional interfaces
     */
    public static void functionalInterfaces() {
        System.out.println("\n=== Functional Interfaces ===");
        
        record Adder(int value) implements java.util.function.IntUnaryOperator {
            @Override
            public int applyAsInt(int operand) {
                return operand + value;
            }
        }
        
        Adder add5 = new Adder(5);
        System.out.println("10 + 5 = " + add5.applyAsInt(10));
        
        // Use as function
        int[] numbers = {1, 2, 3, 4, 5};
        int[] results = java.util.Arrays.stream(numbers)
            .map(add5)
            .toArray();
        
        System.out.println("Results: " + java.util.Arrays.toString(results));
        
        /**
         * FUNCTIONAL INTERFACES:
         * 
         * Records can implement:
         * - Function
         * - Predicate
         * - Consumer
         * - Supplier
         * - Custom functional interfaces
         * 
         * Use for:
         * - Stateful functions
         * - Configured predicates
         * - Parameterized consumers
         * 
         * Example:
         * record RangePredicate(int min, int max) 
         *     implements Predicate<Integer> {
         *   @Override
         *   public boolean test(Integer value) {
         *     return value >= min && value <= max;
         *   }
         * }
         */
    }
    
    public static void main(String[] args) {
        basicInterface();
        comparableInterface();
        multipleInterfaces();
        sealedInterfaces();
        functionalInterfaces();
    }
}
```

## Summary

### Quick Reference

**Basic Record:**

```java
record Person(String name, int age) {}
// Generates: constructor, accessors, equals, hashCode, toString
```

**With Validation:**

```java
record Person(String name, int age) {
    public Person {  // Compact constructor
        Objects.requireNonNull(name);
        if (age < 0) throw new IllegalArgumentException();
    }
}
```

**Custom Methods:**

```java
record Rectangle(int width, int height) {
    public int area() {
        return width * height;
    }
}
```

**With Interface:**

```java
record Point(int x, int y) implements Comparable<Point> {
    @Override
    public int compareTo(Point other) {
        return Integer.compare(x, other.x);
    }
}
```

**Usage:**

```java
Person p = new Person("Alice", 30);
String name = p.name();  // Not getName()!
int age = p.age();
System.out.println(p);  // Person[name=Alice, age=30]
```

---


**Key insights:**

- **50 lines → 1 line**: Record replaces entire data class
- **Compact constructor**: Validation before field assignment
- **Value semantics**: Equals by value, perfect for HashMap
- **Immutable**: All fields final, transparent data carriers
- **Interfaces**: Can implement, cannot extend classes

**The revolution:**

```
Before Records:
- 50+ lines of boilerplate
- Manual constructor
- Manual getters
- Manual equals/hashCode/toString
- Easy to mess up

After Records:
- 1 line of code
- Everything generated
- Correct by default
- Impossible to mess up

From verbose to concise
From error-prone to bulletproof
```
