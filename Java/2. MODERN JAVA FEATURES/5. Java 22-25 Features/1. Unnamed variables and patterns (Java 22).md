# Unnamed Variables and Patterns (Java 22)

## Overview

Unnamed variables and patterns, finalized in Java 22 (JEP 456), allow developers to use the underscore character `_` to explicitly mark variables as unused, improving code clarity and intent. Before Java 22, unused variables created clutter—you had to name them even if you never used them, and IDEs would warn about unused variables. The underscore `_` signals "I know this value exists, but I intentionally don't use it." This works in multiple contexts: lambda parameters, exception parameters, try-with-resources, pattern matching (both instanceof and switch), and record patterns. The feature reduces cognitive load by eliminating meaningless names like `ignored`, `unused`, or single letters, making code more readable and maintainable. Unnamed patterns in switch are particularly powerful when combined with record patterns, allowing you to extract only the components you need while ignoring others. The compiler recognizes `_` as truly unnamed—you cannot reference it, ensuring that declaring something as unused is enforced at compile time. This guide covers all contexts where `_` can be used, before/after comparisons, pattern matching improvements, practical use cases, and best practices for improving code readability.

## 1. Underscore _ for Unused Variables

### The Problem and Solution

```java
/**
 * UNDERSCORE FOR UNUSED VARIABLES
 * 
 * Explicitly marking variables as unused
 */

import java.util.*;

public class UnderscoreUnusedVariables {
    
    /**
     * The problem before Java 22
     */
    public static void beforeUnnamedVariables() {
        System.out.println("=== Before Unnamed Variables ===");
        
        Map<String, Integer> map = Map.of("A", 1, "B", 2, "C", 3);
        
        // Only need keys, not values
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            String key = entry.getKey();
            Integer value = entry.getValue();  // Unused!
            System.out.println("Key: " + key);
            // IDE warns: "value is never used"
        }
        
        /**
         * PROBLEMS:
         * 
         * - Must name unused variables
         * - IDE warnings clutter
         * - Names like "ignored", "unused" are noise
         * - Unclear if intentional or bug
         * - Extra cognitive load
         */
    }
    
    /**
     * After Java 22 with unnamed variables
     */
    public static void afterUnnamedVariables() {
        System.out.println("\n=== After Unnamed Variables ===");
        
        Map<String, Integer> map = Map.of("A", 1, "B", 2, "C", 3);
        
        // Underscore shows we intentionally ignore value
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            String key = entry.getKey();
            Integer _ = entry.getValue();  // Explicitly unused!
            System.out.println("Key: " + key);
            // No IDE warning - clear intent
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ Clear intent: value exists but unused
         * ✓ No IDE warnings
         * ✓ Less noise in code
         * ✓ Cannot accidentally use it
         * ✓ Self-documenting
         */
    }
    
    /**
     * Basic usage
     */
    public static void basicUsage() {
        System.out.println("\n=== Basic Usage ===");
        
        // Extracting from pair
        var pair = Map.entry("key", "value");
        String key = pair.getKey();
        String _ = pair.getValue();  // Don't need value
        
        System.out.println("Key: " + key);
        
        // Can't reference unnamed variable
        // System.out.println(_);  // Compilation error!
        
        /**
         * SYNTAX:
         * 
         * Type _ = expression;
         * 
         * Rules:
         * - Must use single underscore _
         * - Cannot reference it
         * - Type must be specified (or use var)
         * - Shows intentional non-use
         */
    }
    
    /**
     * Multiple unnamed variables
     */
    public static void multipleUnnamedVariables() {
        System.out.println("\n=== Multiple Unnamed Variables ===");
        
        // Multiple underscores in same scope
        String _ = "first unused";
        Integer _ = 42;  // Different type, same name - OK!
        
        // In different contexts
        if (Math.random() > 0.5) {
            Double _ = 3.14;
        }
        
        System.out.println("Multiple underscores allowed!");
        
        /**
         * MULTIPLE USES:
         * 
         * Can use _ multiple times:
         * - Same scope
         * - Different types
         * - Each independent
         * 
         * Unlike normal variables:
         * int x = 1;
         * int x = 2;  // ERROR: duplicate!
         * 
         * With underscore:
         * int _ = 1;
         * int _ = 2;  // OK: both unnamed
         */
    }
    
    /**
     * Loop variables
     */
    public static void loopVariables() {
        System.out.println("\n=== Loop Variables ===");
        
        // Traditional for loop
        for (int _ = 0; _ < 3; _++) {
            // Wait, can't reference _!
            // This doesn't work!
        }
        
        /**
         * FOR LOOP LIMITATION:
         * 
         * Cannot use _ in traditional for loop:
         * for (int _ = 0; _ < 3; _++)
         * 
         * Because you need to reference it in:
         * - Condition check
         * - Increment
         * 
         * But works in for-each where you don't need it:
         * for (String _ : list) { ... }
         */
        
        // Enhanced for loop - works!
        List<String> list = List.of("A", "B", "C");
        for (String _ : list) {
            System.out.println("Processing item");
        }
    }
    
    /**
     * Local variables
     */
    public static void localVariables() {
        System.out.println("\n=== Local Variables ===");
        
        // Method returns multiple values via object
        record Result(boolean success, String message, int code) {}
        
        Result result = processTask();
        
        // Only care about success
        boolean success = result.success();
        String _ = result.message();  // Unused
        int _ = result.code();        // Unused
        
        if (success) {
            System.out.println("Task succeeded");
        }
        
        /**
         * USE CASES:
         * 
         * 1. Extracting from records:
         * record Point(int x, int y) {}
         * Point p = ...;
         * int x = p.x();
         * int _ = p.y();  // Don't need y
         * 
         * 2. Side effects only:
         * String _ = performAction();
         * // Only care that it executed
         * 
         * 3. API requirements:
         * SomeType _ = builder.build();
         * // Must call build() but don't need result
         */
    }
    
    private static Result processTask() {
        record Result(boolean success, String message, int code) {}
        return new Result(true, "OK", 200);
    }
    
    /**
     * Compiler enforcement
     */
    public static void compilerEnforcement() {
        /**
         * COMPILER ENFORCEMENT:
         * 
         * Cannot reference unnamed variable:
         * 
         * int _ = 42;
         * System.out.println(_);  // ERROR!
         * 
         * Error: "cannot find symbol: _"
         * 
         * This is by design:
         * - If you name it _, you promise not to use it
         * - Compiler enforces this promise
         * - If you need it, give it a real name
         * 
         * 
         * COMPARISON:
         * 
         * Before:
         * int unused = 42;
         * // Could still use it accidentally
         * System.out.println(unused);  // Compiles!
         * 
         * After:
         * int _ = 42;
         * System.out.println(_);  // Compilation error!
         */
    }
    
    public static void main(String[] args) {
        beforeUnnamedVariables();
        afterUnnamedVariables();
        basicUsage();
        multipleUnnamedVariables();
        loopVariables();
        localVariables();
    }
}
```

## 2. Unnamed Patterns in Switch

### Pattern Matching Improvements

```java
/**
 * UNNAMED PATTERNS IN SWITCH
 * 
 * Using _ in pattern matching
 */

import java.util.*;

public class UnnamedPatternsInSwitch {
    
    /**
     * Before unnamed patterns
     */
    public static void beforeUnnamedPatterns() {
        System.out.println("=== Before Unnamed Patterns ===");
        
        Object obj = "Hello";
        
        String result = switch (obj) {
            case String s -> "String: " + s.length();
            case Integer i -> "Integer: " + i;
            case Double d -> "Double: " + d;
            case null -> "Null";
            case Object o -> "Other: " + o.getClass().getSimpleName();
            // Even though we don't use 'o', must name it
        };
        
        System.out.println(result);
        
        /**
         * PROBLEM:
         * 
         * default case must name the variable:
         * case Object o -> ...
         * 
         * Even if we don't use 'o'
         * Creates noise
         */
    }
    
    /**
     * After unnamed patterns
     */
    public static void afterUnnamedPatterns() {
        System.out.println("\n=== After Unnamed Patterns ===");
        
        Object obj = "Hello";
        
        String result = switch (obj) {
            case String s -> "String: " + s.length();
            case Integer i -> "Integer: " + i;
            case Double d -> "Double: " + d;
            case null -> "Null";
            case Object _ -> "Other";  // Don't need the variable!
        };
        
        System.out.println(result);
        
        /**
         * BENEFIT:
         * 
         * case Object _ -> ...
         * 
         * Clear: We match Object but don't use it
         * Cleaner than naming it
         */
    }
    
    /**
     * With record patterns
     */
    public static void withRecordPatterns() {
        System.out.println("\n=== With Record Patterns ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        
        Object shape = new Circle(new Point(5, 5), 10);
        
        // Before: Must name all components
        String result1 = switch (shape) {
            case Circle(Point center, double radius) -> 
                "Circle at " + center + " with radius " + radius;
            case null -> "Null";
            case Object obj -> "Not a circle";
        };
        
        // After: Use _ for unused components
        String result2 = switch (shape) {
            case Circle(Point _, double radius) -> 
                "Circle with radius " + radius;
            case null -> "Null";
            case Object _ -> "Not a circle";
        };
        
        System.out.println(result2);
        
        /**
         * RECORD PATTERN WITH _:
         * 
         * case Circle(Point _, double radius) -> ...
         * 
         * Extracts radius, ignores center
         * Clearer than:
         * case Circle(Point ignored, double radius) -> ...
         */
    }
    
    /**
     * Nested unnamed patterns
     */
    public static void nestedUnnamedPatterns() {
        System.out.println("\n=== Nested Unnamed Patterns ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        
        Object shape = new Circle(new Point(5, 5), 10);
        
        // Only need x coordinate from center
        String result = switch (shape) {
            case Circle(Point(int x, int _), double _) -> 
                "Circle at x=" + x;
            case Object _ -> "Other";
        };
        
        System.out.println(result);
        
        /**
         * NESTED PATTERNS:
         * 
         * case Circle(Point(int x, int _), double _) -> ...
         * 
         * Extracts only x from nested Point
         * Ignores:
         * - y coordinate
         * - radius
         * 
         * Super clean!
         */
    }
    
    /**
     * Multiple patterns with underscore
     */
    public static void multiplePatternsWithUnderscore() {
        System.out.println("\n=== Multiple Patterns ===");
        
        sealed interface Result {}
        record Success(String data, int code) implements Result {}
        record Failure(String error, int code) implements Result {}
        
        Result result = new Success("OK", 200);
        
        String message = switch (result) {
            // Only need data, not code
            case Success(String data, int _) -> 
                "Success: " + data;
            
            // Only need error, not code
            case Failure(String error, int _) -> 
                "Failure: " + error;
        };
        
        System.out.println(message);
        
        /**
         * PATTERN:
         * 
         * Both cases ignore code:
         * case Success(String data, int _) -> ...
         * case Failure(String error, int _) -> ...
         * 
         * Clearly shows code is unused
         */
    }
    
    /**
     * Partial record deconstruction
     */
    public static void partialRecordDeconstruction() {
        System.out.println("\n=== Partial Deconstruction ===");
        
        record Person(String name, int age, String email, String city) {}
        
        Object obj = new Person("Alice", 25, "alice@example.com", "NYC");
        
        // Only want name and city
        String info = switch (obj) {
            case Person(String name, int _, String _, String city) -> 
                name + " from " + city;
            case Object _ -> 
                "Not a person";
        };
        
        System.out.println(info);
        
        /**
         * PARTIAL EXTRACTION:
         * 
         * case Person(String name, int _, String _, String city) -> ...
         * 
         * Extracts:
         * ✓ name (position 0)
         * ✗ age (position 1) - ignored
         * ✗ email (position 2) - ignored
         * ✓ city (position 3)
         * 
         * Position still matters!
         * Can't skip to just name and city
         * Must acknowledge all positions
         */
    }
    
    /**
     * Comparison with default
     */
    public static void comparisonWithDefault() {
        /**
         * UNNAMED PATTERN VS DEFAULT:
         * 
         * Option 1: default
         * switch (obj) {
         *   case String s -> ...
         *   default -> ...
         * }
         * 
         * Option 2: case Object _
         * switch (obj) {
         *   case String s -> ...
         *   case Object _ -> ...
         * }
         * 
         * Difference:
         * - default catches everything (including null in statement)
         * - case Object _ only matches Objects (not null)
         * - case Object _ is more explicit
         * 
         * Use case Object _:
         * - When you want explicit type matching
         * - When pattern matching style preferred
         * - When handling null separately
         * 
         * Use default:
         * - When truly a catch-all
         * - When simpler is better
         * - Traditional switch style
         */
    }
    
    public static void main(String[] args) {
        beforeUnnamedPatterns();
        afterUnnamedPatterns();
        withRecordPatterns();
        nestedUnnamedPatterns();
        multiplePatternsWithUnderscore();
        partialRecordDeconstruction();
    }
}
```

## 3. Lambda Parameters

### Cleaner Lambda Expressions

```java
/**
 * LAMBDA PARAMETERS
 * 
 * Using _ in lambda expressions
 */

import java.util.*;
import java.util.function.*;

public class LambdaParameters {
    
    /**
     * Basic lambda with unused parameter
     */
    public static void basicLambda() {
        System.out.println("=== Basic Lambda ===");
        
        List<String> items = List.of("A", "B", "C");
        
        // Before: Must name unused parameter
        items.forEach(item -> System.out.println("Processing..."));
        // 'item' is never used
        
        // After: Use underscore
        items.forEach(_ -> System.out.println("Processing..."));
        // Clear: parameter exists but unused
        
        /**
         * BENEFIT:
         * 
         * _ -> expression
         * 
         * Shows we don't use the parameter
         * Common in:
         * - Side-effect only operations
         * - Fixed responses
         * - Callbacks that ignore input
         */
    }
    
    /**
     * Multiple lambda parameters
     */
    public static void multipleLambdaParameters() {
        System.out.println("\n=== Multiple Parameters ===");
        
        Map<String, Integer> map = Map.of("A", 1, "B", 2, "C", 3);
        
        // Before: Name parameters we don't use
        map.forEach((key, value) -> System.out.println("Entry"));
        
        // After: Only key used
        map.forEach((key, _) -> System.out.println("Key: " + key));
        
        // Or: Only value used
        map.forEach((_, value) -> System.out.println("Value: " + value));
        
        // Or: Neither used
        map.forEach((_, _) -> System.out.println("Entry"));
        
        /**
         * MULTIPLE PARAMETERS:
         * 
         * (key, _) -> ...     // Use key, ignore value
         * (_, value) -> ...   // Ignore key, use value
         * (_, _) -> ...       // Ignore both
         * 
         * Can mix named and unnamed:
         * (name, _, _) -> ... // Use first, ignore rest
         */
    }
    
    /**
     * BiFunction and other functional interfaces
     */
    public static void functionalInterfaces() {
        System.out.println("\n=== Functional Interfaces ===");
        
        // BiFunction - only need one parameter
        BiFunction<String, Integer, String> func1 = 
            (str, _) -> str.toUpperCase();
        
        System.out.println(func1.apply("hello", 42));
        
        // BiConsumer - only need one parameter
        BiConsumer<String, Integer> consumer = 
            (str, _) -> System.out.println("String: " + str);
        
        consumer.accept("test", 100);
        
        // Runnable - no parameters, but showing pattern
        Runnable task = () -> System.out.println("Running");
        task.run();
        
        /**
         * COMMON PATTERNS:
         * 
         * BiFunction<T, U, R>:
         * (t, _) -> process(t)
         * 
         * BiConsumer<T, U>:
         * (t, _) -> consume(t)
         * 
         * BiPredicate<T, U>:
         * (t, _) -> test(t)
         * 
         * Comparator<T>:
         * (t1, t2) -> 0  // All equal
         * Could be: (_, _) -> 0
         */
    }
    
    /**
     * Stream operations
     */
    public static void streamOperations() {
        System.out.println("\n=== Stream Operations ===");
        
        List<String> items = List.of("A", "B", "C", "D", "E");
        
        // Map with index - only need index
        IntStream.range(0, items.size())
            .forEach(i -> System.out.println("Index: " + i));
        
        // Or with items - only need item
        items.stream()
            .forEach(item -> System.out.println("Item: " + item));
        
        // Reduce - complex accumulator
        int sum = IntStream.range(1, 6)
            .reduce(0, (acc, val) -> acc + val);
        
        // If only need accumulator, not current value:
        // (acc, _) -> acc + 1  // Count items
        
        System.out.println("Sum: " + sum);
        
        /**
         * STREAM PATTERNS:
         * 
         * .map(_ -> constantValue)
         * Fixed mapping
         * 
         * .filter(_ -> true/false)
         * Fixed filter (rare)
         * 
         * .forEach(_ -> action())
         * Side effect only
         * 
         * .reduce(identity, (acc, _) -> acc + 1)
         * Count elements
         */
    }
    
    /**
     * Event handlers
     */
    public static void eventHandlers() {
        System.out.println("\n=== Event Handlers ===");
        
        // Simulated button click handler
        handleClick(_ -> System.out.println("Button clicked!"));
        
        // Timer callback
        scheduleTask(1000, _ -> System.out.println("Timer fired"));
        
        /**
         * EVENT HANDLER PATTERN:
         * 
         * button.onClick(_ -> handleClick())
         * timer.schedule(_ -> doWork())
         * stream.subscribe(_ -> process())
         * 
         * Common when:
         * - Event object not needed
         * - Just want to trigger action
         * - Callback for side effects
         */
    }
    
    private static void handleClick(Consumer<Void> handler) {
        handler.accept(null);
    }
    
    private static void scheduleTask(long delay, Consumer<Long> task) {
        task.accept(delay);
    }
    
    /**
     * Builder pattern with callbacks
     */
    public static void builderPattern() {
        System.out.println("\n=== Builder Pattern ===");
        
        // Builder with configuration callbacks
        new RequestBuilder()
            .onSuccess(_ -> System.out.println("Success!"))
            .onError(_ -> System.out.println("Error!"))
            .build();
        
        /**
         * BUILDER CALLBACKS:
         * 
         * builder
         *   .onSuccess(_ -> handleSuccess())
         *   .onError(_ -> handleError())
         *   .onComplete(_ -> cleanup())
         * 
         * Common when callbacks don't need details:
         * - Just trigger action
         * - Log event
         * - Update UI
         */
    }
    
    static class RequestBuilder {
        RequestBuilder onSuccess(Consumer<?> callback) {
            callback.accept(null);
            return this;
        }
        
        RequestBuilder onError(Consumer<?> callback) {
            return this;
        }
        
        void build() {}
    }
    
    /**
     * Comparison with method references
     */
    public static void comparisonWithMethodReferences() {
        /**
         * LAMBDA VS METHOD REFERENCE:
         * 
         * When parameter is unused:
         * 
         * Option 1: Unnamed lambda
         * list.forEach(_ -> System.out.println("Item"))
         * 
         * Option 2: Method reference
         * list.forEach(System.out::println)
         * 
         * Option 3: Custom method reference
         * list.forEach(MyClass::process)
         * 
         * 
         * Use unnamed lambda when:
         * - Simple inline logic
         * - Side effect only
         * - No suitable method exists
         * 
         * Use method reference when:
         * - Method already exists
         * - Better readability
         * - Reusable logic
         */
    }
    
    public static void main(String[] args) {
        basicLambda();
        multipleLambdaParameters();
        functionalInterfaces();
        streamOperations();
        eventHandlers();
        builderPattern();
    }
}
```

## 4. Exception Parameters

### Cleaner Exception Handling

```java
/**
 * EXCEPTION PARAMETERS
 * 
 * Using _ in catch blocks
 */

import java.io.*;
import java.sql.*;

public class ExceptionParameters {
    
    /**
     * Basic exception handling
     */
    public static void basicExceptionHandling() {
        System.out.println("=== Basic Exception Handling ===");
        
        // Before: Must name exception we don't use
        try {
            int result = Integer.parseInt("not a number");
        } catch (NumberFormatException e) {
            System.out.println("Invalid number");
            // 'e' is never used
        }
        
        // After: Use underscore
        try {
            int result = Integer.parseInt("not a number");
        } catch (NumberFormatException _) {
            System.out.println("Invalid number");
            // Clear: exception caught but not used
        }
        
        /**
         * BENEFIT:
         * 
         * catch (ExceptionType _)
         * 
         * Shows we catch but don't use exception
         * Common when:
         * - Just want to suppress
         * - Log fixed message
         * - Don't need exception details
         */
    }
    
    /**
     * Multiple catch blocks
     */
    public static void multipleCatchBlocks() {
        System.out.println("\n=== Multiple Catch Blocks ===");
        
        try {
            performOperation();
        } catch (IOException _) {
            System.out.println("I/O error occurred");
        } catch (SQLException _) {
            System.out.println("Database error occurred");
        } catch (Exception e) {
            // This one we need - log details
            System.out.println("Unexpected error: " + e.getMessage());
        }
        
        /**
         * MIXED APPROACH:
         * 
         * catch (IOException _) { ... }      // Don't need details
         * catch (SQLException _) { ... }     // Don't need details
         * catch (Exception e) { ... }        // Need details
         * 
         * Clear intent:
         * - Some exceptions: just catch
         * - Others: need exception object
         */
    }
    
    private static void performOperation() throws IOException, SQLException {
        // Simulated operation
    }
    
    /**
     * Swallowing exceptions intentionally
     */
    public static void swallowingExceptions() {
        System.out.println("\n=== Swallowing Exceptions ===");
        
        // Before: Unclear if intentional
        try {
            riskyOperation();
        } catch (Exception ignored) {
            // Empty - is this intentional?
        }
        
        // After: Clear intent
        try {
            riskyOperation();
        } catch (Exception _) {
            // Empty - clearly intentional
        }
        
        /**
         * INTENTIONAL SWALLOWING:
         * 
         * catch (Exception _) {
         *   // Intentionally empty
         * }
         * 
         * Use when:
         * - Exception is truly expected and harmless
         * - Best-effort operation
         * - Cleanup code
         * 
         * Still add comment explaining why!
         * 
         * Better than:
         * catch (Exception ignored) { }
         * catch (Exception e) { }  // Looks like forgot to handle
         */
    }
    
    private static void riskyOperation() throws Exception {
        if (Math.random() > 0.5) {
            throw new Exception("Random error");
        }
    }
    
    /**
     * Multi-catch with underscore
     */
    public static void multiCatch() {
        System.out.println("\n=== Multi-Catch ===");
        
        try {
            performComplexOperation();
        } catch (IOException | SQLException _) {
            System.out.println("Operation failed");
            // Don't need exception details for either
        }
        
        /**
         * MULTI-CATCH:
         * 
         * catch (Type1 | Type2 | Type3 _) { ... }
         * 
         * Handle multiple exception types
         * All ignored with single _
         */
    }
    
    private static void performComplexOperation() 
            throws IOException, SQLException {
        // Simulated operation
    }
    
    /**
     * When NOT to use underscore
     */
    public static void whenNotToUse() {
        /**
         * DON'T USE _ IF YOU NEED:
         * 
         * 1. Exception message:
         * catch (Exception e) {
         *   log.error("Failed: " + e.getMessage());
         * }
         * 
         * 2. Exception type:
         * catch (Exception e) {
         *   if (e instanceof SpecificException) { ... }
         * }
         * 
         * 3. Stack trace:
         * catch (Exception e) {
         *   e.printStackTrace();
         * }
         * 
         * 4. Rethrowing:
         * catch (Exception e) {
         *   throw new WrapperException(e);
         * }
         * 
         * 5. Logging:
         * catch (Exception e) {
         *   logger.error("Error", e);
         * }
         * 
         * 
         * USE _ WHEN:
         * ✓ Just catching to suppress
         * ✓ Fixed error message
         * ✓ Best-effort code
         * ✓ Cleanup operations
         */
    }
    
    public static void main(String[] args) {
```

```java
/**
 * TRY-WITH-RESOURCES
 * 
 * Using _ with AutoCloseable resources
 */

import java.io.*;
import java.util.concurrent.locks.*;

public class TryWithResources {
    
    /**
     * Basic try-with-resources
     */
    public static void basicTryWithResources() {
        System.out.println("=== Basic Try-With-Resources ===");
        
        // Before: Must name resource we don't use
        try (BufferedReader reader = new BufferedReader(
                new StringReader("test"))) {
            // Only care that resource is closed
            System.out.println("Resource will be closed");
            // 'reader' is never used
        } catch (IOException e) {
            e.printStackTrace();
        }
        
        // After: Use underscore
        try (BufferedReader _ = new BufferedReader(
                new StringReader("test"))) {
            System.out.println("Resource will be closed");
            // Clear: resource created but not used
        } catch (IOException _) {
            System.out.println("Error occurred");
        }
        
        /**
         * BENEFIT:
         * 
         * try (ResourceType _ = resource) { ... }
         * 
         * Shows we only need auto-close
         * Not the resource itself
         */
    }
    
    /**
     * Lock acquisition
     */
    public static void lockAcquisition() {
        System.out.println("\n=== Lock Acquisition ===");
        
        Lock lock = new ReentrantLock();
        
        // Acquire lock, auto-release
        try (AutoCloseableLock _ = new AutoCloseableLock(lock)) {
            System.out.println("Critical section");
            // Lock auto-released at end
        }
        
        /**
         * LOCK PATTERN:
         * 
         * try (AutoCloseableLock _ = acquire(lock)) {
         *   // Critical section
         * }
         * 
         * Don't need lock object in body
         * Just need it acquired and released
         */
    }
    
    static class AutoCloseableLock implements AutoCloseable {
        private final Lock lock;
        
        AutoCloseableLock(Lock lock) {
            this.lock = lock;
            lock.lock();
        }
        
        @Override
        public void close() {
            lock.unlock();
        }
    }
    
    /**
     * Multiple resources
     */
    public static void multipleResources() throws IOException {
        System.out.println("\n=== Multiple Resources ===");
        
        // Multiple resources, some used, some not
        try (
            BufferedReader reader = new BufferedReader(
                new StringReader("line1\nline2")
            );
            BufferedWriter _ = new BufferedWriter(
                new StringWriter()
            )
        ) {
            // Use reader, ignore writer
            String line = reader.readLine();
            System.out.println("Read: " + line);
        }
        
        /**
         * MIXED RESOURCES:
         * 
         * try (
         *   Resource1 res1 = ...,  // Use this
         *   Resource2 _ = ...,     // Don't use
         *   Resource3 res3 = ...   // Use this
         * ) { ... }
         * 
         * Clear which resources are used
         */
    }
    
    /**
     * Scoped cleanup
     */
    public static void scopedCleanup() {
        System.out.println("\n=== Scoped Cleanup ===");
        
        // Custom cleanup action
        try (CleanupAction _ = new CleanupAction(() -> 
                System.out.println("Cleanup executed"))) {
            System.out.println("Doing work...");
            // Cleanup runs automatically
        }
        
        /**
         * CLEANUP PATTERN:
         * 
         * try (CleanupAction _ = cleanup(() -> { ... })) {
         *   // Do work
         * }
         * // Cleanup runs
         * 
         * Ensures cleanup without resource reference
         */
    }
    
    static class CleanupAction implements AutoCloseable {
        private final Runnable cleanup;
        
        CleanupAction(Runnable cleanup) {
            this.cleanup = cleanup;
        }
        
        @Override
        public void close() {
            cleanup.run();
        }
    }
    
    /**
     * Database connection pattern
     */
    public static void databasePattern() {
        /**
         * DATABASE CONNECTIONS:
         * 
         * Before:
         * try (Connection conn = getConnection()) {
         *   // Just need transaction boundary
         *   executeStatements();
         *   // conn never used directly
         * }
         * 
         * After:
         * try (Connection _ = getConnection()) {
         *   executeStatements();
         * }
         * 
         * 
         * TRANSACTION PATTERN:
         * 
         * try (Transaction _ = beginTransaction()) {
         *   doWork();
         * }
         * // Auto-commit or rollback
         */
    }
    
    /**
     * Real-world example: Scoped metrics
     */
    public static void scopedMetrics() {
        System.out.println("\n=== Scoped Metrics ===");
        
        // Timer that records duration on close
        try (Timer _ = startTimer("operation")) {
            // Do operation
            Thread.sleep(100);
        } catch (InterruptedException _) {
            Thread.currentThread().interrupt();
        }
        
        /**
         * METRICS PATTERN:
         * 
         * try (Timer _ = startTimer("operation")) {
         *   doWork();
         * }
         * // Duration automatically recorded
         * 
         * try (Span _ = tracer.startSpan("operation")) {
         *   doWork();
         * }
         * // Span automatically finished
         * 
         * Clean scoped instrumentation!
         */
    }
    
    static class Timer implements AutoCloseable {
        private final String name;
        private final long start;
        
        Timer(String name) {
            this.name = name;
            this.start = System.nanoTime();
        }
        
        @Override
        public void close() {
            long duration = System.nanoTime() - start;
            System.out.println(name + " took " + 
                duration / 1_000_000 + "ms");
        }
    }
    
    private static Timer startTimer(String name) {
        return new Timer(name);
    }
    
    public static void main(String[] args) {
        basicTryWithResources();
        lockAcquisition();
        scopedCleanup();
        scopedMetrics();
    }
}
```

```java
/**
 * USE CASES FOR READABILITY
 * 
 * When and how to use unnamed variables
 */

import java.util.*;
import java.util.stream.*;

public class ReadabilityUseCases {
    
    /**
     * Side-effect only operations
     */
    public static void sideEffectOperations() {
        System.out.println("=== Side-Effect Operations ===");
        
        List<String> items = List.of("A", "B", "C");
        
        // Just want to execute action, not use parameter
        items.forEach(_ -> System.out.println("Processing item"));
        
        // Counting
        AtomicInteger counter = new AtomicInteger();
        items.forEach(_ -> counter.incrementAndGet());
        System.out.println("Count: " + counter.get());
        
        /**
         * SIDE-EFFECT PATTERN:
         * 
         * collection.forEach(_ -> action())
         * 
         * Clear: We're not using the elements
         * Just executing action for each
         */
    }
    
    /**
     * API compliance
     */
    public static void apiCompliance() {
        /**
         * API REQUIRES RETURN VALUE:
         * 
         * Sometimes API requires method return value
         * but you don't need it:
         * 
         * Before:
         * SomeType result = builder.build();
         * // 'result' never used
         * 
         * After:
         * SomeType _ = builder.build();
         * // Clear: called for side effect
         * 
         * 
         * EXAMPLE:
         * 
         * List _ = new ArrayList<>();  // Create for GC
         * Thread _ = new Thread(task).start();  // Just start
         * boolean _ = collection.add(item);  // Don't care if added
         */
    }
    
    /**
     * Destructuring with partial interest
     */
    public static void destructuringPartialInterest() {
        System.out.println("\n=== Destructuring ===");
        
        record RGB(int red, int green, int blue) {}
        record Color(String name, RGB rgb) {}
        
        Color color = new Color("Sky Blue", new RGB(135, 206, 235));
        
        // Only need name
        if (color instanceof Color(String name, RGB _)) {
            System.out.println("Color: " + name);
        }
        
        // Only need specific RGB component
        if (color instanceof Color(_, RGB(int r, int _, int _))) {
            System.out.println("Red: " + r);
        }
        
        /**
         * PARTIAL EXTRACTION:
         * 
         * Extract only what you need:
         * - Name from Color, ignore RGB
         * - One RGB component, ignore others
         * 
         * Makes code focus on what matters
         */
    }
    
    /**
     * Loop counters you don't reference
     */
    public static void loopCounters() {
        System.out.println("\n=== Loop Counters ===");
        
        // Fixed repetition
        List<String> items = List.of("A", "B", "C");
        
        // Repeat 3 times - don't care about element
        for (String _ : items) {
            System.out.println("Action");
        }
        
        // Or with IntStream
        IntStream.range(0, 3).forEach(_ -> 
            System.out.println("Action"));
        
        /**
         * REPETITION PATTERN:
         * 
         * for (Type _ : collection) { ... }
         * 
         * Shows we're using collection for count
         * Not the elements
         * 
         * Alternative:
         * repeat(3, () -> action());
         */
    }
    
    /**
     * Callback parameters
     */
    public static void callbackParameters() {
        System.out.println("\n=== Callback Parameters ===");
        
        // Event callback - don't need event details
        Button button = new Button();
        button.onClick(_ -> System.out.println("Clicked!"));
        
        // Timer callback - don't need time
        Timer timer = new Timer();
        timer.schedule(1000, _ -> System.out.println("Tick"));
        
        /**
         * CALLBACK PATTERN:
         * 
         * button.onClick(_ -> handleClick())
         * observable.subscribe(_ -> update())
         * future.thenAccept(_ -> cleanup())
         * 
         * Clear: callback triggered but parameter unused
         */
    }

        void onClick(Consumer<?> handler) {
            handler.accept(null);
        }
    }
    
    static class Timer {

            task.accept(delay);
        }
    }
    
    /**
     * Map/reduce with constant

        List<String> items = List.of("A", "B", "C", "D");
        
        // Map everything to constant

            .map(_ -> 1)
            .toList();
        System.out.println("Ones: " + ones);
        
        // Count with reduce
        int count = items.stream()
            .reduce(0, (acc, _) -> acc + 1, Integer::sum);
        System.out.println("Count: " + count);
        
        /**
         * CONSTANT MAPPING:
         * 
         * .map(_ -> constant)
         * 
         * Transform all elements to same value
         * Clear: not using original values
         * 
         * .reduce(0, (acc, _) -> acc + 1)
         * 
         * Count elements
         * Don't need element values
         */
    }
    
    /**
     * Comparison: Before vs After

    public static void beforeAfterComparison() {
```

## Summary

### Quick Reference

**Local Variables:**

```java
String key = entry.getKey();
Integer _ = entry.getValue();  // Unused
```

**Lambda Parameters:**

```java
list.forEach(_ -> action());           // Unused parameter
map.forEach((key, _) -> use(key));    // Unused value
map.forEach((_, _) -> action());      // Both unused
```

**Exception Parameters:**

```java
catch (IOException _) { ... }          // Unused exception
catch (IOException | SQLException _)   // Multi-catch
```

**Try-With-Resources:**

```java
try (Resource _ = resource) { ... }    // Unused resource
```

**Pattern Matching:**

```java
case Point(int x, int _) -> ...        // Unused y coordinate
case Circle(Point _, double r) -> ...  // Unused center
case Object _ -> ...                   // Unused matched object
```

---


**Key insights:**

- **Clear intent**: Shows value exists but intentionally unused
- **No noise**: Eliminates meaningless names like "ignored"
- **Cannot reference**: Compiler enforces non-use
- **Multiple contexts**: Works everywhere
- **Better readability**: Code is self-documenting

**The revolution:**

```
Before:
catch (IOException ignored) { }
for (String item : items) { count++; }
map.forEach((k, v) -> action());

After:
catch (IOException _) { }
for (String _ : items) { count++; }
map.forEach((_, _) -> action());

From noise to clarity
From ambiguous to explicit
From "is this a bug?" to "intentionally unused"
```
