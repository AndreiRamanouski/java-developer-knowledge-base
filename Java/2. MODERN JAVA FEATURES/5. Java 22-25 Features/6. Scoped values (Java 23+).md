# Scoped Values (Java 23+)

## Overview

Scoped Values, finalized in Java 23 (JEP 464), provide a modern replacement for ThreadLocal that is designed for immutable context sharing in concurrent applications, especially those using virtual threads. Before Scoped Values, developers used ThreadLocal to propagate context like user identity, transaction ID, or request metadata through a call stack. However, ThreadLocal has fundamental problems: it requires explicit cleanup (or memory leaks), it's mutable (leading to bugs), it doesn't compose well with nested operations, and it scales poorly with millions of virtual threads. Scoped Values solve these issues by providing immutable, bounded context that is automatically propagated to child tasks, requires no cleanup, and is optimized for virtual threads. A ScopedValue is bound for the lifetime of a specific code block, automatically inherited by child threads (including virtual threads and StructuredTaskScope subtasks), and cleaned up when the scope exits. This guide covers the problems with ThreadLocal, ScopedValue basics, immutability guarantees, per-thread per-scope semantics, performance characteristics, inheritance rules with structured concurrency, common use cases like user context and transaction tracking, and how Scoped Values integrate perfectly with virtual threads to enable millions of concurrent operations without memory overhead.

## 1. The Problem with ThreadLocal

### Why ThreadLocal Doesn't Work for Modern Java

```java
/**
 * THREADLOCAL PROBLEMS
 * 
 * Issues with traditional ThreadLocal
 */

import java.util.concurrent.*;

public class ThreadLocalProblems {
    
    /**
     * Problem 1: Memory leaks
     */
    public static void memoryLeaks() {
        ThreadLocal<String> context = new ThreadLocal<>();
        
        ExecutorService executor = Executors.newCachedThreadPool();
        
        for (int i = 0; i < 10; i++) {
            final int taskId = i;
            executor.submit(() -> {
                // Set ThreadLocal
                context.set("Task " + taskId);
                
                // Do work
                System.out.println("Processing: " + context.get());
                
                // ❌ Forgot to remove!
                // context.remove();
                
                // ThreadLocal data stays in thread!
                // If thread is reused, old data persists!
            });
        }
        
        executor.shutdown();
        
        /**
         * PROBLEM:
         * 
         * ✗ Must call remove() manually
         * ✗ Easy to forget
         * ✗ Memory leak if forgotten
         * ✗ Thread pool reuse → stale data
         * ✗ Hard to track lifecycle
         */
    }
    
    /**
     * Problem 2: Mutability
     */
    public static void mutability() {
        ThreadLocal<StringBuilder> buffer = new ThreadLocal<>();
        
        ExecutorService executor = Executors.newSingleThreadExecutor();
        
        try {
            executor.submit(() -> {
                buffer.set(new StringBuilder("Initial"));
                
                // Somewhere deep in call stack
                doSomething(buffer);
                
                // ❌ Value changed! Was it intentional?
                System.out.println("After: " + buffer.get());
                
            }).get();
            
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            executor.shutdown();
        }
        
        /**
         * PROBLEM:
         * 
         * ✗ ThreadLocal values are mutable
         * ✗ Any code can modify them
         * ✗ Hard to track changes
         * ✗ Race conditions possible
         * ✗ Debugging nightmare
         */
    }
    
    private static void doSomething(ThreadLocal<StringBuilder> buffer) {
        // Someone modifies the shared buffer
        buffer.get().append(" - Modified!");
    }
    
    /**
     * Problem 3: Poor composability
     */
    public static void poorComposability() {
        ThreadLocal<String> outer = new ThreadLocal<>();
        ThreadLocal<String> inner = new ThreadLocal<>();
        
        try {
            outer.set("Outer context");
            
            // Nested operation
            try {
                inner.set("Inner context");
                
                // ❌ No clear nesting relationship
                // ❌ Must manage both independently
                // ❌ What if we forget to clear inner?
                
                System.out.println("Outer: " + outer.get());
                System.out.println("Inner: " + inner.get());
                
            } finally {
                inner.remove();  // Manual cleanup
            }
            
        } finally {
            outer.remove();  // Manual cleanup
        }
        
        /**
         * PROBLEM:
         * 
         * ✗ No automatic nesting
         * ✗ Manual management required
         * ✗ Easy to mess up order
         * ✗ Verbose try-finally blocks
         * ✗ Not composable
         */
    }
    
    /**
     * Problem 4: Virtual threads scalability
     */
    public static void virtualThreadsScalability() throws InterruptedException {
        ThreadLocal<byte[]> largeContext = new ThreadLocal<>();
        
        // With platform threads: 1000 threads = manageable
        // With virtual threads: 1,000,000 threads = disaster!
        
        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 1_000_000; i++) {
                executor.submit(() -> {
                    // Each thread gets 1KB of data
                    largeContext.set(new byte[1024]);
                    
                    // 1 million threads × 1KB = 1GB just for ThreadLocal!
                    // Plus overhead for ThreadLocal itself
                    // Memory explosion!
                    
                    try {
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    } finally {
                        largeContext.remove();
                    }
                });
            }
        }
        
        /**
         * PROBLEM:
         * 
         * ✗ One ThreadLocal per thread
         * ✗ Millions of virtual threads = millions of copies
         * ✗ Memory explosion
         * ✗ GC pressure
         * ✗ Doesn't scale
         * 
         * Virtual threads need better solution!
         */
    }
    
    /**
     * Problem 5: No inheritance control
     */
    public static void noInheritanceControl() {
        ThreadLocal<String> context = new ThreadLocal<>();
        context.set("Parent context");
        
        // Child thread
        Thread child = Thread.ofPlatform().unstarted(() -> {
            // ❌ ThreadLocal NOT inherited by default
            String value = context.get();  // null!
            System.out.println("Child sees: " + value);
        });
        
        child.start();
        
        try {
            child.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        /**
         * PROBLEM:
         * 
         * ✗ ThreadLocal not inherited
         * ✗ InheritableThreadLocal has issues:
         *   - Copies at thread creation
         *   - Not updated if parent changes
         *   - Still requires cleanup
         *   - Still mutable
         * ✗ No good solution
         */
    }
    
    /**
     * What we want
     */
    public static void whatWeWant() {
        /**
         * DESIRED PROPERTIES:
         * 
         * 1. Immutable:
         *    - Once set, cannot change
         *    - No accidental modifications
         *    - Safe to share
         * 
         * 2. Scoped:
         *    - Bound for specific code block
         *    - Automatic cleanup
         *    - Clear lifetime
         * 
         * 3. Inheritable:
         *    - Automatically passed to child tasks
         *    - Works with StructuredTaskScope
         *    - No manual propagation
         * 
         * 4. Scalable:
         *    - Works with millions of virtual threads
         *    - Low memory overhead
         *    - Efficient sharing
         * 
         * 5. Composable:
         *    - Natural nesting
         *    - Rebinding support
         *    - Clean syntax
         * 
         * 
         * ScopedValue provides all of this!
         */
    }
    
    public static void main(String[] args) throws InterruptedException {
        System.out.println("=== ThreadLocal Problems ===");
        memoryLeaks();
        mutability();
        poorComposability();
        // virtualThreadsScalability();  // Commented - uses too much memory
    }
}
```

## 2. ScopedValue Basics

### Modern Context Propagation

```java
/**
 * SCOPEDVALUE BASICS
 * 
 * Core concepts and usage
 */

import java.util.concurrent.*;

public class ScopedValueBasics {
    
    // Define ScopedValue (like ThreadLocal but better)
    private static final ScopedValue<String> USER_ID = ScopedValue.newInstance();
    private static final ScopedValue<String> REQUEST_ID = ScopedValue.newInstance();
    
    /**
     * Basic usage
     */
    public static void basicUsage() {
        System.out.println("=== Basic Usage ===");
        
        // Bind value for a scope
        ScopedValue.where(USER_ID, "user-123").run(() -> {
            // Inside scope, value is available
            String userId = USER_ID.get();
            System.out.println("User ID: " + userId);
            
            doWork();
            
        }); // Scope ends, binding removed automatically
        
        // Outside scope, value not available
        try {
            String userId = USER_ID.get();  // Throws!
        } catch (NoSuchElementException e) {
            System.out.println("Outside scope: " + e.getMessage());
        }
        
        /**
         * KEY CONCEPTS:
         * 
         * 1. ScopedValue.newInstance():
         *    - Creates a ScopedValue
         *    - Type-safe
         *    - Typically static final
         * 
         * 2. ScopedValue.where(...).run(...):
         *    - Binds value for scope
         *    - Executes code
         *    - Automatic cleanup
         * 
         * 3. get():
         *    - Retrieves current value
         *    - Throws if not bound
         *    - No null checks needed
         */
    }
    
    private static void doWork() {
        // Can access USER_ID anywhere in call stack
        String userId = USER_ID.get();
        System.out.println("  doWork sees: " + userId);
    }
    
    /**
     * Multiple bindings
     */
    public static void multipleBindings() {
        System.out.println("\n=== Multiple Bindings ===");
        
        ScopedValue.where(USER_ID, "user-456")
                   .where(REQUEST_ID, "req-789")
                   .run(() -> {
            
            String userId = USER_ID.get();
            String requestId = REQUEST_ID.get();
            
            System.out.println("User: " + userId);
            System.out.println("Request: " + requestId);
        });
        
        /**
         * MULTIPLE BINDINGS:
         * 
         * Chain with .where()
         * All bound for same scope
         * All cleaned up together
         * Clean and readable!
         */
    }
    
    /**
     * Nested scopes
     */
    public static void nestedScopes() {
        System.out.println("\n=== Nested Scopes ===");
        
        ScopedValue.where(USER_ID, "outer-user").run(() -> {
            System.out.println("Outer: " + USER_ID.get());
            
            // Inner scope can rebind
            ScopedValue.where(USER_ID, "inner-user").run(() -> {
                System.out.println("Inner: " + USER_ID.get());
            });
            
            // Outer scope restored
            System.out.println("After inner: " + USER_ID.get());
        });
        
        /**
         * NESTED SCOPES:
         * 
         * ✓ Inner scope can rebind same ScopedValue
         * ✓ Inner value shadows outer
         * ✓ Outer value restored when inner exits
         * ✓ Automatic scope management
         * ✓ Clean nesting
         */
    }
    
    /**
     * With return value
     */
    public static void withReturnValue() {
        System.out.println("\n=== With Return Value ===");
        
        String result = ScopedValue.where(USER_ID, "user-999").call(() -> {
            // Do work and return result
            return "Processed for " + USER_ID.get();
        });
        
        System.out.println("Result: " + result);
        
        /**
         * RETURN VALUES:
         * 
         * run(): void
         * call(): returns value
         * 
         * Both bind scope and clean up automatically
         */
    }
    
    /**
     * isBound() check
     */
    public static void isBoundCheck() {
        System.out.println("\n=== isBound Check ===");
        
        // Check if bound
        if (USER_ID.isBound()) {
            System.out.println("User: " + USER_ID.get());
        } else {
            System.out.println("No user context");
        }
        
        ScopedValue.where(USER_ID, "user-111").run(() -> {
            if (USER_ID.isBound()) {
                System.out.println("User: " + USER_ID.get());
            }
        });
        
        /**
         * isBound():
         * 
         * - Check if value is bound
         * - Avoid NoSuchElementException
         * - Useful for optional context
         */
    }
    
    /**
     * orElse() for defaults
     */
    public static void orElseDefaults() {
        System.out.println("\n=== orElse Defaults ===");
        
        // Not bound - use default
        String user1 = USER_ID.orElse("anonymous");
        System.out.println("User 1: " + user1);
        
        // Bound - use bound value
        ScopedValue.where(USER_ID, "user-222").run(() -> {
            String user2 = USER_ID.orElse("anonymous");
            System.out.println("User 2: " + user2);
        });
        
        /**
         * orElse(default):
         * 
         * - Returns bound value if bound
         * - Returns default if not bound
         * - No exception
         * - Convenient for optional context
         */
    }
    
    /**
     * Type safety
     */
    public static void typeSafety() {
        System.out.println("\n=== Type Safety ===");
        
        // Type-safe at compile time
        ScopedValue<Integer> counter = ScopedValue.newInstance();
        
        ScopedValue.where(counter, 42).run(() -> {
            Integer value = counter.get();  // Type-safe!
            System.out.println("Counter: " + value);
            
            // int primitive = counter.get();  // Works (auto-unboxing)
        });
        
        /**
         * TYPE SAFETY:
         * 
         * ✓ Generic type parameter
         * ✓ Compile-time checking
         * ✓ No casts needed
         * ✓ IDE support
         */
    }
    
    public static void main(String[] args) {
        basicUsage();
        multipleBindings();
        nestedScopes();
        withReturnValue();
        isBoundCheck();
        orElseDefaults();
        typeSafety();
    }
}
```

## 4. Immutable Context Propagation

### Safety Through Immutability

```java
/**
 * IMMUTABLE CONTEXT PROPAGATION
 * 
 * How immutability ensures safety
 */

import java.util.*;
import java.util.concurrent.*;

public class ImmutableContextPropagation {
    
    private static final ScopedValue<String> USER_ID = ScopedValue.newInstance();
    private static final ScopedValue<UserContext> USER_CONTEXT = ScopedValue.newInstance();
    
    /**
     * Immutability guarantee
     */
    public static void immutabilityGuarantee() {
        System.out.println("=== Immutability Guarantee ===");
        
        ScopedValue.where(USER_ID, "user-123").run(() -> {
            String userId = USER_ID.get();
            System.out.println("User ID: " + userId);
            
            // ✓ Cannot modify!
            // USER_ID.set("different");  // Doesn't compile!
            
            // ✓ Can only rebind in nested scope
            ScopedValue.where(USER_ID, "user-456").run(() -> {
                System.out.println("Nested: " + USER_ID.get());
            });
            
            // Original value restored
            System.out.println("After: " + USER_ID.get());
        });
        
        /**
         * IMMUTABILITY:
         * 
         * ✓ No set() method
         * ✓ Only bind() for new scope
         * ✓ Cannot accidentally modify
         * ✓ Safe to share
         * ✓ No race conditions
         */
    }
    
    /**
     * Immutable objects
     */
    public static void immutableObjects() {
        System.out.println("\n=== Immutable Objects ===");
        
        record UserContext(String userId, String role, List<String> permissions) {
            // Immutable record
            public UserContext {
                permissions = List.copyOf(permissions);  // Defensive copy
            }
        }
        
        UserContext ctx = new UserContext(
            "user-123",
            "admin",
            List.of("read", "write", "delete")
        );
        
        ScopedValue.where(USER_CONTEXT, ctx).run(() -> {
            UserContext current = USER_CONTEXT.get();
            System.out.println("User: " + current.userId());
            System.out.println("Role: " + current.role());
            System.out.println("Permissions: " + current.permissions());
            
            // ✓ Record is immutable
            // current.userId = "different";  // Doesn't compile!
            
            // ✓ List is immutable
            try {
                current.permissions().add("execute");  // Throws!
            } catch (UnsupportedOperationException e) {
                System.out.println("Cannot modify: " + e.getClass().getSimpleName());
            }
        });
        
        /**
         * BEST PRACTICE:
         * 
         * Use immutable objects:
         * ✓ Records
         * ✓ Immutable collections
         * ✓ Final fields
         * ✓ No setters
         * 
         * ScopedValue + immutable objects = perfectly safe!
         */
    }
    
    record UserContext(String userId, String role, List<String> permissions) {
        public UserContext {
            permissions = List.copyOf(permissions);
        }
    }
    
    /**
     * Deep propagation
     */
    public static void deepPropagation() {
        System.out.println("\n=== Deep Propagation ===");
        
        ScopedValue.where(USER_ID, "user-789").run(() -> {
            System.out.println("Level 1: " + USER_ID.get());
            
            level2();
        });
    }
    
    private static void level2() {
        System.out.println("Level 2: " + USER_ID.get());
        level3();
    }
    
    private static void level3() {
        System.out.println("Level 3: " + USER_ID.get());
        
        /**
         * DEEP PROPAGATION:
         * 
         * ✓ Available at any depth
         * ✓ No manual passing
         * ✓ Immutable throughout
         * ✓ No hidden dependencies
         */
    }
    
    /**
     * Safe sharing across threads
     */
    public static void safeSharingAcrossThreads() throws InterruptedException, ExecutionException {
        System.out.println("\n=== Safe Sharing ===");
        
        ScopedValue.where(USER_ID, "user-shared").run(() -> {
            
            try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
                
                // Fork multiple tasks
                var task1 = scope.fork(() -> {
                    // USER_ID inherited and immutable
                    return "Task 1: " + USER_ID.get();
                });
                
                var task2 = scope.fork(() -> {
                    return "Task 2: " + USER_ID.get();
                });
                
                var task3 = scope.fork(() -> {
                    return "Task 3: " + USER_ID.get();
                });
                
                scope.join();
                scope.throwIfFailed();
                
                System.out.println(task1.get());
                System.out.println(task2.get());
                System.out.println(task3.get());
                
                /**
```
