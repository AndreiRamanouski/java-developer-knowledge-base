# Record Patterns (Java 19-21)

## Overview

Record patterns, introduced as a preview in Java 19 and finalized in Java 21, enable deconstructing record values inline during pattern matching. Instead of manually calling accessor methods to extract record components, record patterns allow direct extraction using a syntax that mirrors the record declaration. This works in both switch expressions/statements and instanceof checks, making code more concise and readable. Nested record patterns enable deconstructing records that contain other records in a single pattern, eliminating multiple levels of accessor calls. The feature integrates seamlessly with type patterns and guards, enabling sophisticated data processing. Record patterns shine when working with immutable data structures, algebraic data types (ADTs), tree traversal, and data transformation pipelines. This guide covers basic record deconstruction, nested patterns for hierarchical data, instanceof with record patterns, performance characteristics, and extensive real-world examples including JSON processing, tree algorithms, geometric calculations, and functional programming patterns.

## 1. Deconstructing Records

### Basic Record Pattern Syntax

```java
/**
 * DECONSTRUCTING RECORDS
 * 
 * Extracting record components with patterns
 */

public class DeconstructingRecords {
    
    /**
     * Before record patterns
     */
    public static void beforeRecordPatterns() {
        System.out.println("=== Before Record Patterns ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        // Manual deconstruction
        if (obj instanceof Point) {
            Point p = (Point) obj;
            int x = p.x();
            int y = p.y();
            System.out.println("Point at (" + x + ", " + y + ")");
        }
        
        /**
         * PROBLEMS:
         * 
         * - Manual casting
         * - Multiple accessor calls
         * - Verbose (4+ lines)
         * - Temporary variables
         * - Redundant code
         */
    }
    
    /**
     * After record patterns
     */
    public static void afterRecordPatterns() {
        System.out.println("\n=== After Record Patterns ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        // Record pattern deconstruction
        if (obj instanceof Point(int x, int y)) {
            System.out.println("Point at (" + x + ", " + y + ")");
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ One line
         * ✓ Direct extraction
         * ✓ No accessor calls
         * ✓ Clear and concise
         * ✓ Pattern mirrors record
         */
    }
    
    /**
     * Record pattern syntax
     */
    public static void recordPatternSyntax() {
        System.out.println("\n=== Record Pattern Syntax ===");
        
        record Person(String name, int age) {}
        
        Object obj = new Person("Alice", 25);
        
        // Basic pattern
        if (obj instanceof Person(String n, int a)) {
            System.out.println(n + " is " + a + " years old");
        }
        
        /**
         * SYNTAX:
         * 
         * RecordType(component1Type component1Name, 
         *            component2Type component2Name, ...)
         * 
         * Example:
         * record Point(int x, int y) {}
         * 
         * Pattern:
         * Point(int x, int y)
         * 
         * Components:
         * - x: extracted as int
         * - y: extracted as int
         * 
         * Must match:
         * - Component count
         * - Component types
         * - Component order
         */
    }
    
    /**
     * Type inference with var
     */
    public static void typeInference() {
        System.out.println("\n=== Type Inference ===");
        
        record Point(int x, int y) {}
        record Color(String name, int rgb) {}
        
        Object obj = new Point(10, 20);
        
        // Explicit types
        if (obj instanceof Point(int x, int y)) {
            System.out.println("Explicit: " + x + ", " + y);
        }
        
        // Type inference with var
        if (obj instanceof Point(var x, var y)) {
            System.out.println("Inferred: " + x + ", " + y);
            // x is int, y is int
        }
        
        obj = new Color("Red", 0xFF0000);
        
        if (obj instanceof Color(var name, var rgb)) {
            System.out.println("Color: " + name + " = " + rgb);
            // name is String, rgb is int
        }
        
        /**
         * TYPE INFERENCE:
         * 
         * Use var when types are obvious:
         * Point(var x, var y)
         * 
         * Compiler infers from record:
         * record Point(int x, int y)
         * → var x is int, var y is int
         * 
         * Benefits:
         * ✓ Less typing
         * ✓ Still type-safe
         * ✓ Cleaner code
         * 
         * Use explicit types when:
         * - Types not obvious
         * - Want documentation
         * - Complex generics
         */
    }
    
    /**
     * Wildcards for unused components
     */
    public static void wildcards() {
        System.out.println("\n=== Wildcards ===");
        
        record Person(String name, int age, String email) {}
        
        Object obj = new Person("Alice", 25, "alice@example.com");
        
        // Only need name
        if (obj instanceof Person(String name, _, _)) {
            System.out.println("Name: " + name);
        }
        
        // Only need age
        if (obj instanceof Person(_, int age, _)) {
            System.out.println("Age: " + age);
        }
        
        // Only need name and email
        if (obj instanceof Person(String name, _, String email)) {
            System.out.println("Contact: " + name + " <" + email + ">");
        }
        
        /**
         * WILDCARDS (_):
         * 
         * Ignore components you don't need:
         * Person(String name, _, _)
         * 
         * Benefits:
         * ✓ Clear intent
         * ✓ Less noise
         * ✓ Faster (no extraction)
         * 
         * Compiler still validates:
         * - Correct component count
         * - Structure matches
         * 
         * Example:
         * record Point3D(int x, int y, int z) {}
         * 
         * // Only x and y
         * Point3D(int x, int y, _)
         * 
         * // Only z
         * Point3D(_, _, int z)
         */
    }
    
    /**
     * Multiple record types
     */
    public static void multipleRecordTypes() {
        System.out.println("\n=== Multiple Record Types ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        record Rectangle(Point topLeft, Point bottomRight) {}
        
        Object[] shapes = {
            new Point(0, 0),
            new Circle(new Point(5, 5), 10),
            new Rectangle(new Point(0, 0), new Point(10, 10))
        };
        
        for (Object shape : shapes) {
            if (shape instanceof Point(int x, int y)) {
                System.out.println("Point: (" + x + ", " + y + ")");
            } else if (shape instanceof Circle(Point center, double r)) {
                System.out.println("Circle: center=" + center + ", radius=" + r);
            } else if (shape instanceof Rectangle(Point tl, Point br)) {
                System.out.println("Rectangle: " + tl + " to " + br);
            }
        }
        
        /**
         * MULTIPLE PATTERNS:
         * 
         * Can pattern match different record types:
         * 
         * if (obj instanceof Point(int x, int y)) {...}
         * else if (obj instanceof Circle(...)) {...}
         * else if (obj instanceof Rectangle(...)) {...}
         * 
         * Each pattern:
         * - Matches specific type
         * - Extracts components
         * - Different variables
         * 
         * Replaces instanceof chains
         */
    }
    
    /**
     * Record patterns in expressions
     */
    public static void recordPatternsInExpressions() {
        System.out.println("\n=== Patterns in Expressions ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        // In if expression (ternary-like)
        String description = obj instanceof Point(int x, int y)
            ? "Point at (" + x + ", " + y + ")"
            : "Not a point";
        
        System.out.println(description);
        
        // In switch expression
        String result = switch (obj) {
            case Point(int x, int y) when x == 0 && y == 0 -> 
                "Origin";
            case Point(int x, int y) when x == y -> 
                "Diagonal point";
            case Point(int x, int y) -> 
                "Point at (" + x + ", " + y + ")";
            case null -> 
                "Null";
            default -> 
                "Unknown";
        };
        
        System.out.println(result);
        
        /**
         * PATTERNS IN EXPRESSIONS:
         * 
         * Can use record patterns anywhere:
         * - if statements
         * - if expressions (with ternary)
         * - switch expressions
         * - switch statements
         * - while loops (with caution)
         * 
         * Pattern variables scoped to:
         * - True branch (if)
         * - Case body (switch)
         * - Within expression
         */
    }
    
    public static void main(String[] args) {
        beforeRecordPatterns();
        afterRecordPatterns();
        recordPatternSyntax();
        typeInference();
        wildcards();
        multipleRecordTypes();
        recordPatternsInExpressions();
    }
}
```

## 2. Nested Record Patterns

### Deconstructing Hierarchical Data

```java
/**
 * NESTED RECORD PATTERNS
 * 
 * Multi-level deconstruction
 */

public class NestedRecordPatterns {
    
    /**
     * Before nested patterns
     */
    public static void beforeNestedPatterns() {
        System.out.println("=== Before Nested Patterns ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        
        Object obj = new Circle(new Point(5, 5), 10);
        
        // Manual multi-level extraction
        if (obj instanceof Circle) {
            Circle circle = (Circle) obj;
            Point center = circle.center();
            int x = center.x();
            int y = center.y();
            double radius = circle.radius();
            System.out.println("Circle at (" + x + ", " + y + 
                             ") with radius " + radius);
        }
        
        /**
         * PROBLEMS:
         * 
         * - Multiple steps
         * - Many accessor calls
         * - Temporary variables
         * - Verbose (6+ lines)
         * - Hard to read
         */
    }
    
    /**
     * After nested patterns
     */
    public static void afterNestedPatterns() {
        System.out.println("\n=== After Nested Patterns ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        
        Object obj = new Circle(new Point(5, 5), 10);
        
        // One-line nested deconstruction
        if (obj instanceof Circle(Point(int x, int y), double radius)) {
            System.out.println("Circle at (" + x + ", " + y + 
                             ") with radius " + radius);
        }
        
        /**
         * BENEFITS:
         * 
         * ✓ One pattern
         * ✓ Direct access to all components
         * ✓ No intermediate variables
         * ✓ Clear structure
         * ✓ Mirrors data structure
         */
    }
    
    /**
     * Deep nesting
     */
    public static void deepNesting() {
        System.out.println("\n=== Deep Nesting ===");
        
        record Address(String street, String city, String country) {}
        record Person(String name, Address address) {}
        record Company(String name, Person ceo) {}
        
        Object obj = new Company(
            "TechCorp",
            new Person(
                "Alice",
                new Address("123 Main St", "NYC", "USA")
            )
        );
        
        // Triple-nested pattern
        if (obj instanceof Company(
            String companyName,
            Person(String ceoName, Address(_, String city, String country))
        )) {
            System.out.println(companyName + " CEO " + ceoName + 
                             " in " + city + ", " + country);
        }
        
        /**
         * DEEP NESTING:
         * 
         * Can nest patterns arbitrarily deep:
         * Company(
         *   String name,
         *   Person(
         *     String personName,
         *     Address(
         *       String street,
         *       String city,
         *       String country
         *     )
         *   )
         * )
         * 
         * Extracts all in one go:
         * - Company name
         * - Person name
         * - Address components
         * 
         * Use wildcards to skip levels:
         * Company(_, Person(_, Address(_, String city, _)))
         */
    }
    
    /**
     * Nested patterns with collections
     */
    public static void nestedWithCollections() {
        System.out.println("\n=== Nested with Collections ===");
        
        record Point(int x, int y) {}
        record Polygon(java.util.List<Point> vertices) {}
        
        Object obj = new Polygon(
            java.util.List.of(
                new Point(0, 0),
                new Point(10, 0),
                new Point(10, 10),
                new Point(0, 10)
            )
        );
        
        if (obj instanceof Polygon(var vertices)) {
            System.out.println("Polygon with " + vertices.size() + " vertices");
            
            // Can't deconstruct list elements in pattern
            // But can process them
            for (Object vertex : vertices) {
                if (vertex instanceof Point(int x, int y)) {
                    System.out.println("  Vertex: (" + x + ", " + y + ")");
                }
            }
        }
        
        /**
         * COLLECTIONS IN PATTERNS:
         * 
         * Can extract collections:
         * Polygon(List<Point> vertices)
         * 
         * Cannot (yet) pattern match elements:
         * // Not yet supported:
         * Polygon([Point(0, 0), Point(10, 0), ...])
         * 
         * Must process after extraction:
         * if (obj instanceof Polygon(var vertices)) {
         *   for (var v : vertices) {
         *     if (v instanceof Point(int x, int y)) {
         *       // process
         *     }
         *   }
         * }
         */
    }
    
    /**
     * Tree traversal with nested patterns
     */
    public static void treeTraversal() {
        System.out.println("\n=== Tree Traversal ===");
        
        sealed interface Tree<T> {}
        record Leaf<T>(T value) implements Tree<T> {}
        record Branch<T>(Tree<T> left, Tree<T> right) implements Tree<T> {}
        
        // Build tree: ((1, 2), (3, 4))
        Tree<Integer> tree = new Branch<>(
            new Branch<>(
                new Leaf<>(1),
                new Leaf<>(2)
            ),
            new Branch<>(
                new Leaf<>(3),
                new Leaf<>(4)
            )
        );
        
        printTree(tree, 0);
    }
    
    private static <T> void printTree(Tree<T> tree, int depth) {
        String indent = "  ".repeat(depth);
        
        switch (tree) {
            case Leaf(T value) -> 
                System.out.println(indent + "Leaf: " + value);
            
            case Branch(Tree<T> left, Tree<T> right) -> {
                System.out.println(indent + "Branch:");
                printTree(left, depth + 1);
                printTree(right, depth + 1);
            }
        }
        
        /**
         * RECURSIVE PATTERNS:
         * 
         * Perfect for recursive data:
         * case Branch(Tree left, Tree right) -> {
         *   process(left);
         *   process(right);
         * }
         * 
         * Deconstructs structure
         * Then recurses on components
         * 
         * Common for:
         * - Binary trees
         * - Expression trees
         * - File systems
         * - Hierarchical data
         */
    }
    
    /**
     * Partial nesting
     */
    public static void partialNesting() {
        System.out.println("\n=== Partial Nesting ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        record ColoredCircle(Circle circle, String color) {}
        
        Object obj = new ColoredCircle(
            new Circle(new Point(5, 5), 10),
            "red"
        );
        
        // Fully nested
        if (obj instanceof ColoredCircle(
            Circle(Point(int x, int y), double r),
            String color
        )) {
            System.out.println("Fully nested: " + color + " circle at (" + 
                             x + ", " + y + ")");
        }
        
        // Partially nested
        if (obj instanceof ColoredCircle(Circle circle, String color)) {
            if (circle.center() instanceof Point(int x, int y)) {
                System.out.println("Partially nested: " + color + 
                                 " circle at (" + x + ", " + y + ")");
            }
        }
        
        /**
         * NESTING CHOICES:
         * 
         * Full nesting:
         * ColoredCircle(Circle(Point(x, y), r), color)
         * 
         * Partial nesting:
         * ColoredCircle(Circle circle, color)
         * // Then access circle.center()
         * 
         * No nesting:
         * ColoredCircle cc
         * // Then access cc.circle().center()
         * 
         * Choose based on:
         * - What you need
         * - Readability
         * - Complexity
         */
    }
    
    public static void main(String[] args) {
        beforeNestedPatterns();
        afterNestedPatterns();
        deepNesting();
        nestedWithCollections();
        treeTraversal();
        partialNesting();
    }
}
```

## 3. Pattern Matching in instanceof

### Record Patterns Beyond Switch

```java
/**
 * PATTERN MATCHING IN INSTANCEOF
 * 
 * Using record patterns with instanceof
 */

public class PatternMatchingInstanceof {
    
    /**
     * Basic instanceof with record patterns
     */
    public static void basicInstanceof() {
        System.out.println("=== Basic instanceof ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        // Record pattern in instanceof
        if (obj instanceof Point(int x, int y)) {
            double distance = Math.sqrt(x * x + y * y);
            System.out.println("Distance from origin: " + distance);
        }
        
        /**
         * INSTANCEOF WITH RECORD PATTERNS:
         * 
         * Syntax:
         * if (obj instanceof RecordType(components)) {
         *   // components available here
         * }
         * 
         * Combines:
         * - Type check
         * - Cast
         * - Deconstruction
         * 
         * All in one operation!
         */
    }
    
    /**
     * Flow scoping
     */
    public static void flowScoping() {
        System.out.println("\n=== Flow Scoping ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        // Variables scoped to true branch
        if (obj instanceof Point(int x, int y)) {
            System.out.println("In true branch: x=" + x + ", y=" + y);
            // x and y available here
        }
        // x and y NOT available here
        
        // Guard clause pattern
        if (!(obj instanceof Point(int x, int y))) {
            return;
        }
        // x and y available here!
        System.out.println("After guard: x=" + x + ", y=" + y);
        
        /**
         * FLOW SCOPING:
         * 
         * Pattern variables scoped based on control flow:
         * 
         * if (obj instanceof Point(int x, int y)) {
         *   // x, y in scope
         * } else {
         *   // x, y NOT in scope
         * }
         * // x, y NOT in scope
         * 
         * 
         * Guard clause:
         * if (!(obj instanceof Point(int x, int y))) {
         *   return;
         * }
         * // x, y in scope (negation inverts)
         * 
         * 
         * With &&:
         * if (obj instanceof Point(int x, int y) && x > 0) {
         *   // x, y in scope
         * }
         * 
         * With ||:
         * if (obj instanceof Point(int x, int y) || other) {
         *   // x, y NOT in scope (ambiguous)
         * }
         */
    }
    
    /**
     * Combining with logical operators
     */
    public static void logicalOperators() {
        System.out.println("\n=== Logical Operators ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        
        Object obj = new Circle(new Point(0, 0), 5);
        
        // AND operator
        if (obj instanceof Circle(Point(int x, int y), double r) && r > 0) {
            System.out.println("Valid circle at (" + x + ", " + y + 
                             ") with radius " + r);
        }
        
        // Chaining conditions
        if (obj instanceof Circle(Point(int x, int y), double r) 
            && x == 0 
            && y == 0 
            && r > 0) {
            System.out.println("Circle at origin with positive radius");
        }
        
        /**
         * LOGICAL OPERATORS:
         * 
         * AND (&&):
         * if (obj instanceof Point(int x, int y) && x > 0 && y > 0) {
         *   // x, y available
         *   // Can use in subsequent conditions
         * }
         * 
         * 
         * OR (||):
         * if (obj instanceof Point(int x, int y) || other) {
         *   // x, y NOT available (could be from other branch)
         * }
         * 
         * 
         * Pattern for validation:
         * if (obj instanceof Point(int x, int y)
         *     && x >= 0 && x <= 100
         *     && y >= 0 && y <= 100) {
         *   // Valid point in range
         * }
         */
    }
    
    /**
     * Method parameters and returns
     */
    public static void methodsWithPatterns() {
        System.out.println("\n=== Methods with Patterns ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        
        Object shape = new Circle(new Point(3, 4), 5);
        
        double area = calculateArea(shape);
        System.out.println("Area: " + area);
        
        boolean atOrigin = isAtOrigin(shape);
        System.out.println("At origin: " + atOrigin);
    }
    
    private static double calculateArea(Object shape) {
        if (shape instanceof Circle(_, double radius)) {
            return Math.PI * radius * radius;
        }
        return 0.0;
        
        /**
         * METHODS WITH PATTERNS:
         * 
         * Can use in method bodies:
         * double calculate(Object obj) {
         *   if (obj instanceof Point(int x, int y)) {
         *     return Math.sqrt(x*x + y*y);
         *   }
         *   return 0.0;
         * }
         * 
         * Benefits:
         * ✓ Type-safe parameter handling
         * ✓ Clear validation
         * ✓ Direct extraction
         */
    }
    
    private static boolean isAtOrigin(Object shape) {
        return shape instanceof Circle(Point(int x, int y), _) 
            && x == 0 && y == 0;
        
        /**
         * PREDICATE METHODS:
         * 
         * Perfect for validation:
         * boolean isValid(Object obj) {
         *   return obj instanceof Point(int x, int y)
         *       && x >= 0 && y >= 0;
         * }
         * 
         * Clean and readable
         */
    }
    
    /**
     * Early returns
     */
    public static void earlyReturns() {
        System.out.println("\n=== Early Returns ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        String result = processPoint(obj);
        System.out.println(result);
    }
    
    private static String processPoint(Object obj) {
        // Early return for invalid input
        if (!(obj instanceof Point(int x, int y))) {
            return "Not a point";
        }
        
        // x and y available here
        if (x < 0 || y < 0) {
            return "Negative coordinates";
        }
        
        if (x == 0 && y == 0) {
            return "Origin";
        }
        
        return "Point at (" + x + ", " + y + ")";
        
        /**
         * EARLY RETURN PATTERN:
         * 
         * Guard against invalid:
         * if (!(obj instanceof Point(int x, int y))) {
         *   return defaultValue;
         * }
         * // x, y available for rest of method
         * 
         * Benefits:
         * ✓ Clear validation
         * ✓ Pattern variables in scope
         * ✓ Reduce nesting
         * ✓ Early exit for errors
         */
    }
    
    /**
     * Loops with patterns
     */
    public static void loopsWithPatterns() {
        System.out.println("\n=== Loops with Patterns ===");
        
        record Point(int x, int y) {}
        
        Object[] objects = {
            new Point(1, 2),
            "not a point",
            new Point(3, 4),
            42,
            new Point(5, 6)
        };
        
        // Filter and process in loop
        for (Object obj : objects) {
            if (obj instanceof Point(int x, int y)) {
                System.out.println("Point: (" + x + ", " + y + ")");
            }
        }
        
        /**
         * LOOPS:
         * 
         * Can use instanceof in loops:
         * for (Object obj : collection) {
         *   if (obj instanceof Record(components)) {
         *     // process components
         *   }
         * }
         * 
         * While loops:
         * while (queue.poll() instanceof Task(String name, int priority)) {
         *   // process task
         * }
         * 
         * Useful for:
         * - Filtering collections
         * - Processing heterogeneous data
         * - Event handling
         */
    }
    
    public static void main(String[] args) {
        basicInstanceof();
        flowScoping();
        logicalOperators();
        methodsWithPatterns();
        earlyReturns();
        loopsWithPatterns();
    }
}
```

## 4. Use Cases and Examples

### Practical Applications of Record Patterns

```java
/**
 * USE CASES AND EXAMPLES
 * 
 * Real-world applications
 */

import java.util.*;
import java.util.stream.*;

public class RecordPatternUseCases {
    
    /**
     * Use case 1: Geometric calculations
     */
    public static void geometricCalculations() {
        System.out.println("=== Geometric Calculations ===");
        
        record Point(double x, double y) {}
        record Line(Point start, Point end) {}
        record Circle(Point center, double radius) {}
        record Rectangle(Point topLeft, Point bottomRight) {}
        
        Object[] shapes = {
            new Line(new Point(0, 0), new Point(10, 10)),
            new Circle(new Point(5, 5), 3),
            new Rectangle(new Point(0, 0), new Point(10, 10))
        };
        
        for (Object shape : shapes) {
            double measurement = switch (shape) {
                // Line length using nested pattern
                case Line(Point(double x1, double y1), Point(double x2, double y2)) -> {
                    double dx = x2 - x1;
                    double dy = y2 - y1;
                    yield Math.sqrt(dx * dx + dy * dy);
                }
                
                // Circle circumference
                case Circle(_, double r) -> 2 * Math.PI * r;
                
                // Rectangle perimeter
                case Rectangle(Point(double x1, double y1), Point(double x2, double y2)) -> 
                    2 * (Math.abs(x2 - x1) + Math.abs(y2 - y1));
                
                case null -> 0;
                default -> 0;
            };
            
            System.out.println(shape.getClass().getSimpleName() + 
                             " measurement: " + measurement);
        }
        
        /**
         * GEOMETRIC CALCULATIONS:
         * 
         * Record patterns perfect for:
         * - Extracting coordinates
         * - Computing distances
         * - Area/perimeter calculations
         * - Collision detection
         * - Transformations
         * 
         * Benefits:
         * ✓ Direct access to coordinates
         * ✓ No temp variables
         * ✓ Clear formulas
         * ✓ Type-safe
         */
    }
    
    /**
     * Use case 2: Expression evaluation
     */
    public static void expressionEvaluation() {
        System.out.println("\n=== Expression Evaluation ===");
        
        sealed interface Expr {}
        record Const(double value) implements Expr {}
        record Var(String name) implements Expr {}
        record Add(Expr left, Expr right) implements Expr {}
        record Mul(Expr left, Expr right) implements Expr {}
        record Pow(Expr base, Expr exponent) implements Expr {}
        
        Map<String, Double> env = Map.of("x", 2.0, "y", 3.0);
        
        // x^2 + y*3
        Expr expr = new Add(
            new Pow(new Var("x"), new Const(2)),
            new Mul(new Var("y"), new Const(3))
        );
        
        double result = eval(expr, env);
        System.out.println("Result: " + result);  // 4 + 9 = 13
        
        // Derivative
        Expr derivative = differentiate(expr, "x");
        System.out.println("Derivative: " + pretty(derivative));
    }
    
    private static double eval(Expr expr, Map<String, Double> env) {
        return switch (expr) {
            case Const(double v) -> v;
            case Var(String name) -> env.get(name);
            case Add(Expr l, Expr r) -> eval(l, env) + eval(r, env);
            case Mul(Expr l, Expr r) -> eval(l, env) * eval(r, env);
            case Pow(Expr base, Expr exp) -> 
                Math.pow(eval(base, env), eval(exp, env));
        };
    }
    
    private static Expr differentiate(Expr expr, String var) {
        return switch (expr) {
            case Const(_) -> new Const(0);
            case Var(String name) -> 
                name.equals(var) ? new Const(1) : new Const(0);
            case Add(Expr l, Expr r) -> 
                new Add(differentiate(l, var), differentiate(r, var));
            case Mul(Expr l, Expr r) -> 
                // Product rule: d(uv) = u'v + uv'
                new Add(
                    new Mul(differentiate(l, var), r),
                    new Mul(l, differentiate(r, var))
                );
            case Pow(Expr base, Const(double n)) -> 
                // Power rule: d(x^n) = n*x^(n-1)
                new Mul(
                    new Mul(new Const(n), new Pow(base, new Const(n - 1))),
                    differentiate(base, var)
                );
            case Pow(_, _) -> 
                throw new UnsupportedOperationException("General power rule");
        };
    }
    
    private static String pretty(Expr expr) {
        return switch (expr) {
            case Const(double v) -> String.valueOf(v);
            case Var(String name) -> name;
            case Add(Expr l, Expr r) -> "(" + pretty(l) + " + " + pretty(r) + ")";
            case Mul(Expr l, Expr r) -> pretty(l) + "*" + pretty(r);
            case Pow(Expr base, Expr exp) -> pretty(base) + "^" + pretty(exp);
        };
    }
    
    /**
     * Use case 3: Tree algorithms
     */
    public static void treeAlgorithms() {
        System.out.println("\n=== Tree Algorithms ===");
        
        sealed interface BinaryTree<T> {}
        record Empty<T>() implements BinaryTree<T> {}
        record Node<T>(T value, BinaryTree<T> left, BinaryTree<T> right) 
            implements BinaryTree<T> {}
        
        // Build tree:     5
        //               /   \
        //              3     7
        //             / \   / \
        //            1   4 6   9
        BinaryTree<Integer> tree = new Node<>(5,
            new Node<>(3,
                new Node<>(1, new Empty<>(), new Empty<>()),
                new Node<>(4, new Empty<>(), new Empty<>())
            ),
            new Node<>(7,
                new Node<>(6, new Empty<>(), new Empty<>()),
                new Node<>(9, new Empty<>(), new Empty<>())
            )
        );
        
        System.out.println("Size: " + size(tree));
        System.out.println("Height: " + height(tree));
        System.out.println("Contains 6: " + contains(tree, 6));
        System.out.println("Sum: " + sum(tree));
        System.out.println("In-order: " + inOrder(tree));
    }
    
    private static <T> int size(BinaryTree<T> tree) {
        return switch (tree) {
            case Empty<T> _ -> 0;
            case Node<T>(_, var left, var right) -> 
                1 + size(left) + size(right);
        };
    }
    
    private static <T> int height(BinaryTree<T> tree) {
        return switch (tree) {
            case Empty<T> _ -> 0;
            case Node<T>(_, var left, var right) -> 
                1 + Math.max(height(left), height(right));
        };
    }
    
    private static <T> boolean contains(BinaryTree<T> tree, T value) {
        return switch (tree) {
            case Empty<T> _ -> false;
            case Node<T>(T v, var left, var right) -> 
                v.equals(value) || contains(left, value) || contains(right, value);
        };
    }
    
    private static int sum(BinaryTree<Integer> tree) {
        return switch (tree) {
            case Empty<Integer> _ -> 0;
            case Node<Integer>(Integer value, var left, var right) -> 
                value + sum(left) + sum(right);
        };
    }
    
    private static <T> List<T> inOrder(BinaryTree<T> tree) {
        return switch (tree) {
            case Empty<T> _ -> List.of();
            case Node<T>(T value, var left, var right) -> {
                List<T> result = new ArrayList<>(inOrder(left));
                result.add(value);
                result.addAll(inOrder(right));
                yield result;
            }
        };
    }

            .map(user -> {

        }

            if (age < 0 || age > 150) {
                errors.add("Invalid age range");
```

## Best Practices

### Guidelines for Using Record Patterns

```java
/**
 * BEST PRACTICES
 * 
 * How to use record patterns effectively
 */

public class RecordPatternBestPractices {
    
    /**
     * When to use record patterns
     */
    public static void whenToUse() {
        /**
         * USE RECORD PATTERNS FOR:
         * 
         * 1. Immutable data extraction:
         * if (obj instanceof Point(int x, int y)) {
         *   // Process coordinates
         * }
         * 
         * 2. Nested data access:
         * case Circle(Point(int x, int y), double r) -> ...
         * 
         * 3. Multiple component needs:
         * case Person(String name, int age, String email) -> ...
         * 
         * 4. Tree/recursive structures:
         * case Node(T value, Tree left, Tree right) -> ...
         * 
         * 5. Type-safe dispatching:
         * switch (shape) {
         *   case Circle(...) -> ...
         *   case Rectangle(...) -> ...
         * }
         * 
         * 
         * DON'T USE FOR:
         * 
         * 1. Single component access:
         * // Don't:
         * if (obj instanceof Box(var value)) {
         *   return value;
         * }
         * // Do:
         * if (obj instanceof Box box) {
         *   return box.value();
         * }
         * 
         * 2. Mutable objects:
         * // Records are immutable
         * // Use regular instanceof for mutable
         * 
         * 3. When not processing components:
         * // Don't deconstruct if you don't use components
         */
    }
    
    /**
     * Readability guidelines
     */
    public static void readability() {
        /**
         * READABILITY:
         * 
         * 1. Use meaningful names:
         * // Good:
         * case Point(int x, int y) -> ...
         * 
         * // Bad:
         * case Point(int a, int b) -> ...
         * 
         * 
         * 2. Limit nesting depth:
         * // OK (2 levels):
         * case Circle(Point(int x, int y), double r) -> ...
         * 
         * // Consider refactoring (3+ levels):
         * case A(B(C(int x, int y), int z), int w) -> ...
         * 
         * 
         * 3. Use wildcards for unused:
         * // Clear:
         * case Person(String name, _, _) -> ...
         * 
         * // Noisy:
         * case Person(String name, int age, String email) -> ...
         * // Then not using age, email
         * 
         * 
         * 4. Consider var for obvious types:
         * // OK:
         * case Point(var x, var y) -> ...
         * 
         * // Better for documentation:
         * case Point(int x, int y) -> ...
         */
    }
    
    /**
     * Performance considerations
     */
    public static void performance() {
        /**
         * PERFORMANCE:
         * 
         * Record patterns are:
         * ✓ Zero-cost abstraction
         * ✓ Compiled to accessor calls
         * ✓ Inlined by JIT
         * ✓ No runtime overhead
         * 
         * Example:
         * if (obj instanceof Point(int x, int y)) {
         *   use(x, y);
         * }
         * 
         * Compiles to:
         * if (obj instanceof Point) {
         *   Point p = (Point) obj;
         *   int x = p.x();
         *   int y = p.y();
         *   use(x, y);
         * }
         * 
         * Then JIT inlines accessors:
         * - No method call overhead
         * - Direct field access
         * - Same as manual code
         * 
         * 
         * WILDCARDS:
         * 
         * Wildcards may skip extraction:
         * case Point(int x, _, _) -> ...
         * 
         * May only call x(), not y() or z()
         * Compiler optimization
         * 
         * But don't over-optimize:
         * - Accessors are cheap
         * - JIT optimizes anyway
         * - Readability > micro-optimization
         */
    }
    
    /**
     * Type safety
     */
    public static void typeSafety() {
        /**
         * TYPE SAFETY:
         * 
         * Compiler verifies:
         * 
         * 1. Component count:
         * record Point(int x, int y) {}
         * 
         * case Point(int x, int y, int z) -> ...  // ERROR
         * 
         * 
         * 2. Component types:
         * case Point(String x, String y) -> ...  // ERROR
         * 
         * 
         * 3. Component order:
         * case Point(int y, int x) -> ...  // Legal but confusing!
         * // x gets first component (x), y gets second (y)
         * // Don't swap names!
         * 
         * 
         * 4. Pattern structure:
         * sealed interface Shape permits Circle, Rectangle {}
         * 
         * switch (shape) {
         *   case Circle(int r) -> ...  // ERROR if Circle has Point
         * }
         * 
         * 
         * Benefits:
         * ✓ Catch errors at compile time
         * ✓ Refactoring safety
         * ✓ IDE support
         */
    }
    
    /**
     * Migration from old code
     */
    public static void migration() {
        /**
         * MIGRATION:
         * 
         * From instanceof + accessors:
         * 
         * Before:
         * if (obj instanceof Point) {
         *   Point p = (Point) obj;
         *   int x = p.x();
         *   int y = p.y();
         *   double dist = Math.sqrt(x*x + y*y);
         * }
         * 
         * After:
         * if (obj instanceof Point(int x, int y)) {
         *   double dist = Math.sqrt(x*x + y*y);
         * }
         * 
         * 
         * From switch with instanceof:
         * 
         * Before:
         * String result;
         * if (shape instanceof Circle) {
         *   Circle c = (Circle) shape;
         *   result = "Circle: " + c.radius();
         * } else if (shape instanceof Rectangle) {
         *   Rectangle r = (Rectangle) shape;
         *   result = "Rectangle: " + r.width() + "x" + r.height();
         * } else {
         *   result = "Unknown";
         * }
         * 
         * After:
         * String result = switch (shape) {
         *   case Circle(_, double r) -> "Circle: " + r;
         *   case Rectangle(_, int w, int h) -> "Rectangle: " + w + "x" + h;
         *   default -> "Unknown";
         * };
         */
    }
}
```

## Summary

### Quick Reference

**Basic Record Pattern:**

```java
if (obj instanceof Point(int x, int y)) {
    // x and y available
}
```

**Nested Pattern:**

```java
if (obj instanceof Circle(Point(int x, int y), double radius)) {
    // x, y, radius all available
}
```

**With Wildcards:**

```java
case Person(String name, _, _) -> name;
// Only extract name, ignore age and email
```

**In Switch:**

```java
switch (shape) {
    case Circle(Point(int x, int y), double r) -> 
        "Circle at (" + x + ", " + y + ")";
    case Rectangle(Point p1, Point p2) -> 
        "Rectangle";
}
```

**With Guards:**

```java
if (obj instanceof Point(int x, int y) && x > 0 && y > 0) {
    // First quadrant
}
```

---


**Key insights:**

- **One-line deconstruction**: Extract all components at once
- **Nested patterns**: Access deeply nested data directly
- **instanceof patterns**: Works everywhere, not just switch
- **Zero overhead**: Compiles to accessor calls
- **Type-safe**: Compiler verifies everything

**The revolution:**

```
Before Record Patterns:
if (obj instanceof Circle) {

// 3 lines, direct, no temps

From 7 lines to 3 lines
From verbose to concise
From error-prone to type-safe
```
