# Sequenced Collections (Java 21)

## Overview

Sequenced Collections, introduced in Java 21 (JEP 431), provide a uniform API for collections with a defined encounter order. Before Java 21, accessing first and last elements required different methods for different collection types: `getFirst()` for Deque, `get(0)` for List, `first()` for SortedSet, and `firstKey()` for SortedMap. This inconsistency created confusion and made generic programming difficult. Java 21 introduces three new interfaces—SequencedCollection, SequencedSet, and SequencedMap—that retrofit the collections hierarchy to provide consistent methods for accessing and modifying elements at both ends. The key innovation is the `reversed()` method, which returns a reversed view of the collection that shares the same backing storage, enabling efficient bidirectional iteration without copying. New methods like `addFirst()`, `addLast()`, `getFirst()`, `getLast()`, `removeFirst()`, and `removeLast()` work consistently across List, Deque, LinkedHashSet, and LinkedHashMap. This guide covers the new interfaces, their methods, reversed view semantics, migration patterns from old APIs, performance characteristics, and extensive real-world examples showing how sequenced collections simplify code and enable new patterns.

## 1. SequencedCollection Interface

### The New Super-Interface for Ordered Collections

```java
/**
 * SEQUENCED COLLECTION INTERFACE
 * 
 * Uniform API for collections with defined order
 */

import java.util.*;

public class SequencedCollectionBasics {
    
    /**
     * The problem before Java 21
     */
    public static void beforeSequencedCollections() {
        System.out.println("=== Before Sequenced Collections ===");
        
        // Different APIs for different types!
        List<String> list = new ArrayList<>(List.of("A", "B", "C"));
        Deque<String> deque = new ArrayDeque<>(List.of("A", "B", "C"));
        
        // Getting first element - inconsistent
        String listFirst = list.get(0);              // List: get(0)
        String dequeFirst = deque.getFirst();        // Deque: getFirst()
        
        // Getting last element - inconsistent
        String listLast = list.get(list.size() - 1); // List: get(size-1)
        String dequeLast = deque.getLast();          // Deque: getLast()
        
        // Adding at beginning - inconsistent
        list.add(0, "X");                            // List: add(0, e)
        deque.addFirst("X");                         // Deque: addFirst(e)
        
        System.out.println("List: " + list);
        System.out.println("Deque: " + deque);
        
        /**
         * PROBLEMS:
         * 
         * - Different methods for same operation
         * - Hard to write generic code
         * - Easy to make mistakes
         * - Confusing API
         * - No reversed iteration
         */
    }
    
    /**
     * After Java 21 with SequencedCollection
     */
    public static void afterSequencedCollections() {
        System.out.println("\n=== After Sequenced Collections ===");
        
        // All implement SequencedCollection!
        SequencedCollection<String> list = new ArrayList<>(List.of("A", "B", "C"));
        SequencedCollection<String> deque = new ArrayDeque<>(List.of("A", "B", "C"));
        
        // Uniform API for all!
        String listFirst = list.getFirst();
        String dequeFirst = deque.getFirst();
        
        String listLast = list.getLast();
        String dequeLast = deque.getLast();
        
        list.addFirst("X");
        deque.addFirst("X");
        
        System.out.println("List: " + list);
        System.out.println("Deque: " + deque);
        
        /**
         * BENEFITS:
         * 
         * ✓ Consistent API
         * ✓ Easy generic programming
         * ✓ Clear intent
         * ✓ Reversed views
         * ✓ Better code reuse
         */
    }
    
    /**
     * SequencedCollection methods
     */
    public static void sequencedCollectionMethods() {
        System.out.println("\n=== SequencedCollection Methods ===");
        
        SequencedCollection<String> seq = new ArrayList<>(
            List.of("B", "C", "D")
        );
        
        // Access first/last
        System.out.println("First: " + seq.getFirst());     // B
        System.out.println("Last: " + seq.getLast());       // D
        
        // Add at ends
        seq.addFirst("A");
        seq.addLast("E");
        System.out.println("After adds: " + seq);           // [A, B, C, D, E]
        
        // Remove from ends
        seq.removeFirst();
        seq.removeLast();
        System.out.println("After removes: " + seq);        // [B, C, D]
        
        // Reversed view
        SequencedCollection<String> reversed = seq.reversed();
        System.out.println("Reversed: " + reversed);        // [D, C, B]
        
        /**
         * SEQUENCED COLLECTION METHODS:
         * 
         * Access:
         * - E getFirst()
         * - E getLast()
         * 
         * Modification:
         * - void addFirst(E)
         * - void addLast(E)
         * - E removeFirst()
         * - E removeLast()
         * 
         * View:
         * - SequencedCollection<E> reversed()
         * 
         * All throw NoSuchElementException if empty
         * (except add methods)
         */
    }
    
    /**
     * Collections hierarchy
     */
    public static void collectionsHierarchy() {
        /**
         * NEW COLLECTIONS HIERARCHY:
         * 
         * Collection
         *   ├─ SequencedCollection (new!)
         *   │   ├─ List
         *   │   │   ├─ ArrayList
         *   │   │   ├─ LinkedList
         *   │   │   └─ ...
         *   │   ├─ Deque
         *   │   │   ├─ ArrayDeque
         *   │   │   ├─ LinkedList
         *   │   │   └─ ...
         *   │   └─ SequencedSet (new!)
         *   │       └─ LinkedHashSet
         *   │       └─ SortedSet
         *   │           └─ TreeSet
         *   │           └─ ...
         *   └─ Set
         *       ├─ HashSet
         *       └─ ...
         * 
         * 
         * Map
         *   └─ SequencedMap (new!)
         *       ├─ LinkedHashMap
         *       └─ SortedMap
         *           └─ TreeMap
         *           └─ ...
         * 
         * 
         * KEY POINTS:
         * 
         * - List now extends SequencedCollection
         * - Deque now extends SequencedCollection
         * - LinkedHashSet now implements SequencedSet
         * - SortedSet now extends SequencedSet
         * - LinkedHashMap now implements SequencedMap
         * - SortedMap now extends SequencedMap
         * 
         * HashSet does NOT implement SequencedCollection
         * (no defined order)
         */
    }
    
    /**
     * Which collections are sequenced?
     */
    public static void whichAreSequenced() {
        System.out.println("\n=== Which Collections Are Sequenced ===");
        
        // These ARE SequencedCollection:
        SequencedCollection<String> list = new ArrayList<>();
        SequencedCollection<String> linked = new LinkedList<>();
        SequencedCollection<String> deque = new ArrayDeque<>();
        SequencedCollection<String> linkedSet = new LinkedHashSet<>();
        SequencedCollection<String> treeSet = new TreeSet<>();
        
        System.out.println("All these implement SequencedCollection!");
        
        // These are NOT SequencedCollection:
        Collection<String> hashSet = new HashSet<>();        // No order
        Collection<String> concurrentSet = 
            java.util.concurrent.ConcurrentHashMap.newKeySet();  // No order
        
        /**
         * SEQUENCED:
         * ✓ ArrayList, LinkedList
         * ✓ ArrayDeque, LinkedBlockingDeque
         * ✓ LinkedHashSet
         * ✓ TreeSet, ConcurrentSkipListSet
         * 
         * NOT SEQUENCED:
         * ✗ HashSet
         * ✗ ConcurrentHashMap.KeySet
         * ✗ EnumSet (but has order)
         * 
         * Rule: If encounter order is defined, it's sequenced
         */
    }
    
    /**
     * Generic programming
     */
    public static void genericProgramming() {
        System.out.println("\n=== Generic Programming ===");
        
        // Works with any SequencedCollection!
        processSequenced(new ArrayList<>(List.of(1, 2, 3, 4, 5)));
        processSequenced(new LinkedHashSet<>(List.of(1, 2, 3, 4, 5)));
        processSequenced(new ArrayDeque<>(List.of(1, 2, 3, 4, 5)));
    }
    
    private static void processSequenced(SequencedCollection<Integer> seq) {
        System.out.println("Type: " + seq.getClass().getSimpleName());
        System.out.println("First: " + seq.getFirst());
        System.out.println("Last: " + seq.getLast());
        System.out.println("Size: " + seq.size());
        
        /**
         * GENERIC PROGRAMMING:
         * 
         * Can write methods that work with any sequenced collection:
         * 
         * <T> T getMiddle(SequencedCollection<T> seq) {
         *   int mid = seq.size() / 2;
         *   return seq.stream().skip(mid).findFirst().orElse(null);
         * }
         * 
         * <T> void rotateRight(SequencedCollection<T> seq) {
         *   if (!seq.isEmpty()) {
         *     T last = seq.removeLast();
         *     seq.addFirst(last);
         *   }
         * }
         * 
         * Before: Had to handle List, Deque separately
         * After: One implementation for all!
         */
    }
    
    public static void main(String[] args) {
        beforeSequencedCollections();
        afterSequencedCollections();
        sequencedCollectionMethods();
        whichAreSequenced();
        genericProgramming();
    }
}
```

## 2. SequencedSet and SequencedMap

### Sequenced Interfaces for Sets and Maps

```java
/**
 * SEQUENCED SET AND MAP
 * 
 * Ordered sets and maps with uniform API
 */

import java.util.*;

public class SequencedSetAndMap {
    
    /**
     * SequencedSet basics
     */
    public static void sequencedSetBasics() {
        System.out.println("=== SequencedSet Basics ===");
        
        SequencedSet<String> linkedSet = new LinkedHashSet<>();
        linkedSet.add("A");
        linkedSet.add("B");
        linkedSet.add("C");
        
        System.out.println("Set: " + linkedSet);
        System.out.println("First: " + linkedSet.getFirst());
        System.out.println("Last: " + linkedSet.getLast());
        
        // Reversed view
        SequencedSet<String> reversed = linkedSet.reversed();
        System.out.println("Reversed: " + reversed);
        
        /**
         * SEQUENCED SET:
         * 
         * Extends SequencedCollection
         * 
         * Implemented by:
         * - LinkedHashSet
         * - TreeSet (via SortedSet)
         * - ConcurrentSkipListSet (via SortedSet)
         * 
         * Methods:
         * - getFirst(), getLast()
         * - removeFirst(), removeLast()
         * - SequencedSet<E> reversed()
         * 
         * No addFirst/addLast:
         * - Sets maintain uniqueness
         * - Position might violate order
         * - Use add() instead
         */
    }
    
    /**
     * LinkedHashSet as SequencedSet
     */
    public static void linkedHashSetSequenced() {
        System.out.println("\n=== LinkedHashSet ===");
        
        SequencedSet<Integer> set = new LinkedHashSet<>();
        
        // Maintains insertion order
        set.add(5);
        set.add(2);
        set.add(8);
        set.add(1);
        
        System.out.println("Set: " + set);          // [5, 2, 8, 1]
        System.out.println("First: " + set.getFirst());  // 5
        System.out.println("Last: " + set.getLast());    // 1
        
        // Adding duplicate doesn't change order
        set.add(5);
        System.out.println("After add(5): " + set);  // [5, 2, 8, 1]
        
        // Removing first
        set.removeFirst();
        System.out.println("After removeFirst: " + set);  // [2, 8, 1]
        
        /**
         * LINKEDHASHSET:
         * 
         * - Insertion order
         * - Fast lookup (O(1))
         * - Fast add/remove ends (O(1))
         * - No duplicates
         * 
         * Perfect for:
         * - Ordered unique elements
         * - LRU cache keys
         * - Event deduplication
         */
    }
    
    /**
     * TreeSet as SequencedSet
     */
    public static void treeSetSequenced() {
        System.out.println("\n=== TreeSet ===");
        
        SequencedSet<Integer> set = new TreeSet<>();
        
        // Maintains sorted order
        set.add(5);
        set.add(2);
        set.add(8);
        set.add(1);
        
        System.out.println("Set: " + set);          // [1, 2, 5, 8]
        System.out.println("First: " + set.getFirst());  // 1 (smallest)
        System.out.println("Last: " + set.getLast());    // 8 (largest)
        
        // Removing smallest
        set.removeFirst();
        System.out.println("After removeFirst: " + set);  // [2, 5, 8]
        
        /**
         * TREESET:
         * 
         * - Sorted order (natural or comparator)
         * - Fast lookup (O(log n))
         * - Fast add/remove ends (O(log n))
         * - No duplicates
         * 
         * Perfect for:
         * - Sorted unique elements
         * - Range queries
         * - Priority-based processing
         */
    }
    
    /**
     * SequencedMap basics
     */
    public static void sequencedMapBasics() {
        System.out.println("\n=== SequencedMap Basics ===");
        
        SequencedMap<String, Integer> map = new LinkedHashMap<>();
        map.put("A", 1);
        map.put("B", 2);
        map.put("C", 3);
        
        System.out.println("Map: " + map);
        
        // First/last entries
        Map.Entry<String, Integer> first = map.firstEntry();
        Map.Entry<String, Integer> last = map.lastEntry();
        
        System.out.println("First entry: " + first);  // A=1
        System.out.println("Last entry: " + last);    // C=3
        
        // Reversed view
        SequencedMap<String, Integer> reversed = map.reversed();
        System.out.println("Reversed: " + reversed);  // {C=3, B=2, A=1}
        
        /**
         * SEQUENCED MAP:
         * 
         * New interface extending Map
         * 
         * Implemented by:
         * - LinkedHashMap
         * - TreeMap (via SortedMap)
         * - ConcurrentSkipListMap (via SortedMap)
         * 
         * Methods:
         * - Map.Entry<K,V> firstEntry()
         * - Map.Entry<K,V> lastEntry()
         * - Map.Entry<K,V> pollFirstEntry()
         * - Map.Entry<K,V> pollLastEntry()
         * - SequencedMap<K,V> reversed()
         * - SequencedSet<K> sequencedKeySet()
         * - SequencedSet<Map.Entry<K,V>> sequencedEntrySet()
         * - SequencedCollection<V> sequencedValues()
         */
    }
    
    /**
     * SequencedMap methods
     */
    public static void sequencedMapMethods() {
        System.out.println("\n=== SequencedMap Methods ===");
        
        SequencedMap<String, Integer> map = new LinkedHashMap<>();
        map.put("B", 2);
        map.put("C", 3);
        map.put("D", 4);
        
        // Put at ends
        map.putFirst("A", 1);
        map.putLast("E", 5);
        System.out.println("After puts: " + map);  // {A=1, B=2, C=3, D=4, E=5}
        
        // Poll (remove and return)
        Map.Entry<String, Integer> polledFirst = map.pollFirstEntry();
        Map.Entry<String, Integer> polledLast = map.pollLastEntry();
        
        System.out.println("Polled first: " + polledFirst);  // A=1
        System.out.println("Polled last: " + polledLast);    // E=5
        System.out.println("After polls: " + map);            // {B=2, C=3, D=4}
        
        // Sequenced views
        SequencedSet<String> keys = map.sequencedKeySet();
        SequencedCollection<Integer> values = map.sequencedValues();
        SequencedSet<Map.Entry<String, Integer>> entries = map.sequencedEntrySet();
        
        System.out.println("Keys: " + keys);        // [B, C, D]
        System.out.println("Values: " + values);    // [2, 3, 4]
        System.out.println("Entries: " + entries);  // [B=2, C=3, D=4]
        
        /**
         * PUT METHODS:
         * 
         * putFirst(K, V):
         * - Puts entry at beginning
         * - If key exists, moves to first
         * 
         * putLast(K, V):
         * - Puts entry at end
         * - If key exists, moves to last
         * 
         * 
         * POLL METHODS:
         * 
         * pollFirstEntry():
         * - Removes and returns first entry
         * - Returns null if empty
         * 
         * pollLastEntry():
         * - Removes and returns last entry
         * - Returns null if empty
         */
    }
    
    /**
     * LinkedHashMap as SequencedMap
     */
    public static void linkedHashMapSequenced() {
        System.out.println("\n=== LinkedHashMap ===");
        
        SequencedMap<String, Integer> map = new LinkedHashMap<>();
        
        // Insertion order
        map.put("third", 3);
        map.put("first", 1);
        map.put("second", 2);
        
        System.out.println("Map: " + map);
        // {third=3, first=1, second=2}
        
        // Reordering
        map.putFirst("first", 1);
        System.out.println("After putFirst: " + map);
        // {first=1, third=3, second=2}
        
        /**
         * LINKEDHASHMAP:
         * 
         * - Insertion order (or access order)
         * - Fast lookup (O(1))
         * - putFirst/putLast reorder
         * 
         * Perfect for:
         * - LRU caches
         * - Ordered configurations
         * - Event tracking
         */
    }
    
    /**
     * TreeMap as SequencedMap
     */
    public static void treeMapSequenced() {
        System.out.println("\n=== TreeMap ===");
        
        SequencedMap<String, Integer> map = new TreeMap<>();
        
        // Sorted order by key
        map.put("C", 3);
        map.put("A", 1);
        map.put("B", 2);
        
        System.out.println("Map: " + map);
        // {A=1, B=2, C=3} - sorted!
        
        System.out.println("First: " + map.firstEntry());  // A=1
        System.out.println("Last: " + map.lastEntry());    // C=3
        
        /**
         * TREEMAP:
         * 
         * - Sorted order by key
         * - Fast lookup (O(log n))
         * - putFirst/putLast work but maintain sort order
         * 
         * Perfect for:
         * - Sorted key-value pairs
         * - Range queries
         * - Time-series data
         */
    }
    
    public static void main(String[] args) {
        sequencedSetBasics();
        linkedHashSetSequenced();
        treeSetSequenced();
        sequencedMapBasics();
        sequencedMapMethods();
        linkedHashMapSequenced();
        treeMapSequenced();
    }
}
```

## 3. Reversed Views

### Efficient Bidirectional Access

```java
/**
 * REVERSED VIEWS
 * 
 * Reversed views share backing storage
 */

import java.util.*;

public class ReversedViews {
    
    /**
     * Basic reversed view
     */
    public static void basicReversedView() {
        System.out.println("=== Basic Reversed View ===");
        
        SequencedCollection<String> original = new ArrayList<>(
            List.of("A", "B", "C", "D")
        );
        
        // Get reversed view
        SequencedCollection<String> reversed = original.reversed();
        
        System.out.println("Original: " + original);  // [A, B, C, D]
        System.out.println("Reversed: " + reversed);  // [D, C, B, A]
        
        /**
         * REVERSED VIEW:
         * 
         * - Not a copy!
         * - Shares backing storage
         * - Modifications visible in both
         * - Efficient (no duplication)
         */
    }
    
    /**
     * Shared backing storage
     */
    public static void sharedBackingStorage() {
        System.out.println("\n=== Shared Backing Storage ===");
        
        SequencedCollection<String> original = new ArrayList<>(
            List.of("A", "B", "C")
        );
        SequencedCollection<String> reversed = original.reversed();
        
        // Modify original
        original.addLast("D");
        System.out.println("After original.addLast('D'):");
        System.out.println("  Original: " + original);  // [A, B, C, D]
        System.out.println("  Reversed: " + reversed);  // [D, C, B, A]
        
        // Modify reversed
        reversed.addLast("Z");  // Adds to beginning of original!
        System.out.println("After reversed.addLast('Z'):");
        System.out.println("  Original: " + original);  // [Z, A, B, C, D]
        System.out.println("  Reversed: " + reversed);  // [D, C, B, A, Z]
        
        /**
         * BIDIRECTIONAL MODIFICATION:
         * 
         * original.addLast(x)  ≡  reversed.addFirst(x)
         * original.addFirst(x) ≡  reversed.addLast(x)
         * original.getFirst()  ≡  reversed.getLast()
         * original.getLast()   ≡  reversed.getFirst()
         * 
         * Changes in either view affect both!
         */
    }
    
    /**
     * Double reversal
     */
    public static void doubleReversal() {
        System.out.println("\n=== Double Reversal ===");
        
        SequencedCollection<Integer> original = new ArrayList<>(
            List.of(1, 2, 3, 4, 5)
        );
        
        SequencedCollection<Integer> reversed = original.reversed();
        SequencedCollection<Integer> doubleReversed = reversed.reversed();
        
        System.out.println("Original: " + original);           // [1, 2, 3, 4, 5]
        System.out.println("Reversed: " + reversed);           // [5, 4, 3, 2, 1]
        System.out.println("Double reversed: " + doubleReversed);  // [1, 2, 3, 4, 5]
        
        // Are they the same?
        System.out.println("Same object? " + (original == doubleReversed));  // false
        
        /**
         * DOUBLE REVERSAL:
         * 
         * original.reversed().reversed()
         * 
         * - Returns to original order
         * - But NOT the same object
         * - Creates new view
         * - Still shares backing storage
         * 
         * Performance:
         * - Creating reversed view: O(1)
         * - No data copying
         * - Lazy evaluation
         */
    }
    
    /**
     * Reversed iteration
     */
    public static void reversedIteration() {
        System.out.println("\n=== Reversed Iteration ===");
        
        List<String> list = new ArrayList<>(
            List.of("A", "B", "C", "D", "E")
        );
        
        // Forward iteration
        System.out.print("Forward: ");
        for (String s : list) {
            System.out.print(s + " ");
        }
        System.out.println();
        
        // Reversed iteration
        System.out.print("Reversed: ");
        for (String s : list.reversed()) {
            System.out.print(s + " ");
        }
        System.out.println();
        
        /**
         * ITERATION:
         * 
         * Before Java 21:
         * for (int i = list.size() - 1; i >= 0; i--) {
         *   String s = list.get(i);
         *   // process
         * }
         * 
         * Or:
         * Collections.reverse(list);
         * for (String s : list) {
         *   // process
         * }
         * Collections.reverse(list);  // Restore
         * 
         * After Java 21:
         * for (String s : list.reversed()) {
         *   // process
         * }
         * 
         * Much cleaner!
         */
    }
    
    /**
     * Reversed streams
     */
    public static void reversedStreams() {
        System.out.println("\n=== Reversed Streams ===");
        
        List<Integer> numbers = List.of(1, 2, 3, 4, 5);
        
        // Forward stream
        List<Integer> forwardSquares = numbers.stream()
            .map(n -> n * n)
            .toList();
        System.out.println("Forward squares: " + forwardSquares);
        // [1, 4, 9, 16, 25]
        
        // Reversed stream
        List<Integer> reversedSquares = numbers.reversed().stream()
            .map(n -> n * n)
            .toList();
        System.out.println("Reversed squares: " + reversedSquares);
        // [25, 16, 9, 4, 1]
        
        /**
         * STREAMS:
         * 
         * collection.reversed().stream()
         * 
         * Creates stream in reversed order
         * No copying of collection
         * Efficient
         */
    }
    
    /**
     * Reversed views of sets and maps
     */
    public static void reversedSetsAndMaps() {
        System.out.println("\n=== Reversed Sets and Maps ===");
        
        // Reversed Set
        SequencedSet<String> set = new LinkedHashSet<>(
            List.of("A", "B", "C")
        );
        SequencedSet<String> reversedSet = set.reversed();
        
        System.out.println("Set: " + set);              // [A, B, C]
        System.out.println("Reversed set: " + reversedSet);  // [C, B, A]
        
        // Reversed Map
        SequencedMap<String, Integer> map = new LinkedHashMap<>();
        map.put("A", 1);
        map.put("B", 2);
        map.put("C", 3);
        
        SequencedMap<String, Integer> reversedMap = map.reversed();
        
        System.out.println("Map: " + map);              // {A=1, B=2, C=3}
        System.out.println("Reversed map: " + reversedMap);  // {C=3, B=2, A=1}
        
        // Reversed map views
        SequencedSet<String> keys = reversedMap.sequencedKeySet();
        SequencedCollection<Integer> values = reversedMap.sequencedValues();
        
        System.out.println("Reversed keys: " + keys);     // [C, B, A]
        System.out.println("Reversed values: " + values); // [3, 2, 1]
        
        /**
         * REVERSED VIEWS:
         * 
         * All sequenced types support reversed():
         * - SequencedCollection
         * - SequencedSet
         * - SequencedMap
         * 
         * Map views are also reversed:
         * - sequencedKeySet()
         * - sequencedValues()
         * - sequencedEntrySet()
         */
    }
    
    /**
     * Performance characteristics
     */
    public static void performance() {
        /**
         * PERFORMANCE:
         * 
         * Creating reversed view:
         * - Time: O(1) - constant time
         * - Space: O(1) - no copying
         * 
         * Accessing elements:
         * - Same as original
         * - ArrayList reversed: O(1) access
         * - LinkedList reversed: O(n) access
         * 
         * Iteration:
         * - ArrayList reversed: O(n) total
         * - LinkedList reversed: O(n) total
         * - No performance penalty!
         * 
         * Modification:
         * - Same as original
         * - Updates visible in both views
         * 
         * 
         * COMPARISON:
         * 
         * Before (Collections.reverse):
         * Collections.reverse(list);  // O(n), modifies in-place
         * process(list);
         * Collections.reverse(list);  // O(n), restore
         * 
         * After (reversed view):
         * process(list.reversed());   // O(1), no modification
         * 
         * Much more efficient!
         */
    }
    
    public static void main(String[] args) {
        basicReversedView();
        sharedBackingStorage();
        doubleReversal();
        reversedIteration();
        reversedStreams();
        reversedSetsAndMaps();
    }
```

```java
         * METHODS:
         * 
         * addFirst(E element):
         * - Adds at beginning
         * - Shifts existing elements right
         * - Returns void
         * 
         * addLast(E element):
         * - Adds at end
         * - Same as add(E)
         * - Returns void
         */
    }
    
    /**
     * With different collection types
     */
    public static void differentCollectionTypes() {
        System.out.println("\n=== Different Collection Types ===");
        
        // ArrayList
        SequencedCollection<Integer> list = new ArrayList<>(List.of(2, 3));
        list.addFirst(1);
        list.addLast(4);
        System.out.println("ArrayList: " + list);  // [1, 2, 3, 4]
        
        // LinkedList
        SequencedCollection<Integer> linked = new LinkedList<>(List.of(2, 3));
        linked.addFirst(1);
        linked.addLast(4);
        System.out.println("LinkedList: " + linked);  // [1, 2, 3, 4]
        
        // ArrayDeque
        SequencedCollection<Integer> deque = new ArrayDeque<>(List.of(2, 3));
        deque.addFirst(1);
        deque.addLast(4);
        System.out.println("ArrayDeque: " + deque);  // [1, 2, 3, 4]
        
        /**
         * SAME API, DIFFERENT PERFORMANCE:
         * 
         * ArrayList:
         * - addFirst: O(n) - shifts all elements
         * - addLast: O(1) amortized
         * 
         * LinkedList:
         * - addFirst: O(1)
         * - addLast: O(1)
         * 
         * ArrayDeque:
         * - addFirst: O(1) amortized
         * - addLast: O(1) amortized
         * 
         * Choose based on usage pattern!
         */
    }

     * Sets don't have addFirst/addLast
     */
    public static void setsNoAddFirstLast() {

         * SEQUENCED SET LIMITATION:
         * 
         * SequencedSet does NOT have:
         * - addFirst(E)
         * - addLast(E)
         * 
         * Why?
         * - Sets maintain uniqueness
         * - Adding at position might violate order
         * - For TreeSet: must maintain sort order
         * - For LinkedHashSet: insertion order
         * 
         * Use regular add() instead:
         * SequencedSet<String> set = new LinkedHashSet<>();
         * set.add("A");  // Adds at end
         * set.add("B");  // Adds at end
         * 
         * 
         * But can remove from ends:
         * set.removeFirst();  // Works!
         * set.removeLast();   // Works!
         */
    }

     * Maps have putFirst/putLast
     */
    public static void mapsPutFirstLast() {
        System.out.println("\n=== Maps putFirst/putLast ===");
        
        SequencedMap<String, Integer> map = new LinkedHashMap<>();

        map.put("C", 3);

        map.putFirst("A", 1);
        map.putLast("D", 4);

        map.putFirst("B", 20);  // Moves B to first
        System.out.println("After putFirst('B', 20): " + map);
        // {B=20, A=1, C=3, D=4}
        
        /**
         * PUT METHODS:
         * 
         * putFirst(K key, V value):
         * - If key new: adds at beginning
         * - If key exists: updates and moves to beginning
         * 
         * putLast(K key, V value):
         * - If key new: adds at end
         * - If key exists: updates and moves to end
         * 
         * Useful for:
         * - LRU cache implementation
         * - Priority reordering
         * - Most/least recently used

    }

        queue.addLast("third");

         * PATTERNS:
         * 
         * Stack (LIFO):
         * addFirst() - push
         * removeFirst() - pop
         * getFirst() - peek
         * 
         * Queue (FIFO):
         * addLast() - enqueue
         * removeFirst() - dequeue
         * getFirst() - peek
         * 
         * Deque (both ends):
         * addFirst(), addLast()
         * removeFirst(), removeLast()

    }
```

```java
public class UniformAPI {

        System.out.println("Last: " + seq.getLast());

         * UNIFORM API:

         * - getLast()
         * - reversed()
         * - removeFirst()
         * - removeLast()
         * 
         * All Lists/Deques additionally have:
         * - addFirst(E)

         * - firstEntry()
         * - lastEntry()
         * - pollFirstEntry()
         * - pollLastEntry()
         * - putFirst(K, V)
         * - putLast(K, V)

     * Generic algorithms
     */
    public static void genericAlgorithms() {
        System.out.println("\n=== Generic Algorithms ===");
        
        // Works with any SequencedCollection!
        System.out.println("Rotate ArrayList:");
        rotateRight(new ArrayList<>(List.of(1, 2, 3, 4, 5)));
        
        System.out.println("\nRotate LinkedHashSet:");
        rotateRight(new LinkedHashSet<>(List.of(1, 2, 3, 4, 5)));

    private static <T> void rotateRight(SequencedCollection<T> seq) {
        System.out.println("Before: " + seq);
        if (!seq.isEmpty()) {
            T last = seq.removeLast();
            seq.addFirst(last);
        }
        System.out.println("After: " + seq);
        
        /**
         * GENERIC ALGORITHMS:
         * 
         * Can write once, works for all:
         * 
         * <T> T getMiddle(SequencedCollection<T> seq) {
         *   int mid = seq.size() / 2;
         *   return seq.stream().skip(mid).findFirst().orElse(null);
         * }
         * 
         * <T> void swap(SequencedCollection<T> seq) {
         *   if (seq.size() >= 2) {
         *     T first = seq.removeFirst();
         *     T last = seq.removeLast();
         *     seq.addFirst(last);
         *     seq.addLast(first);
         *   }

         *   List<T> result = new ArrayList<>(seq);
         *   result.addAll(seq.reversed());
         *   return result;
         * }

         * After:

     * Real-world example: LRU Cache
     */
    public static void lruCacheExample() {
        System.out.println("\n=== LRU Cache Example ===");
        
        LRUCache<String, Integer> cache = new LRUCache<>(3);
        
        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        System.out.println("After adding A, B, C: " + cache);
        
        cache.get("A");  // Access A
        System.out.println("After accessing A: " + cache);
        
        cache.put("D", 4);  // B is evicted (least recently used)
        System.out.println("After adding D: " + cache);
    }
    
    static class LRUCache<K, V> {
        private final int capacity;
        private final SequencedMap<K, V> map = new LinkedHashMap<>();
        
        LRUCache(int capacity) {
            this.capacity = capacity;
        }
        
        V get(K key) {
            V value = map.remove(key);
            if (value != null) {
                map.putLast(key, value);  // Move to end (most recent)
            }
            return value;
        }
        
        void put(K key, V value) {
            map.remove(key);  // Remove if exists
            map.putLast(key, value);  // Add at end
            
            if (map.size() > capacity) {
                map.pollFirstEntry();  // Remove oldest
            }
        }
        
        @Override
        public String toString() {
            return map.toString();
        }
    }
```

```java
        private final SequencedCollection<String> undoStack = new ArrayDeque<>();
        private final SequencedCollection<String> redoStack = new ArrayDeque<>();

        void execute(String action) {
            undoStack.addLast(currentState);
            currentState = action;
            redoStack.clear();  // Clear redo on new action
        }
        
        void undo() {
            if (!undoStack.isEmpty()) {
                redoStack.addLast(currentState);
                currentState = undoStack.removeLast();
            }
        }
        
        void redo() {
            if (!redoStack.isEmpty()) {
                undoStack.addLast(currentState);
                currentState = redoStack.removeLast();
            }
        }
        
        String currentState() {
            return currentState;
        }
    }
```

### Quick Reference

**SequencedCollection Methods:**

```java
seq.getFirst()      // Get first element
seq.getLast()       // Get last element
seq.addFirst(e)     // Add at beginning
seq.addLast(e)      // Add at end

```

**SequencedMap Methods:**

```java

map.lastEntry()        // Last entry
map.putFirst(k, v)     // Put at beginning
map.putLast(k, v)      // Put at end
map.pollFirstEntry()   // Remove first
map.pollLastEntry()    // Remove last
map.reversed()         // Reversed view
```

**Which Collections:**

```java
// Sequenced:
new ArrayList<>()       ✓
new LinkedList<>()      ✓
new ArrayDeque<>()      ✓
new LinkedHashSet<>()   ✓
```

- **Practical**: Solves 20+ years of API inconsistency
