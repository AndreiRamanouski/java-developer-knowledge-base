# Pattern Matching for Switch (Java 21)

## Overview

Java 21 finalized pattern matching for switch, a feature that dramatically expands the power of switch expressions and statements beyond simple constant matching. Traditional switch only worked with integers, strings, and enums using exact constant values. Pattern matching for switch enables type patterns (matching object types), guarded patterns (adding conditions with `when` clauses), and record patterns (deconstructing records inline). The compiler performs exhaustiveness checking for sealed types, ensuring all possible cases are covered at compile time. Switch now handles null explicitly rather than throwing NullPointerException. This feature combines beautifully with records and sealed classes to enable sophisticated data-oriented programming, making complex conditional logic readable and type-safe. Common patterns include processing algebraic data types, handling polymorphic objects, implementing visitor patterns, and parsing structured data. This guide covers basic type patterns, guarded patterns with when clauses, record patterns for deconstruction, exhaustiveness verification, null handling in switch, and extensive real-world examples showing how this transforms Java code.

### Beyond Constants to Type Patterns

```java
/**
 * PATTERNS IN SWITCH CASES
 * 
 * Type patterns in switch - Java 21
 */

public class PatternsInSwitch {
    
    /**
     * Before: instanceof chains
     */
    public static void beforePatternSwitch() {
        System.out.println("=== Before Pattern Switch ===");
        
        Object obj = "Hello";
        
        String result;
        if (obj instanceof String s) {
            result = "String of length " + s.length();
        } else if (obj instanceof Integer i) {
            result = "Integer: " + i;
        } else if (obj instanceof Double d) {
            result = "Double: " + d;
        } else {
            result = "Unknown type";
        }
        
        System.out.println(result);
        
        /**
         * PROBLEMS:
         * 
         * - Verbose if-else chain
         * - No exhaustiveness checking
         * - Easy to miss a type
         * - Not expression-friendly
         * - Hard to see all cases
         */
    }
    
    /**
     * After: switch with type patterns
     */
    public static void afterPatternSwitch() {
        System.out.println("\n=== After Pattern Switch ===");
        
        Object obj = "Hello";
        
        String result = switch (obj) {
            case String s -> "String of length " + s.length();
            case Integer i -> "Integer: " + i;
            case Double d -> "Double: " + d;
            case null -> "Null value";
            default -> "Unknown type";
        };
        
        System.out.println(result);
        
        /**
         * BENEFITS:
         * 
         * ✓ Concise switch expression
         * ✓ All cases visible together
         * ✓ Pattern variable binding
         * ✓ Expression-friendly
         * ✓ Null handling explicit
         * ✓ More readable
         */
    }
    
    /**
     * Type pattern basics
     */
    public static void typePatternBasics() {
        System.out.println("\n=== Type Pattern Basics ===");
        
        Object[] objects = {
            "Hello",
            42,
            3.14,
            true,
            new int[]{1, 2, 3}
        };
        
        for (Object obj : objects) {
            String description = switch (obj) {
                case String s -> "String: '" + s + "'";
                case Integer i -> "Integer: " + i;
                case Double d -> "Double: " + d;
                case Boolean b -> "Boolean: " + b;
                case int[] arr -> "Array of length " + arr.length;
                case null -> "Null";
                default -> "Unknown: " + obj.getClass().getSimpleName();
            };
            System.out.println(description);
        }
        
        /**
         * TYPE PATTERN SYNTAX:
         * 
         * case Type variableName -> expression
         * 
         * - Type: Any reference type
         * - variableName: Pattern variable
         * - Available in case expression
         * 
         * Examples:
         * case String s -> s.toUpperCase()
         * case Integer i -> i * 2
         * case List<?> list -> list.size()
         */
    }
    
    /**
     * Combining with switch statements
     */
    public static void switchStatements() {
        System.out.println("\n=== Switch Statements ===");
        
        Object obj = 42;
        
        switch (obj) {
            case String s -> {
                System.out.println("Got string: " + s);
                System.out.println("Length: " + s.length());
            }
            case Integer i -> {
                System.out.println("Got integer: " + i);
                System.out.println("Doubled: " + (i * 2));
            }
            case null -> {
                System.out.println("Got null");
            }
            default -> {
                System.out.println("Got something else");
            }
        }
        
        /**
         * SWITCH STATEMENTS:
         * 
         * Works with both:
         * - Switch expressions (->)
         * - Switch statements (: with blocks)
         * 
         * Both support patterns:
         * case String s: {
         *   // Use s here
         *   break;
         * }
         */
    }
    
    /**
     * Dominance and ordering
     */
    public static void dominanceAndOrdering() {
        System.out.println("\n=== Dominance and Ordering ===");
        
        Object obj = "test";
        
        // Order matters!
        String result = switch (obj) {
            case String s -> "Specific: String";
            case Object o -> "General: Object";
            // This order works
        };
        
        System.out.println(result);
        
        /**
         * DOMINANCE RULES:
         * 
         * More specific patterns must come first:
         * 
         * CORRECT:
         * case String s -> ...
         * case Object o -> ...
         * 
         * WRONG (won't compile):
         * case Object o -> ...
         * case String s -> ...  // Error: dominated by Object
         * 
         * Compiler checks:
         * - No unreachable patterns
         * - More specific before general
         * - Prevents dead code
         * 
         * 
         * EXAMPLE HIERARCHY:
         * 
         * case Integer i -> ...
         * case Number n -> ...
         * case Object o -> ...
         * 
         * Integer is more specific than Number
         * Number is more specific than Object
         */
    }
    
    /**
     * Polymorphic dispatch
     */
    public static void polymorphicDispatch() {
        System.out.println("\n=== Polymorphic Dispatch ===");
        
        sealed interface Shape permits Circle, Rectangle, Triangle {}
        record Circle(double radius) implements Shape {}
        record Rectangle(double width, double height) implements Shape {}
        record Triangle(double base, double height) implements Shape {}
        
        Shape shape = new Circle(5.0);
        
        double area = switch (shape) {
            case Circle c -> Math.PI * c.radius() * c.radius();
            case Rectangle r -> r.width() * r.height();
            case Triangle t -> 0.5 * t.base() * t.height();
        };
        
        System.out.println("Area: " + area);
        
        /**
         * POLYMORPHIC DISPATCH:
         * 
         * Replace:
         * if (shape instanceof Circle c) {
         *   return Math.PI * c.radius() * c.radius();
         * } else if (shape instanceof Rectangle r) {
         *   return r.width() * r.height();
         * } ...
         * 
         * With:
         * switch (shape) {
         *   case Circle c -> ...
         *   case Rectangle r -> ...
         *   case Triangle t -> ...
         * }
         * 
         * Benefits:
         * ✓ All cases in one place
         * ✓ Exhaustiveness checking
         * ✓ More readable
         * ✓ Type-safe
         */
    }
    
    public static void main(String[] args) {
        beforePatternSwitch();
        afterPatternSwitch();
        typePatternBasics();
        switchStatements();
        dominanceAndOrdering();
        polymorphicDispatch();
    }
}
```

## 2. Guarded Patterns

### Adding Conditions with when Clauses

```java
/**
 * GUARDED PATTERNS
 * 
 * Using when clauses for additional conditions
 */

public class GuardedPatterns {
    
    /**
     * Basic guarded patterns
     */
    public static void basicGuardedPatterns() {
        System.out.println("=== Basic Guarded Patterns ===");
        
        Object obj = 42;
        
        String result = switch (obj) {
            case String s when s.length() > 5 -> 
                "Long string: " + s;
            case String s -> 
                "Short string: " + s;
            case Integer i when i > 0 -> 
                "Positive integer: " + i;
            case Integer i when i < 0 -> 
                "Negative integer: " + i;
            case Integer i -> 
                "Zero";
            case null -> 
                "Null";
            default -> 
                "Something else";
        };
        
        System.out.println(result);
        
        /**
         * GUARDED PATTERN SYNTAX:
         * 
         * case Pattern when condition -> expression
         * 
         * - Pattern: Type pattern
         * - when: Guard keyword
         * - condition: Boolean expression
         * - Pattern variable available in condition
         * 
         * Execution:
         * 1. Check if type matches
         * 2. Bind pattern variable
         * 3. Evaluate guard condition
         * 4. If true, execute case
         * 5. Otherwise, try next case
         */
    }
    
    /**
     * Multiple guards for same type
     */
    public static void multipleGuards() {
        System.out.println("\n=== Multiple Guards ===");
        
        Object[] numbers = {-5, 0, 1, 5, 10, 100};
        
        for (Object obj : numbers) {
            String category = switch (obj) {
                case Integer i when i < 0 -> "Negative";
                case Integer i when i == 0 -> "Zero";
                case Integer i when i > 0 && i <= 10 -> "Small positive";
                case Integer i when i > 10 && i <= 100 -> "Medium positive";
                case Integer i -> "Large positive";
                default -> "Not an integer";
            };
            System.out.println(obj + " -> " + category);
        }
        
        /**
         * MULTIPLE GUARDS:
         * 
         * Can have multiple cases for same type:
         * case Integer i when i < 0 -> "Negative"
         * case Integer i when i == 0 -> "Zero"
         * case Integer i when i > 0 -> "Positive"
         * 
         * Order matters:
         * - First matching guard wins
         * - More specific guards first
         * - Catch-all guard last
         * 
         * Fall-through:
         * If guard fails, continues to next case
         */
    }
    
    /**
     * Complex guard conditions
     */
    public static void complexGuardConditions() {
        System.out.println("\n=== Complex Guard Conditions ===");
        
        record Person(String name, int age, String city) {}
        
        Object obj = new Person("Alice", 25, "NYC");
        
        String description = switch (obj) {
            case Person p when p.age() < 18 -> 
                "Minor from " + p.city();
            case Person p when p.age() >= 18 && p.age() < 65 -> 
                "Adult from " + p.city();
            case Person p when p.age() >= 65 -> 
                "Senior from " + p.city();
            case Person p -> 
                "Person from " + p.city();
            default -> 
                "Not a person";
        };
        
        System.out.println(description);
        
        /**
         * COMPLEX CONDITIONS:
         * 
         * Guards can be any boolean expression:
         * - Comparisons
         * - Logical operators (&&, ||, !)
         * - Method calls
         * - Field access
         * - Multiple conditions
         * 
         * Examples:
         * case String s when s.startsWith("test") -> ...
         * case List<?> l when !l.isEmpty() -> ...
         * case Person p when isValid(p) -> ...
         * case Integer i when i > 0 && i % 2 == 0 -> ...
         */
    }
    
    /**
     * Guards with method calls
     */
    public static void guardsWithMethodCalls() {
        System.out.println("\n=== Guards with Method Calls ===");
        
        record Email(String address) {
            boolean isValid() {
                return address != null && address.contains("@");
            }
            
            boolean isGmail() {
                return address != null && address.endsWith("@gmail.com");
            }
        }
        
        Object obj = new Email("user@gmail.com");
        
        String result = switch (obj) {
            case Email e when !e.isValid() -> 
                "Invalid email";
            case Email e when e.isGmail() -> 
                "Gmail: " + e.address();
            case Email e -> 
                "Other email: " + e.address();
            default -> 
                "Not an email";
        };
        
        System.out.println(result);
        
        /**
         * METHOD CALLS IN GUARDS:
         * 
         * Can call methods on pattern variable:
         * case Email e when e.isValid() -> ...
         * case String s when s.matches("\\d+") -> ...
         * case List<?> l when l.size() > 10 -> ...
         * 
         * Can call static/external methods:
         * case String s when isValidFormat(s) -> ...
         * case Person p when database.exists(p.id()) -> ...
         * 
         * Benefits:
         * - Encapsulate validation
         * - Reusable logic
         * - Cleaner switch
         */
    }
    
    /**
     * Combining patterns and guards
     */
    public static void combiningPatternsAndGuards() {
        System.out.println("\n=== Combining Patterns and Guards ===");
        
        sealed interface Result permits Success, Failure {}
        record Success(String data) implements Result {}
        record Failure(String error, int code) implements Result {}
        
        Result result = new Failure("Not found", 404);
        
        String message = switch (result) {
            case Success s when s.data().isEmpty() -> 
                "Empty success";
            case Success s -> 
                "Success: " + s.data();
            case Failure f when f.code() == 404 -> 
                "Not found: " + f.error();
            case Failure f when f.code() >= 500 -> 
                "Server error: " + f.error();
            case Failure f -> 
                "Client error: " + f.error();
        };
        
        System.out.println(message);
        
        /**
         * PATTERNS + GUARDS:
         * 
         * Powerful combination:
         * 1. Pattern matches type
         * 2. Guard checks condition
         * 3. Both must be true
         * 
         * Example:
         * case Success s when s.data().isEmpty() -> ...
         * 
         * First: Check if Success
         * Then: Check if data empty
         * Finally: Execute case
         * 
         * Benefits:
         * ✓ Type-safe access
         * ✓ Clear conditions
         * ✓ No null checks needed (if sealed)
         * ✓ Exhaustiveness checking
         */
    }
    
    /**
     * Guard vs pattern comparison
     */
    public static void guardVsPatternComparison() {
        /**
         * WHEN TO USE GUARDS:
         * 
         * Use guards when:
         * - Need runtime condition
         * - Based on value, not type
         * - Multiple conditions for same type
         * - Validation required
         * 
         * Example with guards:
         * case String s when s.length() > 10 -> "Long"
         * case String s when s.isEmpty() -> "Empty"
         * case String s -> "Normal"
         * 
         * 
         * WHEN TO USE PATTERNS:
         * 
         * Use patterns when:
         * - Based on type
         * - Different types need different handling
         * - Polymorphic dispatch
         * 
         * Example with patterns:
         * case String s -> handleString(s)
         * case Integer i -> handleInteger(i)
         * case Double d -> handleDouble(d)
         * 
         * 
         * COMBINATION:
         * 
         * Often use both:
         * case String s when s.startsWith("http") -> handleUrl(s)
         * case String s when s.contains("@") -> handleEmail(s)
         * case String s -> handleGeneric(s)
         * 
         * Type pattern + guard condition = powerful!
         */
    }
    
    public static void main(String[] args) {
        basicGuardedPatterns();
        multipleGuards();
        complexGuardConditions();
        guardsWithMethodCalls();
        combiningPatternsAndGuards();
    }
}
```

```java
public class RecordPatterns {
    
    /**
     * Basic record patterns
     */
    public static void basicRecordPatterns() {
        System.out.println("=== Basic Record Patterns ===");
        
        record Point(int x, int y) {}
        
        Object obj = new Point(3, 4);
        
        String result = switch (obj) {
            case Point(int x, int y) -> 
                "Point at (" + x + ", " + y + ")";
            case null -> 
                "Null";
            default -> 
                "Not a point";
        };
        
        System.out.println(result);
        
        /**
         * RECORD PATTERN SYNTAX:
         * 
         * case RecordType(component1, component2) -> expression
         * 
         * Deconstruction:
         * - Extracts components
         * - Creates pattern variables
         * - Available in case expression
         * 
         * Example:
         * record Point(int x, int y) {}
         * case Point(int x, int y) -> ...
         * 
         * Components x and y available!
         */
    }
    
    /**
     * Nested record patterns
     */
    public static void nestedRecordPatterns() {
        System.out.println("\n=== Nested Record Patterns ===");
        
        record Point(int x, int y) {}
        record Circle(Point center, double radius) {}
        record Rectangle(Point topLeft, Point bottomRight) {}
        
        Object obj = new Circle(new Point(0, 0), 5.0);
        
        String result = switch (obj) {
            case Circle(Point(int x, int y), double r) -> 
                "Circle at (" + x + ", " + y + ") with radius " + r;
            case Rectangle(Point(int x1, int y1), Point(int x2, int y2)) -> 
                "Rectangle from (" + x1 + ", " + y1 + ") to (" + x2 + ", " + y2 + ")";
            case null -> 
                "Null";
            default -> 
                "Unknown shape";
        };
        
        System.out.println(result);
        
        /**
         * NESTED PATTERNS:
         * 
         * Can deconstruct nested records:
         * case Circle(Point(int x, int y), double r) -> ...
         * 
         * Deconstructs:
         * 1. Circle to (center, radius)
         * 2. center (Point) to (x, y)
         * 3. All components available
         * 
         * Before (without record patterns):
         * case Circle c -> {
         *   Point center = c.center();
         *   int x = center.x();
         *   int y = center.y();
         *   double r = c.radius();
         *   ...
         * }
         * 
         * After (with record patterns):
         * case Circle(Point(int x, int y), double r) -> ...
         * 
         * Much cleaner!
         */
    }
    
    /**
     * Partial patterns and wildcards
     */
    public static void partialPatternsAndWildcards() {
        System.out.println("\n=== Partial Patterns ===");
        
        record Person(String name, int age, String city) {}
        
        Object obj = new Person("Alice", 25, "NYC");
        
        String result = switch (obj) {
            // Only care about name and age
            case Person(String n, int a, _) when a < 18 -> 
                n + " is a minor";
            
            // Only care about name and city
            case Person(String n, _, String c) -> 
                n + " lives in " + c;
            
            case null -> 
                "Null";
            default -> 
                "Not a person";
        };
        
        System.out.println(result);
        
        /**
         * WILDCARDS (_):
         * 
         * Use _ to ignore components:
         * case Person(String name, _, _) -> ...
         * 
         * Only extracts what you need:
         * case Circle(_, double radius) -> ...
         * case Rectangle(Point(int x, _), _) -> ...
         * 
         * Benefits:
         * - Clear intent
         * - Less noise
         * - Compiler validates structure
         * - Still type-safe
         */
    }
    
    /**
     * Record patterns with guards
     */
    public static void recordPatternsWithGuards() {
        System.out.println("\n=== Record Patterns with Guards ===");
        
        record Order(String id, double amount, String status) {}
        
        Object obj = new Order("ORD-123", 150.0, "PENDING");
        
        String action = switch (obj) {
            case Order(String id, double amt, String status) 
                when amt > 100 && status.equals("PENDING") -> 
                "Process high-value order: " + id;
            
            case Order(String id, double amt, _) 
                when amt < 10 -> 
                "Reject small order: " + id;
            
            case Order(String id, _, String status) 
                when status.equals("CANCELLED") -> 
                "Archive cancelled order: " + id;
            
            case Order(String id, _, _) -> 
                "Standard processing: " + id;
            
            default -> 
                "Unknown";
        };
        
        System.out.println(action);
        
        /**
         * RECORD PATTERNS + GUARDS:
         * 
         * Powerful combination:
         * 
         * case Order(String id, double amt, String status)
         *   when amt > 100 && status.equals("PENDING") -> ...
         * 
         * 1. Deconstruct Order
         * 2. Extract components
         * 3. Check guard condition
         * 4. Execute if all match
         * 
         * Benefits:
         * ✓ Direct access to components
         * ✓ Clear conditions
         * ✓ Type-safe
         * ✓ Concise
         */
    }
    
    /**
     * Complex example: Expression evaluation
     */
    public static void expressionEvaluation() {
        System.out.println("\n=== Expression Evaluation ===");
        
        sealed interface Expr {}
        record Const(int value) implements Expr {}
        record Add(Expr left, Expr right) implements Expr {}
        record Mul(Expr left, Expr right) implements Expr {}
        record Neg(Expr expr) implements Expr {}
        
        // (2 + 3) * 4
        Expr expression = new Mul(
            new Add(new Const(2), new Const(3)),
            new Const(4)
        );
        
        int result = eval(expression);
        System.out.println("Result: " + result);
    }
    
    private static int eval(Expr expr) {
        return switch (expr) {
            case Const(int value) -> 
                value;
            
            case Add(Expr left, Expr right) -> 
                eval(left) + eval(right);
            
            case Mul(Expr left, Expr right) -> 
                eval(left) * eval(right);
            
            case Neg(Expr e) -> 
                -eval(e);
        };
        
        /**
         * RECURSIVE RECORD PATTERNS:
         * 
         * Can process recursive structures:
         * case Add(Expr left, Expr right) -> 
         *   eval(left) + eval(right)
         * 
         * Pattern matches and deconstructs
         * Recursively processes components
         * 
         * Perfect for:
         * - Abstract syntax trees
         * - Expression evaluators
         * - Tree traversal
         * - Recursive data structures
         */
    }
    
    /**
     * Generic record patterns
     */
    public static void genericRecordPatterns() {
        System.out.println("\n=== Generic Record Patterns ===");
        
        record Box<T>(T value) {}
        
        Object obj = new Box<>("Hello");
        
        String result = switch (obj) {
            case Box(String s) -> 
                "Box contains string: " + s;
            case Box(Integer i) -> 
                "Box contains integer: " + i;
            case Box(var v) -> 
                "Box contains: " + v;
            case null -> 
                "Null";
            default -> 
                "Not a box";
        };
        
        System.out.println(result);
        
        /**
         * GENERIC RECORDS:
         * 
         * Can pattern match generic records:
         * case Box(String s) -> ...
         * case Box(Integer i) -> ...
         * case Box(var v) -> ...  // Type inference
         * 
         * Type inference with var:
         * case Box(var value) -> ...
         * 
         * Compiler infers type from:
         * - Generic type parameter
         * - Runtime type information
         * - Context
         */
    }
    
    public static void main(String[] args) {
        basicRecordPatterns();
        nestedRecordPatterns();
        partialPatternsAndWildcards();
        recordPatternsWithGuards();
        expressionEvaluation();
        genericRecordPatterns();
    }
}
```

## 4. Exhaustiveness and Coverage

### Compiler-Verified Complete Matching

```java
/**
 * EXHAUSTIVENESS AND COVERAGE
 * 
 * Compiler checking for completeness
 */

public class ExhaustivenessAndCoverage {
    
    /**
     * Exhaustiveness with sealed types
     */
    public static void exhaustivenessWithSealed() {
        System.out.println("=== Exhaustiveness with Sealed ===");
        
        sealed interface Status permits Active, Inactive, Pending {}
        record Active(String userId) implements Status {}
        record Inactive(String reason) implements Status {}
        record Pending(long timestamp) implements Status {}
        
        Status status = new Active("user123");
        
        // Exhaustive - no default needed!
        String message = switch (status) {
            case Active a -> "User active: " + a.userId();
            case Inactive i -> "User inactive: " + i.reason();
            case Pending p -> "User pending since " + p.timestamp();
        };
        
        System.out.println(message);
        
        /**
         * EXHAUSTIVENESS:
         * 
         * With sealed types, compiler knows all subtypes:
         * sealed interface Status permits Active, Inactive, Pending
         * 
         * Switch must handle all:
         * case Active a -> ...
         * case Inactive i -> ...
         * case Pending p -> ...
         * 
         * No default needed!
         * Compiler verifies completeness
         * 
         * If you forget a case:
         * switch (status) {
         *   case Active a -> ...
         *   case Inactive i -> ...
         *   // Missing Pending - COMPILATION ERROR!
         * }
         * 
         * Error: "switch expression does not cover all 
         *         possible input values"
         */
    }
    
    /**
     * Adding new sealed subtypes
     */
    public static void addingNewSubtypes() {
        /**
         * SAFE EVOLUTION:
         * 
         * Initial:
         * sealed interface Result permits Success, Failure {}
         * 
         * switch (result) {
         *   case Success s -> ...
         *   case Failure f -> ...
         * }
         * 
         * Add new subtype:
         * sealed interface Result 
         *   permits Success, Failure, Pending {}
         * 
         * ALL switches break!
         * switch (result) {
         *   case Success s -> ...
         *   case Failure f -> ...
         *   // ERROR: Missing case Pending
         * }
         * 
         * Must add:
         * case Pending p -> ...
         * 
         * Compiler finds ALL usages
         * Forces updates everywhere
         * Prevents bugs at compile time
         * 
         * This is type-safe evolution!
         */
    }
    
    /**
     * Non-sealed and coverage
     */
    public static void nonSealedCoverage() {
        System.out.println("\n=== Non-Sealed Coverage ===");
        
        // Not sealed - open hierarchy
        interface Shape {}
        record Circle(double radius) implements Shape {}
        record Square(double side) implements Shape {}
        // Anyone can add more!
        
        Shape shape = new Circle(5.0);
        
        // Must have default - not exhaustive
        String description = switch (shape) {
            case Circle c -> "Circle with radius " + c.radius();
            case Square s -> "Square with side " + s.side();
            default -> "Unknown shape";  // Required!
        };
        
        System.out.println(description);
        
        /**
         * NON-SEALED TYPES:
         * 
         * Open hierarchy:
         * - Unknown subtypes possible
         * - Cannot be exhaustive
         * - Must have default case
         * 
         * switch (shape) {
         *   case Circle c -> ...
         *   case Square s -> ...
         *   // default required - compiler doesn't know all types
         * }
         * 
         * Without default:
         * Compilation error if someone adds new Shape
         */
    }
    
    /**
     * Coverage for primitives and objects
     */
    public static void coverageForTypes() {
        /**
         * COVERAGE RULES:
         * 
         * Sealed types:
         * - Exhaustive possible
         * - No default needed
         * - Compiler knows all subtypes
         * 
         * Example:
         * sealed interface Result permits Ok, Err {}
         * switch (result) {
         *   case Ok ok -> ...
         *   case Err err -> ...
         * }
         * 
         * 
         * Enums:
         * - Exhaustive possible
         * - No default needed
         * - Compiler knows all values
         * 
         * Example:
         * enum Day { MON, TUE, WED }
         * switch (day) {
         *   case MON -> ...
         *   case TUE -> ...
         *   case WED -> ...
         * }
         * 
         * 
         * Open types:
         * - NOT exhaustive
         * - Default required
         * - Unknown subtypes
         * 
         * Example:
         * Object obj = ...;
         * switch (obj) {
         *   case String s -> ...
         *   case Integer i -> ...
         *   default -> ...  // Required!
         * }
         * 
         * 
         * Null:
         * - Must handle explicitly or with default
         * - null case or default
         * 
         * Example:
         * switch (value) {
         *   case String s -> ...
         *   case null -> ...  // Explicit
         * }
         * 
         * OR:
         * 
         * switch (value) {
         *   case String s -> ...
         *   default -> ...  // Catches null
         * }
         */
    }
    
    /**
     * Hierarchical exhaustiveness
     */
    public static void hierarchicalExhaustiveness() {
        System.out.println("\n=== Hierarchical Exhaustiveness ===");
        
        sealed interface Animal permits Mammal, Bird {}
        sealed interface Mammal extends Animal permits Dog, Cat {}
        record Dog(String name) implements Mammal {}
        record Cat(String name) implements Mammal {}
        final class Bird implements Animal {}
        
        Animal animal = new Dog("Rex");
        
        // Option 1: Match leaf types (exhaustive)
        String sound1 = switch (animal) {
            case Dog d -> "Woof";
            case Cat c -> "Meow";
            case Bird b -> "Tweet";
        };
        
        // Option 2: Match intermediate types (exhaustive)
        String sound2 = switch (animal) {
            case Mammal m -> "Mammal sound";
            case Bird b -> "Tweet";
        };
        
        System.out.println("Sound: " + sound1);
        System.out.println("Sound: " + sound2);
        
        /**
         * HIERARCHICAL MATCHING:
         * 
         * Can match at any level:
         * 
         * Leaf level:
         * case Dog d -> ...
         * case Cat c -> ...
         * case Bird b -> ...
         * 
         * Intermediate level:
         * case Mammal m -> ...  // Covers Dog and Cat
         * case Bird b -> ...
         * 
         * Mixed level:
         * case Dog d -> ...     // Specific
         * case Mammal m -> ...  // Covers Cat
         * case Bird b -> ...
         * 
         * All exhaustive with sealed types!
         */
    }
    
    /**
     * Exhaustiveness with guards
     */
    public static void exhaustivenessWithGuards() {
        System.out.println("\n=== Exhaustiveness with Guards ===");
        
        sealed interface Result permits Success, Failure {}
        record Success(String data) implements Result {}
        record Failure(String error) implements Result {}
        
        Result result = new Success("OK");
        
        // Guards don't affect exhaustiveness
        String message = switch (result) {
            case Success s when !s.data().isEmpty() -> "Got: " + s.data();
            case Success s -> "Empty success";
            case Failure f -> "Error: " + f.error();
            // Exhaustive - all types covered
        };
        
        System.out.println(message);
        
        /**
         * GUARDS AND EXHAUSTIVENESS:
         * 
         * Guards don't affect exhaustiveness checking:
         * 
         * switch (result) {
         *   case Success s when condition -> ...
         *   case Success s -> ...  // Catch-all for Success
         *   case Failure f -> ...
         * }
         * 
         * Must still cover all types
         * Each type needs unguarded case
         * 
         * Wrong (not exhaustive):
         * switch (result) {
         *   case Success s when s.data().isEmpty() -> ...
         *   case Failure f -> ...
         *   // Missing: unguarded Success case!
         * }
         * 
         * Right (exhaustive):
         * switch (result) {
         *   case Success s when s.data().isEmpty() -> ...
         *   case Success s -> ...  // Covers other Success
         *   case Failure f -> ...
         * }
         */
    }
    
    public static void main(String[] args) {
        exhaustivenessWithSealed();
        nonSealedCoverage();
        hierarchicalExhaustiveness();
        exhaustivenessWithGuards();
    }
}
```

```java
/**

public class NullHandlingInSwitch {
    
    /**
     * Traditional null problem
     */
    public static void traditionalNullProblem() {
        /**
         * BEFORE JAVA 21:
         * 
         * String s = null;
         * switch (s) {
         *   case "hello" -> ...
         *   case "world" -> ...
         * }
         * 
         * Result: NullPointerException!
         * 
         * Must check before switch:
         * if (s == null) {
         *   // handle null
         * } else {
         *   switch (s) {
         *     case "hello" -> ...
         *     case "world" -> ...
         *   }
         * }
         * 
         * Verbose and error-prone!
         */
    }
    
    /**
     * Null case label
     */
    public static void nullCaseLabel() {
        System.out.println("=== Null Case Label ===");
        
        String value = null;
        
        String result = switch (value) {
            case null -> "Null value";
            case "hello" -> "Hello";
            case "world" -> "World";
            default -> "Other: " + value;
        };
        
        System.out.println(result);
        
        /**
         * NULL CASE:
         * 
         * Explicit null handling:
         * case null -> expression
         * 
         * Benefits:
         * ✓ No NullPointerException
         * ✓ Explicit handling
         * ✓ Part of switch
         * ✓ Clear intent
         * 
         * Order doesn't matter:
         * case null -> ...
         * case String s -> ...
         * 
         * OR:
         * 
         * case String s -> ...
         * case null -> ...
         * 
         * Both work!
         */
    }
    
    /**
     * Null with type patterns
     */
    public static void nullWithTypePatterns() {
        System.out.println("\n=== Null with Type Patterns ===");
        
        Object obj = null;
        
        String result = switch (obj) {
            case null -> "Null object";
            case String s -> "String: " + s;
            case Integer i -> "Integer: " + i;
            default -> "Other";
        };
        
        System.out.println(result);
        
        /**
         * NULL AND TYPE PATTERNS:
         * 
         * Null case before patterns:
         * case null -> ...
         * case String s -> ...
         * 
         * Without null case:
         * case String s -> ...  // Doesn't match null
         * default -> ...        // Catches null
         * 
         * Type pattern doesn't match null:
         * case String s -> ...  // null fails this
         */
    }
    
    /**
     * Combining null with default
     */
    public static void combiningNullWithDefault() {
        System.out.println("\n=== Null with Default ===");
        
        String value = null;
        
        // Option 1: Explicit null case
        String result1 = switch (value) {
            case null -> "Null";
            case "test" -> "Test";
            default -> "Other";
        };
        
        // Option 2: Default catches null
        String result2 = switch (value) {
            case "test" -> "Test";
            default -> "Null or other";
        };
        
        System.out.println("Explicit: " + result1);
        System.out.println("Default: " + result2);
        
        /**
         * NULL HANDLING OPTIONS:
         * 
         * Explicit null case:
         * case null -> "Handle null"
         * case "value" -> "Handle value"
         * 
         * Default catches null:
         * case "value" -> "Handle value"
         * default -> "Handle null or unknown"
         * 
         * 
         * CHOOSE EXPLICIT WHEN:
         * - Null is expected
         * - Different handling for null
         * - Clear documentation
         * 
         * CHOOSE DEFAULT WHEN:
         * - Null is rare
         * - Same handling as unknown
         * - Simpler code
         */
    }
    
    /**
     * Null with sealed types
     */
    public static void nullWithSealedTypes() {
        System.out.println("\n=== Null with Sealed Types ===");
        
        sealed interface Result permits Success, Failure {}
        record Success(String data) implements Result {}
        record Failure(String error) implements Result {}
        
        Result result = null;
        
        String message = switch (result) {
            case null -> "No result";
            case Success s -> "Success: " + s.data();
            case Failure f -> "Failure: " + f.error();
        };
        
        System.out.println(message);
        
        /**
         * NULL WITH SEALED TYPES:
         * 
         * Sealed types don't include null:
         * sealed interface Result permits Success, Failure
         * 
         * Must handle null separately:
         * case null -> ...
         * case Success s -> ...
         * case Failure f -> ...
         * 
         * Or use default:
         * case Success s -> ...
         * case Failure f -> ...
         * default -> ...  // Catches null
         * 
         * Without either:
         * case Success s -> ...
         * case Failure f -> ...
         * // Error: doesn't cover null!
         */
    }
    
    /**
     * Null and total patterns
     */
    public static void nullAndTotalPatterns() {
        System.out.println("\n=== Null and Total Patterns ===");
        
        Object obj = null;
        
        // Total pattern covers everything including null
        String result = switch (obj) {
            case String s -> "String: " + s;
            case Integer i -> "Integer: " + i;
            case null, default -> "Null or unknown";
        };
        
        System.out.println(result);
        
        /**
         * TOTAL PATTERNS:
         * 
         * Pattern that matches everything:
         * case Object o -> ...
         * 
         * Matches all non-null objects
         * Does NOT match null
         * 
         * To be truly total:
         * case null, default -> ...
         * 
         * OR:
         * 
         * case Object o -> ...
         * case null -> ...
         * 
         * 
         * COMBINING NULL AND DEFAULT:
         * 
         * case null, default -> ...
         * 
         * Matches:
         * - null
         * - Anything not matched above
         * 
         * Useful for catch-all case
         */
    }
    
    /**
     * Null safety best practices
     */
    public static void nullSafetyBestPractices() {
        /**
         * NULL SAFETY BEST PRACTICES:
         * 
         * 1. Explicit null case when expected:
         * switch (value) {
         *   case null -> handleNull()
         *   case String s -> handleString(s)
         *   default -> handleOther()
         * }
         * 
         * 2. Use sealed types to avoid null:
         * sealed interface Option<T> permits Some, None
         * // No null needed!
         * 
         * 3. Document null expectations:
         * /**
         *  * @param value nullable string
         *  *\/
         * String process(String value) {
         *   return switch (value) {
         *     case null -> "default"
         *     case String s -> s.toUpperCase()
         *   };
         * }
         * 
         * 4. Consider Optional for return types:
         * Optional<String> getValue() {
         *   // Never returns null
         * }
         * 
         * 5. Fail fast if null unexpected:
         * switch (Objects.requireNonNull(value)) {
         *   case String s -> ...
         *   // No null case - null is error
         * }
         * 
         * 
         * ANTI-PATTERNS:
         * 
         * ✗ Silent null handling:
         * case null -> null  // Don't hide null!
         * 
         * ✗ Default catches null without documenting:
         * default -> ...  // Unclear if null expected
         * 
         * ✓ Explicit is better:
         * case null -> handleNull()
         * default -> handleUnknown()
         */
    }
    
    public static void main(String[] args) {
        nullCaseLabel();
        nullWithTypePatterns();
        combiningNullWithDefault();
        nullWithSealedTypes();
        nullAndTotalPatterns();
    }
}
```

```java
import java.time.*;
import java.util.*;

public class RealWorldExamples {
    
    /**
     * Example 1: JSON processing
     */
    public static void jsonProcessing() {
        System.out.println("=== JSON Processing ===");
        
        sealed interface JsonValue {}
        record JsonString(String value) implements JsonValue {}
        record JsonNumber(double value) implements JsonValue {}
        record JsonBoolean(boolean value) implements JsonValue {}
        record JsonNull() implements JsonValue {}
        record JsonArray(List<JsonValue> values) implements JsonValue {}
        record JsonObject(Map<String, JsonValue> fields) implements JsonValue {}
        
        JsonValue json = new JsonObject(Map.of(
            "name", new JsonString("Alice"),
            "age", new JsonNumber(25),
            "active", new JsonBoolean(true)
        ));
        
        String formatted = formatJson(json, 0);
        System.out.println(formatted);
    }
    
    private static String formatJson(JsonValue json, int indent) {
        String indentStr = "  ".repeat(indent);
        return switch (json) {
            case JsonString(String s) -> "\"" + s + "\"";
            case JsonNumber(double n) -> String.valueOf(n);
            case JsonBoolean(boolean b) -> String.valueOf(b);
            case JsonNull() -> "null";
            case JsonArray(List<JsonValue> values) -> {
                if (values.isEmpty()) yield "[]";
                String items = values.stream()
                    .map(v -> formatJson(v, indent + 1))
                    .reduce((a, b) -> a + ",\n" + indentStr + "  " + b)
                    .orElse("");
                yield "[\n" + indentStr + "  " + items + "\n" + indentStr + "]";
            }
            case JsonObject(Map<String, JsonValue> fields) -> {
                if (fields.isEmpty()) yield "{}";
                String items = fields.entrySet().stream()
                    .map(e -> "\"" + e.getKey() + "\": " + formatJson(e.getValue(), indent + 1))
                    .reduce((a, b) -> a + ",\n" + indentStr + "  " + b)
                    .orElse("");
                yield "{\n" + indentStr + "  " + items + "\n" + indentStr + "}";
            }
        };
    }
    
    /**
     * Example 2: HTTP request routing
     */

                handleDeleteUser(path.substring(7));

            new Number(3),
            new Paren(')'),
            new Operator('*'),
            new Number(4)

        for (Token token : tokens) {

                case Number(double v) when v < 0 ->

                case Number(double v) when v == Math.floor(v) -> 
                    "Integer: " + (int) v;
                case Number(double v) ->

                case Operator(char op) -> 
                    "Unknown operator: " + op;
                case Paren(char p) -> 
                    (p == '(' ? "Open" : "Close") + " paren";
            };
            System.out.println(description);
        }
    }
```

## Summary

### Quick Reference

**Basic Type Pattern:**

```java
switch (obj) {
    case String s -> "String: " + s;
    case Integer i -> "Integer: " + i;
    case null -> "Null";
    default -> "Other";
}
```

**Guarded Pattern:**

```java
switch (obj) {
    case String s when s.length() > 10 -> "Long string";
    case String s -> "Short string";
}
```

**Record Pattern:**

```java
switch (shape) {
    case Circle(Point(int x, int y), double r) -> 
        "Circle at (" + x + ", " + y + ")";
    case Rectangle(Point p1, Point p2) -> 
        "Rectangle";
}
```

**Sealed + Exhaustive:**

```java
sealed interface Result permits Success, Failure {}

switch (result) {
    case Success s -> "OK";
    case Failure f -> "Error";
    // No default needed - exhaustive!
}
```

---

**Key insights:**

- **Type patterns**: Match and bind in one step
- **Guards**: Add conditions with when clause
- **Record patterns**: Deconstruct inline
- **Exhaustiveness**: No default needed for sealed types
- **Null handling**: Explicit case null label
- **Composition**: Patterns + guards + records = powerful!

**The revolution:**

```
Before Pattern Matching:
if (obj instanceof String) {
    String s = (String) obj;
    if (s.length() > 10) {
        return "Long: " + s;
    } else {
        return "Short: " + s;
    }
} else if (obj instanceof Integer) {
    Integer i = (Integer) obj;
    return "Number: " + i;
} else {
    return "Unknown";
}

After Pattern Matching:
return switch (obj) {
    case String s when s.length() > 10 -> "Long: " + s;
    case String s -> "Short: " + s;
    case Integer i -> "Number: " + i;
    default -> "Unknown";
};

From 13 lines to 6 lines
From complex to clear
From error-prone to type-safe
```
