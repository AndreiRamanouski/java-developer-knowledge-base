# New String Methods and Compact Strings (Java 9-11)

## Overview

Java 9-11 introduced significant improvements to the String class, both in terms of new utility methods and internal optimizations. New methods like `isBlank()`, `lines()`, `strip()`, `repeat()`, and `indent()` make common string operations more convenient and expressive. Under the hood, Java 9's Compact Strings feature automatically optimizes memory usage by using byte arrays instead of char arrays for Latin-1 compatible strings, reducing memory footprint by ~50% for typical applications. This guide covers all new String methods introduced in Java 9-11, explains the Compact Strings optimization and when it applies, explores String deduplication for reducing duplicate strings in the heap, and examines the performance improvements these features bring to real-world applications.

```java
import java.util.stream.*;
import java.util.*;

public class BasicStringMethods {
    
    /**
     * isBlank() - Check if string is empty or whitespace
     */
    public static void isBlankExamples() {
        System.out.println("=== isBlank() ===");
        
        // Empty string
        String empty = "";
        System.out.println("Empty isBlank: " + empty.isBlank());  // true
        
        // Whitespace only
        String whitespace = "   \t\n  ";
        System.out.println("Whitespace isBlank: " + whitespace.isBlank());  // true
        
        // Has content
        String content = "  hello  ";
        System.out.println("Content isBlank: " + content.isBlank());  // false
        
        // Comparison with isEmpty()
        System.out.println("\nisEmpty() vs isBlank():");
        System.out.println("\"\" isEmpty: " + "".isEmpty());           // true
        System.out.println("\"\" isBlank: " + "".isBlank());           // true
        System.out.println("\"   \" isEmpty: " + "   ".isEmpty());     // false
        System.out.println("\"   \" isBlank: " + "   ".isBlank());     // true
        
        /**
         * ISBLANK():
         * 
         * Java 11 addition
         * 
         * Returns true if:
         * - String is empty ("")
         * - String contains only whitespace
         * 
         * Whitespace includes:
         * - Space ' '
         * - Tab '\t'
         * - Newline '\n'
         * - Return '\r'
         * - Other Unicode whitespace
         * 
         * vs isEmpty():
         * isEmpty(): true only if length() == 0
         * isBlank(): true if isEmpty() or all whitespace
         * 
         * Use cases:
         * - Validating user input
         * - Checking if string has meaningful content
         * - Form validation
         */
    }
    
    /**
     * lines() - Split string into lines
     */
    public static void linesExamples() {
        System.out.println("\n=== lines() ===");
        
        String multiline = """
            Line 1
            Line 2
            Line 3
            """;
        
        // Stream of lines
        Stream<String> lines = multiline.lines();
        System.out.println("Lines:");
        lines.forEach(line -> System.out.println("  - " + line));
        
        // Count lines
        long count = multiline.lines().count();
        System.out.println("Line count: " + count);
        
        // Filter and process
        multiline.lines()
            .filter(line -> !line.isBlank())
            .map(String::toUpperCase)
            .forEach(System.out::println);
        
        // Different line separators
        String mixed = "Line1\nLine2\rLine3\r\nLine4";
        System.out.println("\nMixed separators:");
        mixed.lines().forEach(System.out::println);
        
        /**
         * LINES():
         * 
         * Java 11 addition
         * 
         * Returns: Stream<String>
         * 
         * Splits on:
         * - \n (LF - Unix)
         * - \r (CR - old Mac)
         * - \r\n (CRLF - Windows)
         * 
         * Benefits:
         * - Lazy evaluation
         * - Memory efficient
         * - Stream operations
         * 
         * vs split("\n"):
         * split("\n"): Array, doesn't handle \r\n
         * lines(): Stream, handles all line separators
         * 
         * Use cases:
         * - Processing log files
         * - Reading CSV/text data
         * - Analyzing multi-line input
         */
    }
    
    /**
     * strip() - Remove leading/trailing whitespace
     */
    public static void stripExamples() {
        System.out.println("\n=== strip() ===");
        
        String text = "  \t  hello world  \n  ";
        
        // strip() - removes all whitespace
        String stripped = text.strip();
        System.out.println("Stripped: '" + stripped + "'");
        
        // stripLeading() - removes leading whitespace
        String stripLeading = text.stripLeading();
        System.out.println("StripLeading: '" + stripLeading + "'");
        
        // stripTrailing() - removes trailing whitespace
        String stripTrailing = text.stripTrailing();
        System.out.println("StripTrailing: '" + stripTrailing + "'");
        
        // Comparison with trim()
        String unicode = "\u2000hello\u2000";  // Unicode space
        System.out.println("\nUnicode spaces:");
        System.out.println("trim(): '" + unicode.trim() + "'");
        System.out.println("strip(): '" + unicode.strip() + "'");
        
        /**
         * STRIP():
         * 
         * Java 11 addition
         * 
         * strip(): Remove leading and trailing whitespace
         * stripLeading(): Remove leading whitespace
         * stripTrailing(): Remove trailing whitespace
         * 
         * vs trim():
         * 
         * trim():
         * - Only removes ASCII whitespace (≤ U+0020)
         * - Based on char <= ' '
         * - Legacy method
         * 
         * strip():
         * - Removes all Unicode whitespace
         * - Based on Character.isWhitespace()
         * - Modern, recommended
         * 
         * Whitespace examples:
         * ASCII: space, tab, newline
         * Unicode: U+2000 (en quad), U+3000 (ideographic space)
         * 
         * Use cases:
         * - Cleaning user input
         * - Parsing data
         * - Internationalized text
```

```java
    /**
     * repeat() - Repeat string n times
     */
    public static void repeatExamples() {

        String dash = "-".repeat(50);

        String stars = "*".repeat(10);

        System.out.println("'" + padded + "'");
        
        // Line separators
        String separator = "=".repeat(40);
        System.out.println(separator);
        System.out.println("SECTION TITLE");
        System.out.println(separator);
        
        /**
         * REPEAT():
         * 
         * Java 11 addition
         * 
         * Syntax:
         * String repeated = string.repeat(count)
         * 
         * Parameters:
         * count: Number of times to repeat (≥ 0)
         * 
         * Returns:
         * New string with count repetitions
         * 
         * Special cases:
         * repeat(0): Returns ""
         * repeat(1): Returns string itself
         * 
         * Throws:
         * IllegalArgumentException if count < 0
         * 
         * Use cases:
         * - Padding strings
         * - Drawing borders
         * - Creating separators
         * - Building test data
         * - ASCII art

    }
```

## 3. Compact Strings (Java 9)

### Internal String Optimization

```java
/**
 * COMPACT STRINGS
 * 
 * Java 9 internal string optimization
 */

public class CompactStrings {
    
    /**
     * Understanding Compact Strings
     */
    public static void understandingCompactStrings() {
        System.out.println("=== Compact Strings ===");
        
        /**
         * BEFORE JAVA 9:
         * 
         * String storage:
         * private final char[] value;
         * 
         * Each char: 2 bytes (UTF-16)
         * 
         * Example:
         * String s = "Hello";
         * Memory: 5 chars × 2 bytes = 10 bytes
         * 
         * Problem:
         * - Most strings are ASCII/Latin-1
         * - Only need 1 byte per character
         * - Wasting 50% of memory
         * 
         * 
         * JAVA 9+ (COMPACT STRINGS):
         * 
         * String storage:
         * private final byte[] value;
         * private final byte coder;  // LATIN1 or UTF16
         * 
         * Latin-1 string (1 byte per char):
         * String s = "Hello";
         * Memory: 5 bytes + 1 byte (coder) = 6 bytes
         * Savings: 40%
         * 
         * UTF-16 string (2 bytes per char):
         * String s = "Hello世界";
         * Memory: 14 bytes + 1 byte (coder) = 15 bytes
         * 
         * Automatic:
         * - JVM decides encoding
         * - Latin-1 if possible (ASCII compatible)
         * - UTF-16 if needed (any char > 255)
         * - Completely transparent
         */
        
        // Latin-1 compatible (compact)
        String latin1 = "Hello World";  // 11 bytes
        
        // UTF-16 required (not compact)
        String unicode = "Hello 世界";  // 2 bytes per char
        
        System.out.println("Latin-1: " + latin1);
        System.out.println("Unicode: " + unicode);
    }
    
    /**
     * When Compact Strings apply
     */
    public static void whenCompactApplies() {
        System.out.println("\n=== When Compact Applies ===");
        
        // Latin-1 compatible (0-255)
        String ascii = "ABCabc123!@#";           // Compact
        String latin1 = "café résumé";           // Compact (Latin-1)
        String extended = "Hello ñ ç ü";         // Compact
        
        // Requires UTF-16 (> 255)
        String chinese = "你好";                  // UTF-16
        String japanese = "こんにちは";           // UTF-16
        String emoji = "Hello 😀";               // UTF-16
        String mixed = "Hello世界";               // UTF-16
        
        /**
         * COMPACT STRING CRITERIA:
         * 
         * Uses Latin-1 (1 byte) if:
         * - All characters in range 0-255
         * - Includes ASCII (0-127)
         * - Includes extended Latin-1 (128-255)
         * 
         * Examples:
         * ✓ "Hello"               (ASCII)
         * ✓ "café"                (Latin-1)
         * ✓ "ñ, ç, ü, ß"          (Latin-1)
         * ✗ "Hello世界"            (Chinese)
         * ✗ "Hello😀"              (Emoji)
         * ✗ Any char > 255
         * 
         * Detection:
         * JVM scans string during creation
         * Chooses encoding automatically
         * No API to check encoding
         */
    }
    
    /**
     * Memory savings example
     */
    public static void memorySavings() {
        System.out.println("\n=== Memory Savings ===");
        
        /**
         * MEMORY CALCULATION:
         * 
         * Before Java 9 (char[]):
         * String overhead: 24 bytes (object + reference)
         * "Hello": 5 chars × 2 bytes = 10 bytes
         * Total: 34 bytes
         * 
         * Java 9+ Latin-1:
         * String overhead: 24 bytes
         * "Hello": 5 bytes
         * coder: 1 byte
         * Total: 30 bytes (12% smaller)
         * 
         * Java 9+ UTF-16:
         * String overhead: 24 bytes
         * "Hello世界": 14 bytes
         * coder: 1 byte
         * Total: 39 bytes
         * 
         * 
         * REAL-WORLD IMPACT:
         * 
         * Typical Java application:
         * - 20-30% of heap is Strings
         * - 80-90% of strings are Latin-1
         * - Compact Strings save ~25% of string memory
         * - Overall heap reduction: 5-7%
         * 
         * Example application:
         * Total heap: 1 GB
         * Strings: 300 MB
         * Latin-1: 270 MB
         * Savings: 135 MB
         * New heap: 865 MB (13.5% reduction)
         */
    }
    
    /**
     * Performance impact
     */
    public static void performanceImpact() {
        System.out.println("\n=== Performance Impact ===");
        
        // Benchmark string operations
        int iterations = 1_000_000;
        String latin1 = "Hello World";
        String unicode = "Hello 世界";
        
        // charAt() performance
        long start = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            char c = latin1.charAt(0);
        }
        long latin1Time = System.nanoTime() - start;
        
        start = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            char c = unicode.charAt(0);
        }
        long unicodeTime = System.nanoTime() - start;
        
        System.out.printf("Latin-1 charAt: %,d ns%n", latin1Time);
        System.out.printf("Unicode charAt: %,d ns%n", unicodeTime);
        
        /**
         * PERFORMANCE:
         * 
         * Latin-1 strings:
         * ✓ Faster - simpler byte access
         * ✓ Better cache utilization
         * ✓ Less memory bandwidth
         * 
         * UTF-16 strings:
         * ○ Slightly slower - need 2 bytes per char
         * ○ Similar to old implementation
         * 
         * Operations:
         * - charAt(): Similar speed
         * - substring(): Slightly faster (less copying)
         * - equals(): Slightly faster (byte comparison)
         * - indexOf(): Faster for Latin-1
         * 
         * Overall:
         * - Same or better performance
         * - Significant memory savings
         * - No downsides
         */
    }
    
    /**
     * Compact Strings configuration
     */
    public static void configuration() {
        /**
         * CONFIGURATION:
         * 
         * Enabled by default in Java 9+
         * 
         * Disable (not recommended):
         * -XX:-CompactStrings
         * 
         * Check if enabled:
         * java -XX:+PrintFlagsFinal -version | grep CompactStrings
         * 
         * Output:
         * bool CompactStrings = true
         * 
         * 
         * WHEN TO DISABLE:
         * 
         * Never in production!
         * 
         * Only for:
         * - Debugging
         * - Performance testing
         * - Comparing before/after
         * 
         * 
         * INTERNALS:
         * 
         * String.java:
         * private final byte[] value;
         * private final byte coder;
         * 
         * Coder values:
         * static final byte LATIN1 = 0;
         * static final byte UTF16 = 1;
         * 
         * Methods check coder:
         * public int length() {
         *   return value.length >> coder;
         *   // Latin-1: >> 0 (no shift)
         *   // UTF-16: >> 1 (divide by 2)
         * }
         */
    }
    
    public static void main(String[] args) {
        understandingCompactStrings();
        whenCompactApplies();
        memorySavings();
        performanceImpact();
    }
}
```

```java
/**

 */

public class StringDeduplication {
    
    /**
     * Understanding string deduplication
     */
    public static void understandingDeduplication() {
        System.out.println("=== String Deduplication ===");
        
        /**
         * THE PROBLEM:
         * 
         * Common scenario:
         * String s1 = new String("Hello");
         * String s2 = new String("Hello");
         * String s3 = new String("Hello");
         * 
         * Result:
         * - 3 String objects
         * - 3 separate char[]/byte[] arrays
         * - Same content: "Hello"
         * - Wasted memory
         * 
         * Memory usage:
         * 3 String objects: 72 bytes
         * 3 arrays: 30 bytes
         * Total: 102 bytes
         * 
         * Could be:
         * 3 String objects: 72 bytes
         * 1 shared array: 10 bytes
         * Total: 82 bytes
         * Savings: 20%
         * 
         * 
         * THE SOLUTION:
         * 
         * String deduplication:
         * - Part of G1 garbage collector
         * - Scans young generation
         * - Finds duplicate strings
         * - Points them to same array
         * 
         * After deduplication:
         * s1, s2, s3 all reference same byte[] array
         * 
         * 
         * HOW IT WORKS:
         * 
         * 1. GC runs
         * 2. Scans String objects
         * 3. Computes hash of content
         * 4. Finds duplicates
         * 5. Updates references to shared array
         * 6. Old arrays garbage collected
         */
    }

     * Enabling string deduplication

    public static void enablingDeduplication() {
        /**

         * REQUIREMENTS:

    public static void vsIntern() {
        /**
         * STRING.INTERN():
         * 
         * String s1 = new String("Hello").intern();
         * String s2 = new String("Hello").intern();

         * ✓ Manual control
         * 
         * Cons:
         * ✗ Must call explicitly
         * ✗ String pool can grow large
         * ✗ Never garbage collected (before Java 7)
         * ✗ Performance overhead
         * 
         * 
         * STRING DEDUPLICATION:
         * 
         * String s1 = new String("Hello");
         * String s2 = new String("Hello");

         * // but may share same byte[] array
         * 
         * Pros:
         * ✓ Automatic
         * ✓ No code changes
         * ✓ Strings can be GC'd
         * ✓ Only during GC
         * 
         * Cons:
         * ✗ G1 GC only
         * ✗ Cannot use == for comparison
         * ✗ Not guaranteed
         * ✗ Delayed (happens during GC)
         * 
         * 
         * WHEN TO USE EACH:
         * 
         * intern():
         * - Need object identity (==)
         * - Small set of known strings
         * - Performance-critical comparison
         * 
         * Deduplication:
         * - Large number of duplicates
         * - Automatic memory savings
         * - Can't modify code
         * - Using G1 GC
         * 
         * 
         * COMBINING:
         * Can use both together
         * intern() for critical strings
         * Deduplication for everything else

    }
```

```
strip(): All Unicode whitespace

split(): Array, doesn't handle \r\n
```
