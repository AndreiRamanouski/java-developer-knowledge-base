# HTTP/2 Client API (Java 11)

## Overview

Java 11 introduced a modern, fully-featured HTTP client that supports HTTP/1.1, HTTP/2, and WebSocket protocols. The new `java.net.http` package provides a fluent builder API for creating requests, supports both synchronous and asynchronous operations, handles responses with type-safe body handlers, and includes native HTTP/2 features like server push and stream multiplexing. This guide covers HttpClient creation and configuration, the difference between synchronous and asynchronous requests, using request builders for clean API calls, handling responses with various body types, WebSocket support for bidirectional communication, HTTP/2-specific features, and how this API compares to legacy HttpURLConnection and popular third-party libraries.

## 1. HttpClient Creation and Configuration

### Building and Configuring HTTP Clients

```java
/**
 * HTTPCLIENT CREATION
 * 
 * Modern HTTP client introduced in Java 11
 */

import java.net.http.*;
import java.net.*;
import java.time.Duration;
import javax.net.ssl.*;

public class HttpClientCreation {
    
    /**
     * Basic client creation
     */
    public static void basicCreation() {
        System.out.println("=== Basic Creation ===");
        
        // Simple default client
        HttpClient client = HttpClient.newHttpClient();
        
        System.out.println("Client created: " + client);
        System.out.println("Version: " + client.version());
        
        /**
         * HTTPCLIENT.NEWHTTPCLIENT():
         * 
         * Creates client with defaults:
         * - HTTP/2 preferred (falls back to HTTP/1.1)
         * - No redirect following
         * - Default SSL context
         * - Default executor
         * - No authenticator
         * - No cookie handler
         * 
         * Thread-safe: Can be shared across application
         * Immutable: Configuration cannot change
         */
    }
    
    /**
     * Builder configuration
     */
    public static void builderConfiguration() {
        System.out.println("\n=== Builder Configuration ===");
        
        HttpClient client = HttpClient.newBuilder()
            // HTTP version preference
            .version(HttpClient.Version.HTTP_2)
            
            // Redirect policy
            .followRedirects(HttpClient.Redirect.NORMAL)
            
            // Connection timeout
            .connectTimeout(Duration.ofSeconds(10))
            
            // Proxy
            .proxy(ProxySelector.of(
                new InetSocketAddress("proxy.example.com", 8080)))
            
            // Authenticator
            .authenticator(new Authenticator() {
                @Override
                protected PasswordAuthentication getPasswordAuthentication() {
                    return new PasswordAuthentication(
                        "user", 
                        "password".toCharArray()
                    );
                }
            })
            
            .build();
        
        System.out.println("Configured client created");
        
        /**
         * BUILDER OPTIONS:
         * 
         * version(Version):
         * - HTTP_1_1: Force HTTP/1.1
         * - HTTP_2: Prefer HTTP/2, fallback to HTTP/1.1
         * 
         * followRedirects(Redirect):
         * - NEVER: Don't follow (default)
         * - ALWAYS: Follow all redirects
         * - NORMAL: Follow except HTTPS→HTTP
         * 
         * connectTimeout(Duration):
         * - Timeout for establishing connection
         * - Applies to initial connection only
         * 
         * proxy(ProxySelector):
         * - Configure HTTP proxy
         * - Can be per-request or client-wide
         * 
         * authenticator(Authenticator):
         * - Handle authentication challenges
         * - Basic, Digest, etc.
         */
    }
    
    /**
     * HTTP version configuration
     */
    public static void httpVersion() {
        System.out.println("\n=== HTTP Version ===");
        
        // Prefer HTTP/2
        HttpClient http2Client = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .build();
        System.out.println("HTTP/2 preferred");
        
        // Force HTTP/1.1
        HttpClient http11Client = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_1_1)
            .build();
        System.out.println("HTTP/1.1 only");
        
        /**
         * HTTP VERSIONS:
         * 
         * HTTP/2:
         * - Binary protocol
         * - Multiplexing (multiple requests per connection)
         * - Server push
         * - Header compression
         * - Requires HTTPS (usually)
         * 
         * HTTP/1.1:
         * - Text protocol
         * - One request per connection
         * - No server push
         * - No header compression
         * - Works with HTTP and HTTPS
         * 
         * Default: Prefer HTTP/2, fallback to HTTP/1.1
         */
    }
    
    /**
     * Redirect policies
     */
    public static void redirectPolicies() {
        System.out.println("\n=== Redirect Policies ===");
        
        // Never follow redirects
        HttpClient neverClient = HttpClient.newBuilder()
            .followRedirects(HttpClient.Redirect.NEVER)
            .build();
        
        // Always follow redirects
        HttpClient alwaysClient = HttpClient.newBuilder()
            .followRedirects(HttpClient.Redirect.ALWAYS)
            .build();
        
        // Normal (except HTTPS to HTTP)
        HttpClient normalClient = HttpClient.newBuilder()
            .followRedirects(HttpClient.Redirect.NORMAL)
            .build();
        
        /**
         * REDIRECT POLICIES:
         * 
         * NEVER:
         * - Don't follow any redirects
         * - Return 3xx response as-is
         * - Application handles redirect
         * 
         * ALWAYS:
         * - Follow all redirects
         * - Including HTTPS → HTTP (security risk!)
         * 
         * NORMAL (recommended):
         * - Follow redirects
         * - Except HTTPS → HTTP (security)
         * - Max 5 redirects by default
         * 
         * Example:
         * http://example.com → http://example.com/new  ✓
         * https://example.com → https://other.com      ✓
         * https://example.com → http://other.com       ✗ (NORMAL)
         */
    }
    
    /**
     * Timeouts
     */
    public static void timeouts() {
        System.out.println("\n=== Timeouts ===");
        
        HttpClient client = HttpClient.newBuilder()
            // Connection timeout (client-level)
            .connectTimeout(Duration.ofSeconds(10))
            .build();
        
        /**
         * TIMEOUT TYPES:
         * 
         * Connect timeout (client):
         * - Time to establish connection
         * - Configured on HttpClient
         * - client.connectTimeout(Duration)
         * 
         * Request timeout (per-request):
         * - Time for entire request/response
         * - Configured on HttpRequest
         * - request.timeout(Duration)
         * 
         * Example:
         * HttpClient client = HttpClient.newBuilder()
         *   .connectTimeout(Duration.ofSeconds(5))  // 5s to connect
         *   .build();
         * 
         * HttpRequest request = HttpRequest.newBuilder()
         *   .uri(URI.create("https://example.com"))
         *   .timeout(Duration.ofSeconds(30))  // 30s for response
         *   .build();
         */
    }
    
    /**
     * SSL/TLS configuration
     */
    public static void sslConfiguration() throws Exception {
        System.out.println("\n=== SSL Configuration ===");
        
        // Custom SSL context
        SSLContext sslContext = SSLContext.getInstance("TLS");
        sslContext.init(null, null, null);
        
        HttpClient client = HttpClient.newBuilder()
            .sslContext(sslContext)
            .sslParameters(new SSLParameters())
            .build();
        
        System.out.println("SSL configured");
        
        /**
         * SSL OPTIONS:
         * 
         * sslContext(SSLContext):
         * - Custom SSL/TLS context
         * - For custom trust stores, client certs
         * 
         * sslParameters(SSLParameters):
         * - Cipher suites
         * - Protocols (TLSv1.2, TLSv1.3)
         * - Server name indication (SNI)
         * 
         * Example use cases:
         * - Self-signed certificates
         * - Client certificate authentication
         * - Custom cipher suites
         * - Certificate pinning
         */
    }
    
    /**
     * Connection pooling
     */
    public static void connectionPooling() {
        /**
         * CONNECTION POOLING:
         * 
         * Automatic:
         * - HttpClient automatically pools connections
         * - Reuses connections for efficiency
         * - HTTP/1.1: Connection per host
         * - HTTP/2: Single connection, multiple streams
         * 
         * Configuration:
         * - No explicit pool size setting
         * - Managed internally
         * - Connections closed when client garbage collected
         * 
         * Best practice:
         * - Share one HttpClient per application
         * - Don't create client per request
         * 
         * Good:
         * private static final HttpClient CLIENT = 
         *   HttpClient.newHttpClient();
         * 
         * Bad:
         * HttpClient client = HttpClient.newHttpClient();
         * // Use once, throw away
         */
    }

            //         return new PasswordAuthentication(

            //     }
            // })
            
            .build();
        
        System.out.println("Production-ready client created");
        
        /**
         * PRODUCTION CONFIGURATION:
         * 
         * Required:
         * ✓ Set timeouts (prevent hangs)

         * 
         * Optional:
         * ○ HTTP version (default is good)
         * ○ SSL context (for custom certs)
```

## 2. Synchronous vs Asynchronous Requests

### Blocking and Non-blocking HTTP Operations

```java
/**
 * SYNCHRONOUS VS ASYNCHRONOUS
 * 
 * Two ways to send HTTP requests
 */

import java.net.http.*;
import java.net.URI;
import java.util.concurrent.*;
import java.util.List;

public class SyncVsAsync {
    
    private static final HttpClient client = HttpClient.newHttpClient();
    
    /**
     * Synchronous request (blocking)
     */
    public static void synchronousRequest() throws Exception {
        System.out.println("=== Synchronous Request ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.github.com/users/octocat"))
            .GET()
            .build();
        
        System.out.println("Sending request...");
        
        // Blocks until response received
        HttpResponse<String> response = client.send(
            request, 
            HttpResponse.BodyHandlers.ofString()
        );
        
        System.out.println("Response received");
        System.out.println("Status: " + response.statusCode());
        System.out.println("Body length: " + response.body().length());
        
        /**
         * SYNCHRONOUS (send):
         * 
         * client.send(request, bodyHandler)
         * 
         * Characteristics:
         * - Blocks calling thread
         * - Waits for complete response
         * - Returns HttpResponse<T>
         * - Throws IOException on error
         * 
         * Use when:
         * ✓ Simple scripts
         * ✓ Sequential operations
         * ✓ Testing/debugging
         * ✓ Single request
         * 
         * Avoid when:
         * ✗ UI thread
         * ✗ High concurrency
         * ✗ Multiple requests
         */
    }
    
    /**
     * Asynchronous request (non-blocking)
     */
    public static void asynchronousRequest() throws Exception {
        System.out.println("\n=== Asynchronous Request ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.github.com/users/octocat"))
            .GET()
            .build();
        
        System.out.println("Sending async request...");
        
        // Returns immediately with CompletableFuture
        CompletableFuture<HttpResponse<String>> futureResponse = 
            client.sendAsync(
                request,
                HttpResponse.BodyHandlers.ofString()
            );
        
        System.out.println("Request sent, doing other work...");
        
        // Do other work while request is in flight
        Thread.sleep(100);
        System.out.println("Still doing other work...");
        
        // Wait for response
        HttpResponse<String> response = futureResponse.join();
        
        System.out.println("Response received");
        System.out.println("Status: " + response.statusCode());
        
        /**
         * ASYNCHRONOUS (sendAsync):
         * 
         * client.sendAsync(request, bodyHandler)
         * 
         * Characteristics:
         * - Returns immediately
         * - Returns CompletableFuture<HttpResponse<T>>
         * - Non-blocking
         * - Can chain operations
         * 
         * Use when:
         * ✓ UI applications
         * ✓ High concurrency
         * ✓ Multiple requests
         * ✓ Need parallelism
         * 
         * Benefits:
         * - Better resource utilization
         * - Higher throughput
         * - Composable operations
         */
    }
    
    /**
     * Chaining async operations
     */
    public static void chainingAsync() throws Exception {
        System.out.println("\n=== Chaining Async ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.github.com/users/octocat"))
            .build();
        
        client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
            .thenApply(HttpResponse::body)
            .thenApply(String::toUpperCase)
            .thenAccept(body -> 
                System.out.println("Processed: " + 
                    body.substring(0, Math.min(50, body.length()))))
            .join();
        
        /**
         * CHAINING PATTERN:
         * 
         * client.sendAsync(request, handler)
         *   .thenApply(response -> response.body())
         *   .thenApply(body -> process(body))
         *   .thenAccept(result -> display(result))
         *   .exceptionally(ex -> handleError(ex));
         * 
         * Operators:
         * - thenApply: Transform result
         * - thenAccept: Consume result
         * - thenCompose: Chain futures
         * - exceptionally: Handle errors
         */
    }
    
    /**
     * Multiple parallel requests
     */
    public static void parallelRequests() throws Exception {
        System.out.println("\n=== Parallel Requests ===");
        
        List<String> urls = List.of(
            "https://api.github.com/users/octocat",
            "https://api.github.com/users/torvalds",
            "https://api.github.com/users/gvanrossum"
        );
        
        // Start all requests in parallel
        List<CompletableFuture<HttpResponse<String>>> futures = urls.stream()
            .map(url -> HttpRequest.newBuilder()
                .uri(URI.create(url))
                .build())
            .map(request -> client.sendAsync(
                request,
                HttpResponse.BodyHandlers.ofString()))
            .toList();
        
        // Wait for all to complete
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .join();
        
        // Process results
        futures.stream()
            .map(CompletableFuture::join)
            .forEach(response -> 
                System.out.println("Status: " + response.statusCode()));
        
        /**
         * PARALLEL PATTERN:
         * 
         * List<CompletableFuture<Response>> futures = urls.stream()
         *   .map(url -> createRequest(url))
         *   .map(req -> client.sendAsync(req, handler))
         *   .collect(Collectors.toList());
         * 
         * CompletableFuture.allOf(futures...)
         *   .thenRun(() -> processAll(futures));
         * 
         * Benefits:
         * - All requests start immediately
         * - No waiting between requests
         * - Maximum parallelism
         * - Efficient resource use
         */
    }
    
    /**
     * Timeout with async
     */
    public static void asyncWithTimeout() throws Exception {
        System.out.println("\n=== Async with Timeout ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/delay/10"))
            .timeout(Duration.ofSeconds(2))  // Request times out after 2s
            .build();
        
        client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
            .orTimeout(3, TimeUnit.SECONDS)  // Future times out after 3s
            .whenComplete((response, ex) -> {
                if (ex != null) {
                    System.out.println("Request failed: " + ex.getMessage());
                } else {
                    System.out.println("Status: " + response.statusCode());
                }
            })
            .join();
        
        /**
         * TIMEOUT LAYERS:
         * 
         * 1. Request timeout (HttpRequest):
         *    request.timeout(Duration)
         *    - Total time for request/response
         * 
         * 2. Future timeout (CompletableFuture):
         *    future.orTimeout(duration, unit)
         *    - Additional timeout on future
         * 
         * 3. Connect timeout (HttpClient):
         *    client.connectTimeout(Duration)
         *    - Time to establish connection
         * 
         * All three can be combined for defense in depth
         */
    }
    
    /**
     * Error handling
     */
    public static void errorHandling() {
        System.out.println("\n=== Error Handling ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://invalid-domain-12345.com"))
            .build();
        
        // Synchronous error handling
        try {
            HttpResponse<String> response = client.send(
                request,
                HttpResponse.BodyHandlers.ofString()
            );
            System.out.println("Success: " + response.statusCode());
        } catch (IOException e) {
            System.out.println("IO Error: " + e.getMessage());
        } catch (InterruptedException e) {
            System.out.println("Interrupted");
            Thread.currentThread().interrupt();
        }
        
        // Asynchronous error handling
        client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
            .thenApply(HttpResponse::body)
            .exceptionally(ex -> {
                System.out.println("Async error: " + ex.getMessage());
                return "Error occurred";
            })
            .thenAccept(System.out::println);
        
        /**
         * ERROR HANDLING:
         * 
         * Synchronous:
         * try {
         *   HttpResponse<T> response = client.send(...);
         * } catch (IOException e) {
         *   // Network error, timeout, etc.
         * } catch (InterruptedException e) {
         *   // Thread interrupted
         *   Thread.currentThread().interrupt();
         * }
         * 
         * Asynchronous:
         * client.sendAsync(...)
         *   .exceptionally(ex -> {
         *     // Handle error
         *     return fallbackValue;
         *   });
         * 
         * Or:
         * client.sendAsync(...)
         *   .whenComplete((response, ex) -> {
         *     if (ex != null) {
         *       // Handle error
         *     } else {
         *       // Handle response
         *     }
         *   });
         */
    }
    
    public static void main(String[] args) throws Exception {
        synchronousRequest();
        asynchronousRequest();
        chainingAsync();
        parallelRequests();
        asyncWithTimeout();
        errorHandling();
    }
}
```

## 3. Request Builders

### Creating HTTP Requests with Fluent API

```java
/**
 * REQUEST BUILDERS
 * 
 * Building HTTP requests with HttpRequest.Builder
 */

import java.net.http.*;
import java.net.URI;
import java.time.Duration;
import java.util.Map;

public class RequestBuilders {
    
    private static final HttpClient client = HttpClient.newHttpClient();
    
    /**
     * Basic GET request
     */
    public static void basicGet() throws Exception {
        System.out.println("=== Basic GET ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.github.com/users/octocat"))
            .GET()  // Default, can be omitted
            .build();
        
        HttpResponse<String> response = client.send(
            request,
            HttpResponse.BodyHandlers.ofString()
        );
        
        System.out.println("Status: " + response.statusCode());
        
        /**
         * GET REQUEST:
         * 
         * HttpRequest.newBuilder()
         *   .uri(URI.create("https://example.com"))
         *   .GET()  // Optional - default method
         *   .build()
         * 
         * GET is default:
         * - Can omit .GET()
         * - No request body
         */
    }
    
    /**
     * POST request with body
     */
    public static void postRequest() throws Exception {
        System.out.println("\n=== POST Request ===");
        
        String json = """
            {
                "name": "John Doe",
                "email": "john@example.com"
            }
            """;
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/post"))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();
        
        HttpResponse<String> response = client.send(
            request,
            HttpResponse.BodyHandlers.ofString()
        );
        
        System.out.println("Status: " + response.statusCode());
        
        /**
         * POST REQUEST:
         * 
         * .POST(BodyPublisher)
         * 
         * BodyPublishers:
         * - ofString(String)
         * - ofByteArray(byte[])
         * - ofFile(Path)
         * - ofInputStream(Supplier<InputStream>)
         * - noBody()
         */
    }
    
    /**
     * PUT, DELETE, PATCH
     */
    public static void otherMethods() throws Exception {
        System.out.println("\n=== Other Methods ===");
        
        // PUT
        HttpRequest putRequest = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/put"))
            .PUT(HttpRequest.BodyPublishers.ofString("data"))
            .build();
        
        // DELETE
        HttpRequest deleteRequest = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/delete"))
            .DELETE()
            .build();
        
        // PATCH
        HttpRequest patchRequest = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/patch"))
            .method("PATCH", HttpRequest.BodyPublishers.ofString("data"))
            .build();
        
        // Custom method
        HttpRequest customRequest = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/anything"))
            .method("CUSTOM", HttpRequest.BodyPublishers.noBody())
            .build();
        
        /**
         * HTTP METHODS:
         * 
         * Standard:
         * .GET()
         * .POST(BodyPublisher)
         * .PUT(BodyPublisher)
         * .DELETE()
         * 
         * Custom:
         * .method("METHOD", BodyPublisher)
         * 
         * Examples:
         * .method("PATCH", BodyPublishers.ofString("data"))
         * .method("HEAD", BodyPublishers.noBody())
         * .method("OPTIONS", BodyPublishers.noBody())
         */
    }
    
    /**
     * Headers
     */
    public static void headers() throws Exception {
        System.out.println("\n=== Headers ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/headers"))
            
            // Single header
            .header("User-Agent", "MyApp/1.0")
            
            // Multiple values for same header
            .header("Accept", "application/json")
            .header("Accept", "application/xml")
            
            // Multiple headers at once
            .headers(
                "X-Custom-Header", "value1",
                "X-Another-Header", "value2"
            )
            
            .build();
        
        HttpResponse<String> response = client.send(
            request,
            HttpResponse.BodyHandlers.ofString()
        );
        
        System.out.println("Response: " + response.body());
        
        /**
         * HEADERS:
         * 
         * Single header:
         * .header("Name", "value")
         * 
         * Multiple values:
         * .header("Name", "value1")
         * .header("Name", "value2")
         * 
         * Multiple headers:
         * .headers(
         *   "Name1", "value1",
         *   "Name2", "value2"
         * )
         * 
         * Common headers:
         * .header("Content-Type", "application/json")
         * .header("Accept", "application/json")
         * .header("Authorization", "Bearer " + token)
         * .header("User-Agent", "MyApp/1.0")
         */
    }
    
    /**
     * Request timeout
     */
    public static void requestTimeout() throws Exception {
        System.out.println("\n=== Request Timeout ===");
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/delay/2"))
            .timeout(Duration.ofSeconds(5))  // Timeout after 5 seconds
            .build();
        
        try {
            HttpResponse<String> response = client.send(
                request,
                HttpResponse.BodyHandlers.ofString()
            );
            System.out.println("Success: " + response.statusCode());
        } catch (HttpTimeoutException e) {
            System.out.println("Request timed out");
        }
        
        /**
         * TIMEOUT:
         * 
         * .timeout(Duration)
         * 
         * - Total time for request + response
         * - Throws HttpTimeoutException
         * - Independent of client connectTimeout
         * 
         * Example:
         * .timeout(Duration.ofSeconds(30))
         * .timeout(Duration.ofMinutes(2))
         */
    }
    
    /**
     * HTTP version per request
     */
    public static void httpVersionPerRequest() throws Exception {
        System.out.println("\n=== HTTP Version Per Request ===");
        
        // Force HTTP/1.1 for this request
        HttpRequest http11Request = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/get"))
            .version(HttpClient.Version.HTTP_1_1)
            .build();
        
        // Prefer HTTP/2 for this request
        HttpRequest http2Request = HttpRequest.newBuilder()
            .uri(URI.create("https://httpbin.org/get"))
            .version(HttpClient.Version.HTTP_2)
            .build();
        
        /**
         * VERSION PER REQUEST:
         * 
         * .version(HttpClient.Version.HTTP_1_1)
         * .version(HttpClient.Version.HTTP_2)
         * 
         * Overrides client-level version setting
         * 
         * Use for:
         * - Testing different versions
```

```java
            public CompletionStage<?> onText(WebSocket webSocket,

                return WebSocket.Listener.super.onClose(webSocket, statusCode, reason);
            }
        };
        
        // Connect to WebSocket
        WebSocket ws = client.newWebSocketBuilder()
            .buildAsync(URI.create("wss://echo.websocket.org"), listener)

        ws.sendText("Message 1", true);
```

```java
import java.net.http.*;
import java.net.URI;
import java.util.concurrent.*;

public class Http2Features {
    
    /**
     * HTTP/2 multiplexing
     */
    public static void multiplexing() throws Exception {
        System.out.println("=== HTTP/2 Multiplexing ===");
        
        HttpClient client = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .build();
        
        // Multiple requests over single connection
        CompletableFuture<HttpResponse<String>> future1 = 
            client.sendAsync(
                HttpRequest.newBuilder()
                    .uri(URI.create("https://http2.golang.org/reqinfo"))
                    .build(),
                HttpResponse.BodyHandlers.ofString()
            );
        
        CompletableFuture<HttpResponse<String>> future2 = 
            client.sendAsync(
                HttpRequest.newBuilder()
                    .uri(URI.create("https://http2.golang.org/reqinfo"))
                    .build(),
                HttpResponse.BodyHandlers.ofString()
            );
        
        CompletableFuture<HttpResponse<String>> future3 = 
            client.sendAsync(
                HttpRequest.newBuilder()
                    .uri(URI.create("https://http2.golang.org/reqinfo"))
                    .build(),
                HttpResponse.BodyHandlers.ofString()
            );
        
        CompletableFuture.allOf(future1, future2, future3).join();
        
        System.out.println("All requests completed");
        System.out.println("Version 1: " + future1.join().version());
        System.out.println("Version 2: " + future2.join().version());
        System.out.println("Version 3: " + future3.join().version());
        
        /**
         * MULTIPLEXING:
         * 
         * HTTP/1.1:
         * - One request per connection
         * - Need multiple connections for parallel
         * - Head-of-line blocking
         * 
         * HTTP/2:
         * - Multiple requests per connection
         * - Streams multiplexed
         * - No head-of-line blocking
         * - Better performance
         * 
         * Benefits:
         * - Fewer connections
         * - Lower latency
         * - Better resource usage
         */
    }
    
    /**
     * Server push (conceptual - not all servers support)
     */
    public static void serverPush() {
        /**
         * SERVER PUSH:
         * 
         * HTTP/2 feature where server can send
         * resources before client requests them
         * 
         * Example:
         * Client requests index.html
         * Server pushes:
         * - style.css
         * - script.js
         * - logo.png
         * 
         * Java API:
         * HttpResponse.PushPromiseHandler
         * 
         * Usage:
         * class MyPushHandler implements 
         *     HttpResponse.PushPromiseHandler<String> {
         *   
         *   @Override
         *   public void applyPushPromise(
         *       HttpRequest initiatingRequest,
         *       HttpRequest pushPromiseRequest,
         *       Function<BodyHandler<String>, 
         *           CompletableFuture<HttpResponse<String>>> acceptor) {
         *     
         *     // Accept or reject push
         *     CompletableFuture<HttpResponse<String>> cf = 
         *       acceptor.apply(BodyHandlers.ofString());
         *     
         *     cf.thenAccept(response -> {
         *       System.out.println("Pushed: " + 
         *         response.request().uri());
         *     });
         *   }
         * }
         * 
         * HttpResponse<String> response = client.send(
         *   request,
         *   BodyHandlers.ofString(),
         *   new MyPushHandler()
         * );
         * 
         * Note: Few servers support push,
         * and Chrome/Firefox deprecated it
         */
    }
    
    /**
     * Header compression
     */
    public static void headerCompression() {
        /**
         * HEADER COMPRESSION:
         * 
         * HTTP/1.1:
         * - Headers sent as text
         * - Repetitive headers (User-Agent, etc.)
         * - Wasted bandwidth
         * 
         * HTTP/2:
         * - HPACK compression
         * - Header table (static + dynamic)
         * - Only send differences
         * 
         * Example:
         * Request 1:
         * :method: GET
         * :path: /index.html
         * user-agent: Mozilla/5.0...
         * accept: text/html...
         * 
         * Request 2 (compressed):
         * :method: GET
         * :path: /style.css
         * [2] [3]  (references to user-agent, accept)
         * 
         * Automatic in Java HttpClient
         * No configuration needed
         * 
         * Benefits:
         * - Reduced overhead
         * - Faster requests
         * - Lower bandwidth
         */
    }

         *

         *

         * High priority: HTML, CSS
         * Low priority: Images, analytics
         *

    public static void versionComparison() throws Exception {

            .version(HttpClient.Version.HTTP_1_1)

        long http11Time = System.currentTimeMillis() - start;

        HttpClient http2Client = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .build();
        
        start = System.currentTimeMillis();
        HttpResponse<String> http2Response = http2Client.send(
            HttpRequest.newBuilder().uri(URI.create(url)).build(),
            HttpResponse.BodyHandlers.ofString()
        );
        long http2Time = System.currentTimeMillis() - start;

        System.out.println("HTTP/2: " + http2Response.version() +

         * ┌─────────────────┬──────────┬──────────┐
         * │ Feature         │ HTTP/1.1 │ HTTP/2   │
         * ├─────────────────┼──────────┼──────────┤
         * │ Protocol        │ Text     │ Binary   │
         * │ Multiplexing    │ No       │ Yes      │
         * │ Server Push     │ No       │ Yes      │
         * │ Header Compress │ No       │ Yes      │
         * │ Prioritization  │ No       │ Yes      │
         * │ TLS Required    │ No       │ Usually  │
         * └─────────────────┴──────────┴──────────┘
         * 
         * When to use HTTP/2:
         * ✓ Multiple requests to same host
         * ✓ HTTPS available
         * ✓ Modern servers
         * 
         * When HTTP/1.1 sufficient:
         * ○ Single request
         * ○ HTTP only (not HTTPS)
         * ○ Legacy servers
         */
    }
    
    public static void main(String[] args) throws Exception {
```

## Summary

### Quick Reference

**HttpClient Creation:**

```java
// Simple
HttpClient client = HttpClient.newHttpClient();

// Configured
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_2)
    .followRedirects(HttpClient.Redirect.NORMAL)
    .connectTimeout(Duration.ofSeconds(10))
    .build();
```

**Requests:**

```java
// GET
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://example.com"))
    .build();

// POST
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://example.com"))
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(json))
    .timeout(Duration.ofSeconds(30))
    .build();
```

**Send:**

```java
// Synchronous
HttpResponse<String> response = client.send(
    request, HttpResponse.BodyHandlers.ofString());

// Asynchronous
CompletableFuture<HttpResponse<String>> future = client.sendAsync(
    request, HttpResponse.BodyHandlers.ofString());
```

**Response:**

```java
int status = response.statusCode();
String body = response.body();
HttpHeaders headers = response.headers();
```

**WebSocket:**

```java
WebSocket ws = client.newWebSocketBuilder()
    .buildAsync(URI.create("wss://example.com"), listener)
    .join();

ws.sendText("Hello", true);
ws.sendClose(WebSocket.NORMAL_CLOSURE, "Done");
```

**Key insights:**

- **Modern API**: Builder pattern, clean, easy to use
- **HTTP/2**: Multiplexing, server push, header compression
- **Async**: CompletableFuture-based, non-blocking
- **Built-in**: No external dependencies needed
- **WebSocket**: Full bidirectional communication support

**The revolution:**

```
HttpURLConnection: Verbose, painful, HTTP/1.1 only
↓
Java 11 HttpClient: Clean, modern, HTTP/2 native

From 30 lines to 5 lines for basic request
```
