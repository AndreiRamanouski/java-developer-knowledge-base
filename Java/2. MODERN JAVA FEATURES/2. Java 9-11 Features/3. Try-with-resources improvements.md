# Try-with-resources Improvements (Java 7-9)

## Overview

Try-with-resources, introduced in Java 7, automatically manages resource cleanup by closing AutoCloseable resources when the try block exits. Java 9 enhanced this feature with cleaner syntax that allows using effectively final variables directly, eliminating redundant variable declarations. This guide covers the evolution from Java 7's verbose syntax to Java 9's streamlined approach, handling multiple resources efficiently, understanding resource closing order (reverse declaration order), and best practices for resource management in modern Java.

## 1. Effectively Final Variables

### Using Existing Variables in Try-with-resources

```java
/**
 * EFFECTIVELY FINAL VARIABLES
 * 
 * Java 9 allows using existing variables in try-with-resources
 */

import java.io.*;
import java.util.*;

public class EffectivelyFinalVariables {
    
    /**
     * Java 7 - Must declare in try statement
     */
    public static void java7Style() throws IOException {
        System.out.println("=== Java 7 Style ===");
        
        // Must create resource in try-with-resources header
        try (BufferedReader reader = new BufferedReader(
                new FileReader("data.txt"))) {
            String line = reader.readLine();
            System.out.println(line);
        }
        
        /**
         * JAVA 7 LIMITATION:
         * 
         * Cannot do:
         * BufferedReader reader = new BufferedReader(...);
         * try (reader) {  // COMPILATION ERROR in Java 7!
         *   ...
         * }
         * 
         * Must declare IN the try statement
         */
    }
    
    /**
     * Java 9 - Can use existing effectively final variable
     */
    public static void java9Style() throws IOException {
        System.out.println("\n=== Java 9 Style ===");
        
        // Create resource before try
        BufferedReader reader = new BufferedReader(
            new FileReader("data.txt"));
        
        // Use in try-with-resources (Java 9+)
        try (reader) {
            String line = reader.readLine();
            System.out.println(line);
        }
        
        /**
         * JAVA 9 ENHANCEMENT:
         * 
         * Can use existing variable if:
         * - Variable is effectively final
         * - Variable is of type AutoCloseable
         * 
         * Benefits:
         * - Less nesting
         * - Cleaner code
         * - Reuse existing variables
         */
    }
    
    /**
     * Effectively final requirement
     */
    public static void effectivelyFinalRequirement() throws IOException {
        System.out.println("\n=== Effectively Final ===");
        
        // Effectively final - assigned once, never reassigned
        BufferedReader reader = new BufferedReader(
            new FileReader("data.txt"));
        
        try (reader) {  // OK - effectively final
            String line = reader.readLine();
            System.out.println(line);
        }
        
        // Cannot do this:
        // BufferedReader reader2 = new BufferedReader(...);
        // reader2 = new BufferedReader(...);  // Reassignment!
        // try (reader2) {  // COMPILATION ERROR - not effectively final
        //   ...
        // }
        
        /**
         * EFFECTIVELY FINAL:
         * 
         * Definition:
         * Variable that is not explicitly final but is never 
         * reassigned after initialization.
         * 
         * Required for:
         * - Try-with-resources (Java 9+)
         * - Lambda expressions
         * - Anonymous inner classes
         * 
         * Examples:
         * 
         * ✓ Effectively final:
         * String s = "hello";
         * // Never reassigned
         * 
         * ✗ Not effectively final:
         * String s = "hello";
         * s = "world";  // Reassigned!
         */
    }
    
    /**
     * Comparison: Java 7 vs Java 9
     */
    public static void comparisonJava7VsJava9() throws IOException {
        System.out.println("\n=== Java 7 vs Java 9 ===");
        
        // JAVA 7: Must create in try statement
        try (BufferedReader reader = new BufferedReader(
                new FileReader("data.txt"))) {
            String line = reader.readLine();
            System.out.println("Java 7: " + line);
        }
        
        // JAVA 9: Can create before, use in try
        BufferedReader reader = new BufferedReader(
            new FileReader("data.txt"));
        try (reader) {
            String line = reader.readLine();
            System.out.println("Java 9: " + line);
        }
        
        /**
         * WHEN JAVA 9 SYNTAX HELPS:
         * 
         * 1. Resource from method call:
         *    Scanner scanner = createScanner();
         *    try (scanner) { ... }
         * 
         * 2. Resource from field:
         *    Reader reader = this.getReader();
         *    try (reader) { ... }
         * 
         * 3. Resource needs setup:
         *    Connection conn = dataSource.getConnection();
         *    conn.setAutoCommit(false);
         *    try (conn) { ... }
         * 
         * 4. Conditional resources:
         *    Reader reader = useFile ? 
         *      new FileReader("file.txt") : 
         *      new StringReader("text");
         *    try (reader) { ... }
         */
    }
    
    /**
     * Cannot reassign
     */
    public static void cannotReassign() {
        /**
         * CANNOT REASSIGN:
         * 
         * InputStream in = new FileInputStream("file.txt");
         * in = new FileInputStream("other.txt");  // Reassignment!
         * try (in) {  // COMPILATION ERROR
         *   ...
         * }
         * 
         * Error: "Local variable in defined in an enclosing 
         *         scope must be final or effectively final"
         */
    }
    
    /**
     * Practical example: Factory method
     */
    public static void factoryMethodExample() throws SQLException {
        System.out.println("\n=== Factory Method ===");
        
        // Get resource from factory method
        Connection connection = DatabaseFactory.createConnection();
        
        // Configure before try
        connection.setAutoCommit(false);
        
        // Use in try-with-resources
        try (connection) {
            Statement stmt = connection.createStatement();
            stmt.execute("INSERT INTO users VALUES (1, 'Alice')");
            connection.commit();
            System.out.println("Transaction committed");
        } catch (SQLException e) {
            System.out.println("Transaction rolled back");
            throw e;
        }
        
        /**
         * BENEFIT:
         * 
         * Java 7:
         * try (Connection conn = DatabaseFactory.createConnection()) {
         *   conn.setAutoCommit(false);  // Setup inside try
         *   ...
         * }
         * 
         * Java 9:
         * Connection conn = DatabaseFactory.createConnection();
         * conn.setAutoCommit(false);  // Setup before try
         * try (conn) {  // Cleaner!
         *   ...
         * }
         */
    }
    
    // Fake classes for examples
    static class DatabaseFactory {
        static Connection createConnection() {
            return null;  // Simplified
        }
    }
    
    interface Connection extends AutoCloseable {
        void setAutoCommit(boolean autoCommit) throws SQLException;
        Statement createStatement();
        void commit() throws SQLException;
        void close() throws SQLException;
    }
    
    interface Statement {
        void execute(String sql);
    }
    
    static class SQLException extends Exception {}
    
    public static void main(String[] args) throws IOException {
        // Note: These examples assume data.txt exists
        // In real code, handle FileNotFoundException
    }
}
```

## 2. Cleaner Syntax in Java 9

### Evolution of Try-with-resources Syntax

```java
/**
 * CLEANER SYNTAX IN JAVA 9
 * 
 * Evolution from Java 7 to Java 9
 */

import java.io.*;
import java.util.*;

public class CleanerSyntax {
    
    /**
     * Before Java 7 - Manual cleanup
     */
    public static void beforeJava7() throws IOException {
        System.out.println("=== Before Java 7 ===");
        
        BufferedReader reader = null;
        try {
            reader = new BufferedReader(new FileReader("data.txt"));
            String line = reader.readLine();
            System.out.println(line);
        } finally {
            if (reader != null) {
                try {
                    reader.close();
                } catch (IOException e) {
                    // Swallow or log
                }
            }
        }
        
        /**
         * PROBLEMS:
         * - Verbose
         * - Error-prone (forget to close)
         * - Exceptions in finally can hide original exception
         * - Must check for null
         * - Nested try-catch for close()
         */
    }
    
    /**
     * Java 7 - Try-with-resources
     */
    public static void java7TryWithResources() throws IOException {
        System.out.println("\n=== Java 7 Try-with-resources ===");
        
        try (BufferedReader reader = new BufferedReader(
                new FileReader("data.txt"))) {
            String line = reader.readLine();
            System.out.println(line);
        }
        // reader.close() called automatically
        
        /**
         * IMPROVEMENTS:
         * ✓ Automatic close
         * ✓ Exception handling correct
         * ✓ Less boilerplate
         * ✓ No null check needed
         * 
         * LIMITATION:
         * ✗ Must declare resource in try statement
         */
    }
    
    /**
     * Java 9 - Cleaner with existing variables
     */
    public static void java9Cleaner() throws IOException {
        System.out.println("\n=== Java 9 Cleaner ===");
        
        BufferedReader reader = new BufferedReader(
            new FileReader("data.txt"));
        
        try (reader) {
            String line = reader.readLine();
            System.out.println(line);
        }
        
        /**
         * ADDITIONAL IMPROVEMENTS:
         * ✓ Can use existing variable
         * ✓ Less nesting
         * ✓ Reuse resources
         * ✓ Separate creation from try
         */
    }
    
    /**
     * Real-world example: Method parameters
     */
    public static void methodParameterExample(InputStream input) 
            throws IOException {
        
        // Java 7: Must wrap
        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(input))) {
            reader.lines().forEach(System.out::println);
        }
        
        // Java 9: Can use parameter directly (if effectively final)
        try (input) {
            new BufferedReader(new InputStreamReader(input))
                .lines()
                .forEach(System.out::println);
        }
    }
    
    /**
     * Conditional resource creation
     */
    public static void conditionalResource(boolean useFile) 
            throws IOException {
        
        // Create resource conditionally
        Reader reader = useFile 
            ? new FileReader("data.txt")
            : new StringReader("default data");
        
        // Java 9: Use in try
        try (reader) {
            int ch;
            while ((ch = reader.read()) != -1) {
                System.out.print((char) ch);
            }
        }
        
        /**
         * JAVA 7 WOULD REQUIRE:
         * 
         * try (Reader reader = useFile 
         *         ? new FileReader("data.txt")
         *         : new StringReader("default data")) {
         *   ...
         * }
         * 
         * Works, but less flexible
         */
    }
    
    /**
     * Resource from builder pattern
     */
    public static void builderPatternExample() throws IOException {
        
        // Build resource with fluent API
        Scanner scanner = new Scanner(System.in)
            .useDelimiter("\\s+")
            .useLocale(Locale.US);
        
        // Java 9: Use configured resource
        try (scanner) {
            while (scanner.hasNext()) {
                System.out.println(scanner.next());
            }
        }
        
        /**
         * BENEFIT:
         * Configuration separate from try block
         * Cleaner separation of concerns
         */
    }
    
    /**
     * Mixing old and new syntax
     */
    public static void mixingOldAndNew() throws IOException {
        
        // Existing resource
        BufferedReader existing = new BufferedReader(
            new FileReader("data1.txt"));
        
        // Can mix: new resource + existing resource
        try (existing;
             BufferedReader created = new BufferedReader(
                new FileReader("data2.txt"))) {
            
            System.out.println("File 1: " + existing.readLine());
            System.out.println("File 2: " + created.readLine());
        }
        
        /**
         * SYNTAX:
         * try (existingResource1;
         *      Type newResource = new ...;
         *      existingResource2) {
         *   ...
         * }
         * 
         * Can mix freely!
         */
    }
    
    /**
     * Complete evolution timeline
     */
    public static void evolutionTimeline() {
        /**
         * EVOLUTION TIMELINE:
         * 
         * Pre-Java 7 (Before 2011):
         * try {
         *   Resource r = new Resource();
         *   // use r
         * } finally {
         *   if (r != null) r.close();
         * }
         * 
         * Java 7 (2011):
         * try (Resource r = new Resource()) {
         *   // use r
         * }
         * 
         * Java 9 (2017):
         * Resource r = new Resource();
         * try (r) {
         *   // use r
         * }
         * 
         * Each step:
         * - Less boilerplate
         * - Safer
         * - More flexible
         */
    }
    
    public static void main(String[] args) throws IOException {
        // Examples would need actual files to run
    }
}
```

## 3. Multiple Resources

### Managing Multiple AutoCloseable Resources

```java
/**
 * MULTIPLE RESOURCES
 * 
 * Handling multiple AutoCloseable resources
 */

import java.io.*;
import java.util.*;

public class MultipleResources {
    
    /**
     * Basic multiple resources
     */
    public static void basicMultipleResources() throws IOException {
        System.out.println("=== Basic Multiple Resources ===");
        
        try (BufferedReader reader = new BufferedReader(
                new FileReader("input.txt"));
             BufferedWriter writer = new BufferedWriter(
                new FileWriter("output.txt"))) {
            
            String line;
            while ((line = reader.readLine()) != null) {
                writer.write(line.toUpperCase());
                writer.newLine();
            }
        }
        // Both resources closed automatically
        
        /**
         * SYNTAX:
         * try (Resource1 r1 = ...;
         *      Resource2 r2 = ...;
         *      Resource3 r3 = ...) {
         *   ...
         * }
         * 
         * Separated by semicolons
         * All resources closed automatically
         */
    }
    
    /**
     * Closing order (reverse declaration)
     */
    public static void closingOrder() {
        System.out.println("\n=== Closing Order ===");
        
        class Resource implements AutoCloseable {
            private String name;
            
            Resource(String name) {
                this.name = name;
                System.out.println("Opening " + name);
            }
            
            @Override
            public void close() {
                System.out.println("Closing " + name);
            }
        }
        
        try (Resource r1 = new Resource("First");
             Resource r2 = new Resource("Second");
             Resource r3 = new Resource("Third")) {
            
            System.out.println("Using resources");
        }
        
        /**
         * OUTPUT:
         * Opening First
         * Opening Second
         * Opening Third
         * Using resources
         * Closing Third
         * Closing Second
         * Closing First
         * 
         * CLOSING ORDER:
         * Reverse of declaration order (LIFO - Last In, First Out)
         * 
         * Reason:
         * Later resources may depend on earlier ones
         * Close dependents before dependencies
         */
    }
    
    /**
     * File copy example
     */
    public static void fileCopyExample() throws IOException {
        System.out.println("\n=== File Copy ===");
        
        try (InputStream in = new FileInputStream("source.dat");
             OutputStream out = new FileOutputStream("dest.dat")) {
            
            byte[] buffer = new byte[8192];
            int bytesRead;
            while ((bytesRead = in.read(buffer)) != -1) {
                out.write(buffer, 0, bytesRead);
            }
        }
        // in and out both closed (out closed first, then in)
    }
    
    /**
     * Database connection example
     */
    public static void databaseExample() throws SQLException {
        System.out.println("\n=== Database Resources ===");
        
        try (Connection conn = DriverManager.getConnection("jdbc:...");
             Statement stmt = conn.createStatement();
             ResultSet rs = stmt.executeQuery("SELECT * FROM users")) {
            
            while (rs.next()) {
                System.out.println(rs.getString("name"));
            }
        }
        // Closed in order: rs, stmt, conn
        
        /**
         * IMPORTANT:
         * Close order matters for database resources:
         * 1. ResultSet (depends on Statement)
         * 2. Statement (depends on Connection)
         * 3. Connection (last)
         * 
         * Try-with-resources handles this automatically!
         */
    }
    
    /**
     * Java 9: Mix existing and new resources
     */
    public static void mixExistingAndNew() throws IOException {
        System.out.println("\n=== Mix Existing and New ===");
        
        // Existing resource
        InputStream existingStream = new FileInputStream("data.txt");
        
        // Mix with new resources
        try (existingStream;
             BufferedInputStream buffered = 
                new BufferedInputStream(existingStream);
             InputStreamReader reader = 
                new InputStreamReader(buffered)) {
            
            int ch;
            while ((ch = reader.read()) != -1) {
                System.out.print((char) ch);
            }
        }
        // All three closed: reader, buffered, existingStream
        
        /**
         * JAVA 9 FLEXIBILITY:
         * 
         * try (existingResource1;
         *      Type newResource = ...;
         *      existingResource2;
         *      Type anotherNew = ...) {
         *   ...
         * }
         * 
         * Can freely mix!
         */
    }
    
    /**
     * Nested try-with-resources (usually avoid)
     */
    public static void nestedTryWithResources() throws IOException {
        System.out.println("\n=== Nested Try (Avoid) ===");
        
        // DON'T DO THIS:
        try (BufferedReader reader1 = new BufferedReader(
                new FileReader("file1.txt"))) {
            
            try (BufferedReader reader2 = new BufferedReader(
                    new FileReader("file2.txt"))) {
                
                // Use both readers
                System.out.println(reader1.readLine());
                System.out.println(reader2.readLine());
            }
        }
        
        // INSTEAD, USE MULTIPLE RESOURCES:
        try (BufferedReader reader1 = new BufferedReader(
                new FileReader("file1.txt"));
             BufferedReader reader2 = new BufferedReader(
                new FileReader("file2.txt"))) {
            
            System.out.println(reader1.readLine());
            System.out.println(reader2.readLine());
        }
        
        /**
         * NESTED vs MULTIPLE:
         * 
         * Nested:
         * - More indentation
         * - Harder to read
         * - Unnecessary
         * 
         * Multiple:
         * - Flatter structure
         * - Clearer
         * - Recommended
         */
    }
    
    /**
     * Limit on number of resources
     */
    public static void noLimitOnResources() throws IOException {
        /**
         * NO LIMIT:
         * 
         * try (Resource r1 = ...;
         *      Resource r2 = ...;
         *      Resource r3 = ...;
         *      Resource r4 = ...;
         *      Resource r5 = ...;
         *      // ... as many as needed
         *      Resource rN = ...) {
         *   ...
         * }
         * 
         * No limit on number of resources
         * All will be closed in reverse order
         * 
         * Practical limit:
         * If you have 10+ resources, consider:
         * - Refactoring into smaller methods
         * - Using a resource manager
         * - Reviewing design
         */
    }
    
    // Fake classes for examples
    interface Connection extends AutoCloseable {
        Statement createStatement() throws SQLException;
        void close() throws SQLException;
    }
    
    interface Statement extends AutoCloseable {
        ResultSet executeQuery(String sql) throws SQLException;
        void close() throws SQLException;
    }
    
    interface ResultSet extends AutoCloseable {
        boolean next() throws SQLException;
        String getString(String column) throws SQLException;
        void close() throws SQLException;
    }
    
    static class DriverManager {
        static Connection getConnection(String url) {
            return null;
        }
    }
    
    static class SQLException extends Exception {}
    
    public static void main(String[] args) throws IOException {
        closingOrder();
    }
}
```

## 4. Resource Ordering

### Understanding Close Order and Dependencies

```java
/**
 * RESOURCE ORDERING
 * 
 * Understanding the order in which resources are closed
 */

import java.io.*;

public class ResourceOrdering {
    
    /**
     * Basic closing order demonstration
     */
    public static void closingOrderDemo() {
        System.out.println("=== Closing Order Demo ===");
        
        class TrackedResource implements AutoCloseable {
            private final String id;
            
            TrackedResource(String id) {
                this.id = id;
                System.out.println(id + " created");
            }
            
            void use() {
                System.out.println(id + " used");
            }
            
            @Override
            public void close() {
                System.out.println(id + " closed");
            }
        }
        
        try (TrackedResource r1 = new TrackedResource("R1");
             TrackedResource r2 = new TrackedResource("R2");
             TrackedResource r3 = new TrackedResource("R3")) {
            
            r1.use();
            r2.use();
            r3.use();
        }
        
        /**
         * OUTPUT:
         * R1 created
         * R2 created
         * R3 created
         * R1 used
         * R2 used
         * R3 used
         * R3 closed  ← Last declared, first closed
         * R2 closed
         * R1 closed  ← First declared, last closed
         * 
         * ORDER: Reverse of declaration (LIFO)
         */
    }
    
    /**
     * Why reverse order matters
     */
    public static void whyReverseOrder() throws IOException {
        System.out.println("\n=== Why Reverse Order ===");
        
        // Example: Stream wrapping
        try (FileInputStream fis = new FileInputStream("data.txt");
             BufferedInputStream bis = new BufferedInputStream(fis);
             DataInputStream dis = new DataInputStream(bis)) {
            
            int value = dis.readInt();
            System.out.println("Value: " + value);
        }
        // Close order: dis → bis → fis
        
        /**
         * WHY REVERSE:
         * 
         * Dependencies:
         * dis depends on bis
         * bis depends on fis
         * 
         * Must close in reverse:
         * 1. Close dis (flushes to bis)
         * 2. Close bis (flushes to fis)
         * 3. Close fis (closes file)
         * 
         * If closed in forward order:
         * - fis closed first
         * - bis and dis would fail
         * - Data loss possible
         */
    }
    
    /**
     * Database resource ordering
     */
    public static void databaseOrdering() throws SQLException {
        System.out.println("\n=== Database Ordering ===");
        
        try (Connection conn = createConnection();
             PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users");
             ResultSet rs = pstmt.executeQuery()) {
            
            while (rs.next()) {
                System.out.println(rs.getString("name"));
            }
        }
        // Close order: rs → pstmt → conn
        
        /**
         * DATABASE DEPENDENCIES:
         * 
         * ResultSet → PreparedStatement → Connection
         * 
         * Must close:
         * 1. ResultSet (release cursor)
         * 2. PreparedStatement (release statement)
         * 3. Connection (return to pool/close)
         * 
         * Reverse order ensures:
         * - No resource leaks
         * - Proper cleanup
         * - Connection available until last moment
         */
    }
    
    /**
     * Exception during resource creation
     */
    public static void exceptionDuringCreation() {
        System.out.println("\n=== Exception During Creation ===");
        
        class Resource implements AutoCloseable {
            private final String id;
            
            Resource(String id) throws IOException {
                this.id = id;
                System.out.println(id + " creating...");
                if (id.equals("R2")) {
                    throw new IOException("R2 creation failed");
                }
                System.out.println(id + " created");
            }
            
            @Override
            public void close() {
                System.out.println(id + " closed");
            }
        }
        
        try {
            try (Resource r1 = new Resource("R1");
                 Resource r2 = new Resource("R2");  // Fails here
                 Resource r3 = new Resource("R3")) {  // Never created
                
                System.out.println("Using resources");
            }
        } catch (IOException e) {
            System.out.println("Caught: " + e.getMessage());
        }
        
        /**
         * OUTPUT:
         * R1 creating...
         * R1 created
         * R2 creating...
         * R1 closed  ← Only R1 closed (successfully created)
         * Caught: R2 creation failed
         * 
         * BEHAVIOR:
         * - R1 created successfully → closed
         * - R2 failed → not closed (never fully created)
         * - R3 never attempted
         * 
         * Only successfully created resources are closed!
         */
    }
    
    /**
     * Exception during close
     */
    public static void exceptionDuringClose() {
        System.out.println("\n=== Exception During Close ===");
        
        class Resource implements AutoCloseable {
            private final String id;
            
            Resource(String id) {
                this.id = id;
                System.out.println(id + " created");
            }
            
            @Override
            public void close() throws IOException {
                System.out.println(id + " closing...");
                if (id.equals("R2")) {
                    throw new IOException("R2 close failed");
                }
                System.out.println(id + " closed");
            }
        }
        
        try {
            try (Resource r1 = new Resource("R1");
                 Resource r2 = new Resource("R2");
                 Resource r3 = new Resource("R3")) {
                
                System.out.println("Using resources");
            }
        } catch (IOException e) {
            System.out.println("Caught: " + e.getMessage());
            System.out.println("Suppressed: " + e.getSuppressed().length);
        }
        
        /**
         * OUTPUT:
         * R1 created
         * R2 created
         * R3 created
         * Using resources
         * R3 closing...
         * R3 closed
         * R2 closing...
         * R1 closing...
         * R1 closed
         * Caught: R2 close failed
         * Suppressed: 0
         * 
         * BEHAVIOR:
         * - R3 closed successfully
         * - R2 threw exception while closing
         * - R1 still closed (exception from R2 not stopping it)
         * 
         * ALL resources attempt to close!
         * Exceptions are collected as suppressed exceptions
         */
    }
    
    /**
     * Multiple exceptions during close
     */
    public static void multipleExceptionsDuringClose() {
        System.out.println("\n=== Multiple Exceptions ===");
        
        class FailingResource implements AutoCloseable {
            private final String id;
            
            FailingResource(String id) {
                this.id = id;
            }
            
            @Override
            public void close() throws IOException {
                throw new IOException(id + " close failed");
            }
        }
        
        try {
            try (FailingResource r1 = new FailingResource("R1");
                 FailingResource r2 = new FailingResource("R2");
                 FailingResource r3 = new FailingResource("R3")) {
                
                System.out.println("Using resources");
            }
        } catch (IOException e) {
            System.out.println("Primary exception: " + e.getMessage());
            
            Throwable[] suppressed = e.getSuppressed();
            System.out.println("Suppressed count: " + suppressed.length);
            for (int i = 0; i < suppressed.length; i++) {
                System.out.println("  Suppressed " + (i+1) + ": " + 
                    suppressed[i].getMessage());
            }
        }
        
        /**
         * OUTPUT:
         * Using resources
         * Primary exception: R3 close failed
         * Suppressed count: 2
         *   Suppressed 1: R2 close failed
         *   Suppressed 2: R1 close failed
         * 
         * EXCEPTION HANDLING:
         * - First exception (R3) becomes primary
         * - Later exceptions (R2, R1) added as suppressed
         * - All exceptions preserved!
         * - Can retrieve via getSuppressed()
         * 
         * This is the "suppressed exception" mechanism
         * Introduced with try-with-resources in Java 7
         */
    }
    
    /**
     * Order visualization
     */
    public static void orderVisualization() {
        /**
         * RESOURCE CLOSING ORDER VISUALIZATION:
         * 
         * Declaration:
         * ┌──────────┐
         * │ try (R1; │
         * │      R2; │
         * │      R3) │
         * └──────────┘
         *     ↓
         * Creation Order:
         * R1 → R2 → R3
         *     ↓
         * Usage:

         *        ↓

         * Close order:
         * dis → bis → fis

    }
```

## Best Practices and Patterns

### Effective Use of Try-with-resources

```java
/**
 * BEST PRACTICES
 * 
 * Effective patterns for try-with-resources
 */

import java.io.*;
import java.util.*;

public class BestPractices {
    
    /**
     * Pattern 1: Prefer try-with-resources over manual close
     */
    public static void preferTryWithResources() throws IOException {
        
        // ❌ BAD: Manual close
        BufferedReader reader = null;
        try {
            reader = new BufferedReader(new FileReader("data.txt"));
            String line = reader.readLine();
            System.out.println(line);
        } finally {
            if (reader != null) {
                reader.close();
            }
        }
        
        // ✅ GOOD: Try-with-resources
        try (BufferedReader reader = new BufferedReader(
                new FileReader("data.txt"))) {
            String line = reader.readLine();
            System.out.println(line);
        }
        
        /**
         * WHY TRY-WITH-RESOURCES:
         * ✓ Automatic resource management
         * ✓ Correct exception handling
         * ✓ Less code
         * ✓ Safer
         * ✓ No resource leaks
         */
    }
    
    /**
     * Pattern 2: Use Java 9 syntax when appropriate
     */
    public static void useJava9WhenAppropriate() throws IOException {
        
        // When resource comes from method/field/parameter
        Scanner scanner = createConfiguredScanner();
        try (scanner) {
            // Use scanner
        }
        
        // When resource needs setup
        Connection conn = getConnection();
        conn.setAutoCommit(false);
        try (conn) {
            // Use connection
        }
        
        /**
         * USE JAVA 9 SYNTAX WHEN:
         * ✓ Resource from factory method
         * ✓ Resource from field
         * ✓ Resource needs configuration
         * ✓ Conditional resource creation
         * ✓ Method parameter
         * 
         * STILL USE JAVA 7 WHEN:
         * ✓ Creating resource inline
         * ✓ Simple one-liner
         */
    }
    
    /**
     * Pattern 3: Order resources by dependency
     */
    public static void orderByDependency() throws IOException {
        
        // ✅ GOOD: Outer to inner
        try (FileInputStream fis = new FileInputStream("data.bin");
             BufferedInputStream bis = new BufferedInputStream(fis);
             DataInputStream dis = new DataInputStream(bis)) {
            
            // dis depends on bis, bis depends on fis
            int value = dis.readInt();
        }
        // Closes: dis → bis → fis (perfect!)
        
        // ❌ BAD: Don't reverse unnecessarily
        // (Java won't prevent this, but it's confusing)
    }
    
    /**
     * Pattern 4: One resource per variable
     */
    public static void oneResourcePerVariable() throws IOException {
        
        // ❌ BAD: Chaining in declaration
        try (DataInputStream dis = new DataInputStream(
                new BufferedInputStream(
                    new FileInputStream("data.bin")))) {
            // Only dis is managed!
            // BufferedInputStream and FileInputStream not closed separately
        }
        
        // ✅ GOOD: Separate variables
        try (FileInputStream fis = new FileInputStream("data.bin");
             BufferedInputStream bis = new BufferedInputStream(fis);
             DataInputStream dis = new DataInputStream(bis)) {
            // All three managed and closed properly
        }
        
        /**
         * WHY SEPARATE:
         * ✓ All resources explicitly managed
         * ✓ Clear closing order
         * ✓ Better for debugging
         * ✓ Each resource tracked
         */
    }
    
    /**
     * Pattern 5: Handle exceptions appropriately
     */
    public static void handleExceptions() {
        
        // Specific exceptions
        try (BufferedReader reader = new BufferedReader(
                new FileReader("data.txt"))) {
            
            String line = reader.readLine();
            processLine(line);
            
        } catch (FileNotFoundException e) {
            System.err.println("File not found: " + e.getMessage());
        } catch (IOException e) {
            System.err.println("I/O error: " + e.getMessage());
        }
        
        /**
         * EXCEPTION HANDLING:
         * ✓ Catch specific exceptions
         * ✓ Handle resource creation failures
         * ✓ Handle usage exceptions
         * ✓ Don't catch Exception in resource close
         *   (handled automatically)
         */
    }
    
    /**
     * Pattern 6: Use wrapper methods for complex setup
     */
    public static void wrapperMethodsForSetup() throws IOException {
        
        // Complex setup extracted to method
        try (BufferedReader reader = createConfiguredReader("data.txt")) {
            reader.lines().forEach(System.out::println);
        }
    }
    
    private static BufferedReader createConfiguredReader(String filename) 
            throws IOException {
        return new BufferedReader(
            new InputStreamReader(
                new FileInputStream(filename),
                "UTF-8"
            )
        );
    }
    
    /**
     * Pattern 7: Null-safe try-with-resources
     */
    public static void nullSafeTryWithResources(String filename) 
            throws IOException {
        
        // Java 7: Null check needed
        BufferedReader reader = filename != null 
            ? new BufferedReader(new FileReader(filename))
            : null;
        
        if (reader != null) {
            try (reader) {
                // Use reader
            }
        }
        
        // Better: Use Optional or validate earlier
        Objects.requireNonNull(filename, "Filename cannot be null");
        try (BufferedReader reader = new BufferedReader(
                new FileReader(filename))) {
            // Use reader
        }
    }
    
    /**
     * Common pitfalls
     */
    public static void commonPitfalls() {
        /**
         * PITFALL 1: Using resource after try block
         * 
         * BufferedReader reader;
         * try (reader = new BufferedReader(...)) {
         *   // Use reader
         * }
         * reader.readLine();  // ❌ Resource already closed!
         * 
         * 
         * PITFALL 2: Reassigning resource variable
         * 
         * BufferedReader reader = new BufferedReader(...);
         * reader = new BufferedReader(...);  // ❌ Not effectively final
         * try (reader) {  // Compilation error
         *   ...
         * }
         * 
         * 
         * PITFALL 3: Closing resource manually
         * 
         * try (BufferedReader reader = ...) {
         *   reader.close();  // ❌ Don't do this!
         *   // Will be closed again automatically
         * }
         * 
         * 
         * PITFALL 4: Forgetting to declare AutoCloseable
         * 
         * class MyResource {  // ❌ Not AutoCloseable
         *   void close() { ... }
         * }
         * 
         * try (MyResource r = new MyResource()) {  // Won't compile
         *   ...
         * }
         * 
         * Fix: Implement AutoCloseable
         * class MyResource implements AutoCloseable {
         *   public void close() { ... }
         * }
         * 
         * 
         * PITFALL 5: Swallowing close exceptions
         * 
         * Don't suppress exceptions in custom close():
         * 
         * ❌ BAD:
         * public void close() {
         *   try {
         *     // cleanup
         *   } catch (Exception e) {
         *     // Swallow - BAD!
         *   }
         * }
         * 
         * ✅ GOOD:
         * public void close() throws IOException {
         *   // cleanup - let exceptions propagate
         * }
         */
    }
    
    // Fake methods/classes
    static Scanner createConfiguredScanner() {
        return new Scanner(System.in);
    }
    
    interface Connection extends AutoCloseable {
        void setAutoCommit(boolean autoCommit);
    }
    
    static Connection getConnection() {
        return null;
    }
    
    static void processLine(String line) {}
}
```

## Summary

### Quick Reference

**Java 7 Try-with-resources:**

```java
// Must declare in try statement
try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"))) {
    String line = reader.readLine();
}
// reader.close() called automatically
```

**Java 9 Enhancement:**

```java
// Can use existing effectively final variable
BufferedReader reader = new BufferedReader(new FileReader("file.txt"));
try (reader) {
    String line = reader.readLine();
}
// reader.close() called automatically
```

**Multiple Resources:**

```java
try (BufferedReader reader = new BufferedReader(new FileReader("in.txt"));
     BufferedWriter writer = new BufferedWriter(new FileWriter("out.txt"))) {
    
    String line;
    while ((line = reader.readLine()) != null) {
        writer.write(line);
    }
}
// Both closed automatically (writer first, then reader)
```

**Resource Closing Order:**

```
Declaration order: R1, R2, R3
Closing order:     R3, R2, R1 (REVERSE - LIFO)

Why: Later resources may depend on earlier ones
```

**Key Rules:**

```
Effectively Final:
✓ Variable assigned once
✗ Never reassigned
→ Required for Java 9 syntax

Closing Order:
✓ Reverse of declaration (LIFO)
✓ Last declared, first closed
✓ Handles dependencies correctly

Exception Handling:
✓ All resources attempt to close
✓ First exception becomes primary
✓ Later exceptions become suppressed
✓ Access via getSuppressed()

AutoCloseable:
✓ Must implement AutoCloseable
✓ close() can throw Exception
✓ close() should be idempotent
```

**Evolution Timeline:**

```
Pre-Java 7:
try {
    Resource r = new Resource();
    // use r
} finally {
    if (r != null) {
        try {
            r.close();
        } catch (Exception e) { }
    }
}

Java 7 (2011):
try (Resource r = new Resource()) {
    // use r
}

Java 9 (2017):
Resource r = new Resource();
try (r) {
    // use r
}
```

---

**Key insights:**

- **Java 9 enhancement**: Can use existing effectively final variables
- **Multiple resources**: Automatically closed in reverse order (LIFO)
- **Exception handling**: All resources close even if exceptions occur
- **Suppressed exceptions**: Later exceptions added to first exception
- **No resource leaks**: Automatic cleanup guaranteed

**The evolution:**

```
Before: Manual close (error-prone, verbose)
Java 7: try-with-resources (automatic, safe)
Java 9: Existing variables (cleaner, flexible)

From 15 lines → 5 lines → 3 lines
```

**Closing order visualization:**

```
try (R1; R2; R3) { ... }

Creation: R1 → R2 → R3
Closing:  R3 → R2 → R1  (REVERSE!)

Why: R3 may depend on R2, R2 may depend on R1
```
